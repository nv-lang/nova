# Plan 33.2: Контракты — Imperative (frame + loops + termination + composition + invariant)

> Часть [Plan 33](33-contracts-implementation.md). Превращает [33.1](33.1-contracts-core.md)
> из «контракты на straight-line код» в инструмент, способный
> верифицировать реальный imperative-код (циклы, mut-структуры,
> рекурсия, composition).

## Scope

**Включено:**
- `reads <exprs>` / `modifies <exprs>` — Dafny-style frame conditions.
- Loop invariants: `invariant <expr>` + `decreases <expr>` после
  `for`/`while`/`loop` до `{`.
- `decreases <expr>` для функций — termination для рекурсии.
- Composition: вызов `#pure` функций в контрактах с использованием
  их `ensures` как axioms (требует termination для рекурсивных).
- Record `invariant`: `invariant <expr>` в `type` декларации,
  автоматически добавляется как pre+post для всех методов на типе.
- `assert_static <expr>` — intermediate proof obligation.
- Расширение кодировки SMT: arrays `[]T` через `(Seq T)` / `(Array Int T)`
  для immut/mut.

**НЕ включено (это в 33.3):**
- `pure_view` + axiom + `#verify_handler`.
- `ghost` state.
- `assume`.
- Bounded quantifiers (`forall` / `exists`).
- FP IEEE 754.
- String reasoning beyond length + equality.
- Sets / Maps теории.
- Incremental cache + parallel verification.
- Cross-check Z3↔CVC5.
- `#must_verify_module`.
- `#trusted` external fn с контрактами.

## Зависит от

[33.1](33.1-contracts-core.md): `SmtBackend`, `ContractCtx`, runtime

## Фазы 33.2

### Ф.5 — Frame conditions (`reads` / `modifies`)

**Файлы:** AST, parser, types, verify/encode.

**AST расширение:**
```rust
pub struct FnDecl {
    // ... 33.1 + ...
    pub reads: Vec<FrameTarget>,
    pub modifies: Vec<FrameTarget>,
}

pub enum FrameTarget {
    Whole(Expr),                    // modifies acc
    Field { receiver: Expr, field: String },  // modifies acc.balance
    ArrayElem { array: Expr, index: Expr },   // modifies xs[i]
    ArrayAll { array: Expr },                  // modifies xs[*]
}
```

**Парсер.** После сигнатуры до `requires` принимает:
```
reads <expr>{, <expr>}*
modifies <expr>{, <expr>}*
```

**Type-check.**
- `reads`/`modifies` targets обязаны быть l-values или receiver references.
- В теле fn: любое присваивание к l-value, не покрытому `modifies`,
  — compile error «assignment outside frame: `r.field` not in modifies».
- Чтение handler-state в `#pure` fn без `reads` — error (заготовка
  под `pure_view` в 33.3, в 33.2 — просто error без alternative).
- При вызове `g(...)` внутри fn: `g.modifies ⊆ this.modifies` через
  substitution параметров.

**SMT encoding.** Frame axiom: после выполнения тела, всё что вне
`modifies` равно `old(значения)`. Реализация:
- Перед body: snapshot значений всех l-values mentioned outside
  `modifies` (используемых в ensures).
- После body: assert `forall l ∉ modifies. l == snapshot(l)`.

**Negative tests:**
- `modifies expr` где `expr` не l-value — error.
- Assignment outside frame — error.
- `g.modifies ⊃ this.modifies` — error.

**Acceptance:** bank-account `withdraw` с `modifies self.balance`
доказывается; пример без `modifies` декларации, но мутирующий self,
— compile error.

### Ф.6 — Loop invariants + loop decreases

**Файлы:** AST, parser, types, codegen, verify.

**AST:**
```rust
pub struct LoopBlock {
    // ... existing body fields ...
    pub invariants: Vec<Contract>,
    pub decreases: Option<Expr>,
}
```

Применяется к `for` / `while` / `loop`.

**Парсер.** После заголовка цикла до `{`:
```
while <cond>
    invariant <expr>
    invariant <expr>
    decreases <expr>
{ ... }
```

**Type-check (LoopInvariant kind в ContractCtx):**
- `result` запрещён.
- `old(...)` относится к состоянию **до entry-loop** (не до fn).
- `decreases` обязан быть `int`/`nat` (`>= 0`) или lex-tuple.
- Type должен быть `bool` для invariant, well-foundable для decreases.

**SMT encoding (стандартный Hoare):**
1. Проверить: `pre ==> Inv` (invariant выполняется на входе).
2. Havoc all modified vars в теле цикла.
3. Assume `Inv ∧ cond`.
4. Symbolically выполнить body.
5. Проверить: post-body `Inv` (preservation) + `decreases_new < decreases_old`.
6. После цикла: assume `Inv ∧ ¬cond`.

**Codegen runtime-check (debug):**
- Перед entry: `if (!Inv) violation(NOVA_LOOP_PRE)`.
- В конце каждой итерации: `if (!Inv) violation(NOVA_LOOP_POST)` +
  `if (!(decreases_new < decreases_old)) violation(NOVA_LOOP_DECREASES)`.

Modified vars для havoc'а собираются flow-аналитикой (уже есть
infrastructure в Plan 16 capability-checks).

**`NovaContractKind` enum расширяется:**
```c
NOVA_LOOP_PRE, NOVA_LOOP_POST, NOVA_LOOP_DECREASES, NOVA_FRAME
```

**Acceptance:** binary search с loop invariants:
```nova
fn binary_search(xs []int, target int) -> int
    requires sorted(xs)
    ensures result == -1 || xs[result] == target
=> {
    let mut lo = 0
    let mut hi = xs.len()
    while lo < hi
        invariant 0 <= lo <= hi <= xs.len()
        invariant forall_pred  // в 33.3, в 33.2 — упрощённый invariant
        decreases hi - lo
    { ... }
}
```
(Полный binary search с quantifiers — после 33.3.)

### Ф.7 — Termination + composition + record invariant

**Termination для функций.**

**AST:**
```rust
pub struct FnDecl {
    // ... 33.1+33.2(F5,F6) + ...
    pub decreases: Option<Expr>,
}
```

**Парсер.** `decreases <expr>` после `ensures` (или в любом месте в
блоке контрактов после сигнатуры).

**SMT:**
- Для каждого call-site `f(args')` в теле рекурсивной `f(args)`:
  push; assert `decreases(args') < decreases(args)`; check_sat; pop.
- Failure → error «decreases not proven for recursive call».
- Рекурсивная fn БЕЗ `decreases`:
  - `#must_verify` → compile error.
  - Default → ensures на этой fn = runtime-only, warning
    «recursive fn without decreases; SMT cannot verify, runtime only».
  - `#unverified` → принимается.

**Codegen runtime decreases-check (debug):**
- Snapshot decreases-value до recursive call.
- После возврата: проверить `m_new < m_old`. Failure → `NOVA_DECREASES`.

**Composition.**

Вызов `g(...)` в контракте теперь **разрешён**, если:
1. `g` — `#pure` (выведенная или объявленная).
2. Если `g` рекурсивна — у неё есть `decreases`.

Иначе compile error с указанием конкретного нарушения.

**SMT encoding:** `g.ensures(args, result)` становится axiom:
```
(declare-fun g (X) Y)
(assert (forall ((x X)) Q_g(x, g(x))))
```

Где `Q_g` — encoded ensures `g`.

Sanity check: после загрузки всех `g.ensures` axioms — `check_sat`.
Если unsat — `g.ensures` противоречивы, error.

**Record `invariant`.**

**AST:**
```rust
pub struct TypeDecl {
    // ... existing ...
    pub invariants: Vec<Invariant>,
}

pub struct Invariant {
    pub label: Option<String>,
    pub expr: Expr,
    pub span: Span,
}
```

**Парсер.** В type-декларации после fields:
```nova
type BankAccount {
    balance money
}
    invariant balance >= 0
```

**Type-check.** Invariant — это `bool`-expr в `ContractCtx::Invariant`.
В скоупе видны fields record'а напрямую (как `self.field` или без префикса).

**SMT semantics.**
- Конструктор `BankAccount { balance: x }` — proof obligation `Inv(self)`.
- Каждая fn с `modifies self.field` или `mut self` receiver —
  автоматически получает `requires Inv(self)` и `ensures Inv(self)`.

**Codegen runtime-check (debug):**
- После record-литерала: check.
- После field-assignment (если record имеет invariant): check.
- На выходе `mut self` method: check.

**Acceptance:**
- `fn fib(n int) -> int decreases n` доказывается.
- `fn loop_forever() decreases 0` (recursive без уменьшения) — error.
- Composition: `ensures is_valid(result)` через `#pure fn is_valid(...) -> bool` работает.
- `type BankAccount { balance } invariant balance >= 0` отвергает функцию,
  оставляющую balance < 0.

### Ф.8 — Расширение кодировки SMT для arrays + `assert_static`

**Arrays `[]T` (immut + mut).**

| Nova | SMT | Theory |
|---|---|---|
| `[]T` immut | `(Seq T_sort)` | Strings/Seq |
| `[]T` mut | `(Array Int T_sort)` | Arrays |
| `xs.len()` | UF `len(xs)` + axiom `len(xs) >= 0` | UF |
| `xs[i]` | `(seq.nth xs i)` или `(select xs i)` | Seq/Arrays |
| `xs.push(x)` | для immut — `(seq.++ xs (seq.unit x))`; mut — store + len update | Seq/Arrays |

Axioms в `axiomatize.rs`:
- `len(empty) == 0`.
- `len(push(xs, x)) == len(xs) + 1`.
- `0 <= i < len(xs) ==> get(push(xs, x), len(xs)) == x`.
- Соответствующие для concat/slice (минимум).

Bounded quantifiers — НЕ в 33.2 (в 33.3). Поэтому ensures про массивы
ограничены length + index-specific cases без `forall`.

**`assert_static`.**

**Парсер.** Statement-level `assert_static <bool-expr>`.

**Type-check.** Тип `bool`, обязан быть `#pure`.

**SMT:** proof obligation в точке вставки.
- Push; assert path-condition; assert `¬expr`; check_sat.
- Unsat → доказано, в release стирается.
- Sat → error с counterexample.
- Unknown → fallback runtime в debug.

**Codegen debug:** `if (!expr) violation(NOVA_ASSERT_STATIC)`.

**Acceptance:**
- `fn first(xs []int) -> int requires xs.len() > 0 ensures result == xs[0] => xs[0]`
  доказывается.
- `assert_static` с заведомо ложным expr — compile error со счётчик-примером.
