# Plan 104.4 — Document/Workspace Symbols + Find-References

> **Статус:** ✅ ЗАКРЫТ 2026-06-16

> **Зависимости:** Plan 104.0 ✅, Plan 104.1 ✅

## Scope

Реализует три LSP методa:

1. **`textDocument/documentSymbol`** — outline для VSCode: иерархический список символов в файле (functions/types/tests/methods nested). DocumentSymbol + children.
2. **`workspace/symbol`** — Ctrl+T поиск по проекту: substring + fuzzy matching; pagination (top 100).
3. **`textDocument/references`** — Shift+F12: find all usages символа; `includeDeclaration` option; cross-file scan.

## Architecture

### DocumentSymbol (outline)

- Parser API: `nova_codegen::parser::parse(src)` → `Module` → items walk.
- AST `Item` enum: `Fn | Type | Let | Const | Test | Bench | Lemma`.
- `FnDecl.receiver` — методы имеют receiver (показываются nested под type).
- `TypeDecl.kind` → `Record(fields)`, `Sum(variants)`, `Protocol(methods)`, etc.
- Outline hierarchy:
  - Top-level: `Fn`, `Type`, `Const`, `Let`, `Test`
  - Nested под `Type`: methods (`FnDecl` с matching receiver) + fields (из `TypeDecl.kind`)
- Cache per-URI: `DashMap<Url, Arc<Vec<DocumentSymbol>>>` — invalidate при `didChange`/`didOpen`.

### WorkspaceSymbol

- Index per-workspace: `DashMap<Url, Vec<WorkspaceSymbolEntry>>` — обновляется при `didOpen/didChange`.
- `WorkspaceSymbolEntry { name: String, kind: SymbolKind, uri: Url, range: Range }`.
- Search: substring match (case-insensitive) + fuzzy fallback; top 100 results.
- Индексирование: при `initialized` → scan workspace root; при `didChange` → incremental re-index для изменённого файла.

### References (find-all-usages)

- V1: regex-scan по тексту файлов (`\bNAME\b`).
- Workspace scan: все `.nv` файлы из `check_workspace` pattern.
- `includeDeclaration` опция (LSP standard).
- Performance: full-scan V1 (acceptable for Nova workspaces, typically <50 files).
- Cache: refs scan запускается lazy per-request, не кешируется (V2: incremental index).

## Performance budgets

- `documentSymbol`: ≤50ms (parse only, no typecheck)
- `workspaceSymbol`: ≤500ms (index lookup + substring match)
- `references`: ≤1s (workspace file scan + regex)

## Tests (≥18 + edge) — план

### documentSymbol (6 pos + 2 neg + 2 edge)

- pos1: функции → `SymbolKind::Function`, правильные spans
- pos2: type → `SymbolKind::Class`, nested fields
- pos3: методы вложены под type (children)
- pos4: `test` → `SymbolKind::Event`
- pos5: mixed outline (fn + type + test + const)
- pos6: generic function name без `[T]` суффикса
- neg1: пустой файл → пустой Vec (no crash)
- neg2: файл с parse-ошибкой → empty Vec (graceful)
- edge1: nested types (type в sum-variant), protocol methods
- edge2: file с только const/let — только Variable symbols

### workspaceSymbol (4 pos + 2 neg)

- pos1: поиск по точному имени → находит
- pos2: поиск подстрокой → matches
- pos3: поиск пустой строкой → top 100
- pos4: cross-file поиск → returns correct URI
- neg1: несуществующее имя → empty list
- neg2: query слишком длинный (>1000 chars) → graceful empty

### references (6 pos + 2 neg + 1 edge)

- pos1: найти usages function в том же файле
- pos2: `includeDeclaration=true` — включает declaration span
- pos3: `includeDeclaration=false` — не включает declaration
- pos4: type usages (variable type annotations)
- pos5: multi-file references
- pos6: references к type в import statement
- neg1: символ не существует → empty list
- neg2: позиция в whitespace → empty list
- edge1: символ с именем substr другого символа — не false-positives (word boundary)

## V1 Simplifications (documented)

- WorkspaceSymbol index rebuilt per-file на didChange, НЕ incremental dep-graph.
- References scan: full filesystem scan per-request (V2: incremental index).
- documentSymbol: no method-under-type nesting via cross-file analysis (нет type-checker); nesting via receiver-name matching в том же файле.
- Generic params в имени типа не отображаются (symbols показывают только `name`).
