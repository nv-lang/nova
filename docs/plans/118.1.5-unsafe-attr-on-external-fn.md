# Plan 118.1.5 — `#unsafe` attribute on `external fn`

> **Создан 2026-06-06.** Production-grade plan-doc.

> **Эстимат:** ~½ dev-day.
> **Model:** Opus 4.7.

## Что

Расширение parser whitelist + type-checker для разрешения `#unsafe` attribute
на `export external fn` declarations. Эквивалент D2 `#unsafe fn` для Nova-side
функций, но для FFI-bindings — обозначает что вызов FFI fn requires `unsafe {}`
context (E_UNSAFE_CALL_REQUIRES_WRAP).

## Почему

1. **`str.@as_cstr_unchecked()` логически unsafe** (skip embedded-NUL scan) — caller
   asserts safety. V1 ships без #unsafe attr — нарушение safety contract.
2. **FFI bindings часто wrap C-side unsafe operations** — типичный pattern
   `external fn dlopen(...)` / `external fn memcpy(...)` нужно метить как unsafe.
3. **Parser сейчас отвергает `#unsafe export external fn ...`** — error: «contract
   attributes (#verify/#unverified/#verify_timeout/#pure/#trusted) only valid before
   fn or external fn». `#unsafe` НЕ в этом whitelist.
