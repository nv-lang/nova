## Фазы

### Ф.6 — MultiError + Error types (D193)

- **Ф.6.1** Удалить `ErrorKind` enum (D158 amend).
- **Ф.6.2** Refactor `MultiError { primary, suppressed }` payload —
  bootstrap: continue `str`; full `any` → `[M-110-multierror-any]` follow-up.
- **Ф.6.3** API: `@walk() -> Iter[Error]`, `@find_first_panic() -> Option[str]`.
- **Ф.6.4** Cycle-safety: identity-check в `nv_compose_error`.
- **Ф.6.5** Depth-limit 256 + truncation entry.
- **Ф.6.6** Concrete error types в prelude — **already landed** в Plan 110
  Ф.0 close (CancelError, CleanupTimeoutError, MultiErrorTruncated в
  `std/prelude/errors.nv`, commit referenced in Plan 110 status).
- **Ф.6.7** Tests T6.1–T6.6.

### Ф.7 — `Cleanup` effect (telemetry only)

- **Ф.7.1** Spec D185 already landed в Plan 110 Ф.0 (commit 044bc06cc24).
- **Ф.7.2** Operations declaration в `std/prelude/effects.nv`.
- **Ф.7.3** Default handler — no-op (zero-overhead).
- **Ф.7.4** Handler throw запрещён (`D185-cleanup-handler-throw`).
- **Ф.7.5** OpenTelemetry wire format implementation.
- **Ф.7.6** Example handler `examples/cleanup_tracing.nv`.
- **Ф.7.7** Tests T7.1–T7.5.

### Ф.8 — `Application` effect + finalizers

- **Ф.8.1** Stdlib `Application` effect (см. D195 spec) — `effect Application
  { register_finalizer, default_exit_timeout }` + `ApplicationHandler` тип.
- **Ф.8.2** Topo-order: registry preserves registration; reverse = topo.
- **Ф.8.3** Idiomatic main pattern через `with Application = ...`.
- **Ф.8.4** Integration с Plan 110.2 Ф.3.5 — codegen `nv_resolve_exit_timeout`
  Level-2 ищет Application handler.
- **Ф.8.5** Nested Application (D195 R1-R4).
- **Ф.8.6** Cross-fiber propagation (D195 R6).
- **Ф.8.7** Документация: finalizers NOT fired на abort/SIGKILL.
- **Ф.8.8** Cross-ref Plan 100.4.1 handler cleanup mechanism reuse.
- **Ф.8.9** Optional: `#[run_on_abort]` атрибут → `[M-110-run-on-abort]`.
- **Ф.8.10** Tests T8.1–T8.8.
