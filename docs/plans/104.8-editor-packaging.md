## Цель

Дать пользователю одну команду установки per editor — и **получить рабочий
setup**: открыл `.nv`-файл → подсветка + ошибки компилятора в редакторе.

**Production criteria** (acceptance):
1. **VSCode/Cursor/VSCodium**: clone repo + `npm install && npm run build` →
   F5 (debug) → открыть .nv-файл → диагностики появляются. Команда
   ≤500 ms (debounced). Server auto-restart при крэше.
2. **Neovim**: `nvim-lspconfig` snippet в репе + 1-shot копи-инструкция →
   `:LspInfo` показывает Nova attached → диагностики в `:lua vim.diagnostic.get()`.
3. **Helix**: добавление 3-block-fragment в `~/.config/helix/languages.toml` →
   `hx file.nv` → diagnostics в gutter. Tree-sitter highlighting от 104.7
   работает совместно с LSP (parallel, не конфликт).
4. **Zed**: extension TOMLs + manifest → side-load → `.nv`-файл показывает
   diagnostics. Marketplace submission — отдельно (manual review process).
5. **Common docs**: `editors/INSTALL.md` — single source of truth для всех
   4 editors.

### LSP binary discovery

Каждый editor-client должен найти `nova-lsp.exe`. Strategy (по приоритету):
1. **User setting**: `nova.lsp.path` (explicit override).
2. **PATH lookup**: `which nova-lsp` / `where.exe nova-lsp`.
3. **Workspace-relative**: `target/release/nova-lsp.exe` (development).
4. **Bundled**: будущее — embed binary in extension (V2; пока external).

Если не найден — graceful error message с инструкцией.

## Production constraints (нельзя сокращать)

1. **Auto-restart** on LSP crash (LanguageClient default behaviour, но явно
   опытом включить и протестировать).
2. **Configurable binary path** в каждом editor — пользователь должен иметь
   возможность override.
3. **Graceful error** если nova-lsp не найден — не падать, показать message.
4. **Workspace-aware root detection** (`nova.toml`, `.git`) — не привязка
   к cwd.
5. **Auto-pairs включают `'` и `` ` ``** в Helix и Zed (Nova char-literals
   и tagged template literals — discovered 2026-05-26).
6. **TextMate grammar coexists with tree-sitter** — VSCode использует TextMate
   (existing), Helix/Zed/Neovim 0.5+ — tree-sitter. Не конфликтуют.
7. **No bundled binary in V1** — экстеншены ищут nova-lsp в PATH/setting.
   Bundled — V2 (требует release pipeline build).

## Risks

| Risk | Mitigation |
|---|---|
| nova-lsp binary discovery fails in PATH | Конфигурируемый `nova.lsp.path` setting + graceful error |
| LSP version mismatch (server vs client) | Server `serverInfo.version`, проверять при initialize, warn |
| TextMate vs tree-sitter highlighting conflict в VSCode | Не должно — VSCode не использует tree-sitter, остаётся TextMate-only там |
| Editor API instability (Zed extension API меняется) | Pin `schema_version = 1` + README с warning |
| Headless testing невозможен для Zed | Manual smoke + documented procedure |
