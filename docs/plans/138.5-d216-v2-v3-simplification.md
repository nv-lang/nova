# Plan 138.5 — D216 V2/V3 simplification (pointer model: pointee-mut only, no prefix modifiers)

> **Scope:** не просто clarification — **упрощение D216 V2/V3**: retire prefix-модификаторов
> (`ro */mut */unsafe *`), retire `safe`-стоппера и `Unsafe(Pointer)`, схлопывание §V3.3/§V3.4
> propagation-машинерии. Pointer = pointee-mut постфикс; reassignability = binding; nullable = Option[*T].
> `unsafe T` value-wrapper (§V2.3) СОХРАНЯЕТСЯ (maybe-uninit значение, ортогонально указателям).

> **Создан:** 2026-06-11.  **Статус:** 📋 PLANNED (spec + docs + small parser/checker enforcement).
> **Эстимат:** ~0.5-1 dev-day.  **Model:** Opus + Thinking (точная spec-формулировка + parser).
> **Триггер:** ревью сессии 138.x — путаница «двух mut» у указателей (outer pointer-mut в
> типе vs pointee-mut), особенно в return-позиции (D184×D216). Подтверждено пользователем.
> **Gate:** после Plan 138.4 (G-D трогает D216 right-binding/pointee — строить поверх его исхода).

---

## Решение (подтверждено 2026-06-11)

**Указательный ТИП несёт ТОЛЬКО pointee-мутабельность:**
- `*mut T` — writable target; `*ro T` ≡ `*T` — read-only target.
- **Перепривязываемость самого указателя** (можно ли `p = other_ptr`) — это **binding**
  (`let` = фиксирован, `mut` = можно переприсвоить, D36), **НЕ часть типа**.
- **Нет** type-level «outer» `mut *` / `ro *` для самого указателя.

В `mut p *mut T`: первый `mut` = **binding** (p reassignable), `*mut T` = **pointee** (writable).
Две роли, чисто разделены. Прецедент: Rust (`*mut T`/`*const T` = pointee; `let mut p` = reassign).

**Почему:** реассайнабельность уже даёт `let`/`mut` для любой переменной → дублировать её в типе
незачем; это и порождало «два mut» + мутный outer-mut в возврате.

## D-блоки к обновлению (читать однозначно)

### D216 §V2.2 / §V2.6 — AMEND (ядро)
- **Retract** трактовку «`mut * T` = `Mut(Pointer(T))` (outer pointer-mut как type-wrapper)».
- Зафиксировать: pointer-тип = цепочка pointee-мутабельностей (`*mut`/`*ro` на каждом `*`-уровне).
  Ведущий `mut`/`ro` ПЕРЕД именем в binding = binding-модификатор (reassignability), не type-wrapper.
- Сохранить §V3.3/§V3.4: pointee-propagation внутри типа (`mut * mut T` inner = E_REDUNDANT) — это
  про **pointee**-уровни, остаётся. RECONCILE: убрать только конфликтующую «outer-pointer-mut-в-типе»
  интерпретацию, не ломая pointee-propagation + `safe`-stopper.

### D216 §1 — CLARIFY + ENFORCE
- `*T ≡ *ro T` (read-only target) — оставить. Добавить явно: это про **pointee**; reassignability — binding.
- **ЗАПРЕТ ВСЕХ prefix-модификаторов перед `*` в типе** — `mut * T` / `ro * T` / **`unsafe * T`** (REVISED 2026-06-11
  после исследования: `unsafe *` тоже запрещаем, см. ниже). Error `E_POINTER_PREFIX_MODIFIER` (расширить
  `E_INVALID_POINTER_MODIFIER`, D216 §1 commit 6d6a18a2ab7). Сообщение: «модификаторы указателя — на pointee
  (после `*`: `*mut T`/`*ro T`/`*unsafe T`) или на binding (`mut x *T`); перед `*` не допускаются».
  Валидно: `*mut T`/`*ro T`/`*unsafe T`/`*T` (pointee, postfix), `mut name *T` (binding). **Никаких prefix.**
- **RETIRE `Unsafe(Pointer(...))`** (форма `unsafe * T` = unsafe/possibly-null POINTER). Обоснование (исследование 2026-06-11):
  (1) usage в std = **0**; (2) коллизия с `Option[*T]` — две формы для «nullable mut ptr»: safe `Option[*mut T]`
  (NPO, 8B, null=None) vs `unsafe * mut T` (raw nullable); (3) `Option[unsafe * T]` теряет NPO → 16B footgun
  ([02-types.md:8169-8173](../../spec/decisions/02-types.md)). Замена:
  - **Nullable — ТОЛЬКО `Option[*T]`** (NPO, канон). Указатели `*T`/`*mut T`/`*ro T` — всегда **non-null**.
  - **FFI nullable-uninit (C `T*`)** — `Option[*unsafe T]` (NPO: None=null, Some=non-null ptr к uninit; validity асертится на deref).
  - dangling — assert на deref (unsafe block), не type-level.
  Ретрактит V2 §V2.2 `unsafe * T`-форму + migration note (8228-8229). Usage=0 → миграция почти пустая.
- Parser/checker: reject `ro`/`mut`/`unsafe` token непосредственно перед `*` в type-position. Аудит +
  миграция существующих `mut */ro */unsafe *` → postfix `*mod T` / `Option[*T]` / binding-`mut`.

### D184 (Return mut default) — AMEND для указателей
- Для pointer-returns: return-mut-default ставит **pointee** mut → `-> *T` (return) = writable target;
  `-> *ro T` = ro target. Reassignability результата — bind-site (`let`/`mut`), не тип. Снимает
  «мутный outer-mut в возврате» (см. сессионный анализ D184×D216).

### D36 (binding readonly default) — CROSS-REF
- Добавить: перепривязываемость указательной переменной регулируется binding'ом (`let`/`mut`),
  как у любой переменной — НЕ типом указателя.

### D216 §V3.4 — RETIRE `safe` модификатор + §V3.3 propagation схлопывается
- `safe` (§V3.4) — propagation-stopper для outer-`unsafe` (только в `unsafe * safe T`). usage std = 0.
  Запрет `unsafe *` (prefix) → нет outer-unsafe-пропагации → `safe` **бесполезен** → **RETIRE**. Standalone `safe T` ≡ `T` и так no-op.
- §V3.3 (right-binding propagation) + §V3.4 (safe-stopper) строились вокруг prefix-модификаторной пропагации.
  Без prefix-модификаторов **пропагировать нечего** → §V3.3/§V3.4 **largely retired** (помечать как superseded).
  Остаётся: pointee-модификаторы постфикс (`*mut/ro/unsafe T`), §V3.1 (ro+mut adjacency на value-T) — проверить, нужен ли.
- `unsafe T` value-wrapper (§V2.3, MaybeUninit-style, `mut x unsafe T`) — **ОСТАЁТСЯ** (про maybe-uninit значение, не про указатель; ортогонально prefix-запрету).

### D216 §V3.2 — FLIP ordering на safety-INNER (`ro unsafe T`)
- Текущее §V3.2 «safety-outer / mutability-inner» (`unsafe ro T`) → **флип на `ro unsafe T`** (mutability-outer,
  unsafe вплотную к базе) для консистентности с `external unsafe fn` (unsafe вплотную к `fn`). Семантически
  свободно (ортогональные оси: `Unsafe(Readonly(T))` ≡ `Readonly(Unsafe(T))`). Reconcile §V3.3 propagation
  с новым порядком. Касается только pointee (`*ro unsafe T`) после forbid-prefix — редкий случай.

### D32 / D2 — CHECK cross-refs
- Если упоминают pointer-mut — выровнять формулировку (pointee-mut в типе).

### Миграция (Ф.2) — НЕ пустая
- `std/runtime/raw_mem.nv` использует prefix `mut * u8` (`dst mut * u8`, ~7 RawMem externs, полагается на
  V2 right-binding mut→pointee). Мигрировать → `*mut u8` (явный pointee-mut). Это основной call-site forbid-prefix.

### docs/typed-pointers.md — REWRITE раздела про mut
- User-facing: «стрелка на коробку» — `*mut T`/`*ro T` = что можно делать с target (в типе);
  `let`/`mut` = можно ли перенаправить стрелку (binding). Примеры: `mut p *mut T`, `let q *ro T`,
  return `-> *T` (writable target by D184), и т.д. Один однозначный набор примеров.
