# Plan 91.8b — Operator dispatch через protocols (D184)

> **Подплан Plan 91** ([91-stdlib-mvp-for-0.1.md](91-stdlib-mvp-for-0.1.md)).
> Статус: 🔴 OPEN.
> Зависимости: **Plan 91.8a** должен быть закрыт (Equatable.equals + Comparable.compare должны существовать).

## Мотивация

Сейчас Nova имеет **два пути** для сравнения (D9 violation):

1. **Magic operator methods** (D46): `@eq`, `@lt`, `@le`, `@gt`, `@ge` дисп

атчат `==`, `<`, `<=`, `>`, `>=`.
2. **Protocols** (D109/D183): `Equatable.equals`, `Comparable.compare`.

Каждый type-author реализует и то и то с boilerplate ×6. Например `Duration` (`std/time/duration.nv:332-348`):

```nova
fn Duration @eq(other) => @nanos == other.nanos
fn Duration @lt(other) => @nanos < other.nanos
fn Duration @le(other) => @nanos <= other.nanos
fn Duration @gt(other) => @nanos > other.nanos
fn Duration @ge(other) => @nanos >= other.nanos
```

После Plan 91.8a — должно быть достаточно implement-only `Comparable.compare`:

```nova
fn Duration @compare(other) -> int => @nanos - other.nanos  // (с overflow-safe вариантом)
```

И тогда `<`, `>`, `==` etc. работают **автоматически** через protocol.

## Scope

### Operator dispatch правила (D184)

Изменить `emit_c.rs` BinOp dispatch:

| Operator | Раньше | Сейчас (D184) |
|---|---|---|
| `a == b` | `@eq(a, b)` | `@equals(a, b)` |
| `a != b` | `!@eq(a, b)` | `!@equals(a, b)` |
| `a < b` | `@lt(a, b)` | `@compare(a, b) < 0` |
| `a <= b` | `@le(a, b)` | `@compare(a, b) <= 0` |
| `a > b` | `@compare(a, b) > 0` | `@compare(a, b) > 0` |
| `a >= b` | `@ge(a, b)` | `@compare(a, b) >= 0` |

**Для primitives** (`int`, `bool`, etc.) — операторы остаются built-in (без вызова метода), C-эмиссия как сейчас.

**Для user types** — dispatch в `@equals` / `@compare` (после 91.8a renames).

### Удаление magic methods

Из `std/time/duration.nv`:
- `Duration @eq/@lt/@le/@gt/@ge` — удалить, заменить на `Duration @compare(other) -> int`
- `Timestamp @eq` — заменить на `@compare`
- `Monotonic @eq` — заменить на `@compare`

Из `std/runtime/string.nv`:
- `str @eq(other str)` external — переименовать `Nova_str_method_eq` → `Nova_str_method_equals` или оставить как alias.

Из `std/collections/set.nv`:
- `Set[T] @minus` — это set difference, **НЕ comparison**, оставить.

### Совместимость

- **Breaking** для existing user-types с `@eq/@lt/@le/@gt/@ge` методами.
- Compiler diagnostic: при встрече `@eq`/`@lt` без `@equals`/`@compare` — warning «deprecated; use Equatable.equals / Comparable.compare».
- Auto-derive: если тип имеет `@compare` но не `@equals`, default body из Comparable заполняет (см. 91.8a).

## Spec / D-block

### D184 — Operator dispatch via protocols (replace magic methods)

**Файл:** `spec/decisions/03-syntax.md` (operator dispatch — D46 territory).

**Content:**

1. Rationale: D9 single path, less boilerplate, protocols как single source.
2. Mapping table (см. выше).
3. Type-checker: для `BinOp::{Eq,Ne}` обязателен `Equatable` (или primitive); для `BinOp::{Lt,Le,Gt,Ge}` обязателен `Comparable` (или primitive).
4. Migration: удалить `@eq/@lt/@le/@gt/@ge` magic methods; deprecation warning для existing usage.
5. Связь: D46 (operator dispatch — этот блок amends D46), D131 (fluent), D183 (canonical protocols).

## Открытые вопросы

1. **Q1:** Deprecation warning для `@eq/@lt/@le/@gt/@ge` — emit warning или ignore при наличии? **Default:** warning (graceful migration); hard error в Plan 92+.
2. **Q2:** Auto-flip operators: `a < b` ↔ `b > a` — implementer не должен писать оба, type-checker derives. Уже сейчас или отдельный followup? **Default:** уже сейчас (через `@compare` sign только).
3. **Q3:** Performance — `compare(a,b) < 0` для record types — extra call vs direct `@lt`. Measure в benchmark? **Default:** acceptable for MVP; benchmark — следующий followup.

> Implementation должна быть **production-grade**. Никаких MVP shortcut'ов,
