## –Ч–∞—З–µ–Љ

Plan 103 V1 (sub-plans 103.1-103.8) ship'–Є—В sync primitives —Б
—В—А–∞–і–Є—Ж–Є–Њ–љ–љ—Л–Љ `lock()/unlock()`, `acquire()/release()` pattern. **V1
mitigation:** `with_lock(fn)` / `with_read(fn)` / `with_permit(fn)`
helpers вАФ closure-form preferred pattern, –Ј–∞—Й–Є—Й–∞–µ—В –Њ—В –Ј–∞–±—Л—В–Њ–≥–Њ release
—З–µ—А–µ–Ј Plan 20 `defer`.

**V2 (Plan 103.9, gated on Plan 100.7):** –њ–µ—А–µ–≤–Њ–і–Є—В –љ–∞ **`MutexGuard
consume`** API вАФ guard-returning `lock()` —Б RAII drop = unlock —З–µ—А–µ–Ј
Plan 100 consume-mechanism. –Ч–∞—Й–Є—В–∞ statically (compile-time) –Њ—В:
- unlock-–±–µ–Ј-lock (impossible вАФ guard required –і–ї—П unlock);
- double-unlock (consume = single-use);
- –Ј–∞–±—Л—В–Њ–≥–Њ unlock (linear-type enforcement –Њ–±—П–Ј—Л–≤–∞–µ—В consume);
- aliasing guard'–∞ –Љ–µ–ґ–і—Г fibers (move-only semantics).

**Plan 100.7 —П–≤–љ–Њ –њ–µ—А–µ—З–Є—Б–ї—П–µ—В** ¬ЂMutex+Lock-guard (std/sync)¬ї –Ї–∞–Ї
–Њ–і–Є–љ –Є–Ј **4 pilot migrations** stdlib. Plan 103.9 вАФ —А–µ–∞–ї–Є–Ј–∞—Ж–Є—П —Н—В–Њ–≥–Њ
pilot (–њ–ї—О—Б —А–∞—Б—И–Є—А–µ–љ–Є–µ –љ–∞ RwLock, Semaphore, Once).

**M16 (master):** Atomics –Э–Х –њ–Њ–ї—Г—З–∞—О—В consume –љ–Є–Ї–Њ–≥–і–∞ (shared-state
primitive, –љ–µ resource).

## Scope

### In scope

**4 —Б–µ–Љ–µ–є—Б—В–≤–∞ –Љ–Є–≥—А–Є—А—Г—О—В—Б—П –љ–∞ consume guards:**

1. **`Mutex` вЖТ `MutexGuard consume`:**
   - `Mutex.lock() -> MutexGuard consume` (new primary API)
   - `Mutex.try_lock() -> Option[MutexGuard consume]`
   - `Mutex.try_lock_for(Duration) -> Option[MutexGuard consume]`
   - `MutexGuard.unlock(consume self)` (explicit release; auto-drop
     also works —З–µ—А–µ–Ј Plan 100.4 cleanup-on-drop)
   - `with_lock(fn)` helper —Б—В–∞–љ–Њ–≤–Є—В—Б—П thin wrapper –љ–∞–і guard-returning core.

2. **`RwLock` вЖТ `ReadGuard consume` / `WriteGuard consume`:**
   - `RwLock.read() -> ReadGuard consume`
   - `RwLock.write() -> WriteGuard consume`
   - `try_*` variants вЖТ `Option[Guard consume]`
   - `*Guard.unlock(consume self)`
   - `with_read(fn)` / `with_write(fn)` вАФ thin wrappers.

3. **`Semaphore` вЖТ `Permit consume`:**
   - `Semaphore.acquire() -> Permit consume`
   - `Semaphore.try_acquire() -> Option[Permit consume]`
   - `Semaphore.try_acquire_for(Duration) -> Option[Permit consume]`
   - `Permit.release(consume self)` (explicit release; auto-drop
     possible —З–µ—А–µ–Ј Plan 100.4)
   - `with_permit(fn)` вАФ thin wrapper.

4. **`Once` вЖТ `OnceGuard consume`:**
   - `Once.try_start() -> Option[OnceGuard consume]` (returns Some
     –і–ї—П exactly-one fiber; None for subsequent вАФ wait for completion).
   - `OnceGuard.commit(consume self)` вАФ successfull completion; transitions
     Once to DONE.
   - `OnceGuard.abort(consume self)` вАФ failed initialization; transitions
     Once to POISONED (–Є–ї–Є back to NEW вАФ D-block decides, similar to
     Plan 100.7 commit/rollback methods –Ї–∞–Ї distinct methods).
   - `Once.call_once(fn)` (V1) вЖТ thin wrapper using try_start + commit/abort
     based on closure result.

**Edition-versioning migration (Plan 62.F.bis):**

- **Edition 2026-Q3 (V2 introduction):**
  - Guard-returning API —Б—В–∞–љ–Њ–≤–Є—В—Б—П primary.
  - `Mutex.unlock()` (bare, –±–µ–Ј guard) вЖТ `#deprecated(since = "0.3")`
    + warning `W_BARE_UNLOCK_DEPRECATED`.
  - `with_lock(fn)` etc. вАФ –±–µ–Ј –Є–Ј–Љ–µ–љ–µ–љ–Є—П (continue —А–∞–±–Њ—В–∞—В—М —З–µ—А–µ–Ј
    new wrapper).
- **Edition 2026-Q4 (V2 enforcement):**
  - Bare `unlock()` removed (or remains –Ї–∞–Ї escape with explicit
    `#unstable` marker).
  - Linear-type enforcement strict: missing `consume MutexGuard` вЖТ
    compile error.

## API design

### `Mutex` V2

```nova
module runtime.sync

/// Guard –і–ї—П locked Mutex. Consume-typed (Plan 100 D131): –Њ–±—П–Ј–∞–љ
/// –±—Л—В—М consumed (—З–µ—А–µ–Ј unlock() –Є–ї–Є auto-drop).
#stable(since = "0.3")
type MutexGuard consume {
    // opaque; internal ref –Ї Mutex
}

#stable(since = "0.3")
export external fn Mutex mut @lock() -> MutexGuard consume

#stable(since = "0.3")
export external fn Mutex mut @try_lock() -> Option[MutexGuard consume]

#stable(since = "0.3")
export external fn Mutex mut @try_lock_for(timeout Duration) -> Option[MutexGuard consume]

#stable(since = "0.3")
export external fn MutexGuard @unlock(consume self)

// V1 helper вАФ preserved, becomes thin wrapper
#stable(since = "0.2")
export fn Mutex mut @with_lock[R](body fn() -> R) -> R {
    let guard = self.lock()
    defer guard.unlock()
    body()
}

// V1 bare API вАФ deprecated
#deprecated(since = "0.3", suggest = "Mutex.lock() returns MutexGuard; use guard.unlock() or with_lock(fn)")
#stable(since = "0.1")
export external fn Mutex mut @unlock()
```

### `RwLock` V2

```nova

#stable(since = "0.3")

#stable(since = "0.3")

```

### `Once` V2

```nova
#stable(since = "0.3")
type OnceGuard consume { /* opaque */ }

#stable(since = "0.3")
export external fn Once mut @try_start() -> Option[OnceGuard consume]

/// Successful completion вАФ transitions Once to DONE, wakes waiters.
#stable(since = "0.3")
export external fn OnceGuard @commit(consume self)

/// Failed init вАФ transitions Once to POISONED (or back to NEW вАФ
/// see D174 decision). Wakes waiters —Б error throw on resume.
#stable(since = "0.3")
export external fn OnceGuard @abort(consume self)

// V1 helper вАФ thin wrapper
#stable(since = "0.2")
export fn Once mut @call_once(body fn() -> ()) {
    match self.try_start() {
        Some(g) => {
            // try/catch –і–ї—П panic-safety
            // success вЖТ commit; panic вЖТ abort
            g.commit()
        }
        None => {}  // already done or in progress
    }
}
```

## Industry comparison

| Feature | Nova 103.9 V2 | Rust | Java | Kotlin | Go |
|---|---|---|---|---|---|
| **MutexGuard RAII unlock** | вЬЕ via consume | вЬЕ Drop trait | вЭМ try/finally | вЭМ withLock {} | вЭМ Lock/Unlock pair |
| **Linear-type guard (cannot leak)** | вЬЕ via consume | вЪ† Drop вЙ† linear (can be `forget`'d) | вЭМ | вЭМ | вЭМ |
| **Distinct commit/abort –љ–∞ init guard** | вЬЕ M of Plan 100 unique | вЭМ | вЭМ | вЭМ | вЭМ |
| **No lifetime annotations** | вЬЕ (M of Plan 100 D157) | вЭМ `MutexGuard<'a, T>` | n/a | n/a | n/a |

**Nova edge:** consume-types unique (Rust Drop вЙ† guaranteed via
`mem::forget`; Java/Kotlin/Go –љ–µ—В statically-enforced); commit/abort
distinct methods (vs single drop) вАФ Plan 100 P-–њ—А–µ–Є–Љ—Г—Й–µ—Б—В–≤–Њ.
