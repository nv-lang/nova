## Идея

Новый модуль `std/unicode` с Unicode-data, закрывающий разрыв с Java `Normalizer`/
`BreakIterator`, JS `normalize`/`Intl.Segmenter`, Rust crates `unicode-*`. Подключается
**явно** — за размер таблиц платит импортирующий.

## D253. `std/unicode` — что в ядре vs библиотеке

Ядро (152.1–152.3a): байты, codepoint decode/encode, ASCII case/classification,
byte-`Compare`. **`std/unicode` (здесь):** полная нормализация, grapheme/word
сегментация, case folding, Unicode case mapping, (152.5b) collation.

API:
- **Нормализация:** `normalize_nfc/nfd/nfkc/nfkd(s) -> str` (decomposition + canonical
  ordering по CCC + composition; compat-формы для NFKC/NFKD).
- **Grapheme-сегментация (UAX #29):** `str.@as_graphemes() -> GraphemesView`
  (extended grapheme clusters: combining marks, ZWJ-emoji, regional-indicator флаги).
  Третья линза, симметрично `as_bytes`/`as_chars`.
- **Case folding:** `fold_case(s) -> str` (для caseless matching, 152.5).
- **Unicode case mapping:** `to_uppercase`/`to_lowercase`/`to_titlecase`
  (multi-codepoint, без локали) — апгрейд ASCII-only `string.nv`.
- **Word/sentence boundaries (UAX #29):** `word_indices`/`sentences` — **roadmap**
  (152.4.4 / `[M-152-word-boundaries]`).

## Q-unicode-data — ЗАКРЫТО (стратегия)

Таблицы генерируются build-time-инструментом **`nova-codegen unicode`** из официального
UCD (`UnicodeData.txt`, `CaseFolding.txt`, `GraphemeBreakProperty.txt`,
`DerivedNormalizationProps.txt`, `CompositionExclusions.txt`, `emoji-data.txt`) в
компактные **range-кодированные** Nova-таблицы `std/unicode/data/`. Пин к версии
(константа `UNICODE_VERSION`, напр. `16.0`), lazy-инициализация. **Без ICU/ОС-зависимостей.**
Прецедент: Rust `unicode-*` (codegen), Go `maketables`.

## Тесты

- **POS:** NFC(«e»+◌́)==NFC(«é»); NFD обратно; NFKC(ﬁ-лигатура)→«fi»;
  graphemes(«👨‍👩‍👧»)==1, («🇺🇸»)==1, («é»=e+◌́)==1; `fold_case(«ß»)==«ss»`;
  `to_uppercase(«straße»)==«STRASSE»`. Прогон по официальным эталонам UAX #15
  (`NormalizationTest.txt`) и UAX #29 (`GraphemeBreakTest.txt`).
- **NEG:** невалидный UTF-8 в normalize → policy (инвариант R-UTF8 ⇒ не должен
  возникать; тест-страховка); неизвестная Unicode-версия данных → ошибка сборки/init.

## Критерии приёмки

- **A1.** NFC/NFD/NFKC/NFKD проходят официальный `NormalizationTest.txt` (UAX #15).
- **A2.** Grapheme-count проходит `GraphemeBreakTest.txt` (UAX #29): эмодзи-ZWJ,
  флаги-RI, combining — каждый = 1 grapheme.
- **A3.** `as_graphemes()` — линза, симметрична `as_bytes`/`as_chars`, не
  `Index[int]`-индексируема (только итерация/`at`).
- **A4.** `fold_case`/Unicode `to_uppercase`/`to_lowercase` корректны (multi-cp);
  `str.to_upper/lower` апгрейжены с ASCII-only.
- **A5.** Unicode-таблицы версионированы (`UNICODE_VERSION`), генерируются `nova-codegen
  unicode`, не зависят от ICU; размер задокументирован.
- **A6.** `std/unicode` подключается явно (импорт) — ядро без него собирается и
  работает (ASCII-complete).
- **A7.** Полный `nova test` без новых FAIL; plan152_4-фикстуры PASS.
