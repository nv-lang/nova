## Проблема (verified state, 2026-05-18)

### 1. Edition не существует

`nova.toml` парсер (`compiler-codegen/src/manifest.rs:41-89`) знает только `[package].name` + `[lib].src`. Поле `edition = "2026.05"` не парсится — silently ignored. Spec [D26 + 07-modules](../../spec/decisions/) предполагает edition-based prelude versioning (per Plan 62 §«Production-grade versioning»).

Resolver (`compiler-codegen/src/imports.rs:103-186`) auto-import'ит фиксированный `std/prelude.nv` без любой edition logic. Если в будущем prelude content изменится breaking-way (e.g. new method on Option) — старые пакеты сломаются. **Industry standard:** Rust uses `edition = "2021"`, Go uses `go 1.21` — стабильность через explicit pin.

### 2. W_PRELUDE_SHADOW lint отсутствует

Per Plan 62 §«Acceptance criteria»: user может shadow'ить prelude names (`type Option { foo int }`) — currently it just works (Plan 62.A разрешил file-based decls). Spec предполагает W_PRELUDE_SHADOW warning, не error — обучающее AI-friendly diagnostic. Currently silent.

### 3. Time/Mem без formal declaration

Plan 62.F документировал DEFER для Time/Mem/Detach (effects.nv DEFER block). Time особенно — это **user-overridable effect** (`with Time = handler Time { sleep(ms) {...} } { body }` — см. examples/effects/effects.nv). Hardcoded в `emit_c.rs:1071` без `.nv` декларации → не виден в `nova doc`.

### 4. Spec D-block amendments

Per Plan 62 §«Spec changes»:
- **D26** (08-runtime.md §249-431): обновить таблицу prelude items (отметить мигрированные vs hardcoded).
- **07-modules.md §962-979** («Prelude opt-out»): добавить `partial_prelude(...)` clause syntax (currently spec только описывает `no_prelude`).

Currently spec немного отстаёт от impl — minor drift.

### Phase 1: Edition versioning ✅ ЗАКРЫТ commit `445904b2b4e`

**Steps:**

1. Extend `Manifest` struct (`compiler-codegen/src/manifest.rs:19-22`):
   ```rust
   pub struct Manifest {
       pub package_name: String,
       pub source_root: PathBuf,
       pub edition: Option<String>,  // NEW (e.g. "2026.05")
   }
   ```

2. Parse `edition` в `parse_manifest`:
   ```rust
   match (section, key) {
       ("package", "name")    => package_name = Some(val),
       ("package", "edition") => edition = Some(val),
       ("lib", "src")         => lib_src = Some(val),
       _ => {}
   }
   ```

3. Thread Manifest через CLI flow → `resolve_imports_inline`. Currently `resolve_imports_inline(module, entry_path, stdlib_dir, ...)` не получает Manifest. Нужно добавить `manifest: Option<&Manifest>`.

5. Создать `std/prelude/2026_05.nv` initial edition pin (content identical to current `std/prelude.nv` для bootstrap).

6. Test: 2 sub-workspace в `nova_tests/plan62/editions/` с разной edition → каждый видит свой prelude content.
