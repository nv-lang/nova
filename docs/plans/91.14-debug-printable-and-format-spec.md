## Что

`DebugPrintable` protocol для отладочного представления значений + расширение
синтаксиса interp-strings новой format-spec формой `${expr:?}` (Rust/Python
`{:?}`-style debug marker). `${expr}` остаётся user-facing display (Printable
protocol); `${expr:?}` — диагностический output (DebugPrintable), полный
memberwise dump со скрытыми/internal полями.

Plan заходит infrastructure для произвольных format-specs (`:hex`, `:pad-N`,
`:.3`, etc. — deferred), но V1 ограничен единственным debug variant.

## Почему

1. **Pointer debugging** — bare `${p}` для `*T` сейчас валит хард-ошибкой
   `E_PTR_NO_DISPLAY_USE_DEBUG_STR`, forcing `(*T).to_debug_str()` в unsafe-
   контексте. Это inelegant (ad-hoc вместо unified protocol path) и leaks
   unsafe semantic в каждый log-statement.
2. **Debug ≠ Display** — auto-derive memberwise output (`TypeName { field:
   value }`) — debug-specific shape. Printable = user-facing display, может
   скрывать internals (priv fields опционально не печатаются) либо
   форматировать short-form (`Vec3(1, 2, 3)` вместо `Vec3 { x: 1.0, y: 2.0,
   z: 3.0 }`); Debug = diagnostic (полное representation, всегда memberwise,
   priv включён).
3. **0.1 release ergonomics** — печать struct'ов в логах без manual `@fmt`
   impl. Plan 91.8a.2 default body даёт Printable auto-synthesis для `${x}`;
   91.14 даёт parallel auto-synthesis для `${x:?}`.
4. **Foundation для format-spec extensions** — `:hex`, `:pad-N`, `:.3` уже
   запрошены пользователями, но требуют design freeze для grammar. V1
   landed единственный spec (`:?`) — extensions добавляются tipa-by-tipa без
   re-architecting AST/lexer.

### Ф.1 — AST extension (~½ dev-day)

- **Ф.1.1** `InterpStrPart::Expr` → split на два variant'а:
  ```rust
  pub enum InterpStrPart {
      Text(String),
      Expr(Expr),                                  // bare ${expr}
      ExprWithFormat { expr: Expr, spec: FormatSpec },   // ${expr:?}
  }
  ```
  (либо unified `InterpStrPart::Expr { expr, spec: FormatSpec }` где
  `FormatSpec::None` — default. Final shape choose в Ф.1 impl — prefer
  unified для downstream simplicity.)
- **Ф.1.2** NEW `enum FormatSpec`:
  ```rust
  pub enum FormatSpec {
      None,            // bare ${expr} → Printable.@fmt(sb)
      Debug,           // ${expr:?} → DebugPrintable.@debug_fmt(sb)
  }
  ```
  Foundation only — `Hex`/`Pad(usize)`/`Precision(u8)` deferred к
  `[M-91.14-format-dsl-extensions]`.
- **Ф.1.3** Display impl updates (`impl Display for InterpStrPart`,
  `impl Display for FormatSpec`) — debug-form для compiler diagnostics.

### Ф.2 — Lexer/parser extension (~½ dev-day)

- **Ф.2.1** Lexer recognizes `:` token внутри `${...}` lexer-state (separate
  state — `:` нормально допустим в Nova outside interp-strings и означает
  type-ascription).
- **Ф.2.2** Parser extends `parse_interp_string` к accept syntax `${expr:?}`:
  - `${expr}` → `FormatSpec::None` (existing behavior).
  - `${expr:?}` → `FormatSpec::Debug`.
  - `${expr:<unknown>}` → `E_FORMAT_SPEC_UNKNOWN` с suggestion list (только
    `?` в V1).
- **Ф.2.3** Reject malformed specs:
  - `${expr:}` (empty spec) → `E_FORMAT_SPEC_EMPTY`.
  - `${expr: ?}` (whitespace) → `E_FORMAT_SPEC_WHITESPACE` (strict; no
    forgiveness — keeps parser simple).
  - `${expr:?suffix}` → `E_FORMAT_SPEC_UNKNOWN` (trailing chars not allowed
    в V1).

### Ф.3 — `DebugPrintable` protocol decl + type-checker (~1 dev-day)

- **Ф.3.1** NEW protocol в `std/prelude/protocols.nv`:
  ```nova
  export type DebugPrintable protocol {
      fn @debug_fmt(sb StringBuilder) -> ()
  }
  ```
  Parallel к existing `Printable` (Plan 91.8a.2 line ~313). Single-method
  protocol, signature mirrors `@fmt` exactly — different method name
  decouples от Printable namespace.
- **Ф.3.2** Default body strategy: NO default body в protocol declaration
  (unlike Equatable→Comparable coercion в 91.8a.2). Instead — **synthesis
  in codegen** (Ф.4.2). Rationale: debug shape всегда memberwise (не
  user-overridable shape), но user может override individual fields поведение
  предоставив explicit `fn T @debug_fmt(sb)`. Synthesis triggered through
  type-checker если type не provides explicit `@debug_fmt`.
- **Ф.3.3** Type-checker: при encountering `FormatSpec::Debug` для interp-
  expr — verify expr type implements DebugPrintable (либо has explicit impl,
  либо eligible для auto-synthesis). Если ineligible → 
  `E_DEBUG_PRINTABLE_NOT_IMPLEMENTED` с hint об auto-derive via 
  `[M-91.14-derive-debug]` (когда landed).
- **Ф.3.4** Plan 97 framework hook — protocol resolution / vtable lookup
  goes через standard Printable infrastructure (Plan 91.8a.2 Ф.3-Ф.4
  precedent). Concrete: `resolve_protocol_method(type, "DebugPrintable",
  "@debug_fmt")` → `Option<FnDecl>`.

### Ф.4 — Codegen + default body synthesis (~1 dev-day)

- **Ф.4.1** `emit_c.rs::emit_interp_str` branch на `FormatSpec`:
  ```rust
  match part {
      InterpStrPart::Text(s) => emit_text_literal(s),
      InterpStrPart::Expr { expr, spec: FormatSpec::None } =>
          emit_call("@fmt", &[sb, expr]),
      InterpStrPart::Expr { expr, spec: FormatSpec::Debug } =>
          emit_call("@debug_fmt", &[sb, expr]),
  }
  ```
- **Ф.4.2** Default body synthesis — Plan 91.8a.2 mechanism extended для
  `@debug_fmt`:
  - For each non-primitive type T, если no explicit `fn T @debug_fmt(sb)`
    AND type used в `${x:?}` context, synthesize:
    ```nova
    fn T @debug_fmt(sb StringBuilder) -> () {
        sb.append("TypeName { ")
        sb.append("f1: ")
        @f1.debug_fmt(sb)
        sb.append(", f2: ")
        @f2.debug_fmt(sb)
        ...
        sb.append(" }")
    }
    ```
  - Recursive — each field's `@debug_fmt` called (либо primitive default,
    либо user-defined, либо synthesized further).
  - Cycle detection: same algorithm as Plan 126 §2.7 (visited set).
- **Ф.4.3** Primitives default `@debug_fmt`:
  | Type | Output |
  |---|---|
  | `int` / `i32` / `i64` | `"-42"` (signed, decimal) |
  | `u32` / `u64` | `"42"` (unsigned, decimal) |
  | `f32` / `f64` | `"3.14"` (lossless decimal, mirrors Rust `{:?}`) |
  | `bool` | `"true"` / `"false"` |
  | `char` | `"'a'"` (single-quoted, escape special) |
  | `str` | `"\"foo\\n\""` (double-quoted, escape `"`, `\n`, `\t`, `\\`, `\r`) |
  | `[]T` | `"[e1, e2, e3]"` (square brackets, comma-sep, recursive) |
  | `Option[T]` | `"Some(<debug>)"` / `"None"` |
  | `Result[T,E]` | `"Ok(<debug>)"` / `"Err(<debug>)"` |
  | `Tuple` | `"(a, b, c)"` (Plan 120 D215) |
- **Ф.4.4** `(*T).debug_fmt` impl для pointer types (в unsafe context):
  - Emits `"0x7f8a1c4d0010 -> Account { ... }"` (hex address + arrow +
    pointee debug repr).
  - Если pointee debug-fmt fails (например null ptr) — `"0x0 (null)"` без
    deref.
  - Codegen path: special-case в emit_c.rs или synthesize wrapper fn —
    decision в Ф.4 impl.

### Ф.5 — Pointer integration + diagnostic update (~½ dev-day)

- **Ф.5.1** `*T` family (`*T`, `*ro T`, `*mut T`, `*unsafe T` — Plan 118
  D216) impls `DebugPrintable`. Implementation в `std/runtime/pointer.nv`
  либо synthesized по Ф.4.4 pattern.
- **Ф.5.2** `${p:?}` теперь работает в unsafe context — emits hex+pointee.
  В safe context still rejected (raw `*T` deref forbidden outside `unsafe`).
- **Ф.5.3** **KEEP** `E_PTR_NO_DISPLAY_USE_DEBUG_STR` для bare `${p}` без
  `:?` (hard error, intentional) — discourages accidental ASLR leaks в
  user-facing output. Update диагностику:
  ```
  error[E_PTR_NO_DISPLAY_USE_DEBUG_STR]: cannot use `*T` в bare
    interpolation — pointers leak ASLR addresses in user-facing output
   --> file.nv:42:18
      |
   42 |     log("ptr = ${p}")
      |                  ^^ pointer here
      |
    help: use debug format spec `${p:?}` instead (unsafe context required)
    help: or call `.to_debug_str()` explicitly in unsafe block
  ```
- **Ф.5.4** Old `(*T).to_debug_str()` API kept (backwards compat) — alias
  для `@debug_fmt(sb) + sb.steal()` для callers без interp-string.

## Followups

- **`[M-91.14-format-dsl-extensions]`** — extensions к format-spec:
  - `:hex` / `:HEX` — hex output для integers (`${42:hex}` → `"0x2a"`).
  - `:pad-N` / `:pad_N` — left-pad с spaces до N chars.
  - `:.N` — float precision (`${3.14159:.3}` → `"3.142"`).
  - `:bin` / `:oct` — binary / octal representation.
  - `:e` / `:E` — scientific notation.
  - Foundation infrastructure landed в this plan; extensions just add
    new `FormatSpec` variants + codegen branches.
- **`[M-91.14-derive-debug]`** — `#impl(DebugPrintable)` auto-derive 
  через Plan 126 mechanism (parallel к Plan 126 Printable derive). 
  Current V1 — implicit synthesis (no annotation needed); future will 
  make this explicit + opt-out-able.
- **`[M-91.14-private-field-policy]`** — policy для priv fields в debug
  output. V1 = always include (full diagnostic). Future may add 
  `#debug_skip` attribute либо priv-respecting variant.
