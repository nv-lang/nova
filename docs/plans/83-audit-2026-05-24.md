## 1. Резюме (TL;DR для re-analysis agent)

**Что реально landed по Plan 83 на 2026-05-24:**

- ✅ **Correctness:** atomic fiber state machine (CAS guards mco_resume),
  uncollectable SpawnCtx, atomic cancel_requested, handler-snapshot capture
  и restore на worker, supervised drain ownership, D93 park-with-predicate,
  Blocking effect → libuv threadpool, AsyncDetach orphan-scope, sysmon
  preemption (Plan 44.7) — **production-grade**.
- ✅ **D138 ACTIVE:** flip default-on M:N landed (2026-05-24). Compiled
  binary без `runtime.init()` использует все CPU.
- ✅ **Test runner:** 1130–1135 PASS / 0 FAIL под armed default, 1141 PASS
  bootstrap baseline. ≥1× speedup acceptance MET через inline parallel-for
  threshold (Plan 83.4.5.10 Ф.3).

**Что **НЕ** landed (gaps vs production-grade, по убыванию impact):**

1. **[P0] Boehm `THREAD_LOCAL_ALLOC` не включён.** Boehm под global lock
   на каждый `GC_malloc[_uncollectable]`. По собственной документации
   Boehm включение `THREAD_LOCAL_ALLOC` + `GC_REDIRECT_TO_LOCAL` "greatly
   reduces locking" — оценка 2-5× spawn-cost speedup. **~1 dev-day fix.**
   → Plan 83.5.
2. **[P0] Per-worker SpawnCtx pool** (Plan 83.4.5.10 Ф.1 deferred). Go
   per-P mcache аналог. Закрывает оставшийся spawn-allocation bottleneck.
   ~1-2 dev-day. → Plan 83.6.
3. **[P0] Stress test coverage НЕ покрывает armed M:N.** Все 3 stress
   теста (`spawn_stress_10k`, `park_wake_stress`, `cancel_stress`) идут
   с `// ENV NOVA_AUTOARM=0` (cooperative mode). Plan 83.4.5.6 §6.4
   acceptance "10⁶ spawn / 10⁵ park-wake / 10⁴ cancel" реализовано
   как 10³/10²/10¹ под **cooperative** path. **Production armed M:N
   stress-валидация отсутствует.** → Plan 83.9.
4. **[P0] Negative-test gap.** В `nova_tests/concurrency/` (149 тестов)
   только **1 negative test** (polymorphic recursion compile-error,
   не runtime). Нет EXPECT_PANIC, EXPECT_RUNTIME_ERROR проверок для:
   double-spawn, cancel race, supervised drain race, GC root leak,
   handler escape. → Plan 83.10.
5. **[P0] Async net/socket stdlib отсутствует.** Plan 18 P0 unblocked,
   но реальной реализации `std/net/{tcp,udp}` через libuv нет.
   Production-grade backend язык без TCP support = hypothetical. → Plan 83.12.
6. **[P1] Нет `runnext`-style LIFO slot.** Go runtime+tokio оба имеют
   single-slot priority queue для cache-warm handler chains. Nova
   каждый spawn идёт через полный Chase-Lev deque push. ~0.5d.
   → Plan 83.7.
7. **[P1] `uv_async_send` слишком медленный для intra-runtime wake.**
   ~5-20µs (mutex+cond) vs Go futex/IOCP <1µs. Можно заменить direct
   eventfd (Linux) / SetEvent (Windows) для cross-worker wake;
   libuv остаётся для real I/O. ~1 dev-week. → Plan 83.8.
8. **[P2] Перфоманс-followups (V2-V3):**
   - mco_coro reuse pool (~1 dev-day).
   - Versioned effect snapshots (~0.5 dev-day).
   - Dynamic NOVA_MAX_EFFECT_STORAGES resize (silent overflow на >32
     эффектов).
   - Epoch-based deque-array reclamation (Plan 44.6 deferred).
   - Linux fiber arena dealloc warnings ([M-82-linux-arena-dealloc]).
9. **[P3] Nova differentiators не упрощены до production-API:**
   - Effect-aware deadline propagation — Nova может extend CancelToken[T]
     с deadline:Time, compiler-warn если call site не успевает. **Никто
     из 4 конкурентов этого не имеет.** → Plan 83.11.
   - panic-safety на spawn (Go `defer recover()` parity) — поведение
     текущее не задокументировано. → Plan 83.10 followup.

## 3. Что обнаружил code-аудит (детально)

Делегировано Explore-агенту. Полный отчёт в conversation log; ключевое:

### 3.1 Spawn hot-path

- **runtime.c:1069-1158** — `nova_runtime_spawn_global` / `spawn_into`.
- **deque.h** — Chase-Lev work-stealing deque (SPAA 2005, PPoPP 2013
  memory ordering correct).
- **SpawnCtx allocation:**
  - Under armed M:N: `nova_alloc_uncollectable` (Boehm
    GC_malloc_uncollectable). См. **emit_c.rs:5091-5094** для условия
    branch.
  - Under bootstrap: `nova_alloc` (managed).
- **Cross-thread wake:** mutex-protected wake_pending list + uv_async_send
  (runtime.c:308-320). ~10-30µs per cross-thread spawn.

**Production-status:** ✅ SEQ_CST fence перед cross-thread deque push
гарантирует видимость SpawnCtx fields (runtime.c:1104-1110).
nova_aint_inc RELEASE ordering на pending_remote.

**Known limitation:** Old deque arrays leak при resize (deque.h:123).
Amortised ~16 KB per worker per 1000 spawns. Epoch-based reclamation —
followup Plan 44.6.

### 3.2 Worker main loop (_worker_main)

State machine (runtime.c:323-666):

```
(0) uv_run(UV_RUN_NOWAIT)      ← service libuv every iteration
(1) lock + drain wake_pending  ← cross-thread wakes
(2) nova_deque_pop (owner LIFO) ← fresh work + woken fibers
(2.5) yielded-FIFO pop          ← preempted CPU-fibers (Plan 44.7)
(3) Steal loop (CAS top peer)
(4) uv_run(UV_RUN_ONCE) if idle ← block until ready
(5) CAS IDLE→RUNNING guard + mco_resume(co)
(6) State cleanup: TLS save/restore
```

**CAS guard (Plan 83.4.5.7, L485-508)** — главный fix для double-resume
race. CAS-loser SKIP'ает resume — защищает от:
- Concurrent steal race (one worker stole, owner pop повторно)
- Duplicate wake-pending от cross-thread cancel

**TLS save/restore (Plan 83.4.2 Ф.2, L434-534)** — critical для GC + handler
correctness:
- outer_scope, outer_fail_top, outer_interrupt_top
- effect_snapshot save/restore per fiber
- Без этого — cross-stack longjmp → SIGSEGV.

### 3.3 Park/wake + cancel

- **nova_sched.h:119-124** — `park` стop store NOVA_FIBER_STATE_PARKED (RELEASE).
- **nova_sched.h:206-245** — `wake` CAS true→false на parked flag (ACQ_REL).
  Только winner вызывает dispatch_ready.
- **Cancel wake-all** (nova_sched.h:318-365) — Plan 83.4.5.1, iterates
  parked-flags + nova_sched_wake (CAS guard inside).

### 3.4 GC integration

- **alloc_boehm.c:108-136** — `nova_alloc_uncollectable` через
  `GC_malloc_uncollectable`. Plan 83.4.5.8 motivation.
- **GC scan range:** `GC_push_other_roots` callback (fiber_arena_win.c)
  walks глобальный append-only список арен (Plan 82 Ф.1).
  Pushes ТОЛЬКО MEM_COMMIT pages — lazy-commit optimization.
- **STW safety:** `GC_register_my_thread` в каждом worker init
  (runtime.c:335-343). Но **нет** explicit validation что все workers
  зарегистрированы. Под `GC_THREADS` compile-time flag, тесты должны
  catch.

**КРИТИЧЕСКИЙ GAP (P0):** **Boehm `THREAD_LOCAL_ALLOC` НЕ включён**
в Nova build. Per Boehm's own scale.html:
> "only one thread can be allocating or collecting at one point.
> This inherently limits performance of multi-threaded applications
> on multiprocessors."

И:
> "Enable `THREAD_LOCAL_ALLOC` + `GC_REDIRECT_TO_LOCAL`, call
> `GC_thr_init()` — greatly reduces locking."

Verified Grep: `grep -r THREAD_LOCAL_ALLOC compiler-codegen/nova_rt/`
не находит defines. **Это самый дешёвый и большой perf win.** → Plan 83.5.

### 3.5 Preemption (Plan 44.7)

- **sysmon** thread (runtime.c:247-280) — uv_sleep(10ms) tick, checks
  каждый worker current_fiber_start vs NOVA_PREEMPT_SLICE_NS.
- **Codegen safepoints** — `nova_preempt_check()` at function prologue +
  loop backedge. TLS pointer `_nova_preempt_ptr` → live read из worker
  struct.
- **vs Tokio**: Nova checkpoints **finer-grained** чем Tokio (Tokio
  только на `.await`). Nova worst-case yield latency = loop body cost;
  Tokio = unbounded для await-free compute loops.
- **vs Go**: Nova не имеет true async preemption (SIGURG). Tight asm
  loops в FFI не preemtable.

### 3.6 Cancel (Plan 83.4.3 B5)

- `nova_atomic_bool cancel_requested` с ACQUIRE/RELEASE fences на всех
  12 read/write сайтах. **Production-grade.**
- Cross-runtime parity: аналог tokio CancellationToken / Go context.Done.

### 3.7 Effect snapshot

- **effects.h:650-714** — NovaEffectRegistry + snapshot save/restore.
- **NOVA_MAX_EFFECT_STORAGES = 32** — **silent overflow** (effects.h:692-693).
  Bootstrap 3 built-in + user <10 эффектов OK. **Production needs
  dynamic resize либо assert.** → Plan 83.5 followup (часть pool plan'а).
- Per-spawn `nova_effect_snapshot_save` копирует table — ~1-5µs.
  Versioning skip — V2.

### 3.8 Fiber arena

- **Linux/macOS** (fiber_arena.c): mmap(MAP_NORESERVE), guard
  PROT_NONE, MADV_NOHUGEPAGE.
- **Windows** (fiber_arena_win.c): VirtualAlloc(MEM_RESERVE) + lazy
  commit per slot. Plan 82 Ф.1: append-only арена list для GC scan.
- **8MB slot** (Plan 83.4.5.10 Ф.2 hypothesis-rejected): уменьшать
  бессмысленно — virtual reservation FREE на 64-bit, lazy commit делает
  physical = actual usage.

**Known issues:**
- P41-3 (fiber_arena.c:188): нет detection `vm.overcommit_memory=2` →
  на Linux OOM-kill в edge cases.
- P41-6: нет SIGSEGV handler с pretty error.
- [M-82-linux-arena-dealloc] (2026-05-24): WSL2 показывает
  `fiber_dealloc ptr outside arena` warnings → likely memory leak per
  fiber на Linux. Не investigated, P3 followup.

### 3.9 Detach / orphan (Plan 83.4.5.2)

- `nova_runtime_spawn_orphan` (runtime.c:1169-1286).
- Global static `_nova_orphan_scope`, lazy-init под mutex.
- Drain on atexit + explicit `nova_runtime_drain_orphans()`.
- Plan 83.4.5.8: pending_remote tracking для drain через
  `nova_runtime_orphan_scope()` API.

### 3.10 Shutdown

Ordered (runtime.c:981-1065):
1. Stop sysmon FIRST (prevent reading freed memory).
2. Signal stop + wake workers.
3. Join workers.
4. Release retired fiber arenas (Plan 82 Ф.3).
5. Cleanup handles + loops.
6. GC_remove_roots + free workers.

**Production-grade.** Sysmon-first ordering — критично, иначе race на
current_fiber_start.

### 3.11 NovaWorker struct

```c
struct NovaWorker {
    int id;
    uv_thread_t thread;
    uv_loop_t loop;                  /* per-worker libuv */
    uv_async_t wake_handle;
    NovaDeque deque;                 /* Chase-Lev */
    NovaFiberQueue scope;            /* home scope */
    nova_atomic_bool stop;
    nova_atomic_int pending_count;
    nova_mutex_t wake_mu;
    mco_coro** wake_pending;
    mco_coro** yielded;              /* FIFO (Plan 44.7) */
    uint64_t current_fiber_start;
    volatile int preempt_flag;       /* sysmon writes */
};
```

**Missing (P0):**
- Per-worker SpawnCtx pool (Plan 83.4.5.10 Ф.1 DEFERRED). → Plan 83.6.
- runnext slot (LIFO priority). → Plan 83.7.

## 4. Сравнение с Go / Tokio / Kotlin / Node (детально)

Делегировано general-purpose агенту. Полный отчёт в conversation log;
ключевая итог-таблица:

| Axis | Go | Tokio | Kotlin | Node Worker | Nova | Rank |
|------|----|----|--------|-------------|------|------|
| Spawn cost | ~50-100 ns | <100 ns | <200 ns | ~30-50 ms | ~10-100 µs | 5/5 ❌ |
| Stack | grow 2K→∞ | stateless | stateless | OS thread | fixed 8MB | 3/5 ⚠ |
| Allocator | per-P mcache | jemalloc | TLAB | V8 | Boehm global lock | 5/5 ❌ |
| Wake latency | <1 µs futex | ~1 µs mio | ~1 µs unpark | ~5-20 µs libuv | ~5-20 µs libuv | 4/5 ⚠ |
| Preemption | true SIGURG | cooperative | cooperative | none | cooperative+finer | 2/5 ✓ |
| Cancel typing | untyped | opaque | exception | AbortError | **typed `CancelToken[T]`** | 1/5 ✅ |
| Structured conc | 3rd-party | std | first-class | none | **first-class `supervised`** | 1/5 ✅ |
| Blocking offload | auto P-handoff | spawn_blocking | Dispatchers.IO | uv_queue_work | **typed `Blocking` effect** | 1/5 ✅ |
| GC pause | <1ms concurrent | N/A | <1ms ZGC | <100ms | <16ms Boehm STW | 4/5 ⚠ |
| **Effect-aware** | no | no | no | no | **yes (compile-time)** | 1/5 ✅✅ |
| **Contract verif on async** | no | no | no | no | **yes (Plan 33)** | 1/5 ✅✅ |

**Nova сильные стороны (defensible):**
1. **Typed CancelToken[T]** + From-cascade + tok.merge — превосходит всех.
2. **Compile-time `Blocking` effect** — невозможно "забыть spawn_blocking"
   как в Tokio.
3. **Effect-aware structured concurrency** — supervised scope знает
   union effects детей статически (Plan 03.4 effect-surface).
4. **Verified contracts на async code** — Plan 33.5 EffectMethod
   contracts, никто из конкурентов не верифицирует cancel semantics
   статически.
5. **cgroup-aware default** (Plan 83.1) — Go 1.22+ догоняет, у Nova
   изначально лучше Tokio/Node defaults.
6. **Finer-grained preemption checkpoints** чем Tokio — каждая loop
   iteration + fn call, не только `.await`.

**Nova слабости (production-blocking):**
1. Spawn cost 100-1000× медленнее Go.
2. Boehm global allocation lock — single biggest scaling bottleneck.
3. uv_async_send overhead для intra-runtime wake.
4. Нет dynamic stack growth (requires precise GC).
5. Нет panic-safety on spawn документировано.

## 6. Декомпозиция новых планов

### 6.1 [P0] Plan 83.5 — Boehm THREAD_LOCAL_ALLOC

**Файл:** `83.5-boehm-thread-local-alloc.md`.
**Сcope:** Включить Boehm thread-local allocation в build chain Nova.
**Effort:** ~1 dev-day.
**Expected:** 2-5× spawn-cost reduction (single biggest perf win).

Acceptance:
- ≥2× speedup на 16 × fib(33) parallel-for bench (current: ≥1× MET через
  inline threshold; ожидаем ≥2× без inline threshold).
- 1141 bootstrap PASS unchanged.
- 1130+ armed PASS unchanged.
- Linux + Windows + macOS все pass.

Tests:
- Positive: existing bench parallel_speedup.nv + stress tests.
- **Negative (НОВЫЕ):**
  - `gc_thread_local_alloc_correctness.nv` — 16 worker × 1000 alloc
    parallel; checksum (sum of alloc'd values) корректный.
  - `gc_thread_local_alloc_no_race.nv` — TSAN gate (Linux).
  - **EXPECT_PANIC**: попытка `GC_malloc` из не-зарегистрированного
    thread'а должна `abort` с readable error (если такое возможно
    воспроизвести).

### 6.2 [P0] Plan 83.6 — Per-worker SpawnCtx pool

**Файл:** `83.6-spawn-ctx-pool.md`.
**Scope:** Аналог Go per-P mcache. Worker держит local free-list пустых
SpawnCtx struct'ов. Spawn → pop из P-local pool без Boehm lock.
**Effort:** 1-2 dev-day (V1 single-size class).
**Expected:** 2× additional spawn-cost reduction поверх 83.5.

Acceptance:
- ≥3× speedup на 16 × fib(33) parallel-for bench (поверх 83.5).
- ≥10⁴ spawn в <60s под armed M:N (current: timeout на 10³+).
- Memory pressure stable (pool capped, excess freed).

Tests:
- Positive: stress tests (после 83.9) на armed M:N actually validate
  pool correctness.
- **Negative:**
  - `spawn_pool_overflow.nv` — alloc N>POOL_MAX, verify excess через
    Boehm path.
  - `spawn_pool_concurrent_alloc.nv` — 16 workers spawn 1000 каждый,
    verify no double-allocation, no leak.
  - TSAN gate (Linux).

### 6.3 [P1] Plan 83.7 — runnext LIFO slot

**Файл:** `83.7-runnext-lifo-slot.md`.
**Scope:** Single-slot priority queue в NovaWorker для cache-warm
handler chains. Аналог Go's `runnext` и tokio's LIFO slot.
**Effort:** 0.5 dev-day.
**Expected:** 1.5× speedup на handler-chain workloads (`recv → process →
send → recv`).

Acceptance:
- Synthetic ping-pong bench (2 fibers exchanging messages 10⁴×): runtime
  reduction ≥30% vs current.
- Existing tests pass (correctness invariant).

Tests:
- Positive: new bench `bench/m_n/handler_chain_pingpong.nv`.
- **Negative:**
  - `runnext_displaced_to_deque.nv` — verify second push displaces previous
    runnext в deque (no loss).
  - `runnext_steal_safety.nv` — другой worker может steal runnext? (нужно
    решить — Go допускает, Tokio нет).

### 6.4 [P1] Plan 83.8 — Direct wake primitive

**Файл:** `83.8-direct-wake.md`.
**Scope:** Заменить `uv_async_send` для intra-runtime cross-worker wake
на direct eventfd (Linux) / SetEvent (Windows) / kqueue (macOS).
libuv остаётся для I/O events.
**Effort:** 1 dev-week.
**Expected:** Wake latency 5-20µs → <1µs.

Acceptance:
- Cross-worker wake latency измерен (per-bench microbench, см. Plan 57):
  Linux <1µs, Windows <2µs, macOS <1µs.
- Existing tests pass (correctness invariant).
- ≥10% reduction in overall spawn-cost (после 83.5+83.6).

Tests:
- Positive: `bench/m_n/wake_latency.nv` (нужно создать).
- **Negative:**
  - `wake_during_shutdown.nv` — verify wake-event arrives после shutdown
    initiation НЕ crash worker.
  - `wake_under_signal.nv` — verify EINTR handling на Linux eventfd read.
  - `wake_handle_exhaustion.nv` — N parallel cross-worker wakes — verify
    no deadlock на handle resource limit.
  - TSAN gate (Linux).
