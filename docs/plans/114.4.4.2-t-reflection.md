# Plan 114.4.4.2 — Type reflection в const fn (sizeof[T]/align_of[T])

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.5 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — useful для generic helpers, но требует deep type
>   layout integration.
> **Оценка:** ~1-2 dev-day.
> **Зависимости:** Plan 114.4.3 V2 (generic const fn baseline).
>
> **Why extracted:** sizeof[T]/align_of[T] требует compile-time type
>   layout info которой evaluator не имеет. Type system должен provide
>   TypeInfo trait — size + align lookup per backend ABI. Primitives
>   simple (hardcoded), records — requires field sum + padding, generics —
>   per-monomorphization.

## Scope

Add intrinsics:
- `sizeof[T]() -> const int` — byte size T per backend ABI.
- `align_of[T]() -> const int` — alignment T.
- `T.const_FIELD` — read assoc const of T (Plan 114.4.1 D200).

## Tests

- Positive: sizeof[int] == 8, sizeof[bool] == 1, align_of[i64] == 8.
- Negative: sizeof[NonExistent] → undefined type.

## Acceptance

A1: sizeof/align_of evaluated at compile time, replaced literal at use site.
A2: Generic const fn with sizeof[T] works per monomorphization.
