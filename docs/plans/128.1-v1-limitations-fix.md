## 1. Контекст

### 1.1 Что V1 не закрыл (Plan 128 Ф.4)

Plan 128 Ф.2 wired `recv.mutable` через `prepare_method_recv` + emit_c.rs
helpers, что correctly работало для **простых identifier-receivers**:

```nova
mut v = Vec3(x: 1.0, y: 2.0, z: 3.0)
v.scale_inplace(2.0)   // ✅ obj_c = "v", is_ident=true → &v
```

Но **lvalue-projections** (field access, array indexing) НЕ работают:

```nova
mut b = Body { v: Vec3(...), mass: 10.0 }
b.v.scale_inplace(2.0)        // ❌ V1 limitation
                              //    obj_c = "b.v", is_ident=false
                              //    → hoist в temp + &temp
                              //    → writes lost on write-back

mut arr = [Vec3(...), Vec3(...)]
arr[0].scale_inplace(2.0)     // ❌ V1 limitation
                              //    obj_c = "arr.data[0]", is_ident=false
                              //    → same hoist-temp bug
```

Workaround в T12/T14 — `extract-rebind-writeback`:

```nova
mut v = b.v        // copy field to local
v.set_x(99.0)      // mutate local
b.v = v            // write back explicitly
```

T13 demonstrates **separate** problem — `[]NamedTuple` construction.
Type-checker `infer_expr_type::Call` для constructor-call
`Vec3(x: 1.0, y: 1.0, z: 1.0)` НЕ распознаёт `TypeDeclKind::NamedTuple`,
поэтому element-type не выводится и array literal fails compile.

### 1.2 Что нужно

| Gap | Fix | Phase |
|---|---|---|
| `prepare_method_recv` string-based `is_ident` rejects `obj.field` / `arr[i]` | AST-aware `is_lvalue_receiver` check на MethodCall AST node — accept Member-access и IndexAccess chains terminating в identifier с mut binding | Ф.1 |

| Regression coverage | 10 fixtures — T12 без workaround, T13 array element, T14 nested record/array, neg для ro-binding projection | Ф.4 |

### Ф.1 — AST-aware `is_lvalue_receiver` in `prepare_method_recv`

**Goal:** заменить string-based `is_ident` на AST inspection — accept
identifier, member access `a.b.c` chain, index access `arr[i]`, mixed
`a.b[i].c`, terminating в identifier binding.

- Pass `recv: &Expr` (the actual MethodCall receiver) в
  `prepare_method_recv` instead of just `obj_c: &str`.
- Helper `is_lvalue_receiver(recv: &Expr) -> bool`:
  - `ExprKind::Ident(_)` → true.
  - `ExprKind::Member { obj, .. }` → recurse on `obj`.
  - `ExprKind::IndexAccess { obj, .. }` → recurse on `obj`.
  - everything else → false.
- Identifier shortcut при `is_lvalue_receiver(recv) = true`: emit
  `&{obj_c}` (no hoist).
- Rvalue path только для не-lvalue (e.g. function call result,
  arithmetic).

**Acceptance:**
- T12 без workaround: `b.v.scale_inplace()` мутирует `b.v` correctly.
- T13: `arr[0].scale_inplace()` мутирует `arr[0]` correctly.
- T14: nested projection `body.parts[i].velocity.set_x()` correctly
  propagates.
- D228 value-record path unchanged (NovaValue_X* uses same helper).

### Ф.2 — `TypeDeclKind::NamedTuple` в `infer_expr_type::Call` arm

**Goal:** type-checker constructor-call для NamedTuple возвращает
`Ty::Named(<name>, [])` так же как для Record/Variant.

- `compiler-codegen/src/types/mod.rs` line ~2268 / ~3500 — `Call` matches
  arm: добавить branch для `TypeDeclKind::NamedTuple { fields, .. }`.
- Validate args против NamedTuple fields (positional или named).
- Return `Ty::Named(decl_name, [])`.

**Acceptance:**
- `let arr = [Vec3(x: 1.0, ...), Vec3(...)]` type-checks без annotation.
- Array element type inferred correctly.
- Pre-existing typed-array fixtures не регрессируют.

### Ф.3 — Mut-binding enforcement для projection writes

**Goal:** type-checker rejects lvalue-projection mut-method call если
**любой** segment в chain — ro-binding.

- `check_method_call`: если method declared `mut @` И receiver — lvalue
  projection → walk chain checking что root binding `mut` И каждый
  intermediate field declared `mut <T>` (D175 readonly field freeze).
- Emit `E_RECEIVER_BINDING_NOT_MUT` если violation.

**Acceptance:**
- `ro b = Body.new(); b.v.set_x(99.0)` rejected.
- `mut b = ...; b.v.set_x(99.0)` где `Body { v Vec3 }` (без `mut`) rejected.
- `mut b = ...; b.v.set_x(99.0)` где `Body { mut v Vec3 }` accepted.

### 4.1. Markers spawned

- 🟡 **`[M-128.1-nonpure-index-key]`** (P2) — opened в Ф.1 (этот коммит).
  Side-effecting subscript keys on a pointer-ABI receiver currently
  evaluate the key twice — once for `&(arr->data[KEY])` taken as the
  receiver pointer, again for any post-call read через the same
  expression. V1 accepts this duplication для `arr[i]` cases (`i` —
  pure local), V2 fix должен hoist non-pure key в temp перед address-of:
  `nova_int __idx = next_idx(); &(arr->data[__idx])`. Detection: walk
  Index AST chain, classify subscript node as «pure» (Ident / IntLit /
  Member-of-pure) или «impure» (Call / Bang / Try / ...). Impure → hoist.
  Scope: `prepare_method_recv` lvalue path только; rvalue hoist уже
  evaluates once (single `obj_c` substitution into temp init).

## 8. Lessons (предварительные)

1. **String-based AST inspection — antipattern.** V1 `is_ident`
   regex-style check (`obj_c.chars().all(|c| c.is_alphanumeric() ||
   c == '_')`) выглядит безобидным shortcut'ом, но silently loses
   correctness для projections. AST-based check — single source of
   truth.

2. **«Known V1 limitation» в фикстурах — load-bearing technical debt.**
   T12/T13/T14 «workaround» pattern был honest acknowledgment в Plan
   128, но без followup плана — debt forever. Plan 128.1 closes debt
   immediately после parent merge.

3. **Constructor-call inference parity.** `TypeDeclKind` variants
   должны иметь симметричные arms во всех inference paths —
   `Record`/`Variant` had arms, `NamedTuple` was added later (Plan
   120) и пропущен в `infer_expr_type::Call`. Code review checklist
   для новых `TypeDeclKind` variants.
