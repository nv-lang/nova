## 1. Контекст

User feedback (2026-05-24): "не использовать инвертированных имен в env".
Существующая env-var `NOVA_NO_AUTOARM=1` — инвертированное имя (двойное
отрицание: "no-autoarm = true" → auto-arm disabled).

Better convention: положительное имя `NOVA_AUTOARM` с inverted semantics:
- `NOVA_AUTOARM=0` — disable auto-arm (cooperative-only mode).
- `NOVA_AUTOARM=1` либо unset либо other — auto-arm enabled (default,
  per D138 default-on M:N).

Cross-runtime parity:
- Go: `GOMAXPROCS=1` (positive value), nothing inverted.
- tokio: `TOKIO_WORKER_THREADS=1`, nothing inverted.
- Node: no equivalent env, positive only.

## 2. Семантика

| Variable                  | Behavior                                                       |
|---------------------------|----------------------------------------------------------------|
| (unset)                   | default — auto-arm enabled (default-on M:N per D138).          |
| `NOVA_AUTOARM=0`          | disable auto-arm; cooperative-only mode (escape hatch).        |
| `NOVA_AUTOARM=false`      | same as `=0`.                                                  |
| `NOVA_AUTOARM=no`         | same as `=0`.                                                  |
| `NOVA_AUTOARM=1`          | explicit auto-arm enable (same as default).                    |
| `NOVA_AUTOARM=true`       | same as `=1`.                                                  |
| `NOVA_AUTOARM=yes`        | same as `=1`.                                                  |
| `NOVA_AUTOARM=<garbage>`  | treat as default (auto-arm enabled); ignore.                   |
