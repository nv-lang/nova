## Ф.1 — `#cfg` predicates `any/all/not` (Rust-style)

**Сейчас:** `#cfg(feature = "X")` / `#cfg(target_os = "Y")` — только
простой predicate, без композиции.

**Цель:** `#cfg(any(target_os = "linux", target_os = "macos"))`,
`#cfg(all(...))`, `#cfg(not(...))` — Rust-style функциональная форма.
Выбрана Rust-форма (не Go `&&/||/!`) — однозначность приоритетов,
единообразие с `#cfg(...)` attribute syntax, расширяемость вложением.

**Реализация:**
- `ast/mod.rs`: `CfgPredicate` → recursive enum:
  ```rust
  enum CfgPredicate {
      Feature(String),
      TargetOs(String),
      Any(Vec<CfgPredicate>),
      All(Vec<CfgPredicate>),
      Not(Box<CfgPredicate>),
  }
  ```
- `parser/mod.rs`: recursive `parse_cfg_predicate()` — `any(`/`all(`/`not(`
  открывают вложенный список.
- `imports.rs::cfg_active`: рекурсивный eval predicate.

**Acceptance:** `#cfg(any(target_os = "linux", target_os = "macos"))`
active на linux И macos. `not(feature = "x")` active когда x off.

## Ф.2 — Item-level `#cfg`

**Сейчас:** `#cfg` только на module-level (peer-file). Целая функция
OS-specific требует отдельного peer-файла.

**Цель:** `#cfg(...)` перед top-level item (Fn/Type/Const).

**Реализация:**
- `ast/mod.rs`: `Item` варианты получают `cfg: Option<CfgPredicate>`
  ИЛИ wrapper `Item::Cfg(CfgPredicate, Box<Item>)`. Выбор: добавить
  `attrs: Vec<ItemAttr>` к FnDecl/TypeDecl/ConstDecl (consistent с
  будущим item-attrs).
- `parser/mod.rs`: парсить `#cfg(...)` в `parse_item` перед item.
- `imports.rs` или codegen pre-pass: фильтровать items с inactive cfg
  ДО merge / emit.

**Acceptance:** `#cfg(target_os = "windows")` перед `fn` — функция
emit'ится только на Windows.

## Ф.3 — [M11] Rule A: cycle detection по module-name

**Сейчас:** `in_progress`/`visited` keyed by canonical `PathBuf`.
Symlink / case-insensitive FS → разные пути для same module →
false-negative.

**Цель:** keyed by declared module name (`Vec<String>` — parent.X).

**Реализация:**
- `imports.rs::resolve_one`: парсить declared module name из peer
  ДО canonicalize. Использовать `Vec<String>` (declared name) как
  ключ `in_progress`/`visited` вместо PathBuf.
- Diamond-dep dedup сохраняется (same module name = same key).
- Edge: single-file legacy modules — declared name тоже работает.

**Риск:** меняет core resolution logic. Mitigation — folder_cycle
тест (42.08) + полная регрессия.

## Ф.4 — 42.09 selective filter strict — ⏭️ RE-DEFER → Plan 42.15

**Статус:** honest re-defer. Реализация требует NameResCtx per-import
visible scope (codegen видит всё для completeness, type-checker —
только selective list). Это тот же refactor что Ф.6 — выделено в
[Plan 42.15](42.15-per-peer-import-isolation.md). Документировано
в simplifications.md [M12]. Сейчас selective list влияет только на
rename (`A as B`), не на visibility.

**Сейчас:** `import X.{A}` парсится, rename работает, но items НЕ в
selective list всё равно merge'атся (backward compat).

**Цель:** items не в `{...}` списке — НЕ видны импортирующему.

**Реализация:**
- `imports.rs`: в merge loop — если `selective_names` задан и item
  не в нём → **skip** (не push в merged_items).
- Caveat: транзитивные зависимости. Если `A` использует `B` (тоже из
  X, но не в selective list) — `B` всё равно нужен для codegen.
  Решение: skip только из **visible scope**, но keep для codegen —
  ИЛИ keep all + mark visibility. Bootstrap: keep all в merged_items,
  но type-checker NameResCtx фильтрует selective. → перенести filter
  в NameResCtx (per-import visible set).

**Acceptance:** `import X.{A}` — `A` доступна, `B` (тоже в X) →
undefined identifier.

## Ф.5 — Rule H actual fix

**Сейчас:** Rule H (`internal/` boundary) сравнивает **import path**
prefix. Spec говорит про **declared module name**. Обычно совпадают,
но при re-export могут разойтись.

**Цель:** сравнение по declared module name (D29 rev-3.1 — теперь
`owner.internal.target` содержит owner explicitly).

**Реализация:**
- `imports.rs`: Rule H check использует declared module name peer'ов
  `internal/` (теперь `owner.internal.X` благодаря Plan 42.13).
  Importing module's declared name должен иметь `owner` как prefix.
- Проще чем раньше: rev-3.1 уже встроил owner в declaration.

**Acceptance:** `internal/` доступен из `<owner>.*`; re-export через
другой path не обходит boundary.

## Ф.6 — [M10] Rule C Path-form enforcement — ⏭️ RE-DEFER → Plan 42.15

**Статус:** honest re-defer. Оценка после Ф.1-Ф.5: полный enforcement
= NameResCtx per-peer visible scope + Path-form head check (~300-500
LOC, high risk). Имеет **общий корень** с Ф.4 — нерационально дважды
трогать NameResCtx. Выделено в [Plan 42.15](42.15-per-peer-import-isolation.md)
единым refactor'ом. Документировано в simplifications.md [M10].

**Сейчас:** Rule C (peers share declarations, NOT imports) — bare
`Ident` проверяется per-peer, но:
- Path-form (`alias.func()`) не проверяется.
- `module.items` flat → imported items одного peer видны другим.

**Цель:** полный per-peer import isolation.

**Реализация (самая объёмная):**
- `imports.rs`: НЕ merge'ить imported items в flat `module.items`.
  Вместо — каждый `PeerFile.imported_items: Vec<Item>` отдельно.
- Codegen: читать union(all peer_files items) для emit, но
  type-checker — per-peer scope.
- `types/mod.rs NameResCtx`: проверять Path-form head-segment против
  per-peer module_names (не только bare Ident).
- Backward compat: codegen flat view сохраняется через accessor.

**Риск:** высокий — затрагивает resolver + codegen + type-checker.
**Mitigation:** делать последней, после Ф.1-5 стабильны. Полная
регрессия. Если ломает >5 тестов нетривиально — откатить в отдельный
under-plan 42.14.6 и пометить snova deferred.
