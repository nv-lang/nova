# Plan 42.10: Module-level `#forbid` / `#requires` (правило I)

> **Создан 2026-05-14, ЗАКРЫТ 2026-05-14.**
>
> **СТАТУС:** ✅ **MVP реализован** через `_module.nv` convention
> (вариант A из плана). Attrs пропагируются через `inherited_attrs` в
> resolve_one и applied ко всему compiled module через CapabilityCtx.
> `#requires` не реализован (отвергнут в комментариях парсера —
> implicit effects в signatures противоречат D62).
>
> **Приоритет:** MEDIUM. Расширяет file-level #forbid (Plan 42.01) на
> module-level scope.

---

## Зачем

Сейчас (после Plan 42.01) `#forbid Net` декларируется per-peer. Если
модуль имеет 5 peers и все должны быть Net-clean, нужно копировать
`#forbid Net` в каждый peer-файл. Это:
- Boilerplate (5 копий)
- Refactor risk (один peer добавлен — забыт `#forbid`)
- Не отражает intent («модуль admin полностью без Net», а не «каждый
  peer независимо»)

**Решение:** один peer (convention: `_module.nv` или `_meta.nv`)
декларирует `#forbid Net` на module-level — наследуется всеми peers.

---

## Архитектурное решение

```
src/admin/
├── _module.nv      // #forbid Net  (module-level)
├── users.nv        // inherits #forbid Net
├── audit.nv        // inherits #forbid Net
└── helpers.nv      // inherits #forbid Net
```

`_module.nv` — special peer convention (по аналогии с Go `doc.go`):
- Содержит ТОЛЬКО `module X` declaration + module-level attributes.
- Никаких items (опционально discourage parser'ом).
- `#forbid`/`#requires` здесь наследуется всеми peers.

**Альтернатива:** `#module #forbid Net` attribute prefix в любом peer.
Менее clean — два механизма (file vs module). Стоит обсудить.

---

## Decisions to make

1. **Mechanism:** `_module.nv` convention vs `#module #forbid` attribute.
2. **`#requires`:** symmetric с `#forbid`? Spec говорит yes.
3. **Test peer inheritance:** `_test.nv` peers тоже наследуют module-level
   forbid? **Yes** — security boundary должна работать в тестах тоже
   (это уже работает на file-level).
4. **Conflict с file-level:** `_module.nv` forbids X, но `peer.nv`
   делает `#requires X` — compile error (contradiction) или allow
   (peer override)?
