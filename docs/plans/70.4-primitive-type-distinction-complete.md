## Inventory (–§.0 ‚ÄĒ confirmed via emit_c.rs:2820-2835)

### –§.1 ‚ÄĒ f32/f64 array distinction (P2, was original 70.4 scope)

Scalar level ‚ÄĒ already distinct (`nova_f32`/`nova_f64` distinct C types).
Array path collapses:

```rust
"f64" | "f32" => "NovaArray_nova_f64*".into(),  // BOTH f32 AND f64 ‚Üí f64 array
```

ABI mismatch (4-byte vs 8-byte —ć–Ľ–Ķ–ľ–Ķ–Ĺ—ā—č) ‚Üí silent data corruption –Ķ—Ā–Ľ–ł
`[]f32` –ĺ–Ī—Ä–į–Ī–į—ā—č–≤–į–Ķ—ā—Ā—Ź –ļ–į–ļ `[]f64` —á–Ķ—Ä–Ķ–∑ FFI –ł–Ľ–ł unsafe cast.

### –§.2 ‚ÄĒ Sized-int array distinction (P1, ABI-real, **highest impact**)

Scalar level distinct (`int32_t`, `int16_t`, etc), but array fallback
collapses ALL sized-int variants:

```rust
_ => "NovaArray_nova_int*".into(),  // catch-all
```

Affected: `[]i8`, `[]i16`, `[]i32`, `[]i64`, `[]u16`, `[]u32`, `[]u64`
‚ÄĒ **8 types**, –≤—Ā–Ķ silently emit `NovaArray<int64_t>*` –≤–ľ–Ķ—Ā—ā–ĺ packed
sized array.

Concrete impact:
- `[]i32` –Ī—É—Ą–Ķ—Ä –ī–Ľ—Ź GPU/audio/network ‚Üí 2x memory overhead + alignment shift
- `[]u32` –ī–Ľ—Ź color/pixel arrays ‚Üí wrong wire format
- FFI –ļ C-libraries –ĺ–∂–ł–ī–į—é—Č–ł–ľ `int32_t buf[N]` ‚Üí silent failure

**SAME bug-class –ļ–į–ļ Plan 70.3 char/int** ‚ÄĒ discovered –ĺ—ā–ī–Ķ–Ľ—Ć–Ĺ–ĺ –Ņ–ĺ—ā–ĺ–ľ—É
—á—ā–ĺ Plan 70.3 audit fokus'–Ĺ—É–Ľ—Ā—Ź –Ĺ–į char-specific.

### –§.3 ‚ÄĒ int/i64 cosmetic alias

Scalar level both ‚Üí `nova_int`. –Ě–Ķ silent miscompilation (semantically
same int64 type), –Ĺ–ĺ Nova-level `Map[int, V]` ‚Č° `Map[i64, V]` mangle
identical ‚Üí cannot type-checker reject swap.

**Decision (proposed):** Document `int` as **alias** `i64` –≤ spec D-block.
Don't introduce distinct mangling ‚ÄĒ —ć—ā–ĺ –Ī—č –∑–į—Ā—ā–į–≤–ł–Ľ–ĺ user code
explicitly choose `int` –ł–Ľ–ł `i64` –ī–Ľ—Ź –ļ–į–∂–ī–ĺ–≥–ĺ generic instantiation,
—á—ā–ĺ hostile. Modern parallels:
- Rust: `isize` distinct from `i64` (platform-pointer width)
- Go: `int` distinct from `int64` (platform-pointer width)
- Nova bootstrap: `int` = `i64` (no platform variation ‚ÄĒ keep alias)

### –§.4 ‚ÄĒ byte/u8 final cleanup (Plan 69 closure leftover)

Plan 68/69 declared ¬ęcanonical name ‚ÄĒ u8¬Ľ, –Ĺ–ĺ codegen –≤—Ā—Ď –Ķ—Č—Ď –ł–ľ–Ķ–Ķ—ā
–ī–≤–į **—Ä–į–∑–Ĺ—č—Ö** mappings:

```rust
"byte" => Ok("nova_byte".into()),  // 8-byte struct typedef (legacy)
"u8"   => Ok("uint8_t".into()),    // canonical
```

`nova_byte` and `uint8_t` are different C type names (both 1-byte
underlying). Generic mono mangle `[]byte` ‚Č† `[]u8` ‚ÄĒ collapse legacy.

**Decision:** Remove `"byte"` mapping, route –ļ `"u8"` via type-alias
resolution. Existing user code using `byte` (–Ķ—Ā–Ľ–ł –Ķ—Ā—ā—Ć) ‚ÄĒ automated
migration tool.
