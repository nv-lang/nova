## 1. Контекст

V5.2 emit'нул semantic tokens через `textDocument/semanticTokens/full`.
Каждый request пересчитывает полный token-set, что для активной
LSP session с typing-edits даёт hundreds of full responses в minute
с в основном идентичными tokens. LSP spec 3.16+ имеет
`semanticTokens/full/delta` request type — client passes back
`previous_result_id` от прошлого ответа, server returns an **edit
script** transforming previous tokens → new tokens.

V5.5 имплементит этот protocol: capability advertisement, per-URI
snapshot cache, monotonic `result_id` allocator, минималистичный
single-edit prefix-suffix algorithm, decision helper для
match/fallback логики, и server handler.

## 2. Изменения

### 2.1 `nova-lsp/src/semantic_tokens_delta.rs` (NEW, ~250 LOC)

Pure module — no IO, no runtime dependency:

- `SemanticTokensSnapshot { result_id: String, tokens: Vec<SemanticToken> }`
  — per-URI cache entry.
- `compute_semantic_token_edits(old, new) -> Vec<SemanticTokensEdit>`
  — single-edit prefix-suffix reduction.
- `build_delta_response(prev_snapshot, prev_result_id, new_tokens,
  new_result_id) -> (SemanticTokensFullDeltaResult, SemanticTokensSnapshot)`
  — decision-and-snapshot-update helper.
- `semantic_tokens_equal(a, b) -> bool` — internal equality predicate.
- 14 unit tests covering algorithm + decision helper.

### 2.2 `nova-lsp/src/state.rs`

- `WorkspaceState` gains two fields:
  - `pub semantic_tokens_cache: DashMap<Url, SemanticTokensSnapshot>`
  - `pub semantic_tokens_counter: AtomicU64`
- `WorkspaceState::Default` initializes both к sensible defaults.
- `WorkspaceState::next_semantic_tokens_result_id() -> String` —
  monotonic allocator, format `st-<N>`.

### 2.3 `nova-lsp/src/server.rs`

- Capability changed: `full: SemanticTokensFullOptions::Bool(true)`
  → `Delta { delta: Some(true) }`.
- `semantic_tokens_full` handler — assigns new `result_id`, inserts
  snapshot, returns tokens with id.
- NEW handler `semantic_tokens_full_delta` — calls
  `build_delta_response`, inserts updated snapshot, returns response.

### 2.4 `nova-lsp/src/lib.rs`

- `pub mod semantic_tokens_delta;` declaration.

## 4. Не в скоупе

- **True LCS multi-edit algorithm** — current single-edit is
  wire-equivalent or better for typical localized edits; LCS
  benefits realize только при interleaved edits which редки в
  practical LSP sessions. Followup V5.5.1.
- **Snapshot eviction policy** — DashMap grows monotonically. OK
  для typical workspace sizes; long-running server sessions могут
  нуждаться в LRU cap. Followup V5.5.2.
- **Range-based semantic-tokens delta** (`semanticTokens/range`) —
  out of V5.5 scope; range provider stays disabled.
