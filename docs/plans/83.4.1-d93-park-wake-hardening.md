## 1. Bug, который закрываем (A1)

**Симптом** (`sleep_bench` / `sleep_precision_bench` / `sleep_real_clock`,
под активным M:N-флипом):
```
nova: FATAL sleep wake before close_cb (stage=0) — D93 protocol bug
```

**Локация:** `compiler-codegen/nova_rt/fibers.h:1552-1557` (sleep-path) +
`:1640+` (blocking-path с тем же паттерном) + любой park-site, который
проверяет post-condition после `nova_sched_park`.

## 2. Root cause — race в park-semantics под M:N

Текущий `nova_sched_park(scope, slot)` (`sched.h`) возвращается ЛИБО:
1. По явному `nova_sched_wake(scope, slot)` (callback inicia).
2. По достижению quiescence в drain-loop (например `pending_remote == 0`).

D93-протокол sleep'а полагается на (1) **строго**: `timer_cb` запускает
`close_cb` асинхронно; реальный wake должен прийти ИЗ `close_cb` (когда
handle уже `CLOSED`).

Под M:N drain-loop может удовлетворить (2) ПОСЛЕ того как `timer_cb`
уведомил scheduler через `pending_remote`-counter (timer's part of
done), но ДО того как `close_cb` фактически отработал. Park возвращает
управление, fiber resume'ится с `stage == OPEN`, FATAL-check срабатывает.

**Это spurious wake**, классическая проблема park/wake. Решение —
индустриальный паттерн **park-with-predicate**.

## 3. Cross-runtime reference

| Runtime | Primitive | Семантика |
|---|---|---|
| **POSIX** | `pthread_cond_wait(cv, mu)` + caller-loop `while (!pred) cond_wait(...)` | Spurious wake handled explicitly в caller'е |
| **C++** | `std::condition_variable::wait(lock, pred)` — predicate-overload | Один call: wait сам loop'ит до pred() == true |
| **Go runtime** | `gopark(unlockf, lock, reason, traceEv, traceskip)` | `unlockf(g, lock)` — атомарный «park-or-not» predicate: если вернул false, goroutine остаётся runnable |
| **tokio** | `Notify` + `Notified` future — `notified().await` | atomic state с PERMIT/WAKER/CONSUMED bits, spurious wakeup невозможен by construction |
| **Loom** (Rust race-detector) | exposes `park` + `unpark` с честной spurious-wake model | для верификации lock-free алгоритмов |

**Общий принцип:** park НИКОГДА не должен возвращаться, если предикат
не выполнен — либо caller-loop, либо встроенный predicate-overload.
Spurious wakeup — реальность любой M:N runtime.

## 4. Production-grade подход

### 4.1. API extension — `nova_sched_park_until`

Новый primitive в `compiler-codegen/nova_rt/sched.h` (header-only):

```c
/* Predicate, evaluated under appropriate ordering by scheduler.
 * Returns true → park returns immediately (or doesn't park at all).
 * Returns false → fiber stays parked, wait for next wake signal. */
typedef nova_bool (*NovaParkPredicate)(void* ctx);

/* Park fiber until predicate returns true. Handles spurious wakeups
 * by re-checking predicate after each wake. Fast-path: if predicate
 * already true on entry, returns without parking.
 *
 * Memory ordering: predicate is called with ACQUIRE-semantics relative
 * to the most recent wake (or to the call site if no wake yet — initial
 * fast-path). Callers must publish state with RELEASE before signalling
 * wake via nova_sched_wake.
 *
 * Cancellation: park returns also if scope->cancel_requested becomes
 * true (caller checks separately; see 83.4.3 cancel hierarchy). */
void nova_sched_park_until(NovaFiberQueue* scope, int slot,
                           NovaParkPredicate pred, void* ctx);
```

Существующий `nova_sched_park(scope, slot)` остаётся, но переоформляется
как тонкий wrapper:
```c
static inline void nova_sched_park(NovaFiberQueue* scope, int slot) {
    nova_sched_park_until(scope, slot, NULL, NULL);  /* legacy: park-once */
}
```
Pred == NULL → legacy «park, wake-once» поведение (без re-check). Это
требуется только для сайтов, где callers внешний loop уже делает
predicate-check (например, drain). Все НОВЫЕ park-сайты ОБЯЗАНЫ
использовать `_until` с явным предикатом.

### 4.2. Реализация (внутри `_until`)

```c
void nova_sched_park_until(NovaFiberQueue* scope, int slot,
                           NovaParkPredicate pred, void* ctx) {
    if (pred && pred(ctx)) return;       /* fast-path, no park */
    for (;;) {
        nova_sched_park(scope, slot);    /* primitive park */
        if (!pred) return;               /* legacy single-shot */
        if (pred(ctx)) return;           /* predicate satisfied */
        /* else: spurious wake, re-park */
    }
}
```

### 4.3. Memory ordering audit

Per pred-call site обязан:
- Writer (releaser): published-state writes ДО `nova_sched_wake` с
  RELEASE-ordering (например `nova_abool_store(&st->stage_closed, true, RELEASE)`).
- Reader (predicate-fn): атомарный load с ACQUIRE-ordering
  (`nova_abool_load(&st->stage_closed, ACQUIRE)`).
- Park-instance: `nova_sched_park` сам выполняет mco_yield, который
  flush'ит регистры и работает как compiler barrier. Между park-сайтом
  и предикат-функцией нет state-shared без atomics.

### 4.4. Sleep-callsite refactor (sleep_bench fix)

В `_nova_sleep_park`:
```c
static nova_bool _sleep_stage_closed(void* ctx) {
    SleepState* st = (SleepState*)ctx;
    return nova_aint_load(&st->stage) == NOVA_SLEEP_CLOSED;
}

/* В sleep-fn: */
nova_aint_init(&st.stage, NOVA_SLEEP_OPEN);
/* ... timer setup ... */
nova_sched_park_until(scope, slot, _sleep_stage_closed, &st);
/* После возврата: stage GUARANTEED == CLOSED — fatal-check можно
 * оставить как debug-assert либо удалить. */
```

State поле `stage` — `nova_atomic_int` (было plain int). Writer
(`close_cb`): `nova_aint_store(&st->stage, NOVA_SLEEP_CLOSED, RELEASE)`
+ `nova_sched_wake(scope, slot)`.

### 4.5. Blocking-callsite refactor

Аналогично `_nova_blocking_park` (`fibers.h:~1640`).

### 4.6. Audit прочих park-сайтов

Перечислить и оценить ВСЕ park-сайты:
- `Channel.send/recv` (в `channels.h`) — park-condition «slot free /
  message available». Уже использует похожий паттерн под Plan 44.1
  audit, но проверить consistency.
- `supervised_run drain` (`fibers.h::nova_supervised_drain_main_scope`)
  — park-condition «все fibers DONE». Не требует extension, drain
  loop сам re-checks.
- `WaitGroup.wait` / `Once.run` — park на counter / state.
- Любые future park-сайты обязаны использовать `_until`.

Acceptance: каждый park-сайт либо использует `_until` с pred, либо
оправдан в комментарии (drain-loop self-rechecks).

## 7. Riski

- **Refactor затрагивает hot park-path** — каждый `park_until` теперь
  делает PRED-call (cold function pointer indirect). Mitigation:
  inline-функция `_until`, hot fast-path inline'ится; pred-fn cost
  amortized over real wait time.
- **Существующий callers с post-park-check могут регрессировать,
  если NULL predicate path не идентичен legacy `nova_sched_park`**.
  Mitigation: NULL-pred branch ровно `nova_sched_park(scope, slot)`
  без loop — старая семантика byte-identical.
- **TSAN на текущем repo может вскрыть pre-existing races НЕ из этого
  plan'а.** Discover-and-document: pre-existing TSAN findings →
  `simplifications.md` как followup, НЕ блокируют 83.4.1.
