## 1. Variant decision: Variant A (workers keep minimal UV loop)

**Decision:** **Variant A**. Workers retain tiny UV loop with one `uv_async_t`
wake handle (= current architecture's `wake_handle`). Driver thread gets one
dedicated UV loop with ALL timers + blocking-pool after_cb + I/O events.

**Rationale:**
- Worker already has `uv_async_t wake_handle` для cross-thread dispatch_ready.
  Variant A reuses this — никакой замены на condvar/Windows event.
- Variant B (condvar workers) require larger refactor — 2 wake mechanisms,
  more code, more bugs.
- Bench-wise Variant A has slight overhead (worker's tiny UV loop spinning
  один handle) but proven path.
- Tokio uses TryLock<Driver> + Condvar fallback (Parker pattern) — но they need это
  потому что нет dedicated driver thread в blocking-only mode. У нас driver
  всегда есть → simpler.

**Trade-off accepted:** worker UV loop остаётся в `_worker_main`, но БЕЗ
own timers/I/O. Минимальный set of handles per worker: только `wake_handle` +
maybe close_queue for deferred close (от current architecture).

## 2. NovaSleepState packed_state encoding — port Tokio TimerEntry

**State enum (8-bit, fits в lower byte of u64 for future expansion):**

```c
typedef enum {
    NOVA_SLEEP_NEW         = 0,  /* allocated, not yet armed на driver loop  */
    NOVA_SLEEP_ARMED       = 1,  /* uv_timer_t init+start, в scope.armed_list */
    NOVA_SLEEP_FIRING      = 2,  /* timer_cb won CAS, uv_close in flight     */
    NOVA_SLEEP_CANCEL_REQ  = 3,  /* cancel job won CAS, uv_close in flight   */
    NOVA_SLEEP_CLOSED      = 4,  /* close_cb fired, safe to wake fiber       */
} NovaSleepStage;

typedef struct {
    /* Identity для wake routing */
    NovaFiberQueue*  scope;             /* worker's scope (для nova_sched_wake) */
    int              slot;              /* worker slot index                    */
    int              home_worker_id;    /* driver wakes via this worker         */
    NovaFiberQueue*  cancel_scope;      /* supervised scope (parent), для cancel */
    /* I/O state — driver-only mutator */
    uv_timer_t       timer;             /* lives on driver_loop                 */
    /* State machine — atomic transitions (port Tokio TimerEntry pattern) */
    nova_atomic_int  packed_state;      /* NovaSleepStage в lower byte          */
    /* Linked-list in scope.armed_list (driver-only mutator) */
    struct NovaSleepState* next_in_scope;
    struct NovaSleepState** pprev_in_scope;  /* для O(1) removal */
} NovaSleepState;
```

**Transitions — все через CAS, single-thread CAS for cancel side, cross-thread
read for park-predicate:**

```
NEW
 └→ ARMED         (driver-thread, after uv_timer_init + uv_timer_start succeeded)

ARMED
 ├→ FIRING        (driver-thread, in timer_cb — CAS ARMED→FIRING. CAS-loser = no-op.)
 └→ CANCEL_REQ    (driver-thread, in cancel-scope handler — CAS ARMED→CANCEL_REQ.
                   CAS-loser = timer already firing/closing, no-op.)

FIRING / CANCEL_REQ
 └→ CLOSED        (driver-thread, in close_cb — Release-store CLOSED)

CLOSED
 └→ (terminal — worker park-predicate Acquire-loads CLOSED, exits park_until)
```

**Memory orderings (port Tokio TimerEntry):**
- CAS transitions: `compare_exchange_weak(AcqRel success, Acquire fail)` — same as
  Tokio's mark_pending / extend_expiration
- close_cb's store CLOSED: `__ATOMIC_RELEASE` — pairs with worker's ACQUIRE in predicate
- Worker predicate load: `__ATOMIC_ACQUIRE` — pairs with close_cb's release
- **No SeqCst.** Tokio doesn't use it here either — AcqRel + driver-thread mutator
  invariant is sufficient.

**Why this eliminates the race:**
- **Single mutator (driver thread)** for all writes to packed_state EXCEPT the
  final close_cb store. close_cb также runs on driver thread (we register с
  driver's UV loop). So **all writes from one thread**.
- Worker reads via ACQUIRE — sees release-store via happens-before.
- No three-load race possible — predicate reads ONE atomic (packed_state).

## 3. Driver thread structure

```c
/* compiler-codegen/nova_rt/driver.h */
typedef struct {
    uv_loop_t          loop;               /* dedicated UV loop                  */
    uv_thread_t        thread;
    uv_async_t         job_async;          /* worker→driver job submission wake */
    uv_async_t         shutdown_async;     /* shutdown signal                    */
    NovaDriverJobQueue jobs;               /* MPSC mutex+linked-list queue       */
    nova_atomic_bool   stop;               /* shutdown flag                      */
    nova_atomic_bool   started;            /* init completed flag                */
} NovaDriver;

extern NovaDriver _nova_driver;

void nova_driver_init(void);
void nova_driver_shutdown(void);
int  nova_driver_submit_job(NovaDriverJob job);  /* returns 0 on success */
```

**Driver main loop (port Tokio's Driver::park pattern):**
```c
static void _nova_driver_main(void* arg) {
    (void)arg;
    /* Register с Boehm GC (если нужно — driver сам не алocates GC objects,
     * но может вызывать callbacks которые могут).
     * Decision: NO Boehm registration — driver кода НЕ должен touch GC heap.
     * Worker callbacks делают GC ops; driver просто dispatches. */
    while (!nova_abool_load(&_nova_driver.stop)) {
        /* uv_run blocks until any handle fires (timer/async/etc).
         * UV_RUN_ONCE — process one batch then return. */
        uv_run(&_nova_driver.loop, UV_RUN_ONCE);
        /* После каждой итерации drain job queue.
         * Также drained via job_async callback — это backup. */
        _nova_driver_drain_jobs();
    }
    /* Shutdown sequence: close all handles, run loop until clean. */
    _nova_driver_shutdown_handles();
    while (uv_loop_alive(&_nova_driver.loop)) {
        uv_run(&_nova_driver.loop, UV_RUN_NOWAIT);
    }
    uv_loop_close(&_nova_driver.loop);
}
```

**Job submission (worker→driver):**
```c
typedef enum {
    NOVA_DRV_JOB_ARM_SLEEP,
    NOVA_DRV_JOB_CANCEL_SCOPE,
    NOVA_DRV_JOB_CANCEL_TIMER,        /* one-off cancel (для linked-tokens) */
    NOVA_DRV_JOB_ARM_BLOCKING,
    NOVA_DRV_JOB_SHUTDOWN,
} NovaDriverJobKind;

typedef struct NovaDriverJob {
    NovaDriverJobKind kind;
    union {
        struct { NovaSleepState* st; uint64_t ms; } arm_sleep;
        struct { NovaFiberQueue* scope; } cancel_scope;
        struct { NovaSleepState* st; } cancel_timer;
        struct { NovaBlockingState* st; void (*work)(void*); void* arg; } arm_blocking;
    };
    struct NovaDriverJob* next;
} NovaDriverJob;

int nova_driver_submit_job(NovaDriverJob* job) {
    nova_mutex_lock(&_nova_driver.jobs.mu);
    job->next = NULL;
    if (_nova_driver.jobs.tail) _nova_driver.jobs.tail->next = job;
    else _nova_driver.jobs.head = job;
    _nova_driver.jobs.tail = job;
    nova_mutex_unlock(&_nova_driver.jobs.mu);
    uv_async_send(&_nova_driver.job_async);
    return 0;
}
```

**Note:** mutex+linked-list is simpler than lock-free queue. Driver has 1 reader
+ N writers (MPSC). Tokio uses crossbeam queue, мы используем mutex —
fine для V1, can optimize later if profiling shows contention.

## 4. Cancel semantics

**Worker fiber calls `tok.cancel()`:**
1. `nova_cancel_token_cancel_reason` (existing) → атомарно set `cancel_requested`
2. **NEW:** submit `NOVA_DRV_JOB_CANCEL_SCOPE` to driver
3. Driver receives job → walks `scope.armed_list` → для каждого NovaSleepState:
   - CAS packed_state ARMED→CANCEL_REQ
   - Если CAS won → uv_close(timer, close_cb)
   - Если CAS lost → timer уже firing/closed, no-op
4. close_cb fires (на driver thread) → store CLOSED → cross-thread wake worker fiber

**Race elimination:**
- Driver thread is SINGLE MUTATOR for armed_list AND packed_state CAS transitions.
- No race because no concurrent writes. Two writers (cancel job vs timer_cb) both
  driver-thread → serialized by uv_run.
- Worker read via ACQUIRE sees Release-store CLOSED.

## 5. Worker idle behavior (Variant A)

Worker's `_worker_main` loop unchanged structurally:
```c
while (!stop) {
    uv_run(&w->loop, UV_RUN_NOWAIT);       /* drain async callbacks (wake events) */
    drain wake_pending;                     /* cross-thread queue → deque */
    co = pop runnext / deque / steal;
    if (!co) uv_run(&w->loop, UV_RUN_ONCE); /* block until wake_handle fires */
    else { run fiber; ... }
}
```

The ONLY UV handle на worker's loop остаётся `wake_handle` (uv_async_t). Plus
`close_queue` (for deferred close from old code — TBD remove if unused after Ф.5).

**Worker fiber timer-arm sequence:**
```c
static inline void _nova_sleep_via_driver(NovaFiberQueue* scope, int slot, nova_int ms) {
    NovaFiberQueue* cancel_scope = /* derive parent scope from base->_nova_parent_scope */;
    if (nova_abool_load(&cancel_scope->cancel_requested)) return;  /* Race 2a — still useful */

    NovaSleepState st = {
        .scope = scope,
        .slot = slot,
        .home_worker_id = _current_worker_id,
        .cancel_scope = cancel_scope,
    };
    nova_aint_init(&st.packed_state, NOVA_SLEEP_NEW);

    /* Submit arm job to driver (async). Driver inits timer на своём loop'е. */
    NovaDriverJob* job = nova_alloc(sizeof(NovaDriverJob));
    job->kind = NOVA_DRV_JOB_ARM_SLEEP;
    job->arm_sleep.st = &st;
    job->arm_sleep.ms = (uint64_t)ms;
    nova_driver_submit_job(job);

    /* Park-until predicate Acquire-loads packed_state. */
    nova_sched_park_until(scope, slot, _nova_sleep_state_is_closed, &st);

    /* После CLOSED — st gets unlinked from armed_list by close_cb. */
}

static nova_bool _nova_sleep_state_is_closed(void* ctx) {
    NovaSleepState* st = (NovaSleepState*)ctx;
    return nova_aint_load(&st->packed_state) == NOVA_SLEEP_CLOSED;
}
```

**Note:** `NovaSleepState st` живёт на fiber's coroutine stack. Driver dereferences
the pointer until close_cb fires + wake delivered. Since fiber is parked while
this happens, stack stays alive. After park_until exits (sees CLOSED), fiber
returns from _nova_sleep_via_driver и st becomes invalid — but driver уже
finished with it (CLOSED state означает close_cb already ran, st unlinked).

## 6. Performance considerations

**Driver as single mutator — bottleneck risk:**
- Tokio mitigates через timer wheel (O(1) insert/remove). Мы используем libuv
  binary heap (O(log n) insert/remove). Acceptable.
- Job queue MPSC mutex: 1 mutex acquire per arm/cancel. At 100k ops/sec contention
  manageable. If profiling shows hot — switch to lock-free SPSC per worker.
- Driver thread context switch overhead: 1 extra thread but no per-worker UV loop
  init cost (savings).

**Home-worker hint:** wake routed to `st->home_worker_id` — fiber resumes on
same worker that armed timer (cache-warm). Если worker busy, fiber waits в
deque — eventually picked up. Tokio uses similar hint в `Shared::schedule_local`.

**Idle CPU:** driver in `uv_run(UV_RUN_ONCE)` blocks until any handle fires (no
spinning). Worker blocked в `uv_run(UV_RUN_ONCE)` similar. No idle CPU burn.

## 7. Spec D-block sketches

**D-NNN-A: Centralized I/O driver topology**
> Runtime Component: NovaDriver — dedicated thread с одним uv_loop_t, owning ALL
> timer instances, blocking-pool after_work_cb, future I/O event sources (sockets,
> file watchers etc). Worker threads (NovaWorker[]) maintain only computational
> state (deque, sched_state, minimal wake handle) — НЕ владеют UV loops с user-
> visible timers/handles.
>
> Cross-thread dispatch: workers submit `NovaDriverJob` via lock-protected MPSC
> queue + uv_async_send(driver). Driver routes wake-events to workers via existing
> dispatch_ready mechanism (`home_worker_id` hint → cross-thread queue + uv_async_send).
>
> Lifecycle: driver thread started в `nova_runtime_init` (after worker pool
> materialization), stopped в `nova_runtime_shutdown` (before worker threads
> join). Driver itself does NOT register с Boehm GC — it never touches GC heap
> directly; callbacks invoke worker-side state machines which do.

**D-NNN-B: Cancel state-machine — single-thread mutator invariant**
> All transitions on NovaSleepState.packed_state EXCEPT the predicate Acquire-load
> happen on the driver thread. This eliminates cross-thread CAS races by
> construction: at most one writer at any time.
>
> Transition table:
> - NEW → ARMED: driver thread, после uv_timer_init/start success
> - ARMED → FIRING: driver thread, в timer_cb (CAS — loser is concurrent cancel)
> - ARMED → CANCEL_REQ: driver thread, в cancel_scope job handler (CAS — loser is concurrent fire)
> - FIRING|CANCEL_REQ → CLOSED: driver thread, в close_cb (Release store)
>
> Memory ordering: CAS uses AcqRel(success)/Acquire(fail). Final CLOSED store
> uses Release. Worker predicate Acquire-load pairs with Release for visibility.
> Pattern ported verbatim from `tokio/src/runtime/time/entry.rs`.

**D-NNN-C: ε-bounded cancel latency formal guarantee**
> If `tok.cancel()` is called at wall-clock time T:
> - All armed sleep timers in `tok.bound_scope` will be in CANCEL_REQ state by
>   T + ε_dispatch where ε_dispatch ≤ 10ms (driver job queue + uv_async wake).
> - Workers будут notified via cross-thread dispatch by T + ε_dispatch + ε_wake
>   where ε_wake ≤ 10ms (worker uv_async wake + deque pop).
> - Total: cancel latency ≤ 20ms under normal load. Bound enforced via
>   `cancel_latency_bench` regression test.

## 9. Decisions log

| Q | A | Why |
|---|---|-----|
| Variant A or B? | **A** | Less refactor, reuse existing wake_handle pattern |
| Worker UV loop removed entirely? | **No, retain minimal** | Already has wake_handle for cross-thread dispatch |
| Driver Boehm-registered? | **No** | Driver code never touches GC heap; callbacks run elsewhere |
| Lock-free job queue? | **No, mutex MPSC V1** | Profiling first; optimize if hot |
| Backward-compat path `_nova_sleep_via_libuv`? | **Yes, gradual deprecation** | Bootstrap-mode (no driver init) needs fallback |
| Migrate channels/mutexes in same plan? | **Ф.5 evaluates** | If same pattern, yes; if different, separate plan |
| SeqCst on packed_state CAS? | **No, AcqRel + Acquire** | Port Tokio TimerEntry verbatim; single-mutator removes need for SeqCst |
| home_worker_id mandatory? | **Yes** | Required for wake routing; capture at park time |

## 10. Risks identified в design

| Risk | Mitigation |
|------|-----------|
| `NovaSleepState st` на fiber stack — driver dereferences pointer | Safe: fiber parked while driver uses; st unlinked from armed_list before close_cb returns |
| Driver thread panics → all I/O dead | Sysmon detects (TODO: add heartbeat); abort с clear error |
| MPSC mutex contention в high-throughput | V1 acceptable; switch lock-free later если bench shows |
| Bootstrap mode (no runtime.init) needs old `_nova_sleep_via_libuv` | Keep both paths; codegen selects based on `_armed` flag |
| Race 2a (cancel_scope->cancel_requested check) — preserved? | **Yes, keep** — early-skip optimization, не correctness |
