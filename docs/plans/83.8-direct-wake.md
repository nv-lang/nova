# Plan 83.8: Direct wake primitive — eventfd/SetEvent intra-runtime wake

> **Создан:** 2026-05-24 (out of Plan 83 audit, [83-audit-2026-05-24.md](83-audit-2026-05-24.md) §6.4).
> **Статус:** 📋 proposed.
> **Приоритет:** P1.
> **Оценка:** 1 dev-week.
> **Родитель:** [Plan 83](83-mn-default-on-gomaxprocs.md).
> **Зависимости:** Plan 83.5 + 83.6 рекомендуется (alloc bottleneck closed
> → wake bottleneck becomes visible).

---

## 1. Проблема

Текущий cross-worker wake (worker A wakes fiber in worker B):

```c
// runtime.c:302-321
nova_mutex_lock(&w->wake_mu);
nova_array_push(w->wake_pending, co);
nova_mutex_unlock(&w->wake_mu);
uv_async_send(&w->wake_handle);  // libuv path
```

`uv_async_send` под капотом — pthread_mutex_lock + pthread_cond_signal
(POSIX) или PostQueuedCompletionStatus (Windows IOCP). Verified в
[libuv src/unix/async.c](https://github.com/libuv/libuv/blob/master/src/unix/async.c):
- POSIX: 2 mutex ops + cond signal = ~5-20µs typical.
- Windows IOCP: 1 syscall + queue ops = ~3-10µs typical.

**Go runtime:** прямой futex (Linux) / SetEvent (Windows) / kqueue
(macOS) — single atomic operation + worst-case 1 syscall = <1µs typical.

**Tokio:** через mio — epoll/kqueue/IOCP directly. ~1µs.

**Nova: 5-20× медленнее на cross-worker wake.** Для workloads с heavy
inter-worker fiber communication (channel send/recv, supervised drain,
cancel propagation) это significant cumulative overhead.

## 2. Дизайн

**Ключевое решение:** libuv остаётся для I/O events (timer, fs, net).
Только intra-runtime wake (worker-to-worker fiber re-dispatch) заменяется
на direct OS primitive.

### 2.1 Per-worker wake handle

```c
struct NovaWorker {
    /* ... existing fields ... */

    /* Plan 83.8: direct wake — replaces uv_async_t для intra-runtime. */
#if defined(__linux__)
    int wake_fd;          /* eventfd(EFD_NONBLOCK|EFD_CLOEXEC) */
#elif defined(_WIN32)
    HANDLE wake_event;    /* CreateEventA(NULL, FALSE, FALSE, NULL) auto-reset */
#else  /* macOS, BSD */
    int wake_pipe[2];     /* pipe2(O_NONBLOCK|O_CLOEXEC) */
#endif

    uv_poll_t wake_poll;  /* libuv polls wake_fd/event */
};
```

### 2.2 Wake signal (worker A → worker B)

```c
void _worker_dispatch_ready(void* ctx, mco_coro* co) {
    NovaWorker* w = (NovaWorker*)ctx;
    /* Plan 83.7 runnext priority (if exists). */
    /* ... */

    /* Cross-thread push: lock-free SPSC queue либо short critical section. */
    nova_mutex_lock(&w->wake_mu);
    nova_array_push(w->wake_pending, co);
    nova_mutex_unlock(&w->wake_mu);

    /* Plan 83.8: direct wake primitive. */
#if defined(__linux__)
    uint64_t one = 1;
    ssize_t n = write(w->wake_fd, &one, sizeof(one));
    if (n != sizeof(one) && errno != EAGAIN) {
        /* EAGAIN означает counter уже notify-pending (acceptable race). */
        fprintf(stderr, "nova: wake_fd write failed errno=%d\n", errno);
    }
#elif defined(_WIN32)
    SetEvent(w->wake_event);  /* idempotent: signal не lost */
#else
    char c = 1;
    write(w->wake_pipe[1], &c, 1);
#endif
}
```

### 2.3 Wait (worker idle)

В `_worker_main` Step (4) (runtime.c, idle wait):

```c
/* Step 4: idle — wait for work. */
#if defined(__linux__)
    uv_poll_start(&w->wake_poll, UV_READABLE, _wake_poll_cb);
    uv_run(&w->loop, UV_RUN_ONCE);
    uv_poll_stop(&w->wake_poll);
    /* Drain eventfd counter. */
    uint64_t drain;
    while (read(w->wake_fd, &drain, sizeof(drain)) > 0) { /* loop drain */ }
#elif defined(_WIN32)
    /* libuv has uv_check_t/UV_RUN_ONCE timeout — but we want direct
     * WaitForSingleObject + libuv. Solution: register HANDLE с
     * uv_loop через uv_poll_t (libuv supports this on Windows). */
    uv_poll_start(&w->wake_poll, UV_READABLE, _wake_poll_cb);
    uv_run(&w->loop, UV_RUN_ONCE);
    uv_poll_stop(&w->wake_poll);
#else
    /* Pipe drain similar to eventfd. */
#endif
```

Note: `uv_poll_t` обёртка вокруг fd/handle — Single syscall в epoll/IOCP
context, по сути same primitive что Go использует.

### 2.4 Memory ordering

- `nova_array_push` под mutex — happens-before with `wake_pending`
  consumer.
- `write(wake_fd)` — kernel synchronization, full barrier.
- `SetEvent` — Windows kernel barrier.

Все cross-thread visibility гарантирован kernel-level primitive.

### 2.5 Initialization

```c
/* runtime.c: nova_runtime_init per-worker */
for (int i = 0; i < _n_workers; i++) {
    NovaWorker* w = &_workers[i];
#if defined(__linux__)
    w->wake_fd = eventfd(0, EFD_NONBLOCK | EFD_CLOEXEC);
    if (w->wake_fd < 0) abort();
    uv_poll_init(&w->loop, &w->wake_poll, w->wake_fd);
    w->wake_poll.data = w;
#elif defined(_WIN32)
    w->wake_event = CreateEventA(NULL, FALSE, FALSE, NULL);
    if (!w->wake_event) abort();
    uv_poll_init_socket(&w->loop, &w->wake_poll, (uv_os_sock_t)w->wake_event);
    /* WaitForSingleObject через libuv. */
#else  /* pipe fallback */
    if (pipe2(w->wake_pipe, O_NONBLOCK | O_CLOEXEC) != 0) abort();
    uv_poll_init(&w->loop, &w->wake_poll, w->wake_pipe[0]);
#endif
}
```

### 2.6 Cleanup

В `nova_runtime_shutdown`:
```c
#if defined(__linux__)
    close(w->wake_fd);
#elif defined(_WIN32)
    CloseHandle(w->wake_event);
#else
    close(w->wake_pipe[0]); close(w->wake_pipe[1]);
#endif
```

## 5. Риски

- **uv_poll_t Windows specificity** — Windows IOCP integration с
  `CreateEventA` через `uv_poll_init_socket` требует verification.
  Альтернатива: separate worker thread polling wake_event + push в
  libuv loop. Adds complexity.
- **macOS pipe** — pipe(2) write неблокирующая ≤ PIPE_BUF (4096). При
  >4K parallel wakes — EAGAIN race. Mitigation: 1-byte write fits.
- **eventfd EAGAIN** — counter wraps at 2⁶⁴-1; никогда practical issue.

## 6. Out-of-scope

- Replacement libuv для I/O — Plan 83 решает что libuv I/O остаётся (NIH
  rebuild months work). Только wake-path replaced.
- Sysmon thread wake — sysmon уже uses uv_sleep(10ms) tick, не нужен
  fast wake.
