## Подход: переиспользовать value-records (Plan 124.8), не «новая инфра»
`type X value priv {...}` уже работает полностью (124.8): value-семантика, **privacy `priv`+`E_PRIV_FIELD`** (бесплатно), field-access, методы, sizeof. str = ещё один value-record, ABI-идентичный существующему `nova_str`. Остаётся **тонкая lang-item линковка** + ABI-мост + миграция C-методов + content-eq + neg-фикстуры.

## Фазы (атомарные; production; per-phase commit; pos+neg via релизный nova)

### Ф.A — Декларация + lang-item линковка + ABI-мост (GATE)
- Объявить **`type str value priv { ptr *u8, len int }`** в core-модуле (.nv), prelude-early
  (str нужен в panic/interp ДО своих методов — R2 bootstrap; literal-lowering зависит только от layout).
- **Lang-item линковка:** компилятор резолвит `str`-keyword + строковые литералы на эту декларацию
  (не хардкод-примитив `types/mod.rs`). Через `#lang(str)`-атрибут ИЛИ распознавание имени в known-core.
- **ABI-мост:** C-тип value-record'а str = существующий `nova_str` (`{const uint8_t*; int64_t}`) →
  354 рантайм-сайта + literal-lowering не трогаем (typedef alias).
- **GATE (E1 core):** тривиальная программа компилится RELEASE + 0 регрессий + **privacy fires**
  (`let s="x"; s.ptr` снаружи str-модуля → `E_PRIV_FIELD`). Если не проходит → НЕ закрывать, чинить.
- **Commit:** `feat(plan139.1 Ф.A): str lang-item decl `type str value priv {ptr *u8,len int}` + ABI-alias`.

### Ф.B — Перенести 10 external C-методов str → Nova-body
- Сейчас `external fn str @...` (string.nv): `@len`, `@byte_at`, `@concat`, `@hash`, `@as_bytes`,
  `str.from_bytes_lossy`, `str.from_bytes_unchecked`, `str.from_bytes_unchecked_steal`, `@split`, `@compare`.
- Переписать на **Nova-body** через `@ptr[i]` byte-access (теперь доступен — privacy позволяет in-module).
  Zero-copy где можно (`as_bytes`/`split`/`from_bytes_*` — view над `*u8`). Методы — в отдельном файле.
- **Commit:** `feat(plan139.1 Ф.B): str methods C→Nova-body via @ptr byte-access (10 external retired)`.

### Ф.C — content-eq override (D228) + 3 neg-фикстуры
- content-eq override (str = content-eq, не pointer-eq; D228) — привязать к lang-item str.
- **3 neg-фикстуры (E1 acceptance, ОБЯЗАТЕЛЬНЫ):** `neg_str_priv_field` (`s.ptr` снаружи → `E_PRIV_FIELD`),
  `neg_str_ptr_write` (`*u8` write → `E_POINTER_RO_ASSIGN`, ro-pointee per 147), `neg_str_construct_direct`
  (`str { ptr:…, len:… }` снаружи → privacy/lang-item construction forbidden).
- pos-фикстуры: литералы/`${}`/`+`/byte-access/methods. **Commit:** `feat(plan139.1 Ф.C): str content-eq + privacy neg fixtures`.
