## Идея

Добить методы `str` до паритета прод-языков (см. матрицу umbrella §1), разместив по
слоям; **все позиции/длины — в байтовых координатах** (или явных char через линзу).
Семантика поиска/нарезки переводится с codepoint (школа B) на **байт**.

## D251. Полный str-surface

- **split-семейство:** `@split(sep)`, `@splitn(n, sep)`, `@rsplit(sep)`,
  `@rsplitn(n, sep)`, `@split_once(sep) -> Option[(str,str)]`, `@rsplit_once(sep)`,
  `@split_terminator(sep)` (без пустого хвоста), `@split_whitespace()` (Unicode WS —
  ASCII-fast + делегат 152.4 для не-ASCII), `@lines()`.
- **trim:** `@trim`/`@trim_start`/`@trim_end` (Unicode WS), `@trim_matches(pred)`/
  `@trim_start_matches`/`@trim_end_matches` (directional, Rust-parity),
  `@strip_prefix(p) -> Option[str]`, `@strip_suffix(s) -> Option[str]`.
- **search:** `@find`/`@rfind` (байт-offset, из 152.1),
  `@match_indices(needle) -> []int` (байт-offset'ы), `@matches(needle) -> []str`
  (подстроки), `@contains`/`@starts_with`/`@ends_with`,
  `@is_char_boundary(byte_idx) -> bool` (Rust-parity — для безопасной нарезки).
- **transform:** `@replace`, `@replacen(from, to, n)`, `@repeat`, `@pad_left`/
  `@pad_right`/`@pad_center` (ширина в **codepoint'ах** → `as_chars().len()`).
- **owned-линзы:** `@to_bytes() -> []u8`, `@to_chars() -> []char`.
- **safe slice:** `@get(a..b) -> Option[str]` (None при OOB / не-границе).

Все аллокации — через `_buffer` (152.0). Семантика байт-в-байт документируется в D251.

## Тесты

- **POS:** `split`/`splitn`/`rsplit`/`split_once`/`split_whitespace`/`lines`;
  `trim`/`trim_matches`/`strip_prefix`/`strip_suffix`; `replacen`;
  `pad_*` (не-ASCII ширина: `"é".pad_left(3,'·') == "··é"`); `match_indices`
  (байт-offset'ы корректны); `get(a..b)` happy-path.
- **NEG:** `splitn(0, …)` (поведение зафиксировано); пустой `sep`;
  `get(a..b)` OOB → `None`; `get` рассекает codepoint → `None`; `pad` width<len → s.

## Критерии приёмки

- **A1.** Полный split/trim/strip/replacen/pad/match_indices присутствует и
  байт-семантичен.
- **A2.** `pad_*` ширина в **codepoint'ах** (`"é".pad_left(3,'·')=="··é"`).
- **A3.** `match_indices`/`find`/`rfind` возвращают **байт**-offset'ы, композируются
  со slice.
- **A4.** `split_whitespace`/`lines` корректны на ASCII; не-ASCII WS — делегат 152.4
  (или `[M-152-ws-unicode]` если 152.4 ещё не сделан).
- **A5.** `@get(a..b)` — safe (None при OOB/не-границе), `s[a..b]` — panic-версия.
- **A6.** Паритет с матрицей umbrella §1 для не-Unicode-строк (Rust/Go-уровень).
- **A7.** Полный `nova test` без новых FAIL; plan152_2-фикстуры PASS.

## Codegen-prerequisite (владеет 152.2)

**`[M-152.1-str-subview-record-ctor]`** (P2, перенесён сюда из floating, 2026-06-13).
Конструкция str-суб-вью через value-record-литерал `str { ptr: @ptr + off, len: n }`
**мис-компилируется** («passing nova_str to nova_int»); т.к. `runtime.string` линкуется в
КАЖДУЮ программу — это ломает весь str-код. Обнаружено в 152.1 Ф.1b; обход — строить view
через inline `@[a..b]` (`slice.nv`). 152.2 — **прямой потребитель**: zero-copy `@trim`/
`@trim_start/end`/`@strip_prefix/suffix` и slice-хелперы хотят возвращать суб-вью над
`@ptr` без alloc-копии. **Чинить в codegen** (`emit_c.rs` — value-record record-literal с
@ptr-арифметикой; класс — value-record codegen, как закрытые в 152.1 Ф.3 self-return/dispatch).
Fix закрывает И парный **`[M-139-f1-trim-view]`** (zero-copy `@trim` вместо alloc — та же
потребность). До фикса trim/strip строят результат через `@[a..b]` или alloc (как Ф.1b).
