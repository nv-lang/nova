## ✅ Реализовано: Вариант B (2026-05-14)

Sysmon-thread + codegen safepoints. CPU-bound fiber без `runtime.yield()`
больше не монополизирует worker — sysmon помечает overrunning worker'ы,
fiber кооперативно yield'ится на ближайшем safepoint'е. Наблюдаемый
Go-style паритет: peer fiber'ы получают CPU даже без явного yield.

### Что сделано

**Runtime (`runtime.c`):**
- `NovaWorker.preempt_flag` (plain `volatile int`) + `current_fiber_start`
  (`uint64_t`, `__atomic_*`). sysmon OS-thread (`_sysmon_main`) каждые
  ~10ms проходит workers; если `now - current_fiber_start >
  NOVA_PREEMPT_SLICE_NS` (10ms) → `preempt_flag = 1`.
- sysmon стартует в `nova_runtime_init` (после workers), останавливается
  ПЕРВЫМ в `nova_runtime_shutdown` (до `free(_workers)`), join.
- Worker loop: `w->preempt_flag = 0` + stamp `current_fiber_start` перед
  `mco_resume`, обнуляет start после.
- **`uv_run(UV_RUN_NOWAIT)` каждую итерацию worker loop** (раньше — только
  когда deque пуст). Обязательно: вытесненный CPU-fiber держит deque
  непустым → старый путь никогда не сервисил бы таймеры → `Time.sleep`
  wakeup'ы не fire'или бы → deadlock.
- **yielded-FIFO per worker** (`NovaWorker.yielded`). Кооперативно
  вытесненный fiber кладётся СЮДА, не в LIFO-deque. Re-push в deque →
  worker сразу re-pop'ит того же fiber'а → peer'ы голодают. Worker loop:
  deque (свежие/разбуженные) → yielded-FIFO → steal → block.

**TLS (`fibers.h` / `effects.c`):**
- `_nova_preempt_ptr` — `volatile int*`, в `_worker_main` указывает на
  `&w->preempt_flag`. NULL на main thread / single-thread → safepoint
  no-op. Указатель, а не снапшот: worker застревает в `mco_resume` на
  весь loop и не может перечитать снапшот.
- `nova_preempt_check()` — inline: `if (ptr && *ptr) { *ptr=0;
  nova_fiber_yield(); }`. `NOVA_UNLIKELY` branch hint.

**Codegen (`emit_c.rs`):**
- `emit_fn` — `nova_preempt_check();` первым стейтментом каждой Nova-функции
  (prologue safepoint).
- `emit_loop_body_inline` — `nova_preempt_check();` первым стейтментом тела
  КАЖДОГО цикла (while/loop/while-let/for). Первым, не последним → ловит
  и `continue`-edge. Покрывает tight loops без вызовов функций.

### Тесты

`nova_tests/concurrency/mn_runtime_preemption.nv` — 2 позитивных + 2
негативных:
- **observable interleave** (1 worker): CPU-fiber крутит loop, peer делает
  `Time.sleep(15)` + ставит флаг; CPU-fiber ВИДИТ флаг посреди цикла →
  доказывает что был вытеснен и peer отработал.
- **state correctness**: 2 fiber'а × 400k `fetch_add` под многократным
  вытеснением → ровно 800k, ничего не потеряно.
- **negative**: no-op без `runtime.init` (single-thread default не задет).
- **negative**: цикл на main thread после `runtime.init` безопасен
  (не-fiber контекст, `_nova_preempt_ptr == NULL`).

### Отличие от Go

Механизм — кооперативные codegen safepoint'ы, а не SIGURG async signal
(minicoro `mco_yield` не async-signal-safe). Observable поведение
совпадает: CPU-bound fiber не морит голодом соседей. Не покрыто:
inline-ASM loop без backedge'а codegen'а (н/п pure FFI-call в цикле) —
нишевой кейс, см. Вариант C ниже если понадобится.

## Проблема

CPU-bound fiber без `runtime.yield()` монополизирует worker thread:

```nova
supervised {
    spawn { 
        // Миллиард итераций без yield — блокирует 1 из N workers
        let mut i = 0
        while i < 1_000_000_000 { i = i + 1 }
    }
    spawn {
        // Не запустится на заблокированном worker'е — ждёт steal
        Time.sleep(10)
    }
}
```

Без preemption у Nova нет честного разделения CPU между fibers.
Cooperative `runtime.yield()` помогает только если программист его ставит.

---

## Go's решение (референс)

Go использует `SIGURG` на Linux/macOS (POSIX) и `SuspendThread` на Windows:

1. **runtime/proc.go**: `sysmon` goroutine (не привязана к P) каждые ~10ms
   проверяет `allp[i]._p_.schedtick` — если goroutine на P дольше 10ms
   без reschedule → mark `gp.preempt = true`.

2. **В каждом function prologue** (вставляется компилятором через `morestack`):
   ```asm
   MOVQ (TLS), R14       ; load g
   CMPQ R14.stackguard0, #stackPreempt
   JBE  morestack_noctxt ; если preempt флаг — yield
   ```

3. При signal SIGURG → `asyncPreempt2` — saves register state → calls `gopreempt_m`
   → goroutine помещается обратно в run queue.

**Ключевые факты:**
- Go устанавливает `stackguard0 = stackPreempt` для marking — это единственный bit.
- `morestack` check в каждом prologue = O(1) overhead на каждый function call.
- Async signal-based preemption для loops без calls: SIGURG → signal handler
  записывает контекст и yields.

## Nova's путь

### Вариант A: Cooperative-only (текущее состояние — достаточно для MVP)

`runtime.yield()` — явный hint. Программист вставляет в CPU-bound loops.

**Плюсы:** Уже реализован. Zero overhead в 99% кода. Простота.
**Минусы:** Нечестный scheduler если код без yield.

**Когда достаточно:** IO-bound workloads (сеть, файлы, каналы) — fibers
park/wake естественно, монополизации нет. Nova — backend/CLI языки, где
типичный spawned fiber = IO operation. CPU-bound concurrency — нетипичный use case.

### Вариант B: TLS-флаг + function prologue check

Добавить в каждый генерируемый C-function prologue:

```c
// Nova codegen вставляет в каждый fn:
if (__builtin_expect(_nova_preempt_flag, 0)) {
    _nova_preempt_flag = 0;
    nova_fiber_yield();
}
```

`_nova_preempt_flag` — `__declspec(thread) volatile int`. `sysmon`-аналог
в runtime.c устанавливает флаг для overdue worker'ов.

**Плюсы:**
- Без signal handling — работает на всех платформах одинаково.
- Overhead: 1 conditional branch per function call (~0.5 ns).
- Loops без function calls (tight arithmetic) не preempt'ятся — acceptable
  для Nova where compiler-level loops без function calls = rare.

**Минусы:**
- Codegen изменение: emit prologue check во ВСЕХ функциях.
- Tight loops (`while i < N { i = i + 1 }`) без function calls → не preempt'ятся.
  Чтобы их покрыть — нужна вставка в каждый loop backedge.
- `nova_fiber_yield()` из TLS-флага context не имеет scope/slot — нужен
  guard `if (_nova_active_slot >= 0)`.

**Оценка:** ~3-5 дней codegen + runtime. Реалистичен как Plan 44.7 V1.

### Вариант C: SIGURG-based async preemption (полный аналог Go)

**Linux/macOS:**

1. `sysmon` thread (отдельный OS thread) каждые 10ms check'ит workers.
2. Для overdue worker → `pthread_kill(w->thread_id, SIGURG)`.
3. Signal handler (`sa_sigaction`):
   - Вызывается на worker thread в середине любой инструкции.
   - Через `mco_coro::context` (ASM context) — save + yield + restore.
   - Но `mco_coro` не имеет async-safe context manipulation API.

**Проблема:** minicoro `mco_yield()` НЕ async-signal-safe. Signal приходит
в середине произвольного кода — yield из signal handler = UB + risk
reentrancy. Go имеет специальный ASM `asyncPreempt` function специально
для signal context. Nova нужен аналог.

**Windows:**
- Нет SIGURG. Аналог: `SuspendThread` + `GetThreadContext` + `SetThreadContext`
  + `ResumeThread`. Работает для thread-level preemption, но fiber-level
  требует знать какой fiber запущен → TLS lookup.
- Альтернатива: `QueueUserAPC` (APC queue) — ставит callback в APC queue
  thread'а, вызывается при alertable wait. Но fiber не делает alertable wait.

**Оценка:** 2-3 недели engineering + TSan validation. Высокий риск,
требует ASM-level careful coding.

## Этапы (Вариант B — когда возьмёмся)

### Ф.1: sysmon thread (runtime.c)

```c
static uv_thread_t _nova_sysmon_thread;
static volatile int _nova_sysmon_running = 0;

static void _nova_sysmon_main(void* arg) {
    while (_nova_sysmon_running) {
        uv_sleep(10);  // 10ms interval
        for (int i = 0; i < _nova_worker_count; i++) {
            NovaWorker* w = &_nova_workers[i];
            uint64_t now = uv_hrtime();
            // Если worker занят одной fiber'ой > 10ms → mark preempt
            if (w->current_fiber_start != 0
                && (now - w->current_fiber_start) > 10000000ULL) {
                w->preempt_flag = 1;  // per-worker flag
            }
        }
    }
}
```

`w->current_fiber_start` — `uv_hrtime()` timestamp, записывается в worker
loop перед каждым `mco_resume(co)`.

### Ф.2: Codegen — prologue check

В `emit_c.rs::emit_fn` — после function signature, перед body:

```rust
// Только для Nova functions (не extern functions)
if !fn_decl.is_external {
    self.line("if (__builtin_expect(_nova_should_yield, 0)) {");
    self.line("    _nova_should_yield = 0;");
    self.line("    if (_nova_active_slot >= 0) nova_fiber_yield();");
    self.line("}");
}
```

`_nova_should_yield` — `_Thread_local volatile int` в nova_rt.h.
Worker loop: `_nova_should_yield = w->preempt_flag; w->preempt_flag = 0;`
перед `mco_resume(co)` (чтобы flag виден на правильном TLS-thread).

### Ф.3: Тест

```nova
test "preemption: CPU-bound fiber не блокирует другие" {
    runtime.init(2)
    let mut counter = 0
    let mut done = false
    supervised {
        spawn {
            // CPU-bound без yield
            let mut i = 0
            while i < 100_000_000 { i = i + 1 }
            done = true
        }
        spawn {
            Time.sleep(50)
            counter = counter + 1
        }
    }
    assert(done)
    assert(counter == 1)
    runtime.shutdown()
}
```

Без preemption: второй spawn не запустится пока первый не завершится
(если оба попали на один worker). С preemption — запустится через 10ms.

### Ф.4: Tight loop inject (опционально)

Для loops без function calls — emit backedge check:

```rust
// В emit_while, emit_for:
// В конце loop body добавляем yield check
self.line("if (__builtin_expect(_nova_should_yield, 0)) {");
self.line("    _nova_should_yield = 0;");
self.line("    if (_nova_active_slot >= 0) nova_fiber_yield();");
self.line("}");
```

Overhead: 1 branch per iteration. Для tight loops ~1 cycle overhead.
Включать только с compile-time флагом `#[preemptible]` — не default.

## Риски

**R1: `nova_fiber_yield()` не async-safe.** В Варианте B yield вызывается
из НАШЕГО кода (не signal handler) — проблемы reentrancy нет. Signal-safe
не требуется для TLS-флаг подхода.

**R2: `w->current_fiber_start` read/write без lock.** sysmon читает,
worker пишет — torn read. На 64-bit платформе 64-bit write atomic.
Или использовать `_Atomic uint64_t`. Non-critical: worst case — лишний
или пропущенный preempt на одном interval.

**R3: Preemption при GC STW.** Если sysmon mark preempt во время GC STW
pause — fiber yield'нется внутри GC. Нужен флаг "GC running → skip preempt".
`GC_set_stop_func` callback или `GC_is_stopped()` check.

**R4: Windows `uv_sleep(10)`** — Sleep(10) реальный период ~15ms (Windows
timer resolution). Приемлемо: preemption interval = 15ms вместо 10ms.
Для production улучшение через `timeBeginPeriod(1)`.
