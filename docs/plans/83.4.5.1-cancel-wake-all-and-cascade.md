## 1. Bug + симптом

**Тесты:** `cancel_semantics_test`, `supervised_cancel_stress_test`,
+ (вероятно) `parallel_for` с cancel-внутри.

**Симптом под активным flip:**
- `cancel_semantics_test`: одна или несколько проверок ожидают, что
  parked fiber пробудится «быстро» (по cancel'у scope), но park
  ждёт собственный wake (timer / channel) — fiber висит на park до
  timeout / external signal.
- `supervised_cancel_stress_test`: deadlock — много nested supervised
  с cancel; cancel publish'нут на parent, но child scope'ы и их
  parked fibers не уведомлены → drain main thread'а ждёт
  pending_remote == 0 которое не падает.

## 2. Root cause

Plan 83.4.3 B5 сделал `scope->cancel_requested` атомарным
(`nova_atomic_bool`), что обеспечивает visibility cancel-флага между
потоками. НО:

- **Cancel НЕ wake'ает parked fibers.** Fiber в park'е (например
  через `Time.sleep` → `nova_sched_park_until(stage==CLOSED)`)
  ждёт ИМЕННО свой wake (от close_cb), не реагирует на cancel-флаг
  scope. До next wake fiber остаётся parked → cancel-cascade не
  доходит → drain hangs.

- **Cancel НЕ cascade'ится в child scope'ы.** Если supervised
  родитель → cancel publish'нут на родителя, child supervised'ы и
  их parked fibers не получают cancel ⇒ deadlock.

Существующий `nova_sched_cancel_all_pending(scope)`
(`nova_sched.h:247+`) **итерирует pending_handle'ы** и зовёт
их stop_cb (для close-async-handle). Это работает для **resource-
holding** parks (sleep, channels, blocking) но НЕ для bare-parks +
не cascade'ится parent→child.

## 3. Cross-runtime reference

### Go `context.Context`

```go
ctx, cancel := context.WithCancel(parent)
// ... goroutine select { case <-ctx.Done(): return }
cancel()  // closes ctx.Done() channel
```

- `cancel()` атомарно закрывает `Done()` channel via `close(ch)`.
- `select { case <-ctx.Done(): }` — все goroutines, ждущие на этом
  channel, мгновенно просыпаются (channel-close = broadcast).
- Hierarchy: `WithCancel(parent)` — child cancelled when parent's
  Done closes (registered via mutex-protected children list).
- **No deadlock** by construction: channel-close — visible to всех
  selects, atomically.

### tokio `CancellationToken`

```rust
let token = CancellationToken::new();
let child = token.child_token();
// ... task: token.cancelled().await
token.cancel();  // atomic + notify_waiters() для всех ждущих
```

- Atomic flag в `CancellationTokenState`.
- `notify_waiters()` budит ВСЕХ wakers, зарегистрированных через
  `.cancelled().await`.
- Hierarchy: child создаётся через parent → cancel propagates
  через explicit linked-list iteration.
- **No deadlock**: `notify_waiters` обходит ВЕСЬ waker-list под
  одним lock acquire.

### Kotlin `Job` cascade

```kotlin
val parent = SupervisorJob()
launch(parent) { … }   // child Job
parent.cancel()        // cascades to all children
```

- `JobSupport.cancel()` устанавливает atomic state, проходит
  child-list через `forEach { it.cancelInternal(cause) }`.
- Parked coroutines возобновляются с `CancellationException`.

### Node `AbortController` (single-thread)

```js
const ctrl = new AbortController();
fetch(url, { signal: ctrl.signal });
ctrl.abort();  // event 'abort' fires on signal
```

- `AbortSignal` — EventTarget; `abort()` dispatches `'abort'` event
  → all listeners (including fetch's internal one) run.
- Single-thread — no race.

## 4. Production-grade design для Nova

### 4.1. `nova_scope_cancel_wake_all(scope)` — wake all parked fibers

В `nova_sched.h` (либо `fibers.h` если зависимость требует):

```c
/* Plan 83.4.5.1: budит ВСЕ parked fiber'ы в scope. Зовётся из
 * cancel-flow (nova_cancel_token_cancel) после atomic store
 * cancel_requested=true. Идемпотентно: wake уже-running fiber'а
 * — no-op (sched.parked[slot] == false ничего не меняет). */
static inline void nova_scope_cancel_wake_all(NovaFiberQueue* scope) {
    if (!scope) return;
    NovaSchedState* st = nova_sched_find_state(scope);
    if (!st) return;
    int n = scope->count < st->capacity ? scope->count : st->capacity;
    for (int i = 0; i < n; i++) {
        if (st->parked[i]) {
            nova_sched_wake(scope, i);
        }
    }
}
```

**Memory ordering**: caller (`nova_cancel_token_cancel`)
ОБЯЗАТЕЛЬНО публикует `cancel_requested = true` **ДО** wake-all
(RELEASE-store). Каждый wake'нутый fiber на park-loop check'ает
`cancel_requested` ACQUIRE-load'ом → видит флаг → бросает
`scope cancelled` (через park-predicate либо post-park check).

### 4.2. Cascade в child scope'ы

`NovaFiberQueue` уже имеет concepts parent/child через
`_nova_parent_scope` в spawn-ctx + первое поле `bound_token` в
scope itself. Подход:

**Подход A — explicit children list в `NovaFiberQueue`:**
```c
struct NovaFiberQueue {
    /* ... */
    nova_mutex_t          children_mu;
    struct NovaFiberQueue** children;     /* dynamic array */
    int                   children_count;
    int                   children_cap;
    struct NovaFiberQueue* parent;        /* nullable */
};
```

`nova_scope_init` зовётся с `parent` параметром (codegen для
nested `supervised{}` передаёт outer scope). Регистрирует child в
`parent->children` под `children_mu`. `nova_scope_destroy`
удаляет.

`nova_scope_cancel(scope)`:
1. RELEASE-store cancel_requested = true.
2. `nova_scope_cancel_wake_all(scope)`.
3. Lock `children_mu`; для каждого child рекурсивно
   `nova_scope_cancel(child)`. Unlock.

**Подход B — cancel propagation через bound_token tree:**

`NovaCancelToken` уже cascade'ит через `linked` array (Plan 49).
Использовать существующую инфраструктуру. Когда scope cancel'ится
без token'а — explicit cascade нужен (подход A).

**Рекомендация:** комбинация: scope-tree (A) для default-cancel,
token-tree (B) для explicit `supervised(cancel: tok)`.

### 4.3. Cancel-flow integration

`nova_cancel_token_cancel` (`fibers.h`) + scope cancel-set sites:

```c
void nova_cancel_token_cancel(NovaCancelToken* t) {
    if (nova_abool_load(&t->cancel_requested)) return;  /* idempotent */
    nova_abool_store(&t->cancel_requested, true);

    if (t->bound_scope) {
        nova_abool_store(&t->bound_scope->cancel_requested, true);
        nova_scope_cancel_wake_all(t->bound_scope);
        /* Plan 83.4.5.1: cascade в child scope'ы */
        nova_scope_cancel_cascade_children(t->bound_scope);
    }
    /* Cascade в linked token'ы (existing Plan 49). */
    for (int i = 0; i < t->linked_count; i++) {
        nova_cancel_token_cancel(t->linked[i]);
    }
}
```

## 5. План реализации

### Ф.1 — `nova_scope_cancel_wake_all` (~0.5 dev-day)

1. Реализация в `nova_sched.h`.
2. Зов из `nova_cancel_token_cancel` после `cancel_requested = true`.
3. Targeted: `cancel_semantics_test` под explicit `runtime.init(2)`
   — parked fiber пробуждается immediately по cancel.

### Ф.2 — Children list + cascade (~1 dev-day)

1. Расширить `NovaFiberQueue` полями `children` array + mutex.
2. `nova_scope_init` принимает optional `parent` (codegen для
   nested supervised передаёт outer scope).
3. `nova_scope_cancel_cascade_children` — DFS по children, recursive
   cancel.
4. Targeted: `supervised_cancel_stress_test` — nested supervised'ы
   получают cancel.
5. Codegen-edit `emit_c.rs::supervised{}`: эмитит
   `nova_scope_init_with_parent(&inner, _nova_active_scope)` для
   nested.

### Ф.3 — Park-predicate cancel check (~0.5 dev-day)

Park-predicate из 83.4.1 уже работает на atomic `stage`. Добавить
optional cancel-pred-check в helper (если нужно — выяснится при
тестах):

```c
static nova_bool _is_done_or_cancelled(void* ctx) {
    SomeState* s = (SomeState*)ctx;
    if (nova_abool_load(&s->scope->cancel_requested)) return true;
    return /* original done check */;
}
```

После park-return: caller check'ает `cancel_requested` и бросает
`scope cancelled`.

## 6. Acceptance

- `cancel_semantics_test`, `supervised_cancel_stress_test` PASS под
  active flip (либо под explicit `runtime.init(2)`).
- 1000-fiber supervised stress с random cancel в child scope'ах —
  drain завершается < 5s, без deadlock.
- 0 регрессий single-thread пути.

## 7. Risk

- **Mutex contention на `children_mu`** под create-heavy load.
  Mitigation: mutex используется ТОЛЬКО на scope-init/destroy +
  cancel-cascade — все rare events; spawn-path не трогает.
- **Cascade ABI** — `NovaFiberQueue` сегодня stack-alloc'ится в
  codegen-генерируемом коде; добавление `children` array требует
  init/destroy симметрии. Mitigation: lazy-alloc детей (NULL до
  первого nested supervised); destroy — free под mutex.
