## –Ч–∞—З–µ–Љ

Nova has `int` (alias `i64`) but no symmetric `uint`. –°–µ–є—З–∞—Б:
- Indexing `arr[i int]` вАФ int –Љ–Њ–ґ–µ—В –±—Л—В—М negative вЖТ runtime check needed
- Sizes `len() -> int` вАФ declared return –Љ–Њ–ґ–µ—В –±—Л—В—М negative semantically
- API asymmetric: `i64` –Є–Љ–µ–µ—В typed alias `int`, `u64` –љ–µ –Є–Љ–µ–µ—В typed alias

Modern languages provide symmetric pair:

| –ѓ–Ј—Л–Ї | signed | unsigned | Width |
|---|---|---|---|
| **Rust** | `isize` | `usize` | platform-pointer (64-bit on x64) |
| **Go** | `int` | `uint` | platform-pointer |
| **Swift** | `Int` | `UInt` | platform-pointer |
| **C** | `int`/`long`/etc | `unsigned`/`size_t` | platform-dependent |
| **Nova (—В–µ–Ї—Г—Й–µ–µ)** | `int` (= `i64`) | (–љ–µ—В) | fixed 64-bit |

## Open Questions (—Б recommended answers)

### Q1: `uint` alias of `u64` –Є–ї–Є distinct primitive?

**Recommended: ALIAS of `u64`** (mirror —В–µ–Ї—Г—Й–Є–є `int` = `i64` pattern).

**Why:**
- Bootstrap Nova –љ–µ –Є–Љ–µ–µ—В multi-arch story вАФ `int` —Г–ґ–µ just-alias `i64`.
  Asymmetry –±—Л–ї–Њ –±—Л strange (signed alias, unsigned distinct).
- Modern parallels:
  - Rust `usize` distinct potentially (platform width) вАФ Nova bootstrap
    pre-multi-arch, –љ–µ —А–µ–ї–µ–≤–∞–љ—В–љ–Њ
  - Go `uint` distinct (platform width) вАФ same
  - Swift `UInt` distinct (platform width) вАФ same
- Future evolution: –Ї–Њ–≥–і–∞ Nova –і–Њ–±–∞–≤–ї—П–µ—В multi-arch support (Plan 58+),
  `int`/`uint` –Љ–Њ–≥—Г—В —Б—В–∞—В—М distinct platform-pointer types. –°–µ–є—З–∞—Б вАФ
  bootstrap-grade alias.

**Codegen:** `"uint" => "uint64_t"` (mirror `"int" => "nova_int"`).
–Т —Д—М—О—З–µ –Љ–Њ–ґ–љ–Њ add typedef `nova_uint = uint64_t` parallel `nova_int =
int64_t`.

### Q2: `int вЖТ uint` cast вАФ explicit only? saturation? wrap?

**Recommended: explicit `as uint` cast —Б saturation** (negative вЖТ 0).

**Why:**
- Nova already has D54 `as`-cast saturation –і–ї—П `float вЖТ int`
  (Plan 07). Apply same pattern: out-of-range вЖТ boundary value.
- Modern parallels:
  - **Rust** `i64 as u64` вАФ bit-cast (wraps; -1 вЖТ u64::MAX). User-hostile
    surprise. Often source of bugs.
  - **Swift** `UInt(value)` initializer вАФ **traps** on negative. Safer
    but verbose. Has separate `UInt(bitPattern:)` for explicit wrap.
  - **Go** `uint(x)` –≥–і–µ `x int` вАФ bit-cast wraps. Same Rust pitfall.
  - **Recommended Nova:** saturation (`-5 as uint вЖТ 0`, matches
    `f64 as int` D54 invariant). Closer –Ї Swift safety –±–µ–Ј runtime trap.

**Bit-cast variant:** `(x as i64).bit_cast[u64]()` –Є–ї–Є explicit method вАФ
–і–ї—П FFI cases –≥–і–µ wrap semantically correct. Don't make —Н—В–Њ default.

### Q3: Index types вАФ `arr[i int]` –Є–ї–Є `arr[i uint]`?

**Recommended: keep `int`** –і–ї—П indexing API.

**Why:**
- **Rust** uses `usize` everywhere –і–ї—П indexing вАФ compile-time
  guarantee non-negative. –Э–Њ: ergonomics suffer (constant `as usize`
  casts –њ—А–Є index math —Б `i32` results).
- **Swift / Go / Kotlin** use `Int` –і–ї—П indexing вАФ runtime check –і–ї—П
  negative. Ergonomic, slight overhead.
- Nova precedent: `arr.len() -> int`, current Range/Iter `-> Option[int]`.
  Switching –Ї uint = breaking change –і–ї—П 100+ APIs.

**Decision:** `arr[i int]` –Њ—Б—В–∞—С—В—Б—П. Type-checker –Љ–Њ–ґ–µ—В add lint
warning ¬Ђconsider non-negative¬ї –≥–і–µ –і–Њ—Б—В—Г–њ–љ–∞ constant-fold proof.

**–Ъ–Њ–≥–і–∞ `uint` –љ—Г–ґ–µ–љ:**
- Explicit non-negative contracts: `fn parse_age(s str) -> Option[uint]`
- Bitfield/flags-like APIs: `Flags { type Mask = uint }`
- FFI –Ї C `size_t`/`uint*_t` parameters

---

### Q4: Default literal type `42` вАФ int –Є–ї–Є uint?

**Recommended: `int` (keep current behavior)**.

**Why:**
- Backward compat: all existing Nova code assumes `42: int`.
- Modern parallels:
  - Rust: integer literal type is inferred (default `i32`); no `uint`
    default.
  - Go: untyped constant; defaults –Ї `int` when used in expression.
  - Swift: literal default `Int`.
- Nova inference: literal `42` defaults `int`, promotes –Ї context
  type. User opt-into uint via `42 as uint` –Є–ї–Є suffix `42u`.

**Suffix syntax (optional):** `42u` –Є–ї–Є `42u64` –і–ї—П uint literals,
mirror Rust `42u64` / Go `uint(42)`. Decision: add suffix syntax
–µ—Б–ї–Є/–Ї–Њ–≥–і–∞ uint use grows. Sometimes annotation enough (`let x uint = 42`).
