# Plan 33.1: Контракты — Core (requires/ensures/old/result + Z3 + runtime)

> Часть [Plan 33](33-contracts-implementation.md). Минимальный end-to-end:
> парсер → typecheck → Z3 SMT → debug runtime fallback → zero-cost release.
> Decision point: после 33.1 решаем стоит ли делать 33.2.

## Scope

**Включено:**
- Грамматика `requires <expr>` / `ensures <expr>` после сигнатуры функции.
- Magic-имена в контрактах: `result`, `old(expr)`, `result.is_ok`,
  `result.is_err`, `result.value`, `result.error` (для `Fail`-функций).
- Оператор `==>` (импликация) и `<==>` (эквивалентность).
- Атрибуты `#must_verify`, `#unverified`, `#verify_timeout(ms)`, `#pure`.
- Purity-inference (через SCC на call-graph).
- SMT backend trait + Z3 driver.
- Кодировка `int`/`bool`/`str` (как Seq Int)/record/sum-type.
- Debug runtime fallback через `nova_contract_violation`.
- Zero-cost release: доказанные контракты стёрты; CI gate проверяет.
- AI-friendly diagnostic с counterexample.

**НЕ включено (это в 33.2 / 33.3):**
- Циклы (loop invariants нужны для верификации тела с while/for).
- Frame conditions (`reads`/`modifies`).
- Record `invariant`.
- Composition (вызов другой fn в контракте).
- Termination (`decreases`) и рекурсивные функции.
- `pure_view` + axioms + `#verify_handler`.
- Ghost / `assume` / `assert_static`.
- Quantifiers (`forall` / `exists`).
- FP IEEE 754, strings beyond equality, sets/maps.
- Incremental cache + parallel verification.
- Cross-check Z3↔CVC5 (Z3-only в 33.1).
- Module strict mode (`#must_verify_module`).
- `#trusted` external fn.

## Ограничения 33.1 — что не верифицируется

- Функции с циклами: ensures на них принимается, runtime-check работает.
  SMT-проверка (Ф.3) может вернуть unknown.
- Рекурсивные функции: ensures runtime-check работает. SMT-проверка
  без `decreases` будет unsound — линт-warning выводится после Ф.3.
- Вызовы других fn в контрактах: **compile error** в 33.1
  («composition not yet supported (Plan 33.2)»).
- Контракты на handler-state: **compile error** в 33.1
  («pure_view not yet supported (Plan 33.3)»).
- Контракты на функциях с `Fail` эффектом: **compile error** в 33.1
  («ContractResult + flow-analysis — Plan 33.1 Ф.3 / Ф.4 follow-up»).
  ContractResult требует SMT-кодировки в Ф.3.

Эти ограничения временные и снимаются в Ф.3/33.2/33.3. Программист видит
явные сообщения, не silent fail.

## Статус 33.1 (2026-05-12)

**Закрыто без SMT** (workable runtime-only stack):
- ✅ Ф.1 AST + parser + lexer + ==> / <==> операторы.
- ✅ Ф.2 ContractCtx: result/old в requires, composition в контрактах
  — compile errors.
- ✅ Ф.2 ext: contracts на Fail-fn — compile error (требует ContractResult).
- ✅ Ф.4 runtime fallback: expression-bodies + block-bodies + early-return

- ✅ D96 attribute syntax (`#name` без скобок) для всей Nova.

- ⏸ Counterexample diagnostic с values.
- ⏸ Zero-cost release CI gate (verify-zero-cost).
- ⏸ Purity inference (SCC по call-graph) — нужно для 33.2 composition.
- ⏸ Fail-fn ContractResult + flow-analysis.

## Фазы 33.1

### Ф.1 — AST + parser + lexer (грамматика минимум)

**Файлы:** [`compiler-codegen/src/ast/mod.rs`](../../compiler-codegen/src/ast/mod.rs),
[`compiler-codegen/src/parser/mod.rs`](../../compiler-codegen/src/parser/mod.rs),
[`compiler-codegen/src/lexer/`](../../compiler-codegen/src/lexer/).

**Архитектурное решение: контекстные keywords (не lexer-level).**

`requires` / `ensures` / `old` остаются **идентификаторами** на уровне
лексера. Парсер распознаёт их **контекстно** в позиции после сигнатуры
функции (до `=>` / `{`).

Обоснование:
- Backward-compat: существующие тесты и std-код используют эти имена как
  обычные переменные без модификаций.
- Меньший diff в lexer + token enum; не нужно править сотни мест
  в parser, которые работают с keyword-токенами.
- Прецедент: Eiffel/D/Ada — `require`/`ensure` контекстные;
  Rust — `union`/`raw` контекстные (не reserved).

Если в 33.3 для `forall`/`exists` возникнет conflict с переменными —
их можно сделать lexer-keyword'ами тогда (не сейчас).

Новые атрибуты: `#must_verify`, `#unverified`, `#verify_timeout`, `#pure`.
Атрибуты — через существующий механизм `parse_function_attributes`
(как `#realtime` в Plan 16).

**AST (✅ DONE 2026-05-12):**

Реализовано в [`compiler-codegen/src/ast/mod.rs`](../../compiler-codegen/src/ast/mod.rs):

```rust
pub struct FnDecl {
    // ... existing ...
    pub contracts: Vec<Contract>,
    pub verify_mode: VerifyMode,           // Default | MustVerify | Unverified
    pub verify_timeout_ms: Option<u32>,    // дефолт 2000 на типчекер-уровне
    pub purity: Purity,                    // Pure | Effectful | Unknown (Ф.2 inference)
}

pub struct Contract {
    pub kind: ContractKind,
    pub expr: Expr,
    pub span: Span,
}

pub enum ContractKind { Requires, Ensures }
pub enum VerifyMode { Default, MustVerify, Unverified }
pub enum Purity { Pure, Effectful, Unknown }
```

В `parser::parse_fn` `FnDecl` создаётся с empty contracts + default
verify_mode + Unknown purity. Этим обеспечивается backward-compat для
всех существующих fn-конструкций.

`result` — НЕ keyword (специальное имя в контракт-скоупе).

**Парсер.**

В [`parse_fn`](../../compiler-codegen/src/parser/mod.rs#L322) после
`parse_effects_until_arrow_or_body`, до тела — цикл:
```
requires <expr>
ensures <expr>
```
на отдельных строках. Стоп: `=>`, `{`, newline+non-keyword.

Атрибуты — через существующий `parse_function_attributes` (паттерн как
для `#realtime`).

`==>` / `<==>` — новые precedence-уровни ниже `||`. `==>` правоассоц.

**Negative tests** (через `EXPECT_COMPILE_ERROR`):
- `requires` в `external fn` без `#trusted` (`#trusted` ещё нет — это error).
- `result` в `requires` — error.
- `old(...)` в `requires` — error.
- Контракт не `bool`-типа — error.

**Acceptance:** 10 R4-функций парсятся; AST сериализуется в golden-файл,
проверяется на стабильность.

### Ф.2 — Type-check + purity-inference

**Файлы:** [`compiler-codegen/src/types/mod.rs`](../../compiler-codegen/src/types/mod.rs).

**Purity inference.** SCC на call-graph; pure ⇔ нет effectful calls
(включая transit). `#pure` атрибут — assertion (если расходится с
выведенным — error на конкретном call-site).

**ContractCtx:**

```rust
pub struct ContractCtx<'a> {
    pub kind: ContractKind,
    pub fn_return: Option<TypeRef>,
    pub fn_fail_row: Vec<TypeRef>,         // [A, B] для Fail[A] Fail[B]
    pub params: &'a [Param],
    pub allow_old: bool,
    pub allow_result: bool,
}
```

**Правила:**

1. Тип контракта = `bool`.
2. `result` биндится: `[]` row → `R`; `[E]` → `ContractResult[R, E]`;
   `[A, B]` → `ContractResult[R, A | B]`.
3. `result.value` / `result.error` — flow-зависимы: доступны только под
   guard (через `==>` или вложенный `if`). Без guard → error с suggestion
   `ensures result.is_ok ==> result.value > 0`.
4. `old(expr)` — type-check `expr` в внешнем ctx; `expr` обязан быть
   `#pure`.
5. Контракт сам обязан быть `#pure`.
6. Composition (вызов другой fn в контракте) — **error в 33.1**
   с указанием на Plan 33.2.

`ContractResult[R, E]` — синтетический prelude-тип:
```nova
type ContractResult[R, E] =
    | Ok(value R)
    | Err(error E)
```

**Negative tests** на каждое правило.

**Acceptance:** R4-примеры типизируются; flow-аналитика для
`result.value` работает.

### Ф.3 — SMT IR + Z3 driver

**Файлы:** новый модуль [`compiler-codegen/src/verify/`](../../compiler-codegen/src/).

```
verify/
  mod.rs                  // VerificationPipeline
  ir.rs                   // SMT-IR engine-neutral
  encode.rs               // Nova AST → IR
  diag.rs                 // AI-friendly diagnostics
  backend/
    mod.rs                // trait SmtBackend
    z3.rs                 // Z3 driver через `z3` crate
```

**Trait `SmtBackend`** (CVC5 impl — в 33.3; в 33.1 только Z3, но trait
готов к расширению):

```rust
pub trait SmtBackend: Send {
    fn name(&self) -> &'static str;
    fn new(timeout_ms: u32) -> Self where Self: Sized;
    fn declare_sort(&mut self, name: &str, arity: u32);
    fn declare_fun(&mut self, name: &str, args: &[SortRef], ret: SortRef);
    fn assert(&mut self, formula: &Formula);
    fn push(&mut self);
    fn pop(&mut self);
    fn check_sat(&mut self) -> SatResult;
    fn extract_model(&self) -> Option<Model>;
    fn extract_unsat_core(&self) -> Option<UnsatCore>;
}

pub enum SatResult {
    Sat(Model),
    Unsat(UnsatCore),
    Unknown(UnknownReason),                // Timeout | NonLinear | OOM | Crash
}
```

**Кодировка типов в 33.1 (минимум):**

| Nova | SMT | Theory |
|---|---|---|
| `int`, `i32`, `i64`, `money` | `Int` + axiom `INT_MIN..INT_MAX` | LIA |
| `bool` | `Bool` | core |
| `str` | `(Seq Int)` для length + equality only | Strings (минимум) |
| `record { ... }` | uninterpreted sort + projections | UF |
| sum-type | algebraic datatype Z3 native | Datatypes |
| `ContractResult[R, E]` | algebraic datatype | Datatypes |
| `result` | fresh sort-var | n/a |
| `old(e)` | fresh sort-var equated с encoded e на entry-state | n/a |
| `==>` | SMT `=>` | core |
| `<==>` | `=` для bool | core |
| `f32`, `f64` | **error в 33.1** (требует Ф.9 в 33.3) | — |
| handler/fn-value | uninterpreted, snapshot запрещён | UF |
| `[]T`, `Map`, `Set` | **error в 33.1** (требует quantifiers в 33.3) | — |

Контракт, использующий `f32`/array/map → compile error с указанием на 33.3.

**Pipeline `verify_fn` (straight-line only в 33.1):**

1. Declare params как fresh vars.
2. Assert all `requires`.
3. SSA-encode body (только без циклов; если есть `while`/`for`/`loop`
   — пропускаем verification, warning «loop not supported (Plan 33.2)»,
   контракт остаётся runtime-only).
4. На каждом return: encode `result := value`; для каждого `ensures Q`:
   push; assert `pc ∧ ¬Q`; check_sat; pop.
5. На `throw`-point: encode `result := Err(err)`, проверка ensures
   под `result.is_err`.

**AI-friendly diagnostic format:**

```
warning C0341: contract not verified statically
   in function `withdraw` at src/account.nv:34
   ┌─ src/account.nv:36:5
   │
   36 │     ensures acc.balance == old(acc.balance) - amount
   │             ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

   counterexample (Z3 sat in 142ms):
     amount               = 1000
     old(acc.balance)     = 500
     acc.balance after f  = 500    ← contradicts ensures
     path                 = branch line 41 (amount > balance)

   reason: branch at line 41 skips update; no requires forbids this

   suggestions:
     1. add `requires acc.balance >= amount`
     2. weaken: `ensures result.is_ok ==> acc.balance == old(...) - amount`
     3. handle the case explicitly at line 41
     4. mark #unverified if intentional (compile error in release)
```

Reason-categories для 33.1: `non-linear-arithmetic`, `timeout`,
`unsupported-feature` (с указанием на 33.2/33.3).

**Build:** Z3 как dependency через `z3` crate; libz3 — vcpkg (как bdwgc)
на Windows, `apt install libz3-dev` на Linux. Документировать в README.

**Acceptance:**
- 10 R4-функций без циклов доказываются Z3.
- Counterexample приводится со значениями для заведомо ложного ensures.
- `SmtBackend` trait готов к расширению (закладка под CVC5 в 33.3).

### Ф.4 — Runtime fallback + zero-cost release

**Файлы:** [`compiler-codegen/src/codegen/emit_c.rs`](../../compiler-codegen/src/codegen/emit_c.rs),
новый `nova_rt/contracts.{h,c}`.

**Codegen pattern:**

```c
R nova_f(P_t p) {
#ifdef NOVA_CONTRACTS_RUNTIME
    if (!P) nova_contract_violation(NOVA_PRE, "f", "P", "file.nv", LINE);
#endif

    /* snapshot old(expr) — только used в ensures */
    SNAP_TY _old_0 = NOVA_SNAPSHOT(expr_0);

    /* setjmp для Fail-catching */
    int _result_is_ok = 1;
    R _result_val;
    E_row _result_err;
    NovaFailFrame _ff;
    nova_fail_push(&_ff);
    if (setjmp(_ff.jmp) == 0) {
        _result_val = /* body */;
        nova_fail_pop();
    } else {
        _result_err = _ff.error_value;
        _result_is_ok = 0;
        nova_fail_pop();
    }

#ifdef NOVA_CONTRACTS_RUNTIME
    if (!Q_substituted) nova_contract_violation(NOVA_POST, ...);
#endif

    if (!_result_is_ok) nova_throw(_result_err);
    return _result_val;
}
```

**Snapshot strategy (минимум для 33.1):**

| Тип | Snapshot |
|---|---|
| Scalar (int/bool/char) | by value |
| `str` | by-value (immutable, shared) |
| Record (value) | recursive field clone |
| Record (mut ref) | recursive deep clone via `nova_record_clone` |
| sum-type | recursive payload clone |
| `ContractResult` | clone |
| handler/fn-value | **compile error** |

Array / Map / Set — snapshot не реализован (они в 33.3).

**Lift detection.** Pre-pass собирает `old(...)` атомы в ensures —
snapshot'им только их.

**Release-режим:**
- Доказанные контракты — отсутствуют в emit_c (lift pass).
- `#must_verify` — error в Ф.3 при unknown.
- `#unverified` — отсутствуют.
- Default unknown в release → **compile error** (R20). Программист
  обязан явно `#unverified`.

**`nova test --verify-zero-cost`:**
1. Build release.
2. На каждом `.o` запускает `nm`/`dumpbin`.
3. Для функций с proven контрактами — нет ссылок на `nova_contract_violation`.
4. Failure — test fails.

CI: dedicated job `contracts-zero-cost-33-1`, обязательный для merge.

**Runtime helper `nova_rt/contracts.{h,c}`:**

```c
typedef enum {
    NOVA_PRE, NOVA_POST
} NovaContractKind;

void nova_contract_violation(
    NovaContractKind kind,
    const char* fn_name,
    const char* contract_src,
    const char* file,
    int line);
```

В fiber-body — `nova_throw` с `ContractViolation` per D13.
В test-frame'е — `nova_test_fail`.
В main без frame — graceful exit non-zero.

**Acceptance:**
- 100% контрактов из 33.1-тестов работают runtime-fallback в debug.
- `nova test --verify-zero-cost` зелёный для 33.1.
