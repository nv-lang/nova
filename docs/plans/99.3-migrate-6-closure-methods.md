## Цель

Перенести 6 closure-applying методов из `emit_c.rs` inline emits в
`std/prelude/core.nv` Nova-body. После 99.3: 14/14 Option/Result-
методов на Nova (`unwrap` — Plan 61 lineage).

## Atomic shipping policy

**КРИТИЧНО:** каждый метод — **один коммит** с парой:
1. `std/prelude/core.nv` — `external fn` → Nova-body `=> match @ {...}`.
2. `emit_c.rs` — удалить соответствующий inline emit-блок.

**Иначе** — C-redefinition collision (Nova-body эмитится с тем же
именем `Nova_Option_method_<m>_<T>`; если inline-emit живёт — collision).

После каждого коммита — **regression run** на `plan62/89/95/95.bis`
+ stdlib consumers. Если non-trivial FAIL — STOP, document, don't ship
broken state.

**Тесты:**
- `plan62/option_methods_from_prelude.nv` (`map(|x| x*2)` if present)
- `plan89/...`
- `plan95/*`

  ```nova

      ro o Option[int] = Some(21)
  ```

## Risk + mitigation

**R1.** C-redefinition collision — strict atomic per-method (core.nv
+ inline-delete в одном коммите).

**R2.** stdlib consumers (`std/encoding/json.nv` etc.) могут использовать
эти методы — verify через targeted regression на каждом шаге.

**R3.** Closure-arg type-inference edge cases (Plan 98 не покрыл
`[]Option[T]` — но closure-applying methods обычно не вызываются на
массивах). Document если выявится.

**R4.** STOP-at-failure: если на любой Ф.N regression non-trivial —
revert commit + investigate + либо fix + re-ship либо document как
known limitation.


## 2026-06-06 — [M-str-len-closure-dispatch] CLOSED

**Marker:** `[M-str-len-closure-dispatch]` 🟢 CLOSED on commit landed
этой session (branch plan-99-str-closure-dispatch).

**Symptom:** `Option[str].map(|s| s.byte_len())` failed at C-codegen с
`member reference base type 'nova_int' is not a structure`. Closure body
emitted with параметром `s` typed `nova_int` instead of `nova_str`.
Variable type for map return was correct (NovaOpt_nova_int len) — only
closure body wrong.

**Root cause:** Option.map / Result.map_err / Result.unwrap_or_else
dispatch sites (emit_c.rs:18961+, 19233+) iterated args через generic
`emit_expr` без setup для closure-typed params. emit_expr matches
`ClosureLight` (line 17115) и вызывает `emit_lambda(...None, None)` —
context_param_tys = None → default к `nova_int` для closure params.

Free-fn generic call site (line 21951+) уже имел корректный pattern:
extract fn-typed param sig from fn_decl, apply current_type_subst,
register inner sig в fn_param_sigs. Но Option/Result method dispatch
пропускал этот pre-pass.

**Fix:** оба dispatch sites теперь pre-emit lambda с explicit
context_param_tys (built from fn_decl param's TypeRef::Func + type_subst).
Pattern mirrors line 22042-22046 в emit_call's hof-param branch.

**Tests:**
- 4 sub-tests re-enabled (был DISABLED с marker reference):
  - Option[str].map(|s| s.byte_len()) — option_map_migrated.nv
  - Result[int,str].unwrap_or_else(|e| e.byte_len()) — result_unwrap_or_else_migrated.nv
  - Result[int,str].map_err(|e| e.byte_len()) — result_map_err_migrated.nv (2 sub-tests)
- plan99: 9/0 PASS (was 8/0 с 3 sub-tests disabled).
- Regression clean: plan100_1/2/3/4_1/4_2/4_4/4_5/6/7/8, plan103_9,
  plan108/108_1.

**Acceptance:**
- A99.M-str: Option[T].map с closure body using T-typed methods (str.len(),
  custom .field access, etc) emits closure с правильным T-substituted
  parameter type.
- A99.M-str-result: same для Result[T,E].map_err (E-substituted) and
  unwrap_or_else (E-substituted).
- A99.M-str-int: closure return type substitution still works для
  `|x| str.from(x)` patterns (NovaClos_ii vs NovaClos_is dispatched
  properly).
- A99.M-str-regr: 0 regressions across closure-heavy test suites.
