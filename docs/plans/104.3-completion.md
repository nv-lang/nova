## Архитектура

### completion.rs — единый модуль

```
nova_lsp::completion
  ├── CompletionContext — enum { TopLevel, FnBody, TypeBody, Import, MethodDot { prefix } }
  ├── detect_context(src, offset) → CompletionContext
  ├── keyword_items(ctx) → Vec<CompletionItem>
  ├── snippet_items(ctx) → Vec<CompletionItem>
  ├── identifier_items(src, offset) → Vec<CompletionItem>
  ├── method_items(src, offset, prefix) → Vec<CompletionItem>
  ├── import_items(src, offset) → Vec<CompletionItem>
  └── completion_for(src, offset) → Vec<CompletionItem>  ← public entry
```

### Context detection (104.3.1)

```
is_in_comment(src, offset) → bool
is_in_string(src, offset) → bool
detect_context(src, offset):
  if in comment/string → None (no completion)
  if line contains "import " + cursor-after → Import
  if current_line ends in "." at offset → MethodDot
  if enclosing fn body (braces) → FnBody
  elif enclosing type body (braces) → TypeBody
  else → TopLevel
```

### Keyword sets per context

TopLevel: `fn`, `type`, `const`, `let`, `import`, `export`, `module`,
          `effect`, `protocol`, `test`, `bench`, `lemma`

FnBody: `let`, `mut`, `ro`, `if`, `else`, `for`, `while`, `return`,
        `match`, `break`, `continue`, `fn`, `type`, `effect`,
        `defer`, `blocking`, `unsafe`, `consume`, `apply`

TypeBody: `fn`, `const`, `type`, `pub`

### Snippets

| trigger | label | insertText |
|---------|-------|-----------|
| fn | fn ...() | `fn ${1:name}(${2:params}) -> ${3:RetTy} => ${4:()}` |
| type | type ... | `type ${1:Name} {\n\t${2:field} ${3:Type}\n}` |
| match | match ... | `match ${1:expr} {\n\t${2:pattern} => ${3:()}\n}` |
| if-let | if let ... | `if let ${1:pattern} = ${2:expr} {\n\t${3:()}\n}` |
| test | test ... | `test "${1:name}" {\n\t${2:()}\n}` |

### Identifier completion (104.3.2)

Scope walk из текущего offset:
1. Scan backwards for fn bodies, let bindings, params
2. Top-level: fns + types + consts from Module items
3. Prelude: common types/fns (hardcoded list)

### Method completion (104.3.3)

1. Detect `expr.` prefix
2. Find type of `expr` via pattern matching on AST
3. Scan `module.fns` for receiver methods matching type
4. For builtins (`int`, `str`, `bool`, `[]T`): hardcoded method lists

### Import completion (104.3.4)

`import std.collections.│`:
- Parse path prefix up to cursor
- Walk std module tree from manifest/hardcoded std modules
- Return matching submodule names

### Ranking (104.3.5)

CompletionItem.sort_text prefix:
- `00_` locals/params
- `01_` module-level
- `02_` std
- `03_` prelude
- `04_` keywords
- `05_` snippets

```rust
CompletionItem {
    label: String,
    kind: Option<CompletionItemKind>,
    detail: Option<String>,        // type signature

    ..Default::default()
}
```
