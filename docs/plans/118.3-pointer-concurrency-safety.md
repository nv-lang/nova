## Зачем

Plan 118 core определяет `*T` family + escape analysis + unsafe model.
**Concurrency dimension отсутствует:**

### Issue 1: Pointer across `supervised{}` boundary

```nova
// Without 118.3 — silent UB risk:
unsafe {
    ro p = &stack_value                              // stack-local pointer
    supervised {
        spawn {
            // ⚠ Other fiber dereferences p AFTER parent stack frame collapses
            use(p)                                    // dangling!
        }
    }
}
```

Currently no compile-time check; relies on user awareness.

### Issue 2: Pointer held across `await`

```nova
unsafe {
    ro acc = Account { name: "Piter" }
    ro p = &acc

    await some_async_op()                            // ⚠ may yield → GC trigger possible
    // p may dangle if GC compaction occurred during await

    p.name                                            // potentially UB
}
```

Plan 118 core §16 GC honor-system addresses **synchronous** allocations
inside unsafe; but `await` introduces yield points compiler currently
doesn't flag specifically для pointer-holding scope.

### Issue 3: Lock-free typed pointer (`AtomicPtr[T]`)

Production-grade lock-free patterns (Treiber stack, hazard pointers, RCU)
require atomic typed pointer swaps. Plan 103.2 ships atomic primitives
(`AtomicI64`, `AtomicU64`), но typed pointer atomic absent — user'у пришлось
бы cast через `usize` (loses type info + alignment).

```nova
// Without AtomicPtr[T]:
mut head AtomicU64 = AtomicU64.new(0)
unsafe {
    head.store(some_ptr as usize, Release)           // type info lost
    ro p = head.load(Acquire) as *mut Node           // explicit re-cast
}

// With AtomicPtr[T]:
mut head AtomicPtr[Node] = AtomicPtr.new_null()
unsafe {
    head.store(some_ptr, Release)                    // typed; ABI same
    ro p = head.load(Acquire)                        // returns Option[*mut Node]
}
```

### Mainstream comparison

| Язык | Cross-fiber pointer | Pointer across yield | Atomic pointer |
|---|---|---|---|
| **Rust** | `Send` marker enforced compile-time | `!Send` for `*const T`/`*mut T` (must use Arc/Mutex для cross-thread) | `std::sync::atomic::AtomicPtr<T>` |
| **C++** | manual discipline (no enforcement) | manual | `std::atomic<T*>` |
| **Java** | references safe across threads (JMM) | NA | `AtomicReference<T>` |
| **Go** | pointer-across-goroutine works (GC tracks); race detector at runtime | escape analysis + GC | `sync/atomic.Pointer[T]` (1.19+) |
| **C#** | manual; `Volatile.Read/Write`; no compile-time check | NA | `Interlocked.CompareExchange` |
| **Zig** | manual; `pub fn send` patterns | NA | `std.atomic.Atomic(T)` (works for ptrs) |
| **Nova V1** (Plan 118 core) | (no check) | (no check) | (нет — workaround через usize) |
| **Nova V2** (Plan 118.3) | `E_POINTER_CROSS_FIBER` (default ban); `#fiber_send` opt-out marker | `W_POINTER_HELD_ACROSS_SUSPEND` warning; `E_POINTER_HELD_REALTIME_SUSPEND` в #realtime | `AtomicPtr[T]` first-class |

## Дизайн

### 1. Cross-fiber pointer ban (default)

```nova
unsafe {
    ro p = &acc

    supervised {
        spawn {
            // ❌ E_POINTER_CROSS_FIBER
            // raw *T cannot be safely shared across fiber boundary
            use(p)
        }
    }
}
```

**Rule:** raw `*T` captured by `spawn` closure crossing supervised boundary
— `E_POINTER_CROSS_FIBER` error.

**Why:** fiber может быть migrated к другому OS thread; pointer captured
becomes unsafe (need atomic / fence guarantees not provided by raw pointer).

**Opt-out marker `#fiber_send`:**

```nova
unsafe {
    #fiber_send ro p = &acc                          // explicit acknowledgement
    supervised {
        spawn {
            use(p)                                    // ✓ allowed
        }
    }
}
```

User explicitly marks pointer as safe для cross-fiber (responsibility
shifted). Marker recorded в LSP diagnostics для audit.

### 2. Pointer across `await` / suspend

```nova
unsafe {
    ro acc = Account { name: "Piter" }
    ro p = &acc

    await some_op()                                  // ⚠ W_POINTER_HELD_ACROSS_SUSPEND
    p.name                                            // potentially dangling
}
```

**Rule:** if `*T` binding is **alive** (still in scope) при `await` / suspend
point — emit `W_POINTER_HELD_ACROSS_SUSPEND` warning.

**Liveness check:** simple per-block analysis (is pointer in scope at suspend
point?).

**Silence:** `// noqa: W_POINTER_HELD_ACROSS_SUSPEND` или explicit
`#fiber_send` marker.

**Realtime escalation:**

```nova
#realtime
fn foo() {
    unsafe {
        ro p = &x
        await something()                            // ❌ E_REALTIME_POINTER_SUSPEND
    }
}
```

В `#realtime fn`, holding pointer across suspend = **error** (not warning) —
realtime guarantees broken.

### 3. `AtomicPtr[T]`

```nova
type AtomicPtr[T]                                    // opaque, compiler-known

// Construction
mut head = AtomicPtr[Node].new_null()                // Option[*mut Node] = None
mut head2 = AtomicPtr[Node].new(some_ptr)

unsafe {
    // Load (returns Option[*mut T] — NPO applies)
    ro p Option[*mut Node] = head.load(Acquire)

    // Store
    head.store(new_ptr, Release)

    // CAS
    ro success bool = head.compare_exchange(
        expected: old_ptr,
        desired:  new_ptr,
        success_ord: Release,
        failure_ord: Acquire,
    )

    // Exchange
    ro prev = head.exchange(new_ptr, AcqRel)
}
```

**API:**
- `AtomicPtr[T].new_null() -> AtomicPtr[T]` — initialize к None (NULL)
- `AtomicPtr[T].new(p Option[*mut T]) -> AtomicPtr[T]` — initialize с value
- `(AtomicPtr[T]).load(ord MemOrdering) -> Option[*mut T]`
- `(AtomicPtr[T]).store(v Option[*mut T], ord MemOrdering) -> ()`
- `(AtomicPtr[T]).exchange(v Option[*mut T], ord MemOrdering) -> Option[*mut T]`
- `(AtomicPtr[T]).compare_exchange(expected, desired, success_ord, failure_ord)
  -> bool`
- `(AtomicPtr[T]).compare_exchange_weak(...)` — faster, may spuriously fail

**Layout:** single pointer (8 bytes); `Option[*mut T]` через NPO.

**Sync class:** `#realtime` (atomic ops are realtime-safe — no park/wake).
Plan 113 D172 cross-ref.

**Cross-platform:** uses C11 `_Atomic` или `std::atomic` (per platform);
codegen consistent с Plan 103.2 atomic family.

### 4. `#realtime` interaction extension

Plan 118 core §«§20 extern C-unwind» mentions pointer ops в `#realtime` =
`E_REALTIME_POINTER_OP`. Plan 118.3 refines:

- **Allowed в `#realtime`:** `AtomicPtr[T]` ops (atomic ops realtime-safe),
  pointer comparison `==`/`!=`
- **Banned в `#realtime`:** raw pointer deref (may GC trigger if reference
  pointee is GC-tracked), `&value` (may allocate via promote), `<`/`>`
  comparison (cross-allocation undefined)

**Updated D172 cross-ref** (Plan 118.3 amend):

| Pointer op | `#realtime fn` | `#blocking fn` |
|---|---|---|
| `*p` deref | ❌ E_REALTIME_POINTER_OP | ✓ |
| `p.field` auto-deref | ❌ E_REALTIME_POINTER_OP | ✓ |
| `&value` | ❌ E_REALTIME_POINTER_OP (may allocate) | ✓ |
| `p ==/!= q` | ✓ | ✓ |
| `p </> q` (unsafe) | ❌ E_REALTIME_POINTER_OP | ✓ |
| `AtomicPtr.load/store/cas` | ✓ | ✓ |

### 5. Suspend-safe pointer pattern

If user needs pointer across suspend, idiomatic pattern:

```nova
unsafe {
    ro acc = Account { name: "Piter" }
    // Option A: take ownership (move)
    ro acc_value = acc                                // value copy or move
    drop(acc)                                          // explicit drop scope
    await something()
    use(acc_value)                                     // ✓ owned, no pointer

    // Option B: use atomic for shared state
    ro shared = AtomicPtr[Account].new(&acc)
    await something()
    match shared.load(Acquire) {                       // ✓ atomic load
        Some(p) => use_pointer(p),
        None    => fallback(),
    }
}
```

Documented in `docs/typed-pointers.md` §«Concurrency patterns».

## Migration impact

### New patterns enabled

**Lock-free typed pointer:**
```nova
mut head = AtomicPtr[Node].new_null()
unsafe {
    head.store(new_node_ptr, Release)
    if head.compare_exchange(expected, desired, Release, Acquire) { ... }
}
```

**Cross-fiber pointer (explicit acknowledgement):**
```nova
unsafe {
    #fiber_send ro shared_ptr = &shared_data
    spawn { use(shared_ptr) }            // ✓ explicit opt-out
}
```

### Migration of existing concurrency patterns

If any code captures raw pointers cross-fiber implicitly — must add
`#fiber_send` marker или restructure. Audit Ф.0.3.
