# Plan 144.1: Heap layout bitmaps (per-type pointer-offset карты)

> **Создан:** 2026-06-14 (вынесен из [Plan 144](144-precise-gc-implementation.md) §8 Ф.1).
> **Статус:** ✅ DONE (аналитическая / emit-nothing половина) 2026-06-15, ветка
>   `plan-144.1-heap-bitmaps`, спец [D277](../../spec/decisions/08-runtime.md#d277). Runtime-половина
>   (layout-id в заголовке объекта + точный tracer) — остаётся [Plan 144.5](144.5-nonmoving-precise-gc-online.md) Ф.5.
> **Родитель:** Plan 144 §8 **Ф.1**.
> **Гейт:** «лёгкая сторона» (layout известен на этапе компиляции) — может идти **параллельно**
>   с [144.0.1](144.0.1-gate-frame-abi-object-start.md); фактическое потребление карт —

---

## Цель
Codegen эмитит **per-type pointer-offset bitmap**; allocator пишет **layout-id** в заголовок
каждого объекта. Heap-сторона достижима: Nova знает layout типов (record/sum/nested-ptr) →
точная карта смещений указателей в объекте (§7 «Heap-сторона достижима»).

## Acceptance criteria
- [✅ аналитика] **Per-type pointer-offset bitmap вычислен** для каждого типа (record / sum
  per-variant / nested-value / scalar-only / heap-контейнерные поля), layout-точно (offset/size/align
  совпадают с эмитимым C — переиспользован `const_fn_eval::type_size_or_align_resolved`).
  [⏳ Ф.5] *Каждая heap-аллокация несёт layout-id* — runtime-половина, [Plan 144.5](144.5-nonmoving-precise-gc-online.md).
- [⏳ Ф.5] *Precise heap-tracer метит объект точно при заданных корнях* — runtime-потребление
  bitmap'ов, [Plan 144.5](144.5-nonmoving-precise-gc-online.md).
- [✅] **Unit-тест на 3–4 типах:** record / sum / nested-ptr + не-GC поле — фактически **5 фикстур**
  (`record_mixed_ptr_scalar`, `sum_per_variant`, `nested_value_record`, `scalar_only_empty`,
  `heap_str_vec_field`) через `nova gc-layout-analyze` + **21 unit-теста** (incl. raw-`*ro u8` НЕ
  помечается, соседний `Vec` помечается).
- [✅] **«без упрощений как для прода»** — layout-точно, soundness-first (unknown → указатель,
  никогда не пропустить), без stubbed-offset'ов и маскировки; emit-nothing-инвариант проверен grep'ом.
