## Цель
Codegen эмитит **per-type pointer-offset bitmap**; allocator пишет **layout-id** в заголовок
каждого объекта. Heap-сторона достижима: Nova знает layout типов (record/sum/nested-ptr) →
точная карта смещений указателей в объекте (§7 «Heap-сторона достижима»).

## Acceptance criteria
- Каждая heap-аллокация несёт **layout-id**.
- Precise heap-tracer метит объект **точно** при заданных корнях.
- Unit-тест на 3–4 типах: record, sum, nested-ptr (и хотя бы один с не-GC полем).
- «без упрощений как для прода» — обязательный критерий.

## Дизайн-источники / связь
Plan 144 §4 (heap bitmaps), §7 (heap-сторона), §7.6 Q8 (flatten/per-type bitmap = compile-time
источник назначения слотов), Q10 (closures env-bitmap). Prior art: Go per-word `gcdata` bitmap,
Mercury/MLton root-list.
