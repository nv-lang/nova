## Что

Plan 136 реализовал `Stmt::TupleAssign` с консервативным V1 codegen:
каждый rhs-элемент, который читает любой lhs-идентификатор, получает
отдельную C-переменную `_nv_ta_N`.

### V1 (текущее поведение)

Swap `(a, b) = (b, a)` генерирует **2 tmp**:

```c
{ nova_int _nv_ta_0 = b; nova_int _nv_ta_1 = a; a = _nv_ta_0; b = _nv_ta_1; }
```

Rotate `(a, b, c) = (b, c, a)` генерирует **3 tmp**:

```c
{ nova_int _nv_ta_0 = b; nova_int _nv_ta_1 = c; nova_int _nv_ta_2 = a;
  a = _nv_ta_0; b = _nv_ta_1; c = _nv_ta_2; }
```

### V2 (цель)

Для **чистых перестановок** (pure permutations) V2 использует
**cycle-decomposition**: каждый цикл требует ровно 1 tmp.
Идентичные присваивания (`(a, b) = (a, b)`) — 0 tmp.

Swap `(a, b) = (b, a)` генерирует **1 tmp**:

```c
{ nova_int _nv_ta_0 = a; a = b; b = _nv_ta_0; }
```

Rotate `(a, b, c) = (b, c, a)` генерирует **1 tmp**:

```c
{ nova_int _nv_ta_0 = a; a = b; b = c; c = _nv_ta_0; }
```

### Сравнительная таблица

| Выражение              | V1 tmp | V2 tmp | Примечание                       |
|------------------------|--------|--------|----------------------------------|
| `(a,b)=(b,a)`          | 2      | 1      | 1 цикл длиной 2                  |
| `(a,b,c)=(b,c,a)`      | 3      | 1      | 1 цикл длиной 3                  |
| `(a,b,c,d)=(b,a,d,c)`  | 4      | 2      | 2 цикла длиной 2                 |
| `(a,b)=(a,b)`          | 0      | 0      | идентичность, никаких tmp        |
| `(a,b)=(b+1,0)`        | 1      | 1 (V1) | не чистая перестановка → fallback |

## Алгоритм

### 1. Определение чистой перестановки

Входные данные: `lhs[0..N]`, `rhs[0..N]`.

Условие **чистой перестановки**:
- Каждый `rhs[i]` структурно равен некоторому `lhs[j]` (lvalue-identity):
  один и тот же корневой идентификатор + одинаковая цепочка
  `.field` / `[idx]` с константным `idx`.
- Отображение `i → j` биективно (никакое `lhs[j]` не встречается
  дважды в rhs, никакое rhs-значение не является «свежим»).

Если условие не выполняется — **V1 fallback** (консервативные tmp).

### 2. Построение перестановки perm[]

```
perm[i] = j  такой, что rhs[i] ≡ lhs[j]
```

### 3. Декомпозиция на циклы

Стандартный алгоритм cycle-following:

```
visited = [false; N]
for start in 0..N:
    if visited[start]: continue
    if perm[start] == start: visited[start] = true; continue  // identity
    // начало нового цикла
    tmp = lhs[start]            // 1 tmp на цикл
    i = start
    while perm[i] != start:
        lhs[i] = lhs[perm[i]]
        visited[i] = true
        i = perm[i]
    lhs[i] = tmp
    visited[i] = true
```

Каждый нетривиальный цикл длиной `k` требует ровно 1 tmp и `k`
присваиваний (vs `k` tmp + `k` присваиваний в V1).

### 4. V1 fallback для нечистых перестановок

Если хотя бы один `rhs[i]` не является lvalue-равным никакому `lhs[j]`
(например, содержит арифметику, вызов функции, литерал),
алгоритм переключается на V1: tmp для каждого зависимого rhs.
