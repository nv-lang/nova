> **Создан:** 2026-05-23. **Ред. 2 (2026-05-23):** production-grade
> rewrite — заменил simple `owner_id` на полный atomic state machine

## 1. Bugs (один комплексный фикс, ОДИН review-unit)

A2, A3, B2 имеют **общую причину**: fiber-state (lifecycle, owner,
handler-snapshot) не атомарен и не путешествует с fiber'ом между main
и worker'ами под M:N. Архитектурный фикс — единый `NovaFiberMeta`
блок с atomic status + per-fiber handler-storage; делается в одном PR.

### A2 — Supervised drain double-resume

**Симптом** (`supervised_errors`, `supervised_cancel_stress_test`):
```
nova: fiber stack overflow in slot 0 (access violation in fiber arena)
```

**Причина:** main thread в `nova_supervised_step` пытается
`mco_resume(fiber)` параллельно с worker'ом, который стащил fiber
через `nova_deque_steal`. Без atomic-status оба resume'ят одну и ту
же mco_coro → re-entrance → corrupted state → stack overflow в
`__chkstk` probe.

### A3 — Per-fiber handlers под M:N

**Симптом** (`per_fiber_handlers`):
```
FAIL: inner with в spawn перекрывает outer для своего fiber — outer_seen == 111
```

**Причина:** handler-scope-snapshot save/restore в `nova_supervised_step`
работает через TLS-массив `_nova_handler_*`. Когда worker запускает
spawned fiber, его handler-snapshot должен быть restored ДО `mco_resume`;
но `supervised_step` крутится на main thread и не знает про worker.

### B2 — `time_handler` storage swap

**Симптом** (`time_handler`): Time-effect handler не виден из spawn'ed
fiber. Та же причина что A3, но конкретно для `_nova_handler_Time`.

## 2. Cross-runtime reference

### Atomic fiber-status state machine

| Runtime | State enum | Атомарность transitions | Где смотреть |
|---|---|---|---|
| **Go runtime** | `g.atomicstatus` — `_Gidle` / `_Grunnable` / `_Grunning` / `_Gsyscall` / `_Gwaiting` / `_Gmoribund_unused` / `_Gdead` / `_Gcopystack` / `_Gpreempted` | `casgstatus(gp, old, new)` — CAS, под STW respects status | `src/runtime/runtime2.go` + `src/runtime/proc.go` |
| **tokio task** | bit-packed atom: `NOTIFIED`/`RUNNING`/`COMPLETE`/`CANCELLED`/`JOIN_INTEREST`/`REF_COUNT` | `state.transition_to_running()` etc. — CAS с conflict-detection (one-poll guarantee) | `tokio/src/runtime/task/state.rs` |
| **Kotlin Job** | `JobState` enum (NEW/ACTIVE/COMPLETING/COMPLETED/CANCELLING/CANCELLED) | atomic `_state` field, CAS-loop | `kotlinx.coroutines.JobSupport` |
| **Nova сейчас** | implicit (`mco_status` — RUNNING/SUSPENDED/DEAD) + non-atomic fiber-flags | non-atomic | `compiler-codegen/nova_rt/fibers.h` |

**Что Nova добавляет** (паритет с Go): explicit `NovaFiberStatus`
enum + atomic CAS transitions + ownership ID для work-stealing.

### Per-fiber storage

| Runtime | Mechanism | Семантика |
|---|---|---|
| **Go** | `g`-struct (per-goroutine) с TLS-like fields (`m.curg`) | per-goroutine state, миграция между M's встроенная |
| **tokio** | `task_local!` macro → `LocalKey<T>` | per-task TLS, save/restore on `poll()` |
| **Node** | `AsyncLocalStorage` | per-async-context, propagates через `await` |
| **Kotlin** | `CoroutineContext.Element` (ThreadLocal.asContextElement()) | per-coroutine context, transparent через suspension |
| **Nova сейчас** | global TLS-массив `_nova_handler_*` + save/restore main-thread'ом | сломан под M:N |

**Что Nova добавляет**: handler-snapshot в `NovaFiberMeta`
(per-fiber), travels with fiber across workers. **Лучше** tokio/Node
потому что поддерживает scoped algebraic-effect `with{ }` — Nova
fiber может shadow handler для самого себя без пересоздания scope.

## 3. Production-grade design

### 3.1. `NovaFiberMeta` — per-fiber state container

```c
typedef enum NovaFiberStatus {
    NOVA_FS_NEW       = 0,  /* created, not yet pushed to any queue */
    NOVA_FS_RUNNABLE  = 1,  /* in some deque, can be popped/stolen */
    NOVA_FS_RUNNING   = 2,  /* currently executing on owner_worker_id */
    NOVA_FS_PARKED    = 3,  /* waiting on park-condition (83.4.1) */
    NOVA_FS_DONE      = 4,  /* finished (return or cancelled) */
} NovaFiberStatus;

typedef struct _Alignas(NOVA_CACHELINE_SIZE) NovaFiberMeta {
    /* Hot CAS state — own cache line (false-sharing prevention) */
    nova_atomic_int status;            /* NovaFiberStatus */
    nova_atomic_int owner_worker_id;   /* -1 = no owner; w->id when RUNNING */
    /* Per-fiber handler-storage snapshot — fixed-size per current K=4-8 effects */
    void* handler_snapshot[NOVA_MAX_EFFECTS];
    /* Existing fields migrated from NovaSpawnCtxBase */
    NovaFiberQueue* parent_scope;      /* для error-propagation/cancel */
    /* ... */
} NovaFiberMeta;
```

`NovaFiberMeta` хранится **в `mco_coro` user_data** (или в начале
`NovaSpawnCtxBase`-структуры, генерируемой codegen'ом). Travels
наживу с fiber'ом — main, worker A, worker B видят один и тот же
блок.

`_Alignas(NOVA_CACHELINE_SIZE)` на status field гарантирует, что
high-contention CAS не делит cache-line с user data.

### 3.2. State-machine transitions (формально)

```
        spawn               ── push to deque ──
NEW ─────────────────► RUNNABLE ◄──────────────────┐
                          │                        │
                          │ worker pop / steal     │ worker yield-back
                          │ (CAS RUNNABLE→RUNNING) │ (CAS RUNNING→RUNNABLE)
                          ▼                        │
                       RUNNING ────────────────────┘
                       │  │
                       │  │ park-call (CAS RUNNING→PARKED)
                       │  ▼
                       │  PARKED
                       │  │ wake (CAS PARKED→RUNNABLE)
                       │  │
                       │  └──► RUNNABLE
                       │
                       │ entry-fn return / cancelled
                       ▼ (CAS RUNNING→DONE)
                      DONE
```

**Memory ordering:**
- Status CAS — `acq_rel` (success), `acquire` (failure).
- Read-only check (skip-decision в drain) — `acquire`.
- Wake — `release` на published state + `nova_sched_wake` (последний
  использует свой ordering из 83.4.1).
- Steal — двойной CAS: сначала deque-pop (`Plan 44.5` PPoPP 2013
  ordering), затем status RUNNABLE→RUNNING; если status CAS fails —
  unconditional push back to deque (другой worker уже захватил).

### 3.3. Supervised drain ownership protocol

`nova_supervised_step` (`fibers.h:1135+`) read-loop по scope's
fibers, ДЛЯ КАЖДОГО fiber'а:

```c
NovaFiberStatus s = nova_aint_load(&meta->status, ACQUIRE);
int owner = nova_aint_load(&meta->owner_worker_id, ACQUIRE);

if (s == NOVA_FS_DONE) {
    /* count / collect result */
    continue;
}
if (s == NOVA_FS_RUNNING || owner != -1) {
    /* worker owns it; just wait via pending_remote */
    continue;
}
if (s == NOVA_FS_PARKED) {
    /* check wake-conditions (timer-due, signal, cancel) — outside scope of A2 */
    continue;
}
/* s == NOVA_FS_RUNNABLE (or NEW just pushed) — try to claim */
int expected = NOVA_FS_RUNNABLE;
if (!nova_aint_cas(&meta->status, &expected, NOVA_FS_RUNNING, ACQ_REL, ACQUIRE)) {
    /* another worker stole — skip */
    continue;
}
nova_aint_store(&meta->owner_worker_id, -1, RELEASE);  /* main = no worker */
mco_resume(co);
/* On return, set status RUNNABLE or DONE based on mco_status */
```

Symmetric в worker (`runtime.c::_worker_main`):
```c
/* steal: */
co = nova_deque_steal(&victim->deque);
if (co) {
    NovaFiberMeta* m = mco_get_user_data(co);
    int expected = NOVA_FS_RUNNABLE;
    if (!nova_aint_cas(&m->status, &expected, NOVA_FS_RUNNING, ACQ_REL, ACQUIRE)) {
        /* lost race — push back to victim's deque (or our deque) */
        nova_deque_push(&victim->deque, co);
        continue;
    }
    nova_aint_store(&m->owner_worker_id, w->id, RELEASE);
    /* handler-snapshot restore (3.4) */
    nova_handler_restore_from_meta(m);
    mco_resume(co);
    /* On return: post-resume status update + handler save */
    nova_handler_save_to_meta(m);
    /* set status back to RUNNABLE or DONE, owner_id = -1 */
}
```

### 3.4. Handler-snapshot per-fiber API

```c
/* Initialized to NULL pointers at fiber spawn; copy current TLS state. */
void nova_handler_snapshot_init(NovaFiberMeta* m);

/* Worker before mco_resume(fiber): copy meta's snapshot into TLS. */
void nova_handler_restore_from_meta(NovaFiberMeta* m);

/* Worker after mco_resume returns (yield/done): copy TLS into meta. */
void nova_handler_save_to_meta(NovaFiberMeta* m);
```

`NOVA_MAX_EFFECTS` (e.g. 16) — статический лимит зарегистрированных
эффектов. Если runtime растёт за это число — расширить (compile-time
check).

Алгебраические эффекты с `with{ }`-блоком (Nova-level): codegen
эмитит push/pop в TLS, snapshot фиксирует STATE в момент save (snapshot
видит push'нутый handler).

### 3.5. Migration invariants (cross-worker)

Когда fiber мигрирует A → B через work-stealing:
1. **Fiber arena slot**: остаётся в A's арене (Plan 82 Ф.3 — address-
   based dealloc). Worker B читает/пишет в A's slot через TIB-swap
   minicoro'й — works (Plan 82 Ф.0).
2. **`NovaFiberMeta`**: stored in mco_coro user_data → travels with
   fiber automatically.
3. **`handler_snapshot`**: travels in meta → restore on B's TLS.
4. **`owner_worker_id`**: updated atomically by B; A's worker drops
   ownership (was -1 после push back to deque).
5. **GC arena scan**: Plan 82 Ф.3 multi-arena push-callback видит
   fiber-стек в A's арене вне зависимости от current owner — корректно.

## 4. План реализации (incremental)

### Ф.1 — `NovaFiberMeta` + atomic status (~2 dev-day)

1. Define `NovaFiberStatus` enum + `NovaFiberMeta` struct в
   `fibers.h`. Aligned cache-line.
2. Migrate existing `NovaSpawnCtxBase` поля into `NovaFiberMeta`
   (parent_scope, etc.).
3. Initialize meta on spawn (`nova_fiber_spawn_into`, `runtime_spawn_*`).
4. Refactor `nova_supervised_step` to use status-CAS skip-protocol
   (§3.3).
5. Refactor `_worker_main` steal + transitions (§3.3 worker side).
6. Targeted-test: `supervised_errors` + `supervised_cancel_stress_test`
   под explicit `runtime.init(2)`. Должны проходить.
7. **No-regression**: full clang `nova test`.

### Ф.2 — Handler-storage migration (~2-3 dev-day)

1. Add `handler_snapshot[NOVA_MAX_EFFECTS]` field to `NovaFiberMeta`.
2. Refactor `nova_register_effect_storage` to assign stable index
   per effect (used to index into snapshot[]).
3. Implement `nova_handler_snapshot_init/restore/save` per §3.4.
4. Hook `restore_from_meta` BEFORE `mco_resume` in both
   `_worker_main` AND `nova_supervised_step`.
5. Hook `save_to_meta` AFTER `mco_resume` returns.
6. Remove old TLS save/restore из `supervised_step` (legacy mechanism).
7. Targeted: `per_fiber_handlers` + `time_handler` под explicit
   `runtime.init(2)`.
8. **Cross-effect test**: spawn nested `supervised{ with handler1 { spawn { with handler2 { ... } } } }`
   — outer/inner handler-state корректно scope'ится.

### Ф.3 — Stress + memory-ordering audit (~1 dev-day)

1. **Stress harness** in `83.4.2-artifacts/`: 1000-fiber supervised
   с work-stealing + random handler-push, без leak/crash.
2. **Profile** `mn_runtime_cpu_workload` to ensure status-CAS не
   стало новым contention hot path (compare BEFORE/AFTER).
3. **Memory ordering review** (paper-style): document every status-CAS
   call с указанием happens-before reasoning. Анализ similar to
   Plan 44.1 R-* audit.
4. **TSAN on Linux** (если возможно): `-fsanitize=thread` на
   `supervised_*` + `per_fiber_handlers` тестах — 0 reported races.

## 6. Риск

| Риск | Mitigation |
|---|---|
| Handler-storage refactor ломает algebraic effects (фундамент языка) | Incremental: Ф.1 (ownership) изолирован; Ф.2 (handler) — targeted-тест per-effect; full nova test после каждого |
| `_Alignas(CACHELINE)` на каждом fiber'е увеличит memory footprint | `NovaFiberMeta` — один блок per-fiber; cache-line align = ~128B per fiber; 10k fiber'ов = 1.25 MB накладных — приемлемо |
| Atomic status CAS под high steal rate → contention | Профиль до/после; если contention визуально проявляется — рассмотреть exponential backoff в steal-loop (Plan 44.5 уже его имеет) |
| Effect-index migration breaks user-registered effects | `nova_register_effect_storage` уже регистрирует — индекс просто становится стабильным; обновить comment + assert (что max не превышен) |
