# Plan 169.2.1 — Развязать `prelude.core` от `std.unicode`

> **Под-план [Plan 169.2](169.2-nova-tests-fix-sweep.md)** (ось «корректность»).
> **Создан:** 2026-06-19. **Статус:** 📋 proposed — execution-ready для агента.
> **Приоритет:** P1 — чинит plan107 `#prelude(partial)` (корень
> `[M-169.2-prelude-partial-stdlib-graph]`).

## Проблема (корень plan107)

`#prelude(core, runtime)` падает: `[D133-not-consumed] sb`. Цепочка:
1. `core.nv:78 import std.unicode` тянет ВЕСЬ unicode folder-module (14 файлов).
2. `std/unicode/normalize.nv::cps_to_str` использует `consume sb =
   StringBuilder…; sb.as_str()`.
3. D133 (type-check, ДО codegen/DCE) проверяет инлайненный `cps_to_str`.
4. StringBuilder consume-методы — в `collections`, которого при partial
   `#prelude(core, runtime)` НЕТ → объявление не подтянуто → D133 видит `sb`
   не-consumed → падение.

`core` тянет весь unicode ради **8 char-методов** (`'a'.is_alphabetic()` и т.д.,
`core.nv:466-497`), которые делегируют в unicode-предикаты. Это намеренный цикл
`prelude.core → std.unicode → std.collections → prelude.core` (`core.nv:74`).

NB: НЕ cross-module-резолв и НЕ DCE — это неполнота транзитивных зависимостей
при partial prelude + полный type-check инлайненного кода. (DCE работает на
codegen, ПОСЛЕ падения в type-check.)

## Решение (утверждено 2026-06-19)

char-unicode-методы → из `core` в `unicode`; «доступность без import» обеспечить
через **полный prelude** (реэкспорт), а не через core. **`core` становится чистым.**

1. Перенести 8 char-методов из `std/prelude/core.nv` (строки ~466–497:
   `@is_alphabetic`/`@is_numeric`/`@is_alphanumeric`/`@is_whitespace`/
   `@is_uppercase`/`@is_lowercase`/`@to_uppercase`/`@to_lowercase`) в
   `std/unicode/` (откуда их перенёс Plan 162 Ф.4 — обратный ход).
2. Убрать `import std.unicode` из `core.nv` → core не зависит от unicode.
3. В `std/prelude.nv` (rolling facade) добавить реэкспорт этих char-методов из
   unicode → обычный код (полный prelude) видит `'a'.is_alphabetic()` без import.

## Результат

- **Обычный код** (полный prelude): char-методы доступны без import (реэкспорт) —
  `plan159/f4_no_import_char_methods` цел.
- **`core` сам**: не тянет unicode → `#prelude(core)` не тянет normalize/
  StringBuilder → plan107 D133 **не падает**.
- **partial `#prelude(core)`**: char-методы недоступны (unicode не подтянут),
  но код unicode туда не инлайнится → ошибки нет (ожидаемо для урезанного prelude).
