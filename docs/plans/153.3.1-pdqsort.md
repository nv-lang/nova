# Plan 153.3.1 — pdqsort: замена heapsort в `@sort_unstable*`

> **Создан:** 2026-06-18. **Статус:** ✅ CLOSED 2026-06-18.
> **Коммиты:** 30139703, 4a77bc92 (ветка plan-153.3.1-pdqsort)
> **Закрывает:** `[M-153.3-sort-pdqsort]` + `[M-91.8c-pdq-sort]`.
> **Тип:** perf-only — API не меняется, семантика не меняется.
> **Зависит от:** Plan 153.3 ✅.

## Проблема

Plan 153.3 реализовал `@sort_unstable*` на основе **heapsort**: алгоритм корректный,
O(n log n) worst-case, но медленный на практике из-за плохой cache-локальности
(доступ к элементам в heap-порядке, не sequential). Стандартный выбор для
«быстрой нестабильной сортировки» в современных языках — **pdqsort** (pattern-defeating
quicksort), который быстрее heapsort на типичных данных при сохранении
O(n log n) worst-case гарантии.

Два followup-маркера (`[M-153.3-sort-pdqsort]` и `[M-91.8c-pdq-sort]`) фиксировали
это как отложенное улучшение.

## Scope

**В scope:**
- Замена внутреннего алгоритма `@sort_unstable` / `@sort_unstable_by` / `@sort_unstable_by_key`
  с heapsort на pdqsort.
- Новые тесты на корректность (random/sorted/reversed/duplicates/nearly-sorted/generic/large).
- Негативные тесты на ошибки типов при неправильном использовании.
- D185 §algorithm amend в spec.

**Вне scope:**
- Стабильная сортировка `@sort` / `@sort_by` / `@sort_by_key` — не затронута.
- Изменения API (сигнатуры остаются прежними).
- Perf-бенчмарки (потребуют отдельной инфраструктуры).
- SIMD/branchless оптимизации.

## Алгоритм (итеративный, без рекурсии)

Реализован **итеративный pdqsort** в `std/collections/vec/sort.nv`:

1. **n ≤ 1** → return immediately (ничего делать не нужно)
2. **n ≤ 16** → insertion sort (cache-warm, O(n²) но быстро для малых массивов)
3. **depth_limit = 2·ilog2(n) исчерпан** → heapsort fallback (O(n log n) worst-case гарантия,
   защита от adversarial inputs)
4. **Иначе:** median-of-3 pivot (lo, mid, hi-1) + Lomuto partition + итеративный стек

**Стек:** `Vec[int]` хранит пары `(lo, hi)` как consecutive int's (lo at i×2, hi at i×2+1).
Это избегает рекурсию и heap allocation структур.

**Вспомогательные функции:**
- `_pdq_ilog2(n)` — целочисленный log₂ через while-цикл
- `Vec[T] mut @_ins_sort_range(lo, hi, cmp)` — insertion sort на диапазоне [lo, hi)
- `Vec[T] mut @_pdqsort(cmp)` — основной driver
