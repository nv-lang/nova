## 2. Design

### 2.1 Global CLI flags

Added к `struct Cli` (clap global = true). Translated to env vars
in `apply_field_cache_cli_flags` before subcommand dispatch.

Flags:
- `--no-field-cache`, `--no-field-cache-licm`,
  `--no-field-cache-pure`, `--no-field-cache-chain`,
  `--no-field-cache-ipa` — disable per-layer.
- `--field-cache-threshold=N`, `--field-cache-licm-threshold=N`,
  `--field-cache-pure-threshold=N`,
  `--field-cache-chain-threshold=N` — thresholds.
- `--field-cache-max=N`, `--field-cache-licm-max=N`,
  `--field-cache-chain-depth=N` — caps.

CLI overrides env vars when both present.

### 2.2 CI perf gate

`nova check --telemetry-cache --telemetry-baseline=baseline.json`:
1. Computes current telemetry.
2. Parses baseline JSON (minimal hand-rolled parser, no serde dep).
3. Compares `methods_affected_pct` (absolute drop > 5pp = fail).
4. Compares `caches_total` (relative drop > 10% = fail).
5. Exit 1 on regression; exit 0 OK.

CI workflow:
```yaml
- run: nova check src/ --telemetry-cache --telemetry-json > metrics.json
- uses: actions/upload-artifact@v3
  with: { name: metrics, path: metrics.json }
# In next PR:
- run: nova check src/ --telemetry-cache --telemetry-baseline=metrics.json
```
