## Цель
Эмит **frame-struct + push/pop + write-back** перед safe-point'ами для функций с heap-локалами,
живущими через safe-point. Инварианты §7.3 (1) init-before, (2) write-back-before, (4)
pop-on-every-exit. Pop на ВСЕХ exit-путях (incl. `?`/Bang error-path, panic-unwind, defer).

**Довести до тира O1** (§7.5: #1 frame-elision если нет корня через safe-point + #2
effect-driven safe-points (`#no_gc`/pure-вызов НЕ safe-point — потребитель may-GC из Plan 144.0)
+ #3 per-safepoint live-set). **O0→O1 — НЕ оптимизация ради скорости, а условие
жизнеспособности схемы** (без неё каждый вызов = safe-point с write-back).

## Acceptance criteria
- Кадр — **авторитетный источник**: тест, где Boehm conservative ложно удерживает (integer
  похож на указатель / dead-but-on-stack) → точная сборка.
- **Pop-discipline** тест: early-return + panic + defer не ломают цепочку.
- **O1-проверка**: leaf/`#no_gc`-вызовы НЕ порождают кадр/write-back.
- AC перфа (Ф.5) меряется на **O1+**, не на O0.
- «без упрощений как для прода» — обязательный критерий.

## Дизайн-источники
