## Зачем

Plan 96 оставил два честно-задокументированных deferred-пункта.
Plan 96.1 закрывает оба:

### 1. `W_VIEW_PUSH_DETACH` lint

`let mut view = arr[1..4]; view.push(99)` — push на view детачит от
parent backing'а (cap==len model). Это **корректное** поведение
(anti-Go-`append`-footgun), но silent — программист ожидает
«push добавит к arr». Lint warning эмитит подсказку:

```
warning W_VIEW_PUSH_DETACH: mut view's push detaches from parent backing;
        parent NOT modified. Use parent directly to grow, or `Array.from(view)`
        to make the detach explicit.
```

### 2. Удалить `str.@slice` — bracket-only (`s[a..b]`)

Сейчас две формы slice для строк:
- `s.slice(a, b)` — старый метод, OOB → **clamp**
- `s[a..b]` — новый bracket (Plan 96), OOB → **panic**

Inconsistency + D9 violation. Решение — удалить метод, оставить bracket.

**Тренд модерн-языков:** Rust/Go/Swift/Python — только bracket; Java/JS —
только метод (legacy). Современная конвергенция → bracket-only.

| Язык | `.slice()` | `s[a..b]` | OOB |
|---|---|---|---|
| Rust | ❌ | ✅ `&s[a..b]` | panic |
| Go | ❌ | ✅ `s[a:b]` | panic |
| Swift | ❌ | ✅ `s[a..<b]` | crash |
| Python | ❌ | ✅ `s[a:b]` | clamp |
| Java | ✅ `substring` | ❌ | throws |
| JS/TS | ✅ `.slice` | ❌ | clamp |
| Nova (цель) | ❌ | ✅ `s[a..b]` | panic |

## Scope

**Входит:**
- `W_VIEW_PUSH_DETACH` lint в `compiler-codegen/src/lints.rs` —
  walker трекает биндинги `let X = obj[Range]` и эмитит warning
  при `X.push(...)`.
- Удаление `nova_str_slice` (clamp-версия) из `nova_rt.h` —
  остаётся только `nova_str_slice_panic` (Plan 96).
- Удаление `external fn str @slice(from int, to int) -> str` из
  `std/runtime/string.nv`.
- Удаление `str_method_to_rt`-mapping для `slice` в codegen.
- Миграция ~30 call-sites `s.slice(a, b)` → `s[a..b]` в `std/`.
- Миграция `nova_tests/str/slice_oob.nv` (тестит clamp — пересматриваем
  или удаляем) + `nova_tests/types/str_search.nv`.
- Spec D26 update — убрать упоминание `@slice`.
- Тесты pos/neg lint (3-4 теста).

**Не входит:**
- Перенос остальных str-методов на Nova (это Plan 94 scope).
- Любые изменения семантики `arr[a..b]` (Plan 96 final).

## Risks

1. **Clamp-зависимые сайты в std.** Если какой-то `s.slice(0, 100)`
   реально полагался на clamp (`s.len() < 100`) — миграция на bracket
   упадёт runtime panic'ом. Mitigation — Ф.2 аудит явно ищет такие
   случаи + добавляет `min(len, ...)` guard'ы.
2. **Lint false-positive на не-view биндингах.** Если `let X =
   func()`, где func возвращает обычный массив (не view) — push не
   детачится; lint не должен warning'ать. Mitigation — Ф.1 walker
   трекает **только** биндинги с RHS=Index-with-Range; обычные fn-
   возвраты не попадают.
3. **`@slice` в user-коде вне std.** Если пользователи (после релиза)
   писали `s.slice(...)` — breaking change. Mitigation — у нас
   bootstrap, нет внешних пользователей; migration внутри.
