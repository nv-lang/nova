## 1. Контекст

V6.1 закрыл CI perf gate machinery с двумя hardcoded thresholds
(5pp affected drop, 10% caches drop). V7 закрыл IPA с hardcoded
iterative-closure cap = 10. V6.3 promotes all три к env+CLI override.

---

## 2. Изменения

### 2.1 FieldCacheConfig

- Новое поле `ipa_iter_limit: usize` (default 10).
- Env var `NOVA_FIELD_CACHE_IPA_ITER` (clamp 1..=1024; n=0 silently
  ignored, не сбрасывает default — escape hatch для IPA-off это
  `NOVA_FIELD_CACHE_IPA=0`).

### 2.2 build_{write,read}_set_registry

Принимают `iter_limit: usize` параметр. Public API
`module_write_sets(module)` сохраняет старую signature (использует
`FieldCacheConfig::default().ipa_iter_limit`).

### 2.3 CLI

Три новых global flag:
- `--field-cache-ipa-iter=N` → `NOVA_FIELD_CACHE_IPA_ITER`
- `--telemetry-gate-affected-drop=F` (passed как Option<f64>)
- `--telemetry-gate-caches-drop=F` (passed как Option<f64>)

`cmd_check_telemetry_cache` signature:
- Добавлены `gate_affected_drop_override: Option<f64>` и
  `gate_caches_drop_override: Option<f64>` параметры.
- `unwrap_or(5.0)` / `unwrap_or(10.0)` сохраняют V6.1 defaults.
- Non-negative validation — отрицательные thresholds rejected.

### 2.4 Messaging

V6.1 perf-gate OK line теперь echoes active threshold values:
```
V6.1 perf gate: OK (vs baseline foo.json; gates -5.00pp affected / -10.0% caches)
```
