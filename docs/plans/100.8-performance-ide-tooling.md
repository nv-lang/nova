## Цель

Закрыть production developer experience gap:
1. **Compile-time performance** — `check_consume` pass (Plan 100.1)
   + расширения (100.2/3/4) добавляют work. Bench + budget.
2. **IDE / LSP integration** — quick fixes (auto-add consume marker,
   auto-add errdefer, auto-add `consume tx =`); hover info; type-
   driven completion.
3. **`nova doc` integration** — consume-status visible в generated
   docs.
4. **Diagnostic ergonomics** — error format spec для AI-first +
   human-readable.
5. **Bench infrastructure** — Plan 57 bench integration; consume-
   pass standalone benchmarks.

Без 100.8 — Plan 100 «works» но developer experience plateaus.
Production требует **best-in-class** error messages + IDE help.

## Дизайн-решения

### D1 — Compile-time performance budget

`check_consume` pass + extensions overhead targets:

| Metric | Budget |
|---|---|
| `check_consume` pass standalone | < 5% от total `nova check` time |
| Memory overhead | < 10MB per 100k SLOC project |
| Worst-case complexity | O(N × depth) где depth = nested field-paths |
| Incremental compilation | minimal-rerun on per-fn basis |

Verification — Plan 57 bench infrastructure (Plan 57 ✅).

### D2 — LSP quick fixes (production-grade list)

| Diagnostic | Quick fix |
|---|---|
| E (D133-not-consumed) | "Add `consume tx = ...` binding; add `tx.<method>()` at exit" |
| E (D133-field-marker-missing) | "Add `consume` to field declaration" |
| E (D133-type-marker-missing) | "Add `consume` to type-decl line" |
| E (D133-assign-live-field) | "Add `@.field.<consume-method>()` before assignment" |
| E (D156-strict-forget) | "Bind value to `_` only if non-consume; otherwise consume via method" |
| E (D157-consume-via-view) | "Use `consume` qualifier instead of `view`" |
| E (D157-view-escape-return) | "Return value, not view; or change return-type" |
| E (D158-defer-fail-not-in-sig) | "Add `Fail[E]` to enclosing function signature" |
| E (D162-uncovered-success-path) | "Add `tx.commit()` or `okdefer { tx.commit() }`" |
| E (D162-uncovered-error-path) | "Add `errdefer { tx.rollback() }`" |
| E (D163-missing-cap) | "Add `needs Fs` (or appropriate capability) to declaration" |
| E (D164-cross-pkg-visibility) | "Add `export` to type declaration in source package" |

Каждая quick-fix имеет machine-applicable edit (Plan 50 D102 pattern).

### D3 — Hover info format

При hover на consume-typed binding:

```
consume tx: Transaction
  Status: Live (consume obligation pending)
  Type defined in: a/types.nv:5 (package 'a' v1.0)
  Consume methods:
    .commit() -> Result[(), CommitErr]
    .rollback() -> Result[(), RollbackErr]
  Coverage:
    Line 7: `errdefer { tx.rollback() }` covers error-path
    Success path: NOT COVERED (suggest okdefer or explicit commit)
```

Comprehensive в одном hover — type + status + coverage + suggestions.

### D4 — `nova doc` integration (Plan 45)

- Section "Resource lifecycle" — open/use/close pattern.

Plan 45 `nova doc` already extensible; Plan 100.8 adds consume-specific

### D5 — Diagnostic format spec

Все Plan 100 errors follow structured format:

```
error[D133-not-consumed]: consume value not consumed before scope exit
   ╭─[file.nv:8:5]
 5 │     consume tx = begin()
   │     ━━━━━━━━━━━━━━━━━━━━ consume binding declared here
 8 │ }
   │ ━ scope exit; tx still Live
   ╰─
   note: type `Transaction` declared `consume` at types.nv:3
   help: consume via one of:
         .commit() -> Result[(), CommitErr]   (success path)
         .rollback() -> Result[(), RollbackErr] (cleanup)
   suggestion (machine-applicable):
         + errdefer { tx.rollback() }
         + tx.commit()?
```

Format: error-code (machine), title (human), source-context (with
markers), notes (additional context), help (consume-methods list),
suggestion (auto-fix template).

JSON variant для LSP/CI:
```json
{
  "code": "D133-not-consumed",
  "severity": "error",
  "ranges": [{ "file": "file.nv", "line": 8, "col": 5 }],
  "suggestions": [
    { "kind": "machine-applicable", "edits": [...] }
  ]
}
```

## Границы

- **AI-driven diagnostic enrichment** (e.g., suggestion ranking via
  LLM) — out of scope; bootstrap deterministic suggestions.
- **Profiling hot-path within check_consume** — bootstrap measures
  total; future plan deep-profiling.
- **IDE extension distribution** — handled by nova-vscode / nova-jetbrains
  extension projects (out of scope для Plan 100.8).

## Risks

1. **Plan 57 bench framework limits** — может быть недостаточно
   precision для < 5% measurement. **Митигация:** averaging over
   multiple runs; Plan 57 hyperfine integration.
2. **LSP performance** — quick-fix evaluation costly. **Митигация:**
   lazy evaluation, cached on incremental builds.
3. **Diagnostic format breaking change** — existing CI tools rely on
   error format. **Митигация:** dual-output (legacy + structured)
   transition period.
