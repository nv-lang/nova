# Plan 83.5: Boehm `THREAD_LOCAL_ALLOC` — снять Boehm global lock на spawn

> **Создан:** 2026-05-24 (out of Plan 83 audit, см. [83-audit-2026-05-24.md](83-audit-2026-05-24.md) §1 пункт 1).
> **Статус:** 📋 proposed.
> **Приоритет:** **P0 — single biggest perf win по аудиту.**
> **Оценка:** ~1 dev-day.
> **Родитель:** [Plan 83](83-mn-default-on-gomaxprocs.md).
> **Зависимости:** нет (build chain change только).

---

## 1. Проблема

Per [Boehm GC scale.html](https://www.hboehm.info/gc/scale.html):

> "only one thread can be allocating or collecting at one point. This
> inherently limits performance of multi-threaded applications on
> multiprocessors."

`nova_alloc` и **`nova_alloc_uncollectable`** (Plan 83.4.5.8) под Boehm
бекендом сериализуются через global allocation mutex. Под 16-worker
armed M:N это даёт ~50-200µs per spawn (см. Plan 83.4.5.10 §2 corner
cause #1).

**Boehm имеет встроенное решение** — `THREAD_LOCAL_ALLOC` + per-thread
free-lists для common size classes. По собственной документации Boehm:

> "Enable `THREAD_LOCAL_ALLOC` + `GC_REDIRECT_TO_LOCAL`, call
> `GC_thr_init()` — greatly reduces locking."

**Verified отсутствие в Nova:** Grep `THREAD_LOCAL_ALLOC` по
`compiler-codegen/nova_rt/` — НЕ найдено (только в Dockerfile/README).
`alloc_boehm.c:61-78` — `GC_INIT()` без thr_init call, без redirect macros.

## 2. Корневая причина почему упущено

При initial Boehm wire-up (Plan 27, Plan 44.2) фокус был на correctness:
GC roots, fiber arena integration, STW под multi-worker. Performance
optimization Boehm-build chain не делалась — все spawn cost-investigations
указывали на mco_create / uv_async_send / arena alloc, но не на Boehm
internal locking pattern.

В Plan 83.4.5.10 §2 это пункт **#1 (PRIMARY bottleneck)**, но fix предлагался
**через per-worker SpawnCtx pool** (Ф.1), а не через включение Boehm own
thread-local alloc. Оба нужны, но `THREAD_LOCAL_ALLOC` — дешевле в 5-10× и
закрывает не только SpawnCtx allocation, но **все** Nova alloc'ы под
contention.

## 3. Дизайн

### Ф.1 — Boehm rebuild с THREAD_LOCAL_ALLOC (~0.5 dev-day)

**Windows (vcpkg):**
- `ports/bdwgc/portfile.cmake` patch — add CMAKE_FLAGS:
  ```
  -Denable_thread_local_alloc=ON
  -Denable_parallel_mark=ON
  -Denable_threads=ON
  ```
- Rebuild vcpkg port → `vcpkg_installed/x64-windows-static/lib/gc.lib`
  refresh.

**Linux:** Ubuntu system `libgc-dev` уже built with `PARALLEL_MARK` (verified
в `alloc_boehm.c:83-88`), но **`THREAD_LOCAL_ALLOC` под вопросом** — check
`pkg-config --variable=defines gc`. Если нет — build from source.

**macOS:** Aналогично через brew или from source.

### Ф.2 — Defines в `alloc_boehm.c` (~0.1 dev-day)

```c
/* nova_rt/alloc_boehm.c — modifications */

/* Plan 83.5: thread-local alloc fast path.
 * Per Boehm scale.html: per-thread size-class free lists, lock-free на
 * common allocations. Closes Plan 83.4.5.10 §2 root cause #1 (Boehm
 * global allocation lock). Available only если libgc built with
 * `--enable-thread-local-alloc` (vcpkg port flag для Windows).
 */
#define GC_THREAD_ISOLATED  /* per-thread heaps for collector workers */

/* MUST be defined BEFORE #include <gc.h>.
 * Effect: GC_malloc redirects to GC_malloc_local (lock-free TL path)
 * for sizes < GC_MAX_LOCAL_SIZE (default 16 free-list buckets).
 */
#define GC_REDIRECT_TO_LOCAL

#include <gc.h>

void nova_gc_init(void) {
    GC_set_no_dls(1);
    GC_INIT();
    /* Plan 83.5: thread-local allocation buffers — MUST call thr_init
     * AFTER GC_INIT for redirect macros to wire up. Boehm requires this
     * sequencing per <gc.h> docs.
     *
     * Effect: subsequent GC_malloc calls на этот thread берут из
     * per-thread free-lists без global mutex. Fallback на global lock
     * только при cache miss (rare).
     */
#ifdef GC_THREADS
    GC_thr_init();
#endif
    GC_set_all_interior_pointers(1);
}
```

**Per-worker `GC_register_my_thread`** уже есть (runtime.c:335-343) —
после `thr_init` Boehm активирует per-thread cache на каждый
зарегистрированный worker автоматически.

## 5. Тесты

### Positive (новые):

**`bench/m_n/spawn_microbench.nv`** — minimal spawn cost measurement:
```nv
module bench_m_n.spawn_microbench

// Bench: 10⁵ empty spawn'ов; measure throughput (spawn/sec).
// Acceptance per Plan 83.5: ≥10⁵ spawn/sec на 16-core (current: ~10³).
bench "10⁵ empty spawn throughput" {
    let n = 100_000
    let mut counter = 0
    measure {
        supervised {
            let mut i = 0
            while i < n {
                spawn { counter += 1 }
                i = i + 1
            }
        }
    }
    assert(counter == n)
}
```

**`nova_tests/plan83_5/gc_thread_local_alloc_correctness.nv`** —
correctness под high allocation parallelism:
```nv
module plan83_5.gc_thread_local_alloc_correctness

test "16 workers × 1000 alloc parallel, checksum correct" {
    let n = 1000
    let mut sum = 0
    supervised {
        let mut i = 0
        while i < n {
            let cur = i
            spawn {
                // Force allocation — record value goes на managed heap.
                let r = make_record(cur)
                sum += r.value
            }
            i = i + 1
        }
    }
    // sum = 0 + 1 + ... + (n-1) = n*(n-1)/2
    assert(sum == n * (n - 1) / 2)
}

record Box { value: int }
fn make_record(v: int) -> Box { Box { value: v } }
```

### Negative (новые):

**`nova_tests/plan83_5/gc_unregistered_thread_alloc.nv`** —
EXPECT_RUNTIME_ERROR (если test runner поддерживает FFI thread spawn
out of nova runtime). Если нет — skip с пометкой "requires FFI test
runner extension".

**`nova_tests/plan83_5/gc_concurrent_collect_safety.nv`** — verify
explicit `GC.collect()` под spawn pressure не race'ит:
```nv
module plan83_5.gc_concurrent_collect_safety

test "explicit collect during heavy spawn no crash" {
    use std::runtime::gc
    supervised {
        let mut i = 0
        while i < 1000 {
            spawn {
                let _r = make_garbage()  // allocate temp
                if i mod 100 == 0 {
                    gc.collect()  // explicit collect during alloc
                }
            }
            i = i + 1
        }
    }
    // Must complete without crash/deadlock.
}

fn make_garbage() -> Box { Box { value: 42 } }
record Box { value: int }
```

### TSAN gate:

`scripts/tsan_concurrency.sh` (existing) — add `gc_thread_local_alloc_*`
к covered tests. Verify 0 races.

## 6. Риски

- **Boehm THREAD_LOCAL_ALLOC compatibility с GC_THREADS** — verified
  в Boehm docs они designed to coexist, но нужно verify rebuild
  работает корректно с existing `GC_register_my_thread` calls.
- **vcpkg port modification** — нужно либо patch upstream port (PR), либо
  держать local override (`vcpkg/ports/bdwgc/portfile.cmake` patch).
  Last-resort: build Boehm from source в `third_party/bdwgc/`.
- **Linux system libgc может не иметь TL_ALLOC** — fallback: build
  from source в worker.
