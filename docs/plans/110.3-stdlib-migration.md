# Plan 110.3: Stdlib migration to `Consumable[E]` (Ф.4 + Ф.5)

> **Создан 2026-05-31.** Sub-plan Plan 110 §«Возможный split». Реализует
> Ф.4 (core) + Ф.5 (extended) stdlib resource migration.
>
> **Статус:** ✅ ЗАКРЫТ (2026-06-01) — sub-plan Plan 110 umbrella (merge `874f5766ca5`). Phases shipped через atomic commits; close summary в `1c09f315f02`. Fixtures в `nova_tests/plan110/` верифицируют integrated behavior. Status update metadata-only commit 2026-06-03.
>
> **Цель:** все stdlib resource-like типы implement `Consumable[E]`:
> Mutex/RwLock/Semaphore family → `Consumable[Never]` (D194 hot-path);
> File/Transaction/BufReader/Writer → `Consumable[IoError]` /
> `Consumable[DbError]`; TCP/UDP/Channel → `Consumable[IoError]` или
> `Consumable[Never]`.
>
> **D-блоки:** активирует D188 (Consumable impls), D192 (`WithExitTimeout`
> impls для Transaction/TcpStream).
>
> **Acceptance:** A5, A6, A8 из Plan 110.
>
> **Зависимости:** Plan 110.1 (compiler pipeline); Plan 103.3/103.4
> (Mutex/Sem family ready); Plan 65 (Channels); Plan 83.12 (TCP/UDP).

## Фазы

### Ф.4 — Stdlib core resources

- **Ф.4.1** `Transaction` (если std/db landed, иначе → `[M-110-stdlib-db]`)
  — `Consumable[DbError]`, commit/rollback по outcome.
- **Ф.4.2** `File` (если std/fs landed, иначе → `[M-110-stdlib-fs]`)
  — `Consumable[IoError]`.
- **Ф.4.3** `MutexGuard`, `ReadGuard`, `WriteGuard`, `ReentrantGuard` —
  `Consumable[Never]` (D194 hot-path).
- **Ф.4.4** `Permit` (Semaphore) — `Consumable[Never]`.
- **Ф.4.5** `BufReader` / `BufWriter` (если std/bufio landed, иначе →
  `[M-110-stdlib-bufio]`) — `Consumable[IoError]`.
- **Ф.4.6** `CancelScope` (Plan 49) — `Consumable[Never]`.
- **Ф.4.7** Tests T4.1–T4.6.

### Ф.5 — Stdlib extended resources

- **Ф.5.1** `TcpStream` / `TcpListener` / `UdpSocket` (Plan 83.12) —
  `Consumable[IoError]`.
- **Ф.5.2** `Channel` / `ChanReader` / `ChanWriter` (Plan 65) —
  `Consumable[Never]`.
- **Ф.5.3** `JoinHandle` / fiber handles (Plan 83.4.2 если ready) —
  `Consumable[Never]`; иначе → `[M-110-supervised-handle]`.
- **Ф.5.4** `Stream` (Plan 84 если exists) — `Consumable[E]`.
- **Ф.5.5** Connection pools — `Consumable[ConnPoolError]`; иначе →
  `[M-110-stdlib-pool]`.
- **Ф.5.6** Tests T5.1–T5.5.

## Notes

`File` / `Transaction` / `BufReader` / `BufWriter` may not exist yet as
concrete std types at execution time. Audit shows main is **clean** of
these types — they're conceptual roadmap (per Plan 91 stdlib MVP).
Sub-plan executor должен **first check** какие типы actually existují в
main и migrate только existing; missing типы → отдельные `[M-110-stdlib-*]`
follow-up плану.

## Acceptance

A5 (core stdlib), A6 (extended), A8 (Consumable[Never] для locks).

## Оценка

~2-3 dev-day. Parallel split возможен (4-6 agent'ов на отдельные resource
families: Mutex, File, TCP, Channels). Recommended: Opus 4.7 final synth.
