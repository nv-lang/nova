## Зачем

GitHub CI сейчас полный регресс `nova_tests` **не гоняет** (только
`contracts-z3` = `--filter contracts` + `nova-doc`). Поэтому поломки

## Ф.1 — CLI category-селекторы

### Модель (аддитивные селекторы, OR)

Две независимые оси теста:
- **Тип** (взаимоисключающий, по `EXPECT_*` маркеру заголовка; positive = нет маркера):
  positive · compile-error · panic · timeout · exit.
- **Скорость** (по суффиксу `*_slow.nv`, Plan 156): fast · slow.

Флаги (каждый = «добавить категорию», результат = объединение):
```
nova test                        → positive ∩ fast  (дефолт, быстрый)
nova test --positive             → то же явно
nova test --slow                 → все *_slow.nv
nova test --compile-error        → только EXPECT_COMPILE_ERROR
nova test --panic                → только EXPECT_RUNTIME_PANIC
nova test --timeout              → только EXPECT_TIMEOUT
nova test --exit                 → только EXPECT_EXIT
nova test --slow --compile-error → объединение
nova test --full                 → всё (positive+slow+compile-error+panic+timeout+exit)
```

### Реализация

- **Фильтр по МАРКЕРУ, не по папке.** Факт (перепись 169.1): 128 compile-error +
  27 panic лежат ВНЕ `neg/`. Фильтр по `neg/`-папке их пропустит. Маркер
  (`// EXPECT_COMPILE_ERROR` / `EXPECT_RUNTIME_PANIC` / `EXPECT_TIMEOUT` /
  `EXPECT_EXIT`) уже парсится в `test_runner.rs` (см. `parse_expect_*`).
- Обобщить `SlowLane` (Plan 156, `test_runner.rs:2633` `slow_lane: SlowLane`) в
  набор независимых category-флагов. Discovery (`walk_nv_filtered`) фильтрует
  файл по (тип-маркер ∈ выбранные) ∨ (slow ∈ выбранные).
- Дефолт без флагов = positive ∩ fast (текущее минус neg/panic/timeout/exit).
- `--full` = все категории.
- Взаимодействие: `*-only`-семантики НЕ нужно (модель аддитивная) — каждый флаг
  добавляет; несколько флагов = объединение. `--full` = шорткат «все».
- CLI: `nova-cli/src/main.rs` — добавить флаги в `Test`-подкоманду; пробросить
  в `TestAllOpts`/`TestBuildOpts`.

### Тесты Ф.1

- `nova test --compile-error` гонит только compile-error (проверить число).
- `nova test --panic` — только panic.
- `nova test --slow --panic` — объединение.
- `nova test` (дефолт) НЕ включает neg/panic/timeout/exit.
- `nova test --full` — всё.

## Ф.2 — CI авто-регресс (`.github/workflows/nova-test-regression.yml`)

- **PR / push** (на каждый коммит, быстро): сборка nova-cli + `nova test`
  (positive-fast) + `nova test std`. Падение → red CI. Закрывает дыру
  молчаливого накопления.
- **Nightly** (`on: schedule: cron`, раз в сутки): `nova test --full` +
  `nova test std`. Полный прогон, отчёт о падениях (artifact с
  `--results-file`).
- Учесть таймаут джоба (полный прогон ~час — см. project-bash-timeout заметку;
  GitHub job-timeout до 6ч, ОК; дробить при необходимости по группам).
- Toolchain: clang (как локально); env NOVA_GC_LIB_DIR/INCLUDE_DIR если нужно.

Зависимость: Ф.2 использует `--full` из Ф.1 → Ф.1 делать первой.
