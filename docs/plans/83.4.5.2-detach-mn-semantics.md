## 1. Bug + симптом

**Тест:** `detach_test` (вероятно ещё `cancel_pattern_var_no_leak_test`).

**Симптом под активным flip:**
Тест запускает `detach { body }` и проверяет наблюдаемые side-
effects body'я. Под bootstrap'ом `SyncDetach` default-handler
исполняет body **инлайн в потоке caller'а** — side-effects видны
сразу. Под flip'ом ожидание меняется: detach должен быть **fire-
and-forget на worker**, не инлайн. Test'ы, которые опираются на
inline-execution, регрессируют.

## 2. Cross-runtime reference

### Go `go fn()`

```go
go func() { /* fire-and-forget */ }()
```

- Параллельно: goroutine отправляется на scheduler runq.
- Caller возвращается мгновенно.
- Goroutine переживает caller (если main не вышел).
- Никакого implicit join'а; explicit sync — `sync.WaitGroup`, channel.

### tokio `tokio::spawn(future)`

```rust
let handle = tokio::spawn(async { /* fire-and-forget */ });
drop(handle);  // task продолжает работать после drop
```

- Future отправляется на multi-thread executor.
- `JoinHandle` возвращается; если caller drop'нет — task продолжает.
- `JoinSet` для structured-await; иначе detached.

### Kotlin `launch { … }`

```kotlin
val job = scope.launch { /* fire-and-forget */ }
// без job.join() — fire-and-forget (но scope joins на close)
```

- В `CoroutineScope`; если scope cancels → child cancels.
- Без scope — `GlobalScope.launch { … }` truly detached.

### Node `setImmediate(cb)` / queueMicrotask

```js
setImmediate(() => { /* fire-and-forget */ });
queueMicrotask(() => { /* same, microtask */ });
```

- Single-thread; callback ставится в event-loop queue.
- Не инлайн (один turn loop'а минимум).

### Nova D50 / bootstrap

D50 spec: `detach { body }` — fire-and-forget, привязан к
**глобальному supervisor'у** (default `LogAndDrop`).

Bootstrap impl: `SyncDetach` handler — **inline в caller'е**.
Тесты пишут body со side-effect'ами и проверяют их сразу после
detach-выражения. Inline-семантика для теста удобна, но НЕ соответствует
spec («fire-and-forget» — caller не должен ждать body).

## 3. Root cause

Codegen `detach { body }` (`emit_c.rs::ExprKind::Detach`) при
bootstrap'е делегирует через built-in `Detach` effect default-
handler'у. Default = `SyncDetach` который исполняет body инлайн.

Под flip:
- Tests ожидают side-effects видимыми сразу.
- Если бы default стал async (`AsyncDetach` через worker spawn) —
  side-effects не видны до того, как worker подхватит fiber и
  выполнит. Test'ы регрессируют.
- Но spec D50 — semantic — fire-and-forget. Inline было bootstrap
  упрощением.

## 4. Дизайн решения

### Опция A — default остаётся `SyncDetach` после flip

Detach под M:N всё ещё inline в caller'е (детач = vibrate
коротко). Тесты не регрессируют. Side-effects видны сразу.
**Минус:** теряется параллельность detach'а — fire-and-forget
де-факто sync. Spec D50 не выполнен в spirit.

### Опция B — flip default на `AsyncDetach`

```nova
// AsyncDetach: spawn body на worker pool, return immediately.
fn Detach.detach(body fn() -> ()) {
    runtime.spawn_orphan(body)  // worker spawn, no scope binding
}
```

Тесты переписать на explicit sync (например wait через channel /
`runtime.yield()` затем check).

**Минус:** ABI change; existing tests рассчитаны на inline.
Migration работа.

### Опция C — context-sensitive default

Detach inside `supervised{}` → spawn в worker pool (parallel).
Detach НА TOP-LEVEL (вне supervised) → inline в main (как сейчас).

**Минус:** rules — programmer surprise; spec D50 needs careful
formulation.

**Рекомендация:** **B** — production-grade fire-and-forget с
explicit sync в тестах. D50 amend'нут в Ф.3.

## 5. План реализации

### Ф.1 — `runtime.spawn_orphan(body)` primitive (~1 dev-day)

New API + codegen:
- `nova_runtime_spawn_orphan(entry, user)` в `runtime.c`: спавн fiber
  на worker без supervised-scope binding (parent_scope = NULL либо
  специальный global orphan-scope для GC root tracking).
- LogAndDrop fail-handler — если orphan fiber throws, runtime logs +
  drops (Plan 50 default).
- Codegen emit_c.rs::ExprKind::Detach → `nova_runtime_spawn_orphan(
  _nova_detach_entry_N, ctx)` вместо inline-handler-call.

### Ф.2 — Test migration (~0.5-1 dev-day)

`detach_test.nv`:
- Tests, проверяющие inline-execution → переписать на explicit
  sync (e.g. signal через atomic counter + busy-wait либо channel).
- Tests, проверяющие fire-and-forget семантику — оставить.
- Documented test-side change в `// Plan 83.4.5.2:`.

### Ф.3 — D50 amend (~0.5 dev-day)

`spec/decisions/06-concurrency.md` D50:
- «Default detach semantics: fire-and-forget на worker pool
  (NOVA_MAXPROCS=1 → 1-worker, последовательное исполнение)».
- Note: bootstrap `SyncDetach` legacy — оставлен для compatibility,
  doc'но deprecated после flip.

## 6. Acceptance

- `detach_test` PASS под active flip.
- Side-effects тесты ждут через explicit sync (а не inline).
- D50 spec amend'ed.
- 0 регрессий single-thread пути (D50 default = AsyncDetach, но
  без armed runtime spawn_orphan → fallback inline через
  nova_fiber_spawn_into в main scope).

## 7. Risk

- **D50 spec drift** — переход на async-default может surprise
  existing user'а. Mitigation: changelog note в release notes;
  bootstrap-fallback inline-execution когда runtime НЕ armed.
- **Orphan fiber GC** — без supervised-scope's ctx_pins, SpawnCtx
  alloc'нутая на heap может быть premature-GC'нута. Mitigation:
  global orphan-scope с persistent ctx_pins (Plan 82 §П3 уже знает
  как).
- **LogAndDrop fail-handler** — если orphan fiber throws,
  необработанная error должна логироваться (D50 spec). Plan 50 уже
  специфицировал; реализация — проверить что под flip активен.
