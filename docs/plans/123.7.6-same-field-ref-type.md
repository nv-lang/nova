---

## 1. Контекст

V7.5/V7.7 conservatively invalidate own-field cache (and chain-root
cache) на `@F.method()`. For **reference-type** fields (`[]T`, `*T`,
`Map`, `String`, …) — field holds header/pointer; mutation through
methods modifies referenced object, не slot bits. Cache of `@F`
survives.

V7.6 closes the gap by integrating TypeDecl reference-type
classification with V7.5/V7.7 dispatch.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**New helper `is_reference_type_ref` (~30 LOC):**

Classifies TypeRef into reference/value categories. Recognizes
Array, Pointer, Readonly/Mut wrappers (recursive peel), and a
well-known set of Named reference-type leaves.

**FieldRegistry extended (~5 LOC):**

```rust
struct FieldRegistry {
    by_type: HashMap<String, HashMap<String, FieldKind>>,
    skip_types: HashSet<String>,
    ref_typed: HashSet<(String, String)>,  // V7.6
}
```

Populated in `register_items` during the existing record-walk.

**IpaCtx extended (~10 LOC):**

```rust
pub(crate) struct IpaCtx<'a> {
    ...
    ref_typed: &'a HashSet<(String, String)>,
}

impl<'a> IpaCtx<'a> {
    pub(crate) fn is_field_ref_type(&self, fname: &str) -> bool { ... }
}
```

All 4 IpaCtx construction sites updated к pass `&reg.ref_typed`.

**V7.5/V7.7 own-field branches refined (~15 LOC):**

Both direct `@F.method()` and chain `@F0...Fn.method()` paths consult
`ctx.is_field_ref_type(fname)` для chain root / own field. Returns
false (don't invalidate) when reference-typed.

## 4. Не в скоупе

- **User generic wrappers** like `Container[T]`. Currently only named
  leaves recognized. Followup `[M-123.7.6-generic-ref-types]`.
- **Method-level realloc detection** — V7.6 assumes any `@F.method()`
  на ref-type is safe для slot bits. True для typical APIs; не для
  swap-and-replace ABI. Followup `[M-123.7.6-method-realloc-flag]`.
