### 3.1 –Р—А—Е–Є—В–µ–Ї—В—Г—А–љ—Л–µ –≤–∞—А–Є–∞–љ—В—Л (decision –≤ –§.1)

**Variant A вАФ minimal worker UV loop:**
- Driver thread owns ALL timers / blocking after_cb / I/O events
- Each worker thread keeps tiny UV loop —Б –Ю–Ф–Э–Ш–Ь `uv_async_t` wake handle
- Worker sleeps –љ–∞ own loop's `uv_run(ONCE)` –Ї–Њ–≥–і–∞ deque empty; driver writes async
  to wake
- **Pros:** minimal refactor, reuse existing UV-based wake mechanism
- **Cons:** still pays per-worker UV loop init cost + thread setup

**Variant B вАФ workers –±–µ–Ј UV loop (condvar/event):**
- Workers used pthread/Windows event for wake
- No UV at all on workers
- **Pros:** lower per-worker overhead, simpler topology
- **Cons:** larger refactor, two wake mechanisms (UV –і–ї—П driver, condvar –і–ї—П workers)

**Decision in –§.1 design phase** based on bench numbers –Є code-size estimate.
Default lean toward **Variant A** (less risky).

#### –§.1 вАФ Architecture design (1 day, **Opus + Thinking ON**)

Deliverable: `docs/plans/83.11-design.md` covering:

1. **Driver thread lifecycle**
   - Init: `nova_driver_init()` called –≤ `nova_runtime_init` (after worker pool
     materialization)
   - Main loop: `for(;;) { uv_run(driver_loop, ONCE); drain_jobs(); check_stop(); }`
   - Shutdown: `nova_driver_stop()` + `uv_async_send(driver_stop_handle)` +
     `uv_thread_join`

2. **Job submission API (worker вЖТ driver)**
   ```c
   typedef enum {
     NOVA_DRV_ARM_SLEEP,
     NOVA_DRV_ARM_BLOCKING,
     NOVA_DRV_CANCEL_SCOPE,
     NOVA_DRV_TIMER_RESET,
     // ...
   } NovaDriverJobKind;

   typedef struct {
     NovaDriverJobKind kind;
     union {
       struct { NovaSleepState* st; uint64_t ms; } arm_sleep;
       struct { struct NovaFiberQueue* scope; } cancel_scope;
       // ...
     };
   } NovaDriverJob;

   int nova_driver_submit_job(NovaDriverJob* job);  // MPSC queue + async_send
   ```

3. **Wake API (driver вЖТ worker)**
   - Driver knows fiber's home worker via NovaSleepState.home_worker_id
   - Wake: push to worker's wake_pending + async_send (same as today's cross-thread
     dispatch_ready)

4. **Per-scope ArmedSleepList**
   - Driver-side data structure: `NovaFiberQueue.armed_sleeps_head` (linked list of
     NovaSleepState*)
   - ONLY driver thread mutates вЖТ no concurrency needed
   - Insert: at driver's arm_sleep job handler
   - Remove: at driver's close_cb handler OR cancel_scope handler

5. **Cancel semantics post-pivot**
   - `tok.cancel()` on worker submits `NOVA_DRV_CANCEL_SCOPE` job
   - Driver walks `scope.armed_sleeps_head` list, –і–ї—П –Ї–∞–ґ–і–Њ–≥–Њ `NovaSleepState*` вАФ
     `uv_close` (same thread = no race) + wake target fiber
   - Single mutator вЖТ no race surface

6. **Worker idle behavior**
   - Variant A: worker's tiny UV loop blocks –≤ `uv_run(ONCE)` on its own
     `uv_async_t`. Driver sends async on wake.
   - Variant B: worker blocks on `pthread_cond_wait` / Windows event. Driver
     signals condvar.

7. **Performance contention analysis**
   - Driver as single mutator –Љ–Њ–ґ–µ—В —Б—В–∞—В—М bottleneck
   - Mitigation: batch dispatch (driver drains job queue once per uv_run iteration);
     home_worker_id hint cache; lock-free MPSC queue (already exists –≤ `wake_pending`
     pattern)

8. **Spec D-block update plan**
   - D-NNN: cancel state-machine semantics (driver-owned, single-thread mutator)
   - D-NNN+1: I/O subsystem architecture (centralized driver vs per-worker)
   - D-NNN+2: ќµ-bounded cancel latency formal guarantees

#### –§.3 вАФ Migrate Time.sleep (2 days, **Opus + Thinking ON**)

**Core migration.** Closes `[M-83.10.4-iso-cancel-startup-race]`.

1. `_nova_sleep_via_libuv` вЖТ `_nova_sleep_via_driver`:
   ```c
   static inline void _nova_sleep_via_driver(NovaFiberQueue* scope, int slot, nova_int ms) {
       NovaFiberQueue* cancel_scope = /* derive parent scope */;
       if (cancel_scope->cancel_requested) return;  // Race 2a (still useful вАФ early skip)

       NovaSleepState st = {
           .scope = scope, .slot = slot,
           .home_worker_id = _current_worker_id,
           .cancel_scope = cancel_scope,
       };
       nova_aint_init(&st.packed_state, NOVA_SLEEP_NEW);

       // Submit arm job to driver. Driver creates uv_timer_t on driver's loop,
       // adds to scope.armed_sleeps_head list, CAS packed_state NEWвЖТARMED.
       NovaDriverJob job = { .kind = NOVA_DRV_ARM_SLEEP, .arm_sleep = { &st, (uint64_t)ms } };
       nova_driver_submit_job(&job);

       // Park-until pattern unchanged вАФ drive sends wake when timer/cancel fires.
       nova_sched_park_until(scope, slot, _nova_sleep_state_is_done, &st);
   }
   ```

2. Driver-side timer arm handler:
   ```c
   static void _nova_driver_handle_arm_sleep(NovaSleepState* st, uint64_t ms) {
       // Single-thread mutator вАФ no atomic CAS needed for state —В—А–∞–љ–Ј–Є—И–µ–љ NEWвЖТARMED.
       // But other threads (worker park-predicate) ACQUIRE-load packed_state.
       uv_timer_init(&_nova_driver_loop, &st->timer);
       st->timer.data = st;
       uv_timer_start(&st->timer, _nova_driver_sleep_timer_cb, ms, 0);

       // Add to scope's armed list (driver-thread only).
       st->next_in_scope = st->cancel_scope->armed_sleeps_head;
       st->cancel_scope->armed_sleeps_head = st;

       nova_aint_store(&st->packed_state, NOVA_SLEEP_ARMED, __ATOMIC_RELEASE);

       // Wake target worker if it might be waiting on park-predicate already
       // (Race: worker submitted arm job, called park_until, predicate returned
       // false –љ–∞ NEW вЖТ parked. Now we ARMED вЖТ predicate would return false still,
       // worker sleeps until timer or cancel.)
       // No wake needed here вАФ worker is parked, will be woken by timer_cb –Є–ї–Є cancel.
   }
   ```

3. Driver-side timer_cb (natural fire):
   ```c
   static void _nova_driver_sleep_timer_cb(uv_timer_t* h) {
       NovaSleepState* st = (NovaSleepState*)h->data;
       // Single-thread mutator. CAS ARMEDвЖТFIRING (loser = cancel already won).
       int32_t expected = NOVA_SLEEP_ARMED;
       if (!nova_aint_cas(&st->packed_state, &expected, NOVA_SLEEP_FIRING)) return;
       _nova_driver_close_and_wake(st);
   }
   ```

4. Driver-side cancel job handler:
   ```c
   static void _nova_driver_handle_cancel_scope(NovaFiberQueue* scope) {
       // Walk scope's armed list (driver-thread only вАФ no concurrent mutation).
       for (NovaSleepState* st = scope->armed_sleeps_head; st; st = st->next_in_scope) {
           int32_t expected = NOVA_SLEEP_ARMED;
           if (nova_aint_cas(&st->packed_state, &expected, NOVA_SLEEP_CANCEL_REQ)) {
               _nova_driver_close_and_wake(st);  // close timer + wake worker
           }
           // CAS loser = timer fired naturally first, already closed.
       }
   }

   static void _nova_driver_close_and_wake(NovaSleepState* st) {
       uv_close((uv_handle_t*)&st->timer, _nova_driver_sleep_close_cb);
   }

   static void _nova_driver_sleep_close_cb(uv_handle_t* h) {
       NovaSleepState* st = (NovaSleepState*)h->data;
       nova_aint_store(&st->packed_state, NOVA_SLEEP_CLOSED, __ATOMIC_RELEASE);

       // Remove from armed list.
       _nova_driver_remove_from_armed_list(st);

       // Wake target worker fiber via cross-thread dispatch.
       nova_worker_wake_fiber(st->home_worker_id, st->scope, st->slot);
   }
   ```

   ```c
   static nova_bool _nova_sleep_state_is_done(void* ctx) {
       NovaSleepState* st = (NovaSleepState*)ctx;

       return s == NOVA_SLEEP_CLOSED;
   }
   ```
