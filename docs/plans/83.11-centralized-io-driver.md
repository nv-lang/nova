## 1. Context — почему architectural pivot

### 1.1 Pattern recognition: 5 planов на тот же area

После Plan 83 (M:N runtime baseline) последовательно потребовались:

| План | Что фиксил | Тип fix |
|------|-----------|---------|
| 83.10.1 | AUTOARM=0 directive sweep | Tactical (test-side) |
| 83.10.2 | armed cancel-timer-hang (3 баг'а: GC aliasing, slot reuse, race 2a/2b, sched_wake branch) | Tactical (4 distinct fixes) |
| 83.10.3 | nested supervised armed routing — worker pump_scope | Tactical (architectural в одной точке) |
| 83.10.4 | TLS handler snapshot race + fail handler cross-mco + 4 residual flaky | Tactical (5+ distinct fixes) |
| 83.10.5 | iso-cancel startup race | Tactical attempts FAILED → architectural Ф.B required |

**Каждый fix решал реальный bug. Каждый bug был в той же зоне:** cross-thread visibility distributed I/O state (timers, cancel-flags, parked-flags) **across N per-worker UV loops**.

**Это не цепочка случайных багов. Это симптомы архитектурного выбора.**

### 1.2 Чем Tokio multi-thread runtime отличается

```
Nova сейчас:                        Tokio (с 1.0):
─────────────                       ─────────────
main thread                         main thread
  ├ worker0 [UV loop]                 ├ driver thread [ОДНА UV-аналог loop]
  ├ worker1 [UV loop]                 │   ├ all timers
  ├ ...                               │   ├ all I/O events
  └ workerN [UV loop]                 │   └ wake jobs queue
                                      ├ worker0 [local queue, БЕЗ UV loop]
                                      ├ worker1 [local queue]
                                      ├ ...
                                      └ workerN [local queue]
```

**Cross-thread cancel в Nova:** walk N worker scopes, читать `pending_stop_cb[]`,
`parked[]`, `count` через ACQUIRE-loads на N different heap pages. Distributed state
→ inherent race surface.

**Cross-thread cancel в Tokio:** атомарный CAS на task-state (один AtomicUsize per
task). Driver на следующем tick видит CANCELLED bit, не запускает task. Один
mutator I/O state — нет races.

### 1.3 Tokio переписали runtime ровно по этой причине

Ранние Tokio (0.1, 0.2 — 2017-2019) пробовали per-worker reactors. Те же races.
1.0 release (2020) переписан на centralized driver. Это в их changelog и blog posts:
- https://tokio.rs/blog/2020-04-preemption
- https://tokio.rs/blog/2019-10-scheduler

Они **знали урок**. Мы можем взять его без повторения 3 лет patching'а.

### 1.4 Go runtime — distributed, платит cost engineering team'ом

Go runtime: per-P (processor) timer heap + per-P run queue. Те же distributed-state
race возможности. У них **десятки timer-race issues** в open-source истории:
#6239, #14383, #27707, #41686, #50190. Решают через mature engineering team,
decades of TSAN/jcstress-like stress testing.

**У нас команда 1 человек.** Невозможно поддерживать distributed-state runtime
качества Go без на порядок больших ресурсов.

### 1.5 Что выберем

Path A из 83.10.5 final report: **architectural pivot на centralized driver**,
**максимально портируя Tokio design**. Никакого reinvent — Tokio решил эти проблемы,
доказал в production, у нас лицензия (MIT) на port их design. См. §3.0 для
explicit Tokio source mapping.

## 2. Цель

Переписать M:N I/O integration: вместо N per-worker UV loops — **одна driver
thread с одним UV loop**, которая владеет всеми timers / blocking work / I/O
event sources. Workers становятся pure compute pumps (deque + local sched_state,
без UV loop ИЛИ с минимальным wake-handle-only loop).

**Эффект:**
- Eliminates ENTIRE class of distributed-state races
- Cancel становится атомарным CAS на single state (driver-side data structure)
- Closes 5 outstanding markers (`[M-83.10.4-iso-cancel-startup-race]`,
  `[M-83.10.4-sleep-bench-scale-cliff]`, `[M-83.10.4-env-parser-30-line-limit]`,
  и preventively — future-race-Ns которые ещё не возникли)
- Plan 83.10.5 PARTIAL → SUPERSEDED (можно закрыть)

### 3.1 Архитектурные варианты (decision в Ф.1)

**Variant A — minimal worker UV loop:**
- Driver thread owns ALL timers / blocking after_cb / I/O events
- Each worker thread keeps tiny UV loop с ОДНИМ `uv_async_t` wake handle
- Worker sleeps на own loop's `uv_run(ONCE)` когда deque empty; driver writes async
  to wake
- **Pros:** minimal refactor, reuse existing UV-based wake mechanism
- **Cons:** still pays per-worker UV loop init cost + thread setup

**Variant B — workers без UV loop (condvar/event):**
- Workers used pthread/Windows event for wake
- No UV at all on workers
- **Pros:** lower per-worker overhead, simpler topology
- **Cons:** larger refactor, two wake mechanisms (UV для driver, condvar для workers)

**Decision in Ф.1 design phase** based on bench numbers и code-size estimate.
Default lean toward **Variant A** (less risky).

#### Ф.1 — Architecture design (1 day, **Opus + Thinking ON**)

Deliverable: `docs/plans/83.11-design.md` covering:

1. **Driver thread lifecycle**
   - Init: `nova_driver_init()` called в `nova_runtime_init` (after worker pool
     materialization)
   - Main loop: `for(;;) { uv_run(driver_loop, ONCE); drain_jobs(); check_stop(); }`
   - Shutdown: `nova_driver_stop()` + `uv_async_send(driver_stop_handle)` +
     `uv_thread_join`

2. **Job submission API (worker → driver)**
   ```c
   typedef enum {
     NOVA_DRV_ARM_SLEEP,
     NOVA_DRV_ARM_BLOCKING,
     NOVA_DRV_CANCEL_SCOPE,
     NOVA_DRV_TIMER_RESET,
     // ...
   } NovaDriverJobKind;

   typedef struct {
     NovaDriverJobKind kind;
     union {
       struct { NovaSleepState* st; uint64_t ms; } arm_sleep;
       struct { struct NovaFiberQueue* scope; } cancel_scope;
       // ...
     };
   } NovaDriverJob;

   int nova_driver_submit_job(NovaDriverJob* job);  // MPSC queue + async_send
   ```

3. **Wake API (driver → worker)**
   - Driver knows fiber's home worker via NovaSleepState.home_worker_id
   - Wake: push to worker's wake_pending + async_send (same as today's cross-thread
     dispatch_ready)

4. **Per-scope ArmedSleepList**
   - Driver-side data structure: `NovaFiberQueue.armed_sleeps_head` (linked list of
     NovaSleepState*)
   - ONLY driver thread mutates → no concurrency needed
   - Insert: at driver's arm_sleep job handler
   - Remove: at driver's close_cb handler OR cancel_scope handler

5. **Cancel semantics post-pivot**
   - `tok.cancel()` on worker submits `NOVA_DRV_CANCEL_SCOPE` job
   - Driver walks `scope.armed_sleeps_head` list, для каждого `NovaSleepState*` —
     `uv_close` (same thread = no race) + wake target fiber
   - Single mutator → no race surface

6. **Worker idle behavior**
   - Variant A: worker's tiny UV loop blocks в `uv_run(ONCE)` on its own
     `uv_async_t`. Driver sends async on wake.
   - Variant B: worker blocks on `pthread_cond_wait` / Windows event. Driver
     signals condvar.

7. **Performance contention analysis**
   - Driver as single mutator может стать bottleneck
   - Mitigation: batch dispatch (driver drains job queue once per uv_run iteration);
     home_worker_id hint cache; lock-free MPSC queue (already exists в `wake_pending`
     pattern)

8. **Spec D-block update plan**
   - D-NNN: cancel state-machine semantics (driver-owned, single-thread mutator)
   - D-NNN+1: I/O subsystem architecture (centralized driver vs per-worker)
   - D-NNN+2: ε-bounded cancel latency formal guarantees

**Acceptance Ф.1:** design doc reviewed (self-review + memory verification), variant
chosen, no architectural ambiguity for Ф.2 implementation.

#### Ф.3 — Migrate Time.sleep (2 days, **Opus + Thinking ON**)

**Core migration.** Closes `[M-83.10.4-iso-cancel-startup-race]`.

1. `_nova_sleep_via_libuv` → `_nova_sleep_via_driver`:
   ```c
   static inline void _nova_sleep_via_driver(NovaFiberQueue* scope, int slot, nova_int ms) {
       NovaFiberQueue* cancel_scope = /* derive parent scope */;
       if (cancel_scope->cancel_requested) return;  // Race 2a (still useful — early skip)

       NovaSleepState st = {
           .scope = scope, .slot = slot,
           .home_worker_id = _current_worker_id,
           .cancel_scope = cancel_scope,
       };
       nova_aint_init(&st.packed_state, NOVA_SLEEP_NEW);

       // Submit arm job to driver. Driver creates uv_timer_t on driver's loop,
       // adds to scope.armed_sleeps_head list, CAS packed_state NEW→ARMED.
       NovaDriverJob job = { .kind = NOVA_DRV_ARM_SLEEP, .arm_sleep = { &st, (uint64_t)ms } };
       nova_driver_submit_job(&job);

       // Park-until pattern unchanged — drive sends wake when timer/cancel fires.
       nova_sched_park_until(scope, slot, _nova_sleep_state_is_done, &st);
   }
   ```

2. Driver-side timer arm handler:
   ```c
   static void _nova_driver_handle_arm_sleep(NovaSleepState* st, uint64_t ms) {
       // Single-thread mutator — no atomic CAS needed for state транзишен NEW→ARMED.
       // But other threads (worker park-predicate) ACQUIRE-load packed_state.
       uv_timer_init(&_nova_driver_loop, &st->timer);
       st->timer.data = st;
       uv_timer_start(&st->timer, _nova_driver_sleep_timer_cb, ms, 0);

       // Add to scope's armed list (driver-thread only).
       st->next_in_scope = st->cancel_scope->armed_sleeps_head;
       st->cancel_scope->armed_sleeps_head = st;

       nova_aint_store(&st->packed_state, NOVA_SLEEP_ARMED, __ATOMIC_RELEASE);

       // Wake target worker if it might be waiting on park-predicate already
       // (Race: worker submitted arm job, called park_until, predicate returned
       // false на NEW → parked. Now we ARMED → predicate would return false still,
       // worker sleeps until timer or cancel.)
       // No wake needed here — worker is parked, will be woken by timer_cb или cancel.
   }
   ```

3. Driver-side timer_cb (natural fire):
   ```c
   static void _nova_driver_sleep_timer_cb(uv_timer_t* h) {
       NovaSleepState* st = (NovaSleepState*)h->data;
       // Single-thread mutator. CAS ARMED→FIRING (loser = cancel already won).
       int32_t expected = NOVA_SLEEP_ARMED;
       if (!nova_aint_cas(&st->packed_state, &expected, NOVA_SLEEP_FIRING)) return;
       _nova_driver_close_and_wake(st);
   }
   ```

4. Driver-side cancel job handler:
   ```c
   static void _nova_driver_handle_cancel_scope(NovaFiberQueue* scope) {
       // Walk scope's armed list (driver-thread only — no concurrent mutation).
       for (NovaSleepState* st = scope->armed_sleeps_head; st; st = st->next_in_scope) {
           int32_t expected = NOVA_SLEEP_ARMED;
           if (nova_aint_cas(&st->packed_state, &expected, NOVA_SLEEP_CANCEL_REQ)) {
               _nova_driver_close_and_wake(st);  // close timer + wake worker
           }
           // CAS loser = timer fired naturally first, already closed.
       }
   }

   static void _nova_driver_close_and_wake(NovaSleepState* st) {
       uv_close((uv_handle_t*)&st->timer, _nova_driver_sleep_close_cb);
   }

   static void _nova_driver_sleep_close_cb(uv_handle_t* h) {
       NovaSleepState* st = (NovaSleepState*)h->data;
       nova_aint_store(&st->packed_state, NOVA_SLEEP_CLOSED, __ATOMIC_RELEASE);

       // Remove from armed list.
       _nova_driver_remove_from_armed_list(st);

       // Wake target worker fiber via cross-thread dispatch.
       nova_worker_wake_fiber(st->home_worker_id, st->scope, st->slot);
   }
   ```

5. Worker park-predicate:
   ```c
   static nova_bool _nova_sleep_state_is_done(void* ctx) {
       NovaSleepState* st = (NovaSleepState*)ctx;
       int32_t s = nova_aint_load(&st->packed_state, __ATOMIC_ACQUIRE);
       return s == NOVA_SLEEP_CLOSED;
   }
   ```

6. Remove old paths:
   - `_nova_sleep_via_libuv` deprecated (keep как bootstrap-only path для compat)
   - `_nova_sleep_timer_cb`, `_nova_sleep_stop_cb`, `_nova_sleep_close_cb` (worker-side)
     removed
   - `nova_runtime_cancel_worker_fibers` removed (replaced by driver-side cancel)
   - `pending_stop_cb[]`, `pending_handle[]` arrays в sched_state could be removed
     (если ONLY sleep used them — but Ф.4/Ф.5 needs to migrate other users first)

#### Ф.5 — Migrate channels / sync primitives (1.5 day, Sonnet + Thinking ON)

Audit park-points outside sleep:
- Channels (`nova_chan_send_blocking`, `nova_chan_recv_blocking`)
- Mutex/RwLock (Plan 103.3)
- Once/CountDownLatch/Semaphore/Barrier/Condvar (Plans 103.4-5)

Most of these don't use UV directly — they use `nova_sched_register_pending` for
cancel-wake. Pattern: cancel reads `pending_stop_cb[]`. Same three-load race
applies.

Migration: replace `pending_stop_cb[]` mechanism with driver-routed cancel. Each
primitive registers a `NovaCancelHook*` with driver via job. Driver walks hooks
on scope cancel.

OR keep sched_state's `pending_stop_cb[]` для bootstrap path, route armed-M:N
cancels through driver only. Cleaner: full migration.

**Acceptance:** Plan 103.x tests pass solo + stress 10×.

#### Ф.6 — Worker loop minimization (1 day, **Opus + Thinking ON**)

Decision per Ф.1 design:

**Variant A:** worker keeps tiny UV loop с одним `uv_async_t` wake handle.
Trivial — already exists (worker's `wake_handle`). Just don't init other handles.

**Variant B:** worker uses pthread cond + mutex или Windows event for sleep/wake.
Replace `uv_run(ONCE)` с `pthread_cond_wait`. Larger refactor.

Per Ф.1 recommendation. Default **Variant A**.

**Acceptance:** workers idle на минимально UV state, no leaked handles, shutdown clean.

## 10. Post-mortem: Ф.3 race investigation (2026-05-27–28)

> **Статус Ф.3:** ✅ ЗАКРЫТА. 30/30 stress PASS. Commit `dd7a4f00bc5`.
>
> Этот раздел документирует ~10 часов / 6 сессий диагностики и исправлений
> одного race condition в `supervised { spawn { sleep(N) } }`.
> Сохранено как обучающий материал для будущих M:N races.

### 10.1 Симптом

```
supervised {
    for _ in 0..100 {
        spawn { sleep(10000) }
    }
    tok.cancel()
}
// → зависает навсегда (без таймаута)
```

`cancel()` вызывается, все timers закрываются (`close_cb` срабатывает), но
`nova_supervised_run_impl` ждёт `pending_remote == 0` вечно.
`cancel_latency_bench` → TIMEOUT >30s. `stress_iso_3e` → 3-5/30 PASS.

### 10.2 Ложные пути (что НЕ сработало)

#### Attempt 1: wake-before-park — unconditional dispatch (Sessions 1-2)

**Гипотеза:** `close_cb` срабатывает ДО того, как fiber успел войти в `nova_sched_park`.
CAS `parked: false → false` fails → wake потерян.

**Fix:** в `close_cb` dispatch fiber unconditionally (без CAS на `parked`).

**Результат:** 1/30 PASS. Вероятная причина — double-dispatch: два wake
на один fiber → double-resume → UB/crash в mco.

**Вывод:** unconditional dispatch без защиты не работает.

#### Attempt 2: Generational futex (Session 2)

**Гипотеза:** нужна версия wake'а — `wake_gen` counter. Fiber проверяет
счётчик после yield, если вырос — wake состоялся (Linux futex idiom).

**Fix:** `NovaSchedState.wake_gen[]` + pre/post checks в `nova_sched_park`.

**Результат:** 1/30 PASS. `dispatch_ready` иерархия имела несколько race
windows сама по себе. wake_gen не помог, так как проблема раньше в стеке.

#### Attempt 3: Simple futex + SEQ_CST stage (Session 3)

**Гипотеза:** добавить `wait_state` per-sleep atomic — NEW/PARKED/WOKEN.
Park CAS NEW→PARKED, wake CAS PARKED→WOKEN, предикат проверяет WOKEN.

**Fix:** `NovaSleepState.wait_state` field (standalone atomic).

**Результат:** 1-19/30 PASS (system-load dependent). SEGFAULT при tight
coupling с fiber arena. Heisenbug: `fprintf` (memory fence) поднимал до
1/1 PASS — классический memory ordering heisenbug.

#### Attempt 4: Single-atomic pure Tokio AtomicWaker port (Session 4)

**Гипотеза:** полностью заменить `parked[]`-based mechanism на
`NovaSleepWaitState` (единый AtomicUsize, packed bits).

**Fix:** remove `parked[]` usage в sleep paths, один CAS per state.

**Результат:** SEGFAULT при arena deallocation + 3/30 PASS в hybrid mode.
Fiber arena жёстко завязана на `parked[]` в `nova_scope_free_slot`.
Полная замена требует переписать arena — ~5 dev-days.

#### Attempt 5: Option A v3 — pending_wake[] integration (Session 5)

**Гипотеза (REVISED):** `pending_remote` leak в spawn-entry epilogue.
7 fibers исчезли из всех state ёмкостей, но `pending_remote` не декрементирован.

**Fix:** `NovaSchedState.pending_wake[]` counter. Park: pre-check CAS 1→0
+ SEQ_CST parked + post-barrier recheck. Wake: CAS pending_wake 0→1 first.

**Результат:** 10/30 PASS (best partial) — но wrong target. `pending_remote`
leak был diagnostic artifact (counters не atomically paired), не реальная
утечка. Сам race был в другом месте.

### 10.3 Финальное исправление (Fix A+B+C)

**Алгоритм который помог локализовать проблему:**

1. Programmatic state-dump при watchdog timeout (`nova_runtime_dump_state`):
   lock-free best-effort snapshot всех workers + sched_state.
2. Атомарные счётчики на каждом шаге wake-delivery chain:
   `close_cb` → `wake_called` → `cas_won` → `sleep_ret`.
3. Dump показал: `close_cb=100, wake=100, cas_won=93` + `fibers[slot]=NULL`.
   **NULL slot при parked=true** — fiber уже вышла, slot переиспользован.

**Root cause — STALE slot condition:**

```
Timeline:
  t1: fiber A занимает slot=5, uv_timer_start
  t2: scope расширяется, slot 5 appears free (parked[5]=true, fibers[5]=NULL)
      → fiber B ЗАНИМАЕТ slot=5 (nova_scope_alloc_slot видит NULL)
  t3: close_cb срабатывает для fiber A
      actual_co = scope->fibers[5] = B (не A!)
      → WRONG-FIBER → wake пропущен → fiber A не просыпается → hang
```

`fibers[slot]=NULL` потому что fiber после `mco_yield` в `nova_sched_park`
не сразу restore'ит `fibers[slot]` — делает это позже в epilogue.
В этот gap `nova_scope_alloc_slot` видит NULL и считает slot свободным.

**Fix A** (`nova_scope_free_slot` в `fibers.h`):
```c
// Пропустить слот если fiber ещё в mco_yield (STALE)
if (__atomic_load_n(&st->parked[i], __ATOMIC_ACQUIRE)) continue;
```
Slot с `parked=true` не считается свободным для переиспользования.

**Fix B** (`_nova_driver_sleep_close_cb` в `driver.c`):
```c
if (actual_co != st->expected_co) {
    // WRONG-FIBER: slot занят другим. Диспатчить expected_co напрямую.
    mco_coro* expected_co = st->expected_co;
    if (expected_co && mco_status(expected_co) == MCO_SUSPENDED) {
        // Очистить parked[slot] для expected_co (не actual_co)
        // Установить DISPLACED sentinel
        displaced_ctx->_nova_worker_slot = -2;
        nova_fiber_state_store(expected_co, NOVA_FIBER_STATE_IDLE);
        sc->dispatch_ready(sc->dispatch_ctx, expected_co);
    }
    return;
}
```
При WRONG-FIBER диспатчим `expected_co` (оригинальный fiber) напрямую,
минуя `nova_sched_wake` который смотрит на slot (теперь занят другим fiber'ом).

**Fix C** (spawn epilogue через `_nova_worker_slot=-2`):
```c
// В epilogue: если slot=-2 (DISPLACED), не вызывать free_slot
if (ctx->_nova_worker_slot != -2) {
    nova_scope_free_slot(scope, ctx->_nova_worker_slot);
}
```
DISPLACED fiber при выходе пропускает `free_slot` — slot уже принадлежит
другому fiber'у (fiber B из Fix B сценария).

**Результат:** stress_iso_3e **30/30 PASS**. Concurrency suite **74-75/78 PASS**.

### 10.4 Lessons learned

1. **Heisenbug = memory ordering**: если `fprintf` fix'ит race — это всегда
   memory fence artifact. Не добавляй `fprintf` и считай «исправлено». Это
   маскировка, не fix.

2. **Counter atomicity важна**: если диагностические счётчики не atomically
   paired с операцией которую считают — их данные misleading. Session 5
   потратила ~2 часа на «утечку pending_remote» которой не было.

3. **State-dump > step-debugger** (при отсутствии debugger): programmatic
   watchdog dump — single most effective diagnostic. Показал WRONG-FIBER
   condition сразу. Без него — 5 сессий слепых попыток.

4. **expected_co как identity**: `NovaSleepState.expected_co` — ключевое поле.
   Без него close_cb не может различить «мой fiber» от «чужой fiber в том же
   slot'е». Аналог Tokio task pointer в `TimerEntry`.

5. **Slot → arena coupling**: `fibers[]`, `parked[]`, `free_slot()` — все три
   должны двигаться атомарно. Попытка заменить только `parked[]` (Attempt 4)
   не работает пока `alloc_slot` смотрит на `fibers[i]==NULL`.

6. **10 часов на один race** — нормально для M:N без debugger под Windows.
   Tokio на тот же класс races потратил ~3 года patching'а. Plan 83.11
   закрывает architectural root cause (centralized driver), Ф.3 был
   tactical must-fix до полной миграции.
