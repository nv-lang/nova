## Что должно работать после Plan 44.6

```nova
import std.runtime.runtime

fn main() {
    runtime.init(4)  // 4 OS workers

    let ch = Channel.new(100)
    supervised {
        for i in 0..100 {
            spawn {
                Time.sleep(10)         // park на own worker loop
                ch.tx.send(i * 2)      // cross-worker channel safe
            }
        }
        spawn {
            var sum = 0
            while let Some(v) = ch.rx.recv() {
                sum += v
            }
            assert(sum == 9900)
        }
    }

    runtime.shutdown()
}
```

**Acceptance:**
- Время сна `Time.sleep(10)` в fiber на worker'е реально занимает ≈10мс
  (а не зависает).
- `Time.after(ms)` в channel-select работает cross-worker.
- 4 workers загружены через `runtime.current_worker_id()` distribution.
- Regression: existing 274 PASS без runtime.init() остаются PASS.

## Архитектурное решение: TLS `_nova_current_loop`

**Ключ:** event-loop является thread-local resource в libuv. `uv_loop_t`
**нельзя** thread-safely использовать из несколькольких threads без
synchronisation. Поэтому каждый worker thread должен иметь own loop.

Минимальный path для Layer 3:

1. Объявить `__thread uv_loop_t* _nova_current_loop`.
2. Main thread init: `_nova_current_loop = nova_evloop()` (в
   `nova_evloop_init`).
3. Worker thread init: `_nova_current_loop = &w->loop` (в
   `_worker_main` runtime.c).
4. Helper `nova_current_loop()`:
   - Если TLS set → return TLS.
   - Иначе fallback на `nova_evloop()` (для C-static init paths где TLS
     не set до `nova_evloop_init`).
5. Заменить `nova_evloop()` на `nova_current_loop()` в всех runtime call-
   sites что регистрируют timer/handle (fibers.h × 6, channels.h × 1).
6. `nova_evloop()` остаётся для **глобальных** операций main thread:
   - `nova_evloop_install_sigint` — SIGINT — single process-wide handler.
   - `nova_evloop_close` — finalize main loop.

**Codegen — БЕЗ ИЗМЕНЕНИЙ.** Под TLS-based подход `nova_supervised_run`
не нуждается в loop parameter — он подбирает loop у TLS. Это снимает
необходимость менять emit_main / emit_supervised в emit_c.rs. Простота.

## Этапы реализации

### Этап 6.1 — TLS infrastructure (eventloop.h + .c)

В `compiler-codegen/nova_rt/eventloop.h` после `uv_loop_t* nova_evloop(void);`
добавить:

```c
/* Plan 44.6 Layer 3: per-thread current event loop.
 *
 * Main thread: set'ится в nova_evloop_init() = глобальный nova_evloop().
 * Worker thread: set'ится в _worker_main = &worker->loop.
 *
 * Все timer / handle registration в runtime — через nova_current_loop(),
 * чтобы handles привязывались к own loop текущего thread'а. */
#ifdef _MSC_VER
extern __declspec(thread) uv_loop_t* _nova_current_loop;
#else
extern __thread uv_loop_t* _nova_current_loop;
#endif

uv_loop_t* nova_current_loop(void);
```

В `eventloop.c`:

```c
#ifdef _MSC_VER
__declspec(thread) uv_loop_t* _nova_current_loop = NULL;
#else
__thread uv_loop_t* _nova_current_loop = NULL;
#endif

uv_loop_t* nova_current_loop(void) {
    if (_nova_current_loop) return _nova_current_loop;
    /* Fallback: main thread где TLS ещё не set, либо thread без
     * runtime.init. Lazily set к глобальному default loop. */
    _nova_current_loop = nova_evloop();
    return _nova_current_loop;
}
```

И в `nova_evloop_init` после успешного init:
```c
_nova_current_loop = _evloop;  /* main thread default */
```

  runtime.is_initialized → Plan 44.7.

  котором park'нулся. uv handle thread-bound, migration требует

  монополизирует worker → Plan 44.9.

- **std.sync.Atomic[T]** → Plan 18 P0.

## Риски

- **R1: Channel waitlist cross-worker.** Plan 44.1 channels уже atomics+
  mutex'ed, M:N-ready. Cross-worker recv/send safe. Risk низкий.
- **R2: Time.after в select-timer cross-worker.** `Time.after` создаёт
  channel + uv_timer. Под Plan 44.6 timer создаётся на current loop
  (worker'а если call site внутри fiber). Если producer worker
  отличается от consumer — channel sync OK, timer fires на producer's
  loop, channel send wakes consumer cross-worker через `uv_async_send`
  (Plan 44.5 wake mechanism). Должно работать. Verify в Layer 5 test.
- **R3: Boehm GC and per-worker loop.** GC_register_my_thread уже в
  Plan 44.5 Layer 4 (Linux/macOS). Worker может вызывать `nova_alloc`
  на own loop — GC safe.
- **R4: SIGINT на worker thread.** SIGINT обрабатывается на main thread
  (`nova_evloop_install_sigint` использует глобальный nova_evloop).
  Workers не имеют signal handler. Cancellation propagation: SIGINT
  cancel'ает main scope; workers' fiber'ы периодически yield → видят
  cancel_requested → throw. Это работает через scope sharing (workers'
  fiber's scope ссылается на parent). TBD verify в test.
- **R5: Windows TLS macro mismatch.** `__declspec(thread)` для MSVC,
  `__thread` для gcc/clang. Уже использован паттерн в runtime.c — повторяем.
