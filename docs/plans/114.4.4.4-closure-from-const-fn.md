# Plan 114.4.4.4 — Closure-returning const fn

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.7 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — builder/curry pattern в comptime.
> **Оценка:** ~1-2 dev-day.
> **Зависимости:** Plan 114.4.3 V2 baseline + Plan 114.4.4.3 (runtime HOF).
>
> **Why extracted:** Closure capture semantics на compile-time complex.
>   Capture-mode design (move vs ref) requires careful integration с
>   existing closure runtime.

## Scope

Allow:
```nova
fn make_adder(const n int) -> const (int -> int) =>
    |x| x + n   // captures const param n
```

- Body checker: accept closure literal в const fn body.
- Closure captures must be const-eligible.
- Codegen: generate specialized closure per (fn, const_args) combo.

## Tests

- Positive: `ro adder5 = make_adder(5); adder5(3) == 8`.

## Acceptance

A1: Const fn returns closure capturing const param.
A2: Different const args → distinct specialized closures.

### Implementation

- New module `compiler-codegen/src/const_fn_closure.rs` (~700 LOC).
- Entry `specialize_closure_returning_const_fns(module)`.
- 4 steps:
  1. **Detect closure-returning fns**: fully-const fn whose body is
     `FnBody::Expr(Lambda | ClosureLight | ClosureFull)`.
  2. **Walk + rewrite call sites**: for each `Call(host, [literals])`,
     evaluate args к `ConstValue`, generate spec name (memoized via
     `HashMap<(host, args), spec_name>`), replace Call expression с
     `Ident(spec_name)`.
  3. **Generate specialized FnDecls**: clone closure body, substitute
     host's const params с literal values; derive param types either
     from closure annotations (Lambda/ClosureFull) или host's
     `-> const fn(T1, ..) -> R` declaration (ClosureLight untyped);
     derive return type similarly. Append к module.items.
  4. **Friendly first-class reject**: any leftover bare-Ident references
     к closure-returning fn names → `E_CONST_FN_CLOSURE_FIRST_CLASS`.

### Body validation extension (types/mod.rs)

- `check_const_fn_decl`: special-case closure literal at top level.
  Extracts closure params, extends scope с host const params + closure
  params, validates closure body по обычным V1 rules через
  `check_const_fn_expr`/`check_const_fn_block`.
- Existing rejection of closure deep inside body unchanged — only
  closure-AT-top-level case allowed.

### Pipeline integration (const_fn_eval.rs:rewrite_const_fn_calls)

- Closure-spec pass invoked **BEFORE** main walker (так что
  `eval_call` не сталкивается с closure body как ConstValue, который
  не поддерживается interpreter'ом).
- Closure-returning fn names остаются в `owned.fns` registry — dropped
  by retain step (как обычные fully-const fns). Specialized fns живут
  с разными именами и переживают retain.
