> **Приоритет:** P0 — 0.1 release blocker.
> **Оценка:** ~4-5 dev-day.
> **Ветка/worktree:** `main` (Ф.-1) / `plan-91.12` / `nova-p91-12` (Ф.0–).
>
> **Зависимости (все ✅ closed):**
>   - Plan 83.12 ✅ — std/net V1 baseline (C runtime, libuv).
>   - Plan 134 ✅ — `ptr` удалён; opaque-handle pattern: `type X(*())` (D214).
>   - Plan 100.x ✅ — consume-types статический анализ.
>   - Plan 73 ✅ — `consume` keyword.
>   - Plan 113 ✅ — `#realtime`/`#blocking` атрибуты.
>
> **D-блоки:** D282 (08-runtime.md, `extern "C" fn`, Ф.-1), D290 (02-types.md, Net C FFI pattern), D291 (04-effects.md, TcpNet/UdpNet/DnsNet).
>
> **Ключевые design-решения (rev 2):**
>
> 1. **Новый синтаксис `extern "nova" fn` / `extern "C" fn` + compiler patch (Ф.-1 ✅).**
>    `extern "C" fn foo(...)` — C-ABI маркер: вызывает C-символ `foo` **буквально**
>    (без `nova_fn_` prefix). `extern "nova" fn bar(...)` — runtime fn с `nova_fn_`
>    prefix (аналог старого `external fn`). `external fn` сохранён как legacy-alias.
>    Пользователь пишет `extern "C" fn libjpeg_decode(...)` и получает прямой вызов.
>    Spec: D282. Реализовано: лексер KwExtern, парсер ABI-строка, AST extern_abi,
>    emit_c c_literal_extern_fns, external_registry skip "C" fns.
>    136 .nv файлов мигрированы: `external fn` → `extern "nova" fn`.
>
> 2. **C-типы — opaque handles: `type CX(*())`.**
>    `CSocketAddr`, `CTcpListener`, `CTcpStream`, `CUdpSocket` — указатели
>    на C-структуры (Plan 134 D214: `type X(*())`). Nova-код не смотрит
>    внутрь; типы различимы compile-time (не взаимозаменяемы).
>
> 3. **Публичные типы — value на стеке.**
>    `type SocketAddr value { priv handle CSocketAddr }` — стек-слот = один
>    pointer. Даёт Nova-API (методы, doc, stable annotations) поверх
>    C-хэндла. Аналог: Rust `TcpStream { inner: Inner { fd: OwnedFd } }`.
>
> 4. **Трёхуровневая архитектура (все файлы — `module std.net` peer):**
>    ```
>    std/net/ffi.nv     ← private C FFI (type CX + external fn без export)
>    std/net/effect.nv  ← effect TcpNet / UdpNet / DnsNet
>    std/net/tcp.nv     ← SocketAddr + TcpListener + TcpStream + real_tcp_net
>    std/net/udp.nv     ← UdpSocket + real_udp_net
>    std/net/dns.nv     ← Dns + real_dns_net  (новый)
>    std/net/error.nv   ← NetError  (без изменений структуры)
>    ```
>    Все файлы объявляют `module std.net`. Пользователь: `use std.net`.
>
> 5. **Нет `export` на FFI-слое.** `ffi.nv` полностью приватен — C-типы
>    и external fns видны только внутри `std.net`.
>
> 6. **Effect-операции вызываются напрямую:** `TcpNet.bind(addr)` (не `perform`).
>    `perform` — не ключевое слово Nova.
>
> 7. **`Blocking Fail[NetError]` в сигнатурах** — основная цель ради
>    `#realtime`-ban enforcement (Plan 113). Public методы несут оба эффекта.
>
> 8. **`str` в FFI слое для V1** — C side возвращает `nova_str` (raw bytes
>    без UTF-8 enforcement). Переход на `[]u8` в C-side — [M-91.12-bytes-ffi]
>    → Plan 91.13.

---

## Зачем

Plan 83.12 (std/net V1) — MVP. Три критических проблемы блокируют 0.1:

1. **Net effect не существует** — mocking невозможен; тесты вынуждены
   использовать реальную сеть.
2. **`Blocking Fail[NetError]` annotations отсутствуют** — type-checker не
   enforce'ит `#realtime` ban. `#realtime fn audio() { TcpStream.connect(...) }`
   компилируется (deadlock в runtime).
3. **Erased errors (`str`)** — `err.contains("connection refused")` fragile,
   cross-platform inconsistent.

## Архитектура

### Слой 1: ffi.nv (private)

```nova
module std.net

// Opaque C handles (Plan 134 D214 pattern).
// Аллоцированы nova_alloc_uncollectable() в C; GC не трогает.
type CSocketAddr(*())
type CTcpListener(*())
type CTcpStream(*())
type CUdpSocket(*())

// `external "C"` — вызов по литеральному C-имени (Ф.-1 compiler patch).
// C side: compiler-codegen/nova_rt/net.h + net.c
//
// Типы параметров — только C-нативные:
//   int, u8, u16, u32, u64, bool, CX(*()) — C знает эти типы.
// Строки: CStr (*u8, null-terminated) — C-строки, не Nova str.
//   Конвертация CStr ↔ str происходит в публичном API (слой 3).

// ── SocketAddr ──────────────────────────────────────────────────────────
extern "C" fn socket_addr_loopback(port u16) -> CSocketAddr
extern "C" fn socket_addr_loopback_v6(port u16) -> CSocketAddr
extern "C" fn socket_addr_v4(a u8, b u8, c u8, d u8, port u16) -> CSocketAddr

extern "C" fn socket_addr_port(addr CSocketAddr) -> u16
extern "C" fn socket_addr_host_str(addr CSocketAddr) -> CStr      // const char*
extern "C" fn socket_addr_is_v4(addr CSocketAddr) -> bool
extern "C" fn socket_addr_is_v6(addr CSocketAddr) -> bool
extern "C" fn socket_addr_to_str(addr CSocketAddr) -> CStr        // const char*

// ── TcpListener ─────────────────────────────────────────────────────────
// out-param pattern: fn возвращает 0 (ok) или errno (<0); handle через out
extern "C" fn tcp_listener_bind(addr CSocketAddr, out mut CTcpListener) -> int
extern "C" fn tcp_listener_accept(lst CTcpListener, out mut CTcpStream) -> int
extern "C" fn tcp_listener_local_port(lst CTcpListener) -> u16
extern "C" fn tcp_listener_local_addr(lst CTcpListener) -> CSocketAddr
extern "C" fn tcp_listener_close(lst CTcpListener) -> ()

// ── TcpStream ───────────────────────────────────────────────────────────
extern "C" fn tcp_stream_connect(addr CSocketAddr, out mut CTcpStream) -> int
extern "C" fn tcp_stream_read(s CTcpStream, buf mut CStr, max_bytes int) -> int  // >0=bytes, 0=EOF, <0=err
extern "C" fn tcp_stream_write(s CTcpStream, buf CStr, len int) -> int           // >0=bytes, <0=err
extern "C" fn tcp_stream_local_port(s CTcpStream) -> u16
extern "C" fn tcp_stream_peer_port(s CTcpStream) -> u16

extern "C" fn tcp_stream_peer_addr(s CTcpStream) -> CSocketAddr
extern "C" fn tcp_stream_close(s CTcpStream) -> ()
// Последнее сообщение об ошибке (thread-local, только после <0-возврата):
extern "C" fn net_last_error() -> CStr

// ── UdpSocket ───────────────────────────────────────────────────────────
extern "C" fn udp_socket_bind(addr CSocketAddr, out mut CUdpSocket) -> int
extern "C" fn udp_socket_send_to(sock CUdpSocket, buf CStr, len int, addr CSocketAddr) -> int
extern "C" fn udp_socket_recv_from(sock CUdpSocket, buf mut CStr, max int, sender_out mut CSocketAddr) -> int
```

### Слой 2: effect.nv (dispatch-point)

```nova
module std.net

// TcpNet — TCP transport (operations take/return Nova value types)
export effect TcpNet {
    fn bind(addr SocketAddr) -> TcpListener
    fn accept(listener mut TcpListener) -> TcpStream
    fn connect(addr SocketAddr) -> TcpStream
    fn read(stream mut TcpStream, max int) -> str
    fn write(stream mut TcpStream, data str) -> int
    fn close_stream(stream consume TcpStream) -> ()
    fn close_listener(listener consume TcpListener) -> ()
    // socket options
    fn set_nodelay(stream mut TcpStream, on bool) -> ()
    fn set_keepalive(stream mut TcpStream, on bool) -> ()
    fn set_read_timeout(stream mut TcpStream, secs int) -> ()
    fn set_write_timeout(stream mut TcpStream, secs int) -> ()
    fn set_reuse_address(listener mut TcpListener, on bool) -> ()
}

// UdpNet — UDP transport (recv возвращает кортеж atomically, без race)
export effect UdpNet {
    fn bind(addr SocketAddr) -> UdpSocket
    fn send_to(sock mut UdpSocket, data str, addr SocketAddr) -> int
    fn recv_from(sock mut UdpSocket, max int) -> (str, SocketAddr)
    fn close_socket(sock consume UdpSocket) -> ()
}

// DnsNet — async DNS
export effect DnsNet {
    fn lookup(host str) -> []SocketAddr
}
```

**Rationale:**
- Effect-operations take/return Nova value types (`SocketAddr`, `TcpListener` и т.д.) — не C-хэндлы. Handler'ы оборачивают/разворачивают `.handle`.
- `UdpNet.recv_from -> (str, SocketAddr)` — атомарный tuple, устраняет race `recv_from + last_sender()` из V1.
- `close_stream`/`close_listener`/`close_socket` — consume-параметры; double-close = compile error.
- `str` для read/write в V1; `[]u8` — [M-91.12-bytes-ffi] → Plan 91.13.

## Фазы

### Ф.-1 — Parser + compiler: синтаксис `external "C" fn`

**Шаг 1 — Parser** (`compiler-codegen/src/parser/mod.rs`):

В парсере `external fn` — добавить опциональную ABI-строку после `external`:
```
external_fn ::= "external" ( STRING_LIT )? "fn" fn_signature
```
Если строка присутствует и равна `"C"` — `FnDecl.abi = Some("C")`. Иначе `abi = None`.

**Шаг 2 — AST** (`compiler-codegen/src/ast/mod.rs`):
```rust
pub struct FnDecl {
    // ... existing fields ...
    pub abi: Option<String>,   // Some("C") для `external "C" fn`
}
```

**Шаг 3 — emit_c.rs**, функция `free_fn_c_name` (строка 11502):

```rust
fn free_fn_c_name(&self, name: &str) -> String {
    // Если функция объявлена с `external "C"` — используем литеральное имя.
    if self.is_c_abi_fn(name) {
        return name.to_string();
    }
    // ... existing logic (nova_fn_ prefix для builtins и legacy) ...
}
```

`is_c_abi_fn` ищет FnDecl с `is_external=true` и `abi=Some("C")` по имени.

**Шаг 4 — Проверить обратную совместимость:**
- `external fn fence()` (no ABI) → `nova_fn_fence` (unchanged).
- `extern "C" fn tcp_listener_bind(...)` → `tcp_listener_bind`.

**Тест Ф.-1** — fixture `nova_tests/plan91_12/pos_ffi_c_abi.nv`:
Вызов `external "C" fn` с заглушкой в отдельном `.c` файле; проверка что линкуется
и возвращает ожидаемое значение (единственный способ верифицировать emit без инспекции
сгенерированного C).

### Ф.0 — C-side: новые entry-points в net.h/net.c

**Задача:** добавить C-функции с литеральными именами как тонкие обёртки вокруг
существующих `Nova_*_static_*` / `Nova_*_method_*`.

```c
// net.h — добавить после существующих объявлений

/* Plan 91.12 Ф.0: literal-name entry-points для Nova external fn. */
Nova_SocketAddr* socket_addr_loopback(uint16_t port);
Nova_SocketAddr* socket_addr_loopback_v6(uint16_t port);
Nova_SocketAddr* socket_addr_v4(uint8_t a, uint8_t b, uint8_t c, uint8_t d, uint16_t port);
/* ... аналогично для всех ffi.nv функций */
```

```c
// net.c — реализации (тонкие делегаты)
Nova_SocketAddr* socket_addr_loopback(uint16_t port) {
    return Nova_SocketAddr_static_loopback(port);
}
/* ... */
```

**Result-returning functions:** тип возврата `NovaRes_*` для `Result[CTcpListener, str]`

### Ф.8 — Spec D-блоки

#### D280 — Net C FFI pattern (в `spec/decisions/02-types.md`)

**Net C FFI pattern для Nova std:** opaque C handles объявляются через
`type CX(*())` (Plan 134 D214). Публичные Nova types — `value`-records
с приватным handle-полем. External fns — без `export`, без `@`, только
simple C parameters; называются по паттерну `<resource>_<action>` и
вызываются по литеральному C-имени (Ф.-1 compiler patch).

#### D281 — TcpNet/UdpNet/DnsNet effect family (в `spec/decisions/04-effects.md`)

`std.net` предоставляет три эффекта:
- `TcpNet` — TCP transport + socket options.
- `UdpNet` — UDP datagram transport; `recv_from` возвращает `(data, SocketAddr)`
  атомарно (устраняет race V1).
- `DnsNet` — DNS resolution (V1: sync via libuv; V2: async [M-91.12-async-dns]).

Использование:
```nova
fn fetch(url str) -> str TcpNet DnsNet Fail[NetError] Blocking { ... }
```

`real_tcp_net()` / `real_udp_net()` / `real_dns_net()` — public export из `std.net`;
устанавливаются в runtime root:
```nova
with TcpNet = real_tcp_net(), UdpNet = real_udp_net(), DnsNet = real_dns_net() {
    run_app()
}
```

#### D282 — `external "C" fn` — явный C-ABI маркер (в `spec/decisions/09-tooling.md`)

`extern "C" fn foo(args...)` — новая форма external fn:
- Вызывает C-символ `foo` **буквально** (без `nova_fn_` prefix).
- Параметры и возвращаемые типы — только C-нативные: `int`, `u8`…`u64`,
  `bool`, `CStr` (`*u8`), opaque handles (`type CX(*())`). Nova-specific
  типы (`str`, `[]T`, `Option`, `Result`) — **запрещены** в `external "C"`.
- `external fn` (без `"C"`) — прежнее поведение: `nova_fn_<name>` для
  runtime builtins; зарезервировано для Nova-aware C код (знает `nova_rt.h`).
- Ответственность за коллизии с C stdlib именами — на авторе external fn.
- Открывает путь для `external "stdcall"`, `external "swift"` и т.п.
