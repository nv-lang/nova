> **Design audit 2026-05-31** — per-method-name + ergonomics + effect-architecture
> review vs Rust/Go/TS/Kotlin/Java; **7 решений**:
>   - **`read_bytes(max)` → `read(max)`** (match mainstream brevity). `read_str`/
>     `write_str` convenience с UTF-8 validation остаётся.
>   - **UDP multicast TTL/loop split на v4/v6** (`multicast_v4_set_ttl`,
>     `multicast_v6_set_ttl`, etc) — Rust-style type-safety vs runtime family
>     dispatch.
>   - **`Dns.reverse` → `Dns.reverse_lookup`** — explicit > short.
>   - **`TcpReader/TcpWriter`** — оставить short names (vs Rust's
>     `OwnedReadHalf/OwnedWriteHalf`). Ownership уже clear через consume types.
>   - **`shutdown(ShutdownDir)` УДАЛЁН** — runtime flag inconsistent с consume
>     philosophy. Half-close ТОЛЬКО через `consume @split() -> (TcpReader,
>     TcpWriter)` + `consume @close()` на половине. **D205 → WITHDRAWN status.**
>     Type-system compile-time enforce'ит — Nova best-in-class vs Rust std/Go/
>     Java/Python (все runtime). +1 строка для half-close use case (worth it).
>   - **Single `Net` effect → split на 4 focused effects** (`TcpNet`/`UdpNet`/
>     `UnixNet`/`DnsNet`). Pro: granular mocking (mock только нужный sub-system
>     в test); function-signature precision (`fn fetch(url) TcpNet DnsNet …` —
>     точно видно что используется); layered architecture для Plan 116 TLS
>     (wraps только TcpNet, не full Net). Con: +3 lines в runtime root install
>     (one-time cost vs per-signature gain).
>   - **Method names внутри effect — БЕЗ prefix** (namespace задан effect-
>     именем): `TcpNet.bind` не `TcpNet.tcp_bind`; clusters со type names
>     (`TcpListener`, `TcpStream`, `TcpReader`, `TcpWriter`, `TcpNet`) —
>     auto-complete by feature ergonomics.
>   - **Public API methods (`@set_*`/`@flush`/`@write_all`) carry `-> @`**
>     (D181 / Plan 91.7 fluent chain pattern; Plan 114 D176 amend — `-> @`
>     inherits mut receiver). Allows `TcpStream.connect(addr)?.set_nodelay(true)?
>     .set_keepalive(true, …)?.set_read_timeout(…)?` — fluent config chain
>     через `?` propagation.
>   - **Effect ops НЕ `-> @`** — effect ops free-function style (parameters
>     explicit, no receiver), `@` syntax не имеет meaning. Effect op

>     и wraps в `Fail.throw(map_libuv_error(rc))`. **Hard dependency на
>     Plan 115** (`ptr` built-in type + tuple-by-value FFI return ABI).

## Зачем

Plan 83.12 (async net V1) закрыт 2026-05-27 как **MVP**. Корпус std/net (4
файла, ~300 строк) предоставляет TCP/UDP базовый async API. **Comments в
коде честно признают V1 limitations**:

```nova
// error.nv: "V1 bootstrap implementation, external functions return
//   Result[T, str] (erased representation). … Future versions will use
//   typed Result[T, NetError]"
// addr.nv:  "V1 note: DNS resolution is synchronous (uv_getaddrinfo on
//   calling thread). Requires Blocking effect in the caller."  ← но в
//   signature Blocking НЕ объявлен.
```

**9 problems V1 std/net** (отсортированы по severity):

### Critical (0.1 blockers)

1. **`Net` effect не существует.** Effect-density example
   ([examples/effect-density/main.nv:30](examples/effect-density/main.nv#L30))
   документирует `Net` как expected handler-effect:
   `fn run(cfg Config) Net Io Fail[StartupError] -> ()`. **Но в std/net
   `effect TcpNet/UdpNet/UnixNet/DnsNet { … }` нигде не объявлены**. Все net ops — прямые `external
   fn` в libuv. **Mocking невозможен**: тесты вынуждены либо использовать
   реальную сеть (slow, flaky, requires localhost ports), либо избегать
   тестирования net-using кода.

2. **`Blocking Fail[NetError]` annotations отсутствуют в signature.**
   `external fn TcpStream.connect(addr) -> Result[TcpStream, str]` — нет
   эффектов в подписи (только в docstring `# Effects` comment). Type-checker
   **не enforce'ит**:
   - Plan 113 `#realtime` ban: `#realtime fn audio_callback() { TcpStream.connect(...) }`
     **компилируется** (sic!). В runtime — deadlock real-time потока на park'е.
   - Effect-handler contracts: handler-state может быть оставлен в
     невалидном виде при park'е.
   - Cancel-propagation: docs обещают «cancel-safe» но signature не
     отражает.
   - Errors через text-matching (`err.contains("connection refused")`)
     fragile + cross-platform inconsistent.

3. **`read_bytes`/`write` используют `str` вместо `[]u8`.**
   `str` в Nova = UTF-8 invariant. Двоичные протоколы (HTTP/2, gRPC,
   protobuf, любой binary): `read_bytes` либо UTF-8 panic, либо silent
   corruption через `from_bytes_unchecked`. **Fundamental design bug**.
   Mainstream: Rust `&[u8]`, Go `[]byte`, TS `Buffer`/`Uint8Array`,
   Kotlin/Java `ByteArray`/`byte[]`.

### High (production-readiness gaps)

4. **`UdpSocket.recv_from + last_sender()` — race-prone API.** Stateful
   на shared socket: multi-fiber receiver получит **чужой адрес** под
   определённым timing'ом. Mainstream все возвращают tuple `(data, addr)`
   атомарно: Rust `(usize, SocketAddr)`, Go `(n, addr, err)`, Java
   `DatagramPacket`. Nova V1 единственная с separate `last_sender()`.

5. **Нет DNS async API.** Только `SocketAddr.parse("host:port")` —
   синхронный `uv_getaddrinfo` на calling thread. Real async DNS
   (через `uv_getaddrinfo` callback) нет. `Dns.lookup("api.example.com") ->
   []SocketAddr` — необходим для современного клиента.

6. **Нет socket options.** Ни одного: TCP_NODELAY, SO_KEEPALIVE,
   SO_REUSEADDR, read/write timeouts, buffer sizes. Production gRPC/HTTP
   сервер невозможен.

### Medium (ergonomics gaps)

7. **Только полный `close()`, нет half-close.** Half-close patterns
   (HTTP/1.1 connection-close, graceful gRPC shutdown) невозможны. **Решение
   2026-05-31:** force split для half-close — нет `shutdown(direction)`
   метода вообще (runtime-flag inconsistent с Nova consume philosophy),
   вместо этого `TcpStream consume @split() -> (TcpReader, TcpWriter)` +
   `consume @close()` на нужной half'е. Type-system enforce'ит «нельзя
   read after reader.close()». Это **best-in-class** среди mainstream
   (Rust std/Go/Java/Python имеют runtime shutdown без compile-time
   guarantees; только Rust tokio split — близкий аналог).

8. **Нет convenience-методов:** `read_exact`/`read_to_end`/`write_all`.
   Каждый caller — manual loop вокруг partial reads.

9. **Дополнительные missing features:**
   - `IpAddr` отдельно от `SocketAddr` (только host без port)
   - `SocketAddr.v6(...)` constructor (только `loopback_v6` и `parse`)
   - UDP multicast (`join_multicast_v4/v6/set_multicast_ttl_v4/v6/set_multicast_loop_v4/v6`)
   - Unix domain sockets (`UnixListener`/`UnixStream`)
   - TcpStream **split** на `TcpReader`/`TcpWriter` — type-safe
     concurrent read/write через consume-types (Nova-уникальная фича).

## Дизайн

### Foundation: `Net` effect — handler-dispatched, mockable

```nova
// std/net/effect.nv — новый файл, ядро Plan 91.12
//
// Net layer разделён на 4 focused effects per protocol/concern:
//   TcpNet  — TCP transport + halves + socket options
//   UdpNet  — UDP transport + multicast
//   UnixNet — Unix domain sockets
//   DnsNet  — DNS lookup + reverse lookup
//
// Naming: `<Protocol>Net` suffix (TcpNet/UdpNet/UnixNet/DnsNet) — clusters
// со существующими type names (TcpListener/TcpReader/TcpStream/TcpWriter),
// auto-complete by feature: `Tcp<Tab>` shows all TCP items (types + effect)
// together. Method names inside effect — без prefix (namespace задан
// effect-именем): `TcpNet.bind`, не `TcpNet.tcp_bind`.

export effect TcpNet {
    // Basic transport
    fn bind(addr SocketAddr) -> TcpListener
    fn accept(listener mut TcpListener) -> TcpStream
    fn connect(addr SocketAddr) -> TcpStream
    fn read(stream mut TcpStream, max int) -> []u8
    fn write(stream mut TcpStream, data ro []u8) -> int
    fn close(stream consume TcpStream)
    fn close_listener(listener consume TcpListener)
    // NOTE: нет shutdown(dir) — для half-close используется split + close half

    // Split halves (Ф.12) — единственный способ для half-close
    fn split(stream consume TcpStream) -> (TcpReader, TcpWriter)
    fn read_half(reader mut TcpReader, max int) -> []u8
    fn write_half(writer mut TcpWriter, data ro []u8) -> int
    fn flush_half(writer mut TcpWriter) -> ()
    fn close_reader(reader consume TcpReader)    // → shutdown(Read) под капотом
    fn close_writer(writer consume TcpWriter)    // → shutdown(Write) под капотом

    // Socket options (Ф.6)
    // Socket options — effect ops returns `()` (free-fn style, no @-receiver
    // here). Fluent chain через `-> @` живёт на public API wrappers уровне
    // (см. ниже в Public API + Ф.6.3); perform-уровень не chain'ится.
    fn set_nodelay(stream mut TcpStream, on bool) -> ()
    fn set_keepalive(stream mut TcpStream, on bool, idle Option[Duration]) -> ()
    fn set_read_timeout(stream mut TcpStream, t Option[Duration]) -> ()
    fn set_write_timeout(stream mut TcpStream, t Option[Duration]) -> ()
    fn set_recv_buffer_size(stream mut TcpStream, n int) -> ()
    fn set_send_buffer_size(stream mut TcpStream, n int) -> ()
    fn listener_set_reuse_address(listener mut TcpListener, on bool) -> ()
    fn set_ttl(stream mut TcpStream, ttl u8) -> ()
}

export effect UdpNet {
    // Basic transport
    fn bind(addr SocketAddr) -> UdpSocket
    fn send(sock mut UdpSocket, data ro []u8, addr SocketAddr) -> int
    fn recv(sock mut UdpSocket, max int) -> ([]u8, SocketAddr)
    fn close(sock consume UdpSocket)

    // Multicast (Ф.10) — split v4/v6 для type-safety
    fn multicast_v4_join(sock mut UdpSocket, group Ipv4Addr, iface Option[Ipv4Addr])
    fn multicast_v4_leave(sock mut UdpSocket, group Ipv4Addr, iface Option[Ipv4Addr])
    fn multicast_v4_set_ttl(sock mut UdpSocket, ttl u8)
    fn multicast_v4_set_loop(sock mut UdpSocket, on bool)
    fn multicast_v6_join(sock mut UdpSocket, group Ipv6Addr, iface_idx int)
    fn multicast_v6_leave(sock mut UdpSocket, group Ipv6Addr, iface_idx int)
    fn multicast_v6_set_ttl(sock mut UdpSocket, ttl u32)
    fn multicast_v6_set_loop(sock mut UdpSocket, on bool)
}

export effect UnixNet {
    fn bind(path str) -> UnixListener
    fn accept(listener mut UnixListener) -> UnixStream
    fn connect(path str) -> UnixStream
    fn read(stream mut UnixStream, max int) -> []u8
    fn write(stream mut UnixStream, data ro []u8) -> int
    fn close_stream(stream consume UnixStream)
    fn close_listener(listener consume UnixListener)
}

export effect DnsNet {
    fn lookup(host str) -> []SocketAddr
    fn reverse_lookup(addr IpAddr) -> []str
}
```

**Дизайн-rationale (revised 2026-05-31):**

1. **Split на 4 focused effects** (`TcpNet`/`UdpNet`/`UnixNet`/`DnsNet`).
   Reason: granular mocking (test mocks только нужный sub-system, не full
   stack); function-signature precision (`fn fetch(url) TcpNet DnsNet
   Blocking Fail` — точно видно что используется); layered architecture
   для Plan 116 (TLS wraps only `TcpNet`, не full Net).
2. **Naming `<Protocol>Net`** suffix — clusters с type names (`Tcp*`
   family alphabetically); auto-complete by feature.
3. **Methods без prefix внутри effect** — namespace задан effect-именем
   (`TcpNet.bind`, не `TcpNet.tcp_bind`).
4. **All ops через handler dispatch** — `external fn` libuv-binding'и
   становятся **private** (`libuv_*` prefix), only real handlers их
   вызывают.
5. **`consume`-ops для close** (`TcpNet.close`, `TcpNet.close_reader`,
   `UdpNet.close`, etc) — type-system enforce'ит «нельзя use after close».
6. **Tuple-returns** для multi-value ops (`UdpNet.recv -> ([]u8,
   SocketAddr)`, `TcpNet.split -> (TcpReader, TcpWriter)`).
7. **`[]u8` для всех bytes** — никакого `str` для двоичных payload'ов.

### Real (production) handlers — 4 separate factories

```nova
// std/net/real.nv — production handlers

export fn real_tcp_net() Effect[TcpNet] => effect TcpNet {
    bind(addr) => libuv_tcp_bind(addr)?
    accept(l) => libuv_tcp_accept(l)?
    connect(addr) => libuv_tcp_connect(addr)?
    read(s, max) => libuv_tcp_read(s, max)?
    write(s, data) => libuv_tcp_write(s, data)?
    close(s) => libuv_tcp_close(s)?
    close_listener(l) => libuv_tcp_close_listener(l)?
    split(s) => libuv_tcp_split(s)?
    // close_reader/close_writer → shutdown(fd, SHUT_RD/WR) + refcount decrement
    // refcount=0 → uv_close(handle)
    close_reader(r) => libuv_tcp_close_reader(r)?
    close_writer(w) => libuv_tcp_close_writer(w)?
    // ... ~16 ops total
}

export fn real_udp_net() Effect[UdpNet] => effect UdpNet {
    bind(addr) => libuv_udp_bind(addr)?
    send(s, data, addr) => libuv_udp_send(s, data, addr)?
    recv(s, max) => libuv_udp_recv(s, max)?
    close(s) => libuv_udp_close(s)?
    // multicast ops через raw setsockopt
    multicast_v4_join(s, group, iface) => libuv_udp_multicast_v4_join(s, group, iface)?
    // ... ~12 ops total
}

export fn real_unix_net() Effect[UnixNet] => effect UnixNet {
    bind(path) => libuv_unix_bind(path)?
    accept(l) => libuv_unix_accept(l)?
    connect(path) => libuv_unix_connect(path)?
    read(s, max) => libuv_unix_read(s, max)?
    write(s, data) => libuv_unix_write(s, data)?
    close_stream(s) => libuv_unix_close_stream(s)?
    close_listener(l) => libuv_unix_close_listener(l)?
}

export fn real_dns_net() Effect[DnsNet] => effect DnsNet {
    lookup(host) => libuv_dns_lookup(host)?         // uv_getaddrinfo callback
    reverse_lookup(addr) => libuv_dns_reverse(addr)?
}

// ─── Typed opaque handles (Plan 115 pattern) ─────────────────────────────────
//
// std/net/types.nv — typed opaque pointers wrapping `ptr` (D52 tuple newtype).
// Universal FFI pattern (Plan 115 D214) — works для stdlib И user FFI.
//
// Каждый handle — distinct type над `ptr`:
//   - Construct: `TcpListenerHandle(some_ptr_value)`
//   - Access inner: `handle.0` (positional)
//   - Type-safe: compiler не даст mix TcpListenerHandle и UdpSocketHandle
//   - Zero overhead — same ABI as `ptr` (void*)

type TcpListenerHandle(ptr)
type TcpStreamHandle(ptr)
type UdpSocketHandle(ptr)
type UnixListenerHandle(ptr)
type UnixStreamHandle(ptr)

// ─── Public Nova wrapper types (records over handles) ────────────────────────

export type TcpListener { handle TcpListenerHandle }
export type TcpStream   { handle TcpStreamHandle }
export type UdpSocket   { handle UdpSocketHandle }
export type UnixListener { handle UnixListenerHandle }
export type UnixStream  { handle UnixStreamHandle }

// ─── External fn declarations — RAW C ABI (POSIX/libuv + tuple returns) ──────
//
// Convention (Plan 115 D214):
//   - typed opaque handles (Plan 115 pattern) для socket/stream identities
//   - tuple-by-value returns для multi-value: (handle, err) / (data, err) / etc
//   - int return = UV_E* error code (0 = success, negative = error)
//   - read/write: int с dual meaning (>= 0 = bytes count, < 0 = error)
//
// Allowed types в external fn signatures:
//   ✅ Primitives (int, bool, f64) — direct C ABI
//   ✅ str — Nova (ptr, len) struct, defined ABI
//   ✅ []u8 (and other arrays) — NovaArray_T (ptr, len, cap), defined ABI
//   ✅ POD Nova structs (SocketAddr — matches sockaddr_storage layout)
//   ✅ typed handles `type X(ptr)` (Plan 115 D214) — distinct opaque pointer
//   ✅ tuples-by-value `-> (T1, T2)` — small struct return per platform ABI
//   ✅ mut/consume refs — pointer ABI with Nova ownership tracking
//   ❌ Result[T, E], Option[T] для не-pointer T — sum-type per-instance fragile
//   ❌ Effect[X] — Nova-only abstraction
//   ❌ Generic T без monomorphization context

// Handle allocation — returns (handle, err) tuple
external fn libuv_tcp_listener_new() -> (TcpListenerHandle, i64)
external fn libuv_tcp_stream_new() -> (TcpStreamHandle, i64)
external fn libuv_udp_new() -> (UdpSocketHandle, i64)
external fn libuv_unix_listener_new() -> (UnixListenerHandle, i64)
external fn libuv_unix_stream_new() -> (UnixStreamHandle, i64)

// TCP operations — int return = error code (0=ok, negative=UV_E*)
external fn libuv_tcp_bind(l TcpListenerHandle, addr SocketAddr) -> i64
```

**Архитектурные принципы external fns (Layer 4 — raw FFI):**

1. **No Nova effects** в signature (no `Fail`, no `Blocking`, no other handler-effects)
2. **No Nova abstract types** в return (no `Result`, no `Option`, no sum-types) —
   only **primitives** (int, bool, str, []u8) и **opaque pointers** (TcpStream/etc — wrap'еры вокруг uv_*_t structs)
3. **POSIX/libuv convention**: int return = 0 ok / negative UV_E* error;
   for read/write — int = bytes count (>=0) или negative error code
4. **Results via out-params**: `mut` references (`out_stream mut TcpStream`,
   `out_buf mut []u8`, `out_addrs mut []SocketAddr`) — pre-allocated buffers
   filled by C; standard libuv pattern
5. **Handle creation separate from bind/connect**: `libuv_tcp_*_new()` allocates
   opaque handle (uv_tcp_init called inside); subsequent ops modify
6. **Private** (no `export`) — user не имеет direct access; handler — single user
7. **Naming convention `libuv_*` prefix** — visual differentiation от Nova
   business logic; compiler не requires
8. **C-side implementation** в `compiler-codegen/nova_rt/net.h`/`.c` (или новый
   `net_libuv.c`); each external fn = один C function

**Handler делает Nova-side error mapping (Layer 3):**

```nova
fn map_libuv_error(code i64) -> NetError =>
    match code {
        UV_ECONNREFUSED  => NetError.ConnectionRefused
        UV_ECONNRESET    => NetError.ConnectionReset
        UV_ECONNABORTED  => NetError.ConnectionAborted
        UV_EADDRINUSE    => NetError.AddressInUse
        UV_EADDRNOTAVAIL => NetError.AddressNotAvailable
        UV_ENOTFOUND     => NetError.NotFound
        UV_ECANCELED     => NetError.Cancelled
        UV_ETIMEDOUT     => NetError.TimedOut
        UV_EPIPE         => NetError.BrokenPipe
        UV_EOF           => NetError.UnexpectedEof
        UV_EINVAL        => NetError.InvalidInput(uv_strerror(code))
        _                => NetError.IoError(code, uv_strerror(code))
    }

// Handler unwraps tuple returns + wraps в Nova Fail effect:
export fn real_tcp_net() Effect[TcpNet] => effect TcpNet {
    bind(addr) => {
        ro (h, rc1) = libuv_tcp_listener_new()         // tuple return: handle + err
        if rc1 != 0 { Fail.throw(map_libuv_error(rc1)) }
        ro rc2 = libuv_tcp_bind(h, addr)               // int return: err code
        if rc2 != 0 {
            libuv_tcp_close_listener(h)                 // cleanup partial state
            Fail.throw(map_libuv_error(rc2))
        }
        TcpListener { handle: h }                      // wrap handle в Nova type
    }
    
    accept(l) => {
        ro (h, rc) = libuv_tcp_accept(l.handle)        // unwrap public type → handle
        if rc != 0 { Fail.throw(map_libuv_error(rc)) }
        TcpStream { handle: h }
    }
    
    read(s, max) => {
        ro (data, rc) = libuv_tcp_read(s.handle, max)
        if rc != 0 { Fail.throw(map_libuv_error(rc)) }
        data
    }
    
    write(s, data) => {
        ro (n, rc) = libuv_tcp_write(s.handle, data)
        if rc != 0 { Fail.throw(map_libuv_error(rc)) }
        n
    }
    
    set_nodelay(s, on) => {
        ro rc = libuv_tcp_set_nodelay(s.handle, on)
        if rc != 0 { Fail.throw(map_libuv_error(rc)) }
        ()
    }
    
    // ... все ops follow same pattern
}
```

### Network stack layered effects architecture (architectural intent)

```

                       ↓
```

**Каждый layer:**

  установлен)

### Half-close via split (вместо `shutdown(direction)`)

> **Решение 2026-05-31:** `shutdown(ShutdownDir)` метод УДАЛЁН. Mainstream
> (Rust std/Go/Java/Python) использует runtime flag — это inconsistent с
> Nova consume philosophy (type-system не tracks которые capabilities ещё
> valid после shutdown(Read)). Force split для любого half-close:

```nova
// Полный close — consume на whole stream
stream.close()                          // = full close (FIN+RST, fd released)

// Half-close — обязательный split + consume на нужной half
consume (reader, writer) = stream.split()
writer.close()?                         // = shutdown(Write) под капотом; FIN sent
                                        // writer consumed — type-checker не даст use after
reader.read_to_end()?                   // продолжаем читать
reader.close()?                         // = shutdown(Read); refcount=0 → full close
```

**Тип `ShutdownDir` НЕ существует.** Direction implicit через какую half
ты close'ишь. Type-system enforce'ит compile-time что после
`writer.close()` writer недоступен → нельзя случайно `writer.write(...)`
после.

**Trade-off**: half-close use case теперь требует explicit split (+1
строка) vs `stream.shutdown(Write)`. Worth it за compile-time safety.

**Concurrent r/w usage:**

```nova
fn proxy(stream consume TcpStream) TcpNet Fail[NetError] -> () {
    consume (reader, writer) = stream.split()

    // Каждая half — owned по type-system. Compile-time safe для concurrent r/w.
    spawn |r consume TcpReader| {
        read_loop(r)                // owns reader
    }(reader)

    write_loop(writer)               // owns writer in this fiber
    writer.close()?                  // consume close → shutdown(Write)
}
```

**Type-system enforce'ит:**
- Reader/Writer — consume types: forgetting close → Plan 100.4 cleanup-on-failure
- После split — stream consumed; нельзя случайно использовать
- После close — half consumed; нельзя use-after-close

## D-block changes

### D201 (NEW) — Network stack layered effects architecture (split per-protocol)

**Локация:** `spec/decisions/04-effects.md` (после D61 effects foundation).

**Что.** Архитектурный pattern для всего network stack'а Nova: каждый
сетевой layer — алгебраический effect с handler dispatch.

**Layering:**

```
Application layer  →  HttpClient / HttpServer    ← effect (Plan 117/118)
                       ↓
                      Tls                         ← effect (Plan 116)
                       ↓
                      TcpNet                      ← effect (Plan 91.12, this plan)
                       ↓
                      libuv                       ← private external fn
```

**Каждый layer:**
1. Declares `effect XxxLayer { … }` — set of operations.
2. Provides production handler (`real_xxx()`) — uses lower-layer's `perform`.
3. Mock-friendly — tests заменяют handler через `with XxxLayer = mock`.
4. Type-safe — `XxxLayer` в signature; compile-error если handler не
   установлен (или нет в scope).

**Зачем layered effects (vs single MegaNet effect):**
- Independent mockability: можно mock'ать только Http, оставив real Tls/Net.
- Independent testing: test TLS-only без HTTP overhead.
- Independent evolution: HTTP/3 поверх Net (skip Tls на HTTP/3 over QUIC).
- Composition: один Net handler — multiple consumers (Tls, raw socket fn).

**Acceptance.** См. A1 в Plan 91.12; будущие Plan 116/117/118 наследуют
этот D-block как architecture contract.

### D202 (NEW) — Net effects family contract (`TcpNet`/`UdpNet`/`UnixNet`/`DnsNet`)

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Точные contracts 4 focused effects + production handlers + mock
requirements:

- **`TcpNet`** — TCP transport (~24 ops): basic (bind/accept/connect/read/
  write/close/close_listener), halves (split/read_half/write_half/
  flush_half/close_reader/close_writer), socket options (set_nodelay/
  set_keepalive/set_read_timeout/set_write_timeout/set_recv_buffer_size/
  set_send_buffer_size/listener_set_reuse_address/set_ttl).
- **`UdpNet`** — UDP transport + multicast (~12 ops): bind/send/recv/
  close + 8 multicast ops (v4/v6 join/leave/set_ttl/set_loop).
- **`UnixNet`** — Unix domain sockets (~7 ops): bind/accept/connect/read/
  write/close_stream/close_listener.
- **`DnsNet`** — DNS resolution (~2 ops): lookup/reverse_lookup.

**Production handler factories:**
- `real_tcp_net() -> Effect[TcpNet]`
- `real_udp_net() -> Effect[UdpNet]`
- `real_unix_net() -> Effect[UnixNet]`
- `real_dns_net() -> Effect[DnsNet]`

**Production handler invariants** (apply ко всем 4):
- Thread-safe (multiple fibers могут вызывать concurrently).
- Все blocking ops park fiber через libuv async callback.
- Cancel-aware: supervised-cancel пропагирует через handler в libuv close.
- Errors маппятся из `UV_*` codes в `NetError` variants (полный mapping
  table в D203).

**Mock requirements** (per-effect):
- Mock реализует только нужные ops для данного теста — non-implemented
  throw `NetError.OperationNotPermittedInTest`.
- Tests явно `with TcpNet = mock { … }` (или комбинации); runtime НЕ
  устанавливает default mocks.
- **Granular mocking**: тест который тестирует только DNS — mock'ает
  только `DnsNet`, без необходимости stubить весь network stack.

**Runtime root install** — все 4 handlers:
```nova
fn main() Io -> () {
    with TcpNet = real_tcp_net(),
         UdpNet = real_udp_net(),
         UnixNet = real_unix_net(),
         DnsNet = real_dns_net() {
        user_main()
    }
}
```

Convenience helper `install_all_net()` в `std/net/install.nv` для one-liner.

**Cross-ref:** D50 (Blocking effect), D61 (effects foundation), D201
(layered architecture — TLS Plan 116 wraps only TcpNet, не full Net
family).

### D203 (NEW) — `NetError` typed error contract

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Список variants `NetError` enum (см. секцию «Дизайн»), mapping
из libuv `UV_*` error codes, cross-platform consistency requirements.

**Mapping table** (libuv → NetError):
- `UV_ECONNREFUSED` → `ConnectionRefused`
- `UV_ECONNRESET` → `ConnectionReset`
- `UV_ECONNABORTED` → `ConnectionAborted`
- `UV_EADDRINUSE` → `AddressInUse`
- `UV_EADDRNOTAVAIL` → `AddressNotAvailable`
- `UV_ENOTFOUND` (DNS) → `NotFound`
- `UV_ECANCELED` → `Cancelled`
- `UV_ETIMEDOUT` → `TimedOut`
- `UV_EPIPE` → `BrokenPipe`
- `UV_EOF` → `UnexpectedEof` (если read был partial; иначе clean EOF не
  считается error)
- `UV_EINVAL` → `InvalidInput(message)`
- Прочие → `IoError(errno, message)`

### D204 (NEW) — `[]u8` for raw bytes, `str` for explicit text

**Локация:** `spec/decisions/02-types.md`.

**Что.** Контракт: все network I/O API использует `[]u8` для raw bytes.
`str` доступен только через explicit convenience wrappers (`read_str`/
`write_str`), которые делают UTF-8 validation (returns `Fail[NetError]` на
invalid UTF-8).

**Rationale:** `str` в Nova = UTF-8 invariant; network payload часто
binary; смешивать → silent corruption или panic.

### D205 (WITHDRAWN) — `ShutdownDir` removed

**Локация:** `spec/decisions/08-runtime.md`.

> **Withdrawn 2026-05-31:** `ShutdownDir` type и `shutdown(direction)` method
> УДАЛЕНЫ из API. Runtime flag inconsistent с Nova consume philosophy
> (type-system не tracks которые capabilities valid после shutdown(Read)).
>
> **Replacement:** force split (D206) для любого half-close. `consume (r, w)
> = stream.split()`; затем `consume w.close()` (= shutdown(Write) под
> капотом) или `consume r.close()` (= shutdown(Read)). Type-system
> compile-time enforce'ит «нельзя use after consume».
>
> D205 number retained as «withdrawn marker» для traceability; не peуse'ится
> для другого D-блока.

### D206 (NEW) — `TcpStream.split` via consume-types (единственный способ half-close)

**Локация:** `spec/decisions/05-memory.md` (рядом с D131 consume).

**Что.** `TcpStream consume @split() -> (TcpReader, TcpWriter)` — splits
ownership на две half'ы для concurrent r/w. Каждая half — consume type;
`consume @close()` → `shutdown(direction)`.

**Implementation note:** under-the-hood — shared `uv_tcp_t` handle с
atomic refcount. Refcount=0 → full `uv_close`. Type-system на Nova-level
гарантирует safe ownership без runtime locks.

### D207 (NEW) — DNS async via libuv callback

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** `Dns.lookup(host str) Net Blocking Fail[NetError] -> []SocketAddr`
— async DNS через `uv_getaddrinfo` с callback (не blocking calling thread,
но fiber parks).

### D208 (NEW) — Socket options contract

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Список socket-option ops в `Net` effect (TCP_NODELAY, SO_KEEPALIVE,
read/write timeouts, buffer sizes, SO_REUSEADDR, SO_TTL), cross-platform
semantics, error behavior.

### D50 amend (Blocking effect) — cross-ref на Net

**Локация:** `spec/decisions/06-concurrency.md`.

Добавить cross-ref: «`Net` effect ops (Plan 91.12) — все carry `Blocking`
для parking-операций. Production handler offload'ит через libuv. Mock
handler typically не parks (synchronous response)».

## Risk register

| # | Риск | Митигация |
|---|---|---|
| R-1 | Effect dispatch для ~50 ops добавляет measurable runtime overhead | Bench `perform Net.xxx` vs direct `libuv_*` call в micro-bench — должно быть <5ns overhead. Если больше — optimize codegen (inline-handler через type-specialization, известный handler set in scope). Если не удаётся — extract advanced ops (multicast, Unix, socket options) в Plan 91.13, оставить core TCP/UDP/DNS в 91.12 |
| R-2 | Mock infrastructure требует test framework integration (auto-install mocks в `nova test`?) | V1 решение: `nova test` НЕ устанавливает default mocks — каждый test пишет явно `with TcpNet = …, DnsNet = …` (только нужные sub-effects). Forces explicit + granular mocking. Если ergonomics плохая — Plan 91.13 followup для test-framework integration |
| R-3 | Plan 110 (scoped resources + Cancel) ещё не landed — Cancel-effect interaction с Net handler неясен | Plan 91.12 ships без explicit Cancel effect; supervised-scope cancel пропагирует через existing runtime mechanism (Plan 47/83.10.x). Cross-ref в D202: «Cancel effect (Plan 110) будет augment'ить Net contract когда landed». `[M-91.12-cancel-effect-integration]` followup marker |
| R-4 | Plan 114 (keyword refresh) ещё не landed — Plan 91.12 пишется в каком syntax'е? | Plan 91.12 implementation uses **current** syntax (`let`/`readonly`/`if let`). Когда Plan 114 lands — codemod конвертирует. Если Plan 91.12 ships первым (вероятно) — Plan 114 включит conversion в свой scope. Если Plan 114 ships первым — Plan 91.12 пишется в new syntax (`ro`/`mut`/`consume`/`if Some(x) = e`) |
| R-5 | TcpStream split implementation сложнее expected (refcount + shutdown direction routing) | **Safety hatch Ф.12 preamble**: extract split в Plan 91.13, ship 91.12 без split. Decision point: конец Ф.12.3 |
| R-6 | Socket options cross-platform differences (Windows WSA codes, macOS variants) | Per-option testing на Windows + Linux + macOS. Известные differences mapped в D208 spec. Edge cases (e.g. SO_KEEPALIVE timing на Windows) — documented как «platform-specific behavior may differ» |
| R-7 | Unix domain sockets on Windows — AF_UNIX support только Windows 10 1803+ | Detect at compile time через cfg; older Windows → `NetError.UnsupportedOperation`. Documented в D202 |
| R-8 | Effect declaration с ~50 ops большая — compile time effect-table overhead | Bench codegen time для `effect Net { … }` declaration; если значительный — split effect на под-effects (`TcpEffect`/`UdpEffect`/`DnsEffect` etc.) И каждый top-level handler implement'ит все three. **Decision point:** конец Ф.0.4 |
| R-9 | `Net` ops `consume @close()` могут забываться → resource leaks | Plan 100.4 cleanup-on-failure ловит forgotten consume. Plus `defer close(stream)` pattern в docs/examples |
| R-10 | Migration существующих net-using fixtures (~20 файлов) | Bulk-script + compiler errors (`<SubNet> effect not in signature`) показывают сайты; agent fixes итеративно. Standard rewrite pattern |
| R-11 | DNS async через callback требует thread-safe fiber park/wake — может коллидировать с existing scheduler invariants | Re-use existing pattern из other libuv async ops (TCP connect/read/write — все callback-based уже работают). Если есть corner case — `[M-91.12-dns-async-edge]` followup |
| R-12 | Mock handler в test может leaked resources (если mock возвращает fake `TcpStream` который никогда не close'ится) | Mock test handlers should `consume` the fake stream через `Plan 100.4` mechanism (Plan 100.4 enforces consume even на mock types). Test framework может add `assertNoLeakedNetResources()` helper |

## Rollback strategy

Если после merge выявляется fundamental проблема:

1. **Revert PR** на main — atomic, один merge commit.
2. Worktree `nova-p91-12` preserved для diagnosis (не удаляется автоматически).
3. **Per-phase rollback**: каждая фаза = отдельный commit; можно revert
   подмножество фаз (e.g. revert Ф.12 split если ломается, оставить
   Ф.0-Ф.11 effect+hardening).
4. Cross-platform CI smoke за ~30 минут.
