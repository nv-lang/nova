## Зачем

Plan 83.12 (async net V1) закрыт 2026-05-27 как **MVP**. Корпус std/net (4
файла, ~300 строк) предоставляет TCP/UDP базовый async API. **Comments в
коде честно признают V1 limitations**:

```nova
// error.nv: "V1 bootstrap implementation, external functions return
//   Result[T, str] (erased representation). … Future versions will use
//   typed Result[T, NetError]"
// addr.nv:  "V1 note: DNS resolution is synchronous (uv_getaddrinfo on
//   calling thread). Requires Blocking effect in the caller."  ← но в
//   signature Blocking НЕ объявлен.
```

**9 problems V1 std/net** (отсортированы по severity):

### Critical (0.1 blockers)

1. **`Net` effect не существует.** Effect-density example
   ([examples/effect-density/main.nv:30](examples/effect-density/main.nv#L30))
   документирует `Net` как expected handler-effect:
   `fn run(cfg Config) Net Io Fail[StartupError] -> ()`. **Но в std/net
   `effect Net { … }` нигде не объявлен**. Все net ops — прямые `external
   fn` в libuv. **Mocking невозможен**: тесты вынуждены либо использовать
   реальную сеть (slow, flaky, requires localhost ports), либо избегать
   тестирования net-using кода.

2. **`Blocking Fail[NetError]` annotations отсутствуют в signature.**
   `external fn TcpStream.connect(addr) -> Result[TcpStream, str]` — нет
   эффектов в подписи (только в docstring `# Effects` comment). Type-checker
   **не enforce'ит**:
   - Plan 113 `#realtime` ban: `#realtime fn audio_callback() { TcpStream.connect(...) }`
     **компилируется** (sic!). В runtime — deadlock real-time потока на park'е.
   - Effect-handler contracts: handler-state может быть оставлен в
     невалидном виде при park'е.
   - Cancel-propagation: docs обещают «cancel-safe» но signature не
     отражает.
   - Errors через text-matching (`err.contains("connection refused")`)
     fragile + cross-platform inconsistent.

3. **`read_bytes`/`write` используют `str` вместо `[]u8`.**
   `str` в Nova = UTF-8 invariant. Двоичные протоколы (HTTP/2, gRPC,
   protobuf, любой binary): `read_bytes` либо UTF-8 panic, либо silent
   corruption через `from_bytes_unchecked`. **Fundamental design bug**.
   Mainstream: Rust `&[u8]`, Go `[]byte`, TS `Buffer`/`Uint8Array`,
   Kotlin/Java `ByteArray`/`byte[]`.

### High (production-readiness gaps)

4. **`UdpSocket.recv_from + last_sender()` — race-prone API.** Stateful
   на shared socket: multi-fiber receiver получит **чужой адрес** под
   определённым timing'ом. Mainstream все возвращают tuple `(data, addr)`
   атомарно: Rust `(usize, SocketAddr)`, Go `(n, addr, err)`, Java
   `DatagramPacket`. Nova V1 единственная с separate `last_sender()`.

5. **Нет DNS async API.** Только `SocketAddr.parse("host:port")` —
   синхронный `uv_getaddrinfo` на calling thread. Real async DNS
   (через `uv_getaddrinfo` callback) нет. `Dns.lookup("api.example.com") ->
   []SocketAddr` — необходим для современного клиента.

6. **Нет socket options.** Ни одного: TCP_NODELAY, SO_KEEPALIVE,
   SO_REUSEADDR, read/write timeouts, buffer sizes. Production gRPC/HTTP
   сервер невозможен.

### Medium (ergonomics gaps)

7. **Только полный `close()`, нет half-close.** Half-close patterns
   (HTTP/1.1 connection-close, graceful gRPC shutdown) невозможны. **Решение
   2026-05-31:** force split для half-close — нет `shutdown(direction)`
   метода вообще (runtime-flag inconsistent с Nova consume philosophy),
   вместо этого `TcpStream consume @split() -> (TcpReader, TcpWriter)` +
   `consume @close()` на нужной half'е. Type-system enforce'ит «нельзя
   read after reader.close()». Это **best-in-class** среди mainstream
   (Rust std/Go/Java/Python имеют runtime shutdown без compile-time
   guarantees; только Rust tokio split — близкий аналог).

8. **Нет convenience-методов:** `read_exact`/`read_to_end`/`write_all`.
   Каждый caller — manual loop вокруг partial reads.

9. **Дополнительные missing features:**
   - `IpAddr` отдельно от `SocketAddr` (только host без port)
   - `SocketAddr.v6(...)` constructor (только `loopback_v6` и `parse`)
   - UDP multicast (`join_multicast_v4/v6/set_multicast_ttl_v4/v6/set_multicast_loop_v4/v6`)
   - Unix domain sockets (`UnixListener`/`UnixStream`)
   - TcpStream **split** на `TcpReader`/`TcpWriter` — type-safe
     concurrent read/write через consume-types (Nova-уникальная фича).

## Дизайн

### Foundation: `Net` effect — handler-dispatched, mockable

```nova
// std/net/effect.nv — новый файл, ядро Plan 91.12

export effect Net {
    // TCP transport
    fn bind_tcp(addr SocketAddr) -> TcpListener
    fn accept(listener mut TcpListener) -> TcpStream
    fn connect_tcp(addr SocketAddr) -> TcpStream
    fn read_tcp(stream mut TcpStream, max int) -> []u8
    fn write_tcp(stream mut TcpStream, data ro []u8) -> int
    fn close_tcp(stream consume TcpStream)
    // NOTE: нет shutdown_tcp(dir) — для half-close используется split + close half

    // TCP split halves (Ф.12) — только способ для half-close
    fn split_tcp(stream consume TcpStream) -> (TcpReader, TcpWriter)
    fn read_half(reader mut TcpReader, max int) -> []u8
    fn write_half(writer mut TcpWriter, data ro []u8) -> int
    fn flush_half(writer mut TcpWriter) -> ()
    fn close_reader(reader consume TcpReader)    // semantically = shutdown(Read)
    fn close_writer(writer consume TcpWriter)    // semantically = shutdown(Write)

    // UDP transport
    fn bind_udp(addr SocketAddr) -> UdpSocket
    fn send_udp(sock mut UdpSocket, data ro []u8, addr SocketAddr) -> int
    fn recv_udp(sock mut UdpSocket, max int) -> ([]u8, SocketAddr)
    fn close_udp(sock consume UdpSocket)

    // UDP multicast (Ф.10) — split v4/v6 для type-safety (Rust-style)
    fn join_multicast_v4(sock mut UdpSocket, group Ipv4Addr, iface Option[Ipv4Addr])
    fn join_multicast_v6(sock mut UdpSocket, group Ipv6Addr, iface_idx int)
    fn leave_multicast_v4(sock mut UdpSocket, group Ipv4Addr, iface Option[Ipv4Addr])
    fn leave_multicast_v6(sock mut UdpSocket, group Ipv6Addr, iface_idx int)
    fn set_multicast_ttl_v4(sock mut UdpSocket, ttl u8)
    fn set_multicast_ttl_v6(sock mut UdpSocket, ttl u32)
    fn set_multicast_loop_v4(sock mut UdpSocket, on bool)
    fn set_multicast_loop_v6(sock mut UdpSocket, on bool)

    // Unix domain sockets (Ф.11)
    fn bind_unix(path str) -> UnixListener
    fn accept_unix(listener mut UnixListener) -> UnixStream
    fn connect_unix(path str) -> UnixStream
    fn read_unix(stream mut UnixStream, max int) -> []u8
    fn write_unix(stream mut UnixStream, data ro []u8) -> int
    fn close_unix_stream(stream consume UnixStream)
    fn close_unix_listener(listener consume UnixListener)

    // DNS (Ф.5)
    fn dns_lookup(host str) -> []SocketAddr
    fn dns_reverse_lookup(addr IpAddr) -> []str

    // Socket options (Ф.6)
    fn set_nodelay(stream mut TcpStream, on bool)
    fn set_keepalive(stream mut TcpStream, on bool, idle Option[Duration])
    fn set_read_timeout(stream mut TcpStream, t Option[Duration])
    fn set_write_timeout(stream mut TcpStream, t Option[Duration])
    fn set_recv_buffer_size(stream mut TcpStream, n int)
    fn set_send_buffer_size(stream mut TcpStream, n int)
    fn set_reuse_address(listener mut TcpListener, on bool)
    fn set_ttl(stream mut TcpStream, ttl u8)
}
```

**Дизайн-rationale:**

1. **Single `Net` effect, NOT separate `TcpEffect`/`UdpEffect`/`DnsEffect`.**
   Reason: tests want to mock the **entire network stack** at once. Multiple
   effects → composability nightmare (`fn foo() Tcp Udp Dns Fail -> …`
   verbose).
2. **All ops через handler dispatch** — `external fn` libuv-binding'и
   становятся **private** (`libuv_*` prefix), only `real_net` handler их
   вызывает.
3. **`consume`-ops для close** (`close_tcp`, `close_reader`, etc) —
   type-system enforce'ит «нельзя use after close».
4. **Tuple-returns** для multi-value ops (`recv_udp -> ([]u8, SocketAddr)`,
   `split_tcp -> (TcpReader, TcpWriter)`).
5. **`[]u8` для всех bytes** — никакого `str` для двоичных payload'ов.

### Real (production) handler

```nova
// std/net/real.nv — production Net handler

export fn real_net() Effect[Net] => effect Net {
    bind_tcp(addr) => libuv_tcp_bind(addr)?
    accept(l) => libuv_tcp_accept(l)?
    connect_tcp(addr) => libuv_tcp_connect(addr)?
    read_tcp(s, max) => libuv_tcp_read(s, max)?
    write_tcp(s, data) => libuv_tcp_write(s, data)?
    close_tcp(s) => libuv_tcp_close(s)?
    // close_reader(r) → shutdown(fd, SHUT_RD) + refcount decrement
    // close_writer(w) → shutdown(fd, SHUT_WR) + refcount decrement
    // refcount=0 → uv_close(handle)
    // ... все остальные ops дispatch'ятся в libuv
}

// Private external fn — only real_net handler их использует
external fn libuv_tcp_bind(addr SocketAddr) Blocking Fail[NetError] -> TcpListener
external fn libuv_tcp_accept(l mut TcpListener) Blocking Fail[NetError] -> TcpStream
external fn libuv_tcp_connect(addr SocketAddr) Blocking Fail[NetError] -> TcpStream
external fn libuv_tcp_read(s mut TcpStream, max int) Blocking Fail[NetError] -> []u8
external fn libuv_tcp_write(s mut TcpStream, data ro []u8) Blocking Fail[NetError] -> int
// ... и т.д.
```

### Public API surface — все через `perform Net.xxx`

```nova
// std/net/tcp.nv — public API rewrite

// TcpListener
export fn TcpListener.bind(addr SocketAddr) Net Blocking Fail[NetError] -> TcpListener =>
    perform Net.bind_tcp(addr)

export fn TcpListener mut @accept() Net Blocking Fail[NetError] -> TcpStream =>
    perform Net.accept(@)

export fn TcpListener consume @close() Net Fail[NetError] -> () =>
    perform Net.close_listener(@)

// TcpStream
export fn TcpStream.connect(addr SocketAddr) Net Blocking Fail[NetError] -> TcpStream =>
    perform Net.connect_tcp(addr)

export fn TcpStream mut @read(max int) Net Blocking Fail[NetError] -> []u8 =>
    perform Net.read_tcp(@, max)

export fn TcpStream mut @write(data ro []u8) Net Blocking Fail[NetError] -> int =>
    perform Net.write_tcp(@, data)

// Text convenience — UTF-8 validation on read; encode on write
export fn TcpStream mut @read_str(max int) Net Blocking Fail[NetError] -> str =>
    str.from_utf8(@.read(max)?)

export fn TcpStream mut @write_str(s ro str) Net Blocking Fail[NetError] -> int =>
    @.write(s.as_bytes())

export fn TcpStream consume @close() Net Fail[NetError] -> () =>
    perform Net.close_tcp(@)

// Split (Ф.12) — единственный способ для half-close
export fn TcpStream consume @split() Net -> (TcpReader, TcpWriter) =>
    perform Net.split_tcp(@)

// Convenience (Ф.8) — implemented in Nova, не в effect
export fn TcpStream mut @read_exact(n int) Net Blocking Fail[NetError] -> []u8 {
    mut buf = Vec.new()
    while buf.len() < n {
        ro chunk = @.read(n - buf.len())?
        if chunk.is_empty() { Fail.throw(NetError.UnexpectedEof) }
        buf.extend(chunk)
    }
    buf.to_array()
}
```

### Runtime root устанавливает `real_net` автоматически

`nova` runtime перед запуском user main:

```nova
// Built-in: nova-cli main entry
fn main() Io -> () {
    with Net = real_net() {

```

Так что user пишет `fn main() Net Io -> ()` и handler уже привязан runtime'ом.
Net effect не «протекает» из binary boundary'а — handled в самом верху.

### Network stack layered effects architecture (architectural intent)

```
Application layer  →  HttpClient / HttpServer    ← effect (Plan 116/117 followup)
                       ↓
                      Tls                         ← effect (Plan 115 followup)
                       ↓
                      Net                         ← effect (this plan, Ф.0)
                       ↓
                      libuv                       ← private external fn (FFI)
```

**Каждый layer:**
- Свой algebraic effect (`effect XxxLayer { … }`)
- Real handler (production) — uses lower-layer's perform
- Mock-friendly (tests заменяют handler через `with`)
- Type-safe — нельзя случайно "skip layer" (compile-error если handler не
  установлен)

**Plan 91.12 закладывает фундамент** для всего network stack. TLS/HTTP/WS —
отдельные планы (см. Out of scope), все следуют **тому же effect-handler
pattern** что Net.

### Half-close via split (вместо `shutdown(direction)`)

> **Решение 2026-05-31:** `shutdown(ShutdownDir)` метод УДАЛЁН. Mainstream
> (Rust std/Go/Java/Python) использует runtime flag — это inconsistent с
> Nova consume philosophy (type-system не tracks которые capabilities ещё
> valid после shutdown(Read)). Force split для любого half-close:

```nova
// Полный close — consume на whole stream
stream.close()                          // = full close (FIN+RST, fd released)

// Half-close — обязательный split + consume на нужной half
consume (reader, writer) = stream.split()
writer.close()?                         // = shutdown(Write) под капотом; FIN sent
                                        // writer consumed — type-checker не даст use after
reader.read_to_end()?                   // продолжаем читать
reader.close()?                         // = shutdown(Read); refcount=0 → full close
```

**Тип `ShutdownDir` НЕ существует.** Direction implicit через какую half
ты close'ишь. Type-system enforce'ит compile-time что после
`writer.close()` writer недоступен → нельзя случайно `writer.write(...)`
после.

**Trade-off**: half-close use case теперь требует explicit split (+1
строка) vs `stream.shutdown(Write)`. Worth it за compile-time safety.

### `TcpReader`/`TcpWriter` — split via consume (Ф.12)

```nova
// std/net/tcp.nv — split halves

export type TcpReader consume       // owned read-half, consume @close = shutdown(Read) под капотом
export type TcpWriter consume       // owned write-half, consume @close = shutdown(Write) под капотом

// Получаем через TcpStream.split()
export fn TcpStream consume @split() Net -> (TcpReader, TcpWriter) =>
    perform Net.split_tcp(@)

// Reader API
export fn TcpReader mut @read(max int) Net Blocking Fail[NetError] -> []u8
export fn TcpReader mut @read_str(max int) Net Blocking Fail[NetError] -> str
export fn TcpReader mut @read_exact(n int) Net Blocking Fail[NetError] -> []u8
export fn TcpReader mut @read_to_end() Net Blocking Fail[NetError] -> []u8
export fn TcpReader @peer_addr() -> SocketAddr
export fn TcpReader consume @close() Net Fail[NetError] -> ()       // → shutdown(Read), refcount decrement

// Writer API
export fn TcpWriter mut @write(data ro []u8) Net Blocking Fail[NetError] -> int
export fn TcpWriter mut @write_str(s ro str) Net Blocking Fail[NetError] -> int
export fn TcpWriter mut @write_all(data ro []u8) Net Blocking Fail[NetError] -> ()
export fn TcpWriter mut @flush() Net Blocking Fail[NetError] -> ()
export fn TcpWriter @peer_addr() -> SocketAddr
export fn TcpWriter consume @close() Net Fail[NetError] -> ()       // → shutdown(Write), refcount decrement
```

**Concurrent r/w usage:**

```nova
fn proxy(stream consume TcpStream) Net Blocking Fail[NetError] -> () {
    consume (reader, writer) = stream.split()

    // Каждая half — owned по type-system. Compile-time safe для concurrent r/w.
    spawn |r consume TcpReader| {
        read_loop(r)                // owns reader
    }(reader)

    write_loop(writer)               // owns writer in this fiber
    writer.close()?                  // consume close → shutdown(Write)
}
```

**Type-system enforce'ит:**
- Reader/Writer — consume types: forgetting close → Plan 100.4 cleanup-on-failure
- После split — stream consumed; нельзя случайно использовать
- После close — half consumed; нельзя use-after-close

## D-block changes

### D201 (NEW) — Network stack layered effects architecture

**Локация:** `spec/decisions/04-effects.md` (после D61 effects foundation).

**Что.** Архитектурный pattern для всего network stack'а Nova: каждый
сетевой layer — алгебраический effect с handler dispatch.

**Layering:**

```
Application layer  →  HttpClient / HttpServer    ← effect (Plan 116/117)
                       ↓
                      Tls                         ← effect (Plan 115)
                       ↓
                      Net                         ← effect (Plan 91.12, this plan)
                       ↓
                      libuv                       ← private external fn
```

**Каждый layer:**
1. Declares `effect XxxLayer { … }` — set of operations.
2. Provides production handler (`real_xxx()`) — uses lower-layer's `perform`.
3. Mock-friendly — tests заменяют handler через `with XxxLayer = mock`.
4. Type-safe — `XxxLayer` в signature; compile-error если handler не
   установлен (или нет в scope).

**Зачем layered effects (vs single MegaNet effect):**
- Independent mockability: можно mock'ать только Http, оставив real Tls/Net.
- Independent testing: test TLS-only без HTTP overhead.
- Independent evolution: HTTP/3 поверх Net (skip Tls на HTTP/3 over QUIC).
- Composition: один Net handler — multiple consumers (Tls, raw socket fn).

**Acceptance.** См. A1 в Plan 91.12; будущие Plan 115/116/117 наследуют
этот D-block как architecture contract.

### D202 (NEW) — `Net` effect contract

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Точный contract `effect Net` — список operations, semantics
каждого op'а, production handler invariants, mock requirements.

**Operations** — см. секцию «Дизайн» Plan 91.12 (TCP/UDP/multicast/Unix/
DNS/socket-options, ~50 ops total).

**Production handler invariants:**
- `real_net()` thread-safe (multiple fibers могут вызывать concurrently).
- Все blocking ops park fiber через libuv async callback.
- Cancel-aware: supervised-cancel пропагирует через handler в libuv close.
- Errors маппятся из `UV_*` codes в `NetError` variants (полный mapping
  table в spec).

**Mock requirements:**
- Mock не обязан implement'ить все ops — non-implemented ops can throw
  `NetError.UnsupportedOperation` или `OperationNotPermittedInTest`.
- Tests должны явно `with Net = mock { … }` — runtime НЕ устанавливает
  default mock для tests.

**Cross-ref:** D50 (Blocking effect), D61 (effects foundation), D201
(layered architecture).

### D203 (NEW) — `NetError` typed error contract

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Список variants `NetError` enum (см. секцию «Дизайн»), mapping
из libuv `UV_*` error codes, cross-platform consistency requirements.

**Mapping table** (libuv → NetError):
- `UV_ECONNREFUSED` → `ConnectionRefused`
- `UV_ECONNRESET` → `ConnectionReset`
- `UV_ECONNABORTED` → `ConnectionAborted`
- `UV_EADDRINUSE` → `AddressInUse`
- `UV_EADDRNOTAVAIL` → `AddressNotAvailable`
- `UV_ENOTFOUND` (DNS) → `NotFound`
- `UV_ECANCELED` → `Cancelled`
- `UV_ETIMEDOUT` → `TimedOut`
- `UV_EPIPE` → `BrokenPipe`
- `UV_EOF` → `UnexpectedEof` (если read был partial; иначе clean EOF не
  считается error)
- `UV_EINVAL` → `InvalidInput(message)`
- Прочие → `IoError(errno, message)`

### D204 (NEW) — `[]u8` for raw bytes, `str` for explicit text

**Локация:** `spec/decisions/02-types.md`.

**Что.** Контракт: все network I/O API использует `[]u8` для raw bytes.
`str` доступен только через explicit convenience wrappers (`read_str`/
`write_str`), которые делают UTF-8 validation (returns `Fail[NetError]` на
invalid UTF-8).

**Rationale:** `str` в Nova = UTF-8 invariant; network payload часто
binary; смешивать → silent corruption или panic.

### D205 (WITHDRAWN) — `ShutdownDir` removed

**Локация:** `spec/decisions/08-runtime.md`.

> **Withdrawn 2026-05-31:** `ShutdownDir` type и `shutdown(direction)` method
> УДАЛЕНЫ из API. Runtime flag inconsistent с Nova consume philosophy
> (type-system не tracks которые capabilities valid после shutdown(Read)).
>
> **Replacement:** force split (D206) для любого half-close. `consume (r, w)
> = stream.split()`; затем `consume w.close()` (= shutdown(Write) под
> капотом) или `consume r.close()` (= shutdown(Read)). Type-system
> compile-time enforce'ит «нельзя use after consume».
>
> D205 number retained as «withdrawn marker» для traceability; не peуse'ится
> для другого D-блока.

### D206 (NEW) — `TcpStream.split` via consume-types (единственный способ half-close)

**Локация:** `spec/decisions/05-memory.md` (рядом с D131 consume).

**Что.** `TcpStream consume @split() -> (TcpReader, TcpWriter)` — splits
ownership на две half'ы для concurrent r/w. Каждая half — consume type;
`consume @close()` → `shutdown(direction)`.

**Implementation note:** under-the-hood — shared `uv_tcp_t` handle с
atomic refcount. Refcount=0 → full `uv_close`. Type-system на Nova-level
гарантирует safe ownership без runtime locks.

### D207 (NEW) — DNS async via libuv callback

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** `Dns.lookup(host str) Net Blocking Fail[NetError] -> []SocketAddr`
— async DNS через `uv_getaddrinfo` с callback (не blocking calling thread,
но fiber parks).

### D208 (NEW) — Socket options contract

**Локация:** `spec/decisions/08-runtime.md`.

**Что.** Список socket-option ops в `Net` effect (TCP_NODELAY, SO_KEEPALIVE,
read/write timeouts, buffer sizes, SO_REUSEADDR, SO_TTL), cross-platform
semantics, error behavior.

### D50 amend (Blocking effect) — cross-ref на Net

**Локация:** `spec/decisions/06-concurrency.md`.

Добавить cross-ref: «`Net` effect ops (Plan 91.12) — все carry `Blocking`

  intercepts perform — `with Net = mock { TcpStream.connect(addr) }` returns
  mock_stream, не real socket.

### T1 — Effect annotations

- **T1.1** All public net fn'ы имеют `Net Blocking Fail[NetError]` в
  signature (grep verify).
- **T1.2** `#realtime fn foo() { TcpStream.connect(addr) }` → compile
  error `E_REALTIME_BLOCKING_OP`.
- **T1.3** `fn caller() -> ()` (без `Fail[NetError]`) вызывает
  `TcpStream.connect(addr)?` → compile error «`Fail[NetError]` not in
  caller's effect set».

### T2 — `[]u8` rewrite

- **T2.1** `stream.read(1024)` returns `[]u8` (не `str`; renamed from `read_bytes`).
- **T2.2** `stream.write(some_bytes)` accepts `ro []u8`.
- **T2.3** Binary payload (e.g. random bytes containing 0x80-0xff) — pass
  через read/write без UTF-8 panic.
- **T2.4** Convenience `write_str("hello")` works (UTF-8 known valid).
- **T2.5** `read_str(max)` returns `str` если UTF-8 valid; `Fail` иначе.

### T3 — `recv_from` tuple

- **T3.1** `sock.recv_from(512)` returns `([]u8, SocketAddr)` tuple.
- **T3.2** Multi-fiber receivers — каждый получает свой `(data, addr)`
  атомарно (no race).
- **T3.3** `last_sender()` external fn deleted; grep confirm.

### T4 — Typed `NetError`

- **T4.1** `Result[T, str]` API gone; все public fn returns through `Fail`.
- **T4.2** `connect_to_dead_port` → `Fail.throw(NetError.ConnectionRefused)`
  (не string match).
- **T4.3** `bind_on_used_port` → `Fail.throw(NetError.AddressInUse)`.
- **T4.4** Cross-platform error mapping consistent (test runs same way
  Windows/Linux/macOS).

### T5 — DNS

- **T5.1** `Dns.lookup("localhost")` returns `[]SocketAddr` containing
  127.0.0.1.
- **T5.2** `Dns.lookup("invalid.host.nonexistent.test.")` →
  `Fail.throw(NetError.NotFound)`.
- **T5.3** Async DNS — fiber parks, не blocks event loop (concurrent
  fiber на same loop progresses).

### T6 — Socket options

- **T6.1** `stream.set_nodelay(true)` — works (no error).
- **T6.2** `stream.set_keepalive(true, Some(60.seconds))` — works.
- **T6.3** `stream.set_read_timeout(Some(1.seconds))` + read с no data →
  `Fail.throw(NetError.TimedOut)` после 1s.
- **T6.4** `listener.set_reuse_address(true)` permits immediate rebind
  после close.
- **T6.5** Cross-platform validation.

### T7 — [REMOVED] Half-close via split вместо shutdown

> **Removed 2026-05-31:** не отдельная test serie. Half-close покрывается
> T12 (TcpStream split tests) — close на reader/writer half эквивалентен
> shutdown(Read)/shutdown(Write) под капотом. T7 переименована на T12 если
> агент видит refers.

- **NEG-T7.1** `stream.shutdown(...)` (any direction) — compile error
  `E_METHOD_NOT_FOUND` («`TcpStream.shutdown` retracted in Plan 91.12 —
  use `consume (r, w) = stream.split()` + `consume w.close()` / `consume
  r.close()` для half-close»).
- **NEG-T7.2** Refer to `ShutdownDir` type — compile error «type does not
  exist; half-close via split + consume».

### T8 — Convenience methods

- **T8.1** `read_exact(100)` blocks until exactly 100 bytes received.
- **T8.2** `read_exact(100)` с EOF после 50 → `Fail.throw(NetError.UnexpectedEof)`.
- **T8.3** `read_to_end()` reads until peer close, returns full bytes.
- **T8.4** `write_all(data)` writes all bytes, retries partial writes
  internally.

### T9 — `IpAddr` + `SocketAddr.v6`

- **T9.1** `IpAddr.V4(Ipv4Addr { a: 127, b: 0, c: 0, d: 1 })` constructs.
- **T9.2** `SocketAddr.v6(0xfe80, 0, 0, 0, 0, 0, 0, 1, 8080)` constructs.
- **T9.3** `addr.ip()` returns IpAddr separately from port.

### T10 — UDP multicast

- **T10.1** Two sockets bound; first joins group; sends from second;
  first receives.
- **T10.2** Leave group; first stops receiving.
- **T10.3** `set_multicast_ttl_v4(1)` limits to local network (IPv4); `set_multicast_ttl_v6(1)` для IPv6.

### T11 — Unix domain sockets

- **T11.1** `UnixListener.bind("/tmp/test.sock")` + `UnixStream.connect`
  works (POSIX).
- **T11.2** Windows 10 1803+ — same test passes (AF_UNIX).
- **T11.3** Streaming send/receive works идентично TCP.

### T12 — TcpStream split

- **T12.1** `stream.split()` returns `(TcpReader, TcpWriter)` tuple.
- **T12.2** Spawned fiber owns reader, parent owns writer — concurrent
  r/w работает.
- **T12.3** `reader.close()` (consume) → shutdown(Read).
- **T12.4** `writer.close()` (consume) → shutdown(Write).
- **T12.5** Both closed → underlying handle released (memory leak check
  через `nova bench` GC introspection).
- **T12.6** Use reader after `reader.close()` → compile error (consume
  type already consumed).

### T13 — Mock end-to-end

- **T13.1** Mock Net handler intercepts all calls; recording_net helper
  records ops; assertion on recorded sequence.
- **T13.2** Test `retry_connect` retries 5x on ConnectionRefused mock,
  succeeds on 5th.
- **T13.3** Test без `with Net = …` block — compile error при попытке
  вызвать net op.

### T14 — Cross-layer integration (future-compat smoke)

- **T14.1** Stub TLS layer (mock) over real Net — verify perform
  composition работает.
- **T14.2** Stub Http layer over mock Net — chained handler стек работает.

### Regression

- **R1** Full `nova test` ≥ 1559/74 baseline.
- **R2** Cross-platform Windows + Linux × clang + MSVC.
- **R3** `examples/net/echo_server.nv` + `echo_client.nv` (existing) —
  pass после rewrite.
- **R4** `examples/effect-density/main.nv` — compiles с new Net API.

## Risk register

| # | Риск | Митигация |
|---|---|---|
| R-1 | Effect dispatch для ~50 ops добавляет measurable runtime overhead | Bench `perform Net.xxx` vs direct `libuv_*` call в micro-bench — должно быть <5ns overhead. Если больше — optimize codegen (inline-handler через type-specialization, известный handler set in scope). Если не удаётся — extract advanced ops (multicast, Unix, socket options) в Plan 91.13, оставить core TCP/UDP/DNS в 91.12 |
| R-2 | Mock infrastructure требует test framework integration (auto-install mock в `nova test`?) | V1 решение: `nova test` НЕ устанавливает default mock — каждый test pisа явно `with Net = … { … }`. Forces explicit mocking. Если ergonomics плохая — Plan 91.13 followup для test-framework integration |
| R-3 | Plan 110 (scoped resources + Cancel) ещё не landed — Cancel-effect interaction с Net handler неясен | Plan 91.12 ships без explicit Cancel effect; supervised-scope cancel пропагирует через existing runtime mechanism (Plan 47/83.10.x). Cross-ref в D202: «Cancel effect (Plan 110) будет augment'ить Net contract когда landed». `[M-91.12-cancel-effect-integration]` followup marker |
| R-4 | Plan 114 (keyword refresh) ещё не landed — Plan 91.12 пишется в каком syntax'е? | Plan 91.12 implementation uses **current** syntax (`let`/`readonly`/`if let`). Когда Plan 114 lands — codemod конвертирует. Если Plan 91.12 ships первым (вероятно) — Plan 114 включит conversion в свой scope. Если Plan 114 ships первым — Plan 91.12 пишется в new syntax (`ro`/`mut`/`consume`/`if Some(x) = e`) |
| R-5 | TcpStream split implementation сложнее expected (refcount + shutdown direction routing) | **Safety hatch Ф.12 preamble**: extract split в Plan 91.13, ship 91.12 без split. Decision point: конец Ф.12.3 |
| R-6 | Socket options cross-platform differences (Windows WSA codes, macOS variants) | Per-option testing на Windows + Linux + macOS. Известные differences mapped в D208 spec. Edge cases (e.g. SO_KEEPALIVE timing на Windows) — documented как «platform-specific behavior may differ» |
| R-7 | Unix domain sockets on Windows — AF_UNIX support только Windows 10 1803+ | Detect at compile time через cfg; older Windows → `NetError.UnsupportedOperation`. Documented в D202 |
| R-8 | Effect declaration с ~50 ops большая — compile time effect-table overhead | Bench codegen time для `effect Net { … }` declaration; если значительный — split effect на под-effects (`TcpEffect`/`UdpEffect`/`DnsEffect` etc.) И каждый top-level handler implement'ит все three. **Decision point:** конец Ф.0.4 |
| R-9 | `Net` ops `consume @close()` могут забываться → resource leaks | Plan 100.4 cleanup-on-failure ловит forgotten consume. Plus `defer close(stream)` pattern в docs/examples |
| R-10 | Migration существующих net-using fixtures (~20 файлов) | Bulk-script + compiler errors (`Net effect not in signature`) показывают сайты; agent fixes итеративно. Standard rewrite pattern |
| R-11 | DNS async через callback требует thread-safe fiber park/wake — может коллидировать с existing scheduler invariants | Re-use existing pattern из other libuv async ops (TCP connect/read/write — все callback-based уже работают). Если есть corner case — `[M-91.12-dns-async-edge]` followup |
| R-12 | Mock handler в test может leaked resources (если mock возвращает fake `TcpStream` который никогда не close'ится через real_net) | Mock test handlers should `consume` the fake stream через `Plan 100.4` mechanism (Plan 100.4 enforces consume even на mock types). Test framework может add `assertNoLeakedNetResources()` helper |

## Rollback strategy

Если после merge выявляется fundamental проблема:

1. **Revert PR** на main — atomic, один merge commit.
2. Worktree `nova-p91-12` preserved для diagnosis (не удаляется автоматически).
3. **Per-phase rollback**: каждая фаза = отдельный commit; можно revert
   подмножество фаз (e.g. revert Ф.12 split если ломается, оставить
   Ф.0-Ф.11 effect+hardening).
4. Cross-platform CI smoke за ~30 минут.
