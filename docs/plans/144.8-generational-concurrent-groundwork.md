## Цель
**Write-barriers** (§4.3) для incremental/concurrent GC; подготовка **tri-color**. Concurrent
tri-color GC в C-версии Go (≤1.4) отсутствовал — появился только в go1.5 (C→Go); для Nova это
масштаб post-v1.0 (в пределе — владеть кодогеном: LLVM IR statepoints / свой backend, §6).

## Acceptance criteria
- **Deferred** — отдельная design-сессия; AC определяются тогда.
- «без упрощений как для прода» — обязательный критерий (на момент реализации).

## Дизайн-источники

корень — владеть кодогеном для настоящего Go-уровня concurrent GC).
