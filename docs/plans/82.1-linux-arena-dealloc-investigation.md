## 1. Контекст

Per `[M-82-linux-arena-dealloc]` (simplifications.md, discovered
2026-05-24):

Под Linux (WSL2 + Ubuntu 22.04, glibc 2.35, clang-15) при выполнении
bench `parallel_speedup.nv` выводятся warnings:
```
nova: fiber_dealloc ptr outside arena (0x72972e804000)
nova: fiber_dealloc ptr outside arena (0x72972e004000)
nova: fiber_dealloc ptr outside arena (0x72972d804000)
...
```

Pointer'ы передаваемые в `nova_fiber_dealloc` не попадают в active
arena range. Windows показывает no warnings — другая arena impl
(`fiber_arena_win.c` отдельный путь).

**Hypothesis (initial):**
1. Worker thread exit race — fiber arena retired до того, как fiber's
   dealloc callback fired.
2. Cross-arena dealloc — fiber stack из arena A dealloc'ается из
   arena B context (e.g., после work-stealing thread migration).
3. Plan 83.4.5.8 uncollectable ctx free order — мы free'им ctx
   в worker_main после mco_destroy, но fiber stack belongs to
   mco_coro arena — possible race с arena retirement.

**Последствие:** warning-only (process не abort'ает); fiber's
stack memory **likely leaks** на Linux (dealloc skip'ается).
Под Windows такого нет — другой path.

### 3.3 Hypothesis formation

After code reading, identify exact mechanism:

**Hypothesis 1 — cross-thread dealloc:**
Fiber spawned на worker A, work-stolen worker B, dies на worker B.
Dealloc callback runs на worker B's thread — checks worker B's TLS
arena, не worker A's. → warning.

Verification: grep `_current_worker_id` либо `nova_current_worker` usage
вокруг dealloc. Если worker thread changes mid-fiber-life — confirm
hypothesis.

**Hypothesis 2 — retired arena race:**
Worker A exits, arena retired (added to retire list). Some leftover
fiber dealloc fires after retire → ptr no longer в active range.

Verification: trace retire mechanism. Когда arena moved к retired list?
Какие fibers still alive в этот момент?

**Hypothesis 3 — thread-local arena pointer staleness:**
TLS arena pointer cleared in worker exit, но fiber dealloc reads it
later → NULL либо stale, "outside arena" check fails.

Verification: TLS lifecycle vs fiber lifecycle. Read `_nova_fiber_arena_tls`
либо аналог.

### 3.5 V2 fix outline

Document concrete fix approach для V2 plan:

**Approach A (preferred — TLS-based dispatch):**
Store arena pointer **в fiber ctx** при alloc. Dealloc reads ctx's
arena pointer, не TLS. Fiber migration не breaks (ctx carries owner).

**Approach B (alternative — global registry):**
Global thread-safe registry maps ptr_range → arena. Dealloc lookup
by range. Slower (lock либо tree lookup) but robust.

**Approach C (defer dealloc к owner thread):**
Dealloc callback posts ptr к owner worker's wake_pending list. Owner
thread releases. Adds latency but simple.

Comparison table в V2 plan: complexity, performance, correctness, risk.
