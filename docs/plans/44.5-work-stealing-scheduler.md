# План 44.5: Work-stealing scheduler + per-worker libuv loop

> **Статус:** executable sub-plan, начат 2026-05-13.
> **Parent:** [Plan 44 (M:N runtime roadmap)](44-mn-runtime-roadmap.md).
> **Prerequisite:** [Plan 44.4](44.4-mn-runtime-stage0.md) closed
> (NovaWorker infrastructure live).
>
> **Цель:** реальное распределение workload между workers с work-stealing
> (Chase-Lev deque) + параметризация `nova_supervised_run` per-worker
> libuv loop. После этого `runtime.spawn_global` actually работает —
> fiber'ы запускаются на разных threads, blocked fibers (Time.sleep/
> Channel.recv) корректно park'ятся на own worker loop.
>
> **Соответствие с Go:** Chase-Lev deque = тот же алгоритм что Go's
> `runtime/proc.go::runqsteal`. Per-worker loop = Go's M-P-G model
> (каждый P имеет own scheduler state).

---

## Что должно работать после Plan 44.5

```nova
import std.runtime.runtime

fn main() {
    runtime.init(4)  // 4 OS workers

    let ch = Channel.new(100)
    supervised {
        // 100 fibers spawned — Plan 44.5 распределяет round-robin,
        // workers steal'ят work если очередь uneven.
        for i in 0..100 {
            spawn {
                Time.sleep(10)         // park на own worker loop
                ch.tx.send(i * 2)      // cross-worker channel safe
            }
        }
        spawn {
            var sum = 0
            while let Some(v) = ch.rx.recv() {
                sum += v
            }
            assert(sum == 9900)        // 0+2+4+...+198
        }
    }

    runtime.shutdown()
}
```

**Acceptance:**
- 4 workers полностью загружены (через `runtime.current_worker_id()` — fibers распределены).
- Total throughput ≥ 3× single-thread на 4-core.
- Time.sleep в worker fiber park'ится на own loop, не global.
- Cross-worker channel send/recv корректно работают.
- Work-stealing: если один worker занят, другие steal'ят.

## Архитектура

### Layer 1: Chase-Lev deque (lock-free work-stealing)

`compiler-codegen/nova_rt/deque.h` — single-producer multi-consumer deque.
Каноническая реализация (Chase & Lev, 2005). ~250 LOC C11 atomics.

```c
typedef struct {
    /* Power-of-2 size, циркулярный буфер. */
    nova_atomic_intptr     top;       /* steal-side index (FIFO read) */
    nova_atomic_intptr     bottom;    /* owner-side index (LIFO push/pop) */
    nova_atomic_intptr*    buffer;    /* указатель на array[size] */
    size_t                 mask;      /* size - 1 для modulo */
} NovaDeque;

/* Owner-side ops (called только current thread). Wait-free. */
void   nova_deque_init(NovaDeque* d, size_t initial_size);
void   nova_deque_push(NovaDeque* d, void* item);   /* LIFO push */
void*  nova_deque_pop(NovaDeque* d);                 /* LIFO pop, NULL if empty */

/* Steal-side op (called any thread). Lock-free. */
void*  nova_deque_steal(NovaDeque* d);              /* FIFO steal, NULL if empty/contended */

void   nova_deque_destroy(NovaDeque* d);
```

**Properties (Chase-Lev guarantees):**
- Owner push/pop **wait-free** (no atomic CAS in hot path).
- Steal **lock-free** (CAS may fail under contention, but progresses).
- No false steals от same item.
- Resize support через atomic buffer pointer swap (рост по требованию).

### Layer 2: Worker integration

Замена `NovaWorker.scope` (текущий push queue) на Chase-Lev deque:

```c
typedef struct NovaWorker {
    int               id;
    uv_thread_t       thread;
    uv_loop_t         loop;
    uv_async_t        wake_handle;
    NovaDeque         local_deque;       /* NEW — replaces queue_mu+scope */
    /* fiber scope для cancellation propagation — отдельно от deque. */
    NovaFiberQueue    scope;
    nova_atomic_bool  stop;
    nova_atomic_int   parked;            /* workers idle counter */
} NovaWorker;
```

`scope` остаётся для supervised-style cancel propagation; **fiber'ы в
deque** распределяются между workers, но **per-fiber state** ссылается
на parent scope.

### Layer 3: Worker main loop с work-stealing

```c
static void _worker_main(void* arg) {
    NovaWorker* self = (NovaWorker*)arg;
    _current_worker_id = self->id;
    _nova_active_scope = &self->scope;

    while (!nova_abool_load(&self->stop)) {
        mco_coro* co = NULL;

        /* (1) Local deque — owner-side LIFO pop. */
        co = nova_deque_pop(&self->local_deque);

        /* (2) Idle — try steal у соседей. */
        if (!co) {
            for (int i = 0; i < _n_workers; i++) {
                if (i == self->id) continue;
                co = nova_deque_steal(&_workers[i].local_deque);
                if (co) break;
            }
        }

        /* (3) Still nothing — block в libuv (own loop). */
        if (!co) {
            nova_aint_inc(&self->parked);
            uv_run(&self->loop, UV_RUN_ONCE);
            nova_aint_dec(&self->parked);
            continue;
        }

        /* (4) Run fiber. */
        _nova_active_slot = /* slot index в self->scope */;
        mco_resume(co);

        /* (5) Fiber finished or yielded — handle accordingly. */
        if (mco_status(co) == MCO_DEAD) {
            mco_destroy(co);
        } else if (mco_status(co) == MCO_SUSPENDED) {
            /* Yielded — back to deque. Если fiber парк'нулся (waiter
             * на channel), он НЕ в deque — wake'нет его cross-worker
             * push через uv_async_send. */
            nova_deque_push(&self->local_deque, co);
        }
    }
}
```

### Layer 4: Per-worker libuv loop параметризация

Сейчас `nova_supervised_run` использует global `nova_evloop()`. Под M:N
worker должен использовать own loop. Изменения:

1. `nova_supervised_run(scope)` → `nova_supervised_run(scope, loop)`.
2. `Nova_Time_after(ms)` — registers `uv_timer_t` на current TLS loop:
   ```c
   uv_loop_t* loop = _nova_current_loop();  /* TLS, set by worker_main */
   uv_timer_init(loop, &timer);
   ```
3. `nova_sched_park(scope, slot)` — если slot park'ится waiting libuv
   event, callback fires на том же thread'е что зарегистрировал handle
   (libuv guarantee). Cross-worker wake через `uv_async_send`.

### Layer 5: Cross-worker channel correctness

Plan 44.1 (channels) уже **M:N-ready** (atomics, mutex, doubly-linked,
direct-copy). Под Plan 44.5 ничего не меняется в channel runtime.

Edge case: `send` блокируется до recv. Сейчас sender park'ится на own
scope. Когда recv'er на другом worker'е → cross-worker wake через
`nova_sched_wake(sender->scope, sender->slot)`. Это вызывает
`uv_async_send(&sender_worker->wake_handle)`. Sender worker awakes,
видит sender в parked list, resume'ит его на own thread.

**Critical:** waiter state (BaseWaiter struct) живёт на sender's fiber
stack. Если sender fiber **мигрирует** на другой worker между park и
wake — `_nova_active_scope` / `_nova_active_slot` invalid. Plan 44.5
**запрещает migration** — fiber pin'ится к worker'у где park'нулся.

Migration = Plan 44.6 (TLS migration + per-fiber state).

### Layer 6: Boehm GC_THREADS

Workers вызывают `nova_alloc` (channels, NovaSpawnCtx, NovaAfterState
malloc'нут). Boehm под multi-thread требует `GC_THREADS=1` build +
`GC_register_my_thread()` per worker.

- **Linux Docker**: `libgc-dev` Ubuntu 22.04 уже built с threads.
- **Windows vcpkg**: `bdwgc[multithreaded]` feature flag (custom vcpkg config).

Plan 44.5 включает GC registration в worker main loop:
```c
struct GC_stack_base sb;
GC_get_stack_base(&sb);
GC_register_my_thread(&sb);
```

И `GC_unregister_my_thread()` в worker cleanup.

Если Boehm build не имеет GC_THREADS — вернуться к Plan 44.4 single-thread
PoC через runtime flag.

## Не входит в Plan 44.5

- **TLS migration** — fiber pin'ится к worker'у. Migration = Plan 44.6.
- **Preemption** — cooperative-only. Long CPU-bound fiber монополизирует
  worker. = Plan 44.7.
- **Blocking pool** — `Blocking` effect остаётся inline. = Plan 44.8.
- **std.sync Atomic[T]** — Plan 18 P0.
- **Concurrent GC tuning** — BDW-GC default, свой GC = отдельный milestone.

## Риски Plan 44.5

- **R1: Chase-Lev complexity.** Канонический алгоритм 200 строк, но
  edge cases (ABA problem, resize correctness) тонкие. Тестировать
  под TSan на Linux Docker.
- **R2: Per-worker libuv loop**: handle migration между threads —
  libuv `uv_xxx_t` handles **thread-bound**. Если fiber park'ится на
  worker A's timer, потом worker A освобождает thread → handle
  orphan'ится. Решение: pin fiber к worker'у пока handle active.
- **R3: Boehm GC_THREADS Windows build**: vcpkg feature flag может
  не быть, custom build complex. Fallback на single-thread.
- **R4: Cross-worker contention** на steal. Chase-Lev минимизирует через
  FIFO steal-side, но 16 workers × 16 stealers = N² CAS per cycle.
  Mitigation: try steal только от neighbor, random victim selection.
