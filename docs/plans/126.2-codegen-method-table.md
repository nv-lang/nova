## 1. Проблема

Plan 126 V1 заложил **type-check level** auto-derive: `verify_impl_protocols`
(`compiler-codegen/src/types/mod.rs`) вызывает
`auto_derive::synthesize_method(...)` и при успехе **suppress'ит**
`E_IMPL_MISSING_METHODS`. Это позволяет коду с `#impl(Equatable + ...)`
проходить type-check без explicit method body.

**Но synthesized `FnDecl` выбрасывается** — он НЕ регистрируется в
method_table. Прямая цитата из `types/mod.rs` (Ф.4 V1):

```rust
match crate::protocols::auto_derive::synthesize_method(
    &mut derive_ctx, td, proto_name,
) {
    Ok(_fn_decl) => {
        // V1: synthesized FnDecl не register'ится в method_table
        // immediately — Plan 126 Ф.5+ extension покрывает codegen
        // integration. Здесь только suppress'им E_IMPL_MISSING_METHODS,
        // подтверждая что фактическая synthesis возможна.
        synthesized_ok = true;
    }
    ...
}
```

Последствие: runtime `a == b` на `#impl(Equatable)`-type **НЕ вызывает**
synthesized `@equals` — резолвится к default (identity-eq для heap-record,
либо C compile error для value-record stack-slot ABI). Аналогично
`a.clone()` / `a.hash()` / `a.compare(b)` / `a.fmt(sb)` не имеют C-тел.

**Net:** Plan 126 V1 = type-check теория без runtime реализации. Это
половина фичи.

## 2. Fix overview

### Ф.1 — Register synthesized FnDecls в method_table

При `check_impl` / `verify_impl_protocols`, когда `synthesize_method` →
`Ok(fn_decl)`:
- Регистрировать `fn_decl` в method_table под `(type_name, method_name)`
  ключом — так чтобы method-resolution (operator `==` → `@equals`,
  method-call `.clone()`) находил synthesized метод как обычный
  user-declared метод.
- Synthesized FnDecl должен пройти ту же type-check / lowering pipeline,
  что и user-declared методы (либо инжектиться до lowering phase).
- Idempotency: повторная регистрация одного `(type, method)` не дублирует.
- Explicit method override (Plan 126 A126.10): если user предоставил
  `fn T @equals(...)` — synthesis НЕ запускается (уже guarded V1
  `find_method_decl` check), synthesized не перетирает user-метод.

### Ф.2 — Codegen emit C-тел + operator dispatch

- `emit_c.rs` (или соответствующий codegen pass) должен emit'ить C-тела
  для synthesized методов точно как для user-declared (они теперь в
  method_table → должны попасть в codegen iteration естественно, либо
  через explicit injection в declaration list).
- Operator dispatch: `a == b` на `#impl(Equatable)`-type →
  `Nova_<T>_method_equals(a, b)` (вместо identity / stack-slot fallback).
  D109 amend: heap-record `==` resolution с auto-derived `@equals`
  overrides identity default.
- Value-record (Plan 124.8 / 127.x ABI): synthesized методы должны
  использовать корректный receiver ABI (stack `NovaValue_X` vs promoted
  `Nova_X*` — leverage Plan 127.1 `prepare_method_recv` + field-access
  codegen).
- `.clone()` → `Nova_<T>_method_clone`; `.hash()` → `Nova_<T>_method_hash`;
  `.compare()` → `Nova_<T>_method_compare`; `.fmt(sb)` →
  `Nova_<T>_method_fmt`.

### Ф.3 — Tests (runtime behavior)

`nova_tests/plan126_2/` — end-to-end runtime fixtures (требуют
исполнения, не только type-check):
- value-record `==` memberwise eq → correct true/false runtime.
- heap-record `#impl(Equatable)` → memberwise (overrides identity).
- `.clone()` deep copy → mutate clone не трогает original.
- `.hash()` → equal objects equal hashes.
- `.compare()` → lexicographic ordering runtime.
- `.fmt(sb)` → correct string output `T { f: v, ... }`.
- Recursive (nested record) runtime correctness.
- Explicit override wins (user `@equals` called, not synthesized).
- ≥10 fixtures (mix POS runtime + regression).
- Regression: plan126 21/21 + plan124_8 + plan127 unchanged.

## 4. Acceptance criteria (A126.2.1–A126.2.10)

- **A126.2.1** Synthesized `FnDecl` от `synthesize_method` регистрируется
  в method_table под `(type_name, method_name)` ключом.
- **A126.2.2** value-record `#impl(Equatable)`: runtime `a == b` вызывает
  synthesized `@equals` (memberwise), корректный true/false.
- **A126.2.3** heap-record `#impl(Equatable)`: runtime `a == b` memberwise
  overrides identity default (D109 amend).
- **A126.2.4** `#impl(Cloneable)`: runtime `a.clone()` deep copy — mutate
  clone не модифицирует original.
- **A126.2.5** `#impl(Hashable)`: runtime `a.hash()` — equal objects
  produce equal hashes.
- **A126.2.6** `#impl(Comparable)`: runtime `a.compare(b)` lexicographic
  ordering correct (-1/0/1).
- **A126.2.7** `#impl(Printable)`: runtime `a.fmt(sb)` outputs
  `T { f1: v1, f2: v2 }`.
- **A126.2.8** Recursive synthesis runtime correctness (nested record 2+
  levels): inner synthesized methods вызываются.
- **A126.2.9** Explicit method override (A126.10 holds at runtime): user
  `fn T @equals(...)` called, synthesized НЕ генерируется/не перетирает.
- **A126.2.10** Regression clean: plan126 21/21 PASS + 30/30 auto_derive
  unit tests + plan124_8 + plan127 fixtures unchanged; lib build clean.
