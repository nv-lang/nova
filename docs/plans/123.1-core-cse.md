## 1. Контекст

Plan 123 umbrella (`docs/plans/123-receiver-field-cse.md`) §1 описал
проблему: codegen Nova эмитит literal `self->X` на каждый `@X`-доступ
в method body, hot-path methods accessing `@field` 5-15 раз →
redundant pointer dereferences. C-оптимизатор частично спасает на
`-O2` через NoAlias analysis, но non-deterministic perf
characteristic; на `-O0` (debug) — zero CSE.

V1 sub-plan — **core CSE** на уровне Nova-codegen AST-pass:
- `ro` (readonly) field unconditional caching;
- `mut` field straight-line write-region caching;
- escape hatch `--field-cache-threshold=0`;
- debug-info preservation;
- semantic equivalence guarantee.

V2-V7 (sub-plans 123.2-123.7) — LICM, pure-call caching, chain
caching, LSP code-lens, telemetry, IPA — отдельные sub-plans.

## 2. Design

### 2.1 Pipeline position (DECISION-A)

Pass `field_cache::cache_module` инвокируется в pipeline ПОСЛЕ:
- `parser::parse` ✅ (AST built)
- `types::check_module` ✅ (types validated → ro/mut classification
  derivable из RecordField + DocAttr)
- `const_fn_eval::rewrite_const_fn_calls` ✅ (const literals inlined)
- `types::annotate_map_literals` + `desugar::desugar_module` ✅
  (map literals desugared)
- `types::infer_effects` ✅ (effect-row stable)
- `callnorm::normalize_module` ✅ (call-args normalized — последняя AST
  трансформация)

И ПЕРЕД:
- `codegen::CEmitter::emit_module` ❌ (codegen видит уже cached AST)

Для `nova run` / `nova test-interp` (interpreter path) — пасс работает
аналогично (после `desugar`, перед `Interpreter::load_module`).

**Обоснование:** на этой позиции (1) типы валидированы, (2) AST
стабилен (никаких новых трансформаций), (3) codegen и interpreter
видят transformation transparently — pass возвращает обычный AST с
дополнительными `let _at_<F> = @<F>` bindings + replaced `@<F>` ->
`_at_<F>` reads.

### 2.2 Cache local naming (DECISION-B)

- Pattern: **`_at_<field>`** (per Plan 123 umbrella §3.2; D217 §4).
  Подчёркивает «cache for `@<field>`» и читаемо в debug-info.
- Collision avoidance: если user уже имеет binding `_at_<field>` в
  scope (через let / param), append numeric suffix `_at_<field>_<N>`
  где N — fn-local counter (counter incremented только когда collision
  detected — keep names stable в no-conflict case для bit-identical
  output across builds).
- Counter — **per-fn** (reset between methods → deterministic output).

### 2.3 ro vs mut classification (DECISION-C)

V1 cache eligibility — определяется через `RecordField` flags +
mutation analysis:

1. **`readonly` field (D175 `ro` modifier):** `RecordField.readonly ==
   true`. **Unconditional cache** — поле frozen post-construction, no
   mutation possible → safe across entire method body.
2. **Default-mode field (neither `readonly` nor `mutable`):** в
   bootstrap-edition default = mutable. V1 treat как mut field
   (straight-line analysis).
3. **`mut` field (`RecordField.mutable == true`):** straight-line
   write-region analysis. Cache live между mutation boundaries.

V1 **НЕ** cache: `consume` fields (D131 linearity — separate semantics,
Plan 100.x cross-ref), embed fields (`use _ Type`), и поля типов,
объявленных без `RecordField` (`Sum` variants — отдельный path).

### 2.4 Threshold (DECISION-D)

- Default threshold N=2: cache emit'ится только если field accessed
  **≥2** times в method body.
- CLI flag `--field-cache-threshold=N`:
  - `N=0` → feature OFF полностью (escape hatch).
  - `N=1` → cache даже single access (отладка, не recommended для
    production — stack-frame bloat без winn'а).
  - `N=2` (default) → 2+ accesses required.
  - `N≥3` → conservative — cache только heavily-accessed.
- Bound на cache locals per fn: **8** (`--field-cache-max=N`).

### 2.5 V1 scope (DECISION-E)

ONLY: direct `@field` access = `ExprKind::Member { obj:
ExprKind::SelfAccess, name }`.

**Skipped в V1** (sub-plans):
- Chain `@a.b.c` → Plan 123.4.
- `@field.method()` call caching → Plan 123.3.
- Loop-invariant hoisting → Plan 123.2.
- Inter-procedural → Plan 123.7.

### 2.6 Edge case handling (DECISION-F)

**Closures:** если closure body (ClosureLight / ClosureFull / Lambda)
referencing `@field` встречается **anywhere в method body** →
conservative skip caching that specific field (закрытая closure может
outlive scope и captured `self`-pointer вызывает aliasing surprise).

**`perform Effect.X`:** для mut fields, `perform` boundary
invalidates cache → re-cache after если accessed снова. Pass detects
perform через Call с callee resolved to effect method (effect_surface
crate). V1 conservative: ANY method-call в body может trigger effect
handler → mut-field cache invalidated after любого call. Ro-fields —
unaffected (frozen).

**Vtable dispatch:** pass runs ПОСЛЕ type-check (types resolved). Для
protocol-typed receiver (`fn ProtocolFoo @method`) — pass skip
caching полностью (protocol method bodies — separate dispatch path,
caching без знания concrete impl unsafe).

**Generic mono:** RecordField.readonly/mutable — type-decl-level
property, generic-instantiation-agnostic. Pass читает классификацию из
TypeDecl-уровня; codegen-mono pipeline downstream видит уже cached
AST.

**Receiver mutation (`mut self`):** receiver itself (whole self-pointer)
re-binding запрещён в Nova syntax (нет `@= X` для всего self).
Mutation в Nova методах — только через `@field = ...`. Pass tracks
exactly это.

**`consume self` methods:** receiver consumed at exit — мутации внутри
тела работают, но receiver не может pass дальше. V1 — рассматриваем
тело как обычный method body (consume аффектирует только post-method
linearity, не intra-method).

### 2.7 Semantic equivalence guarantee (§8.1 umbrella)

Pass — **pure AST-to-AST трансформация**:

For any input AST `A`, output AST `T(A)`:
- Every read `Member{SelfAccess, F}` in `A` becomes either:
  - same `Member{SelfAccess, F}` (если threshold не met / в excluded
    region) — identical semantics;
  - OR `Ident(_at_F_X)` где `_at_F_X` bound earlier к ровно
    `Member{SelfAccess, F}` evaluated в the same control-flow region
    без intervening mutation of `@F` — identical value.
- Every write `Assign { target: Member{SelfAccess, F}, ... }` —
  unchanged + cache re-emission inserted на subsequent reads.
- Closure / `perform` boundaries — conservative invalidation.
- No reordering of side effects.
- No additional allocations (let-binding на C int / pointer ≈ register
  reuse).
- Throws / break / continue / return — work as before (cache let'ы
  scope'ом охватывают всю function-body, invalidation reset не
  меняет semantics).

Verification methods (§9):
- Unit tests AST-level (input → expected output).
- Codegen `.c` diff before/after.
- Semantic `.nv` programs run identically (`nova test` ON vs OFF).
- Property tests: random AST → equivalence check.
- Regression: full nova test — 0 new FAIL.

### 2.8 Debug-info preservation

Generated `_at_<F>` locals имеют `Span = span of original @field
access`. Codegen DWARF/PDB emit reflects это — debugger показывает
`_at_<F>` mapped back to source `@<F>`. V1 implementation: span
clone'ится от первого `Member{SelfAccess, F}` access в region —
sufficient для bootstrap (subsequent reads все mapped к этому
prefix).

## 5. Production-grade requirements

Per Plan 123 umbrella §8. Specifically для V1:

- **Semantic equivalence (§8.1):** unit + codegen-diff + semantic +
  property + differential — все 5 verification methods применяются.
- **Cross-platform determinism (§8.2):** pass deterministic
  (no HashMap iteration order leak — sort keys где applicable).
  Windows MSVC / Linux clang — same input → same `.c`.
- **Bounded overhead (§8.3):** compile-time <5%, ≤8 cache locals/fn,
  binary size ≤2% bloat, debug-info ≤5% growth.
- **Escape hatch (§8.4):** `--field-cache-threshold=0` → bypass pass.
- **Edition compat (§8.5):** V1 — enable unconditionally (semantic
  equivalence guarantee достаточная). No edition gating.
- **Debug-info (§8.6):** span preserved через AST node cloning.
- **Telemetry (§8.7):** V1 — minimal (debug-emit counter under
  `--field-cache-threshold=N` logging). Production telemetry — Plan
  123.6.

## 6. Risk register (V1 specific)

Per Plan 123 umbrella §7.1 (R-1.1, R-1.2, R-1.3):

- **R-1.1: closure capture aliasing.** Mitigation: closure detection
  → skip cache.
- **R-1.2: vtable dispatch type.** Mitigation: pass invokes ПОСЛЕ
  type-check; protocol receiver — skip.
- **R-1.3: AST mutation debugging UX.** Mitigation: span preserved
  через clone.
- **R-1.4 (new): name collision с user locals.** User имеет `let
  _at_pos`. Mitigation: detect-before-emit + numeric suffix.
- **R-1.5 (new): hidden mutation via call.** Method call may mutate
  `@field` через alias. V1 conservative — ANY call invalidates mut
  cache. Refined в Plan 123.7 (IPA).
