### 1.2 Сравнение с Tokio (полезный референс)

Проведено сравнение через WebFetch [tokio source](https://github.com/tokio-rs/tokio):

**Tokio TimerEntry** (`tokio/src/runtime/time/entry.rs`):
- Один `AtomicU64` encoded: timestamp ∣ STATE_DEREGISTERED ∣ STATE_PENDING_FIRE
- Pattern "register-first": waker регистрируется (Release) **ДО** финальной
  проверки state (Acquire). Гарантия: «либо canceller увидит зарегистрированный
  waker, либо мы увидим racing-cancel, либо оба»
- Один `compare_exchange_weak(AcqRel/Acquire)` атомарно «проверить + изменить»

**Tokio Task State** (`tokio/src/runtime/task/state.rs`):
- Один `AtomicUsize` packed: RUNNING + COMPLETE + NOTIFIED + CANCELLED + refcount
- `transition_to_notified_and_cancel()` = **один CAS** на cancel + schedule

**Tokio Notify** (`tokio/src/sync/notify.rs`):
- `AtomicUsize` lower 2 bits state (EMPTY/WAITING/NOTIFIED) + upper bits notify counter
- Pattern: optimistic CAS → fail → acquire lock → reload state → recheck → register waker

**Общий принцип:** ОДИН атомик на ВЕСЬ lifecycle waiter'а. SeqCst или AcqRel
на CAS. Нет "окон" между операциями.

### 1.3 Текущая Nova-архитектура (для сравнения)

```c
// _nova_sleep_via_libuv (compiler-codegen/nova_rt/fibers.h)
NovaFiberQueue* cancel_scope = base->_nova_parent_scope;

// Race 2a:
if (nova_abool_load(&cancel_scope->cancel_requested)) return;     // [1]
NovaSleepState st = ...
uv_timer_init(...)
uv_timer_start(...)
nova_sched_register_pending(scope, slot, &st.timer, _nova_sleep_stop_cb);  // [2]
// Race 2b:
if (nova_abool_load(&cancel_scope->cancel_requested)) {           // [3]
    CAS PENDING→CLOSING; uv_close(&st.timer, _close_cb)
}
nova_sched_park_until(scope, slot, _nova_sleep_stage_is_closed, &st);  // [4]
```

**Три раздельных атомика** на разных address'ах разной памяти:
- `cancel_scope->cancel_requested` (supervised scope heap)
- `pending_stop_cb[slot]` (worker's sched_state heap)
- `st.stage` (stack of parked fiber's coroutine)

`cancel_worker_fibers` читает их в три отдельных load'а:
```c
co = w->scope.fibers[j];                              // worker scope memory
base = mco_get_user_data(co);
base->_nova_parent_scope                              // SpawnCtx heap
__atomic_load(&st->pending_stop_cb[j], &cb, ACQUIRE); // sched_state heap
hdl = st->pending_handle[j];                          // sched_state heap
```

### 1.4 Гипотеза race'а (из Tokio paradigm)

Возможный путь по которому теряется wake (для будущей диагностики):

1. Worker A: `register_pending` RELEASE store cb[j] = `_nova_sleep_stop_cb`
2. Cancel thread (другой worker C, после `tok.cancel()`):
   - ACQUIRE-load `w->scope.count` → видит slot j
   - read `fibers[j]` → OK
   - ACQUIRE-load `pending_stop_cb[j]` → читает stale NULL (slot reuse + cache aliasing?)
3. Cancel thread: skip slot (cb==NULL, parked[j]==false)
4. Worker A: Race 2b ACQUIRE-load `cancel_requested` → false (cancel-store ещё не виден)
5. Worker A: `park_until` → `mco_yield` → **fiber parked permanently**

В Tokio это невозможно: ОДИН атомик `compare_exchange` сразу делает обе операции.
У нас three-load → ordering между ними не гарантирована между двумя независимыми
heap страницами под GC, особенно с slot reuse.

**Подтверждение через bench-vs-iso difference:** warm workers — slot'ы уже использовались,
cache topology установлена. Cold workers (first/only test) — свежеаллоцированные slot'ы,
cross-CPU cache invalidation more likely → race window шире.

## 3. Стратегия — попытка по возрастающей сложности

### Ф.A — Tactical: predicate self-close (≤0.5 dev-day)

**Идея:** В `nova_sched_park_until` predicate уже re-check'ает
`_nova_sleep_stage_is_closed` на каждый spurious-wake. Добавить ещё одну проверку
там же: если `cancel_scope->cancel_requested == true` И стейдж всё ещё `PENDING`
→ self-initiate close (`CAS PENDING→CLOSING` + `uv_close`) прямо из callback'а.

Это аналог Tokio's **"recheck under lock"** в `Notified::poll`: после регистрации
waker'а под locking перепроверить state. Только у нас "lock" — это park-loop
predicate.

**Файлы:** `compiler-codegen/nova_rt/fibers.h`

**Эскиз изменения:**
```c
// Заменить _nova_sleep_stage_is_closed на:
static nova_bool _nova_sleep_stage_check_or_self_close(void* ctx) {
    NovaSleepState* st = (NovaSleepState*)ctx;
    int32_t stage = nova_aint_load(&st->stage);
    if (stage == NOVA_SLEEP_CLOSED) return true;

    // Race 2c (Tokio-style recheck-under-park):
    // Cancel мог стать requested ПОСЛЕ Race 2b — проверяем здесь.
    NovaFiberQueue* cancel_scope = st->cancel_scope_cache;  // capture at setup
    if (cancel_scope && nova_abool_load(&cancel_scope->cancel_requested)) {
        int32_t expected = NOVA_SLEEP_PENDING;
        if (nova_aint_cas(&st->stage, &expected, NOVA_SLEEP_CLOSING)) {
            // Same-worker thread → uv_close direct (как в Race 2b).
            uv_close((uv_handle_t*)&st->timer, _nova_sleep_close_cb);
        }
    }
    return false;  // park (close_cb разбудит)
}
```

Потребуется добавить `cancel_scope_cache` в `NovaSleepState` (capture'ить в
`_nova_sleep_via_libuv` после Race 2a).

**Почему предположительно сработает:** Если cancel_worker_fibers пропустил slot
из-за stale ACQUIRE-load (гипотеза 1.4), но `cancel_requested` уже стало true
к моменту когда park_until крутит predicate-loop (spurious wake'и приходят от
worker'а который продолжает обрабатывать события), predicate увидит cancel и
само закроет timer.

**Возможный недостаток:** spurious wake'и в этой ситуации могут не приходить
(никто не разбудил fiber). Тогда predicate вызывается только один раз — на
первый вход в park_until. Это то же самое что Race 2b просто чуть позже. Может
не помочь.

**Поэтому Ф.A — попытка дешёвая, но без гарантий.** Если не помогает → Ф.B.

### Ф.B — Architectural: single state field (если Ф.A не помогла)

**Идея (Tokio paradigm):** Сложить `pending_stop_cb`, `pending_handle`, `stage`
в **один `AtomicU64`** per-fiber-NovaSleepState. State machine:

```c
enum {
    NOVA_SLEEP_NEW         = 0,
    NOVA_SLEEP_REGISTERED  = 1,   // timer started, stop_cb known
    NOVA_SLEEP_FIRING      = 2,   // timer fired naturally
    NOVA_SLEEP_CANCEL_REQ  = 3,   // cancel requested, stop_cb pending
    NOVA_SLEEP_CLOSING     = 4,   // uv_close in flight
    NOVA_SLEEP_CLOSED      = 5,   // close_cb fired
};

// Возможный encoding в один u64:
// [63:8] = optional pointer to handle (or hash)
// [7:0]  = state
typedef struct {
    nova_atomic_u64 packed_state;
    uv_timer_t timer;
    NovaFiberQueue* scope;
    int slot;
} NovaSleepState;
```

`register_pending` → `compare_exchange(NEW → REGISTERED, AcqRel/Acquire)`.
`cancel_worker_fibers` → `compare_exchange(REGISTERED → CANCEL_REQ, AcqRel/Acquire)`,
если success — proceed with `uv_close` via defer.
`_nova_sleep_timer_cb` → `compare_exchange(REGISTERED → FIRING)`, если success
— proceed.
Loser of every CAS → no-op (someone else won).

**Эффект:** ОДИН атомик на ВЕСЬ lifecycle → нет "окон" между ordering'ами.
Аналогично Tokio TimerEntry.

**Усложнение:** Нужно убрать `pending_stop_cb[]` + `pending_handle[]` массивы
из `NovaSchedState` — они становятся не нужны. Cancel дёргает stop_cb через
другой путь — может через `scope->pending_sleep_state[slot]` (один pointer
на NovaSleepState).

**Файлы:**
- `compiler-codegen/nova_rt/fibers.h` — NovaSleepState, _nova_sleep_via_libuv
- `compiler-codegen/nova_rt/nova_sched.h` — register_pending / cancel_pending API
- `compiler-codegen/nova_rt/runtime.c` — nova_runtime_cancel_worker_fibers
- `nova_tests/plan83_10_5/` — новые fixture'ы под new contract

**Cross-runtime test:** specifically test the bench-vs-iso difference — должен
быть NO different behavior между warm/cold workers после fix'а.

### Diagnostic findings (Ф.A.1)
Counter values от stress_iso_3e (99+1 fibers, sleep(10000), cancel(30ms)):
```
enter=100 parked=100 canceled_by_worker=90-97 close_cb=91-98 return=91-98

```

Pattern: **3-9 fibers stuck** every run. close_cb fires for them BUT
nova_sched_wake's CAS on `parked[slot]` either:
1. Already cleared by broadcast bare-wake (race: fiber unpark → predicate sees
   CLOSING (Race 2b's CAS) → returns false → re-park sets parked=true → close_cb's
   CAS races with re-park, can fail)
2. Or close_cb fires before nova_sched_park's SEQ_CST store completes (window
   between predicate-returns-false и SEQ_CST store).

### Tactical attempts (Ф.A.2)
1. **SEQ_CST store на cancel_requested** (replaced RELEASE): ✅ committed, correctness
   improvement. Не достаточно само по себе.
2. **SEQ_CST load на pending_stop_cb в cancel_worker_fibers** (replaced ACQUIRE):
   ✅ committed, correctness improvement. Не достаточно.
3. **NovaSleepState.cancel_scope_cached + Race 2c в predicate**: ✅ committed,
   defense-in-depth. race_2c counter=0 — никогда не fired в реальных tests'ах
   (фибер всегда уже видел Race 2b TRUE по факту).
4. **Broadcast bare-wake в cancel_worker_fibers second pass**: ❌ REVERTED —
   создаёт хуже race с close_cb (clears parked, re-park race с close_cb's CAS).
5. **uv_run drain spin в predicate**: ❌ REVERTED — reentrant uv_run
   из fiber context interferred с worker UV loop other-fiber processing
   (5/20 PASS — worse than no-spin baseline).
6. **Three-pass cancel с uv_sleep(1)**: ❌ REVERTED — same broadcast race
   issue, 1-2/15 PASS.

Final Tactical result: **8-9/15 (~55%) PASS**. НЕ production-grade.

### Conclusion
Tactical fixes на edge'ах atomics не закроют race robustly. Root cause —
структурная: разделённые pending_stop_cb / parked / stage / cancel_requested

**Architectural Ф.B нужен:** один AtomicU64 state field per NovaSleepState
encode'ает [state || handle hash]. Один CAS на каждый переход (Tokio
