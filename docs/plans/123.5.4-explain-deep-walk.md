## 1. Контекст

V5 `analyze_module → analyze_fn_for_explain` scanned only top-level
`_at_*` lets in fn body's Block. V1.1 generalized к full top-level
scan. But V1.2 nested-region cache lets `_at_<F>_n<N>` live INSIDE
nested blocks (if/while/match arms / for loops) — invisible к the
explain report.

Practical effect: `nova check --explain-cache` / V5 LSP code-lens /
`nova check --telemetry-cache` reported "0 mut caches" для fns where
V1.2 had actually inserted multiple nested caches. False negative.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**Signature change:**
- `analyze_fn_for_explain(f, recv, b)` → `(f, recv, b, registry)`.
- `collect_fn_caches(items, report)` → `(items, registry, report)`.
- `analyze_module` builds `registry = build_registry(&module_copy)`
  once, threads через collect_fn_caches.

**Helper rebuild (~190 LOC NEW + ~50 LOC refactor):**
- `explain_walk_block(b, type_fields, info)` — iterates stmts +
  trailing.
- `explain_walk_stmt(s, type_fields, info)` — descends into all
  Stmt variants carrying Expr/Block.
- `explain_walk_expr(e, type_fields, info)` — exhaustive ExprKind
  walk (If/IfLet/Match/For/ParallelFor/While/WhileLet/Loop/With/
  Forbid/Realtime/Detach/Blocking/Supervised/Block-Expr/Call с
  trailing/etc.). Closures excluded.
- `explain_classify_at_let(d, name, type_fields, info)` — priority-
  ordered classification: suffix-based fixed kind → TypeDecl-aware
  ro/mut lookup → name-suffix fallback.
- `explain_name_has_region_suffix(name)` — helper для fallback
  heuristic detecting `_r<digits>` / `_n<digits>` suffixes.
