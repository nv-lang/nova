# Plan 91.15 — std/net API Polish

STATUS: CLOSED (branch plan-91-net, 2026-06-17)

## Changes (by priority)

### P0 — Blocking removal (D172 amend, commit 2951c485)
- blocking{} expression is already a compile error: [D172-block-form-removed] (Plan 113).
- Blocking effect removed from compiler: realtime_suspend_effect() + builtin-effect registry.
- ExprKind::Blocking checker arm collapsed to plain walk_block (parser never fires this path).
- Blocking annotation stripped from: TcpListener.accept(), TcpStream.connect(),
  TcpStream.write(), TcpStream.read(), TcpReadHalf.read(), TcpWriteHalf.write(),
  UdpSocket.send_to(), UdpSocket.recv_from().
- Retained: negative_capability/blocking_*.nv (4 files) — Plan 113 migrated them to expect
  D172-block-form-removed; they pass and provide useful retraction coverage.

### P1 — Core API corrections (commit 1d01b05d, bundled with P2)
- NetError.Eof: returned when TCP peer closes connection (was Ok("") — ambiguous).
- NetError @to_str(): human-readable string for all 14 variants.
- SocketAddr.ip(): renamed from host_str() to match ecosystem conventions.
- TcpStream.write_all() / TcpWriteHalf.write_all(): C-backed (libuv uv_write is atomic).

### P2 — New error variants + spec (commit 1d01b05d)
- NetError.PermissionDenied: UV_EACCES -> bind on privileged port.
- NetError.ConnectionReset: UV_ECONNRESET -> peer sent RST.
- C normalization: _nova_net_uv_err() switches UV_EACCES/UV_ECONNRESET to canonical strings.
- D302: documents all changes (spec/decisions/04-effects.md).
