## 1. Контекст

V5 shipped CLI `--explain-cache` для per-fn cache decisions. V5.1
exposes same information through LSP:
- **code-lens** над method headers — "N caches inserted".
- **hover** над `@field` access — "cached as `_at_field`".

Both use `field_cache::analyze_module` API (V5 infrastructure).

---

## 2. Design

### 2.1 LSP capabilities

Add к InitializeResult.capabilities:
- `code_lens_provider: Some(CodeLensOptions { resolve_provider: false })`.
- `hover_provider: Some(HoverProviderCapability::Simple(true))`.

### 2.2 code_lens handler

For each `FnCacheInfo` в report:
- Compute lens range from `info.span`.
- Title: "N cache(s): ro=X mut=Y licm=Z pure=W chain=V".
- Command: `nova-lsp.fieldCache.show` (no-op стуб; client can extend).

### 2.3 hover handler

- Find `@<name>` token at hover position.
- Look up в report: find fn whose span covers position и whose
  caches contain `name`.
- Compose markdown с cache origin info.
