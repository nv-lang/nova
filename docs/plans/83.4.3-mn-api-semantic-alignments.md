## 1. Bugs (4 таргетных фикса, объединены в один architectural review-unit)

### B1 — `fiber_arena_stats` main vs worker aggregation

**Симптом** (`fiber_arena_stats`):
```
FAIL: spawn увеличивает slots_active под arena — now_active > before_active
```

**Причина:** API `fibers.slots_active()` возвращает stats per-thread
арены **текущего потока**. Main thread query — main's arena. spawn
под M:N идёт на worker → worker's arena. Main видит 0.

### B3 — `parallel_for` ordering tests

**Симптом** (`parallel_for`): 9 из 14 sub-тестов падают на assert'ах
порядка элементов в encoded log.

**Причина:** тесты записывают порядок исполнения в массив-log и
assert'ят specific order. Под M:N work-stealing порядок исполнения
fiber'ов **недетерминирован by design**.

### B4 — `main_yield` семантика под armed runtime

**Симптом** (`main_yield`): `runtime.yield()` на main thread'е теряет
fiber когда runtime armed.

**Причина:** yield на main thread'е роутится в worker-scheduler через
текущий код; main thread сам не является worker'ом → роутинг no-op
либо неправильный.

### B5 — `cancel_semantics` propagation через worker boundary

**Симптом** (`cancel_semantics_test`): cancellation flag, выставленный
на main, может не быть прочитан worker fiber'ом в park-condition
(race window).

**Причина:** `scope->cancel_requested` flag читается non-atomically в
park-loop. Под M:N main write / worker read могут разъехаться по
ordering. Дополнительно: parked fiber не wake'ится автоматически на
cancel — нужно либо явное polling, либо wake-all-on-cancel.

## 2. Cross-runtime reference

### 2.1. Cancellation

| Runtime | Mechanism | Hierarchical? | Wake-on-cancel? |
|---|---|---|---|
| **Go** | `context.Context` + `WithCancel(parent)` | да (tree) | да — `<-ctx.Done()` channel close = atomic broadcast |
| **tokio** | `CancellationToken` + `child_token()` | да (tree) | да — `.cancelled().await` resolves on cancel |
| **AbortController** (web/Node) | `AbortController.signal` + `signal.addEventListener('abort', ...)` | flat (signal-per-controller; tree via `AbortSignal.any([...])`) | да — event-based |
| **Kotlin** | `Job` cancellation cascades через `coroutineScope` | да (tree) | да — `isActive` polling + checkpoints |
| **Nova сейчас** | `scope.cancel_requested` plain bool | плоский | НЕТ — fiber polls в next yield |

**Что Nova добавляет** (паритет с tokio): atomic flag + automatic
wake-all parked fibers in scope on cancel + child-scope cancel
cascade (parent cancel → child cancel via scope-tree walk).

### 2.2. Cooperative yield

| Runtime | API | Семантика |
|---|---|---|
| **Go** | `runtime.Gosched()` | yields current goroutine; scheduler picks next runnable |
| **tokio** | `tokio::task::yield_now().await` | returns Pending once, scheduler reschedules |
| **JS** | `await new Promise(r => setImmediate(r))` или `setTimeout(r, 0)` | yields to event-loop; pending I/O + queued microtasks run |
| **Nova сейчас** | `runtime.yield()` | broken on main thread under armed M:N |

**Что Nova добавляет**: branch на main vs worker — main goes via
`uv_run(loop, UV_RUN_NOWAIT)` (один turn libuv loop'а — JS-стиль);
worker — push-back в deque + scheduler tick.

### 2.3. Runtime introspection

| Runtime | API |
|---|---|
| **Go** | `runtime.NumGoroutine()` (count), `runtime.NumCPU()`, `runtime.GOMAXPROCS(-1)`, pprof for richer info |
| **tokio** | `Handle::metrics()` → `RuntimeMetrics`: `num_workers`, `num_alive_tasks`, `queue_depth`, `steal_count`, `worker_overflow_count`, ... |
| **Node** | `process.eventLoopUtilization()`, `perf_hooks.PerformanceObserver` |
| **Nova сейчас** | `fibers.slots_active/_count/_high_water/_virtual_reserved` — все per-thread arena |

**Что Nova добавляет**: global aggregation (Plan 82 Ф.3 multi-arena
list уже даёт нужную инфраструктуру) + optional metrics
`fibers.worker_queue_depth(w)`, `fibers.steal_count()`.

## 3. Production-grade design (per-bug)

### 3.1. B5 — hierarchical CancellationToken

#### API addition

`NovaFiberQueue` (scope) уже имеет `cancel_requested` (плоский bool).
Replace with atomic + wake-all + parent-link:

```c
typedef struct NovaFiberQueue {
    /* ... existing fields ... */
    nova_atomic_bool      cancel_requested;     /* атомарный флаг */
    struct NovaFiberQueue* parent_scope;        /* parent в tree (NULL = root) */
    /* List of child scopes — для propagation. Linked-list head + lock. */
    nova_mutex_t          children_mu;
    struct NovaFiberQueue* children_head;       /* singly-linked */
    struct NovaFiberQueue* sibling_next;
} NovaFiberQueue;
```

#### Cancel-flow

```c
void nova_scope_cancel(NovaFiberQueue* scope) {
    /* Atomic publish flag. Workers/parked fibers see ACQUIRE on next check. */
    nova_abool_store(&scope->cancel_requested, true, RELEASE);

    /* Wake all parked fibers in this scope so they observe cancel quickly. */
    for (int slot = 0; slot < scope->count; slot++) {
        if (nova_sched_is_parked(scope, slot)) {
            nova_sched_wake(scope, slot);  /* wakes; predicate re-check sees cancel */
        }
    }

    /* Propagate to children (DFS). Children list under children_mu. */
    nova_mutex_lock(&scope->children_mu);
    for (NovaFiberQueue* child = scope->children_head; child; child = child->sibling_next) {
        nova_scope_cancel(child);  /* recursive — terminates because tree finite */
    }
    nova_mutex_unlock(&scope->children_mu);
}

nova_bool nova_scope_is_cancelled(NovaFiberQueue* scope) {
    return nova_abool_load(&scope->cancel_requested, ACQUIRE);
}
```

#### Park-predicate integration (uses 83.4.1)

Park-predicate include cancel check:
```c
static nova_bool _my_wait_pred(void* ctx) {
    MyCtx* c = (MyCtx*)ctx;
    if (nova_scope_is_cancelled(c->scope)) return true;  /* wake on cancel */
    return c->user_condition_met();
}
```

After park returns, caller re-checks cancel:
```c
nova_sched_park_until(scope, slot, _my_wait_pred, &ctx);
if (nova_scope_is_cancelled(scope)) {
    /* propagate cancel: throw Cancelled, etc. */
    return;
}
```

#### Scope-tree maintenance

On `nova_scope_init(child, parent)`:
```c
void nova_scope_init_with_parent(NovaFiberQueue* child, NovaFiberQueue* parent) {
    /* existing init ... */
    child->parent_scope = parent;
    if (parent) {
        nova_mutex_lock(&parent->children_mu);
        child->sibling_next = parent->children_head;
        parent->children_head = child;
        nova_mutex_unlock(&parent->children_mu);
        /* Inherit cancel state on init (if parent already cancelled). */
        if (nova_scope_is_cancelled(parent)) {
            nova_abool_store(&child->cancel_requested, true, RELEASE);
        }
    }
}
```

On scope destroy (`nova_scope_destroy`): remove from parent's children
list under `children_mu`.

### 3.2. B4 — `runtime.yield()` main-thread branch

```c
void nova_runtime_yield(void) {
    NovaWorker* w = _nova_current_worker_or_null();
    if (w) {
        /* Worker context: re-enqueue current fiber to own deque, scheduler picks next. */
        /* existing logic */
        nova_deque_push(&w->deque, mco_current_co());
        mco_yield(mco_current_co());
    } else {
        /* Main thread: one libuv tick — pending I/O + scheduler events fire. */
        if (nova_evloop()) {
            uv_run(nova_evloop(), UV_RUN_NOWAIT);
        }
        /* Note: на main вне fiber context'а — semantically a hint;
         * doesn't suspend (нет fiber to suspend); just lets pending I/O fire.
         * Mirrors Node setImmediate semantics. */
    }
}
```

### 3.3. B1 — global arena stats aggregation

`nova_fiber_arena_stats()` (Windows: `fiber_arena_win.c`; POSIX:
`fiber_arena.c`) сейчас возвращает per-thread stats. Расширить —
global aggregation через обход `_nova_fw_arena_list` (Plan 82 Ф.3
already provides this list).

```c
/* In fiber_arena_win.c */
NovaFiberArenaStats nova_fiber_arena_stats_global(void) {
    NovaFiberArenaStats s = { 0 };
    /* List read is lock-free (Plan 82 Ф.3 — append-only). */
    for (NovaFiberArenaWin* a = _nova_fw_arena_list; a; a = a->next) {
        char* base = a->base;
        if (!base) continue;  /* retired */
        s.slots_active   += nova_aint_load(&a->slots_active, ACQUIRE);
        s.slot_count     += a->slot_count;
        s.high_water     += a->high_water;
        s.virtual_reserved += a->virtual_size;
    }
    return s;
}

/* Existing per-thread API renamed; new global as default for fibers.slots_active(). */
NovaFiberArenaStats nova_fiber_arena_stats(void) {
    return nova_fiber_arena_stats_global();
}

NovaFiberArenaStats nova_fiber_arena_stats_local(void) {
    /* old per-thread query, kept for diagnostic */
}
```

POSIX (`fiber_arena.c`): на текущем bootstrap — single arena
per-process; global == local. Имплементировать как `local()` для
паритета API.

`std/runtime/fibers.nv`: add `fibers.local_slots_active()` etc. для
тех кто хочет per-thread (rare).

### 3.4. B3 — `parallel_for` test rewrite

Test-side fix. Two patterns:
1. **Set-equality** (порядок-независимо): сортировать log + compare.
2. **`// ENV NOVA_MAXPROCS=1` directive**: для тестов которые проверяют
   именно sequential ordering.

```nova
// До:
ro log = [Int]
parallel for x in [1, 2, 3, 4, 5] {
    log.push(x * 2)
}
assert(log == [2, 4, 6, 8, 10])  // ОШИБКА: order не гарантирован

// После — set-equality:
ro log = [Int]
parallel for x in [1, 2, 3, 4, 5] {
    log.push(x * 2)
}
assert(log.sorted() == [2, 4, 6, 8, 10].sorted())

// ИЛИ под директивой если тест честно требует order:
// ENV NOVA_MAXPROCS=1
test "encoded log preserves submit order under single worker" {
    /* unchanged */
}
```

## 5. Acceptance

### Корректность

- `fiber_arena_stats`, `cancel_semantics_test`, `main_yield`,
  `parallel_for` — все PASS под explicit `runtime.init(2)`.
- 0 регрессий single-thread пути.
- 0 регрессий полного `nova test` clang.

### Новые тесты (negative + positive)

- `cancel_propagates_to_nested_supervised.nv` — позитивный: cancel
  parent → nested supervised's fibers wake quickly и видят cancel.
- `cancel_during_park.nv` — позитивный: parked fiber wake'ится
  немедленно на parent-scope cancel (не ждёт next yield).
- `fibers_global_stats.nv` — позитивный: spawn в N workers → global
  `slots_active` reflects total.

### Cross-runtime parity

- Cancellation behaves like tokio `CancellationToken`: tree
  propagation + wake-all parked + cooperative re-check.
- `runtime.yield()` semantics документирована (spec D-block в
  [83.4.4](83.4.4-flip-activation-and-83.2-closure.md)).

## 6. Риск

| Риск | Mitigation |
|---|---|
| Scope-tree links + mutex — ABI change в `NovaFiberQueue`; затрагивает все scope-callers (Codegen-генерируемые scope-inits) | Backwards-compatible: parent параметр optional (default NULL); existing scope-inits продолжают работать без cascade |
| `children_mu` mutex contention под create-heavy load | Mutex используется ТОЛЬКО на scope-init/destroy (rare); spawn-path не трогает. Не hot path. |
| Wake-all on cancel может разбудить fiber'ов которые НЕ cancelled (ошибочное pred wake) | Спецификация park-predicate из [83.4.1](83.4.1-d93-park-wake-hardening.md): predicate re-check после wake — fiber re-park'ится если cancel false и его собственное condition не выполнено |
| `nova_fiber_arena_stats_global` lock-free обход списка — race с retire | Plan 82 Ф.3 уже решил: append-only list, retire — только в shutdown под эксклюзивный момент |
