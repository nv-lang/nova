## 1. Bugs (4 мелких таргетных фикса)

Несвязанные small-surface API/semantic-вопросы, объединены в один
review-unit для эффективности.

### B1 — `fiber_arena_stats` main vs worker aggregation

**Симптом** (`fiber_arena_stats`):
```
FAIL: spawn увеличивает slots_active под arena — now_active > before_active
```

**Причина:** API `fibers.slots_active()` возвращает stats per-thread
арены **текущего потока**. Main thread query — main's arena. spawn
под M:N идёт на worker → worker's arena. Main видит 0.

**Фикс:** `nova_fiber_arena_stats()` (`fiber_arena_win.c` /
`fiber_arena.c`) расширить **global aggregation** — обход
`_nova_fw_arena_list` (Plan 82 Ф.3) с суммированием `slots_active` /
`high_water` всех арен. Per-thread query остаётся доступным под
отдельным API `fibers.local_slots_active()` если кому нужен.

**Файлы:** `compiler-codegen/nova_rt/fiber_arena_win.c::nova_fiber_arena_stats`,
`compiler-codegen/nova_rt/fiber_arena.c::nova_fiber_arena_stats`,
`std/runtime/fibers.nv` (export + doc).

### B3 — `parallel_for` ordering tests

**Симптом** (`parallel_for`): 9 из 14 sub-тестов падают на assert'ах
порядка элементов в encoded log.

**Причина:** тесты записывают порядок исполнения в массив-log и
assert'ят specific order. Под M:N work-stealing порядок исполнения
fiber'ов **недетерминирован by design**.

**Фикс — test-side:**
- Переписать ordering-assertions на **set-equality** (sorted log
  совпадает с sorted expected) — корректно как single-thread, так и
  M:N.
- Альтернатива: пометить такие тесты как «требуют `NOVA_MAXPROCS=1`»
  через test-directive (если есть подобный механизм; иначе через env
  в test-spec line).

**Файлы:** `nova_tests/concurrency/parallel_for.nv`.

### B4 — `main_yield` семантика под armed runtime

**Симптом** (`main_yield`): `runtime.yield()` на main thread'е теряет
fiber когда runtime armed.

**Причина:** yield на main роутится в worker-scheduler через текущий
код; main thread сам не является worker'ом → роутинг no-op либо
неправильный.

**Фикс:** `nova_runtime_yield` (`runtime.c`) — branch на «текущий
поток — main»: если да, делать `uv_run(loop, UV_RUN_NOWAIT)` (один
turn main loop'а — кооперативный yield в libuv-стиле); если worker —
текущая логика (deque re-enqueue).

**Файлы:** `compiler-codegen/nova_rt/runtime.c::nova_runtime_yield`.

### B5 — `cancel_semantics` propagation через worker boundary

**Симптом** (`cancel_semantics_test`): cancellation flag, выставленный
на main, может не быть прочитан worker fiber'ом в park-condition (race
window).

**Причина:** `scope->cancel_requested` flag читается non-atomically в
park-loop. Под M:N main write / worker read могут разъехаться по
ordering.

**Фикс:** `cancel_requested` → `nova_atomic_bool`; load с `ACQUIRE`,
store с `RELEASE`. Park-condition check после atomic load.

**Файлы:** `compiler-codegen/nova_rt/fibers.h` (`NovaFiberQueue`
struct definition + park-condition).

## 2. План реализации

Каждый фикс — самостоятельный mini-PR, делать в порядке (B1 → B5 → B4
→ B3, по возрастанию риска):

### Ф.1 — B1 arena stats global aggregation (~0.5 day)

1. Обход `_nova_fw_arena_list` под list-lock, суммирование.
2. POSIX-аналог в `fiber_arena.c` (single arena per-process на
   текущем bootstrap — global == local).
3. `fibers.slots_active()` doc — обновить «global aggregated».
4. Targeted: `fiber_arena_stats` под explicit `runtime.init(2)`.

### Ф.2 — B5 atomic cancel flag (~0.5 day)

1. Change type `NovaFiberQueue.cancel_requested` to `nova_atomic_bool`.
2. Все write-сайты — через `nova_abool_store(..., RELEASE)`.
3. Все read-сайты — `nova_abool_load(..., ACQUIRE)`.
4. Park-condition recheck.
5. Targeted: `cancel_semantics_test` + stress-loop.

### Ф.3 — B4 main_yield branch (~0.5 day)

1. `nova_runtime_yield` (`runtime.c`): if `current_worker == NULL`
   (main thread) → `uv_run(nova_evloop(), UV_RUN_NOWAIT)`; else
   existing.
2. Targeted: `main_yield`.

### Ф.4 — B3 parallel_for test rewrite (~1 day)

1. Переписать ordering-assertions в `parallel_for.nv` на set-equality.
2. Где set-equality семантически неприемлемо (тест на конкретный
   sequence) — пометить через test-directive `// ENV
   NOVA_MAXPROCS=1`.
3. Verify: 14/14 PASS под explicit `runtime.init(0)`.
