**Test categories** (new files в `nova_tests/plan103_8/`):

1. **Cross-cutting stress (5 файлов):**
   - 16-fiber mixed sync stress (Mutex+Atomic+Semaphore+Condvar+Barrier)
   - High-contention CAS-loop (32 fibers competing для single AtomicI64)
   - RwLock read-heavy + occasional writer (8 readers, 1 writer × 10k iter)
   - Producer-consumer bounded buffer (M:N stress, 4 producers + 4 consumers)
   - Once + OnceCell + Lazy combined (multiple init types concurrent)

2. **Memory ordering litmus (4 файла):**
   - **IRIW (Independent Reads of Independent Writes):** classic litmus;
     2 writers (X.store, Y.store) + 2 readers ((X.load, Y.load),
     (Y.load, X.load)) — SeqCst forbids `(0,1), (1,0)`; AcqRel allows.
   - **Store Buffer (SB):** 2 fibers each `X.store(1) → Y.load`;
     SeqCst forbids `(0,0)`; Relaxed/Acquire allows.
   - **Message Passing (MP):** writer `data=v; flag.store(1, Release)`;
     reader `flag.load(Acquire); read(data)` — happens-before holds.
   - **Refcount drop:** `fetch_sub_release` + `fence(Acquire)` —
     verify last-fiber can safely free.

3. **Property tests (5 файлов):**
   - CAS-loop convergence: N fibers compete, all eventually succeed.
   - Semaphore bounded: never N+1 concurrent.
   - Barrier all-or-none: parties either all proceed or all abort.
   - Condvar no-lost-wakeup: 1000 produce/consume cycles, no miss.
   - Once exactly-once: init counter == 1 under all contention levels.

4. **Backward compat migration (2 файла):**
   - Existing `nova_tests/concurrency/sync_atomic.nv` PASS без изменения
     (M14: AtomicInt deprecated alias).

## Тест-файлы breakdown

### Stress (5 файлов)

1. **`stress_mixed_sync_mn.nv`** — 16 fibers × 1000 iter:
   - 4 fibers: AtomicI64.fetch_add competing
   - 4 fibers: Mutex.with_lock critical sections
   - 4 fibers: Semaphore.with_permit (Semaphore.new(2))
   - 4 fibers: Barrier.wait per-iter
   - M:N(4 workers). Property: no deadlock, no torn state,
     final atomic counter exact, semaphore never overcommitted.

2. **`stress_cas_loop_high_contention.nv`** — 32 fibers each runs
   CAS-loop увеличить AtomicI64 на 1 × 100 раз; final = 3200, no
   spurious "lost update". Both compare_exchange и compare_exchange_weak
   variants.

3. **`stress_rwlock_read_heavy.nv`** — 8 readers continuously check
   shared `[]int` length; 1 writer extends array × 100 iter. Property:
   readers never see torn data (length always matches actual capacity);
   writer eventually progresses (no starvation).

4. **`stress_producer_consumer_bounded.nv`** — bounded buffer
   (Mutex+Condvar+Vec): 4 producers + 4 consumers × 5000 items each;
   total produced = total consumed; no item lost; ordering invariants
   соответствуют design.

5. **`stress_once_oncecell_lazy_combined.nv`** — 32 fibers concurrently
   call:
   - One `Once.call_once { init1() }`
   - One `OnceCell.get_or_init { init2() }`
   - One `Lazy.force()` (Lazy.new at module scope)
   Each init runs exactly once.

### Memory ordering litmus (4 файла)

1. **`litmus_iriw_seqcst_forbids.nv`** — IRIW with SeqCst stores; both
   readers should see consistent total order. Run 1000 iterations; verify
   inconsistent observation never happens. (На x86 SeqCst почти free;
   на ARM критичен. Тест cross-platform-aware: skip ARM-specific check
   до Linux ARM CI готов.)

2. **`litmus_iriw_acqrel_allows.nv`** — same litmus с AcqRel; inconsistent
   observation IS possible (rare on x86); verify не panic. Документирует
   что AcqRel ≠ SeqCst, ordering tradeoffs.

3. **`litmus_message_passing.nv`** — classic MP:
   - Writer fiber: `payload = 42; flag.store(true, Release)`
   - Reader fiber: `while !flag.load(Acquire) { yield(); }; assert(payload == 42)`
   Property: paired Acquire/Release establishes happens-before
   (payload read sees 42, не junk).

4. **`litmus_refcount_drop.nv`** — refcount idiom:
   - N producers init shared data + `refcount.fetch_add(1, Relaxed)`
   - Each fiber done: `if refcount.fetch_sub(1, Release) == 1 { fence(Acquire); free(data) }`
   Property: data freed exactly once, by the fiber that drove refcount
   to 0; no use-after-free.

### Property (5 файлов)

1. **`prop_cas_loop_convergence.nv`** — N=8 fibers × 1000 attempts;
   eventual convergence: all attempts eventually succeed (counter ==
   N×1000). Both strong + weak CAS variants.

2. **`prop_semaphore_bounded_concurrency.nv`** — Semaphore.new(K);
   N >> K fibers; observable max-in-flight never exceeds K (sampled
   1000 times during run).

3. **`prop_barrier_all_or_none.nv`** — Barrier.new(N); fibers may
   timeout (`wait_for(50ms)`) or succeed; verify atomicity: either all
   N fibers see "rendezvous succeeded" or all see "barrier broken".

4. **`prop_condvar_no_lost_wakeup.nv`** — producer-consumer bounded
   buffer; 1000 produce/consume cycles; consumer never misses a
   produced item.

5. **`prop_once_exactly_once_under_pressure.nv`** — 64 fibers compete
   на `Once.call_once`, `OnceCell.get_or_init`, `Lazy.force()` —
   каждый init counter == 1 across all runs.

### Backward compat migration (1 audit file)

1. **`backward_compat_audit.md`** — verify все existing
   `nova_tests/concurrency/sync_*.nv` (`sync_atomic.nv`, `sync_mutex.nv`,
   `sync_once.nv`, `sync_waitgroup.nv`) PASS без изменения после
   Plan 103.2-103.5. AtomicInt → AtomicI64 alias verified; Once.run/done
   emit deprecation warning but functional.
