## 1. Контекст

### 1.1 Баг (как обнаружен Plan 128.1 Ф.3)

Plan 108.2 (D36 enforcement) ввёл инвариант: **`mut`-binding обязателен
для любой мутации локального state'а** — присваивание, index-assign,
mut-method call. Реализация в `types/mod.rs::consume_walk_expr` Call arm
проверяет mutability **только** когда receiver — голый identifier:

```rust
// types/mod.rs ~13690
if let ExprKind::Ident(recv) = &obj.kind {
    // … if ctx.local_mut.get(&recv) == Some(false) AND method is mut-method
    //   → E_LOCAL_NOT_MUT
}
```

Для `obj.field.mut_method()` `obj` — `ExprKind::Member { obj: Ident(b),
name: "v" }`, и guard молча пропускается:

```nova
ro b = Body.new()
b.v.set_x(99.0)              // ❌ должно быть E_LOCAL_NOT_MUT, сейчас OK
assert(b.v.x == 99.0)        // … и реально мутирует ro-slot
```

До Plan 128.1 Ф.1 это было **скрытым** багом (string-based `is_ident`
в `prepare_method_recv` отвергал Member chain → hoist в temp → writes
терялись, observable correctness совпадал с intent). Plan 128.1 Ф.1
заменил `is_ident` на AST-aware `is_lvalue_receiver` — теперь
`&(b->v)` идёт в `set_x`, mutation реально landит в ro-slot. Дыра
стала **observable** и **soundness-critical**.

Plan 128.1 Ф.3 запиннил current behaviour в
`nova_tests/plan128/t_gated_M_128_1_ro_field_chain_not_mut.nv` (без
`// EXPECT_COMPILE_ERROR`) и открыл P1 marker.

### 1.2 Inversion (что должно быть)

Инвариант D33 (locality of mutation, Plan 108.2 D36 enforcement):

> Любая мутация — присваивание, index-assignment, mut-method call —
> ТРЕБУЕТ `mut` binding'а у root корня lvalue chain'а. Field
> chain (`b.v.x = ...`) и index chain (`arr[i] = ...`) — формы
> lvalue projection, проверка применяется ровно так же, как для
> голого identifier'а.

Конкретно для mut-method call:

| Receiver shape | Текущее (баг) | Должно быть |
|---|---|---|
| `b.set_x()` где `ro b` | E_LOCAL_NOT_MUT ✅ | E_LOCAL_NOT_MUT |
| `b.v.set_x()` где `ro b` | OK (mutation landит) ❌ | E_LOCAL_NOT_MUT |
| `arr[0].set_x()` где `ro arr` | OK ❌ | E_LOCAL_NOT_MUT |
| `b.parts[i].v.set_x()` где `ro b` | OK ❌ | E_LOCAL_NOT_MUT |
| `b.v.set_x()` где `mut b`, `v` mut field | OK ✅ | OK |
| `b.v.set_x()` где `mut b`, `v` ro field (D175) | TBD | E_FIELD_NOT_MUT (cross-ref Plan 108) |

Параметр-симметрия (Plan 108.1 D176):

| Receiver shape | Должно быть |
|---|---|
| `b.v.set_x()` где `b` — param без `mut` | E_PARAM_NOT_MUT |
| `b.v.set_x()` где `mut b` param | OK |

### Ф.1 — `lvalue_root_ident` helper + Call arm rewrite

**Goal:** receiver-mutability check в `consume_walk_expr` Call arm
рекурсивно walks Member/IndexAccess chain к корневому Ident,
применяет existing `param_mut`/`local_mut` enforcement.

- New helper в `types/mod.rs`:
  ```rust
  /// Walk Member/IndexAccess chain до root Ident, return its name.
  /// None для chain'ов начинающихся с rvalue (Call result, literal, …).
  fn lvalue_root_ident(obj: &Expr) -> Option<&str> {
      match &obj.kind {
          ExprKind::Ident(name) => Some(name.as_str()),
          ExprKind::Member { obj, .. } => lvalue_root_ident(obj),
          ExprKind::IndexAccess { obj, .. } => lvalue_root_ident(obj),
          _ => None,
      }
  }
  ```
- Refactor existing Call arm (~13690-13800) — extract Ident-only path
  в `if let Some(recv) = lvalue_root_ident(obj)` wrapper. Inside:
  - `ctx.use_var(recv, …)` — touch root binding regardless.
  - `ctx.is_view_param(recv) && ctx.is_consume_method(recv, method)`
    — D157 view-via-chain check (extend coverage).
  - `param_mut[recv] == Some(false)` + registered/builtin mut-method
    → E_PARAM_NOT_MUT (diag text adjusted для chain case).
  - `local_mut[recv] == Some(false)` + … → E_LOCAL_NOT_MUT.
- Diag text amendment — when `obj.kind != Ident`, append chain hint:
  «mut-метод вызван через chain `b.v.set_x(...)` — root binding `b`
  должен быть `mut`».
- Edge cases:
  - Chain через rvalue (`make_body().v.set_x()`): `lvalue_root_ident`
    возвращает None — no enforcement, hoist path в codegen работает
    как раньше (mutation теряется, но это expected — temporary).
  - Self-receivers (`@field.set_x()` в impl): `ExprKind::SelfAccess`
    — НЕ Ident, НЕ Member chain root. Plan 108.2 уже трактует `@field`
    как mut-by-default-if-mut-self. Helper возвращает None → existing
    self-handling path сохраняется без regression.
  - Pure-read methods НЕ должны fail'иться (`b.v.x.abs()` где `abs`
    pure) — enforcement gate'ится через `registered || builtin_mut_method`,
    как раньше.

**Acceptance:**
- A128.2.1 — `ro b = ...; b.v.set_x(99.0)` → E_LOCAL_NOT_MUT.
- A128.2.2 — `fn f(b Body) { b.v.set_x(...) }` → E_PARAM_NOT_MUT.
- A128.2.3 — `mut b = ...; b.v.set_x(...)` — OK (root `mut`, field
  `mut v`).
- A128.2.4 — Index chain `ro arr; arr[0].set_x(...)` → E_LOCAL_NOT_MUT.

## 8. Lessons (предварительные)

1. **Enforcement vs codegen ABI — independent axes.** Plan 128.1 Ф.1
   correctly расширил codegen `is_ident` → `is_lvalue_receiver`. Этим
   он inadvertently сделал latent mutability-bug observable. Lesson:
   при расширении lvalue-acceptance в codegen — **обязателен**
   симметричный pass через type-checker enforcement audit (cross-ref
   D33/D36/D175/D176 invariants).

2. **«Pin current behaviour» fixture pattern — production-grade
   technical debt tracker.** Gated fixture
   `t_gated_M_128_1_ro_field_chain_not_mut.nv` без `EXPECT_COMPILE_ERROR`
   служит «honest» tombstone'ом для known bug: CI green, но reader
   видит FIXME header + marker ref. После fix'а — promote в
   `EXPECT_COMPILE_ERROR` (или удалить как новый POS-NEG split).
   Pattern из Plan 128.1 worth doc'ing в simplifications.md.

3. **Chain-walking helpers — reusable infrastructure.**
   `lvalue_root_ident` symmetric с `is_lvalue_receiver` (codegen) и
   `compute_array_elem_type_for_obj` (Plan 56 stdlib unlock). Все
   три — paths-through-chain helpers. Worth extracting в общий
   `expr_chain.rs` module если в Plan 128.x появится 4-й use-case.
