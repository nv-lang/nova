## 1. Зачем зонтик

Переработка типовой системы Nova породила несколько связанных верхнеуровневых планов. Зонтик
собирает их в один отслеживаемый набор и фиксирует **порядок** работ. Корень — у Nova **нет
единого типового движка**: аудит 2026-06-19 (3 параллельных агента) нашёл **четыре** независимых
движка типов над разными представлениями, с дублированием и дрейфом (детальная карта — в
[175.1 §8](175.1-unified-type-engine.md)).

## 2. Под-планы

| # | План | Суть | Статус |
|---|---|---|---|
| **175.1** | [Unified type engine](175.1-unified-type-engine.md) | Свести 4 движка к **одному** семантическому проходу → типизированный IR → codegen лоуэрит → C не ловит типы. Фазы U.1-U.7. Ядро рефактора. | 📋 proposed |
| **175.2** | [Method-arg type-checking](175.2-method-arg-type-checking.md) | Типизация аргументов методов + `[E_IMPLICIT_NARROWING]` на scalar-narrowing через method-arg. **Первый конкретный шаг** 175.1 (узкий кейс U.1-U.3). | 📋 proposed |
| **175.3** | [Type-set bounds](175.3-type-set-bounds.md) | Go-style generic-constraints (`fn[T IntNumber] …`) — набор конкретных типов как bound, не только протокол. Type-system фича, которую единый движок должен нести. | 📋 proposed |

**Не входят** (остаются top-level, другая тема): Plan 171 (primitive parse API — stdlib-поверхность,
зависит от 175.3), Plan 170 (file-private visibility — видимость).

## 3. Порядок работ

```
175.1 (unified engine)
   U.1 единое знание stdlib  ─┐
   U.2 один реестр           ─┼─→ разблокируют 175.2 (method-arg) и убирают дрейф
   U.3 резолв в чекере       ─┘
   U.4..U.7 консолидация
175.3 (type-set bounds) — может идти параллельно как фича; единый движок (175.1) её упрощает
```

- **175.1 U.1-U.3** — фундамент: единое знание stdlib + один реестр + резолв в чекере. Они же
  разблокируют полноценный **175.2** (резолв методов в чекере для method-arg типизации).
- **175.2** — первый измеримый результат на узком кейсе (scalar-narrowing в method-arg);
  частично уже сделано вне методов (`E_IMPLICIT_NARROWING`, commit `f96016e6`).
- **175.3** — независимая фича; после landing единого движка (175.1) выражается чище (схлопывает
  ~13 per-type обёрток Plan 171 в ~2-3 generic).

## 4. Критерий «готово» (зонтик)

Зонтик закрыт, когда выполнены критерии приёмки **175.1** (единый источник истины, codegen не
резолвит/не выводит типы заново, C не ловит ошибок типов, никакого хардкода stdlib) и закрыты
**175.2**/**175.3**.

## 5. Принципы выполнения

Per [`compiler-conventions.md`](../compiler-conventions.md): этапами (не «большим взрывом»), каждый —
с регрессом против чистого бинаря (§6), blast-radius до правки (§6), без молчаливой ломки std.
