## 1. Проблема

`nova_tests/concurrency/` — 149 тестов. **Только 1 negative test**:
`polymorphic_recursion_compile_error_test.nv` — generics, not runtime.

**Нет EXPECT_PANIC / EXPECT_RUNTIME_ERROR / EXPECT_COMPILE_ERROR** для:
- Double-resume (Plan 83.4.5.7 атомик state machine защищает, но нет
  test exercising defence).
- Cancel race на shutdown.
- Supervised drain panic propagation.
- GC root leak detection.
- Detach-after-shutdown.
- FFI blocking без `Blocking` effect.
- Spawn в `realtime` capability (D64).
- Recursive supervised internal себя.

**Production-grade quality assurance невозможна без negative-tests** —
неизвестно что invariants реально enforce'ены либо silent corruption.

### 2.1 Compile-time enforcement (EXPECT_COMPILE_ERROR)

Тесты verify что type checker / capability checker ловит запрещённое
**до runtime**:

**`blocking_without_effect_compile_fail.nv`**:

### 2.3 Panic-safety positive tests

Verify runtime survives panic в spawn body, propagates correctly:

**`spawn_body_panic_propagates_to_supervised.nv`**:
```nv
test "panic в spawn body доходит до supervised" {
    use std::throw
    try {
        supervised {
            spawn {
                throw "user-error"  // panic propagates to supervised
            }
            spawn {
                // sibling continues until cancel'нут
            }
        }
        assert(false)  // unreachable — supervised must re-throw
    } catch e {
        assert(e == "user-error")
    }
}
```

**`spawn_body_panic_does_not_crash_runtime.nv`**:
```nv
test "panic в одном spawn — runtime survives, другие fibers продолжают" {
    use std::throw
    let mut completed = 0
    try {
        supervised {
            spawn { throw "fail" }
            spawn {
                completed += 1
                // Other fiber должен получить CANCEL и завершиться gracefully.
            }
        }
    } catch _e {
        // expected
    }
    // Runtime still functional after.
    supervised {
        spawn { completed += 100 }
    }
    assert(completed == 101)
}
```

**`panic_during_drain_recoverable.nv`** — panic в одной fiber не
блокирует drain других.

**`panic_in_handler_method.nv`** — verify panic в effect handler
method routed правильно к interrupt frame.

**`cancel_race_no_orphan_state.nv`**:
```nv
test "cancel during park — no orphan parked state" {
    use std::channels::*
    let (tx, rx) = chan[int].make()
    use std::runtime::cancel

    let tok = cancel.token()
    supervised(cancel: tok) {
        spawn {
            let _v = rx.recv()  // parks
        }
        // Race: cancel mid-park.
        tok.cancel()
    }
    // Verify scope properly cleaned (no orphan park-flag, no leaked sched_state).
}
```

**`gc_root_leak_under_spawn_pressure.nv`** (depends on Plan 32):
```nv
test "10K spawn → collect → heap returns to baseline" {
    use std::runtime::gc
    let baseline = gc.heap_size()
    let mut i = 0
    while i < 10_000 {
        supervised {
            spawn { }
        }
        i = i + 1
    }
    gc.collect()
    // Allow small overhead, but no per-spawn leak.
    assert(gc.heap_size() - baseline < 10 * 1024 * 1024)
}
```

**`handler_escape_via_spawn_isolated.nv`** — verify handler scoping
correct под M:N (Plan 83.4.5.4):
```nv
test "outer handler not visible in child spawn after inner override" {
    // Plan 83.4.5.4 capture-time snapshot validation.
    with TimeProvider = h_outer {
        supervised {
            spawn {
                with TimeProvider = h_inner {
                    // child fiber sees h_inner
                    let t = Time.now()
                    assert(t == h_inner_value)
                }
            }
        }
        // Main flow sees h_outer
        let t = Time.now()
        assert(t == h_outer_value)
    }
}
```
