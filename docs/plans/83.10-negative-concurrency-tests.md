## ⚠ Discovered runtime bug: [M-83.10-armed-user-throw-routing]

**Под armed default-on M:N, USER throw из worker fiber НЕ surfaces
к main thread's `with Fail` handler.** Discovered via 5 panic-safety
positive tests — все требовали `// ENV NOVA_AUTOARM=0` для PASS.

**Repro pattern:**
```nv
let _ = with Fail[int] = |e| { caught = true; 0 } {
    supervised {
        spawn { throw 42 }   // worker thread под armed M:N
    }
    99
}
assert(caught)               // FAILS под armed
```

**Hypothesis:** Plan 49 Ф.5 `first_error_atomic` CAS captures error
на worker thread, но cross-worker routing к main thread's handler
chain не fires. Existing `cancel_semantics_test.nv` (Plan 49 Ф.4
positive coverage) сам использует `NOVA_AUTOARM=0` ровно по этой
причине — было known, just not surfaced как explicit gap.

**Impact:** production armed M:N user code who relies on `with Fail`
handlers to catch fiber errors — handlers won't fire. Currently
fiber errors abort process с "unhandled Fail" stderr — likely
not user-intended.

**V2 followup:** investigate Plan 49 Ф.5 routing chain through
worker fiber → first_error_atomic → main supervised_run_impl exit →
re-throw chain. Specific test:
- Where does USER kind error stay в worker thread?
- Does main supervised_run_impl pickup atomic_err?
- Где losing path?

### 2.3 Panic-safety positive tests

Verify runtime survives panic в spawn body, propagates correctly:

**`spawn_body_panic_propagates_to_supervised.nv`**:
```nv
test "panic в spawn body доходит до supervised" {
    use std::throw
    try {
        supervised {
            spawn {
                throw "user-error"  // panic propagates to supervised
            }
            spawn {
                // sibling continues until cancel'нут
            }
        }
        assert(false)  // unreachable — supervised must re-throw
    } catch e {
        assert(e == "user-error")
    }
}
```

**`spawn_body_panic_does_not_crash_runtime.nv`**:
```nv
test "panic в одном spawn — runtime survives, другие fibers продолжают" {
    use std::throw
    let mut completed = 0
    try {
        supervised {
            spawn { throw "fail" }
            spawn {
                completed += 1
                // Other fiber должен получить CANCEL и завершиться gracefully.
            }
        }
    } catch _e {
        // expected
    }
    // Runtime still functional after.
    supervised {
        spawn { completed += 100 }
    }
    assert(completed == 101)
}
```

**`panic_during_drain_recoverable.nv`** — panic в одной fiber не

**`panic_in_handler_method.nv`** — verify panic в effect handler

**`gc_root_leak_under_spawn_pressure.nv`** (depends on Plan 32):
```nv
test "10K spawn → collect → heap returns to baseline" {
    use std::runtime::gc
    let baseline = gc.heap_size()
    let mut i = 0
    while i < 10_000 {
        supervised {
            spawn { }
        }
        i = i + 1
    }
    gc.collect()
    // Allow small overhead, but no per-spawn leak.
    assert(gc.heap_size() - baseline < 10 * 1024 * 1024)
}
```
