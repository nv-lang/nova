## Идея

Дефолтное сравнение — byte-lexicographic (быстрое, детерминированное, locale-
независимое, как Rust/Go). Locale-aware collation — отдельный явный opt-in слой; `str`
никогда не делает collation молча.

## D254. Модель сравнения

### 152.5a — Ядро (Фаза A)
- **`@compare`/`Ord`/`Equal`/`Hash`** на `str` — byte-lexicographic (есть `@compare`,
  D178; закрепить как дефолт `Ord`). Детерминированная сортировка.
- **`@eq_ignore_ascii_case(other) -> bool`** — ASCII case-insensitive eq (без таблиц).

### 152.5b — Unicode/locale (Фаза B, зависит 152.4)
- **`@eq_ignore_case(other) -> bool`** — Unicode case folding (`std/unicode.fold_case`).
- **Locale collation:** `std/unicode/collate.Collator` (UCA / DUCET ordering +
  опц. CLDR-tailoring), аналог JS `localeCompare` / Java `Collator`.

## Q-string-collation — ЗАКРЫТО

Дефолт = byte-`Ord` (быстрый, детерминированный, locale-независимый). Locale-aware —
явный `std/unicode/collate.Collator` на UCA (DUCET) + опц. CLDR-tailoring (Phase B).
`str` никогда не делает collation молча. Прецедент: Rust (byte Ord + crate), Go (byte
+ `x/text/collate`).

## Тесты

- **POS (A):** byte-`Ord` сортировка детерминирована и стабильна; `["b","A","a"]`
  сортируется byte-lex; `eq_ignore_ascii_case("Foo","foo")==true`.
- **POS (B):** `eq_ignore_case("STRASSE","straße")==true` (folding);
  `Collator(locale).compare(...)` упорядочивает по UCA (если в scope).
- **NEG:** `eq_ignore_ascii_case("é","É")==false` (не-ASCII вне ASCII-folding);
  collation без locale-данных → fallback на byte-`Ord` + диагностика.

## Критерии приёмки

- **A1.** `str` реализует `Compare`/`Equal`/`Hash` byte-lexicographic; сортировка
  детерминирована, locale-независима.
- **A2.** `eq_ignore_ascii_case` корректен (ASCII-only, без таблиц).
- **A3 (B).** `eq_ignore_case` через Unicode folding совпадает с эталоном.
- **A4 (B).** `Collator` (UCA) упорядочивает по DUCET; tailoring опционален; `str`
  не делает collation неявно.
- **A5.** Полный `nova test` без новых FAIL; plan152_5-фикстуры PASS.
