## Идея

`char` (u32 codepoint) получает прод-API, как `char` в Rust / `Character` в Java /
`unicode` в Go. Сейчас классификация разбросана приватно (`is_alpha`/`is_digit` в
[json.nv](../../std/encoding/json.nv), [url.nv](../../std/_experimental/encoding/url.nv));
консолидируем на `char`-методы. **Двухуровнево:** ASCII-fast в ядре (без таблиц) +
Unicode-aware (делегат в `std/unicode`).

## D252. `char`-тип API

### 152.3a — ASCII-core (Фаза A, без Unicode-таблиц)
`@is_ascii`, `@is_ascii_digit`, `@is_ascii_alphabetic`, `@is_ascii_alphanumeric`,
`@is_ascii_whitespace`, `@is_ascii_uppercase`/`@is_ascii_lowercase`,
`@to_ascii_uppercase`/`@to_ascii_lowercase`, `@is_ascii_hexdigit`,
`@to_digit(radix) -> Option[int]`, **`@len_utf8() -> int`** (байт-длина кодировки
codepoint'а, 1–4), **`@encode_utf8() -> str`** (= `str.from(c)`, для симметрии).

### 152.3b — Unicode-aware (Фаза B, делегат 152.4)
`@is_alphabetic`, `@is_numeric`, `@is_alphanumeric`, `@is_whitespace`,
`@is_uppercase`/`@is_lowercase`, `@is_control`,
`@general_category() -> GeneralCategory`,
`@to_uppercase() -> CharsView`/`@to_lowercase()` (могут давать НЕСКОЛЬКО
codepoint'ов — ß→ss; тип возврата — последовательность, не один `char`).

## Фазы

- **Ф.0 — Spec D252** (ASCII-часть полностью; Unicode-часть — сигнатуры + делегаты).
- **Ф.1 — ASCII-core** в `std/runtime/char.nv` (или `runtime.char`-расширение).
- **Ф.2 — Консолидация ad-hoc** хелперов (`json.nv`/`url.nv` → `char`-методы).
- **Ф.3 — Unicode-делегаты** (152.3b, после 152.4): `is_alphabetic`/`general_category`/
  `to_uppercase`(multi-cp) через `std/unicode`.
- **Ф.4 — Фикстуры + close.**

## Тесты

- **POS (A):** `'A'.is_ascii_alphabetic()`, `'7'.to_digit(10)==Some(7)`,
  `'f'.to_digit(16)==Some(15)`, `'Z'.to_ascii_lowercase()=='z'`,
  `' '.is_ascii_whitespace()`.
- **POS (B):** `'Ω'.is_alphabetic()`, `'٣'.is_numeric()`, `'ß'.to_uppercase()`→«SS»
  (multi-cp), `'A'.general_category()==Lu`.
- **NEG:** `'g'.to_digit(16)==None`, `'\n'.is_ascii_alphanumeric()==false`,
  `'7'.to_digit(2)==None` (radix), `'@'.is_ascii_alphabetic()==false`.

## Критерии приёмки

- **A1.** ASCII-API полон и совпадает с Rust `char::is_ascii_*`/`to_digit` по
  семантике.
- **A2.** `to_digit(radix)` корректен для 2..36, None вне диапазона/не-цифра.
- **A3.** ad-hoc `is_alpha`/`is_digit` из `json.nv`/`url.nv` мигрированы на
  `char`-методы (ноль приватных дублей — grep-аудит).
- **A4 (B).** Unicode-классификация совпадает с Unicode-эталоном (UCD); `to_uppercase`
  multi-codepoint корректен (ß→ss, ﬁ→FI и т.п.).
- **A5.** Полный `nova test` без новых FAIL; plan152_3-фикстуры PASS.

## Связь

D252; зависит от 152.4 (`std/unicode`) для 152.3b. Phase-split: A = ASCII, B = Unicode.

## Прогресс ✅ 152.3a (ASCII) ЗАКРЫТ 2026-06-13 (ветка plan-152)

- **Ф.1** — ASCII-core API в `std/runtime/defaults.nv` (рядом с `char @compare`):
  `is_ascii`/`is_ascii_digit`/`is_ascii_alphabetic`/`is_ascii_alphanumeric`/
  `is_ascii_whitespace`/`is_ascii_uppercase`/`is_ascii_lowercase`/`is_ascii_hexdigit`;
  `to_ascii_uppercase`/`to_ascii_lowercase`; `to_digit(radix)->Option[int]`; `len_utf8`;
  `encode_utf8`. Через char-сравнения + `char.try_from(@ as int ±32)`, ноль таблиц.
- **Ф.0** — D252 ([03-syntax.md](../../spec/decisions/03-syntax.md#d252), ASCII полностью +
  Unicode-сигнатуры). **Ф.2** — json.nv `is_digit`/`is_hex_digit`/`is_num_start` мигрированы
  на char-методы (url.nv _experimental — pre-existing broken, пропущен). **Ф.4** —
  [plan152_3/char_ascii](../../nova_tests/plan152_3/char_ascii.nv) 1/0 (pos+neg, multibyte len_utf8).
- **Ключевая находка:** Nova-body методы на **builtin char** эмитятся ТОЛЬКО из disk-loaded
  prelude-модуля (`defaults.nv`), НЕ из embedded `char.nv` (тот парсится лишь для checker'а —
  type-check проходил, но codegen падал в member-access). Критерии A1/A2/A3 (ASCII) ✅.
- **Остаток:** 152.3b (Unicode-aware: is_alphabetic/general_category/to_uppercase multi-cp)
  — Phase B, gated на Plan 152.4 (std/unicode).
