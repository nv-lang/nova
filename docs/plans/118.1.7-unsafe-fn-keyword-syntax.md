# Plan 118.1.7 — `unsafe fn` как часть типа: миграция с `#unsafe` атрибута

> **Создан:** 2026-06-08.  **Статус:** 📋 PLANNED.
> **Parent:** Plan 118.1 (FFI intrinsics + cstring).
> **Опирается на:** Plan 118.1.5 (closed) · Plan 118.1.6 (closed) · Plan 118.5 (V3 landed).
> **Closes:** `[M-118.1.7-unsafe-fn-keyword]`
> **Эстимат:** ~1 dev-day.  **Model:** Sonnet 4.6.

---

## Что

Заменить синтаксис `#unsafe fn` (атрибут) на `unsafe fn` (ключевое слово —
часть типа функции). Убрать парсинг `#unsafe` как contract-атрибута на fn/external fn;
парсить `unsafe` перед `fn` в позиции declaration modifier.

### До / После

```nova
// БЫЛО (Plan 118.1.5, атрибут)
export #unsafe fn str @as_cstr_unchecked() -> CStr { ... }
export #unsafe external fn RawMem.copy(src *u8, dst mut * u8, n usize) -> ()

// СТАЛО (Plan 118.1.7, тип)
export unsafe fn str @as_cstr_unchecked() -> CStr { ... }
export external unsafe fn RawMem.copy(src *u8, dst mut * u8, n usize) -> ()
```

---

## Почему

### Согласованность с системой типов

Plan 118.5 ввёл `unsafe T` как first-class type wrapper (`TypeRef::Unsafe`).
Plan 118.1.6 ввёл `*unsafe fn(...)` как тип function pointer.
Вывод: **`unsafe fn(...)` — это тип функции** (не атрибут на декларации).

Симметрия:

| Позиция              | safe                     | unsafe                       |
|----------------------|--------------------------|------------------------------|
| Тип fn-pointer       | `* fn(x int) -> str`     | `* unsafe fn(x int) -> str`  |
| External fn (было)   | `external fn foo()`      | `#unsafe external fn foo()`  |
| External fn (станет) | `external fn foo()`      | `external unsafe fn foo()`   |
| Nova fn (было)       | `fn foo()`               | `#unsafe fn foo()`           |
| Nova fn (станет)     | `fn foo()`               | `unsafe fn foo()`            |

### Прецедент: `const fn`

В Nova `const fn` выражается через типы параметров и возврата
(`fn calc(const a int) -> const int`): `const` — часть типа, не атрибут.
Аналогично `unsafe fn` — `unsafe` маркирует тип функции.

### Устранение несоответствия Plan 118.1.5

Plan 118.1.5 решал задачу через whitelist расширение contract_attrs — tactical,
не type-consistent. После Plan 118.5 (TypeRef::Unsafe) и Plan 118.1.6
(`*unsafe fn(...)`) `#unsafe` на fn declarations стал orphan legacy syntax.

### Ф.1 — Parser: `unsafe fn` declaration syntax (~¼ dev-day)

**Цель:** `unsafe fn foo()` и `external unsafe fn foo()` парсятся в FnDecl с `unsafe_attr = true`.

- **Ф.1.1** В `parse_fn_common` (или `parse_fn`): перед `fn` keyword — проверить `KwUnsafe`.
  Если `unsafe fn` → set `is_unsafe = true`, bump `unsafe`, continue к `fn` parsing.

- **Ф.1.2** Для external fn: обработать оба порядка:
  - `external unsafe fn foo()` — `KwExternal` → peek `KwUnsafe` → bump `unsafe` → `fn`
  - (reverse order `unsafe external fn` — опционально, или только один порядок)

  **Рекомендация:** только `external unsafe fn` (unsafe описывает тип `fn`, стоит прямо перед `fn`).

- **Ф.1.3** `#unsafe` на fn/external fn → **hard error**:
  `E_UNSAFE_ATTR_DEPRECATED: use \`unsafe fn\` syntax instead of \`#unsafe fn\` (D2 amend Plan 118.1.7)`
  Это breaking change. Grace period не нужен — все 17 файлов мигрируем в Ф.3.

- **Ф.1.4** Убрать `unsafe_attr` из `parse_contract_attrs` whitelist.
  `KwUnsafe` там больше не допускается: «`#unsafe` is only valid as `unsafe fn` keyword».

- **Ф.1.5** POS parser fixture: `unsafe fn foo() -> int` parses.
- **Ф.1.6** POS parser fixture: `external unsafe fn bar() -> ()` parses.
- **Ф.1.7** NEG parser fixture: `#unsafe fn foo()` → E_UNSAFE_ATTR_DEPRECATED.
- **Ф.1.8** NEG parser fixture: `#unsafe external fn bar()` → E_UNSAFE_ATTR_DEPRECATED.

**Commit:** `feat(plan118.1.7 Ф.1): parser — unsafe fn keyword syntax, retire #unsafe attr on fn`

## D-block changes

**D2 amend (Plan 118.1.7 closeout, 04-effects.md):**
> `unsafe fn` keyword syntax replaces `#unsafe` attribute на fn declarations.
> `unsafe` — часть типа функции, симметрично `*unsafe fn(...)` (Plan 118.1.6).
> `#unsafe fn` → hard error E_UNSAFE_ATTR_DEPRECATED.
> Migration: `#unsafe fn` → `unsafe fn`; `#unsafe external fn` → `external unsafe fn`.

**D216 §9 amend (Plan 118.1.7 closeout, 02-types.md):**
> `unsafe fn foo(...)` — declares function of unsafe fn type. Call requires `unsafe {}`.
> `external unsafe fn foo(...)` — external fn of unsafe type.
> Type notation consistent: declaration `unsafe fn(...)` ≡ fn-ptr type `*unsafe fn(...)`.
