## Зачем

Текущий baseline `std/runtime/sync.nv` имеет только `WaitGroup` и
`Once` из coordination-family. **Industry gaps:**

| Gap | Use case | Workaround сейчас |
|---|---|---|
| **Semaphore** | bounded concurrency (rate-limit, pool capacity) | `Channel[unit].new(N)` token-channel — verbose, не idiomatic |
| **Barrier (CyclicBarrier)** | N-party rendezvous, reusable (epoch-based parallel algorithm) | WaitGroup похож, но не reusable; нужны N WaitGroup'ов per epoch |
| **CountDownLatch** | one-shot signal (init complete; shutdown initiated) | WaitGroup допускает `add()` после `wait()` — fragile; CountDownLatch immutable count |
| **Condvar** | `wait until predicate` pattern на shared mutable state under mutex | `AtomicBool.swap` + polling loop — burn CPU, no fiber park |

**M11 (master):** Semaphore — bounded (init permits); unbounded counting —
через `AtomicI32` напрямую (не нужен dedicated тип).
**M10 (master):** Condvar всегда tied to Mutex (Rust-style); нельзя
wait без mutex.

## Scope

### In scope

**`Semaphore`:**
- `new(permits int) -> Self` (initial permits ≥ 0)
- `acquire()` (park пока permit unavailable)
- `release()` (incr permits, wake one waiter)
- `try_acquire() -> bool`
- `try_acquire_for(timeout Duration) -> bool`
- `available_permits() -> int` (best-effort)
- `with_permit[R](body fn() -> R) -> R` helper (M15 V1 pattern)
- N-permit variants: `acquire_n(n int)` / `release_n(n int)` для batch
- **Fair FIFO** (Plan 103.3 M6 consistency).

**`Barrier`** (CyclicBarrier-style — reusable):
- `new(parties int) -> Self`
- `wait() -> int` (returns arrival index 0..parties-1; party-(parties-1)
  is "last arrival", может trigger barrier action)
- `wait_with_action(action fn() -> ())` — action выполняется one fiber'ом
  (last arrival) внутри barrier; остальные fibers wake после.
- `is_broken() -> bool` (state observability)
- `reset()` (force reset, освобождает waiters с broken=true)
- `wait_for(timeout Duration) -> Option[int]` (returns None если timeout
  → barrier broken, все waiters released с broken state)

**`CountDownLatch`** (one-shot, Java-style):
- `new(count int) -> Self` (initial count > 0)
- `count_down()` (decr count by 1; if reaches 0 → wake all waiters)
- `count_down_n(n int)` (decr by n; saturates at 0)
- `await()` (park до count = 0)
- `try_await() -> bool`
- `try_await_for(timeout Duration) -> bool`
- `current_count() -> int` (best-effort observability)
- **Не allow `add()` после `wait()`** (vs WaitGroup): immutable initial
  count, only `count_down`. This is the safer-vs-WaitGroup variant.

**`Condvar`** (tied to Mutex — M10):
- `new() -> Self`
- `wait(m mut Mutex)` (releases mutex, parks, re-acquires on wake)
- `wait_for(m mut Mutex, timeout Duration) -> WaitResult`
  - `WaitResult { TimedOut | Notified }`
- `wait_until(m mut Mutex, predicate fn() -> bool)` — predicate-loop
  helper (handles spurious wakeup automatically)
- `notify_one()` (wake one waiter)
- `notify_all()` (wake all waiters)
- **Spurious wakeup contract:** `wait()` может return without notify;
  must wrap в loop. `wait_until(m, predicate)` handles it automatically.
- **Warning hook:** если `Condvar.wait(reentrant_mutex)` called →
  `W_REENTRANT_CONDVAR_RECOMMEND` (M3 from Plan 103.3 cross-ref).

### Out of scope

- `Permit consume` guard для Semaphore — V2 sub-plan 103.9 (M16-clarified:
  Semaphore guards = applicable to consume).
- `Barrier` cyclic-action как closure-with-effects — V1 без; V2 если запрос.
- Unbounded counting Semaphore — М11 reject (use AtomicI32).
- `RwLock` writer-priority `Condvar` (Rust parking_lot) — отдельный future.
- `Phaser` (Java advanced barrier) — over-engineered, не нужен.

## API design

### `Semaphore`

```nova
module runtime.sync

/// Fiber-aware counting semaphore. Bounded permits (M11).
/// Fair FIFO waiter queue (Plan 103.3 M6 consistency).
///
/// # Use case
/// - Connection pool capacity
/// - Rate limiting (N concurrent requests)
/// - Resource pool (parallel workers, bounded)
#stable(since = "0.1")
export external fn Semaphore.new(permits int) -> Self

#stable(since = "0.1")
export external fn Semaphore mut @acquire()
#stable(since = "0.1")
export external fn Semaphore mut @release()
#stable(since = "0.1")
export external fn Semaphore mut @try_acquire() -> bool
#stable(since = "0.1")
export external fn Semaphore mut @try_acquire_for(timeout Duration) -> bool

#stable(since = "0.1")
export external fn Semaphore @available_permits() -> int

/// Batch acquire/release.
#stable(since = "0.1")
export external fn Semaphore mut @acquire_n(n int)
#stable(since = "0.1")
export external fn Semaphore mut @release_n(n int)

/// PREFERRED PATTERN (M15 V1). V2 (Plan 103.9): wraps Permit consume.
#stable(since = "0.1")
export fn Semaphore mut @with_permit[R](body fn() -> R) -> R {
    self.acquire()
    defer self.release()
    body()
}
```

### `Barrier`

```nova
/// Reusable N-party rendezvous (CyclicBarrier-style).
/// Cycles automatically: after `parties` fibers `wait()`, next round
/// starts.
///
/// # Use case
/// - Epoch-based parallel computation (all workers complete phase N,
///   then start phase N+1)
/// - Synchronized startup (all workers initialized before first task)
#stable(since = "0.1")
export external fn Barrier.new(parties int) -> Self

/// Park until all `parties` fibers reach barrier. Returns arrival index
/// (0..parties-1); last-arrival party gets index `parties-1`.
#stable(since = "0.1")
export external fn Barrier mut @wait() -> int

/// Wait + last-arrival fiber executes action before all wake.
/// Useful для cross-epoch state reset.
#stable(since = "0.1")
export external fn Barrier mut @wait_with_action(action fn() -> ()) -> int

#stable(since = "0.1")
export external fn Barrier mut @wait_for(timeout Duration) -> Option[int]

#stable(since = "0.1")
export external fn Barrier @is_broken() -> bool

/// Force-reset barrier. All current waiters wake с broken=true; future
/// waits use fresh round.
#stable(since = "0.1")
export external fn Barrier mut @reset()
```

### `CountDownLatch`

```nova
/// One-shot count-down rendezvous (Java-style). Immutable initial count.
///
/// # Vs WaitGroup
/// WaitGroup allows `add(n)` after creation — fragile если `wait()`
/// already started. CountDownLatch immutable count, безопаснее для
/// "init complete" / "shutdown initiated" signals.
#stable(since = "0.1")
export external fn CountDownLatch.new(count int) -> Self

#stable(since = "0.1")
export external fn CountDownLatch mut @count_down()
#stable(since = "0.1")
export external fn CountDownLatch mut @count_down_n(n int)

#stable(since = "0.1")
export external fn CountDownLatch @await()
#stable(since = "0.1")
export external fn CountDownLatch @try_await() -> bool
#stable(since = "0.1")
export external fn CountDownLatch @try_await_for(timeout Duration) -> bool

#stable(since = "0.1")
export external fn CountDownLatch @current_count() -> int
```

### `Condvar`

```nova
type WaitResult {
    Notified
  | TimedOut
}

/// Fiber-aware condition variable. Tied to Mutex (M10). Supports
/// spurious wakeup (must wrap predicate в loop).
///
/// # Use case
/// "Wait until shared state satisfies predicate" pattern.
///
/// # Spurious wakeup
/// `wait()` может return without `notify_*`. Always wrap в loop:
///
/// ```nova
/// m.with_lock { ||
///     while !predicate() {
///         cv.wait(m)
///     }
/// }
/// ```
///
/// Or use `wait_until` helper.
///
/// # Reentrant interaction
/// `Condvar.wait(reentrant_mutex)` releases ENTIRE recursive lock count
/// (Java pitfall avoided). `W_REENTRANT_CONDVAR_RECOMMEND` lint warns.
#stable(since = "0.1")
export external fn Condvar.new() -> Self

/// Atomically release mutex + park; re-acquire on wake.
#stable(since = "0.1")
export external fn Condvar @wait(m mut Mutex)
#stable(since = "0.1")
export external fn Condvar @wait_for(m mut Mutex, timeout Duration) -> WaitResult

/// Predicate-loop helper. Handles spurious wakeup automatically.
#stable(since = "0.1")
export fn Condvar @wait_until(m mut Mutex, predicate fn() -> bool) {
    while !predicate() {
        self.wait(m)
    }
}

#stable(since = "0.1")
export external fn Condvar mut @notify_one()
#stable(since = "0.1")
export external fn Condvar mut @notify_all()
```

## Industry comparison

| Primitive | Nova 103.4 | Go | Rust | Java | Kotlin |
|---|---|---|---|---|---|
| **Semaphore** | ✅ bounded | ⚠ через `chan struct{}` token | ✅ `tokio::sync::Semaphore` | ✅ `Semaphore` | ✅ `kotlinx.coroutines.Semaphore` |
| **Semaphore `try_acquire_for(Duration)`** | ✅ | n/a | ✅ tokio | ✅ `tryAcquire(time, unit)` | ✅ |
| **Semaphore fair** | ✅ FIFO | n/a | tokio: configurable | ✅ constructor flag | ✅ |
| **Semaphore batch acquire_n** | ✅ | n/a | ✅ tokio | ✅ `acquire(int)` | ❌ single only |
| **Barrier (CyclicBarrier)** | ✅ reusable | ❌ | ✅ `std::sync::Barrier` (reusable) | ✅ `CyclicBarrier` | ❌ (use Mutex+CondVar) |
| **Barrier `wait_for(Duration)`** | ✅ | n/a | ❌ stdlib (parking_lot only) | ✅ `await(time, unit)` | n/a |
| **CountDownLatch** | ✅ | ❌ (use sync.WaitGroup) | ❌ stdlib | ✅ `CountDownLatch` | n/a |
| **Condvar** | ✅ tied to Mutex (M10) | ❌ (use channels) | ✅ `Condvar` tied to Mutex | ✅ `Condition` (loosely tied to Lock) | ❌ (use channels) |
| **Condvar `wait_for(Duration)`** | ✅ → WaitResult | n/a | ✅ `wait_timeout` | ✅ `awaitNanos` | n/a |
| **Condvar `wait_until(predicate)` helper** | ✅ | n/a | ✅ `wait_while` | ❌ user-implements | n/a |
| **Spurious wakeup handled** | ✅ via wait_until | n/a | ✅ via wait_while | ⚠ user-responsibility | n/a |

**Nova edge:**
- Complete primitive set vs Go (Go relies entirely on channels — fine for many cases но не all);
- M10 (Condvar tied to Mutex) — type-safer than Java loose Condition;
- `wait_until(predicate)` baked-in — Java requires user loop;
- `with_permit(fn)` helper для Semaphore (M15 V1 path к V2 consume Permit);
- Reentrant + Condvar warning baked in (Java pitfall preempted).

### Ф.0 — Baseline audit (~0.1d)

### Ф.1 — Semaphore impl (~0.4d)

- `nova_rt/sync_primitives.h`: `Nova_Semaphore` struct (mu, permits,
  waiter list FIFO).
- `acquire()`: if permits > 0 → decr, return; else park as waiter.
- `release()`: incr permits; if waiter list non-empty → wake head waiter
  (она decr'ит permits после wake).
- `try_acquire_for(Duration)`: libuv timer cancellation pattern.
- `acquire_n/release_n`: batch decrement/increment.
- `with_permit[R]` Nova-body.

Acceptance: 4 positive tests (bounded concurrency observable, batch,
timeout, with_permit).

### Ф.2 — Barrier impl (~0.4d)

- `nova_rt/sync_primitives.h`: `Nova_Barrier` struct (mu, parties_total,
  arrived_count, generation, waiter list, broken flag).
- `wait()`:
  - mu.lock(); arrived++; my_index = arrived - 1
  - if arrived == parties_total → arrived=0, generation++, wake all,
    return parties_total - 1
  - else → record generation, park до generation change OR broken
  - on wake: return saved my_index; if broken → throw/return error
- `wait_with_action(action)`: last-arrival path выполняет action перед
  wake all.
- `reset()`: broken=true; wake all waiters; broken=false; reset
  counters for next generation.

Acceptance: 3 positive tests (8 fibers rendezvous, reusable across
epochs, wait_with_action).

### Ф.3 — CountDownLatch impl (~0.2d)

- Reuse Mutex+waiter pattern (similar to WaitGroup но immutable
  initial count, count_down only decreases).
- `await()`: park пока count > 0.
- `count_down()`: atomic decr; if reaches 0 → wake all.

Acceptance: 2 positive tests (one-shot signal, batch count_down).

### Ф.4 — Condvar impl (~0.6d)

- `nova_rt/sync_primitives.h`: `Nova_Condvar` struct (waiter list FIFO,
  generation counter для spurious wake detection).
- `wait(m)`:
  - assert m is locked by current fiber (debug check; runtime panic
    for misuse)
  - register self as waiter, save current generation
  - m.unlock() + park (atomic via park_with_unlock pattern)
  - on wake: m.lock() before return
- `wait_for(m, timeout)`: same + libuv timer; if timer fires first →
  return TimedOut; if notify first → return Notified.
- `notify_one()`: pop FIFO waiter, set its wake flag, signal scheduler.
- `notify_all()`: pop all waiters, set wake flags, signal all.
- **Spurious wakeup acceptance:** scheduler may wake fiber spontaneously
  (e.g., M:N rebalance); `wait_until` helper handles via predicate loop.
- **Reentrant mutex interaction:** if `m` is ReentrantMutex, save
  full lock_count, release all (count → 0); on wake, re-acquire as
  count=1 (Java-pitfall-aware design). Emit `W_REENTRANT_CONDVAR_RECOMMEND`
  if mutex type is ReentrantMutex (type-checker hook).

Acceptance: 5 positive tests (notify_one wakes one, notify_all wakes
all, wait_for timeout, wait_until predicate, spurious wake survives loop).

## Risks + mitigations

| Риск | Mitigation |
|---|---|
| Spurious wakeup на Condvar.wait — пользователь не знает, забывает loop | `wait_until(predicate)` helper baked-in; D173 (Plan 103.7) AI-guidance явно warning'ает; lint `W_CONDVAR_WAIT_WITHOUT_LOOP` (optional, V1 если простой detection — иначе V2) |
| Barrier reset() с уже parked waiters — race condition | Generation counter + atomic broken flag; test 6 (cyclic reusable) явно покрывает |
| ReentrantMutex + Condvar — Java pitfall — `wait()` releases full count | Type-checker hook + warning (документ design choice); Ф.4 implements full release explicitly |
| `Semaphore.release()` без `acquire()` — drift unbounded permits | D170 specifies: permits может exceed initial (Java behavior); если pathological — emit `W_SEMAPHORE_OVER_RELEASE` (optional) |
| `CountDownLatch.new(0)` confusion (no-await needed?) | Reject at runtime panic; D170 specifies count > 0 |
| Barrier deadlock если fiber dies before wait() | Если supervised — propagates; if detached — barrier broken через `reset()` или timeout — pattern в D173 |

## Parallel execution playbook (4 agents)

**Mode:** 4 independent agents in 4 separate worktrees + 1 final
merge-agent (sequential, отдельная сессия после всех 4).

**Why parallel:** 4 примитива независимы friction-wise — agent A
(Semaphore) ничего не знает про agent D (Condvar) и наоборот.
Risk = merge conflicts на shared files (`sync_primitives.h`,
`sync.nv`, `external_registry.rs`). Решается через **per-primitive
header file** + **explicit marker blocks** + **alphabetical ordering**.

**Recommended model для каждого sub-agent:** Sonnet 4.6, effort HIGH.
Thinking — см. таблицу ниже. Fallback на Opus 4.7 если застрянут >1
retry (план явно flag'ует Ф.2/Ф.4 как тяжёлые).

   /* AGENT-D */  // #include "sync_condvar.h"

   ```

   **Final merge agent** добавляет эти marker-блоки в `sync_primitives.h`
   до старта sub-agents (см. Ф.0.5 ниже). Каждый sub-agent затем
   ОБЯЗАН uncomment'ить ТОЛЬКО свою строку — это даёт детерминированный
   alphabetical merge без line-conflicts.

2. **`std/runtime/sync.nv`** — конец файла; каждый агент добавляет
   свой блок declarations в собственной marker-секции:

   ```nova
   // === PLAN-103.4 BARRIER (Agent B) ===
   // (declarations…)
   // === END PLAN-103.4 BARRIER ===
   ```

   Order при final merge: alphabetical (Barrier → Condvar → CountDownLatch
   → Semaphore).

3. **`compiler-codegen/src/codegen/emit_c.rs`** — minor routing entries
   (Plan 103.3 потребовал 16 строк для 3-х примитивов). Каждый агент
   добавляет свою routing-entry в `// === PLAN-103.4 ROUTING (alphabetical) ===`
   marker блоке (pre-flight его создаст). Conflicts маловероятны
   (~3-5 строк на примитив).

   **`external_registry.rs` НЕ модифицируется** — sync.nv декларации
   автозагружаются через `type_decls` infrastructure (Plan 103.5
   pattern).
