## Ф.2.6 — str API cleanup + extensions (D178) ✅ closed 2026-05-28

### Мотивация

Привести str API к единому стилю и устранить несоответствия:
- `bytes()`/`chars()` звучат как zero-copy views, но аллоцируют — переименовать в `to_bytes()`/`to_chars()`
- `as_bytes()` → `readonly []u8` уже существует (D176, Plan 108)
- `parse_int()` и `parse_int_radix()` — два метода для одного действия; объединить через default-параметр
- `readonly param_name type` синтаксис параметров не поддерживается (только `name readonly type`)
- `split` возвращает `[]str` но фактически это zero-copy views → `readonly []str`
- Отсутствует `@compare` для лексикографического сравнения

### Изменения D178

| Было | Стало | Тип |
|---|---|---|
| `@bytes() -> []u8` | `@to_bytes() -> []u8` | rename (аллоцирующая копия) |
| `@chars() -> []char` | `@to_chars() -> []char` | rename (аллоцирующая копия) |
| `@split(sep str) -> []str` | `@split(sep str) -> readonly []str` | return-type (zero-copy views) |
| `from_bytes_lossy(bytes readonly []u8)` | `from_bytes_lossy(readonly bytes []u8)` | синтаксис параметра |
| `from_bytes_unchecked(bytes readonly []u8)` | `from_bytes_unchecked(readonly bytes []u8)` | синтаксис параметра |
| `@parse_int() + @parse_int_radix(r)` | `@parse_int(radix int = 10)` | API merge, убрать `parse_int_radix` |
| — | `@compare(other str) -> int` | новый (lexicographic, как C `strcmp`) |
| `readonly []u8` vs `readonly [] readonly u8` | задокументировать эквивалентность | spec only |

`as_bytes() -> readonly []u8` — уже есть (D176).

## Ф.2.6 amendment — D132 `-> @` поправки (Plan 77/91, 2026-05-28)

### Мотивация

При реализации `StringBuilder @plus` обнаружен дефект: метод
`fn StringBuilder mut @plus(s str) -> Self => @append(s)` должен быть
ошибкой компиляции — `@append` объявлен `-> @`, поэтому `@plus`
обязан тоже использовать `-> @`, а не `-> Self`. Одновременно
старый `check_fluent_return` не принимал вызов `-> @` метода
как допустимое завершение тела другого `-> @` метода.

### Исправления

**1. `std/runtime/string_builder.nv`** — `@plus` исправлен: `-> Self` → `-> @`:
```nova
// было (BUG):
export fn StringBuilder mut @plus(s str) -> Self => @append(s)
// стало (CORRECT):
export fn StringBuilder mut @plus(s str) -> @ => @append(s)
```

**2. `compiler-codegen/src/types/mod.rs` — `check_fluent_return` рефакторинг:**

- Добавлен `fluent_methods: HashSet<(type, method)>` — множество всех
  `-> @` методов модуля.
- Добавлены `expr_always_returns_receiver` и `body_always_returns_receiver`:
  принимают `@`, вызов известного `-> @` метода на `@`, и
  `if/else` где все ветки удовлетворяют условиям.
- **Check 1 (уточнён):** `-> @` тело должно пройти
  `body_always_returns_receiver` — теперь принимает вызовы `-> @`
  методов как допустимые окончания.
- **Check 2 (новый):** `-> Self` метод, где `body_always_returns_receiver`
  = true → compile error `E_FLUENT_SELF`: "метод `X` объявлен `-> Self`,
  но все пути возвращают receiver (`@`). Используйте `-> @`."

### Тесты

**Позитивные:**
- `nova_tests/plan77/fluent_wrapper_ok.nv` — wrapper `@push() -> @ => @write()` делегирует в `-> @` метод; цепочка `b.push().push().push()` работает
- `nova_tests/plan77/fluent_wrapper_ok.nv` — `(sb + "hello" + " " + "world").into()` → `"hello world"` через `@plus -> @`

**Негативные:**
- `nova_tests/plan77/fluent_self_instead_of_at_err.nv` — `fn Bldr mut @step() -> Self => @bump()` где `@bump()` это `-> @` → E_FLUENT_SELF
