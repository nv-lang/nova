## 1. Контекст

V3 caches only `@<method>()` без arguments (e.g. `@len()`,
`@is_empty()`). V3.1 extends to args-with-literal-arguments:
`@get(0)`, `@nth(5)` — args are integer/string/bool/char literals.

Pure method call с same literal arguments produces same result.
V3.1 caches per (method, args) key.

---

## 2. Design

### 2.1 Eligibility

- Method has `Purity::Pure`.
- Method receiver type = current fn receiver type.
- All args are **literal expressions only** (V3.1 scope):
  - `ExprKind::IntLit(_)`, `FloatLit`, `StrLit`, `BoolLit`,
    `CharLit`, `UnitLit`, `NullPtrLit`.
- Same `(method_name, args_tuple)` accessed ≥ pure_threshold times.

### 2.2 Args canonicalization

Each call's args canonicalized к `Vec<String>` (literal repr).
Two calls share cache iff canonical args match.

### 2.3 Naming

Cache local = `_at_<method>_<argsig>_call`. Where `argsig` =
hash-like compact suffix of args (e.g. `_arg_0i` for IntLit(0),
`_arg_42i` for IntLit(42)).

For V3.1 simplicity: `_at_<method>_args_<hash>_call` where hash is
SipHash или simple FNV of args. Collision avoidance via numeric
suffix.

### 2.4 Composition

V3.1 extends V3. Same composition order (D218 LICM → V4 chain →
V3+V3.1 pure → D217 V1).
