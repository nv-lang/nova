## Scope

### 1. Refactor protocols.nv

**Было (91.8a part 1):**
```nova
type Equatable protocol {
    equals(other Self) -> bool                  // abstract
}

type Comparable protocol {
    use Equatable
    compare(other Self) -> int
    equals(other Self) -> bool => @compare(other) == 0     // default (override of embedded)
}
```

**Будет (91.8a.2) — canonical coercion form:**
```nova
type Equatable protocol {
    equals(other Self) -> bool {
        let cmp Comparable = @                      // explicit dependency
        cmp.compare(other) == 0
    }
}

type Comparable protocol {
    compare(other Self) -> int                      // abstract
}
```

**Rationale (canonical form = coercion-style):**
- Orthogonal protocols — каждый stand-alone, без embed-зависимости.
- `let cmp Comparable = @` **явно декларирует** cross-protocol dependency:
  читая Equatable, сразу видно «использует Comparable».
- Self-documenting: IDE/nova doc показывает dependency graph.
- **Conditional default**: T satisfies Equatable если has @equals (explicit)
  ИЛИ satisfies Comparable (default body synth via @compare). Type только
  Equatable (Vector3, Complex) пишет @equals explicit (coercion fails — no @compare).
- **Devirtualization at synthesis** (см. секцию 4) делает coercion form
  zero-cost в hot path; equivalent to `@compare(other) == 0` после оптимизации.

**Direct call form тоже валидна** (terser):
```nova
type Equatable protocol {
    equals(other Self) -> bool => @compare(other) == 0
}
```
Codegen emits **identical C code** для обеих форм (после devirtualization).
Coercion form предпочтительна для canonical protocol declarations потому
что explicitly документирует cross-protocol зависимость.

### 2. Printable.fmt с default body

```nova
type Printable protocol {
    fmt(sb StringBuilder) {
        sb.append(str.from(@))
    }
}
```

Default использует `str.from(@)` — primitive C function для primitives,
overload для user types. Implementer выбирает:
- Path 1 (efficient): пишет `@fmt(sb)` явно (no intermediate string)
- Path 2 (короче): пишет `fn str.from(MyType) -> str` overload (default fmt works)

**Note:** `str.from` тоже generic. Resolution:
- `str.from(int_val)` → primitive (Nova_int_to_str)
- `str.from(my_type)` → user-declared overload
- НЕТ blanket для arbitrary T (нет canonical default для str representation)

### 3. From[T] identity blanket

Generic identity method available для каждого типа:

```nova
/// Blanket identity: каждый тип T можно конвертировать из себя.
/// Аналог Rust `impl<T> From<T> for T`. Override запрещён — identity
/// is identity (D9 single canonical path).
#stable(since = "0.1")
export fn[T] T.from(t T) -> T => t
```

Use case'ы:
- `let m = Money.from(other_money)` — identity, работает для любого T
- Generic `fn copy[T](x T) -> T => T.from(x)` — works for any T
- Identity Into auto-derived из identity From (D77)

Strict override rule: попытка явно объявить `fn Money.from(m Money) -> Money`
даёт E_BLANKET_IDENTITY_OVERRIDE.

#### Resolution order для `T.from(value)`

При call site `TargetType.from(arg)`:

1. **Explicit user declaration** — `fn TargetType.from(arg_type) -> TargetType`
   объявленный в module → win, blanket suppressed
2. **Blanket identity** — `fn[T] T.from(t T) -> T` match'ит только если
   `arg_type == TargetType` (identity case) → synth + use
3. **D77 auto-derive из From[arg_type]** — если `TargetType From[arg_type]`
   satisfied через chain (existing D77 infrastructure: `from_targets`/
   `into_targets` в `emit_c.rs:401-409`) → synth derived call
4. **Error** — `[E_NO_FROM_IMPL] type TargetType cannot be constructed
   from arg_type`

**Coexistence с existing infrastructure:**

- `std/prelude/protocols.nv:81-83` — `From[T]` protocol declaration (abstract)
  не конфликтует с blanket — это разные syntactic constructs:
  - Protocol decl = abstract contract for `[T From[U]]` bounds
  - Blanket = concrete free function with generic on receiver-type
- `emit_c.rs::from_targets` + `into_targets` registries (D77 4-way derive)
  — blanket additive: registers identity per concrete T при mono;
  D77 auto-derive продолжает работать для non-identity cases.

### 4. Devirtualization в synthesis (canonical coercion form → direct call)

**Key insight:** При synthesis default body для **concrete T**, все типы
known statically. Coercion-form `let cmp Protocol = @` есть syntactic
construct который devirtualizes к direct call.

**Pass: SSA-style devirtualization**

При synthesis Nova_T_method_M из protocol P default body:

1. **Substitute:** `Self → T`, `@ → self_var`. Walk body AST.
2. **Detect coercion patterns:** `let var ProtocolName = expr` где:
   - `expr` имеет statically-known concrete type (post-substitution)
   - `ProtocolName` satisfied by expr's type structurally
3. **Replace coercion с type ascription** (no fat ptr construction):
   - `let cmp Comparable = self_var` (self_var: T) → `let cmp: T_alias = self_var`
   - var теперь treated как concrete T для последующих method lookups
4. **Replace virtual calls с direct calls:**
   - `cmp.compare(other)` где cmp: T → `Nova_T_method_compare(cmp, other)`
5. **Emit:** final body identical to direct-form synthesis.

**Result:** zero runtime cost vs `=> @compare(other) == 0`. Same C output.

**Когда devirtualization НЕ применима** (true dynamic dispatch):

| Контекст | Type info | Codegen |
|---|---|---|
| Synthesis для concrete T | static | direct call (devirt) |
| `fn print(x Printable)` param call site | type varies per call | fat ptr + vtable |
| `fn print(...args []Printable)` variadic | element types vary | fat ptr + vtable per arg |
| `let x Printable = m` coercion + later x.fmt() | coercion lost original type | fat ptr + vtable |
| `Iterable[T]` element iteration где T abstract | dynamic | vtable |

Variadic Printable — **truly dynamic**, vtable необходим. Это не workaround,
это semantically correct (каждый arg может быть разным concrete type).

### 4. Codegen lazy synthesis at use-site

**Trigger points** (когда synthesis активна):

| Контекст | Что |
|---|---|
| `a == b` / `a != b` operator (Plan 91.8b) | Equatable required → synth default `equals` если type только @compare |
| `a < b` / `<=` / `>` / `>=` operator (Plan 91.8b) | Comparable required → no default for @compare, error |
| `[T Equatable]` bound | T satisfies Equatable если has @equals OR has @compare (для default) |
| `[T Comparable]` bound | T satisfies Comparable если has @compare |
| `[T Printable]` bound | T satisfies Printable если has @fmt OR has str.from(T) (для default) |
| `let x Equatable = m` coercion | same as bound check |
| `"${expr}"` string interpolation (Plan 91.10) | Printable required for expr type |
| `fn[T] T.from(t T)` blanket call site | always synth для concrete T |
| **NOT triggered:** `m.equals(other)` direct method call | bare lookup; не нашли — error |

**Synthesis mechanism (production-grade):**

1. **Cache:** per compilation unit, `HashMap<(TypeId, MethodName), SynthFnDecl>`. Synth раз — переиспользуем.
2. **Eligibility check:** для типа T и метода M из протокола P:
   - T уже declares @M → use explicit (no synth)
   - P has abstract M (no body) → T должен иметь @M, иначе satisfaction fails
   - P has default body for M:
     - Substitute Self → T in body
     - **Devirtualization pass** (см. секцию 4) — coercions become type
       ascriptions, virtual calls become direct calls
     - Check body's method calls (@compare, str.from(@), etc.) resolve для T
     - All resolve → synthesize Nova_T_method_M; bound satisfied
     - Not all resolve → satisfaction fails (clear error message)
3. **Emit:** synth function added к module's method_overloads + emit_c.rs
   emission queue. Called normally.
4. **Generic blanket (`fn[T] T.from(t T) -> T => t`):** mono per concrete T.
   Treated as существующее generic free function через Plan 101 mono pass;
   только с static-on-T receiver вместо instance.

**NOT in scope 91.8a.2:**
- Operator dispatch refactor (`==` → @equals) — Plan 91.8b
- String interpolation rewrite — Plan 91.10
- `#impl(P + Q)` annotation — Plan 91.9

## Открытые вопросы

1. **Q1:** Cycle detection в default body resolution. Например `Equatable.equals
   default = @compare(other) == 0`; gипотетический `Comparable.compare default =
   if @equals(other) { 0 } else { ... }` → инфинитный recursion. **Default:**
   detect cycle при synthesis, error `E_SYNTH_CYCLE`. Сейчас нет такого
   паттерна в stdlib, но защита нужна.

2. **Q2:** Order of synthesis при multiple defaults. Type T satisfies both
   protocol P (with default M1) and protocol Q (with default M1 same name).
   Conflict? **Default:** error «ambiguous default body source for method M1
   на type T: protocols P and Q both provide»; implementer должен явно
   написать M1.

3. **Q3:** Generic blanket `fn[T] T.from(t T)` — как mono'd для `T = T` где
   T сам generic param? Recursive instantiation? **Default:** mono per
   concrete T только; generic T в bound context — synth deferred to call
   site с concrete T.

4. **Q4:** From identity blanket overrride strict vs lax? **Default:** strict
   (E_BLANKET_IDENTITY_OVERRIDE). Identity is identity; lax allows confusion.

5. **Q5:** Synthesis для `pure` метода protocol должна сохранять pure marker?
   **Default:** да; synthesized function inherits effects/pure marker из
   protocol method declaration.

6. **Q6:** Canonical form в protocols.nv — coercion (`let cmp Protocol = @`)
   или direct call (`=> @method()`)? Оба генерируют identical C output
   после devirtualization. **Default:** **coercion form в stdlib protocols
   как canonical** — explicit cross-protocol dependency documentation.
   Direct form допустима в user-defined protocols (terser).

7. **Q7:** Devirtualization pass — emit warning если pattern не
   recognizable (`let cmp Protocol = some_complex_expr`)? **Default:** да,
   `W_DEVIRT_FAILED` warning + fallback к fat ptr (correct codegen,
   suboptimal performance). Implementer переписывает на simpler form.

8. **Q8:** From blanket interaction с D77 `Into`-derive. Если есть
   identity From → identity Into auto-derived. Test что `42.into()` для
   `42: int` → `42` (identity). **Default:** verify через test
   `from_blanket_into_auto_derive_test.nv`.

## Принцип реализации — production-grade

> Implementation должна быть **production-grade**. Никаких MVP shortcut'ов
> вроде «cache disabled for simplicity», «error message TBD», «handle only
> Equatable, others later». Lazy synthesis работает **uniformly** для всех
> protocol contexts (bounds, coercion, operators ready for 91.8b, etc.).
>
> **Error messages должны включать:**
> - Имя type T и method M
> - Список протоколов с default body для M
> - Why default не synth'able (e.g., «@compare missing on T»)
> - Hint: «implement `@equals` explicitly OR provide `@compare`»
>
> **Cache discipline:** synth per (TypeId, MethodName) ОДИН раз per
> compilation unit. Verify duplicate-free emission.
>
> **Edge cases (все покрыты тестами):**
> - Cycles в default body method calls → E_SYNTH_CYCLE
> - Multiple protocol defaults same method → E_SYNTH_AMBIGUOUS
> - Generic param T в default body → mono per concrete T at synthesis time
> - From blanket override → E_BLANKET_IDENTITY_OVERRIDE
> - Self в param-type — резолвится в receiver type
> - **Devirtualization fails** для complex expression в `let var Protocol =
>   complex_expr` → fallback fat ptr + W_DEVIRT_FAILED warning
>
> **Devirtualization correctness:**
> - Coercion-form default body должен emit **identical C output** что
>   direct-form (verify через snapshot test сравнения сгенерированных .c файлов)
> - Devirtualization tested на multiple protocols
> - Variadic / dynamic dispatch — vtable correctly emitted когда type unknown
>
> **Plan 101 mono pass extension** — production-quality reuse existing
> infrastructure для generic mono; don't bypass.
>
> Если упирается в архитектурную проблему — **поднять вопрос пользователю**,
> не обходить.

**Lessons learned (part 1):**
1. **Orthogonal protocols cleaner than embed.** Embed creates duplicate-method detection edge case (Comparable.equals overrides Equatable.embedded.equals). Separating позволяет default body использовать другой protocol's methods declaratively.
2. **Devirtualization key insight (user contribution):** coercion-form `let cmp Comparable = @` compiles to direct call when type статически known. Same machine code as direct form (`@compare(other) == 0`). Performance не аргумент за direct form; choose form for clarity.
3. **Self в param fix** ровно тот же pattern что для return-type fix в Plan 91.7 — mirror existing infrastructure (`current_receiver_type`), no new mechanism.
4. **Plan splitting (part 1/part 2)** — protocols refactor + small fix valid sam по себе. Part 2 substantial codegen work (multiple modules, 21 tests). Better commit history + reviewable changes vs monolithic implementation.
