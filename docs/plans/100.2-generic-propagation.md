# Plan 100.2: generic propagation — `[T consume]` bound + collection-aware

> **Создан 2026-05-23 как sub-plan [Plan 100 umbrella](100-linear-must-consume.md).**
> **Статус:** 📋 proposed, **GATED на [100.1](100.1-core-must-consume.md)**.
> **Оценка:** ~4 dev-day.
> **D-блок:** **D156**.

---

## Цель

Закрыть **silent-leak в generic-коде** для consume-типов. После 100.1
foundation generic-функции типа `fn first[T](pair (T,T)) -> T => pair.0`
**молча теряют** `pair.1` если T=consume. Это самый серьёзный hole
bootstrap'а — именно generic-helpers есть в каждой stdlib. Rust решает
через `Move` trait + ownership; Nova решает через **`[T consume]` bound**
+ collection-aware iteration.

Параллельно — закрыть collection-API с 3 mode'ами (Ред. 2):
- `for tx in vec` — view (default; vec остаётся Live).
- `for mut tx in vec` — mut-view (in-place modify; vec Live).
- `for consume tx in vec` — consume каждый (vec → Consumed).

Плюс `vec.map` / `vec.filter` / `vec.fold` через `[T consume]` bound +
view-default для closure-params; `HashMap` generic-заразность.

## Сравнение с другими языками

| Capability | Go | Rust | TS | Kotlin | Nova target |
|---|---|---|---|---|---|
| Generic linear bound | n/a | ✅ `T: Move` (default) | n/a | n/a | ✅ `[T consume]` |
| Detection «generic drops linear arg» | n/a | ✅ compile-error | n/a | n/a | ✅ |
| Backward-compat: generic без bound | n/a | n/a | n/a | n/a | ✅ **silent-ignore остаётся** (Plan 100.1 default) |
| `Vec<T>` ownership iteration | n/a | ✅ `for x in vec { x.commit() }` consume каждое | n/a | n/a | ✅ |
| Collection HOF (map/filter/fold) с T=consume | n/a | ✅ (через `FnMut`/`FnOnce` + `Move`) | n/a | n/a | ✅ через bound |
| `HashMap[K, V consume]` propagation | n/a | ✅ | n/a | n/a | ✅ generic-заразность |

Превосходит Rust на одном — **backward-compat**: generic-functions без
явного `[T consume]` bound продолжают работать как раньше для не-
consume T (silent-ignore retained). Rust требует `Move` всегда (но
default). Nova даёт **opt-in strict**.

## Дизайн-решения

### D1 — синтаксис bound: `[T consume]`

```nova
fn first[T consume](pair (T, T)) -> T => pair.0
fn box[T consume](consume x T) -> Box[T] => Box { val: x }
fn map[T consume, U consume](items []T, f fn(consume T) -> U) -> []U
```

`consume` в generic-position — bound, означающий «T должен быть
consume-типом ИЛИ non-consume-типом, и компилятор обращается строго».
Симметричный keyword reuse (D131/D133 + D9 binding).

### D2 — что меняется внутри `[T consume]` generic-body

Внутри функции с `[T consume]` bound параметр T трактуется как
**возможно consume**. Strict mode:

| Действие с T-значением | Без bound | С `[T consume]` |
|---|---|---|
| `let _ = x` (silent drop) | ✅ OK | ❌ error E (D156-strict-forget) |
| передача в non-consume fn | ⚠️ silently | ❌ error |
| destructure tuple, discard part | ⚠️ silently | ❌ error |
| `return x` | ✅ | ✅ (передача наверх) |
| передача в `consume` fn-param | ✅ | ✅ (consume) |

Compiler внутри generic-body считает T = unknown-but-possibly-consume,
все non-handle-обязательно паттерны → error.

### D3 — backward-compat и migration policy

- **Default = silent-ignore** для generic-functions без bound (Plan
  100.1 behavior preserved). Иначе сломается весь stdlib generic-код.
- **Opt-in `[T consume]`** для функций, которые хотят strict mode.
- **Migration:** stdlib generic-functions (Plan 17/26/30/52/57
  collection API) — постепенно аннотируются `[T consume]` где это
  имеет смысл. Не-аннотированные продолжают silent-ignore.

### D4 — collection ABI propagation

`type_is_consume(TypeRef)` (D6 из 100.1) уже распространяется на
generic-wrap'ы: `[]Transaction` consume через generic-args. Что нужно
доп. в 100.2:

**`for x in []Transaction { ... }`** — каждое x в arm'е считается
Live linear, обязан Consumed/Returned в arm-теле. Branch-join по
итерациям как D3 из 100.1 (loop pessimistic) — каждая итерация
независима, но **накапливаемый счёт** consume требует special
trackingа.

Pragmatic подход: пометить весь vec как Consumed после `for` (даже
если break early), а каждое x в теле проверять стандартным
`check_consume`.

**`vec.push(consume tx)`** — стандартный push через consume-param;
работает из D131 + D2 (100.1).

**`vec.map(f)`** где `f: fn(consume T) -> U` — требует `[T consume,
U consume]` bound на `map`. Закрывает HOF case.

### D5 — pattern destructure в generic-context

`fn first[T consume](pair (T, T)) -> T => pair.0` — `pair.0` access.
Внутри `[T consume]` body — `pair.1` остаётся живым после `pair.0`,
обязан consumed → error (т.к. функция не консьюмит, не передаёт
дальше). 

Чтобы legitimately drop элемент — нужен **явный consume-метод** для
T-аргумента, что невозможно (T неизвестен). Значит `first` нельзя
написать как `pair.0` — нужен другой signature:

```nova
fn first[T consume](consume a T, drop_b T) -> T => a
//                                ^^^^^^^^^^ — другой `consume` param для drop'а;
//                                            если caller не хочет drop, использует
//                                            не-strict generic.
```

Это форсирует honest API.

### D6 — HashMap / user-generic propagation

`HashMap[K, V consume]` — generic-заразность из 100.1 D6 распространяет.
HashMap consume — value-ownership обязательство. Migration: HashMap
API в std/collections должен аннотировать `[V consume]` методы.

### D7 — runtime cost = 0

Все проверки — compile-time. Runtime-представление generic'ов не
меняется. Bound `[T consume]` — type-level only, не влияет на codegen
mono'd functions.

bound`. Cross-ref D133/D72/D77.

### Ф.2 — Parser + AST для `[T consume]` (~0.5 dev-day)

- AST: `GenericParam { consume_bound: bool, ... }`.
- Parser: `[T consume]` (после имени параметра, аналогично trait-
  bound D72 `[T Iter[U]]`).
- Может комбинироваться с trait-bound: `[T consume + Clone]` —
  отложено (parse error в bootstrap; одно из двух).

### Ф.3 — `type_is_consume` для generic-param + strict-mode check (~1 dev-day)

- `type_is_consume(TypeRef::Param(name), ctx)` — true если `name` имеет
  `consume_bound`.
- Внутри generic-body с `[T consume]` — `check_consume` обращается с
  T-значениями строго (D2 actions table).
- E (D156-strict-forget) при дропе T в strict mode.

**Acceptance Ф.3:** 5 probe-фикстур в `nova_tests/plan100.2/strict/`
(forget, drop-tuple-half, pass-non-consume, branch-forget, loop-forget)
ловят expected errors.

### Ф.4 — Collection-aware iteration (~0.6 dev-day)

- `for x in []T` где `type_is_consume(T)` — x как Live linear в arm'е,
  стандартный `check_consume`.
- После `for` — весь vec considered Consumed (pragmatic).
- `vec.push(consume tx)` — standard через D131.
- HashMap / Set propagation через generic-args.

### Ф.5 — Stdlib migration (audit) (~0.5 dev-day)

- Audit `std/collections/`, `std/iter/`, `std/result/`, `std/option/` —
  identify generic-functions, которые потенциально работают с consume-
  T.
- Аннотировать `[T consume]` там, где это имеет смысл (`map`, `filter`,
  `fold`, `for_each`, `chain`). НЕ аннотировать там где silent-ignore
  достаточен (`println` debug-helpers и т.п.).
- Sub-task: список аннотированных API в `docs/plans/100.2-artifacts/stdlib-migration.md`.

**Acceptance Ф.5:** ~10–15 stdlib generic-functions аннотированы, 0
регрессий в `nova test`.

## Acceptance criteria

- [ ] `[T consume]` bound парсится; работает в комбинации с D131
      `consume`-param.
- [ ] Внутри `[T consume]` body — silent-forget T → compile-error
      E (D156-strict-forget).
- [ ] Без `[T consume]` bound — silent-ignore сохраняется (backward-
      compat для existing stdlib generics).
- [ ] `for tx in vec { ... }` где vec=[]ConsumeType — каждое tx
      обязан Consumed/Returned в arm-теле.
- [ ] `HashMap[K, V consume]` propagation работает (V — consume,
      HashMap сам consume).
- [ ] stdlib generics, отмеченные миграцией Ф.5, не дают регрессий.
- [ ] 9 pos + 6 neg фикстур PASS как expected.
- [ ] Полный `nova test` → 0 регрессий.
- [ ] Spec D156 published; cross-ref D133/D72.
- [ ] Generic silent-leak `[M-generic-consume-leak]` (если был
      заведён в Plan 100.1) → CLOSED.

## Границы

- **Trait-bound + consume combined (`[T consume + Clone]`)** —
  отложено. Bootstrap parse-error если оба; future plan.
- **`[T !consume]` (anti-bound «T не должен быть consume-type»)** —
  не вводится; нет use-case'а в bootstrap.
- **Variance** linear-typed wrappers — не входит в scope (общая
  variance system — separate plan).

## Risks

1. **Stdlib migration explosion.** Audit может выявить много мест
   нуждающихся в bound. **Митигация:** Ф.5 строго bounded — только
   `map/filter/fold/for_each/chain` обязательны; остальное deferred.
2. **Tuple destructure ambiguity** (D5). **Митигация:** force
   honest API через явные `consume` params для каждого args.
3. **Performance** mono'd `[T consume]` functions — нет cost'а,
   bound — pure type-level.
