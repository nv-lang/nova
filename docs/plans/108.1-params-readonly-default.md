# Plan 108.1 — Параметры readonly по умолчанию

> **Sub-plan Plan 108** (Plan 108 = D175 readonly field freeze + D176 readonly T modifier; закрыт 2026-05-28).
> Plan 108.1 amends D176 + закрывает 2 honest-defer markers в D176 enforcement.

## TL;DR

Параметры функций без явного `mut` — **read-only**. Хочешь менять — пиши `fn f(mut b []int)`. Согласует param mutability с consume-семантикой (`consume X` уже неявно mut) и привязывает «mutation» к явному ключу.

```nova
// Было (V1):
fn f(b []int) { b.push(1) }       // компилируется — silent mutability

// Стало (Plan 108.1):
fn f(b []int) { b.push(1) }       // ✗ E_PARAM_NOT_MUT
fn f(mut b []int) { b.push(1) }   // ✓ explicit mut
fn f(readonly b []int) {...}      // ✓ явная гарантия (как было до 108.1)
```

### Ф.3 — Type-checker enforce (~2 часа)

В `compiler-codegen/src/types/mod.rs`:

1. **Закрытие [M-108-readonly-mut-method-check]:** при вызове mut-метода на параметре, проверить наличие `mut` модификатора. Если нет — `E_PARAM_NOT_MUT` + machine-applicable suggestion («добавь `mut`»).
2. **Закрытие [M-108-readonly-coerce-on-param]:** при передаче аргумента в параметр, проверить assignability:
   - `T → T` — OK.
   - `T → readonly T` — OK (сужение).
   - `T → mut T` (param) — OK (mut позволяет, передаём fresh).
   - `readonly T → T` — `E_READONLY_COERCE`.
   - `readonly T → mut T` — `E_READONLY_COERCE`.
3. Если callee вызывает `param.field = ...` или `param[i] = ...` или `param.mut_method()` без `mut` — `E_PARAM_NOT_MUT`.

## Open questions (для Ф.1)

1. **`consume X T` + `mut`** — ❌ `consume` подразумевает mut (per user decision). Parser-level reject `consume mut` и `mut consume`.
2. **Generic параметр `[T]`** — параметры с типом `T` (generic) трактуются как readonly по умолчанию; mut допустим если T поддерживает mut-методы (через impl-bound).
3. **Closures** — параметры lambda. Аналогично: без `mut` — readonly.
