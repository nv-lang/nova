## 1. Контекст (полное знание из investigation)

### 1.1 Проблема

10 из 18 тестов с `// ENV NOVA_AUTOARM=0` directive падают/TIMEOUT под
armed M:N. Pattern: `supervised(cancel: tok) { spawn { Time.sleep(N) } }`,
затем `tok.cancel()` — TIMEOUT 64s.

Тесты:
1. `concurrency/cancel_latency_bench.nv`
2. `concurrency/supervised_cancel_stress_test.nv`
3. `concurrency/supervised_cancel_test.nv`
4. `plan65/f10_select_cancel_propagation.nv`
5. `plan65/f11a_timer_metrics.nv`
6. `plan65/f7_cancel_via_token.nv`
7. `plan83_4_5_6_stress/cancel_stress.nv`
8. `plan83_4_5_6_stress/park_wake_stress.nv`
9. `plan83_4_5_6_stress/spawn_stress_10k.nv` (частично, overhead-related)
10. `concurrency/main_yield.nv` (cooperative ordering, отдельная причина — НЕ
    закрывается этим планом, остаётся с directive)

### 1.2 Root cause (точно установлено investigation)

**Файл:** `compiler-codegen/nova_rt/fibers.h:1709-1720`

```c
static NovaStopMode _nova_sleep_stop_cb(void* handle) {
    uv_timer_t* timer = (uv_timer_t*)handle;
    NovaSleepState* st = (NovaSleepState*)timer->data;
    int32_t expected = NOVA_SLEEP_PENDING;
    if (nova_aint_cas(&st->stage, &expected, NOVA_SLEEP_CLOSING)) {
        uv_timer_stop(timer);                                      // ← UB cross-thread
        uv_close((uv_handle_t*)timer, _nova_sleep_close_cb);       // ← UB cross-thread
    }
    return NOVA_STOP_ASYNC;
}
```

**Сценарий хэнга:**

1. Fiber spawn'ится под armed M:N → запускается на worker thread W.
2. Fiber вызывает `Time.sleep(10_000)` → `_nova_sleep_via_libuv` создаёт
   `uv_timer_t timer` на `nova_current_loop()` (TLS = `w->loop` worker'а W).
3. Fiber park'ится через `nova_sched_park_until` с predicate
   `_nova_sleep_stage_is_closed`.
4. Main thread вызывает `tok.cancel()` →
   `nova_cancel_token_cancel_reason` → `nova_sched_cancel_all_pending(scope)`.
5. `cancel_all_pending` (на main thread!) walks `pending_stop_cb[]` scope'а,
   вызывает `_nova_sleep_stop_cb(timer)` напрямую.
6. `uv_timer_stop()` + `uv_close()` вызываются **с main thread на handle,
   принадлежащем w->loop'у worker'а W** — это **UB** (libuv contract:
   handle operations only from the loop's thread).
7. На практике: close_cb либо не fire'ает, либо fire'ает в неожиданный
   момент → fiber остаётся `parked[slot]=true` навсегда.
8. Main drain loop в `nova_supervised_run_impl` (fibers.h:1434) видит
   `alive > 0` (fiber жив), `parked == alive` → крутит
   `uv_run(nova_current_loop(), UV_RUN_ONCE)` на **main loop'е**, но
   close_cb queued на **worker's loop**. Хэнг до 64s timeout.

### 1.3 Existing infrastructure (готовое для переиспользования)

- **`uv_async_t` per worker** — `w->wake_handle` (runtime.c, init в worker
  preamble). Callback `_worker_async_cb` сейчас no-op (signal-only).
- **`uv_async_t` для main** — `_main_wake` (runtime.c:284-310).
  `_main_wake_cb` no-op.
- **`wake_pending` queue per worker** — `mco_coro** wake_pending`,
  `wake_pending_count`, `wake_pending_cap`, под `wake_mu` mutex.
- **`dispatch_ready`** — `_worker_dispatch_ready(ctx, co)` runtime.c:489,
  knows cross-thread vs same-thread (через `_current_worker_id` TLS).
- **`nova_scope_cancel_wake_all`** — Plan 83.4.5.1 (nova_sched.h:318)
  walks slots, calls `nova_sched_wake` для каждого parked. Уже cross-thread
  safe через dispatch_ready.

### 1.4 Reference implementation pattern

Plan 83.10 fix `[M-83.10-armed-user-throw-routing]` (commit `57d0faaabc0`)
расширил `NovaFiberQueue` атомарными полями для cross-thread данных. **Тот
же pattern применяется здесь** — атомарная queue close-jobs per loop.

## 2. Scope

**Цель:** Routing of `uv_close` (и `uv_timer_stop`) от cancel-thread к
timer-owning loop thread через `uv_async_send`-based deferred dispatch.

**В scope:**
- New per-loop "deferred close" queue + mutex.
- New `uv_async_t` callback на worker'ах и main для draining queue.
- Изменения в `_nova_sleep_stop_cb` (и аналогичных stop_cb для других
  async ops, например `_nova_after_timer_stop_cb` если он также UB).
- Также проверить `cleanup_cbs` registered через
  `nova_cancel_token_register_resource` — те же UB?
- Verify acceptance на 9 тестах (10й — main_yield — cooperative ordering).

**НЕ в scope:**
- Изменения compile-time semantics или codegen (только runtime fix).
- Architectural rewrite scheduler.
- TLS handler race ([M-83.10.1-per-fiber-handler-tls-race] — separate).
- Cross-mco longjmp ([M-83.10.1-fail-handler-cross-mco-longjmp] — separate).
- Nested supervised routing ([M-83.10-nested-armed-routing] — Plan 83.10.3).

## 3. Design

### 3.1 Data structures

**New: `NovaDeferredCloseQueue`** (eventloop.h):

```c
/* Plan 83.10.2 (2026-05-26): cross-thread uv_close dispatch.
 * libuv requires handle ops only on loop's thread. Cancel runs on
 * any thread; timer/handle may be on a worker's loop. Schedule the
 * close via this queue + uv_async signal.
 *
 * Producer: any thread (e.g. cancel) — push job + uv_async_send.
 * Consumer: loop's thread — _nova_loop_drain_closes drains and
 * invokes uv_close for each.
 *
 * Lifetime: embedded in NovaWorker (workers) or static global (main).
 * Mutex-protected for cross-thread push.
 */
typedef struct {
    uv_handle_t*      handle;
    uv_close_cb       close_cb;
} NovaDeferredCloseJob;

typedef struct {
    NovaDeferredCloseJob* jobs;
    int                   count;
    int                   cap;
    nova_mutex_t          mu;
} NovaDeferredCloseQueue;
```

**Add to `NovaWorker` struct** (runtime.c near `wake_handle`):

```c
NovaDeferredCloseQueue close_queue;  /* Plan 83.10.2 */
```

**Add global для main loop** (runtime.c near `_main_wake`):

```c
static NovaDeferredCloseQueue _main_close_queue;
static bool                   _main_close_queue_inited = false;
```

### 3.2 API

**New functions** (eventloop.h declarations, eventloop.c definitions):

```c
/* Initialize a deferred-close queue. Called once per worker preamble
 * + once for main during pool materialization. */
void nova_close_queue_init(NovaDeferredCloseQueue* q);

/* Destroy a deferred-close queue. Workers free on exit, main on
 * runtime shutdown. */
void nova_close_queue_destroy(NovaDeferredCloseQueue* q);

/* Schedule `handle` to be closed on `loop`'s thread. Thread-safe —
 * may be called from any thread. Triggers uv_async_send on the
 * loop's wake handle.
 *
 * Caller must ensure:
 *   - handle->loop == loop (asserted in debug).
 *   - close_cb is non-NULL.
 *   - handle won't be closed elsewhere (idempotency required, see below).
 *
 * Returns 0 on success, -1 on enqueue failure.
 */
int nova_loop_defer_close(uv_loop_t* loop,
                          uv_handle_t* handle,
                          uv_close_cb close_cb);

/* Drain pending close jobs on the current loop. Must be called from
 * loop's thread (typically from async cb). */
void nova_loop_drain_closes(NovaDeferredCloseQueue* q);
```

### 3.3 Loop → queue resolution

`nova_loop_defer_close(loop, handle, cb)` must find:
- Worker queue: iterate `_workers[]`, match `&w->loop == loop` → push to
  `w->close_queue`, `uv_async_send(&w->wake_handle)`.
- Main queue: `loop == nova_evloop()` → push to `_main_close_queue`,
  `uv_async_send(&_main_wake)`.

Add static helper:

```c
static NovaDeferredCloseQueue* _nova_find_close_queue_for_loop(uv_loop_t* loop) {
    if (loop == nova_evloop()) return &_main_close_queue;
    for (int i = 0; i < _n_workers; i++) {
        if (&_workers[i].loop == loop) return &_workers[i].close_queue;
    }
    return NULL;  /* unknown loop — caller's bug */
}

static uv_async_t* _nova_find_wake_for_loop(uv_loop_t* loop) {
    if (loop == nova_evloop()) return &_main_wake;
    for (int i = 0; i < _n_workers; i++) {
        if (&_workers[i].loop == loop) return &_workers[i].wake_handle;
    }
    return NULL;
}
```

### 3.5 Modify `_nova_sleep_stop_cb`

**Файл:** `compiler-codegen/nova_rt/fibers.h:1709-1720`

Before:
```c
static NovaStopMode _nova_sleep_stop_cb(void* handle) {
    uv_timer_t* timer = (uv_timer_t*)handle;
    NovaSleepState* st = (NovaSleepState*)timer->data;
    int32_t expected = NOVA_SLEEP_PENDING;
    if (nova_aint_cas(&st->stage, &expected, NOVA_SLEEP_CLOSING)) {
        uv_timer_stop(timer);
        uv_close((uv_handle_t*)timer, _nova_sleep_close_cb);
    }
    return NOVA_STOP_ASYNC;
}
```

After:
```c
static NovaStopMode _nova_sleep_stop_cb(void* handle) {
    uv_timer_t* timer = (uv_timer_t*)handle;
    NovaSleepState* st = (NovaSleepState*)timer->data;
    int32_t expected = NOVA_SLEEP_PENDING;
    if (nova_aint_cas(&st->stage, &expected, NOVA_SLEEP_CLOSING)) {
        /* Plan 83.10.2 (2026-05-26): cross-thread safe.
         * timer->loop may not be current thread's loop under armed M:N
         * (timer created on worker, cancel on main). Defer close to
         * timer's loop thread via async dispatch. uv_timer_stop is also
         * not thread-safe — perform it inside drain on loop's thread. */
        nova_loop_defer_close(timer->loop, (uv_handle_t*)timer,
                              _nova_sleep_close_cb);
    }
    return NOVA_STOP_ASYNC;
}
```

The `uv_timer_stop` is moved into the drain (see §3.6). For uv_timer_t,
`uv_close` already implicitly stops the timer — `uv_timer_stop` was
redundant. (Confirm with libuv docs: yes, uv_close stops handle.)

### 3.6 Drain implementation

`nova_loop_drain_closes` (eventloop.c):

```c
void nova_loop_drain_closes(NovaDeferredCloseQueue* q) {
    if (!q) return;
    /* Move jobs out under lock, then invoke unlocked. Producer may
     * enqueue more during invocation — next async wake handles them. */
    nova_mutex_lock(&q->mu);
    int n = q->count;
    NovaDeferredCloseJob* batch = q->jobs;
    q->jobs = NULL;
    q->count = 0;
    q->cap = 0;
    nova_mutex_unlock(&q->mu);
    for (int i = 0; i < n; i++) {
        uv_close(batch[i].handle, batch[i].close_cb);
    }
    free(batch);
}
```

`nova_loop_defer_close` (eventloop.c):

```c
int nova_loop_defer_close(uv_loop_t* loop,
                          uv_handle_t* handle,
                          uv_close_cb close_cb) {
    if (!loop || !handle || !close_cb) return -1;
    NovaDeferredCloseQueue* q = _nova_find_close_queue_for_loop(loop);
    uv_async_t* wake = _nova_find_wake_for_loop(loop);
    if (!q || !wake) return -1;
    nova_mutex_lock(&q->mu);
    if (q->count >= q->cap) {
        int new_cap = q->cap > 0 ? q->cap * 2 : 8;
        NovaDeferredCloseJob* new_jobs =
            (NovaDeferredCloseJob*)realloc(q->jobs,
                                            (size_t)new_cap * sizeof(*new_jobs));
        if (!new_jobs) {
            nova_mutex_unlock(&q->mu);
            return -1;
        }
        q->jobs = new_jobs;
        q->cap = new_cap;
    }
    q->jobs[q->count].handle = handle;
    q->jobs[q->count].close_cb = close_cb;
    q->count++;
    nova_mutex_unlock(&q->mu);
    uv_async_send(wake);
    return 0;
}
```

### 3.10 Spurious wake invariant

`nova_sched_park_until` (Plan 83.4.1, nova_sched.h:280) уже handles
spurious wake. Our fix preserves this: fiber wakes when close_cb sets
`stage = NOVA_SLEEP_CLOSED`, predicate satisfies, returns.

### 3.11 GC safety

`NovaDeferredCloseQueue` использует `realloc` (не GC). Jobs hold raw
pointers to uv handles — handles are в GC heap или embedded в state
structs. После uv_close handle становится invalid (libuv invariant).
Pending close на already-closed handle → libuv asserts; idempotency
controlled by CAS на NovaSleepState.stage (NOVA_SLEEP_PENDING →
NOVA_SLEEP_CLOSING happens once).

## 7. Risks

- **Performance:** per-cancel uv_async overhead. Acceptable for cancel
  (rare path). Bench `cancel_latency_bench` — slack 50ms p99 баунд должен
  оставаться достижим.
- **Deadlock risk:** if drain happens while another thread holds queue
  mutex — won't happen, drain takes mutex briefly to swap out batch.
- **Race on close_queue init:** worker preamble runs before any spawn;
  init complete before any cancel can route to worker. Main init pre-pool.
- **Idempotent close:** CAS in `_nova_sleep_stop_cb` ensures single
  enqueue. Subsequent stop_cb invocations skip.
