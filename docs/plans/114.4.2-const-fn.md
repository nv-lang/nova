### –§.1 вАФ Parser + body checker

#### Parser

- `parse_param` —А–∞—Б—И–Є—А–Є—В—М: `const NAME TYPE` вАФ –і–Њ–±–∞–≤–Є—В—М `const`
  qualifier –≤ param-modifier choice (–љ–∞—А—П–і—Г —Б `mut`/`ro`/`consume`).
- `parse_fn_return` —А–∞—Б—И–Є—А–Є—В—М: `-> const T` вАФ –і–Њ–±–∞–≤–Є—В—М `const` –њ—А–µ—Д–Є–Ї—Б
  –≤ return type position.
- **All-or-nothing check** –љ–∞ parse-time:
  - –Х—Б–ї–Є —Е–Њ—В—М –Њ–і–Є–љ param `const` –Ш–Ы–Ш return `const` вАФ –њ—А–Њ–≤–µ—А–Є—В—М –Т–°–Х
    params `const` –Ш return `const`.
  - Mixed вЖТ `E_CONST_FN_PARTIAL_CONSTNESS`.
- **Modifier-conflicts** –љ–∞ param: `mut const` / `ro const` /
  `consume const` вЖТ `E_CONST_PARAM_MOD_CONFLICT`.
- **Effect-list –Ј–∞–њ—А–µ—Й—С–љ**: `fn calc(const a int) Log -> const int { вА¶ }`
  вЖТ parse-time `E_CONST_FN_EFFECT_IN_SIGNATURE`.

#### Body checker

Whitelist allowed:
- –Ы–Є—В–µ—А–∞–ї—Л (Int/Float/Str/Bool/Char/Unit).
- –Р—А–Є—Д–Љ–µ—В–Є–Ї–∞/bitwise/comparison –љ–∞–і constexpr.
- `as`-casts –Љ–µ–ґ–і—Г primitive-—В–Є–њ–∞–Љ–Є.
- –°—Б—Л–ї–Ї–Є –љ–∞ const-–њ–∞—А–∞–Љ–µ—В—А—Л –Є local `const`-bindings.
- –Ы–Њ–Ї–∞–ї—М–љ—Л–µ `const c = expr` declarations.
- Final expression (–њ–Њ—Б–ї–µ–і–љ–Є–є statement вАФ expression).
- Calls –љ–∞ –і—А—Г–≥–Є–µ `const fn` —Б constexpr args.

Blacklist (V1) вАФ –Ї–∞–ґ–і—Л–є —Б–Њ —Б–≤–Њ–Є–Љ error code:
- `if`/`else`/`match` вЖТ `E_CONST_FN_CONTROL_FLOW`.
- `for`/`while`/`loop` вЖТ `E_CONST_FN_CONTROL_FLOW`.
- `mut`/`consume` bindings вЖТ `E_CONST_FN_MUT_BINDING`.
- Effects (calls –љ–∞ non-const fn, `Time.now()`, `print()`) вЖТ
  `E_CONST_FN_EFFECT_IN_BODY`.
- Allocations (`Vec.new()`, `StringBuilder`, etc.) вЖТ
  `E_CONST_FN_ALLOCATION`.
- Generic type params –≤ body –Є–ї–Є signature вЖТ `E_CONST_FN_GENERIC`.
- Recursion (direct –Є–ї–Є mutual) вЖТ `E_CONST_FN_RECURSION`.

### –§.2 вАФ Comptime evaluator subsystem (safety-hatched)

> **Safety hatch:** decision point –≤ –Ї–Њ–љ—Ж–µ –§.2.3 smoke –љ–∞ minimal
> example. –Х—Б–ї–Є evaluator —В—А–µ–±—Г–µ—В > вЕФ day –Є–ї–Є deep integration refactor
> —Б runtime interpreter вЖТ extract –≤ Plan 114.4.2.1. Plan 114.4.2 ships
> —Б –§.1 only (checker rejects const fn calls —Б
> `E_CONST_FN_EVALUATOR_NOT_IMPLEMENTED`).

#### Architecture

**Reuse + extend Plan 14 –§.2 constexpr-engine.** –°—Г—Й–µ—Б—В–≤—Г—О—Й–Є–є engine
–≤—Л—З–Є—Б–ї—П–µ—В literals + arithmetic + record literals. Plan 114.4.2 extends:
- **Fn-call evaluation** вАФ –љ–Њ–≤—Л–є case –≤ evaluator dispatch.
- **Environment-based interpreter** вАФ param env + local const env
  –Њ—В–і–µ–ї—М–љ—Л–µ scope'—Л.
- **Sequential statements** вАФ –≤—Л–њ–Њ–ї–љ–µ–љ–Є–µ –њ–Њ –Њ–і–љ–Њ–Љ—Г (V1 –±–µ–Ј branching).
- **Final expression** вАФ last expression –≤ block-body –≤–Њ–Ј–≤—А–∞—Й–∞–µ—В
  —А–µ–Ј—Г–ї—М—В–∞—В.

#### Recursion-limit V1=1

- No recursion –≤ V1 (checker rejects –љ–∞ static-time —З–µ—А–µ–Ј call-graph
  analysis).
- Mutual recursion detection: build call-graph –њ—А–Є body-check; cycle вЖТ
  `E_CONST_FN_RECURSION`.

#### Memoization

- –Ч–∞ –Њ–і–Є–љ compilation, –Ї—Н—И `(fn_id, arg_tuple_hash) вЖТ result`.
- –Ш–і–µ–љ—В–Є—З–љ—Л–µ call sites evaluate –Њ–і–Є–љ —А–∞–Ј.
- Cache invalidated per compilation (no cross-compilation cache).

#### Errors –љ–∞ evaluator-side

- `E_CONST_FN_EVAL_OVERFLOW` вАФ arithmetic overflow (signed/unsigned
  saturate vs wrap вАФ explicit compile error, –љ–µ silent).
- `E_CONST_FN_DIV_ZERO` вАФ division by zero.
- `E_CONST_FN_EVAL_PANIC` вАФ –Є–љ–Њ–є evaluator panic (defensive, –љ–µ –і–Њ–ї–ґ–љ–Њ
  –њ—А–Њ–Є—Б—Е–Њ–і–Є—В—М per body checker).

### –§.3 вАФ Call-site validation + codegen replace

#### Call-site validation

- –Э–∞ call site `f(arg1, arg2, вА¶)` –≥–і–µ `f` вАФ `const fn`:
  - –Ъ–∞–ґ–і—Л–є arg –Њ–±—П–Ј–∞–љ –±—Л—В—М constexpr-evaluable (reuse Plan 114.4 –§.1
    `check_const_constexpr`).
  - –Ш–љ–∞—З–µ `E_CONST_FN_NON_CONST_ARG` —Б pointer –љ–∞ offending arg.
- –Х—Б–ї–Є –≤—Б–µ args constexpr вЖТ invoke evaluator вЖТ replace call —Б
  result-–ї–Є—В–µ—А–∞–ї–Њ–Љ –≤ AST.

#### Codegen

- `const fn` –Э–Х emit'–Є—В—Б—П –≤ C-output (–љ–µ—В runtime fn symbol).
- –Т—Б–µ call sites replaced –ї–Є—В–µ—А–∞–ї–Њ–Љ (–њ–Њ—Б–ї–µ evaluator).
- Dead `const fn` (–±–µ–Ј call sites) вАФ silently dropped –Є–Ј output.

#### First-class reject

- `ro f = calc` –≥–і–µ `calc` вАФ `const fn` вЖТ `E_CONST_FN_FIRST_CLASS`
  (V1 reject; followup `[M-114.4.2-first-class]` —З–µ—А–µ–Ј runtime-wrapper).
