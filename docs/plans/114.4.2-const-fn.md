## Зачем

Plan 114.4 D199 (draft) декларирует `const fn` — функции вычисляемые
компилятором. Аналог Rust `const fn`, C++ `constexpr fn`, Zig
`comptime` params.

**Без них:**
- `const X = …` ограничен литералами + арифметикой над literals + record
  literals.
- Computed `[N]T` sizes требуют hand-rolled hex literals.
- Default params (`fn open(n int = ???)`) — только literals.
- Lookup tables / type-driven dispatch constants — невозможны.

**С Plan 114.4.2:**
```nova
fn calc(const a int, const b char) -> const int {
    const c = b as int
    a + c * 10
}

const RESULT = calc(5, 'A')              // → литерал 655 на call site
ro buf [calc(2, '0')]u8 = ...            // array size 482
fn open(n int = calc(3, ' ')) { ... }    // default param 323

type Config {
    const BUFFER_SIZE int = calc(64, '@')   // assoc const computed
    name str
}
```

## Дизайн

Полный дизайн — в [D199](../../spec/decisions/03-syntax.md#d199)

### Ф.1 — Parser + body checker

#### Parser

- `parse_param` расширить: `const NAME TYPE` — добавить `const`
  qualifier в param-modifier choice (наряду с `mut`/`ro`/`consume`).
- `parse_fn_return` расширить: `-> const T` — добавить `const` префикс
  в return type position.
- **All-or-nothing check** на parse-time:
  - Если хоть один param `const` ИЛИ return `const` — проверить ВСЕ
    params `const` И return `const`.
  - Mixed → `E_CONST_FN_PARTIAL_CONSTNESS`.
- **Modifier-conflicts** на param: `mut const` / `ro const` /
  `consume const` → `E_CONST_PARAM_MOD_CONFLICT`.
- **Effect-list запрещён**: `fn calc(const a int) Log -> const int { … }`
  → parse-time `E_CONST_FN_EFFECT_IN_SIGNATURE`.

#### Body checker

Whitelist allowed:
- Литералы (Int/Float/Str/Bool/Char/Unit).
- Арифметика/bitwise/comparison над constexpr.
- `as`-casts между primitive-типами.
- Ссылки на const-параметры и local `const`-bindings.
- Локальные `const c = expr` declarations.
- Final expression (последний statement — expression).
- Calls на другие `const fn` с constexpr args.

Blacklist (V1) — каждый со своим error code:
- `if`/`else`/`match` → `E_CONST_FN_CONTROL_FLOW`.
- `for`/`while`/`loop` → `E_CONST_FN_CONTROL_FLOW`.
- `mut`/`consume` bindings → `E_CONST_FN_MUT_BINDING`.
- Effects (calls на non-const fn, `Time.now()`, `print()`) →
  `E_CONST_FN_EFFECT_IN_BODY`.
- Allocations (`Vec.new()`, `StringBuilder`, etc.) →
  `E_CONST_FN_ALLOCATION`.
- Generic type params в body или signature → `E_CONST_FN_GENERIC`.
- Recursion (direct или mutual) → `E_CONST_FN_RECURSION`.

### Ф.2 — Comptime evaluator subsystem (safety-hatched)

> **Safety hatch:** decision point в конце Ф.2.3 smoke на minimal
> example. Если evaluator требует > ⅔ day или deep integration refactor
> с runtime interpreter → extract в Plan 114.4.2.1. Plan 114.4.2 ships
> с Ф.1 only (checker rejects const fn calls с
> `E_CONST_FN_EVALUATOR_NOT_IMPLEMENTED`).

#### Architecture

**Reuse + extend Plan 14 Ф.2 constexpr-engine.** Существующий engine
вычисляет literals + arithmetic + record literals. Plan 114.4.2 extends:
- **Fn-call evaluation** — новый case в evaluator dispatch.
- **Environment-based interpreter** — param env + local const env
  отдельные scope'ы.
- **Sequential statements** — выполнение по одному (V1 без branching).
- **Final expression** — last expression в block-body возвращает
  результат.

#### Recursion-limit V1=1

- No recursion в V1 (checker rejects на static-time через call-graph
  analysis).
- Mutual recursion detection: build call-graph при body-check; cycle →
  `E_CONST_FN_RECURSION`.

#### Memoization

- За один compilation, кэш `(fn_id, arg_tuple_hash) → result`.
- Идентичные call sites evaluate один раз.
- Cache invalidated per compilation (no cross-compilation cache).

#### Errors на evaluator-side

- `E_CONST_FN_EVAL_OVERFLOW` — arithmetic overflow (signed/unsigned
  saturate vs wrap — explicit compile error, не silent).
- `E_CONST_FN_DIV_ZERO` — division by zero.
- `E_CONST_FN_EVAL_PANIC` — иной evaluator panic (defensive, не должно
  происходить per body checker).

### Ф.3 — Call-site validation + codegen replace

#### Call-site validation

- На call site `f(arg1, arg2, …)` где `f` — `const fn`:
  - Каждый arg обязан быть constexpr-evaluable (reuse Plan 114.4 Ф.1
    `check_const_constexpr`).
  - Иначе `E_CONST_FN_NON_CONST_ARG` с pointer на offending arg.
- Если все args constexpr → invoke evaluator → replace call с
  result-литералом в AST.

#### Codegen

- `const fn` НЕ emit'ится в C-output (нет runtime fn symbol).
- Все call sites replaced литералом (после evaluator).
- Dead `const fn` (без call sites) — silently dropped из output.

#### First-class reject

- `ro f = calc` где `calc` — `const fn` → `E_CONST_FN_FIRST_CLASS`
  (V1 reject; followup `[M-114.4.2-first-class]` через runtime-wrapper).

## Tests (T3 series carried-over from Plan 114.4)

### Positive

- **T3.1** `fn calc(const a int, const b char) -> const int { const c =
  b as int; a + c * 10 }`; `calc(5, 'A')` evaluates to literal `655`.
- **T3.2** Expression body: `fn add(const a int, const b int) -> const
  int => a + b`; `add(2, 3)` → 5.
- **T3.3** Const fn в `[N]T` size: `ro buf [calc(2, '0')]u8 = …`;
  array size 482.
- **T3.4** Const fn в default param: `fn open(n int = calc(3, ' '))`;
  default = 323.
- **T3.5** Const fn в record-field assoc-const (зависит от Plan 114.4.1
  shipped): `type T { const SIZE int = add(2, 3) }`; `T.SIZE == 5`.
- **T3.6** Const fn calling const fn: `fn outer(const x int) -> const
  int => add(x, 1)`; works.

- **T3-NEG.1** `calc(x, 'A')` где `x` runtime → `E_CONST_FN_NON_CONST_ARG`.

### Subsystems

1. **Parser** (`compiler-codegen/src/parser/mod.rs`):
   - `Param.is_const: bool` (Plan 114.4.2 AST extension).
   - `FnDecl.return_is_const: bool` (Plan 114.4.2 AST extension).
   - `parse_param`: `const` qualifier handling + 5 conflict checks.
   - `parse_fn`: `-> const T` parsing + all-or-nothing post-validation +
     7 sign-level reject codes.

2. **Body checker** (`compiler-codegen/src/types/mod.rs`):
   - `check_const_fn_expr` / `check_const_fn_block` / `check_const_fn_decl`
     — V1 whitelist + 7 error codes для blacklist.
   - `check_module` pass: build const_fn_names, validate each, build
     call_graph, DFS cycle detection (WHITE/GRAY/BLACK marker, full
     path в error message для mutual recursion).
   - `check_const_constexpr_ex`: extended variant — allows `As` cast +
     `Call(const_fn)` с constexpr args. Backward-compat через
     `check_const_constexpr` delegation.
   - TypeCheckCtx: `const_fn_names: HashSet<String>` + `in_const_fn:
     Cell<bool>` fields.
   - `check_fn` устанавливает in_const_fn flag (RAII restore via
     `ConstFnFlagGuard`).

3. **Comptime evaluator** (`compiler-codegen/src/const_fn_eval.rs`):
   - `ConstValue` enum (Int/Float/Bool/Str/Char/Unit) с manual
     PartialEq + Hash (float through `to_bits`).
   - `ConstFnEvaluator` + `OwnedEvaluator` (owns FnDecl clones для
     mutation borrow flow).
   - Memoization: `(fn_name, args) → ConstValue` per compilation.
   - `eval_call_inner` / `eval_block` / `eval_expr`: env-based
     interpreter.
   - `eval_binary`: typed dispatch — int (checked arithmetic +
     bitwise + shl/shr range [0..63] + cmp), float (Div 0.0 explicit),
     bool, str concat + cmp, char cmp.
   - `eval_as_cast`: identity + char↔int (Unicode range check) + int
     narrowing с range checks + float↔int + bool→int.
   - Errors: `E_CONST_FN_EVAL_OVERFLOW` / `E_CONST_FN_DIV_ZERO` /
     `E_CONST_FN_EVAL_PANIC` (defensive).

4. **AST rewriter** (`rewrite_const_fn_calls`):
   - Walks module.items + peer_files.items_here.
   - Skip Item::Fn если is_const_fn (interpreted at eval time).
   - For every `Expr::Call` where callee = const fn name AND все args
     constexpr → invoke evaluator → replace Call node с literal Expr.
   - After walk: retain filter removes const fn FnDecls.
   - Full AST coverage: control flow (If/IfLet/Match/For/ParallelFor/
     While/WhileLet/Loop), closures (Lambda/ClosureLight/ClosureFull),
     spawn/supervised/detach/blocking, with/forbid/realtime, handler/
