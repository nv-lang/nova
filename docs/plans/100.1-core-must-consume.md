## Цель

Compile-time гарантия «инстанс consume-типа обязан быть consumed до
выхода из scope'а на каждом code-path'е». Foundation для production-
grade resource management в Nova. Закрывает 5 базовых дизайн-измерений
(D1–D10 + D5.1), задаёт инфру (`LinearityRegistry`, field-aware flow,
binding-form), на которой строятся 100.2 (generics), 100.3 (borrow),
100.4 (cleanup-on-failure).

## Дизайн-решения (locked 2026-05-23)

### D1 — место маркера: type-decl

```nova
type Transaction consume { id int }
type File consume { fd i32 }
type Lock consume { mutex *Mutex }
```

`consume` стоит **после имени типа** (зеркало `mut`/`consume` на
receiver'е, D131). Type-level — единственная точка истины: обязательство
наследуется на все instance'ы консистентно.

### D2 — что считается consume

| Действие | Эффект на VarState |
|---|---|
| `tx.commit()` — consume-метод | `tx` → `Consumed` |
| `f(tx)` где `f(consume tx Tx)` | `tx` → `Consumed` |
| `return tx` (тип = consume) | `tx` → `Returned` (передача caller'у) |
| `record.field = tx` где record consume и field declared consume | `tx` → `Moved` (в record), `record.field` → `Live` |
| `f(tx)` где `f` без `consume` qualifier | ❌ compile error E (D133-move-to-non-consume) |
| `let _ = tx` | ❌ error (silent forget — no escape) |

### D3 — точки проверки (scope-exits)

На каждом exit-point'е проход по `ConsumeCtx.states`; live consume-var
→ error E (D133-not-consumed):

- конец function body;
- `return expr` (все live consume-vars кроме возвращаемой);
- `panic` / `expr!!` / `expr?` / unwinding-paths;
- `loop break`;
- branch join `if`/`match` — `Live ⊔ Consumed = MaybeConsumed`,
  использование MaybeConsumed на exit'е → error;
- `defer`/`errdefer` тело — учитывается (полная поддержка — Plan 100.4).

### D4 — заразность через поля + explicit double-marker

```nova
type TxState consume {                    // ← ОБЯЗАТЕЛЬНО
    consume tx Transaction,               // ← ОБЯЗАТЕЛЬНО (field type = consume)
    writes []Write,                       // обычное поле
}
```

Compiler enforces consistency:
- consume-поле без `consume`-маркера на field → error E (D133-field-marker-missing);
- consume-маркер на field без `consume` на type-decl → error E (D133-type-marker-missing);
- `consume` на type-decl без хотя бы одного consume-поля → error
  E (D133-empty-consume);
- Несоответствие field-marker vs field-type (`consume f int` где int
  не consume) → warning W (D133-marker-on-non-consume) (для DRY-стиля
  generic'ов 100.2 — silent ignore).

### D5 — consume-поле в методах record'а: field-aware flow analysis

`@field` — независимый VarState slot в `ConsumeCtx`. На exit'е метода:

| Тип метода | consume-поля должны быть |
|---|---|
| `fn X consume @method(...)` | `Consumed` (record closes) |
| `fn X mut @method(...)` | **`Live`** (invariant preserved) |
| `fn X @method(...)` (regular) | **`Live`** (invariant preserved) |

Реальные паттерны (reopen / rotate / replace):

```nova
fn Service mut @reopen() -> Result[(), OpenErr] {
    let new_file = File.open()?           // сначала добываем замену
    @file.close()                          // только теперь закрываем старое
    @file = new_file                       // rebind — @file опять Live
}                                          // mut exit: @file Live ✅
```

#### D5.1 — assignment в Live consume-поле / locals

Прямое присваивание `@field = expr` разрешено **только** когда
`@field` уже Consumed. Иначе compile error E (D133-assign-live-field)
с suggestion «consume the existing value first via `<consume-method>()`».

```nova
fn Service mut @overwrite_naive() {
    @file = File.open()?                  // ❌ @file Live, перезапись затёрла бы
                                          //    старый без close → утечка.
}

fn Service mut @overwrite_correct() {
    @file.close()
    @file = File.open()?                  // ✅ @file Consumed → assign OK
}
```

То же для локальных consume-var: `consume tx = ...; consume tx = ...`
без consume старой — error. Re-binding через shadow — отдельный случай
(см. D2 + Plan 73 alias-tracking).

#### D5.2 — nested field paths (full support, не bootstrap)

Multi-level field tracking — `ConsumeCtx` хранит state не только по
`@.field`, но по **произвольно глубокому пути** `@.f1.f2.f3`:

```nova
type Inner consume { consume tx Transaction }
type Outer consume { consume inner Inner }

fn Outer mut @commit_inner() {
    @inner.tx.commit()                    // ✅ deep path consume; @.inner.tx → Consumed
    @inner = Inner.new()                  // ✅ rebind inner (внутренний tx был Consumed)
}

fn Outer mut @broken() {
    @inner.tx.commit()                    // ❌ @inner.tx Consumed
    // exit без rebind @inner → @.inner.tx остаётся Consumed
}                                          // ❌ error: @.inner Live (это Outer) но содержит
                                          //    Consumed-поле, invariant nested-record broken
```

Реализация: `ConsumeCtx::states: HashMap<FieldPath, VarState>` где
`FieldPath = Vec<String>`. На каждый field access — extend path.

### D6 — generic-заразность

`type_is_consume(TypeRef)` — рекурсивная функция (общая, не Option-
специфичная):

```rust
fn type_is_consume(t: &TypeRef, registry: &LinearityRegistry) -> bool {
    if registry.contains(t.name) { return true; }      // объявлен `consume`
    match t.kind {
        TypeKind::Record(fields) =>
            fields.iter().any(|f| f.consume || type_is_consume(&f.ty)),
        TypeKind::Sum(variants) =>
            variants.iter().any(|v| v.payload.iter().any(|p| type_is_consume(p))),
        TypeKind::Generic { name, args } =>
            registry.contains(name) || args.iter().any(|a| type_is_consume(a)),
        TypeKind::Param(_) => false,                    // bootstrap: silent-ignore (100.2 закроет)
        _ => false,
    }
}
```

`Option[Transaction]` / `Result[Transaction, E]` / `Box[Transaction]` /
user `Wrapper[Transaction]` — все автоматически consume через wrap.
**Никакого Option-специфичного хардкода.**

### D7 — 3 mode'а binding-position (view-default)

**Единое правило:** в позиции param / for / match / if-let / let-alias
без qualifier'а = **view** (read-only borrow). С `mut` = mut-view.
С `consume` = ownership transfer.

```nova
fn print_id(tx Transaction) {              // view (default)
    println(tx.id)                          // ✅ read
    tx.commit()                             // ❌ consume через view — error
    tx.reopen()                            // ❌ mut через view — error (см. mut)
    finish(tx)                              // ❌ передача в consume-param — error
    storage.last = tx                       // ❌ store — error
    return tx                               // ❌ возврат — error (view не escape)
    let f = || tx.commit()                  // ❌ closure consume-capture — view-source escape
    let local = tx                          // ✅ view-alias (Plan 73 alias-class)
}

fn modify(mut tx Transaction) {            // mut-view (+ mut-методы OK)
    tx.reopen()                            // ✅
    tx.commit()                             // ❌ consume через mut-view — error
}

fn close(consume tx Transaction) {         // consume (transfer)
    tx.commit()                             // ✅
}
```

### D8 — `consume` + `-> @` несовместимы

`fn Tx consume @prepare() -> @ { ... }` → **parse error**. Противоречие
между «забираю целиком» и «возвращаю тот же объект».

### D9 — binding: `consume` keyword обязателен для ownership

Для consume-типов `consume` keyword **обязателен** в LHS когда binding
становится Live-linear-owner:

```nova
let tx = begin()                           // ❌ ERROR D133-consume-needs-keyword
consume tx = begin()                       // ✅ initial binding — owns

let alias = tx                             // ✅ view-alias (Plan 73 alias-class)
let mut alias = tx                         // ✅ mut-view-alias
consume new_owner = tx                     // ✅ transfer: tx → Consumed
```

Без strict-mode (отвергнут): обе формы `let tx`/`consume tx` для
fresh-value RHS — equivalent (но `let tx` для consume-type → error,
обязателен `consume`). `consume` keyword меняет behavior в alias-
position (transfer vs view-alias).

Без `consume` keyword'а LHS = view-alias. Симметрично с param/for/
match.

### D10 — runtime mental model (Option-projection, не ABI)

Концептуально consume-тип проецируется в `Option[T]`-space: `Live`
≡ `Some(t)`, `Consumed` ≡ `None`, `MaybeConsumed` ≡ branch-зависимо.
Mental model для spec/docs. **Реализация**:
- pointer-based consume: NULL = None (zero overhead);
- value consume: zero-out fields после consume;
- compile-time `check_consume` — основной механизм; runtime null-deref
  panic — defense-in-depth.

User-facing pattern-match `match tx { Some(t) => ... }` для runtime-
проверки **не вводится** — ослабит compile-time гарантии.

### Ф.4 — `check_consume` extension: must-consume на exit'ах + field-aware (~1.8 dev-day)

Расширяем existing `check_consume` pass (Plan 73 Ф.4):

**4.1 ConsumeCtx с FieldPath:**
```rust
type FieldPath = Vec<String>;     // [] = local var; ["@"] = receiver itself;
                                  // ["@", "file"] = @.file; ["@", "inner", "tx"] = @.inner.tx
struct ConsumeCtx {
    states: HashMap<(ScopeId, FieldPath), VarState>,
    aliases: AliasClasses,        // Plan 73 reuse
}
```

**4.2 Exit-point check** (D3) — на каждом exit'е (`return` / `panic` /
`expr!!` / `expr?` / `loop break` / function-end / branch-join):
- Local consume-vars: live → error E (D133-not-consumed) с suggestion
  списка consume-методов.
- Field paths внутри метода:
  - non-consume метод exit → все consume field-paths Live → error
    E (D133-field-not-restored).
  - consume метод exit → consume field-paths Consumed/Returned → OK.

**4.3 Assignment rules** (D5.1):
- Assign в Live consume-field/local → error E (D133-assign-live-field).
- Re-binding через shadow → first binding должен быть Consumed.

**4.4 Nested field paths** (D5.2):
- `@.f1.f2.commit()` — extract path `["@", "f1", "f2"]`, mark Consumed.
- При exit'е invariant nested: если `@.f1` Live, то `@.f1.fX` consume-
  поля должны быть Live.

**4.5 Передача в non-consume param** (D2 / D7) — error E (D133-move-
to-non-consume).

**4.6 Match на consume-value** — destructive по умолчанию (`Some(f)`
arm даёт Live linear f, обязан consumed в arm'е). Borrow-match
(`match view`) — отдельно в **100.3**.

**Acceptance Ф.4:** 4 probe-фикстуры (forget / branch-forget / loop-
forget / field-not-restored / nested-broken) дают expected errors;
`reopen_test` ✅ и `nested_reopen_test` ✅ PASS.

## Acceptance criteria

- [ ] `type X consume { ... }` парсится; согласованность маркеров (D4)
      enforced.
- [ ] `consume tx = ...` (binding) парсится с strict-семантикой (D9).
- [ ] Consume-instance, не consumed до scope exit'а → compile error
      E (D133) с named consume-методами в diagnostic.
- [ ] Branch-forget / loop-forget / panic-path-forget → compile error.
- [ ] Передача consume в non-consume non-receiver fn-param → compile
      error (D2 / D7).
- [ ] Read-only param (без `consume`) — только чтение + alias; consume-
      метод / store / return / closure-capture-с-consume → error (D7).
- [ ] Field-aware flow внутри методов (D5): mut-метод reopen pattern
      работает; D5.1 assign в Live consume-поле → error.
- [ ] **Nested field paths (D5.2):** `@.inner.tx.commit()` tracks
      Consumed; `@.inner` cannot exit с Consumed-внутренностями в non-
      consume методе.
- [ ] `Option[Transaction]` / `Box[Transaction]` / user-generic с
      consume-arg — автоматически consume через generic-заразность
      (D6); никакого Option-специфичного кода.
- [ ] `consume` + `-> @` на одном методе → parse error (D8).
- [ ] D131 affine остаётся доступным; `consume` без type-level
      `consume` работает как раньше (forget OK).
- [ ] Pilot `nova_tests/plan100.1/` — 8 pos + 9 neg PASS как expected.
- [ ] Полный `nova test` → 0 регрессий vs baseline (~1130 PASS).
- [ ] Spec D133 опубликован; cross-ref'ы D131/D132/D75/D90 ✅.

## Risks

1. **Field-aware flow для nested paths (D5.2)** — расширение
   `ConsumeCtx` на `FieldPath = Vec<String>`. Сложность — alias-class
   через field-paths. **Митигация:** Plan 73 alias-tracking уже работает
   на flat именах; nested — straightforward extension через path-keys.
2. **Suppress-pressure** — pull-request с «forget» escape → gateway-
   bug. **Митигация:** явная позиция в spec D133 §«Anti-pattern».
3. **Migration cost** реальных типов — pilot 100.1 без миграций;
   реальные типы в Plan 18.
