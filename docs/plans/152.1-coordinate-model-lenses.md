## Идея

Ядро-разворот: `str` — тонкий «кусок текста», работа через линзы-представления.
Асимметрия: **`as_bytes()` — reinterpretation** (байты лежат подряд → настоящий
`ro []u8`, O(1) `[i]`/`len`); **`as_chars()` — decoding lens** (codepoint'ы
вычисляются → отдельный view-тип, O(n)). Целочисленной индексации `str` нет.

---

## D249. Строковая координатная модель: линзы вместо плоских методов

1. **`str` не индексируется целым** — не реализует `Index[int,V]`. `str[i]` →
   `E_STR_NO_INT_INDEX` (fix-it: символ `as_chars().at(i)`, байт `as_bytes()[i]`,
   срез `s[a..b]`).
2. **`str[a..b]`** — byte-range zero-copy view (`Index[Range,str]`), O(1), panic при
   OOB / рассечении codepoint-границы.
3. **`@as_bytes() -> ro []u8`** — байтовый слой (бесплатно из `Vec[u8]`, D239):
   `[i]`/`len()`/срезы/итерация O(1). `@byte_at` ретайрнут.
4. **`@as_chars() -> CharsView`** — codepoint-слой (D250). Плоские
   `@char_at`/`@char_len`/`@char_indices` со `str` убраны → в линзу.
5. **`@find`/`@rfind` → байт-offset** (композируются с `s[k..]` за O(1)).
6. **`for c in s` → `char`** (= `s.as_chars().iter()`).
7. **Бэар `@len()` убран** → `E_STR_NO_LEN`. `@byte_len()` (O(1)) — единственный
   length-метод на `str`; char-длина только `as_chars().len()`. Поле `len` (storage)
   остаётся.
8. **Инвариант R-UTF8** (D26 AMEND): значение `str` всегда валидный UTF-8 →
   `as_chars()`-decode тотален.

**Отвергнуто:** `str[i]→u8` (Go) / `str[i]→char` (Python) — индексация строки
провоцирует `for i in 0..len`-антипаттерн; нужное через линзу. Дефолт-grapheme
(Swift) — Unicode-data в ядро → библиотека (152.4). Индексируемый `CharsView` —
«O(n) под видом O(1)».

## D250. `CharsView` (decoding lens) + конвенция `as_`/`to_`

```nova
type CharsView value priv {
    ptr      *ro u8     // тот же буфер, что у str (read-only, алиас)
    byte_len int        // длина буфера в БАЙТАХ
}
```

| Метод | Смысл | Сложность |
|---|---|---|
| `@len() -> int` | число codepoint'ов | O(n) |
| `@at(i int) -> Option[char]` | i-й codepoint (имя сигналит скан) | O(n) |
| `mut @next() -> Option[char]` | курсор → `Next[char]` | аморт. O(1) |
| `@iter() -> CharsView => self` | `Iter[CharsView]` | O(1) |
| `@indices() -> CharIndicesView` | пары `(byte_offset, char)` | O(n) |
| `@is_empty() -> bool` | `byte_len == 0` | O(1) |

- **`CharsView` НЕ реализует `Index[int,char]`** — только `at(i)`/итерация.
- Decode-курсор — UTF-8-алгоритм из нынешних `@char_at`/`@to_chars`, переезжает сюда.
- **Конвенция:** `as_<repr>()` = линза (zero-copy, алиас), `to_<repr>()` = owned копия.
- **GC/lifetime:** `CharsView` алиасит GC-буфер `str` (как `as_bytes`); `str`
  иммутабелен (R8), conservative GC видит `ptr` → буфер жив, пока жив view.

- **Ф.1 — Снять `Index[int,char]` со `str`.** Compiler str-index dispatch + checker:

- **Ф.2 — `@find`/`@rfind` → байт-offset** (убрать cp-счётчик).
- **Ф.3 — `CharsView` + `CharIndicesView` + `str.@as_chars()`** (перенос decode-курсора).

  (Index[int])/`@byte_at`; оставить `@byte_len`/`@as_bytes`/`@as_chars`/`@to_bytes`/
  `@to_chars`; `for c in s` → `as_chars().iter()`; `E_STR_NO_LEN`.
- **Ф.5 — Фикстуры + миграция** (`str[i]`/`s.len()`/`char_*`-callers → линзы).

## Тесты

- **POS:** `for c in s` (ASCII+мультибайт «héllo»/эмодзи); `s[a..b]` zero-copy;
  `s.as_bytes()[i]`/`.len()`; `s.as_chars().at(i)`/`.len()`/`.indices()`;
  `s.find(x)`→`s[k..]` композиция; конвенция `as_`/`to_`.
- **NEG:** `s[i]` целым → `E_STR_NO_INT_INDEX`; `s.len()` → `E_STR_NO_LEN`;
  `chars[i]` целым → нет `Index[int]`; `s[a..b]` рассекает codepoint → panic;
  `as_chars()` пережил `str` (lifetime/GC — регресс-тест).

## Критерии приёмки

- **A1.** `str[i]`(int) → `E_STR_NO_INT_INDEX` с fix-it.
- **A2.** `str[a..b]` zero-copy byte-slice; panic при OOB / рассечении codepoint.
- **A3.** `@as_bytes() -> ro []u8` даёт O(1) `[i]`/`len()`/итерацию; `@byte_at` ретайрнут.
- **A4.** `@as_chars() -> CharsView` с `len/at/next/iter/indices/is_empty`; НЕ
  реализует `Index[int,char]`.
- **A5.** `for c in s` итерирует `char` == `@to_chars()`-порядок/значения.
- **A6.** `@find`/`@rfind` → байт-offset; `find`→`s[k..]` композируется zero-copy.
- **A7.** У `str` нет `@len()` (`E_STR_NO_LEN`) и плоских `@char_*`; `@byte_len()`
  (O(1)) остаётся.
- **A8.** `str`: `Index[Range,str]` ∧ ¬`Index[int,_]`; `CharsView`:
  `Iter[CharsView]`/`Next[char]`.
- **A9.** Инвариант R-UTF8 зафиксирован в D26; unchecked-конструкторы несут контракт.
- **A10.** Полный `nova test` без новых FAIL vs baseline; plan152_1-фикстуры PASS.
