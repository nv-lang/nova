## Идея

Ядро-разворот: `str` — тонкий «кусок текста», работа через линзы-представления.
Асимметрия: **`as_bytes()` — reinterpretation** (байты лежат подряд → настоящий
`ro []u8`, O(1) `[i]`/`len`); **`as_chars()` — decoding lens** (codepoint'ы
вычисляются → отдельный view-тип, O(n)). Целочисленной индексации `str` нет.

## D249. Строковая координатная модель: линзы вместо плоских методов

1. **`str` не индексируется целым** — не реализует `Index[int,V]`. `str[i]` →
   `E_STR_NO_INT_INDEX` (fix-it: символ `as_chars().at(i)`, байт `as_bytes()[i]`,
   срез `s[a..b]`).
2. **`str[a..b]`** — byte-range zero-copy view (`Index[Range,str]`), O(1), panic при
   OOB / рассечении codepoint-границы.
3. **`@as_bytes() -> ro []u8`** — байтовый слой (бесплатно из `Vec[u8]`, D239):
   `[i]`/`len()`/срезы/итерация O(1). `@byte_at` ретайрнут.
4. **`@as_chars() -> CharsView`** — codepoint-слой (D250). Плоские
   `@char_at`/`@char_len`/`@char_indices` со `str` убраны → в линзу.
5. **`@find`/`@rfind` → байт-offset** (композируются с `s[k..]` за O(1)).
6. **`for c in s` → `char`** (= `s.as_chars().iter()`).
7. **Бэар `@len()` убран** → `E_STR_NO_LEN`. `@byte_len()` (O(1)) — единственный
   length-метод на `str`; char-длина только `as_chars().len()`. Поле `len` (storage)
   остаётся.
8. **Инвариант R-UTF8** (D26 AMEND): значение `str` всегда валидный UTF-8 →
   `as_chars()`-decode тотален.

**Отвергнуто:** `str[i]→u8` (Go) / `str[i]→char` (Python) — индексация строки
провоцирует `for i in 0..len`-антипаттерн; нужное через линзу. Дефолт-grapheme
(Swift) — Unicode-data в ядро → библиотека (152.4). Индексируемый `CharsView` —
«O(n) под видом O(1)».

## D250. `CharsIter` (codepoint-итератор) + конвенция `as_`/`to_`

> **РЕВИЗИЯ 2026-06-13 (Q1, апрув автора): `CharsView` → `CharsIter` (вариант C).**
> codepoint'ы — **поток** (декод на проходе), не коллекция. View с `at`/`len` —
> ложная коллекция: кэша нет, `at(i)`/`len()` каждый = O(n), а наличие обоих
> провоцирует `for i in 0..len { at(i) }` = **O(n²)** — ровно тот footgun, что убран у
> `str[i]` (нарушение I2 «no hidden O(n)»). Честный примитив для потока — **итератор**
> (как Rust `str::chars()` = `Chars`, без `len`/index). Держим `str`, а не `ptr+byte_len`:
> ноль priv-poking, ноль `unsafe`, переиспользуем `as_bytes`.

```nova
type CharsIter value priv {
    buf str    // borrows str (алиас буфера; GC видит ptr через str-поле)
    pos int    // текущий байтовый offset в buf
}
```

| Метод | Смысл | Сложность |
|---|---|---|
| `mut @next() -> Option[char]` | декод codepoint в `buf[pos]`, `pos += step` (`Next[char]`) | аморт. O(1) |
| `@iter() -> CharsIter => self` | `Iter` (self-iterator, для D58 `for c in s`) | O(1) |
| `@count() -> int` | число codepoint'ов (терминатор, потребляет) | O(n) |
| `@nth(i int) -> Option[char]` | i-й codepoint (явный скан) | O(n) |
| `@is_empty() -> bool` | `pos >= buf.byte_len()` | O(1) |

- **`CharsIter` НЕ реализует `Index[int,char]`** и НЕ имеет позиционных `at(i)`/`len()`
  коллекции — только `next`/`count`/`nth`. Нет `char_len()`/`char_at()` на `str` (это и
  есть минимализм, хендофф 153 п.3).
- **`str @as_chars() -> CharsIter`** (ленивый, borrows str). **`str @iter() -> CharsIter
  => @as_chars()`** → `for c in s` работает (D58). Codepoint-count = `s.as_chars().count()`;
  N-й символ = `s.as_chars().nth(i)`.
- Decode-курсор — UTF-8-алгоритм из нынешних `@char_at`/`@to_chars`, переезжает в `next`
  (читает `@buf.as_bytes()[pos]`, без `unsafe`).
- **`char_indices()` — ОТДЕЛЬНЫЙ итератор** `CharIndicesIter` (yields `(byte_offset, char)`),
  для slice-after-walk. Можно **отложить в Phase B** (`[M-152-char-indices]`), если не нужен
  сразу в 152.1.
- **Конвенция:** `as_chars()` = ленивый итератор (borrows str); `to_chars() -> []char` =
  owned копия (alloc). `as_<repr>()` = линза/итератор-вью, `to_<repr>()` = owned.
- **GC/lifetime:** `CharsIter.buf` (str) держит `ptr` → conservative GC видит буфер живым,
  пока жив итератор; `str` иммутабелен (R8).

- **Ф.0 — Spec.** D249 + D250 в `03-syntax.md`; AMEND D26(MAJOR + R-UTF8)/D238/D58;

  `@to_chars`; `for c in s` → `@as_chars()`; `E_STR_NO_LEN`. codepoint-count =
  `as_chars().count()`, N-й = `as_chars().nth(i)`.

## Тесты

- **POS:** `for c in s` (ASCII+мультибайт «héllo»/эмодзи); `s[a..b]` zero-copy;
  `s.as_bytes()[i]`/`.len()`; `s.as_chars().count()`/`.nth(i)`; `s.find(x)`→`s[k..]`
  композиция; конвенция `as_`/`to_`.
- **NEG:** `s[i]` целым → `E_STR_NO_INT_INDEX`; `s.len()` → `E_STR_NO_LEN`;
  `s.as_chars()[i]` целым → нет `Index[int]`; нет `s.char_len()`/`s.char_at()` (методы
  убраны со `str`); `s[a..b]` рассекает codepoint → panic; `as_chars()` пережил `str`
  (lifetime/GC — регресс-тест).

## Критерии приёмки

- **A1.** `str[i]`(int) → `E_STR_NO_INT_INDEX` с fix-it.
- **A2.** `str[a..b]` zero-copy byte-slice; panic при OOB / рассечении codepoint.
- **A3.** `@as_bytes() -> ro []u8` даёт O(1) `[i]`/`len()`/итерацию; `@byte_at` ретайрнут.
- **A4.** `@as_chars() -> CharsIter` с `next/count/nth/iter/is_empty`; НЕ реализует
  `Index[int,char]` и НЕ имеет позиционных `at(i)`/`len()` (вариант C, Q1). `@iter()` =>
  `@as_chars()`.
- **A5.** `for c in s` итерирует `char` == `@to_chars()`-порядок/значения.
- **A6.** `@find`/`@rfind` → байт-offset; `find`→`s[k..]` композируется zero-copy.
- **A7.** У `str` нет `@len()` (`E_STR_NO_LEN`) и плоских `@char_len`/`@char_at`;
  `@byte_len()` (O(1)) остаётся; codepoint-count = `as_chars().count()`.
- **A8.** `str`: `Index[Range,str]` ∧ ¬`Index[int,_]`; `CharsIter`: `Iter`/`Next[char]`,
  `{buf str, pos int}` (ноль `unsafe`/priv-poking, через `as_bytes`).
- **A9.** Инвариант R-UTF8 зафиксирован в D26; unchecked-конструкторы несут контракт.
- **A10.** Полный `nova test` без новых FAIL vs baseline; plan152_1-фикстуры PASS.
