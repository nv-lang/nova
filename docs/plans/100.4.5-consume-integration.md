# Plan 100.4.5: consume-integration final — check_consume + defer/errdefer/okdefer + cancel-aware

> **Создан 2026-05-23 как sub-sub-plan [Plan 100.4 umbrella](100.4-cleanup-on-failure.md).**
> **Статус:** 📋 proposed. **GATED на 100.1 + 100.4.1 + 100.4.2 + 100.4.3 + 100.4.4**.
> **Оценка:** ~3 dev-day.
> **D-блок:** **D140**.

---

## Цель

Финальный sub-sub-plan — интегрирует consume-семантику (Plan 100.1)
с амеандированными defer/errdefer/okdefer (Plan 100.4.1–4) для
production-grade автоматического cleanup-coverage consume-vars на
ВСЕХ exit-paths (включая supervised-cancel, async-await, handler-flow-
interrupt).

Без Plan 100.4.5 consume-проверка из 100.1 видит `errdefer { tx.rollback()
}` как «непонятный statement», не cover'ит обязательство. Caller'у
приходится явно потреблять tx на каждом code-path вручную — defeats
ergonomics.

Plan 100.4.5 закрывает: `check_consume` распознаёт defer/errdefer/
okdefer как покрывающие consume-vars на соответствующих exit-paths.

on cleanup-panic; explicit commit/rollback distinction через okdefer;
effect-typed cleanup-fail).

## Дизайн-решения

### D1 — check_consume распознаёт defer-family cover

Расширение `check_consume` pass (Plan 100.1 Ф.4):

При scan'е scope: для каждого defer/errdefer/okdefer статемента —
анализ body на consume-операции над live consume-vars.

Если body содержит `var.consume_method()` или `consume_fn(var)` —
markers cover'ing:

| Statement | Покрывает consume на path'е |
|---|---|
| `defer { tx.commit() }` | **все exit-paths** (success, error, panic, interrupt) |
| `errdefer { tx.rollback() }` | **error-paths** (throw, panic, interrupt после D90 §7 amend) |
| `okdefer { tx.commit() }` | **success-path** (normal exit, return) |

Каждый exit-point check:
- Если consume-var Live: covered ли каким-то defer-family entry на
  этом path'е?
  - Да → OK.
  - Нет → error E (D140-not-consumed-on-path).

### D2 — Multiple defers на одну consume-var

```nova
consume tx = begin()
errdefer { tx.rollback() }
okdefer  { tx.commit() }
do_work()?
// tx covered:
//   - error-path: errdefer rollback
//   - success-path: okdefer commit
// no explicit tx.commit() needed
```

`check_consume` видит: tx covered errdefer + okdefer = exhaustive
(error + success). No live consume-var на exit. OK.

### D3 — Explicit commit + okdefer = double-consume error

```nova
consume tx = begin()
okdefer { tx.commit() }
tx.commit()                                  // ❌ ERROR: double-consume
                                             //    (okdefer also commits)
```

Compile error E (D140-double-cover) с suggestion «remove explicit
commit or remove okdefer».

### D4 — Partial coverage — error на uncovered path

```nova
consume tx = begin()
errdefer { tx.rollback() }
do_work()?
// SUCCESS PATH: tx Live, errdefer не fires, no explicit commit
// ❌ ERROR: tx not consumed on success path
```

Force programmer'у написать `tx.commit()` или добавить `okdefer { tx.commit() }`.

### D5 — Conditional consume в defer body — warning

```nova
defer {
    if condition {
        tx.commit()                          // conditional
    }
}
```

Compiler видит: cover только если condition true. **Pragmatic решение:**
warning W (D140-conditional-cover), не error. Programmer ответственен.

Strict mode (опциональный flag `--strict-consume`) — escalates до
error.

### D6 — Cancel-aware: D90 §7 amend

D90 §7 текущее: `interrupt` НЕ триггерит errdefer. Для consume — это
дыра: handler-flow `interrupt v` exit'ит scope с consume Live, errdefer
не cover'ит. Compile error на каждом fn-с-interrupt-возможностью.

**Решение:** D90 §7 amended — `interrupt` тоже триггерит errdefer
(симметрично с throw/panic).

Логика: errdefer = «exit без normal completion». throw / panic /
interrupt — все «abnormal» exits относительно success-path.

Backward-compat impact: handler-flow user code — errdefer'ы now fire
on interrupt. Аудит существующих фикстур в 100.4.5 Ф.0 GATE.

### D7 — Supervised + spawned-fiber consume cleanup

```nova
supervised(cancel: tok) {
    spawn {
        consume tx = begin()
        errdefer { tx.rollback() }
        long_op()                                // may cancel
        tx.commit()
    }
}
```

Cancel сценарий:
- outer cancels.
- spawned fiber unwinding с cancel.
- errdefer triggers — `tx.rollback()` runs (cancel-shielded по 100.4.2).
- rollback completes (или Time.timeout fail composes).
- fiber dies.
- supervised continues unwinding.

`check_consume` для spawned-closure body: видит errdefer cover'ит на
error-path; на cancel-path errdefer fires (per amended D90 §7);
consume-обязательство удовлетворено.

### D8 — Async-await preservation

```nova
fn process() Fail Async -> () {
    consume tx = begin()
    errdefer { tx.rollback() }
    await long_async_op()                       // suspend
    tx.commit()
}
```

`await` — regular call (not exit-point). check_consume:
- pre-await: tx Live, errdefer registered.
- post-await: tx still Live.
- success exit: explicit commit → tx Consumed.
- if cancel-during-await → errdefer fires → tx Consumed via rollback.

Standard flow, no special handling beyond D7.

### D9 — Cleanup-fail integration с consume

Plan 100.4.1 (failable cleanup body) интегрируется тривиально:

```nova
consume tx = begin()
errdefer { tx.rollback() }                  // rollback may Fail (D136)
do_work()?
tx.commit()                                  // commit may Fail
```

Flow на error:
- do_work fails Err1.
- errdefer rollback fires.
- If rollback succeeds — propagate Err1.
- If rollback fails Err2 — composite { primary: Err1, suppressed: [Err2] }.

check_consume не различает — все Fail-effects в cleanup body — допустимы
если fn-sig declares Fail[E] (D136).

### D10 — Pilot: full Transaction lifecycle

```nova
type Transaction consume { id int }
fn Transaction consume @commit() -> Result[(), CommitErr]
fn Transaction consume @rollback() -> Result[(), RollbackErr]

fn process_order(data Data) Fail[OrderErr] Db -> Receipt {
    consume tx = Db.begin()
    errdefer { tx.rollback()? }              // rollback failable
    okdefer  { tx.commit()?   }              // commit failable

    let order = Db.insert(data)?
    let receipt = Db.notify(order)?
    return receipt                            // okdefer fires → commit
}
// Error: errdefer fires → rollback
//        composite если rollback fail
// Success: okdefer fires → commit
//          throw если commit fail
```

Это canonical pattern. Каждая часть — production-grade.

## Границы

- **Deep field-path cover** (`@.inner.tx.commit()` в defer cover'ит
  `@.inner.tx`) — поддерживается через 100.1 D5.2 nested paths +
  recognized in 100.4.5 Ф.2.
- **Generic consume cover в defer** — если `[T consume]` generic-param
  (Plan 100.2) — defer-cover'ит T-binding если type_is_consume(T) при
  instantiation; уже из D6 (100.1).

## Risks

1. **D90 §7 amend** (interrupt → errdefer trigger) — breaks existing
   handler-flow user-code. **Митигация:** Ф.0 GATE audit; если breakage
   значителен — documented as semantic-version-breaking change (Plan 91
   relevance).
2. **check_consume complexity** — multi-path cover analysis. **Митигация:**
   path enumeration finite (success / error / panic / interrupt /
   cancel); каждая path-cover check — O(N defer-stack).
3. **Diagnostic clarity** для multi-cover scenarios. **Митигация:** AI-
   first diagnostic с explicit listing covered/uncovered paths.
