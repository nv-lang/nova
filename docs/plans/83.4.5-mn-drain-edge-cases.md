## 1. Контекст

После закрытия 5 named-bugs из `[M-83.2-supervised-mn-bugs]`
(A1 D93 park, A2 supervised double-resume, A3 per-fiber handlers,
B2 time_handler, B5 atomic cancel) **попытка активации flip**
(уkomментировать `nova_runtime_auto_arm()` в `emit_main_wrapper`)
дала **18 RUN-FAIL** в concurrency suite:

```
cancel_semantics_test
detach_test
main_yield
parallel_for          (даже с // ENV NOVA_MAXPROCS=1 директивой)
per_fiber_handlers    (handler-migration уже включён, но не покрывает 100%)
plan40_perf_bench
sleep_precision_bench
sleep_real_clock
supervised_cancel_stress_test
time_handler
+ ещё ~8 тестов
```

Это значит — 5 «named» багов были только **видимыми** проявлениями,
а под flip активацией всплывает **серия дополнительных edge cases**
в supervised-drain layer.

## 2. Категории edge cases (предполагаемые)

### A. Supervised drain deadlock под cancel-stress

`supervised_cancel_stress_test` — много supervised{} вложенных с
cancel — race между cancel propagation и worker drain.

### B. Cooperative-vs-M:N ordering divergence

`parallel_for` помечен `NOVA_MAXPROCS=1`, но даже с 1 worker'ом
порядок исполнения через M:N scheduler ≠ cooperative drain. Тесты
с encoded-log ordering всё равно падают.

Решение: переписать на set-equality либо явно сделать тест
single-thread (без M:N вообще через ENV маркер `NOVA_NO_MN=1`,
которого пока нет).

### C. detach семантика

`detach_test` опирается на inline-выполнение detach body. Под flip
M:N detach уходит на worker — async dispatch, не inline. Семантика
меняется.

### D. Timing precision

`sleep_precision_bench`, `sleep_real_clock` — измеряют wall-clock
точность Time.sleep. Под M:N worker scheduling добавляет джиттер;
precision-budget теста (≤1ms) может превышаться.

### E. main_yield race

`main_yield` — мой B4 fix добавил `uv_run(NOWAIT)` на main; под
активным flip main thread теперь имеет armed runtime, конфликт
с supervised drain'ом.

### F. handler-storage corner cases

`per_fiber_handlers` всё ещё падает несмотря на handler-migration.
Возможно corner cases с nested supervised{} или с `with`-блоками
внутри parallel for.

### G. plan40_perf_bench

Latency-bench — measured timing assertions. Под M:N добавляется
overhead.

## 3. Подход

### Ф.0 — Reproducible enumeration (~1 dev-day)

Прогнать flip-activated build, для КАЖДОГО из 18 FAIL:
1. Capture exact error output.
2. Изолировать root cause.
3. Классифицировать (A-G выше) либо новая категория.
4. Оценить дёшево-vs-дорого fix.

Документ: `83.4.5-artifacts/f0-enumeration.md`.

### Ф.1-Ф.N — Per-category fixes

Каждая категория — отдельная фаза с targeted-тестом.

Категории A-C-F (runtime races) — самое тонкое, ~1-2 dev-day каждая.
Категории D-G (timing) — пометить `NOVA_MAXPROCS=1` или скип под
M:N с честным комментарием (semantic differences expected).
Категория B (parallel_for ordering) — test rewrite на set-equality.

### Ф.Final — Re-activate flip + closure

Раскомментировать `nova_runtime_auto_arm()` в `emit_main_wrapper`.
Полный `nova test`: ≥1107 PASS, 0 FAIL (или известные флэки).
Закрыть [Plan 83.2](83.2-mn-default-flip.md) Ф.2+Ф.3 + [Plan 83](83-mn-default-on-gomaxprocs.md)
целиком. Spec D-block (default-on M:N model). Speedup-бенчмарк.

## 4. Acceptance

- Все 18 RUN-FAIL'ов из попытки 2026-05-23 проходят (либо явно
  скипнуты с обоснованием).
- Полный clang `nova test` после активации: ≥ 1107 PASS / 0 FAIL.
- Plan 83.2 §4 acceptance выполнен (compiled-программа без
  `runtime.*` использует все CPU).
- Spec D-block по default-on M:N + cross-runtime comparison.
- Speedup-bench на parallel_sum / parallel_hash / parallel_fib.

## 5. Risk

- **Каждый edge case требует отдельной investigation** — нельзя
  предсказать сложность fix'ов до Ф.0 enumeration.
- **Test fallout cascade** — fix одного теста может породить fail
  в другом (handler scoping сложно изолировать).
- **Не все 18 могут быть быстро fix-able** — некоторые могут стать
  «известными ограничениями M:N» в spec.
