```
cancel_semantics_test          ← cancel propagation race
detach_test                    ← inline-vs-async detach semantics
main_yield                     ← yield на armed runtime
parallel_for                   ← ordering 1-worker M:N ≠ cooperative
per_fiber_handlers             ← handler corner cases за пределами Ф.2

sleep_precision_bench          ← wall-clock precision под M:N jitter
```

вскрывает следующий слой — настоящие M:N edge cases в supervised
drain layer, которые до 83.4 были закрыты codegen-веткой
`if (is_initialized()) M:N else single-thread` (без явного init шла
single-thread кооперативная ветка).

## 2. Cross-runtime reference — что должен дать паритет

Для КАЖДОЙ категории edge cases выбран **production-grade reference
implementation** из mature M:N runtime; Nova цель — не хуже:

| Edge case category | Go runtime | tokio | Kotlin | Node | **Nova цель** |
|---|---|---|---|---|---|
| Cancel propagation + wake-all parked | `context.Context.Done()` channel close → all selects fire | `CancellationToken.cancel()` → `Notify::notify_waiters()` | `Job.cancel()` → propagates via `CoroutineContext` | `AbortSignal.dispatchEvent('abort')` | scope cancel → atomic flag + `nova_sched_wake` всех parked в scope-tree |
| Detach semantics | `go fn()` — fire-and-forget на scheduler | `tokio::spawn(future)` + drop `JoinHandle` | `launch { … }` без `await` | `setImmediate(cb)` / queueMicrotask | `detach { … }` под M:N → spawn на worker (не inline) |
| Parallel ordering tests | `golang.org/x/sync/errgroup` + `WaitGroup`; ordering не специфицирован | `try_join!` / `FuturesUnordered` — order не специфицирован | `awaitAll(Deferred<T>)` — index-preserved результат, exec-order нет | `Promise.all([…])` — index-preserved, exec-order нет | переписать тесты на **set-equality** (sorted log compare) либо явно `NOVA_MAXPROCS=1` directive |
| Timing precision under scheduler jitter | `time.Sleep` ≥ d (overshoot allowed); benchmarks с relaxed budgets (±5-20ms) | `tokio::time::sleep` ≥ d, driver tick 1ms; bench с criterion outliers | `delay(d)` ≥ d | `setTimeout(cb, d)` ≥ d, event-loop jitter ~4-15ms | relax budgets под M:N (±10ms slack) либо single-worker для precision tests |
| Per-fiber handler scoping (nested) | Goroutine не имеет (нет algebraic effects) | `task_local!` не handles nested with{} | `CoroutineContext.Element` — immutable, fork-on-set | `AsyncLocalStorage.run(store, fn)` snapshot per call | snapshot save/restore вокруг nested `with X = h { … }` — корректный для arbitrary nesting под M:N |
| Yield on main thread | `runtime.Gosched()` — yields current G; на main = goroutine #1, нормально | `tokio::task::yield_now()` work только в task context | `yield()` suspend fn только в coroutine | `await new Promise(r => setImmediate(r))` | branch: fiber yield → mco_yield; main yield → `uv_run(NOWAIT)` (B4 fix); проверить interaction с armed runtime |
| Supervised drain deadlock | scheduler централизованный, нет supervised-drain primitive | task tree через `JoinSet`/`select!` | `coroutineScope { … }` joins дочерние; cancel cascade автоматический | n/a | `nova_supervised_run` + cancel cascade race-free под M:N: cancel-publish → wake all parked → drain pending_remote |

**Что Nova добавляет лучше Go/tokio:** algebraic effects (handler-
scoping per `with`-block) — этого нет в Go (handlers глобальные либо
через context), в tokio (task_local плоский), в Kotlin (`CoroutineContext.Element`
fork-on-set — близкий, но без scoped with-block semantic).

**Где Nova может быть в одной лиге с Go/tokio:** cancel propagation
(если done через wake-all + scope-tree cascade), detach semantics
(если переведён на worker), drain race-freedom (если ownership-
protocol чёткий).

## 3. Декомпозиция в 6 sub-plan'ов

### Принцип декомпозиции

По **корневой причине**, не по тест-имени. 18 fails — это
проявления **6 distinct architectural issues**:

| Sub-plan | Корневая причина | Тесты затронуты | Размер |
|---|---|---|---|
| [83.4.5.1](83.4.5.1-cancel-wake-all-and-cascade.md) | Cancel published, но parked fibers не wake'аются автоматически — ждут next yield/timeout → deadlock под cancel-stress | `cancel_semantics_test`, `supervised_cancel_stress_test`, частично `parallel_for` (если cancel внутри) | ~2 dev-day |
| [83.4.5.2](83.4.5.2-detach-mn-semantics.md) | Detach под M:N inline-vs-async ABI: bootstrap inline-`SyncDetach`-handler через codegen-default; под flip — должен идти worker, но codegen не отличает | `detach_test` | ~2-3 dev-day |
| [83.4.5.3](83.4.5.3-test-suite-mn-cleanup.md) | Test-side: ordering assumptions (encoded log), precision budgets (latency benches под M:N jitter), tight wall-clock budgets | `parallel_for` (rewrite на set-equality), `sleep_precision_bench`, `sleep_real_clock`, `plan40_perf_bench` | ~1-2 dev-day |
| [83.4.5.4](83.4.5.4-handler-scoping-nested.md) | Handler-snapshot save/restore (Plan 83.4.2 Ф.2) корректен для basic case, но nested `with X = h { spawn { with X = h2 { … } } }` / cross-spawn handler timing / Time-effect specifics — corner cases | `per_fiber_handlers`, `time_handler` | ~2-3 dev-day |
| [83.4.5.5](83.4.5.5-main-yield-armed-runtime.md) | `nova_fiber_yield` на main thread (B4 fix `uv_run(NOWAIT)`) под armed runtime — interaction с worker async-event-callbacks; re-entrance risk | `main_yield` | ~1 dev-day |
| [83.4.5.6](83.4.5.6-flip-reactivation-and-closure.md) | После 83.4.5.1-83.4.5.5: re-uncomment `nova_runtime_auto_arm`; C1+C2 ассерты; Plan 83.2 Ф.2 sweep + Ф.3 spec D-block + speedup-bench; полная suite stress + TSAN gate; closure | C1+C2 + flip + sweep + spec + bench | ~2-3 dev-day |

## 5. Production-grade acceptance (umbrella)

### Корректность

- Все 18 RUN-FAIL'ов из flip-attempt проходят (либо явно скипнуты
  под `NOVA_MAXPROCS=1` directive с обоснованием в Ф.0
  enumeration).
- Полный clang `nova test` после активации (83.4.5.6 Ф.2): **≥ 1111
  PASS / 0 FAIL / 56 SKIP** (baseline до flip — 1111/0).
- **TSAN gate на Linux**: clang `-fsanitize=thread` на
  concurrency-сюите — 0 reported races (gate в 83.4.5.6).

### Производительность

- **Stress soak**: 10⁶ spawn + 10⁵ park-wake + 10⁴ supervised-scopes
  ×100 fibers с work-stealing migration — без deadlock/leak.
- **Speedup на multi-core**: `bench/micro/parallel_*` (parallel
  sum/hash/fib):
  - На N=4 ядрах: ≥ 3.0× speedup vs `NOVA_MAXPROCS=1` (≥ 0.75×
    линейного).
  - vs Go equivalent: ≥ 0.85× Go time (не хуже 15% от Go).

### Spec / документация

- spec D-block в `06-concurrency.md` — «M:N default model»: NOVA_MAXPROCS
  resolution order, lazy worker pool, cancel hierarchy semantics,
  detach M:N behavior, handler scoping invariants, cross-runtime
  comparison table.
- D50 amendment (если detach semantics меняются) — Plan 83.4.5.2 Ф.X.
- D93 уже amend'нут (Plan 83.4.1 Ф.5).
- `docs/plans/README.md`: Plan 83.2 ✅, Plan 83.4 ✅, Plan 83 ✅.
- `[M-83.2-supervised-mn-bugs]` → ✅ ЗАКРЫТО.

### Regression budget

- 0 регрессий полного `nova test` clang.
- 0 регрессий MSVC suite (1049/16/56 baseline от Plan 82 followup).
- 0 регрессий Linux suite (когда автоматизирован).

## 6. Risk audit

| Риск | Mitigation |
|---|---|
| Ф.0 enumeration может показать что моя 6-категорийная декомпозиция неверна | sub-plan'ы стабильны на architectural-уровне; Ф.0 может объединить категории либо добавить новые (например «migration race» если deeper investigation покажет) — план легко refactor'ится |
| Cancel-wake-all (83.4.5.1) — требует обхода всех parked fibers; mutex contention | `nova_sched_wake` идемпотентен (Plan 22 Ф.3 §D93); wake-all через `sched_state.parked` iteration — O(scope.count) под scope-mutex, не hot path |
| Detach M:N semantics (83.4.5.2) — может потребовать D50 amendment | если ABI change существенный, открыть `Plan 83.4.5.2.1` с D50 spec обновлением; semantics уточняется в нём |
| Handler nested corner cases (83.4.5.4) — algebraic effects глубоко в codegen; ошибка регрессирует прод-код | incremental: каждый corner case с targeted-тестом; снапшот тестов effects suite до/после; full `nova test` после каждого |
| Test cleanup (83.4.5.3) — risk потерять валидное coverage переписав на set-equality | для тестов которые ДОКАЗЫВАЛИ parallelism (encoded log) — переписать в `bench/micro/parallel_*` (отдельный workload), а в `nova_tests/` оставить только correctness-set-equality |
| Speedup ≥0.85× Go не достижим из-за Boehm GC overhead | измерить GC-вклад через `--gc malloc`; если разрыв > 30% — открыть Plan 27.2 concurrent GC; не блокировать closure |
| TSAN на Linux недоступен из текущего Windows-окружения | honest-defer с инструкцией для Linux-сессии; documented в acceptance |
