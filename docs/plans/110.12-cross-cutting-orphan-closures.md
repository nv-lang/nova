### 110.12.1 — MultiError payload `str` → `any` migration

**Scope:**
Plan 110.4 Ф.6.2 ship'нул `MultiError` с `primary` / `suppressed` payloads
typed как `str` (compatibility shim). Real spec D193 says `any`. Migrate.

**Pre-implementation:** Verify `any` codegen maturity.
- Check if `any` type works в struct fields (current?).
- Check if `any` cross-module mangling works.
- Check if `any` payload deref + downcast works.

**Implementation:**
- `std/prelude/errors.nv`: `MultiError` struct field type `str` → `any`.
- Migration tool: `nova fix --migrate-multierror-any` для existing user code
  patterns `me.primary as str`.
- Spec D193 amend — status changed to ACTIVE without `str` shim.

**Tests:**
- POS: multierror_any_payload_ok.nv — `me.primary` is `any`,
  `me.primary as MyError` works.
- POS migration: existing fixtures still PASS post-migration.
- NEG: `me.primary as Unrelated` rejected (downcast safety).

**Acceptance:**
- A110.12.1.a: MultiError fields typed `any` (per spec).
- A110.12.1.b: Existing user code migrates с auto-fix tool.
- A110.12.1.c: Zero regression на error-handling fixtures.

**Estimate:** ~1-2 dev-day (depends on `any` codegen state).

**Dependencies:** mature `any` codegen — gated. **Pre-check phase
recommended** before committing к full implementation.

### 110.12.2 — `#[run_on_abort]` attribute + platform signal handling

**Scope:**
Plan 110.4 Ф.8.9 originally deferred attribute design для finalizers
на abort/SIGKILL. D195 R8 currently states «abort/SIGKILL не fires
finalizers (OS limitation)». Этот sub-plan opens new design space:
**graceful-shutdown finalizers** для controlled abort paths.

**Design considerations:**
- **POSIX:** SIGTERM (graceful) handler — installs `nova_app_run_finalizers`
  как signal handler safely (async-signal-safe constraints).
- **Windows:** SetConsoleCtrlHandler для CTRL_C_EVENT / CTRL_CLOSE_EVENT —
  installs equivalent.
- **SIGKILL / TerminateProcess:** STILL не firable (OS-enforced) — explicit
  documentation, не feature gap.

**Implementation:**
- Attribute parse: `#[run_on_abort] fn shutdown_logger() -> () { ... }`.
- Codegen: register fn в global `nova_abort_finalizer_stack` at startup.
- Runtime: signal handler walks stack LIFO (async-signal-safe — store
  fn ptr + no allocation в handler).
- Cross-platform: POSIX sigaction + Windows SetConsoleCtrlHandler.

**Tests:**
- POS: run_on_abort_sigterm_ok.nv — POSIX-only fixture (gated). Spawn
  child process, send SIGTERM, verify finalizer ran.
- POS Windows: run_on_abort_ctrl_c_ok.nv — child process, simulate
  CTRL+C, verify finalizer.
- Documentation: D195 R8 amend — clarifies SIGKILL vs SIGTERM distinction.

**Acceptance:**
- A110.12.2.a: `#[run_on_abort]` attribute parses + codegens.
- A110.12.2.b: SIGTERM (POSIX) fires registered finalizers.
- A110.12.2.c: CTRL_C_EVENT (Windows) fires registered finalizers.
- A110.12.2.d: SIGKILL documented as not-firable (по design).

**Estimate:** ~2-3 dev-day (cross-platform + async-signal-safe correctness).

**Dependencies:** None — но **high implementation risk** due к platform
signal handling subtleties. Recommend separate dev-day per platform.

### 110.12.3 — `Time.sleep(0)` × multi-fiber scheduler race investigation

**Scope:**
Investigation marker — Plan 110.6.5 noted `Time.sleep(0)` inside body
× multi-fiber может cause test-runner timeout. Root cause unclear.
Этот sub-plan = **structured investigation** с deliverables.

**Investigation phases:**
1. **Reproduce:** create minimal test fixture с deterministic timeout.
2. **Diagnose:** instrument scheduler (`nova_sched_park` / `nova_sched_wake`)
   с counter dumps. Identify where stall occurs.
3. **Hypothesize fix:** based on diagnose data — possible candidates:
   - Yield не goes back к scheduler в edge case.
   - Wake CAS race с zero-budget shield.
   - Test runner blocking SIGCHLD-style wait.
4. **Implement fix or document «WONTFIX»:**
   - If actionable: ship fix + regression test.
   - If platform-level limit: document scenario + workaround pattern.

**Estimate:** ~1-2 dev-day investigation; +1-2 dev-day fix (if found).

**Dependencies:** Scheduler-level access (Plan 83.x family).

**Risk:** High — scheduler changes могут regression concurrency tests.
Mitigation: extensive baseline regression check post-fix.

### 110.12.4 — Quantitative benchmark infra (Plan 110.8.5)

**Scope:**

110.8.5 was extracted как «quantitative bench results» — number-driven
verification of cancel-shield + consume-scope perf budgets.

**Implementation:**

**Acceptance:**
- A110.12.4.a: 3 bench fixtures land + run в CI.
- A110.12.4.b: Baseline numbers recorded + tracked.
- A110.12.4.c: Regression gate fires on > 10% degradation.

**Estimate:** ~1-2 dev-day (mostly fixture writing + CI wiring).

**Dependencies:** Plan 110.8.4 CI infra closure.
