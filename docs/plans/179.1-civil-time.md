## 1. Зачем

После 179 у Nova есть **instant** (`Timestamp` — Unix-ns) и **interval** (`Duration`), но **нет «человеческого» календаря**:
год/месяц/день, час, таймзоны, форматирование/парсинг. Нельзя ни вывести `Timestamp` как `2026-06-22T14:00:00Z`, ни
распарсить дату, ни прибавить «1 месяц». Это большой пласт реальных задач (логи, API, БД, расписания) и снимает
зависимость из 179 (full-datetime `@display`).

## 1a. Где Nova может быть ЛУЧШЕ peers (differentiators — в доку)

- **Compile-time `Period` vs `Duration` + DateBased/TimeBased split** → `date.plus(duration)` / «add hours to a Date» —
  **compile-error**. Это чинит **худший задокументированный изъян Temporal** (single unified `Duration`, узнаёшь что она
  «календарная» только когда арифметика бросает в рантайме) и отсутствие `Period` в Go (только 3 голых int в `AddDate`).
- **Type-separated Plain vs Zoned** → «хранить wall-clock как момент» (корень почти всех Go-date-багов, единый `time.Time`)
  становится **ошибкой типа**, а не тихим рантайм-багом.
- **Value-records** (`type Date value { epoch_day i64 }`, D215/172.4): stack, zero-GC, copy, структурное `==` — строго
  лучше heap-объектов Java.
- **4-way DST-disambiguation как Result-значение** (не throw, не silent) — превосходит java.time (только earlier/later +
  ручной `getValidOffsets`), Go (silent), kotlinx (silent earlier), chrono (None путает gap с load-error).
- **Effect-handler clock** → `ZonedDateTime.now(tz) => Time.timestamp().to_zoned(tz)` мокается детерминированно (Java —
  awkward Clock-DI, Go — никак).
- **Result + структурный `DateError`** вместо throw-everything (java.time `DateTimeException` непрозрачен) / `Invalid Date` NaN (JS).

## 3. Модель типов (D319) — value-records, immutable, proleptic Gregorian

**Type-ladder (нормативный инвариант D319): Plain (нет зоны, *неоднозначно*, НЕ точка на оси) → Offset (точка, фикс.
сдвиг, без DST-правил) → Zoned (точка, IANA-rule-aware).** Plain → `Timestamp` **только** через явный `Offset`/`TimeZone`
+ disambiguation, и это **fallible**. Никаких неявных дефолтных зон.

```nova
type Date        value { ro epoch_day i64 }                 // дни от 1970-01-01 (proleptic Gregorian); range §3.0-Q4
type TimeOfDay   value { ro nanos_of_day i64 }              // 0 .. 86_399_999_999_999
type DateTime    value { ro date Date, ro time TimeOfDay }  // plain, без зоны
type Offset      value { ro seconds i32 }                   // фикс. UTC-сдвиг, ±64800 (±18h)
type ZonedDateTime value { ro dt DateTime, ro offset Offset, ro zone TimeZone }  // момент: offset резолвит, zone — для round-trip
type Period      value { ro years i32, ro months i32, ro days i32 }   // КАЛЕНДАРНАЯ величина (≠ Duration)
type YearMonth   value { ro year i32, ro month i32 }        // billing-периоды
type MonthDay    value { ro month i32, ro day i32 }         // годовщины (Feb-29 валиден как анниверсари)
type Weekday | Mon | Tue | Wed | Thu | Fri | Sat | Sun     // ISO 1..7 (D52 sum-type)
type Month   | Jan | Feb | Mar | Apr | May | Jun | Jul | Aug | Sep | Oct | Nov | Dec  // 1..12
type TimeZone | Utc | Fixed(Offset) | Iana(IanaZone)       // Utc/Fixed = value; Iana несёт правила (ref)
type IanaZone reference { ro id str, ro rules ZoneRules }  // именованная зона + transition-таблица (Ф.4)
```

**`TimeZone` — sum-type, не flat value-record:** `Utc`/`Fixed(Offset)` — value (zero-weight); `Iana(IanaZone)` несёт
правила → `IanaZone` = reference-тип (handle к `ZoneRules` transition-таблице). **Полная реализация (без MVP-gating):**
именованные IANA-зоны (`TimeZone.try_from("Europe/Moscow")`) — в scope сразу (**Ф.4**), вместе с реальным DST gap/overlap.
`ZonedDateTime.offset` всегда хранит резолвнутый сдвиг (round-trip-safe даже при смене правил), `zone` — для лосслесс
round-trip имени (RFC-9557 `[zone]`).

**`OffsetDateTime` отдельным типом НЕ вводим** (jiff-подход): `ZonedDateTime` с `zone = Fixed(Offset)` это покрывает —
меньше surface, единый round-trip.

## 3a. Методы (эскиз; финал при реализации)

**Naming-конвенции** (по коду std, не java-измы): конструктор = `X.new(...)` (как `Vec.new`/`Barrier.new`; `.of` только у
`Vec` для variadic); fallible-вариант = `try_`-префикс → `Result` (как `try_parse_int`/`try_lock_for`); ошибки парсинга =
`Parse<X>Error` (как `ParseIntError`/`ParseJsonError`); конверсия = `from`/`try_from`/`TryInto` (D77/Plan 85). Операторы
арифметики **trap-on-overflow** (D317), `try_*` — Result-вариант.

- **`Date`:** `Date.new(y i32, m Month, d i32) -> Result[Date, DateError]` (validate-by-default), `Date.today(tz TimeZone) Time -> Date`,
  `Date.from_epoch_day(d i64) -> Date`, `@year/@month/@day/@weekday/@day_of_year/@epoch_day`, `@plus(Period)/@minus(Period) -> Date`
  (**оператор `+`/`-`, trap на range-overflow**) + `@try_plus(Period)/@try_minus(Period) -> Result[Date, DateError]` (структурная ошибка),
  `@minus(other Date) -> Period` (overload) **и** `@days_until(other Date) -> int`, `@is_leap_year`, `Date.MIN`/`MAX`,
  ISO-week `@iso_week()/@iso_week_year()`, адъюстеры `@last_day_of_month/@next(Weekday)/@nth_weekday(n,Weekday)/@first_day_of_month`,
  `@compare`, ISO `@to_iso()` / `Date.parse_iso(s) -> Result[Date, ParseDateTimeError]`, `Date.from_normalized(y,m,d) -> Date` (явный opt-in).
- **`TimeOfDay`:** `TimeOfDay.new(h,m,s,ns) -> Result[TimeOfDay, DateError]`, `@hour/@minute/@second/@nano`,
  `@plus(Duration) -> (TimeOfDay, days_overflow i64)` (явный day-carry, не молча), `@compare`, `TimeOfDay.MIDNIGHT`.
- **`DateTime`:** `DateTime.new(date Date, time TimeOfDay) -> DateTime` / `Date @at(time TimeOfDay) -> DateTime`, `@date/@time`,
  `@plus(Period) -> DateTime` (через Date) + `@try_plus -> Result`, `@plus(Duration) -> DateTime` (инстант-семантика — doc
  wall-vs-elapsed), `@to_zoned(tz TimeZone, disamb Disambiguation) -> Result[ZonedDateTime, DateError]`, ISO.
- **`ZonedDateTime`:** `ZonedDateTime.now(tz TimeZone) Time -> ZonedDateTime` (`=> Time.timestamp().to_zoned(tz)`),
  `@to_timestamp() -> Result[Timestamp, DateError]` (Err вне ±292y; также impl `TryInto[Timestamp]` → `@try_into()`),
  `@offset/@zone/@datetime`, `@with_zone(tz, disamb)`, date-арифметика сохраняет wall-time / time-арифметика — elapsed
  (D319 асимметрия), RFC-3339/RFC-9557 round-trip.
- **`Timestamp` (из 179):** `+@to_zoned(tz TimeZone) -> ZonedDateTime` (инфаллибельно при in-range), `+@to_rfc3339() -> str`,
  `+Timestamp.parse_rfc3339(s) -> Result[Timestamp, ParseDateTimeError]`, full-datetime `@display` (**закрывает зависимость 179**).
- **`Period`:** `Period.new(y,m,d) -> Period`, `Period.from_weeks(w) -> Period` (days=7w), `@plus/@minus`, `@normalize() -> Period`
  (bare-verb non-mut → новое значение, как `str @trim_ascii`; **months↔years ТОЛЬКО**, days НИКОГДА не сворачиваются),
  `Period.between(Date,Date) -> Period`, ISO-8601 `PnYnMnD` `@to_iso()`.
- **`Weekday`/`Month`:** `@number` (1-based), `@next/@prev`, `Into[str]`, `Month @length(leap bool) -> i32`.

## 3b. `Period` vs `Duration` + DateBased/TimeBased (D320)

Два **тип-различных** amount: `Duration` (точный ns, из 179) и `Period` (календарный y/m/d). **Никакой неявной коэрсии.**
Enum kind для арифметики: `DateBased`(days/months/years) vs `TimeBased`(hours/…/ns). Enforcement **на уровне типов** →
`Date.plus(Duration)` / «add hours to a Date» = **compile-error** (бьёт Temporal runtime-RangeError). Правила приёма:
`Date` принимает `Period`, **отвергает** `Duration`; `TimeOfDay` — наоборот; `DateTime`/`ZonedDateTime` — оба. Wall-vs-elapsed:
`Zoned.plus(Period(days:1))` через DST даёт **ту же wall-time** (23/25h elapsed); `Zoned.plus(Duration(24h))` — фикс. 24h
(сдвигает wall). Документировать.

## 3c. DST-disambiguation + OffsetConflict + parse (D321)

```nova
type Disambiguation | Compatible | Earlier | Later | Reject   // default Compatible
type OffsetConflict | Reject | Use | Prefer | Ignore          // default Reject (для explicit-offset строк)
```
`DateTime.to_zoned(tz, disamb)`: gap → (`Compatible`) push-forward / (`Reject`) `Err`; overlap → (`Compatible`) earlier /
`Earlier`/`Later` явно. Ambiguity surface как значение (`@is_gap()/@is_overlap()` на промежуточном resolvable). **НЕ путать
gap с load-error** (warт chrono). Parse zoned-строки: `OffsetConflict` решает рассогласование сохранённого offset с правилами
зоны. `Utc`/`Fixed` всегда unambiguous (DST нет); **именованные IANA-зоны (Ф.4) дают реальный gap/overlap** → 4-way Disambiguation работает не вхолостую.

## 3d. Ошибки — структурные (D321)

```nova
type DateError
    | InvalidField(field FieldKind, given i64, min i64, max i64)   // Feb 30, month 13, hour 25
    | RangeOverflow(kind str)                                       // civil↔Timestamp вне ±292y
    | Ambiguous(kind str)                                           // Reject на gap/overlap
type ParseDateTimeError
    | FormatMismatch(position int, expected str)
    | InvalidValue(position int, field FieldKind, given i64)
type FieldKind | Year | Month | Day | Hour | Minute | Second | Nano | OffsetSec | Weekday
```
(shape — time-crate `ComponentRange` + java.time `getErrorIndex`.) Конструкторы/parse возвращают `Result`; **нет
default-panicking конструктора** (урок: chrono deprecated `from_ymd`).

## 3e. Алгоритмы

Date↔epoch-day — **Howard Hinnant `days_from_civil` / `civil_from_days`** (branch-light, overflow-safe; ровно как chrono).
Leap: `÷4 кроме веков, если не ÷400` (2000 да, 1900 нет). `day_of_week` из epoch-day по модулю. ISO-week: yyyy-Www-D,
**week-based-year ≠ calendar-year** у границ Jan1/Dec31 — задокументировать остро (java.time `Y` vs `y` footgun).

  (читать TZif из `/usr/share/zoneinfo`, `/etc/localtime` для Local) + **opt-in embedded versioned snapshot** (build-flag,
  ~450KB, **обязательный fallback на Windows** + CLDR `windowsZones`-map) + raw-TZif-bytes loader + `$ZONEINFO` override;
  **expose tzdb-version**. Здесь **реальный gap/overlap** делает 4-way `Disambiguation` (Ф.3) рабочим, а не вхолостую.
  ⚠ **Самая тяжёлая фаза** (TZif-парсер + embed-данные + Windows-mapping); см. §10 (фоновые агенты, батчи). DEP: Ф.3.
- **Ф.5 — Format/parse (ISO + RFC-3339 + кастомный pattern).** `Timestamp.to_rfc3339`/`parse_rfc3339`, `Date`/`TimeOfDay`/`DateTime`
  ISO; **RFC-9557 `[zone]`-bracket** (лосслесс round-trip именованной зоны); **strict-default parser** + `OffsetConflict`; `:60`→clamp;
  **кастомный pattern-формат — type-safe builder DSL** (field-директивы + литералы + optional-секции; **НЕ** strftime-as-primary,
  **НЕ** Go reference-layout; опц. CLDR/Unicode-pattern мост); full `@display`/`@debug` для civil **и** `Timestamp`
  (**закрывает зависимость 179 full-datetime `@display`**). Использует [91.18](91.18-str-unicode-api.md). DEP: Ф.1-Ф.4.
- **Ф.6 — тесты + spec/docs.** §7 pos+neg (вкл. реальные DST gap/overlap на именованных зонах); D319-321 amend; новый
  `docs/datetime.md` (модель + «Nova↔java.time» + §1a + wall-vs-elapsed). DEP: all.

**Ортогональные оси — вне scope civil-time-ядра (НЕ MVP-cut, отдельные концерны):** **локализация** (имена месяцев/дней на
языках) — i18n/CLDR-locale-данные, отдельная подсистема (даже Temporal делегирует в `Intl`); **non-Gregorian-календари**
(Japanese/Hijrah/…) — явный design-reject (Q10; java.time/Temporal тоже default ISO); **cron validation-bridge** (`Cron.matches`
валидирует day_month/month против реальных длин месяцев) — изменение experimental-потребителя, отдельный followup.

## 7. Тесты (pos + neg; раскладка `nova_tests/civil179_1/{,neg/}`)

Раскладка как в 179 (pos standalone; neg `module neg.<name>` + `EXPECT_COMPILE_ERROR`); классификация по маркеру.

**pos:** `Date.new` валидные; `@plus(Period)` clamp (`feb_clamp`: Jan31+1mo=Feb28); leap (`leap_feb29`: 2020-02-29 Ok,
2021-02-29 Err); epoch-day round-trip (Hinnant); `@weekday`/`@day_of_year`/ISO-week (вкл. week-year≠calendar-year граница);
`DateTime.to_zoned(Fixed)` + `@to_timestamp` round-trip; **`iso_roundtrip`** (parse(to_iso(x))==x на каждый тип); RFC-9557
`[zone]`-bracket round-trip; `Period.between`; `YearMonth`/`MonthDay` (Feb-29 анниверсари); `ZonedDateTime.now(tz)` под mock
`Time`-handler (детерминизм); `:60`→`:59` clamp на parse. **IANA (Ф.4):** `tz_load` (`TimeZone.try_from("Europe/Moscow")` Ok,
unknown → Err); **`dst_gap_named`** (America/New_York spring-forward 02:30 → push 03:30, Compatible); **`dst_overlap_named`**
(fall-back → earlier offset); `dst_roundtrip` (RFC-9557 `[zone]` лосслесс через transition); кастомный pattern round-trip (builder DSL).

**neg (`EXPECT_COMPILE_ERROR`):** **`period_not_duration`**: `date.plus(some_duration)` → type-error (D320);
`time_of_day.plus(period)` → type-error; «add hours to a Date» → type-error.
**neg (`DateError`/runtime):** `feb30_rejected` (Feb30→`InvalidField`); `month13`/`hour25`; `range_292y` (civil→Timestamp вне
±292y → `RangeOverflow`) + один tick за границей; `dst_reject` (Reject на gap/overlap → `Ambiguous`); parse-fail
(`FormatMismatch`/`InvalidValue` с позицией); civil-арифметика overflow → trap (не wrap).

**контрактные:** структурное `==`/`@compare` на value-records; value-const (`Date.MIN/MAX`, `TimeOfDay.MIDNIGHT`)
компилируются; `@display` ASCII (нет байтов >0x7F); большие sweeps (leap-годы, DST-таблицы) — **в репо, но вне дефолт-регресса**
([[feedback-large-tests-stored-not-in-regress]]).

## 8. Критерии приёмки

0. **🔴 ОБЯЗАТЕЛЬНО: «без упрощений, как для прода».** Ни одного «решим потом» на критическом пути, ни silent-normalize, ни
   untested-поведения; каждая behavior-change закрыта pos+neg + аргументом звучности.
1. Type-ladder: Plain (`Date`/`TimeOfDay`/`DateTime`) / `Offset` / `ZonedDateTime` реализованы как **value-records**;
   `Weekday`/`Month` 1-based enums; `YearMonth`/`MonthDay` есть; нет неявного Plain→Timestamp (compile-error/Result).
2. **`Period` тип-различён с `Duration`**; `date.plus(Duration)` / «hours to Date» — **compile-error** (neg-fixture).
3. **Validate-by-default**: `Date.new(2024,Feb,30)`→`DateError` (не panic, не silent-normalize); та же строгая модель на parse.
4. **calendar-арифметика**: month-add clamp biggest-unit-first (Jan31+1mo=Feb28/29); `checked_*`/`saturating_*`; **trap-on-overflow**
   (не wrap, наследует D317); non-invertible/non-associative задокументировано + fixture.
5. **DST**: `Disambiguation` 4-way (Compatible default) surface как Result-значение; `Reject`→`Err`; `OffsetConflict` 4-way; не путать gap с error.
5a. **IANA в scope (Ф.4)**: `TimeZone.try_from(name)` грузит `ZoneRules` (OS-tzdata-first + embedded fallback + Windows CLDR); **реальный**
   DST gap/overlap на именованных зонах резолвится per `Disambiguation`; tzdb-version доступна; named-zone round-trip через RFC-9557 `[zone]`.
5b. **Кастомный pattern-формат (Ф.5)**: type-safe builder DSL (НЕ strftime/Go-layout); round-trip pattern-формата.
6. **ISO-8601/RFC-3339 round-trip** per-тип (`parse(to_iso(x))==x`); RFC-9557 `[zone]`-bracket для zoned (offset-only теряет zone-identity); strict-parse; `:60`→clamp.
7. **Range**: civil↔Timestamp вне ±292y → `Result::Err` (не clamp/wrap); MIN/MAX подобраны так, что `Timestamp+Offset→civil` инфаллибелен; fixture на границе ±292y.
8. **`now()` мокабелен**: `ZonedDateTime.now(tz)` через `Time.timestamp()`-handler; `Time.local_offset()` мокабелен.
9. full-datetime `@display`/`@debug` (закрывает зависимость 179); structured `DateError`/`ParseDateTimeError`.
10. Полный регресс зелёный (батчами <10мин); leap/DST-sweeps в репо вне дефолт-сэмпла.
11. spec: D319/320/321 написаны; `docs/datetime.md`; §1a differentiators зафиксированы.
