# Plan 57.E.3 — AI-driven regression interpretation (design sketch)

> **Status:** design-sketch (not implemented в этой phase).
> **Estimate when implemented:** ~300 LOC, 1-2 dev-days + recurring
> API costs.
> **External dependencies:** LLM API (Anthropic/OpenAI) + tokens budget.

## Motivation

`nova bench diff` сейчас сообщает **что** изменилось (+8.3% bench X,
p < 0.001), но не **почему**. Developer should вручную сопоставлять
с git diff между commits и угадывать причину.

LLM consumer JSON diff может суммировать в natural language:
- "Likely cause: mono-pass change в commit abc123 added 2 extra type
  substitution allocations per iteration; observable в +200ns mono time."
- "Noise spike — distribution shape changed, but no clear single-source
  attribution. Re-run recommended."
- "Fix: revert the binary search inversion в hashmap.nv:142
  (mentioned в commit message as 'optimization' but increased branch
  mispredict)."

## Architecture

```
git diff <base>..<new>     ┐
nova bench diff JSON       ├──► LLM prompt builder ──► LLM API
recent commit messages     │                              │
flame graph (если есть)    ┘                              ▼
                                                  JSON response
                                                  + suggestions
                                                       │
                                                       ▼
                                                Terminal display
                                                + diff.md annotation
```

## API sketch

```
# Config: NOVA_AI_API_KEY env or ~/.nova-ai.toml
[ai]
provider = "anthropic"  # or "openai"
model = "claude-opus-4-7"
max_tokens = 4000
include_git_diff = true   # default true
include_flame = false     # default false (large)

# Command:
nova bench diff baseline.json new.json --explain

# Output (verbose):
[diff table as usual]
                                          geomean delta: +2.3%

═══ AI interpretation ═══
Confidence: high (statistical + code analysis)

Most likely cause:
  hashmap_insert/n=10000 regression (+15.5%, p<0.001) correlates с
  commit abc1234 "refactor: HashMap.insert" which removed empty-bucket
  check before probing — каждая insert теперь делает 1 extra hash
  comparison.

Recommended actions:
  1. Revert the empty-check removal (hashmap.nv:78).
  2. If intentional (clearer code), accept regression + update
     bench.toml threshold для hashmap_insert/* к 20%.

Confidence: medium (no flame graph data — install samply для
  improved attribution).

Tokens used: 2,341 / 4,000 budget.
```

## Implementation plan

1. **Phase 1: prompt construction (~100 LOC).**
   - bench/ai.rs: build context from diff JSON + git log + commit msgs.
   - Prompt template: structured XML с `<diff>`, `<commits>`, `<question>`.

2. **Phase 2: LLM API integration (~120 LOC).**
   - HTTP client (reqwest? — adds dep; manual via std::net? — sufficient
     для simple JSON POST).
   - Backoff/retry для rate limits.
   - Per-provider адаптеры (Anthropic vs OpenAI API shape).

3. **Phase 3: output rendering (~50 LOC).**
   - Terminal: structured display с sections.
   - Markdown: append к `diff.md` для PR comment.

4. **Phase 4: cost guards (~30 LOC).**
   - `--max-tokens N` flag (default budget).
   - `--dry-run` prints prompt без API call (estimate cost).
   - Log token usage per invocation.

## Risks

- **R1: Hallucination** — LLM может invent root causes. Mitigation:
  display "Confidence: low/medium/high" based on signals (clear single-
  source vs scattered changes).
- **R2: API cost** — production CI с per-PR explain runs может cost
  $1-5/PR. Mitigation: opt-in `--explain` (not default); per-PR budget
  cap.
- **R3: Privacy** — git diff sent к external API may contain confidential
  code. Mitigation: `--explain` opt-in + warning; document data handling.
- **R4: Vendor lock-in** — different providers различные API. Mitigation:
  trait `AiProvider` + per-vendor implementations; switchable via config.

## Acceptance criteria (when implemented)

- [ ] `nova bench diff ... --explain` produces structured AI output.
- [ ] Token budget enforced (no runaway costs).
- [ ] At minimum 2 providers supported (Anthropic + OpenAI).
- [ ] Privacy warning printed на first use.
- [ ] Markdown output suitable для PR comment.

## Alternative considered

**Static heuristic** — rule-based "what changed + which benches
correlate" without LLM. Pros: no API cost, deterministic. Cons:
brittle, low-value insights, требует maintained rule database.
LLM подход scales к novel patterns.

## When to implement

После Phase D real-world usage shows что developer тратит significant
time interpreting diffs manually. Если diff format уже readable + git
log integration sufficient — defer indefinitely.
