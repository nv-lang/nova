## 2. Sub-plan details

### 2.1 Plan 123.5.4 (V5.4) — explain deep-walk

`analyze_fn_for_explain` currently scans only top-level `_at_*` lets
в fn body's Block. V1.2 nested-region caches inject lets inside
nested blocks (if/while/match arms) — these не surfaced by V5
code-lens / hover / `nova check --telemetry-cache`.

**Approach:** generalize `analyze_fn_for_explain` (and
`emit_explain_report_for_file` if needed) to recursively walk ALL
top-level + nested blocks, collecting every `_at_*` let regardless
of depth.

**Closes:** `[M-123.1.2-explain-deep-walk]`.

### 2.2 Plan 123.7.7 (V7.7) — chain receiver IPA

V7.5 detects direct `@F.method()` (receiver `Member{SelfAccess, F}`).
V7.7 extends к chains `@a.b.method()` (receiver
`Member{Member{...{SelfAccess, a}, b}, ..}, m}`).

**Approach:** generalize `call_recv_self_field` to return
`Option<Vec<String>>` (chain path), then refine
`expr_contains_invalidating_call_for` so calls на `@<chain>` survive
caches of fields outside `@<chain>` root.

**Closes:** `[M-123.7.5-chain-receiver]`.

### 2.3 Plan 123.2.1 (V2.1) — loop-body LICM coordination

V1.2 nested-region caching inside loop bodies uses single-iteration
read counts. Loop bodies execute N times → real cost is N×reads. V2
LICM hoists из loops independently. Coordinated cost model needed
для not-double-counting / not-undercounting decisions.

**Approach:** add loop-iteration weight в read counting. For reads
inside loop bodies, multiply by `cfg.loop_iters_estimate` (env-
configurable, default 8 — matches V6.2 `NOVA_FC_LOOP_ITERS`). Apply
in `count_field_reads_in_*` recursion via context flag.

**Closes:** `[M-123.1.2-loop-body-licm-coordination]`.

### 2.4 Plan 123.7.6 (V7.6) — same-field ref-type IPA

V7.5 conservatively invalidates own-field cache (`fname == recv_field`)
across `@F.method()`. For reference-type fields (`[]T`, `String`,
`Map`), mutation methods modify referenced object но не the field's
slot itself — cache should survive.

**Approach:** integrate с TypeDecl. Look up `recv_field`'s declared
type; if it's a known reference type (Array/String/Map/Set/...) →
mark `@F.method()` non-invalidating even for own-field cache.

**Closes:** `[M-123.7.5-same-field-ref-type]`.

### 2.5 Plan 123.4.4 — codegen chain-root temp

`@buf.push(a).push(b).push(c)` lowers к 3× `nova_self->buf` due к
recursive `emit_expr(obj)` propagation в `emit_c.rs:19567`. The
**actual fix** for WriteBuffer.@write_char pattern.

**Approach:** new AST pre-pass `chain_norm.rs` adjacent к callnorm.
Detect fluent chains (Call{Member{Call{...}}}) where root receiver is
non-trivial (`@F`, `@a.b`, etc.) AND chain depth ≥ 2. Rewrite к
Block с let-binding chain root once.

Pattern:
```
e.m1(a).m2(b).m3(c)
  → { let _chain_<N> = e; _chain_<N>.m1(a); _chain_<N>.m2(b); _chain_<N>.m3(c); _chain_<N> }
```

Scope V1: only when each method returns `@` (self) — known fluent
methods (builtin push/append/extend_from/copy_from + user fns с
`-> @` ret type).

**Closes:** `[M-123.4.4-codegen-fluent-chain-root-temp]`.
