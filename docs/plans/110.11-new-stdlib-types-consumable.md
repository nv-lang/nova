## Sub-plans (separate plan-docs created per sub-task)

### 110.11.1 — std/fs.File Consumable

**Scope:**
- Type design: `File` struct с handle + path + mode.
- Methods: `File.open(path, mode)`, `read`, `write`, `seek`, `close`.
- Consumable impl: `@on_exit(...)` → close handle.
- FFI: libc fopen/fread/fwrite/fclose (POSIX) + Windows CreateFile/ReadFile/WriteFile/CloseHandle.

**Estimate:** ~2-3 dev-day (FFI + cross-platform).

**Dependencies:** Plan 118 (typed pointers для FILE*/HANDLE) optional but
recommended; Plan 115 FFI cookbook references.

### 110.11.2 — std/bufio.BufReader / BufWriter Consumable

**Scope:**
- Type design: `BufReader[R]` / `BufWriter[W]` generics over Reader/Writer protocols.
- Methods: `read_line`, `read_until`, `flush`, etc.
- Consumable impl: `@on_exit(Success)` → flush + close; `@on_exit(Failure)` →
  close без flush (data integrity policy).

**Estimate:** ~1-2 dev-day.

**Dependencies:** 110.11.1 (File) для default backing type.

### 110.11.3 — std/db.Transaction Consumable

**Scope:**
- Type design: `Transaction[E]` generic, abstract over driver.
- Methods: `begin`, `commit`, `rollback`, `exec`.
- Consumable impl: `@on_exit(Success)` → commit; `@on_exit(Failure/Cancel)` →
  rollback.

**Estimate:** ~2-3 dev-day (driver abstraction design).

**Dependencies:** Driver layer (SQLite/Postgres) — likely separate plan
or stdlib roadmap decision needed first. Plan 115 examples/ffi/sqlite_mini_ffi.h
provides minimal proof.

### 110.11.4 — std/pool.ConnPool Consumable

**Scope:**
- Type design: `ConnPool[C]` generic с size + acquire/release semantics.
- Methods: `acquire`, `release`, `close_all`.
- Consumable impl: `@on_exit(...)` → close_all (drain pool).
- Threading: Mutex protection (Plan 103.3).

**Estimate:** ~2 dev-day.

**Dependencies:** 110.11.3 (Transaction conn type) OR другой backing type.

### 110.11.5 — std/concurrency.CancelScope Consumable

**Scope:**
- Type design: `CancelScope` struct с deadline + child token registry.
- Methods: `new`, `with_deadline`, `request_cancel`, `is_cancelled`.
- Consumable impl: `@on_exit(...)` → propagate cancel к children.
- Integration: Plan 110 cancel-shield runtime hooks.

**Estimate:** ~2 dev-day.

**Dependencies:** Plan 110.2 cancel-shield runtime (already landed).

### 110.11.6 — Stream[T] + Consumable

**Scope:**
- Type design: `Stream[T]` lazy iterator protocol (pull-based).
- Methods: `next`, `close`, `map`, `filter`, `for_each`.
- Consumable impl: `@on_exit(...)` → release underlying resource.
- Source adapters: from Channel, from File, from network.

**Estimate:** ~3-4 dev-day (design + integration).

**Dependencies:** 110.11.1 (File source) + 110.10.1 (Channel source). Plan 84
**reference was wrong** — corrected here (Plan 84 is relative-imports plan).

## Coordination с Plan 91 (std MVP)

Plan 91 — overall std MVP umbrella. Some sub-tasks этого plan'а могут
overlap с Plan 91 sub-tasks (e.g., std/fs could land via Plan 91 Ф.X).

**Coordination rule:** If Plan 91 closes a type design, Plan 110.11.X
sub-task scope reduces к «add Consumable impl only». If Plan 91 doesn't
plan a type, Plan 110.11.X owns full design.

**Action:** Pre-implementation per sub-plan, check Plan 91 roadmap для
overlap.

## Phases

Каждый sub-plan independent — ship in any order based on user priority
+ external dependencies. Recommended order:

1. 110.11.5 CancelScope (least deps).
2. 110.11.1 File (foundation для bufio/stream).
3. 110.11.2 BufReader/Writer (builds on File).
4. 110.11.6 Stream[T] (builds on File + Channel).
5. 110.11.4 ConnPool (generic, can land без Transaction).
6. 110.11.3 Transaction (needs driver decision).
