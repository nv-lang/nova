## 1. Контекст

nudge добавить `#pure` annotation там, где compiler IPA уже знает
что метод effectively pure. V5.3 закрывает этот gap.

## 2. Изменения

### 2.1 field_cache.rs

Public API: `pure_annotation_candidates(module) -> Vec<(String, String, Span)>`.
Возвращает analytically-pure instance methods, lacking `#pure`,
with empty write-set closure и no effects/concurrent constructs.

### 2.2 nova-lsp/src/server.rs

- `Backend::code_action` extended: pre-diagnostic pass calls
  `compute_pure_annotation_actions(src, params.range)` и appends
  any returned `(insert_range, label)` pairs as QUICKFIX
  CodeActions с `is_preferred: false` (suggestion, not corrective).
- Public helper `compute_pure_annotation_actions(src, range) ->
  Option<Vec<(Range, String)>>`.
- Internal helper `position_to_byte_offset_via_starts`.
- Insertion point = column 0 of `fn` line. New text = `"#pure\n"`.

### 2.3 Tests (nova-lsp/tests/field_cache_lens.rs)

3 new test cases — total 10:
- `v53_pure_annotation_suggested_for_analytic_pure_fn` — suggestion
  fires for `Box.sum` (ro fields only).
- `v53_pure_annotation_skipped_outside_fn_span` — no suggestion
  when range is on `type` decl.
- `v53_pure_annotation_insertion_at_fn_line_start` — insertion at
  column 0 of fn line.
