# Plan 42.13: `internal/` extended naming (D29 rev-3.1)

> **Создан 2026-05-14, ЗАКРЫТ 2026-05-14.**
>
> **СТАТУС:** ✅ Реализован. Manifest special-case для `internal/`
> устраняет naming collision.

---

## Проблема

Rev-3 правило `parent.X` (D29) — declaration всегда 2 segments. Для
single-file внутри `internal/`:

```
admin/internal/token.nv     → module internal.token
billing/internal/token.nv   → module internal.token   // COLLISION
```

Оба объявляют одинаковый `internal.token`. Резолвится по filesystem
path, но в error messages / debugging — не видно владельца. При росте
stdlib (Plan 18 P0 — каждый модуль имеет свой `internal/`) — visual
collision повсеместна.

---

## Решение: extended `parent.X` для `internal/`

Если path содержит сегмент `internal`, declaration =
**`owner.internal.target`** (3 segments):

| Файл | owner | target | declaration |
|---|---|---|---|
| `admin/internal/token.nv` (single) | admin | token | `admin.internal.token` |
| `billing/internal/token.nv` (single) | billing | token | `billing.internal.token` |
| `admin/internal/codec/enc.nv` (peer of `codec/`) | admin | codec | `admin.internal.codec` |

**Edge case:** если `internal/` САМА folder-module (peers прямо в
`internal/`, не в sub-folder) — declaration = `owner.internal`
(2 segments, без дублирования `internal.internal`):

| Файл | declaration |
|---|---|
| `folder_internal/internal/encode.nv` (peer) | `folder_internal.internal` |
| `folder_internal/internal/token.nv` (peer) | `folder_internal.internal` |

- **owner** = directory сразу перед `internal`. Если `internal` на
  root level — owner = package name.
- **target** = file basename (single-file) или folder name
  (folder-module peer).

---

## Реализация

`compiler-codegen/src/manifest.rs::expected_module_path_rev3`:

- Перед обычной rev-3 логикой — проверка `parts.iter().position(|s| s == "internal")`.
- Если найдено: owner = `parts[idx-1]` (или package name), target по
  is_folder_module флагу.
- Если `target == "internal"` → 2-segment declaration (edge case).
- Иначе → 3-segment `[owner, "internal", target]`.

~35 LOC special-case.

---

## Миграция

Затронут 1 файл:
- `nova_tests/modules/reexport_facade/internal/impl.nv` — single-file,
  `module internal.impl` → `module reexport_facade.internal.impl`.

`folder_internal/internal/{encode,token}.nv` — НЕ затронуты (они peers
самой `internal/` folder-module → declaration `folder_internal.internal`
не изменилась благодаря edge case).

---

## Что НЕ входит

- Nested `internal/` глубже одного уровня (`X/internal/Y/internal/Z`) —
  берётся **первый** `internal` сегмент. Глубокая вложенность не
  поддерживается (как и обычные sub-modules — через nested folders).
- Rule H enforcement logic (imports.rs) не меняется — он работает по
  **import path** prefix, не declared module name. См. Plan 42.08 Ф.3
  comment.

---

## Acceptance criteria

- `internal/` single-file → 3-segment `owner.internal.target`.
- `internal/` folder-module peer → 2-segment `owner.internal` (edge case)
  ИЛИ 3-segment если `internal/` содержит sub-folder.
- Naming collision между разными модулями устранена на declaration level.
- Existing tests PASS — reexport_facade + folder_internal.
- Regression untouched.
