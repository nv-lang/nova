## Design refinement 2026-06-03 (user discussion finalization)

### Naming + namespace

**`RawMem`** вАФ namespace type (unit-type declaration per spec/02-types.md:233):
- `type RawMem` (no body, no fields) вАФ Nova valid unit type
- Static methods: `RawMem.copy_from[T](...)` style
- Use case: low-level memory operations (driver/embedded/perf-critical)
- Rejected alternatives: `Raw` (too short, semantic vague), `Unsafe` (doubled keyword
  —Б `unsafe { }` block), `Intrinsics` (broad), `Mem` (too narrow вАФ addr_of –љ–µ fits)

### CStr type

**`type CStr(*u8)`** вАФ tuple newtype wrapping null-terminated bytes:
- Plan 115 D52 precedent (canonical FFI handle pattern, zero-overhead)
- C ABI: `const char*` (single pointer, NO header)
- PascalCase user-declared type (lowercase `cstr` would require compiler
  special-case –Ї–∞–Ї `ptr`/`str` вАФ deferred –Ї future plan –µ—Б–ї–Є ergonomic demand)
- Location: `std/ffi/cstr.nv` (–ї–Є–±–Њ `std/prelude/cstr.nv` –µ—Б–ї–Є global-visible)

### Conversions —З–µ—А–µ–Ј D73/D77 From/Into protocols (REFINED 2026-06-03)

**Implementer pi≈°et `try_from` (richest form per D77)** вАФ compiler auto-synthesizes
short `from`/`into` forms.

```nova
// ============================================================================
// CStr вЖР str (Nova-managed allocation, NO raw deref of user-supplied ptr)
// ============================================================================

fn CStr.try_from(s str) -> Result[CStr, CStrError]
// SAFE вАФ validates no embedded NUL, allocates s.len + 1, copies + NUL.
// Compiler auto-synthesizes via D77:
//   CStr.from(s str) Fail[CStrError] -> CStr     (short, throws)
//   s.into() / s.try_into()

#unsafe
fn CStr.from_unchecked(s str) -> CStr
// Skip NUL validation; allocates + copies + NUL.
// Caller promises s has no embedded NUL.
// Rust precedent: CString::from_vec_unchecked (unsafe fn).

// ============================================================================
// str вЖР CStr (raw deref of user-supplied pointer вАФ all #unsafe)
// ============================================================================

#unsafe
fn str.try_from(c CStr) -> Result[str, Utf8Error]
// Allocates new buffer, memcpy + UTF-8 validate.
// Compiler auto-synthesizes via D77:
//   str.from(c CStr) Fail[Utf8Error] -> str      (short, throws)
//   c.into() / c.try_into()

#unsafe
fn str.from_view(c CStr) -> str
// VIEW: no copy, no validation. Returns str with ptr=c.0, len=strlen(c.0).
// Caller promises:
//   (1) CStr buffer outlives the resulting str view
//   (2) bytes ARE valid UTF-8
// Differs –Њ—В str.from_bytes_unchecked (which COPIES + skip validate); from_view
// ALSO skips copy. Rust equivalent: combines CStr::from_ptr + to_bytes +
// std::str::from_utf8_unchecked –≤ –Њ–і–Є–љ step (Nova –љ–µ –Є–Љ–µ–µ—В lifetime tracking).

// ============================================================================
// str вЖТ CStr (view-if-possible OR materialize)
// ============================================================================

fn str @as_cstr() -> CStr
// SAFE primitive. Runtime check: ptr[len] == 0?
//   TRUE вЖТ zero-copy view (CStr wraps str.ptr directly)
//   FALSE вЖТ allocate + copy + NUL (fallback –Ї @to_cstr semantics)
// Validates no embedded NUL (panic –µ—Б–ї–Є –љ–∞–є–і–µ–љ вАФ C-side truncation prevention).
// Resolves D26 ¬ІNul-termination Q-cstring (spec amend 2026-06-03).

fn str @to_cstr() -> CStr
// SAFE вАФ always allocates new buffer + copy + NUL.
// Validates no embedded NUL (panic).

#unsafe
fn str @as_cstr_unchecked() -> CStr
// Skip embedded NUL validation. Caller promises str has no '\0' –≤–љ—Г—В—А–Є [0..len).
// For perf-critical paths where validation cost matters.
```

**D73 auto-derive –і–∞–µ—В `.into()` —Д–Њ—А–Љ—Г –±–µ—Б–њ–ї–∞—В–љ–Њ** –і–ї—П –≤—Б–µ—Е `try_from`/`from`:
```nova
ro c = my_str.into()              // вЖТ CStr (auto-derived from CStr.try_from)
unsafe {
    ro s = my_cstr.into()          // вЖТ str (auto-derived from str.try_from)
                                    // вАФ auto-throws Fail[Utf8Error]
}
```

**Naming convention** (added –Ї spec convention follow-on):
- `from_X` вАФ safe (–Є–ї–Є auto-synthesized throwing form)
- `try_from_X` вАФ fallible, explicit Result form (richest implementer form)
- `from_X_unchecked` вАФ unsafe variant, skip validation BUT still copy
- `from_X_view` вАФ unsafe variant, no copy (view + lifetime contract + skip validation)
- `as_X` вАФ view if possible (zero-copy primitive)
- `to_X` вАФ always materialize (copy)

### RawMem method declarations

–Т—Б–µ pointer-touching methods —В—А–µ–±—Г—О—В `#unsafe` attribute (Plan 118 D216 ¬І9 вАФ
caller must wrap –≤ `unsafe { }`):

```nova
type RawMem                              // unit type namespace

#unsafe
fn RawMem.copy_from[T](src *T, dst *mut T, count int) -> ()
// memmove (overlap-safe). Mirror []T.copy_from semantics (spec/08-runtime.md ¬І3267).

#unsafe
fn RawMem.copy_nonoverlapping[T](src *T, dst *mut T, count int) -> ()
// memcpy (undefined –њ—А–Є overlap; perf-critical optimization).
// Rust precedent: ptr::copy vs ptr::copy_nonoverlapping.

#unsafe
fn RawMem.copy_within[T](p *mut T, src_offset int, dst_offset int, count int) -> ()
// memmove –≤–љ—Г—В—А–Є –Њ–і–љ–Њ–≥–Њ –±—Г—Д–µ—А–∞. Mirror []T.copy_within (¬І3268).

#unsafe
fn RawMem.fill[T](dst *mut T, value T, count int) -> ()
// Element-typed fill. Mirror []T.fill (¬І3269). Compile-time fast-path:
// u8/i8 вЖТ memset, wider T вЖТ scalar loop (auto-vectorizable).

#unsafe
fn RawMem.write_bytes[T](dst *mut T, byte_val u8, count int) -> ()
// memset(dst, byte_val, count * sizeof(T)).
// Generalized byte-pattern fill (covers fill_zero via byte_val=0).
// Rust precedent: ptr::write_bytes. Use cases: zero-init, debug patterns
// (0xCC/0xDEADBEEF), all-ones (-1) init.

#unsafe
fn RawMem.compare[T](a *T, b *T, count int) -> int
// memcmp. Returns 0=equal, <0=a less, >0=a greater.

#unsafe
fn RawMem.read[T](src *T) -> T
// Aligned single-element load (UB if src misaligned –і–ї—П T).
// Use case: read T from properly-aligned pointer –±–µ–Ј auto-deref ergonomic.

#unsafe
fn RawMem.read_unaligned[T](src *T) -> T
// Any-alignment safe load (~2x slower; copies bytes one-by-one –µ—Б–ї–Є –љ—Г–ґ–љ–Њ).
// Use cases: binary protocol parsing (network packets, ELF headers),
// firmware images, #pragma pack(1) C structs.
// Rust precedent: ptr::read vs ptr::read_unaligned split.

#unsafe
fn RawMem.write[T](dst *mut T, value T) -> ()
// Aligned single-element store (UB if dst misaligned –і–ї—П T).

#unsafe
fn RawMem.write_unaligned[T](dst *mut T, value T) -> ()
// Any-alignment safe store. Rust precedent: ptr::write_unaligned.

#unsafe
fn RawMem.volatile_read[T](src *T) -> T
// MMIO / driver primitive. Compiler hint: do not optimize away, do not reorder.
// Distinct –Њ—В Atomic* (which adds LOCK prefix / thread synchronization).

#unsafe
fn RawMem.volatile_write[T](dst *mut T, value T) -> ()
// MMIO / driver primitive.

fn RawMem.addr_of[T](x ro T) -> ptr      // NO #unsafe вАФ pure read of address
// Returns raw `ptr` (opaque integer). NO escape analysis, NO auto-promote
// (vs `&x` which promotes –Ї heap if escape detected). For address-as-identity
// (HashMap key, hash, debug ID).
```

**–Т—Б–µ signatures –Є—Б–њ–Њ–ї—М–Ј—Г—О—В `int` –і–ї—П count/offset (D226 signed indexing)**.
FFI boundary calls –Ї C `size_t`-typed args —В—А–µ–±—Г—О—В explicit `as usize` cast.

### Volatile vs Atomic differentiation

Documented –≤ spec D216 ¬І6 –Є–ї–Є new D217:

| Need | Use | Why not the other |
|---|---|---|
| Shared mutable state across fibers/threads | `Atomic{I8..I64,U8..U64,Ptr}` + MemOrdering | Atomic emits LOCK prefix (CPU sync), –љ–µ –њ–Њ–і—Е–Њ–і–Є—В –і–ї—П MMIO |
| MMIO register / signal handler / volatile semantics | `RawMem.volatile_read/write` | Volatile is compiler hint only (no CPU sync); Atomic over-synchronizes |

### CStr edge cases вАФ RESOLVED 2026-06-03 (Q-cstring closed)

**Research finding** (`nova_rt/nova_rt.h:117-136` + spec D26 amend):

Nova str allocator INVARIANT вАФ full str always has `ptr[len] == '\0'`:
- All paths (`nova_str_concat`, `nova_str_to_upper/lower`, `string_builder.h`,
  `conv.h`, literals –≤ `.rodata`) allocate `len + 1` + explicit `buf[len] = '\0'`.
- Documented –≤ spec/decisions/08-runtime.md ¬І¬ЂNul-termination (C-interop)¬ї
  (amended 2026-06-03 вАФ Q-cstring closed).

**BUT substring views (D144 `s[a..b]`) вАФ `view.ptr[view.len]` NOT guaranteed**:
- View shares parent's buffer; parent's NUL at `parent.ptr[parent.len]`
- For sub-views ending mid-buffer (e.g., `s[2..5]`), `view.ptr[view.len]` is
  arbitrary byte (e.g., space, next char), NOT `'\0'`.
- For sub-views ending at parent's len (e.g., `s[5..]`), `view.ptr[view.len]`
  == parent's terminating NUL.

**Resolution –≤ `@as_cstr()`:**
```nova
fn str @as_cstr() -> CStr
// Runtime check: ptr[len] == 0?
//   TRUE вЖТ zero-copy view (CStr wraps view.ptr directly)
//   FALSE вЖТ allocate len + 1, copy + NUL (fallback –Ї @to_cstr)
// Memory access at ptr[len] always safe вАФ within parent's len+1 buffer.
```

Single runtime check (1 byte read + branch) вАФ minimal overhead. Resolves
D26 Q-cstring open question.

Closes `[M-118.1-cstr-nul-invariant]` followup (no further audit needed).

### Plan 118.2 simplification (cross-impact)

Per design discussion: **NO new `Slice[T]` type** –≤ Plan 118.2. Resolution per D144
single-type design ("`[]T` вАФ –Њ–і–Є–љ —В–Є–њ –і–ї—П owner –Є view"). Instead:
```nova
fn []T @as_ptr() -> *T                  // NEW вАФ ro view of internal data ptr (FFI read)
fn []T mut @as_mut_ptr() -> *mut T      // NEW вАФ mut view (FFI write, requires mut binding)
fn []T @len() -> int                    // exists (D32)
fn []T @cap() -> int                    // exists (D32)
```

**Naming `@as_ptr` / `@as_mut_ptr`** (not bare `@ptr` / `@ptr_mut`):
- Avoid visual confusion —Б `ptr` primitive type
- Match `as_X` view convention (zero-copy view)
- Rust precedent: `Vec::as_ptr()` / `Vec::as_mut_ptr()`

User accesses parts directly:
```nova
unsafe {
    ro view = arr[0..1024]                          // []T view (D144)
    read(fd, view.as_ptr() as *mut u8, view.len() as usize)
}
```

This is `[]T` parts extraction, not new slice type. Plan 118.2 refactored accordingly
(see Plan 118.2 design refinement section).

`p.method()`. –Э–Њ **memory access intrinsics** –і–ї—П FFI –Њ—В—Б—Г—В—Б—В–≤—Г—О—В:

```nova
// Plan 118 core gives us:

    ro buf *u8 = some_ffi_call()

    ro field = buf.field          // (if buf points to struct)
}

// What Plan 118 core CANNOT do:

    // memcpy 1024 bytes from src to dst
    src.copy_to(dst, 1024)        // вЭМ no such method

    // read 4 bytes as i32 at typed offset
```

**–°—Ж–µ–љ–∞—А–Є–Є –±–µ–Ј 118.1:**
1. **Bulk buffer copy** (image data, network packets) вАФ user –њ–Є—И–µ—В C-shim per call
2. **Memory-mapped I/O** (hardware registers, framebuffers) вАФ –љ–µ–і–Њ—Б—В—Г–њ–љ–Њ
3. **Typed reads –Є–Ј FFI buffers** (parse binary protocols) вАФ manual cast hacks
4. **C-string FFI** (sqlite, curl, png вАФ –≤—Б–µ –њ—А–Є–љ–Є–Љ–∞—О—В `const char*`) вАФ boilerplate
   string-conversion –≤ shim'–∞—Е
5. **Out-params** (`int func(out int* result)`) вАФ –љ—Г–ґ–µ–љ `addr_of_mut!`

## –Ф–Є–Ј–∞–є–љ

### 1. Typed read/write primitives

```nova
unsafe {
    ro buf *u8 = malloc(16).unwrap()

    // Read 1 byte
    ro byte = buf.read()                       // u8

    // Read typed value (offset already encoded –≤ pointer type)
    ro int_ptr = buf as *i32
    ro x = int_ptr.read()                       // i32 (4 bytes consumed)

    // Write
    int_ptr.write(0xDEAD_BEEF)
}
```

**API:**
- `(*T).read() -> T` вАФ typed read; equivalent to `*p` –љ–Њ explicit form –і–ї—П clarity
- `(*mut T).write(v T) -> ()` вАФ typed write; equivalent to `*p = v` –љ–Њ explicit

**Why duplicate `*p`?** Two reasons:
1. **Method form** –Є–љ—В–µ–≥—А–Є—А—Г–µ—В—Б—П —Б `.copy_to()` / `.read_volatile()` chain
   pattern (mainstream familiarity)
2. **Explicit –і–ї—П `*unsafe T`**: nested unsafe wrap proxied —З–µ—А–µ–Ј method call
   reads cleaner

### 2. Bulk copy primitives

```nova
unsafe {
    ro src *ro u8 = source_buffer()
    ro dst *mut u8 = dest_buffer()
    ro n usize = 1024

    src.copy_to(dst, n)                 // memmove (overlapping safe)
    src.copy_to_nonoverlapping(dst, n)  // memcpy (UB if overlap)
}
```

**API:**
- `(*ro T).copy_to(dst *mut T, count usize) -> ()` вАФ memmove; handles overlap
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` вАФ memcpy;
  faster –љ–Њ UB on overlap
- Both compile to direct `memmove` / `memcpy` calls (linker resolves
  –Ї platform libc)

**Units:** `count` вАФ element count (not bytes); compiler multiplies by
`sizeof(T)` for actual byte count.

**Type safety:** `src` –Є `dst` –і–Њ–ї–ґ–љ—Л –Є–Љ–µ—В—М same `T` (or one cast –≤ unsafe).

### 3. Volatile reads/writes (memory-mapped I/O)

```nova
unsafe {
    ro reg *mut u32 = 0xFFFF_0010 as *mut u32   // memory-mapped register

    reg.write_volatile(0xDEAD_BEEF)              // visible to hardware immediately
    ro status = reg.read_volatile()              // reread (no compiler caching)
}
```

**API:**
- `(*T).read_volatile() -> T` вАФ volatile read; compiler –љ–µ cache'–Є—В
- `(*mut T).write_volatile(v T) -> ()` вАФ volatile write; immediate side-effect

**Codegen:** emit C `volatile T*` cast at access point:
```c
((volatile uint32_t*)reg)[0]               // read_volatile
*((volatile uint32_t*)reg) = 0xDEAD_BEEF;  // write_volatile
```

**Compiler guarantee:** no reordering, no caching, no elision (per C
`volatile` semantics). Critical –і–ї—П hardware registers + lock-free protocols.

### 4. `addr_of!` / `addr_of_mut!` macros

```nova
type Packed {
    a u8
    b u32                                // unaligned field!
}

unsafe {
    ro packed = Packed { a: 0xFF, b: 0xDEAD_BEEF }

    // вЭМ &packed.b вАФ –Љ–Њ–ґ–µ—В dereference unaligned value temporarily
    // вЬУ addr_of!(packed.b) вАФ pointer without deref
    ro p_b = addr_of!(packed.b)                  // *ro u32 (may be unaligned!)

    // Similarly –і–ї—П mut:
    mut packed_mut = ...
    ro p_b_mut = addr_of_mut!(packed_mut.b)      // *mut u32
}
```

**API:**
- `addr_of!(expr)` вАФ macro/intrinsic; returns `*ro T` where T = type of expr
- `addr_of_mut!(expr)` вАФ returns `*mut T`
- **Does not deref** the value вАФ safe even on packed/uninit memory
- Compile-time check: expr must be lvalue (addressable)

**Why needed:** `&packed.b` semantically creates a `*T` —З–µ—А–µ–Ј value access,
which requires aligned load (UB –љ–∞ packed structs). `addr_of!` provides
pointer construction –±–µ–Ј implicit load вАФ Rust 1.51+ pattern, critical –і–ї—П
FFI —Б C-packed structs.

**Codegen:** emit direct `&(packed.b)` C expression (no temporary value
materialization).

### 5. C-string convention

C libraries —Г–љ–Є–≤–µ—А—Б–∞–ї—М–љ–Њ –њ—А–Є–љ–Є–Љ–∞—О—В/–≤–Њ–Ј–≤—А–∞—Й–∞—О—В **null-terminated `const char*`**.
Nova `str` —В–Є–њ вАФ length-prefixed (`{ data: *ro u8, len: usize }`); NOT
null-terminated. Conversion was C-shim boilerplate –≤ Plan 115 V1.

**`cstr"..."` literal:**

```nova
unsafe {
    // Plan 115 V1 вАФ needed C-shim:
    //   char path_buf[1024];
    //   memcpy(path_buf, path.ptr, path.len);
    //   path_buf[path.len] = '\0';
    //   sqlite3_open(path_buf, &db);

    // Plan 118.1:
    ro path = cstr"database.sqlite3"             // *ro u8, null-terminated literal
    sqlite3_open(path, &mut db)                  // direct FFI
}
```

**Literal semantics:**
- `cstr"foo"` вАФ compile-time string literal —Б trailing `\0` byte
- Type: `*ro u8` (pointer to first byte)
- Stored –≤ read-only segment (`.rodata`)
- Length: `4` bytes (`"foo\0"`); user code –љ–µ sees the null
- **Cannot** contain embedded `\0` (compile error `E_CSTR_EMBEDDED_NULL`)

**Interop functions:**
- `(*ro u8).cstring_len() -> usize` вАФ runtime `strlen` (counts to first `\0`)
- `(*ro u8).from_cstring() -> str` вАФ copy bytes to GC-allocated Nova str
  (length determined by `strlen`)
- `str.to_cstring(buf *mut u8, buf_len usize) -> usize` вАФ copy str + null
  terminator to provided buffer; returns bytes written (or err if buf too small)

**Why explicit `cstr"..."` vs auto-convert `"..."`:**
- `str` literals –≤ Nova вАФ Nova-managed (length-prefixed); auto-convert
  –љ–∞ –Ї–∞–ґ–і–Њ–Љ FFI call = silent heap allocation
- Explicit `cstr"..."` вАФ compile-time string –≤ `.rodata`, zero allocation,
  zero copy

**Limits:**
- `cstr"..."` вАФ only ASCII/UTF-8 byte content (no escape –і–ї—П NULs)
- Maximum –ї–Є—В–µ—А–∞–ї–∞ length вАФ 64KB (compile-time limit –і–ї—П `.rodata` sanity)
- Runtime `str.to_cstring(buf, len)` вАФ for dynamic case —Б user-provided buffer

### 6. Out-param FFI pattern

Combining `addr_of_mut!` + `Option[*T]`:

```nova
external fn try_alloc(out *mut Option[*u8]) -> i64
//                       ^^^^^^^^^^^^^^^^^^^
//                       caller passes pointer to Option storage

unsafe {
    mut buf Option[*u8] = None
    ro rc = try_alloc(addr_of_mut!(buf))         // C fills *buf via address
    match buf {
        Some(p) => use_buffer(p),
        None    => Fail.throw(OutOfMemory),
    }
}
```

**Pattern documented –≤ ffi-cookbook.md update (Plan 118.1 –§.5).**

### 7. Alignment + size queries

```nova
unsafe {
    ro align = align_of[*ro u8]()                  // 1
    ro align32 = align_of[*ro u32]()                // 4
    ro size = size_of[*ro Account]()                // 8 (pointer-width)
    ro size_t = size_of[Account]()                  // size of Account record
}
```

**API:**
- `align_of[T]() -> usize` вАФ compile-time constant; alignment of T
- `size_of[T]() -> usize` вАФ compile-time constant; size of T

**Use case:** manual buffer allocation + alignment compliance for FFI.

## D-block changes

### D217 (NEW) вАФ FFI memory access primitives + C-string convention

**–Ы–Њ–Ї–∞—Ж–Є—П:** `spec/decisions/02-types.md` (–њ–Њ—Б–ї–µ D216).

#### ¬І1. Typed read/write

- `(*T).read() -> T` вАФ typed read; equivalent to `*p`
- `(*mut T).write(v T) -> ()` вАФ typed write; equivalent to `*p = v`
- All –≤ unsafe context

#### ¬І2. Bulk copy

- `(*ro T).copy_to(dst *mut T, count usize) -> ()` вАФ memmove (overlap safe)
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` вАФ memcpy
- Units: element count (compiler scales by sizeof(T))

#### ¬І3. Volatile

- `(*T).read_volatile() -> T` вАФ no caching
- `(*mut T).write_volatile(v T) -> ()` вАФ immediate side-effect
- Codegen: C `volatile` qualifier –љ–∞ access point

#### ¬І4. Address-of intrinsics

- `addr_of!(expr) -> *ro T` вАФ pointer –±–µ–Ј temp materialization
- `addr_of_mut!(expr) -> *mut T` вАФ same, mut variant
- Critical –і–ї—П packed structs / uninit memory

#### ¬І5. Alignment + size

- `align_of[T]() -> usize` вАФ compile-time const
- `size_of[T]() -> usize` вАФ compile-time const

#### ¬І6. C-string convention

- `cstr"..."` literal вАФ null-terminated, `.rodata`, type `*ro u8`
- Cannot contain embedded `\0` (`E_CSTR_EMBEDDED_NULL`)
- Maximum 64KB literal length (`E_CSTR_TOO_LARGE`)
- Interop: `.cstring_len()` (runtime strlen), `.from_cstring()` (copy to str),
  `str.to_cstring(buf, len)` (write null-term to buffer)

**Cross-ref:** D216 (typed pointer foundations), D26 (str type), D214 (ptr).

### D26 cross-ref (existing string types) вАФ C-string convention paragraph added

Cross-ref existing D26 (str type) —Б new `cstr"..."` literal pattern –Є
interop conversion functions. No D26 amend вАФ just ¬І¬ЂFFI interop¬ї paragraph.
