## Design refinement 2026-06-03 (user discussion finalization)

### Naming + namespace

**`RawMem`** ‚Äî namespace type (unit-type declaration per spec/02-types.md:233):
- `type RawMem` (no body, no fields) ‚Äî Nova valid unit type
- Static methods: `RawMem.copy_from[T](...)` style
- Use case: low-level memory operations (driver/embedded/perf-critical)
- Rejected alternatives: `Raw` (too short, semantic vague), `Unsafe` (doubled keyword
  —Å `unsafe { }` block), `Intrinsics` (broad), `Mem` (too narrow ‚Äî addr_of –Ω–µ fits)

### CStr type

**`type CStr(*u8)`** ‚Äî tuple newtype wrapping null-terminated bytes:
- Plan 115 D52 precedent (canonical FFI handle pattern, zero-overhead)
- C ABI: `const char*` (single pointer, NO header)
- PascalCase user-declared type (lowercase `cstr` would require compiler
  special-case –∫–∞–∫ `ptr`/`str` ‚Äî deferred –∫ future plan –µ—Å–ª–∏ ergonomic demand)
- Location: `std/ffi/cstr.nv` (–ª–∏–±–æ `std/prelude/cstr.nv` –µ—Å–ª–∏ global-visible)

### Conversions —á–µ—Ä–µ–∑ D73/D77 From/Into protocols (REFINED 2026-06-03)

**Implementer pi≈°et `try_from` (richest form per D77)** ‚Äî compiler auto-synthesizes
short `from`/`into` forms.

```nova
// ============================================================================
// CStr ‚Üê str (Nova-managed allocation, NO raw deref of user-supplied ptr)
// ============================================================================

fn CStr.try_from(s str) -> Result[CStr, CStrError]
// SAFE ‚Äî validates no embedded NUL, allocates s.len + 1, copies + NUL.
// Compiler auto-synthesizes via D77:
//   CStr.from(s str) Fail[CStrError] -> CStr     (short, throws)
//   s.into() / s.try_into()

#unsafe
fn CStr.from_unchecked(s str) -> CStr
// Skip NUL validation; allocates + copies + NUL.
// Caller promises s has no embedded NUL.
// Rust precedent: CString::from_vec_unchecked (unsafe fn).

// ============================================================================
// str ‚Üê CStr (raw deref of user-supplied pointer ‚Äî all #unsafe)
// ============================================================================

#unsafe
fn str.try_from(c CStr) -> Result[str, Utf8Error]
// Allocates new buffer, memcpy + UTF-8 validate.
// Compiler auto-synthesizes via D77:
//   str.from(c CStr) Fail[Utf8Error] -> str      (short, throws)
//   c.into() / c.try_into()

#unsafe
fn str.from_view(c CStr) -> str
// VIEW: no copy, no validation. Returns str with ptr=c.0, len=strlen(c.0).
// Caller promises:
//   (1) CStr buffer outlives the resulting str view
//   (2) bytes ARE valid UTF-8
// Differs –æ—Ç str.from_bytes_unchecked (which COPIES + skip validate); from_view
// ALSO skips copy. Rust equivalent: combines CStr::from_ptr + to_bytes +
// std::str::from_utf8_unchecked –≤ –æ–¥–∏–Ω step (Nova –Ω–µ –∏–º–µ–µ—Ç lifetime tracking).

// ============================================================================
// str ‚Üí CStr (view-if-possible OR materialize)
// ============================================================================

fn str @as_cstr() -> CStr
// SAFE primitive. Runtime check: ptr[len] == 0?
//   TRUE ‚Üí zero-copy view (CStr wraps str.ptr directly)
//   FALSE ‚Üí allocate + copy + NUL (fallback –∫ @to_cstr semantics)
// Validates no embedded NUL (panic –µ—Å–ª–∏ –Ω–∞–π–¥–µ–Ω ‚Äî C-side truncation prevention).
// Resolves D26 ¬ßNul-termination Q-cstring (spec amend 2026-06-03).

fn str @to_cstr() -> CStr
// SAFE ‚Äî always allocates new buffer + copy + NUL.
// Validates no embedded NUL (panic).

#unsafe
fn str @as_cstr_unchecked() -> CStr
// Skip embedded NUL validation. Caller promises str has no '\0' –≤–Ω—É—Ç—Ä–∏ [0..len).
// For perf-critical paths where validation cost matters.
```

**D73 auto-derive –¥–∞–µ—Ç `.into()` —Ñ–æ—Ä–º—É –±–µ—Å–ø–ª–∞—Ç–Ω–æ** –¥–ª—è –≤—Å–µ—Ö `try_from`/`from`:
```nova
ro c = my_str.into()              // ‚Üí CStr (auto-derived from CStr.try_from)
unsafe {
    ro s = my_cstr.into()          // ‚Üí str (auto-derived from str.try_from)
                                    // ‚Äî auto-throws Fail[Utf8Error]
}
```

**Naming convention** (added –∫ spec convention follow-on):
- `from_X` ‚Äî safe (–∏–ª–∏ auto-synthesized throwing form)
- `try_from_X` ‚Äî fallible, explicit Result form (richest implementer form)
- `from_X_unchecked` ‚Äî unsafe variant, skip validation BUT still copy
- `from_X_view` ‚Äî unsafe variant, no copy (view + lifetime contract + skip validation)
- `as_X` ‚Äî view if possible (zero-copy primitive)
- `to_X` ‚Äî always materialize (copy)

### RawMem method declarations

–í—Å–µ pointer-touching methods —Ç—Ä–µ–±—É—é—Ç `#unsafe` attribute (Plan 118 D216 ¬ß9 ‚Äî
caller must wrap –≤ `unsafe { }`):

```nova
type RawMem                              // unit type namespace

#unsafe
fn RawMem.copy_from[T](src *T, dst *mut T, count int) -> ()
// memmove (overlap-safe). Mirror []T.copy_from semantics (spec/08-runtime.md ¬ß3267).

#unsafe
fn RawMem.copy_nonoverlapping[T](src *T, dst *mut T, count int) -> ()
// memcpy (undefined –ø—Ä–∏ overlap; perf-critical optimization).
// Rust precedent: ptr::copy vs ptr::copy_nonoverlapping.

#unsafe
fn RawMem.copy_within[T](p *mut T, src_offset int, dst_offset int, count int) -> ()
// memmove –≤–Ω—É—Ç—Ä–∏ –æ–¥–Ω–æ–≥–æ –±—É—Ñ–µ—Ä–∞. Mirror []T.copy_within (¬ß3268).

#unsafe
fn RawMem.fill[T](dst *mut T, value T, count int) -> ()
// Element-typed fill. Mirror []T.fill (¬ß3269). Compile-time fast-path:
// u8/i8 ‚Üí memset, wider T ‚Üí scalar loop (auto-vectorizable).

#unsafe
fn RawMem.write_bytes[T](dst *mut T, byte_val u8, count int) -> ()
// memset(dst, byte_val, count * sizeof(T)).
// Generalized byte-pattern fill (covers fill_zero via byte_val=0).
// Rust precedent: ptr::write_bytes. Use cases: zero-init, debug patterns
// (0xCC/0xDEADBEEF), all-ones (-1) init.

#unsafe
fn RawMem.compare[T](a *T, b *T, count int) -> int
// memcmp. Returns 0=equal, <0=a less, >0=a greater.

#unsafe
fn RawMem.read[T](src *T) -> T
// Aligned single-element load (UB if src misaligned –¥–ª—è T).
// Use case: read T from properly-aligned pointer –±–µ–∑ auto-deref ergonomic.

#unsafe
fn RawMem.read_unaligned[T](src *T) -> T
// Any-alignment safe load (~2x slower; copies bytes one-by-one –µ—Å–ª–∏ –Ω—É–∂–Ω–æ).
// Use cases: binary protocol parsing (network packets, ELF headers),
// firmware images, #pragma pack(1) C structs.
// Rust precedent: ptr::read vs ptr::read_unaligned split.

#unsafe
fn RawMem.write[T](dst *mut T, value T) -> ()
// Aligned single-element store (UB if dst misaligned –¥–ª—è T).

#unsafe
fn RawMem.write_unaligned[T](dst *mut T, value T) -> ()
// Any-alignment safe store. Rust precedent: ptr::write_unaligned.

#unsafe
fn RawMem.volatile_read[T](src *T) -> T
// MMIO / driver primitive. Compiler hint: do not optimize away, do not reorder.
// Distinct –æ—Ç Atomic* (which adds LOCK prefix / thread synchronization).

#unsafe
fn RawMem.volatile_write[T](dst *mut T, value T) -> ()
// MMIO / driver primitive.

fn RawMem.addr_of[T](x ro T) -> ptr      // NO #unsafe ‚Äî pure read of address
// Returns raw `ptr` (opaque integer). NO escape analysis, NO auto-promote
// (vs `&x` which promotes –∫ heap if escape detected). For address-as-identity
// (HashMap key, hash, debug ID).
```

**–í—Å–µ signatures –∏—Å–ø–æ–ª—å–∑—É—é—Ç `int` –¥–ª—è count/offset (D226 signed indexing)**.
FFI boundary calls –∫ C `size_t`-typed args —Ç—Ä–µ–±—É—é—Ç explicit `as usize` cast.

### Volatile vs Atomic differentiation

Documented –≤ spec D216 ¬ß6 –∏–ª–∏ new D217:

| Need | Use | Why not the other |
|---|---|---|
| Shared mutable state across fibers/threads | `Atomic{I8..I64,U8..U64,Ptr}` + MemOrdering | Atomic emits LOCK prefix (CPU sync), –Ω–µ –ø–æ–¥—Ö–æ–¥–∏—Ç –¥–ª—è MMIO |
| MMIO register / signal handler / volatile semantics | `RawMem.volatile_read/write` | Volatile is compiler hint only (no CPU sync); Atomic over-synchronizes |

### CStr edge cases ‚Äî RESOLVED 2026-06-03 (Q-cstring closed)

**Research finding** (`nova_rt/nova_rt.h:117-136` + spec D26 amend):

Nova str allocator INVARIANT ‚Äî full str always has `ptr[len] == '\0'`:
- All paths (`nova_str_concat`, `nova_str_to_upper/lower`, `string_builder.h`,
  `conv.h`, literals –≤ `.rodata`) allocate `len + 1` + explicit `buf[len] = '\0'`.
- Documented –≤ spec/decisions/08-runtime.md ¬ß¬´Nul-termination (C-interop)¬ª
  (amended 2026-06-03 ‚Äî Q-cstring closed).

**BUT substring views (D144 `s[a..b]`) ‚Äî `view.ptr[view.len]` NOT guaranteed**:
- View shares parent's buffer; parent's NUL at `parent.ptr[parent.len]`
- For sub-views ending mid-buffer (e.g., `s[2..5]`), `view.ptr[view.len]` is
  arbitrary byte (e.g., space, next char), NOT `'\0'`.
- For sub-views ending at parent's len (e.g., `s[5..]`), `view.ptr[view.len]`
  == parent's terminating NUL.

**Resolution –≤ `@as_cstr()`:**
```nova
fn str @as_cstr() -> CStr
// Runtime check: ptr[len] == 0?
//   TRUE ‚Üí zero-copy view (CStr wraps view.ptr directly)
//   FALSE ‚Üí allocate len + 1, copy + NUL (fallback –∫ @to_cstr)
// Memory access at ptr[len] always safe ‚Äî within parent's len+1 buffer.
```

Single runtime check (1 byte read + branch) ‚Äî minimal overhead. Resolves
D26 Q-cstring open question.

Closes `[M-118.1-cstr-nul-invariant]` followup (no further audit needed).

### Plan 118.2 simplification (cross-impact)

Per design discussion: **NO new `Slice[T]` type** –≤ Plan 118.2. Resolution per D144
single-type design ("`[]T` ‚Äî –æ–¥–∏–Ω —Ç–∏–ø –¥–ª—è owner –∏ view"). Instead:
```nova
fn []T @as_ptr() -> *T                  // NEW ‚Äî ro view of internal data ptr (FFI read)
fn []T mut @as_mut_ptr() -> *mut T      // NEW ‚Äî mut view (FFI write, requires mut binding)
fn []T @len() -> int                    // exists (D32)
fn []T @cap() -> int                    // exists (D32)
```

**Naming `@as_ptr` / `@as_mut_ptr`** (not bare `@ptr` / `@ptr_mut`):
- Avoid visual confusion —Å `ptr` primitive type
- Match `as_X` view convention (zero-copy view)
- Rust precedent: `Vec::as_ptr()` / `Vec::as_mut_ptr()`

User accesses parts directly:
```nova
unsafe {
    ro view = arr[0..1024]                          // []T view (D144)
    read(fd, view.as_ptr() as *mut u8, view.len() as usize)
}
```

This is `[]T` parts extraction, not new slice type. Plan 118.2 refactored accordingly
(see Plan 118.2 design refinement section).

// What Plan 118 core CANNOT do:

## –î–∏–∑–∞–π–Ω

### 1. Typed read/write primitives

```nova
unsafe {
    ro buf *u8 = malloc(16).unwrap()

    // Read 1 byte
    ro byte = buf.read()                       // u8

    // Read typed value (offset already encoded –≤ pointer type)
    ro int_ptr = buf as *i32
    ro x = int_ptr.read()                       // i32 (4 bytes consumed)

    // Write
    int_ptr.write(0xDEAD_BEEF)
}
```

**API:**
- `(*T).read() -> T` ‚Äî typed read; equivalent to `*p` –Ω–æ explicit form –¥–ª—è clarity
- `(*mut T).write(v T) -> ()` ‚Äî typed write; equivalent to `*p = v` –Ω–æ explicit

**Why duplicate `*p`?** Two reasons:
1. **Method form** –∏–Ω—Ç–µ–≥—Ä–∏—Ä—É–µ—Ç—Å—è —Å `.copy_to()` / `.read_volatile()` chain
   pattern (mainstream familiarity)
2. **Explicit –¥–ª—è `*unsafe T`**: nested unsafe wrap proxied —á–µ—Ä–µ–∑ method call
   reads cleaner

### 2. Bulk copy primitives

```nova
unsafe {
    ro src *ro u8 = source_buffer()
    ro dst *mut u8 = dest_buffer()
    ro n usize = 1024

    src.copy_to(dst, n)                 // memmove (overlapping safe)
    src.copy_to_nonoverlapping(dst, n)  // memcpy (UB if overlap)
}
```

**API:**
- `(*ro T).copy_to(dst *mut T, count usize) -> ()` ‚Äî memmove; handles overlap
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` ‚Äî memcpy;
  faster –Ω–æ UB on overlap
- Both compile to direct `memmove` / `memcpy` calls (linker resolves
  –∫ platform libc)

**Units:** `count` ‚Äî element count (not bytes); compiler multiplies by
`sizeof(T)` for actual byte count.

**Type safety:** `src` –∏ `dst` –¥–æ–ª–∂–Ω—ã –∏–º–µ—Ç—å same `T` (or one cast –≤ unsafe).

### 3. Volatile reads/writes (memory-mapped I/O)

```nova
unsafe {
    ro reg *mut u32 = 0xFFFF_0010 as *mut u32   // memory-mapped register

    reg.write_volatile(0xDEAD_BEEF)              // visible to hardware immediately
    ro status = reg.read_volatile()              // reread (no compiler caching)
}
```

**API:**
- `(*T).read_volatile() -> T` ‚Äî volatile read; compiler –Ω–µ cache'–∏—Ç
- `(*mut T).write_volatile(v T) -> ()` ‚Äî volatile write; immediate side-effect

**Codegen:** emit C `volatile T*` cast at access point:
```c
((volatile uint32_t*)reg)[0]               // read_volatile
*((volatile uint32_t*)reg) = 0xDEAD_BEEF;  // write_volatile
```

**Compiler guarantee:** no reordering, no caching, no elision (per C
`volatile` semantics). Critical –¥–ª—è hardware registers + lock-free protocols.

### 4. `addr_of!` / `addr_of_mut!` macros

```nova
type Packed {
    a u8
    b u32                                // unaligned field!
}

unsafe {
    ro packed = Packed { a: 0xFF, b: 0xDEAD_BEEF }

    // ‚ùå &packed.b ‚Äî –º–æ–∂–µ—Ç dereference unaligned value temporarily
    // ‚úì addr_of!(packed.b) ‚Äî pointer without deref
    ro p_b = addr_of!(packed.b)                  // *ro u32 (may be unaligned!)

    // Similarly –¥–ª—è mut:
    mut packed_mut = ...
    ro p_b_mut = addr_of_mut!(packed_mut.b)      // *mut u32
}
```

**API:**
- `addr_of!(expr)` ‚Äî macro/intrinsic; returns `*ro T` where T = type of expr
- `addr_of_mut!(expr)` ‚Äî returns `*mut T`
- **Does not deref** the value ‚Äî safe even on packed/uninit memory
- Compile-time check: expr must be lvalue (addressable)

**Why needed:** `&packed.b` semantically creates a `*T` —á–µ—Ä–µ–∑ value access,
which requires aligned load (UB –Ω–∞ packed structs). `addr_of!` provides
pointer construction –±–µ–∑ implicit load ‚Äî Rust 1.51+ pattern, critical –¥–ª—è
FFI —Å C-packed structs.

**Codegen:** emit direct `&(packed.b)` C expression (no temporary value
materialization).

### 5. C-string convention

C libraries —É–Ω–∏–≤–µ—Ä—Å–∞–ª—å–Ω–æ –ø—Ä–∏–Ω–∏–º–∞—é—Ç/–≤–æ–∑–≤—Ä–∞—â–∞—é—Ç **null-terminated `const char*`**.
Nova `str` —Ç–∏–ø ‚Äî length-prefixed (`{ data: *ro u8, len: usize }`); NOT
null-terminated. Conversion was C-shim boilerplate –≤ Plan 115 V1.

**`cstr"..."` literal:**

```nova
unsafe {
    // Plan 115 V1 ‚Äî needed C-shim:
    //   char path_buf[1024];
    //   memcpy(path_buf, path.ptr, path.len);
    //   path_buf[path.len] = '\0';
    //   sqlite3_open(path_buf, &db);

    // Plan 118.1:
    ro path = cstr"database.sqlite3"             // *ro u8, null-terminated literal
    sqlite3_open(path, &mut db)                  // direct FFI
}
```

**Literal semantics:**
- `cstr"foo"` ‚Äî compile-time string literal —Å trailing `\0` byte
- Type: `*ro u8` (pointer to first byte)
- Stored –≤ read-only segment (`.rodata`)
- Length: `4` bytes (`"foo\0"`); user code –Ω–µ sees the null
- **Cannot** contain embedded `\0` (compile error `E_CSTR_EMBEDDED_NULL`)

**Interop functions:**
- `(*ro u8).cstring_len() -> usize` ‚Äî runtime `strlen` (counts to first `\0`)
- `(*ro u8).from_cstring() -> str` ‚Äî copy bytes to GC-allocated Nova str
  (length determined by `strlen`)
- `str.to_cstring(buf *mut u8, buf_len usize) -> usize` ‚Äî copy str + null
  terminator to provided buffer; returns bytes written (or err if buf too small)

**Why explicit `cstr"..."` vs auto-convert `"..."`:**
- `str` literals –≤ Nova ‚Äî Nova-managed (length-prefixed); auto-convert
  –Ω–∞ –∫–∞–∂–¥–æ–º FFI call = silent heap allocation
- Explicit `cstr"..."` ‚Äî compile-time string –≤ `.rodata`, zero allocation,
  zero copy

**Limits:**
- `cstr"..."` ‚Äî only ASCII/UTF-8 byte content (no escape –¥–ª—è NULs)
- Maximum –ª–∏—Ç–µ—Ä–∞–ª–∞ length ‚Äî 64KB (compile-time limit –¥–ª—è `.rodata` sanity)
- Runtime `str.to_cstring(buf, len)` ‚Äî for dynamic case —Å user-provided buffer

### 6. Out-param FFI pattern

Combining `addr_of_mut!` + `Option[*T]`:

```nova
external fn try_alloc(out *mut Option[*u8]) -> i64
//                       ^^^^^^^^^^^^^^^^^^^
//                       caller passes pointer to Option storage

unsafe {
    mut buf Option[*u8] = None
    ro rc = try_alloc(addr_of_mut!(buf))         // C fills *buf via address
    match buf {
        Some(p) => use_buffer(p),
        None    => Fail.throw(OutOfMemory),
    }
}
```

**Pattern documented –≤ ffi-cookbook.md update (Plan 118.1 –§.5).**

### 7. Alignment + size queries

```nova
unsafe {
    ro align = align_of[*ro u8]()                  // 1
    ro align32 = align_of[*ro u32]()                // 4
    ro size = size_of[*ro Account]()                // 8 (pointer-width)
    ro size_t = size_of[Account]()                  // size of Account record
}
```

**API:**
- `align_of[T]() -> usize` ‚Äî compile-time constant; alignment of T
- `size_of[T]() -> usize` ‚Äî compile-time constant; size of T

**Use case:** manual buffer allocation + alignment compliance for FFI.

## D-block changes

### D217 (NEW) ‚Äî FFI memory access primitives + C-string convention

**–õ–æ–∫–∞—Ü–∏—è:** `spec/decisions/02-types.md` (–ø–æ—Å–ª–µ D216).

#### ¬ß1. Typed read/write

- `(*T).read() -> T` ‚Äî typed read; equivalent to `*p`
- `(*mut T).write(v T) -> ()` ‚Äî typed write; equivalent to `*p = v`
- All –≤ unsafe context

#### ¬ß2. Bulk copy

- `(*ro T).copy_to(dst *mut T, count usize) -> ()` ‚Äî memmove (overlap safe)
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` ‚Äî memcpy
- Units: element count (compiler scales by sizeof(T))

#### ¬ß3. Volatile

- `(*T).read_volatile() -> T` ‚Äî no caching
- `(*mut T).write_volatile(v T) -> ()` ‚Äî immediate side-effect
- Codegen: C `volatile` qualifier –Ω–∞ access point

#### ¬ß4. Address-of intrinsics

- `addr_of!(expr) -> *ro T` ‚Äî pointer –±–µ–∑ temp materialization
- `addr_of_mut!(expr) -> *mut T` ‚Äî same, mut variant
- Critical –¥–ª—è packed structs / uninit memory

#### ¬ß5. Alignment + size

- `align_of[T]() -> usize` ‚Äî compile-time const
- `size_of[T]() -> usize` ‚Äî compile-time const

#### ¬ß6. C-string convention

- `cstr"..."` literal ‚Äî null-terminated, `.rodata`, type `*ro u8`
- Cannot contain embedded `\0` (`E_CSTR_EMBEDDED_NULL`)
- Maximum 64KB literal length (`E_CSTR_TOO_LARGE`)
- Interop: `.cstring_len()` (runtime strlen), `.from_cstring()` (copy to str),
  `str.to_cstring(buf, len)` (write null-term to buffer)

**Cross-ref:** D216 (typed pointer foundations), D26 (str type), D214 (ptr).

### D26 cross-ref (existing string types) ‚Äî C-string convention paragraph added

Cross-ref existing D26 (str type) —Å new `cstr"..."` literal pattern –∏
interop conversion functions. No D26 amend ‚Äî just ¬ß¬´FFI interop¬ª paragraph.
