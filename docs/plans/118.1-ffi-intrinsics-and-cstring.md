## Design refinement 2026-06-03 (user discussion finalization)

### Naming + namespace

**`RawMem`** — namespace type (unit-type declaration per spec/02-types.md:233):
- `type RawMem` (no body, no fields) — Nova valid unit type
- Static methods: `RawMem.copy_from[T](...)` style
- Use case: low-level memory operations (driver/embedded/perf-critical)
- Rejected alternatives: `Raw` (too short, semantic vague), `Unsafe` (doubled keyword
  с `unsafe { }` block), `Intrinsics` (broad), `Mem` (too narrow — addr_of не fits)

### CStr type

**`type CStr(*u8)`** — tuple newtype wrapping null-terminated bytes:
- Plan 115 D52 precedent (canonical FFI handle pattern, zero-overhead)
- C ABI: `const char*` (single pointer, NO header)
- PascalCase user-declared type (lowercase `cstr` would require compiler
  special-case как `ptr`/`str` — deferred к future plan если ergonomic demand)
- Location: `std/ffi/cstr.nv` (либо `std/prelude/cstr.nv` если global-visible)

### Conversions через D73/D77 From/Into protocols (REFINED 2026-06-03)

**Implementer pišet `try_from` (richest form per D77)** — compiler auto-synthesizes
short `from`/`into` forms.

```nova
// ============================================================================
// CStr ← str (Nova-managed allocation, NO raw deref of user-supplied ptr)
// ============================================================================

fn CStr.try_from(s str) -> Result[CStr, CStrError]
// SAFE — validates no embedded NUL, allocates s.len + 1, copies + NUL.
// Compiler auto-synthesizes via D77:
//   CStr.from(s str) Fail[CStrError] -> CStr     (short, throws)
//   s.into() / s.try_into()

#unsafe
fn CStr.from_unchecked(s str) -> CStr
// Skip NUL validation; allocates + copies + NUL.
// Caller promises s has no embedded NUL.
// Rust precedent: CString::from_vec_unchecked (unsafe fn).

// ============================================================================
// str ← CStr (raw deref of user-supplied pointer — all #unsafe)
// ============================================================================

#unsafe
fn str.try_from(c CStr) -> Result[str, Utf8Error]
// Allocates new buffer, memcpy + UTF-8 validate.
// Compiler auto-synthesizes via D77:
//   str.from(c CStr) Fail[Utf8Error] -> str      (short, throws)
//   c.into() / c.try_into()

#unsafe
fn str.from_view(c CStr) -> str
// VIEW: no copy, no validation. Returns str with ptr=c.0, len=strlen(c.0).
// Caller promises:
//   (1) CStr buffer outlives the resulting str view
//   (2) bytes ARE valid UTF-8
// Differs от str.from_bytes_unchecked (which COPIES + skip validate); from_view
// ALSO skips copy. Rust equivalent: combines CStr::from_ptr + to_bytes +
// std::str::from_utf8_unchecked в один step (Nova не имеет lifetime tracking).

// ============================================================================
// str → CStr (view-if-possible OR materialize)
// ============================================================================

fn str @as_cstr() -> CStr
// SAFE primitive. Runtime check: ptr[len] == 0?
//   TRUE → zero-copy view (CStr wraps str.ptr directly)
//   FALSE → allocate + copy + NUL (fallback к @to_cstr semantics)
// Validates no embedded NUL (panic если найден — C-side truncation prevention).
// Resolves D26 §Nul-termination Q-cstring (spec amend 2026-06-03).

fn str @to_cstr() -> CStr
// SAFE — always allocates new buffer + copy + NUL.
// Validates no embedded NUL (panic).

#unsafe
fn str @as_cstr_unchecked() -> CStr
// Skip embedded NUL validation. Caller promises str has no '\0' внутри [0..len).
// For perf-critical paths where validation cost matters.
```

**D73 auto-derive дает `.into()` форму бесплатно** для всех `try_from`/`from`:
```nova
ro c = my_str.into()              // → CStr (auto-derived from CStr.try_from)
unsafe {
    ro s = my_cstr.into()          // → str (auto-derived from str.try_from)
                                    // — auto-throws Fail[Utf8Error]
}
```

**Naming convention** (added к spec convention follow-on):
- `from_X` — safe (или auto-synthesized throwing form)
- `try_from_X` — fallible, explicit Result form (richest implementer form)
- `from_X_unchecked` — unsafe variant, skip validation BUT still copy
- `from_X_view` — unsafe variant, no copy (view + lifetime contract + skip validation)
- `as_X` — view if possible (zero-copy primitive)
- `to_X` — always materialize (copy)

### RawMem method declarations

Все pointer-touching methods требуют `#unsafe` attribute (Plan 118 D216 §9 —
caller must wrap в `unsafe { }`):

```nova
type RawMem                              // unit type namespace

#unsafe
fn RawMem.copy_from[T](src *T, dst *mut T, count int) -> ()
// memmove (overlap-safe). Mirror []T.copy_from semantics (spec/08-runtime.md §3267).

#unsafe
fn RawMem.copy_nonoverlapping[T](src *T, dst *mut T, count int) -> ()
// memcpy (undefined при overlap; perf-critical optimization).
// Rust precedent: ptr::copy vs ptr::copy_nonoverlapping.

#unsafe
fn RawMem.copy_within[T](p *mut T, src_offset int, dst_offset int, count int) -> ()
// memmove внутри одного буфера. Mirror []T.copy_within (§3268).

#unsafe
fn RawMem.fill[T](dst *mut T, value T, count int) -> ()
// Element-typed fill. Mirror []T.fill (§3269). Compile-time fast-path:
// u8/i8 → memset, wider T → scalar loop (auto-vectorizable).

#unsafe
fn RawMem.write_bytes[T](dst *mut T, byte_val u8, count int) -> ()
// memset(dst, byte_val, count * sizeof(T)).
// Generalized byte-pattern fill (covers fill_zero via byte_val=0).
// Rust precedent: ptr::write_bytes. Use cases: zero-init, debug patterns
// (0xCC/0xDEADBEEF), all-ones (-1) init.

#unsafe
fn RawMem.compare[T](a *T, b *T, count int) -> int
// memcmp. Returns 0=equal, <0=a less, >0=a greater.

#unsafe
fn RawMem.read[T](src *T) -> T
// Aligned single-element load (UB if src misaligned для T).
// Use case: read T from properly-aligned pointer без auto-deref ergonomic.

#unsafe
fn RawMem.read_unaligned[T](src *T) -> T
// Any-alignment safe load (~2x slower; copies bytes one-by-one если нужно).
// Use cases: binary protocol parsing (network packets, ELF headers),
// firmware images, #pragma pack(1) C structs.
// Rust precedent: ptr::read vs ptr::read_unaligned split.

#unsafe
fn RawMem.write[T](dst *mut T, value T) -> ()
// Aligned single-element store (UB if dst misaligned для T).

#unsafe
fn RawMem.write_unaligned[T](dst *mut T, value T) -> ()
// Any-alignment safe store. Rust precedent: ptr::write_unaligned.

#unsafe
fn RawMem.volatile_read[T](src *T) -> T
// MMIO / driver primitive. Compiler hint: do not optimize away, do not reorder.
// Distinct от Atomic* (which adds LOCK prefix / thread synchronization).

#unsafe
fn RawMem.volatile_write[T](dst *mut T, value T) -> ()
// MMIO / driver primitive.

fn RawMem.addr_of[T](x ro T) -> ptr      // NO #unsafe — pure read of address
// Returns raw `ptr` (opaque integer). NO escape analysis, NO auto-promote
// (vs `&x` which promotes к heap if escape detected). For address-as-identity
// (HashMap key, hash, debug ID).
```

**Все signatures используют `int` для count/offset (D226 signed indexing)**.
FFI boundary calls к C `size_t`-typed args требуют explicit `as usize` cast.

### Volatile vs Atomic differentiation

Documented в spec D216 §6 или new D217:

| Need | Use | Why not the other |
|---|---|---|
| Shared mutable state across fibers/threads | `Atomic{I8..I64,U8..U64,Ptr}` + MemOrdering | Atomic emits LOCK prefix (CPU sync), не подходит для MMIO |
| MMIO register / signal handler / volatile semantics | `RawMem.volatile_read/write` | Volatile is compiler hint only (no CPU sync); Atomic over-synchronizes |

### CStr edge cases — RESOLVED 2026-06-03 (Q-cstring closed)

**Research finding** (`nova_rt/nova_rt.h:117-136` + spec D26 amend):

Nova str allocator INVARIANT — full str always has `ptr[len] == '\0'`:
- All paths (`nova_str_concat`, `nova_str_to_upper/lower`, `string_builder.h`,
  `conv.h`, literals в `.rodata`) allocate `len + 1` + explicit `buf[len] = '\0'`.
- Documented в spec/decisions/08-runtime.md §«Nul-termination (C-interop)»
  (amended 2026-06-03 — Q-cstring closed).

**BUT substring views (D144 `s[a..b]`) — `view.ptr[view.len]` NOT guaranteed**:
- View shares parent's buffer; parent's NUL at `parent.ptr[parent.len]`
- For sub-views ending mid-buffer (e.g., `s[2..5]`), `view.ptr[view.len]` is
  arbitrary byte (e.g., space, next char), NOT `'\0'`.
- For sub-views ending at parent's len (e.g., `s[5..]`), `view.ptr[view.len]`
  == parent's terminating NUL.

**Resolution в `@as_cstr()`:**
```nova
fn str @as_cstr() -> CStr
// Runtime check: ptr[len] == 0?
//   TRUE → zero-copy view (CStr wraps view.ptr directly)
//   FALSE → allocate len + 1, copy + NUL (fallback к @to_cstr)
// Memory access at ptr[len] always safe — within parent's len+1 buffer.
```

Single runtime check (1 byte read + branch) — minimal overhead. Resolves
D26 Q-cstring open question.

Closes `[M-118.1-cstr-nul-invariant]` followup (no further audit needed).

### Plan 118.2 simplification (cross-impact)

Per design discussion: **NO new `Slice[T]` type** в Plan 118.2. Resolution per D144
single-type design ("`[]T` — один тип для owner и view"). Instead:
```nova
fn []T @as_ptr() -> *T                  // NEW — ro view of internal data ptr (FFI read)
fn []T mut @as_mut_ptr() -> *mut T      // NEW — mut view (FFI write, requires mut binding)
fn []T @len() -> int                    // exists (D32)
fn []T @cap() -> int                    // exists (D32)
```

**Naming `@as_ptr` / `@as_mut_ptr`** (not bare `@ptr` / `@ptr_mut`):
- Avoid visual confusion с `ptr` primitive type
- Match `as_X` view convention (zero-copy view)
- Rust precedent: `Vec::as_ptr()` / `Vec::as_mut_ptr()`

User accesses parts directly:
```nova
unsafe {
    ro view = arr[0..1024]                          // []T view (D144)
    read(fd, view.as_ptr() as *mut u8, view.len() as usize)
}
```

This is `[]T` parts extraction, not new slice type. Plan 118.2 refactored accordingly
(see Plan 118.2 design refinement section).

## Зачем

После Plan 118 core у нас есть typed `*T` family + auto-deref `p.field` /
`p.method()`. Но **memory access intrinsics** для FFI отсутствуют:

```nova
// Plan 118 core gives us:
unsafe {
    ro buf *u8 = some_ffi_call()
    ro byte = (*buf)              // ✓ one byte deref
    ro field = buf.field          // (if buf points to struct)
}

// What Plan 118 core CANNOT do:
unsafe {
    // memcpy 1024 bytes from src to dst
    src.copy_to(dst, 1024)        // ❌ no such method

    // read 4 bytes as i32 at typed offset
    ro x = base.offset(16).read()  // ❌ no .read() / .offset()

    // memory-mapped register I/O
    register_addr.write_volatile(0xDEAD_BEEF)  // ❌ no volatile

    // FFI с C-string (null-terminated)
    sqlite_open(cstr"db.sqlite3")  // ❌ no cstr"..." literal
}
```

**Сценарии без 118.1:**
1. **Bulk buffer copy** (image data, network packets) — user пишет C-shim per call
2. **Memory-mapped I/O** (hardware registers, framebuffers) — недоступно
3. **Typed reads из FFI buffers** (parse binary protocols) — manual cast hacks
4. **C-string FFI** (sqlite, curl, png — все принимают `const char*`) — boilerplate
   string-conversion в shim'ах
5. **Out-params** (`int func(out int* result)`) — нужен `addr_of_mut!`

### Mainstream comparison

| Язык | Bulk copy | Typed read/write | Volatile | C-string |
|---|---|---|---|---|
| **C** | `memcpy` / `memmove` | `*p = v` / `v = *p` | `volatile` qualifier | `"..."` (null-term по умолчанию) |
| **Rust** | `ptr::copy` / `copy_nonoverlapping` | `ptr::read` / `ptr::write` | `read_volatile` / `write_volatile` | `c"..."` literal (2024) / `CStr` |
| **Zig** | `@memcpy` builtin | `p.*` / `p.* = v` | `volatile` modifier (`@as(*volatile T, ...)`) | `[*:0]const u8` (sentinel-term) |
| **C#** | `Buffer.MemoryCopy` / `Unsafe.CopyBlock` | `Unsafe.Read<T>` / `Write<T>` | `Volatile.Read` / `Write` | `MarshalAs(UnmanagedType.LPStr)` |
| **Swift** | `UnsafeMutableRawPointer.copyMemory` | `pointee` accessor | `withUnsafePointer(volatile:)` | `withCString { ... }` |
| **D** | `core.stdc.string.memcpy` import | `*p` | `core.volatile` package | `toStringz` |
| **Go** | `copy(dst, src)` для slice; `memmove` via cgo | через `unsafe.Pointer` casts | `sync/atomic.LoadPointer` | `C.CString(...)` |
| **Nova V1** (Plan 118 core) | (нет) | one-byte `*p` only | (нет) | (нет — wrap в C-shim) |
| **Nova V2** (Plan 118.1) | `.copy_to()` / `.copy_to_nonoverlapping()` | `.read()` / `.write()` | `.read_volatile()` / `.write_volatile()` | `cstr"..."` literal + interop |

## Дизайн

### 1. Typed read/write primitives

```nova
unsafe {
    ro buf *u8 = malloc(16).unwrap()

    // Read 1 byte
    ro byte = buf.read()                       // u8

    // Read typed value (offset already encoded в pointer type)
    ro int_ptr = buf as *i32
    ro x = int_ptr.read()                       // i32 (4 bytes consumed)

    // Write
    int_ptr.write(0xDEAD_BEEF)
}
```

**API:**
- `(*T).read() -> T` — typed read; equivalent to `*p` но explicit form для clarity
- `(*mut T).write(v T) -> ()` — typed write; equivalent to `*p = v` но explicit

**Why duplicate `*p`?** Two reasons:
1. **Method form** интегрируется с `.copy_to()` / `.read_volatile()` chain
   pattern (mainstream familiarity)
2. **Explicit для `*unsafe T`**: nested unsafe wrap proxied через method call
   reads cleaner

### 2. Bulk copy primitives

```nova
unsafe {
    ro src *ro u8 = source_buffer()
    ro dst *mut u8 = dest_buffer()
    ro n usize = 1024

    src.copy_to(dst, n)                 // memmove (overlapping safe)
    src.copy_to_nonoverlapping(dst, n)  // memcpy (UB if overlap)
}
```

**API:**
- `(*ro T).copy_to(dst *mut T, count usize) -> ()` — memmove; handles overlap
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` — memcpy;
  faster но UB on overlap
- Both compile to direct `memmove` / `memcpy` calls (linker resolves
  к platform libc)

**Units:** `count` — element count (not bytes); compiler multiplies by
`sizeof(T)` for actual byte count.

**Type safety:** `src` и `dst` должны иметь same `T` (or one cast в unsafe).

### 3. Volatile reads/writes (memory-mapped I/O)

```nova
unsafe {
    ro reg *mut u32 = 0xFFFF_0010 as *mut u32   // memory-mapped register

    reg.write_volatile(0xDEAD_BEEF)              // visible to hardware immediately
    ro status = reg.read_volatile()              // reread (no compiler caching)
}
```

**API:**
- `(*T).read_volatile() -> T` — volatile read; compiler не cache'ит
- `(*mut T).write_volatile(v T) -> ()` — volatile write; immediate side-effect

**Codegen:** emit C `volatile T*` cast at access point:
```c
((volatile uint32_t*)reg)[0]               // read_volatile
*((volatile uint32_t*)reg) = 0xDEAD_BEEF;  // write_volatile
```

**Compiler guarantee:** no reordering, no caching, no elision (per C
`volatile` semantics). Critical для hardware registers + lock-free protocols.

### 4. `addr_of!` / `addr_of_mut!` macros

```nova
type Packed {
    a u8
    b u32                                // unaligned field!
}

unsafe {
    ro packed = Packed { a: 0xFF, b: 0xDEAD_BEEF }

    // ❌ &packed.b — может dereference unaligned value temporarily
    // ✓ addr_of!(packed.b) — pointer without deref
    ro p_b = addr_of!(packed.b)                  // *ro u32 (may be unaligned!)

    // Similarly для mut:
    mut packed_mut = ...
    ro p_b_mut = addr_of_mut!(packed_mut.b)      // *mut u32
}
```

**API:**
- `addr_of!(expr)` — macro/intrinsic; returns `*ro T` where T = type of expr
- `addr_of_mut!(expr)` — returns `*mut T`
- **Does not deref** the value — safe even on packed/uninit memory
- Compile-time check: expr must be lvalue (addressable)

**Why needed:** `&packed.b` semantically creates a `*T` через value access,
which requires aligned load (UB на packed structs). `addr_of!` provides
pointer construction без implicit load — Rust 1.51+ pattern, critical для
FFI с C-packed structs.

**Codegen:** emit direct `&(packed.b)` C expression (no temporary value
materialization).

### 5. C-string convention

C libraries универсально принимают/возвращают **null-terminated `const char*`**.
Nova `str` тип — length-prefixed (`{ data: *ro u8, len: usize }`); NOT
null-terminated. Conversion was C-shim boilerplate в Plan 115 V1.

**`cstr"..."` literal:**

```nova
unsafe {
    // Plan 115 V1 — needed C-shim:
    //   char path_buf[1024];
    //   memcpy(path_buf, path.ptr, path.len);
    //   path_buf[path.len] = '\0';
    //   sqlite3_open(path_buf, &db);

    // Plan 118.1:
    ro path = cstr"database.sqlite3"             // *ro u8, null-terminated literal
    sqlite3_open(path, &mut db)                  // direct FFI
}
```

**Literal semantics:**
- `cstr"foo"` — compile-time string literal с trailing `\0` byte
- Type: `*ro u8` (pointer to first byte)
- Stored в read-only segment (`.rodata`)
- Length: `4` bytes (`"foo\0"`); user code не sees the null
- **Cannot** contain embedded `\0` (compile error `E_CSTR_EMBEDDED_NULL`)

**Interop functions:**
- `(*ro u8).cstring_len() -> usize` — runtime `strlen` (counts to first `\0`)
- `(*ro u8).from_cstring() -> str` — copy bytes to GC-allocated Nova str
  (length determined by `strlen`)
- `str.to_cstring(buf *mut u8, buf_len usize) -> usize` — copy str + null
  terminator to provided buffer; returns bytes written (or err if buf too small)

**Why explicit `cstr"..."` vs auto-convert `"..."`:**
- `str` literals в Nova — Nova-managed (length-prefixed); auto-convert
  на каждом FFI call = silent heap allocation
- Explicit `cstr"..."` — compile-time string в `.rodata`, zero allocation,
  zero copy

**Limits:**
- `cstr"..."` — only ASCII/UTF-8 byte content (no escape для NULs)
- Maximum литерала length — 64KB (compile-time limit для `.rodata` sanity)
- Runtime `str.to_cstring(buf, len)` — for dynamic case с user-provided buffer

### 6. Out-param FFI pattern

Combining `addr_of_mut!` + `Option[*T]`:

```nova
external fn try_alloc(out *mut Option[*u8]) -> i64
//                       ^^^^^^^^^^^^^^^^^^^
//                       caller passes pointer to Option storage

unsafe {
    mut buf Option[*u8] = None
    ro rc = try_alloc(addr_of_mut!(buf))         // C fills *buf via address
    match buf {
        Some(p) => use_buffer(p),
        None    => Fail.throw(OutOfMemory),
    }
}
```

**Pattern documented в ffi-cookbook.md update (Plan 118.1 Ф.5).**

### 7. Alignment + size queries

```nova
unsafe {
    ro align = align_of[*ro u8]()                  // 1
    ro align32 = align_of[*ro u32]()                // 4
    ro size = size_of[*ro Account]()                // 8 (pointer-width)
    ro size_t = size_of[Account]()                  // size of Account record
}
```

**API:**
- `align_of[T]() -> usize` — compile-time constant; alignment of T
- `size_of[T]() -> usize` — compile-time constant; size of T

**Use case:** manual buffer allocation + alignment compliance for FFI.

## D-block changes

### D217 (NEW) — FFI memory access primitives + C-string convention

**Локация:** `spec/decisions/02-types.md` (после D216).

#### §1. Typed read/write

- `(*T).read() -> T` — typed read; equivalent to `*p`
- `(*mut T).write(v T) -> ()` — typed write; equivalent to `*p = v`
- All в unsafe context

#### §2. Bulk copy

- `(*ro T).copy_to(dst *mut T, count usize) -> ()` — memmove (overlap safe)
- `(*ro T).copy_to_nonoverlapping(dst *mut T, count usize) -> ()` — memcpy
- Units: element count (compiler scales by sizeof(T))

#### §3. Volatile

- `(*T).read_volatile() -> T` — no caching
- `(*mut T).write_volatile(v T) -> ()` — immediate side-effect
- Codegen: C `volatile` qualifier на access point

#### §4. Address-of intrinsics

- `addr_of!(expr) -> *ro T` — pointer без temp materialization
- `addr_of_mut!(expr) -> *mut T` — same, mut variant
- Critical для packed structs / uninit memory

#### §5. Alignment + size

- `align_of[T]() -> usize` — compile-time const
- `size_of[T]() -> usize` — compile-time const

#### §6. C-string convention

- `cstr"..."` literal — null-terminated, `.rodata`, type `*ro u8`
- Cannot contain embedded `\0` (`E_CSTR_EMBEDDED_NULL`)
- Maximum 64KB literal length (`E_CSTR_TOO_LARGE`)
- Interop: `.cstring_len()` (runtime strlen), `.from_cstring()` (copy to str),
  `str.to_cstring(buf, len)` (write null-term to buffer)

**Cross-ref:** D216 (typed pointer foundations), D26 (str type), D214 (ptr).

### D26 cross-ref (existing string types) — C-string convention paragraph added

Cross-ref existing D26 (str type) с new `cstr"..."` literal pattern и
interop conversion functions. No D26 amend — just §«FFI interop» paragraph.

#### Ф.2 — volatile read/write (commit пенd next-commit)

Codegen extension в compiler-codegen/src/codegen/emit_c.rs:
  - Typed-pointer dispatch arm (~line 19720) extended:
    - `read_volatile()` → emits `*((volatile T*)p)` C cast
    - `write_volatile(v)` → emits `((*((volatile T*)p)) = v, NOVA_UNIT)`

    - write_volatile → nova_unit

    on ro pointers (parallel к existing write enforcement)

**Architectural decision:** volatile is COMPILER-LEVEL qualifier — orthogonal
к atomics (Plan 103.2 fences handle CPU-level ordering). Mirrors Rust
precedent (`ptr::read_volatile` + `atomic::fence` distinct concerns). For
MMIO + multi-thread user combines volatile (compiler) + fence (CPU)
explicitly.

  - NEW arm `TypeRef::Pointer(_, _)` → 8 (uniform pointer ABI per D216 §1)
  - NEW arm `TypeRef::Mut(inner, _) | TypeRef::Unsafe(inner, _)` →
    transparent recurse on inner (Plan 118.5 V2 zero-cost wrapper invariant)

Coverage:
  - `size_of[*T]()` / `size_of[mut * T]()` / etc. → 8 bytes ✅
  - `size_of[unsafe int]()` → 8 (transparent recurse) ✅
  - `size_of[unsafe u8]()` → 1, `size_of[unsafe i32]()` → 4 ✅
  - Nested chains via recursive walk
