# Plan 138.3 — `Clone` protocol = deep/recursive (collections element-wise)

> **Создан:** 2026-06-10.  **Статус:** ✅ CLOSED ПОЛНОСТЬЮ — spec-complete + deep-collection-clone

> **Закрыт:** 2026-06-10 (spec) / 2026-06-11 (deep-impl via Plan 138.4).  **Эстимат:** ~0.5-1 dev-day.
> **Model:** Sonnet HIGH (codegen-light; spec + stdlib).
> **Триггер:** ревью Vec.clone (Plan 138.2 design-cleanup) — текущий Vec.clone shallow
> (bitwise `RawMem.copy`), а по Rust-rule `Clone` должен быть deep.
> **Gate:** после `wj7opwd1b` (design-cleanup) — общий файл vec_owned.nv.

## Решение (подтверждено пользователем 2026-06-10)

`Clone` — **deep / рекурсивный** (Rust-семантика). Collections клонируют **поэлементно**
через `T.@clone()`, с conditional bound **`T: Clone`** (как Rust `impl<T: Clone> Clone for Vec<T>`).
Закрывает класс aliasing-багов: shallow-clone коллекции ref-типов тихо разделяет вложенные
объекты → мутация «копии» бьёт оригинал.

**Различие (зафиксировать в спеке):**

| Операция | Семантика | Bound | Применение |
|---|---|---|---|
| `@clone()` | **deep** — рекурсивный клон элементов via `T.@clone()` | `T: Clone` | независимая копия |
| `from`/`extend`/`push`/`copy_from`/`realloc` | **shallow** value-copy (bit-copy; ref-T разделяет pointee) | любой T | move/перенос/построение буфера |

`RawMem.copy`/`copy_nonoverlapping` (bitwise) — **только** для shallow-контекстов, **не** для clone.

## Текущее состояние (баг)

- `Vec.clone()` — shallow: раньше `Vec[T].from(@as_slice())` (push-copy), сейчас (Plan 138.2
  `[M-138-rawmem-bulk-ops]`) `RawMem.copy` (bitwise). Оба shallow → `Vec[Point]` разделяет Point'ы.
- Прецедент deep в спеке: `fn Tree @clone() -> Self => match @ {...}` (02-types.md:3028) — user-типы
  клонируют рекурсивно. Значит protocol-контракт Clone = deep, но Vec его нарушает.
- HashMap.clone / Set.clone — **audit** (вероятно тоже shallow; line 3743 `HashMap.@clone`).

## Фазы

### Ф.1 — Spec: Clone = deep (~0.5h)

- Найти Clone protocol D-блок (std/prelude/protocols.nv decl + spec D-block; Clone переименован
  в Plan 137 D237 — отталкиваться оттуда). Зафиксировать:
  - `Clone` = deep/recursive; `@clone()` возвращает независимую копию.
  - Для **collections** (Vec/HashMap/Set): element-wise deep via `T.@clone()`, conditional bound
    `[T Clone]` (Rust `where T: Clone`).
  - **shallow value-copy** (from/copy_from/extend/push/realloc) — отдельная операция, любой T,
    bit-copy; для ref-T разделяет pointee. `RawMem.copy` — только здесь.
- Q-блок если нужно (open-questions): conditional-Clone-bound на generic-методах.

**Commit:** `spec(plan138.3): Clone protocol = deep/recursive; collections element-wise [T Clone]`

### Ф.2 — Vec.clone → deep (~2h)

`std/collections/vec_owned.nv`:
```nova
export fn Vec[T Clone] @clone() -> Self {
    mut out = Vec[T].with_capacity(@len)
    mut i = 0
    while i < @len {
        out.push(unsafe { @data[i] }.clone())   // per-element deep
        i = i + 1
    }
    out
}
```
- Убрать `RawMem.copy` из `@clone` (оставить в `@realloc_to`/`from`/`@copy_from`/`@append` — shallow корректен).
- Conditional bound `[T Clone]` — проверить, что bootstrap поддерживает bound на instance-методе
  (если нет — fallback: отдельный метод или document gap).
- Тест nova_tests/plan138_3/t1_vec_clone_deep.nv: `Vec[Point]`, `c = v.clone()`, мутировать `c[0]`
  поле → `v[0]` **не** меняется (доказывает deep). + value-T `Vec[int]` clone (deep==shallow для int).

**Commit:** `fix(plan138.3): Vec.clone deep per-element (T Clone bound), not bitwise`

### Ф.3 — HashMap/Set clone audit + fix (~2h)

- `HashMap.@clone` / `Set.@clone`: проверить shallow/deep. Если shallow — перевести на deep
  (element/value-wise via K/V `.clone()`, bound `[K Clone, V Clone]` / `[T Clone]`).
- Тесты deep для HashMap[str, Point] / Set[Point].

**Commit:** `fix(plan138.3): HashMap/Set clone deep element-wise`

### Ф.4 — Bound-implication audit + close (~1-2h)

- `Vec[T].clone()` теперь требует `T: Clone` — найти call-site'ы `.clone()` на `Vec[non-Clone-T]`,
  убедиться что T везде Clone (примитивы/records авто-Clone? — проверить auto-derive D73).
  Если есть Vec[T] без Clone, требующий clone — решить (auto-derive Clone для records / document).
- docs/simplifications.md, project-creation.txt, nova-private/discussion-log.md, memory.

**Commit:** `docs(plan138.3): Clone deep complete + bound audit`

## Followups
- `[M-138.3-clone-bound-unsupported]` ✅ **CLOSED (Plan 138.4 Ф.1 G-C, 2026-06-11, commits
  `88432dd6f02` + `363f4b53788`).** ROOT CAUSE отличался от изначальной гипотезы: не
  «монформизатор мис-диспатчит», а single-key last-wins `method_receivers["clone"]` instance-fallback
  (emit_c.rs ~22159 — lookup ТОЛЬКО по имени метода, игнорируя receiver-тип), который роутил unbound
  primitive-T `.clone()` в произвольный неродственный `@clone` (`Nova_WriteBuffer_method_clone` etc).
  FIX: `PrimBuiltin::Identity` variant для всех primitive-C-типов (nova_int/bool/f64/f32/char/str/byte/unit
  + int8_t..uint64_t — `.clone()` = bitwise self) + record/heap identity-clone arm раньше fallback'а
  (Nova_*-pointer receiver без `(type,"clone")` в all_methods → emit receiver unchanged); зеркало в
  `infer_expr_c_type`. Real user/synthesized `@clone` сохраняет precedence (в all_methods). Deep
  Vec[T Clone]/HashMap[K,V Clone]/Set[T Clone] clone восстановлены (commit `363f4b53788`) — снят блокер
  с G2/G4.
- `[M-138.3-autoclone-records]` — auto-derive `#impl(Clone)` для records РАБОТАЕТ (plan126_2 p3/p7);
  расширить покрытие если bound-audit вскроет gap (bare-record без `#impl(Clone)` → bound-error).
- `[M-138.3-clone-cycle]` — deep-clone циклических структур (Vec[Vec[T]] с shared) — out of scope V1.
