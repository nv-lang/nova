## Цель

После 99.1: probe-метод `fn Option[T] @my_map[U](f fn(T)->U) ->
Option[U] => match @ { Some(v) => Some(f(v)), None => None }` (с
**другим** именем, не сталкивающимся с inline emit `Option.map`)
компилируется и работает для произвольных (T, U) включая non-primitive.

## Что меняем

### Ф.1 — Extract `method_extra_subst` в reusable helper (~0.4 д)

**Где:** `emit_c.rs:16386-16494` — внутри user-generic dispatch'а.

**Что:** Вынести method-level inference логику в метод
`CEmitter::resolve_method_level_subst(&mut self, fn_decl: &FnDecl,
args: &[CallArg], receiver_subst: &[(String,String)]) -> Result<
Vec<(String,String)>, String>`.

Helper должен:
1. Set `current_type_subst` = receiver_subst (saved/restored).
2. Initialize `subst_slots: Vec<(String, Option<String>)>` из
   `fn_decl.generics`.
3. **Step 1** (non-closure args): standard `infer_type_param_binding`
   (Plan 98 расширенный) для каждого param + arg.
4. **Step 2** (ClosureLight args): pre-infer closure return type с
   typed var_types для params (substituted), recurse на return_type_ref.
5. Loud-error при unresolved method-level type-params (existing
   behavior, line 16465-16488).
6. Return `Vec<(name, c_ty)>` filtered.

**Refactor:** user-generic dispatch (`:16386-16494`) использует
`self.resolve_method_level_subst(&fn_decl, args, &type_subst)`
вместо inline кода.

**Regression:** `plan48_mpm` (7), `generics` (5), `plan88` (5),
любые user-generic-method тесты. Должно быть 0 дельт.

### Ф.2 — Integrate в Option DeclaredBody dispatch (~0.5 д)

**Где:** `emit_c.rs:14910-14964` (Plan 95 Ф.3.1 / Plan 95.bis).

**Что:**

1. После сборки receiver-only `type_subst` (line 14946),
   **before** `register_mono_method_instance`:
   ```rust
   let method_extras = if !fn_decl.generics.is_empty() {
       self.resolve_method_level_subst(&fn_decl, args, &type_subst)?
   } else {
       Vec::new()
   };
   for entry in &method_extras { type_subst.push(entry.clone()); }
   ```

2. **Расширить mono_name** с U-суффиксом (line 14948-14950):
   ```rust
   let base = format!("Nova_Option_method_{}_{}", method, elem_ty);
   let mono_name = if method_extras.is_empty() {
       base
   } else {
       Self::compute_mono_name(&base, &method_extras)
   };
   ```

3. **Trigger `register_novaopt_decl(U)`** для return-type Option[U]:
   - Если `fn_decl.return_type` это `Option[<type-param>]` и
     type-param ∈ method_extras names — register typedef для resolved U:
     ```rust
     let saved = std::mem::replace(&mut self.current_type_subst, ...);
     if let Some(ret) = &fn_decl.return_type {
         self.ensure_novaopt_for_typeref(ret);  // helper
     }
     self.current_type_subst = saved;
     ```

4. **Fix `infer_expr_c_type` для Option methods** (`:23215+`):
   - Для `map[U]` нужен U из closure-arg return. Использовать
     preview inference (как при dispatch).
   - Для `ok_or[E]` — E из arg `err: E`.
   - Helper: `infer_method_level_return_for_option(method, args,
     receiver_elem_c) -> String`.

**Verify:** probe `nova_tests/plan99_probe/my_option_map_probe.nv` —
PASS для int→int, int→str, str→User.

### Ф.3 — Integrate в Result DeclaredBody dispatch (~0.4 д)

**Где:** `emit_c.rs:15148+` (Plan 95.bis Ф.2 patched).

**Что:** Параллельно Ф.2:

2. mono_name suffix: `Nova_Result_method_<m>_<ok>_<err>_<U>...`.
3. `register_novares_decl(U_or_F, err_or_ok)` triggered для return.
4. `infer_expr_c_type` для Result methods (`:23227+`) — same fix.

**Verify:** `my_result_map_probe.nv` — PASS.

### Ф.4 — Probe fixtures (~0.1 д)

`nova_tests/plan99_probe/`:
- `my_option_map_probe.nv` — `fn Option[T] @my_map[U](f fn(T)->U)
  -> Option[U]`, тесты int→str, str→char, User-record→int.
- `my_result_map_probe.nv` — same для Result.
- `my_option_ok_or_probe.nv` — `fn Option[T] @my_ok_or[E](err E)
  -> Result[T, E]`.

Все имена с префиксом `my_` чтобы избежать C-redefinition с
существующими inline emit'ами.
