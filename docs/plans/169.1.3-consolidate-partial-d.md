## Что сделать

Для каждой папки категории (D):
1. **Вынести negative** (`EXPECT_COMPILE_ERROR`/`EXPECT_*`) в `neg/` подпапку
   (`module neg.<stem>`) — по конвенции test-conventions.md (Ф.8).
2. **Объединить pos-остаток** в folder-module `module nova_tests.<dir>`
   (ordinal-suffix rename при конфликтах).
3. Regress папки — 0 новых FAIL; negative по-прежнему standalone в `neg/`.

## Метод

Ручная или скриптовая обработка (расширить логику `scripts/catb_convert.py`,
которая уже умеет «положительные → folder-module, EXPECT_* → neg/»). Начать с
крупнейших по pos (`plan91_14` 16, `plan147` 14, `plan100_1`/`plan106` 9).

## Замечания

- `EXPECT_RUNTIME_PANIC` среди вынесенных — кандидаты на консолидацию через
  `panics`-клаузулу ([169.1.4](169.1.4-panics-clause.md)) ПОЗЖЕ; пока выносятся
  в `neg/` как standalone.
- Не трогать pos-файлы, которые на деле cross-resolve (проверять `import` на
  соседние peer внутри папки).
