## Phases

### Ф.1 — 3 missing negative tests

`nova_tests/negative_capability/`:

1. **`folder_inconsistent_decl/`** — folder где peers объявляют разные
   `module X` (нарушение spec rev-3). Entry test ожидает clear error.
2. **`folder_file_vs_folder_conflict/`** — `parent/admin.nv` + `parent/admin/`
   с direct `.nv` файлом. Должна быть compile error «ambiguous module
   admin: file vs folder».
3. **`folder_cycle_between_modules.nv`** — folder-module A импортирует
   folder-module B, B импортирует A. Rule A cycle detection должен
   ловить (текущий `import_cycle_rejected.nv` использует single-file
   modules, не folder).

**Acceptance:**
- 3 новых negative tests добавлены с `// EXPECT_COMPILE_ERROR <pat>`.
- Каждый PASS под `nova test`.

### Ф.2 — Diagnostic для ambiguous X.nv + X/

`compiler-codegen/src/imports.rs::resolve_module_paths`:

Текущее поведение — silent `None` когда есть и `X.nv` и `X/` (с direct
.nv). Заменить на explicit error в diagnostic с:
- Имя модуля `X`
- Путь к `X.nv`
- Путь к `X/`
- Hint: «remove one or rename».

**Acceptance:**
- Test from Ф.1 #2 (folder_file_vs_folder_conflict) passes.
- Error message содержит оба пути.

### Ф.3 — Rule H comment fix + Rule A semantic clarify

**Rule H** (`imports.rs:184-212`):

Code использует `import_chain.last()` — это **import path string**,
не canonical declared module name. Спека Plan 42 правило H:

> `<X>/internal/` доступен только из `<X>.*` descendants.

`<X>.*` — descendants по **declared module name** (D29 rev-3 — `parent.X`).
Текущая проверка через **import path prefix** работает, потому что
import path обычно coincides с declared module path. Но если модуль
reachable через re-export (Plan 35.A R26 `export import`), пути
расходятся.

**Действие:**
1. Update comment в `imports.rs:184-212` — описать что сейчас
   проверяется import path, не declared module.
2. Добавить TODO в коде с reference на этот sub-plan для будущего fix.
3. **НЕ менять логику сейчас** — re-export still rare, нет регрессии.
   Зафиксировать known limitation в simplifications.md.

**Rule A** (cycle detection):

Спека: `HashSet<Vec<String>>` keyed by module-name segments.
Реальность: `HashSet<PathBuf>` keyed by canonical path первого peer'а.

Symlinks / case-insensitive FS (NTFS) могут дать different paths для
same module → false-negative cycle. **Действие:**
1. Document в simplifications.md как [M11].
2. Negative test из Ф.1 #3 проверяет основной case (works).
3. Symlink edge case — separate plan если когда-нибудь stop'нет на
   реальном bug.

**Acceptance:**
- Comment in `imports.rs` updated to match actual behavior.
- `[M10]`/`[M11]` entries в simplifications.md.
