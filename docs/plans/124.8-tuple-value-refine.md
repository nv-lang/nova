### 1.2 Conceptual overlap ¬Ђstack vs heap¬ї —З–µ—А–µ–Ј —Б–Ї–Њ–±–Ї–Є

Nova вАФ –µ–і–Є–љ—Б—В–≤–µ–љ–љ—Л–є mainstream-—П–Ј—Л–Ї, –≥–і–µ **—Д–Њ—А–Љ–∞ —Б–Ї–Њ–±–Њ–Ї** –Ї–Њ–і–Є—А—Г–µ—В

- Go: ONE syntax + escape analysis.
- Rust: `()` vs `{}` вАФ –Њ–±–∞ stack, –Њ—В–ї–Є—З–∞—О—В—Б—П access pattern.
- Kotlin: `value class` keyword.
- Java Valhalla: `value class` (incoming).
- C#: `class` vs `struct` keyword.

User decision (–њ–Њ—Б–ї–µ discussion): —Г–љ–Є—Д–Є—Ж–Є—А–Њ–≤–∞—В—М —З–µ—А–µ–Ј **`value`
modifier** –і–ї—П records, **upgrade tuples** –і–Њ first-class data type
(no priv, multi-line OK).

### 1.3 New semantic axes (per user)

- **Tuples** = pure data carriers, all-public, stack-only, no priv,
  no per-field mut (binding-level mutability).
- **Records** = encapsulation-capable, heap by default, optional
  `value` keyword –і–µ–ї–∞–µ—В stack.
- **`ro`/`mut` —А–∞—Б–њ—А–Њ—Б—В—А–∞–љ—П—О—В—Б—П –њ–Њ default** –љ–∞ —В–Є–њ —Б–њ—А–∞–≤–∞ –≤ binding;
  explicit duplication вАФ `E_REDUNDANT_TYPE_MODIFIER`.

## 2. Design

### 2.1 Tuple syntax вАФ multi-line + restrictions

```nova
// Single-line (current)
type Vec3(x f64, y f64, z f64)

// Multi-line —Б commas (NEW)
type Vec3(
    x f64,
    y f64,
    z f64
)

// Trailing comma (NEW)
type Vec3(x f64, y f64, z f64,)

// Multi-line —Б trailing comma (NEW)
type Vec3(
    x f64,
    y f64,
    z f64,
)
```

**–Ч–∞–њ—А–µ—Й–µ–љ–Њ:**
- `priv` / `pub` modifiers –љ–∞ tuple field вЖТ `E_TUPLE_NO_PRIV`.
- `mut` / `ro` per-field вЖТ tuple fields –≤—Б–µ–≥–і–∞ –±–µ–Ј per-field modifier
  (binding-level mutability).

**–†–∞–Ј—А–µ—И–µ–љ–Њ:**
- Newlines –Љ–µ–ґ–і—Г –њ–Њ–ї—П–Љ–Є (–њ–Њ—Б–ї–µ comma вАФ for now keep comma required).
- Trailing comma –њ–Њ—Б–ї–µ –њ–Њ—Б–ї–µ–і–љ–µ–≥–Њ –њ–Њ–ї—П.

**Mutability via binding (Rust-style, NEW):**
```nova
ro p = Vec3(x: 1.0, y: 2.0, z: 3.0)
// p.x = 5.0    // вЭМ E_READONLY_FIELD вАФ ro binding

mut p = Vec3(x: 1.0, y: 2.0, z: 3.0)
p.x = 5.0       // вЬЕ вАФ mut binding allows ALL fields mut
```

### 2.2 Value-record вАФ unified syntax

```nova
// Heap record (current default, unchanged)
type Account {
    ro id u64
    mut balance f64
}

// Value record (stack-allocated, NEW)
type Vec3 value {
    mut x f64
    mut y f64
    mut z f64
}

// Composable —Б priv (Plan 124 carries over)
type Secret value priv {
    pub ro id u64
    key str
}

// Composable —Б consume вАФ auto-becomes consume (per user)
type Token value {
    consume raw [32]u8     // consume field вЖТ Token implicitly consume
}
```

### 2.3 Modifier ordering (canonical)

`type X [value] [consume] [priv] { ... }` вАФ **allocation вЖТ ownership вЖТ visibility** (outer to inner).

Parser **order-independent** (accept any order). Lint `W_NON_CANONICAL_TYPE_MODIFIER_ORDER` вАФ V2.

### 2.4 Binding mutability propagation rules (NEW, D33/D176 amend)

`ro`/`mut` –љ–∞ binding **–њ–Њ default —А–∞—Б–њ—А–Њ—Б—В—А–∞–љ—П–µ—В—Б—П** –љ–∞ —В–Є–њ —Б–њ—А–∞–≤–∞.
Explicit duplication = redundant error.

| –Ф–µ–Ї–ї–∞—А–∞—Ж–Є—П | –Я–∞—А—Б–Є—В—Б—П | –°–µ–Љ–∞–љ—В–Є–Ї–∞ |
|---|---|---|
| `ro x T` | вЬЕ default | binding ro, type implicit ro |
| `mut x T` | вЬЕ default | binding mut, type implicit mut |
| `ro x ro T` | вЭМ `E_REDUNDANT_TYPE_MODIFIER` | —В–Њ –ґ–µ —З—В–Њ `ro x T` |
| `mut x mut T` | вЭМ `E_REDUNDANT_TYPE_MODIFIER` | —В–Њ –ґ–µ —З—В–Њ `mut x T` |
| `ro x mut T` | вЬЕ NEW | binding ro, content mut (cannot reassign, can mutate) |
| `mut x ro T` | вЬЕ existing | binding mut, content ro (can reassign, cannot mutate) |

вЖТ –Ч–∞–Ї—А—Л–≤–∞–µ—В **partial implementation gap** (D176 ¬І¬Ђtype-modifier –≤ –ї—О–±–Њ–є
–њ–Њ–Ј–Є—Ж–Є–Є¬ї –і–µ–Ї–ї–∞—А–Є—А—Г–µ—В –≤—Б–µ 4, –њ–∞—А—Б–µ—А —А–µ–∞–ї—М–љ–Њ –њ—А–Є–љ–Є–Љ–∞–µ—В —В–Њ–ї—М–Ї–Њ 2).

### 2.5 D175 amend вАФ binding dominates

| Field declaration | `ro acc` binding | `mut acc` binding |
|---|---|---|
| `field T` | –њ–µ—А–µ–њ—А–Є—Б–≤–Њ–Є—В—М вЭМ, –Љ—Г—В–Є—А–Њ–≤–∞—В—М вЭМ | вЬЕ / вЬЕ |
| `ro field T` | вЭМ / вЭМ | вЭМ / вЭМ (always frozen) |
| `field ro T` | вЭМ / вЭМ | вЬЕ / вЭМ |
| `mut field T` | вЭМ / вЭМ (**binding dominates**) | вЬЕ / вЬЕ |
| `mut field ro T` | вЭМ / вЭМ (**binding dominates**) | вЬЕ / вЭМ |

**Changed:** `mut field` —В–µ–њ–µ—А—М –Э–Х "always-mutable" вАФ `ro acc` –±–ї–Њ–Ї–Є—А—Г–µ—В
–њ–µ—А–µ–њ—А–Є—Б–≤–Њ–µ–љ–Є–µ –Є –Љ—Г—В–∞—Ж–Є—О (Rust-style).

### 2.6 Value-record semantics

- **Allocation:** stack (inline struct –≤ callee frame).
- **Pass semantics:** copy on parameter pass (D32 amend).
- **Method receiver `@`:** pointer –љ–∞ stack-slot (mutates –Њ—А–Є–≥–Є–љ–∞–ї).
- **`[]Vec3` storage:** inline (Vec3 instances –ї–µ–ґ–∞—В —А—П–і–Њ–Љ –≤ –Љ–∞—Б—Б–Є–≤–µ).
- **Reference fields:** —Е—А–∞–љ—П—В—Б—П –Ї–∞–Ї handles inline (ptr+len+cap –і–ї—П `[]T`,
  ptr+len –і–ї—П `str`); data –љ–∞ heap (GC-tracked).
- **Fixed array fields:** fully inline (`[32]u8` = 32 bytes inline).
- **Auto-equality:** —З–µ—А–µ–Ј `#impl(Equatable)` вАФ **Plan 126**, –љ–µ 124.8.

### 2.7 Tuple semantics (existing + clarification)

- **Allocation:** stack (–Ї–∞–Ї Plan 120).
- **Pass semantics:** copy on parameter pass.
- **Method receiver:** pointer (–Ї–∞–Ї value-record).
- **Mutability:** binding-level (`mut p` вЖТ –≤—Б–µ –њ–Њ–ї—П mut; `ro p` вЖТ frozen).
- **All-public:** `priv` / `pub` –Ј–∞–њ—А–µ—Й–µ–љ—Л.
- **No per-field modifier:** consistent —Б Rust.
- **Methods supported:** `fn Vec3 mut @rotate()` –≤–∞–ї–Є–і–µ–љ.

### 2.8 Composability matrix

| Modifier | Tuple `()` | Record `{}` | Value record `value {}` |
|---|---|---|---|
| `priv` (Plan 124) | вЭМ `E_TUPLE_NO_PRIV` | вЬЕ | вЬЕ |
| `consume` (Plan 100) | вЭМ (tuple is value data, no ownership) | вЬЕ | вЬЕ |
| `mut`/`ro` per-field | вЭМ | вЬЕ | вЬЕ |
| `#test_access` (Plan 124.6) | n/a (no priv) | вЬЕ | вЬЕ |
| `#visible_to` (Plan 124.6) | n/a (no priv) | вЬЕ | вЬЕ |
| Methods `fn T @method()` | вЬЕ | вЬЕ | вЬЕ |
| Generic `T[A, B]` | вЬЕ | вЬЕ | вЬЕ |
| Protocol `#impl(P)` | вЬЕ | вЬЕ | вЬЕ |
