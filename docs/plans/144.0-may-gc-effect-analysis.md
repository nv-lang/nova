## 1. Зачем это, и почему сейчас (вне очереди Ф.0)

Plan 144 §7.6 перечисляет soundness-дыры precise-GC. **H4** — самая опасная:

> **H4 (high) — may-GC: косвенные/FFI-callback рёбра.** First-class замыкания
> (`(*fn)(env,…)`) + C→Nova callbacks — fixpoint может пропустить ребро → ложный NoGC →
> **UAF**. Фикс (soundness): дефолт решётки = **MayGC (top)**; NoGC доказывать.

Если may-GC-вывод неверен, **весь** codegen Ф.2+ (shadow-frame elision, отказ от
write-back перед `#no_gc`/pure-вызовами — тир O1) строится на песке: пропущенный
safe-point = пропущенный корень = use-after-free. Поэтому доказать корректность
may-GC-вывода **до** того, как на него обопрётся эмиссия кадров — настоящий de-risk.

**Почему можно делать сейчас, в отрыве от остального Ф.0:**
- Анализ **чисто compile-time** и **ничего не эмитит** в генерируемый C → нулевой
  риск для рантайма, не зависит от стабильности scheduler.
- Переиспользует уже готовую инфраструктуру
  [`preempt_keep.rs`](../../compiler-codegen/src/codegen/preempt_keep.rs) (Plan 143.2):
  call-graph, `fn_key`, overload-резолюция, итеративный Tarjan SCC.
- Результат — верифицируемый артефакт (`nova gc-effect-analyze` CLI + фикстуры),
  который Ф.2 потом **потребляет напрямую** (driver тира O1).

## 2. Что это: решётка эффекта may-GC

Каждой функции (на уровне исходного шаблона, см. §3 про мономорфизацию)
присваивается эффект из двухточечной решётки:

```
        MayGC   (top, ⊤)   ← ДЕФОЛТ (soundness — H4)
          │
        NoGC    (bottom, ⊥) ← доказывается
```

- **NoGC** — функция **доказуемо** не запускает GC: не аллоцирует сама И каждый
  достижимый вызов доказуемо NoGC.
- **MayGC** — всё остальное. **Любое сомнение → MayGC** (H4: дефолт = top).

`is_no_gc(f)` истинно **только** когда весь статический конус вызовов из `f`
полностью разрешён по именам И не содержит ни одной аллокации.

## 3. Алгоритм (переиспользует preempt_keep.rs)

### 3.1. Узлы и рёбра (как в preempt_keep)
- **Узлы** — не-external `FnDecl`, ключ `fn_key` (receiver::name::param-sig).
- **Рёбра** — прямые разрешённые вызовы (с overload-резолюцией по
  approx-arg-signatures — переиспользовать `approx_arg_sig`/`add_candidate_edges`).

### 3.2. Seed: `self_may_gc(node)`
Узел сам по себе MayGC, если ИСТИННО любое из:
1. **Аллоцирует** — тело содержит аллоцирующее выражение (см. §3.3).
2. **Косвенный вызов** (`makes_indirect`) — closure/fn-ptr/метод-на-не-self/
   trailing-block/`with`/`spawn`/`select`/`parallel-for` и пр. (как в preempt_keep:
   цель неизвестна → не можем доказать NoGC цели → top).
3. **FFI/extern** (`makes_ffi`) — внешняя функция; её may-GC неизвестен → top.
4. **Неразрешённый callee** — bare-ident/Type.method/module.func, не сматченный ни
   к одному known `FnDecl` (cross-module / closure-local) → top.

> NB: `address_taken` сам по себе НЕ делает узел MayGC (быть взятым по адресу — не
> значит аллоцировать). Он влияет лишь на КОЛЛЕРА, который делает indirect-вызов
> (там сработает `makes_indirect`). Это отличие от preempt_keep, где address_taken
> входил в KEEP-набор. Учесть при переносе.

### 3.3. Детекция аллокации (`allocates(expr)`)
**Дизайн-принцип (soundness): allowlist provably-non-allocating, всё прочее → аллоцирует.**
Конструкции, которые в текущем codegen эмитят GC-аллокацию (черновик, Ф.0 уточняет
по `emit_c.rs`):
- `RecordLit` (heap-record), sum-конструкторы, `ArrayLit`, `MapLit`,
  `InterpolatedStr` / str-конкатенация (буфер), `Lambda`/`ClosureLight`/`ClosureFull`
  (env), `Spawn`/`Detach`/`Blocking`/`Supervised`/`ParallelFor` (fiber/runtime),
  boxing escaping-значений, vec/StringBuilder/Map конструкторы и `.clone()` на
  heap-типах, range-в-коллекцию.
- **Provably non-allocating** (allowlist): целочисленная/плавающая арифметика,
  сравнения, доступ к полю/индексу без копии, `as`-касты скаляров, literal-скаляры,
  return/break/continue, чтение локала. **str-литерал** — Ф.0 решает: интернированный
  `static const u8[]` НЕ аллоцирует, но при сомнении → аллоцирует (conservative).
- Любой **неизвестный** AST-узел → считать аллоцирующим (top).

### 3.4. Транзитивная пропагация (новое vs preempt_keep)
preempt_keep считал KEEP локально + цикл. may-GC требует, чтобы MayGC **текла вверх
по коллерам**: если `g` MayGC и `f` вызывает `g`, то `f` MayGC.

Реализация — **конденсация SCC + обратный топо-проход** (переиспользовать
`cycle_members`/итеративный Tarjan, расширив до полного SCC-разбиения):
1. Построить SCC-компоненты графа.
2. SCC помечается MayGC, если: **любой** его член имеет `self_may_gc=true`, ИЛИ
   **любое** исходящее ребро (в ДРУГУЮ SCC) ведёт в MayGC-SCC.
3. Обработать SCC в обратном топологическом порядке (callee → caller).
4. `NoGC = {узлы, чья SCC не помечена MayGC}`.

**Цикл с аллокацией:** рекурсивная SCC, где хоть один член аллоцирует → вся SCC
MayGC (член `self_may_gc` ⇒ компонент MayGC). Чистая рекурсия без аллокаций и без
внешних MayGC-рёбер → NoGC.

### 3.5. Соундность над мономорфизацией
Тот же аргумент, что в preempt_keep (§ комментарий в `preempt_keep.rs`):
свойства seed (аллокация / indirect / FFI / unresolved) — **исходные**; KEEP/MayGC,
вычисленный на шаблоне, наследуется КАЖДЫМ мономорфизированным/erased-инстансом.
Over-approximation рёбер ⇒ over-approximation MayGC ⇒ ни один реальный
may-GC-инстанс не пропущен. Spurious-рёбра лишь добавляют MayGC (теряем элизию,
остаёмся соундны).

### 3.6. Conservative-gating (как PreemptKeepSet)
`MayGcSet { may_gc: HashSet<FnKey>, populated: bool }`.
`is_no_gc(key) = populated && !may_gc.contains(key)`.
Если `populated == false` (анализ не прогонялся над непустой вселенной) — **никто не
NoGC** (всё MayGC). Зеркалит `PreemptKeepSet::populated`-контракт.

## 5. Критерии приёмки
1. **Дефолт решётки = MayGC (top).** NoGC доказывается только при полностью
   разрешённом + неаллоцирующем статическом конусе. (Закрывает H4 в этом слайсе.)
2. **Соундность над мономорфизацией** — §3.5; покрыто generic/erased-негативом.
3. **Транзитивная пропагация корректна** — SCC-конденсация, обратный топо; покрыто
   позитивом (NoGC-форвардер) и негативом (аллок в callee → caller MayGC).
4. **Emit-nothing** — нулевое изменение генерируемого C; полная сборка + регрессионный
   сэмпл зелёные (поведение рантайма не меняется).
5. **CLI** `gc-effect-analyze` работает (json + text), корректные exit-коды.
6. **Позитивные + негативные фикстуры** проходят через **РЕЛИЗНЫЙ nova & компилятор**.
7. **Adversarial review** (≥3 soundness-линзы: indirect/FFI-дыра, направление
   пропагации, SCC-с-аллокацией, default-top) — все находки разрешены.
8. **«Без упрощений как для прода» — ОБЯЗАТЕЛЬНЫЙ критерий.** Никаких stub'ов/TODO-
   срезов; conservative-MayGC при любом сомнении; soundness > агрессивность элизии.
