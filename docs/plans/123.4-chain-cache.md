## 1. Контекст

D217 (Plan 123.1) caches `@<field>` direct accesses. Hot-path methods
часто accessing nested chains `@parent.inner.cfg` multiple times.
Codegen эмитит `nova_self->parent->inner->cfg` каждый раз — N
pointer derefs × M occurrences.

V4 chain caching caches intermediate path segments:
- Cache `@parent` (D217 baseline).
- Additionally cache `@parent.inner` if accessed ≥ threshold times.
- Cache `@parent.inner.cfg` if accessed ≥ threshold times.

---

## 2. Design

### 2.1 V4 scope

**Targeted:** chain access patterns `Member { obj: Member{obj:
SelfAccess, name: A}, name: B }` (= `@a.b`) и deeper.

**Heuristic:** cache subpath if accessed ≥ threshold times; nested
depth limit ≤4 (avoid stack bloat).

### 2.2 Eligibility

- Each path prefix counted: `@a` (D217 baseline), `@a.b`, `@a.b.c`.
- Cache emit для prefixes с count ≥ threshold (default 2).
- Depth limit: 4 (V4 conservative).
- No mutation of `@a` anywhere in body (top-level field; nested
  field mutation rare и tracked separately).
- No closure capture / no concurrent body.

### 2.3 Naming

Cache local = `_at_<a>_<b>_<c>_chain` для chain prefix `@a.b.c`.
Distinct от D217 `_at_<F>` (single-level).

### 2.4 Composition с D217+D218+D219

V4 phase runs **between** D218 LICM и D219 pure-cache:
- LICM hoists invariant @F reads.
- V4 chain detection sees post-LICM AST с potentially hoisted @F.
- D219 pure-cache then handles pure-method calls.
- D217 final per-fn cache fills in remaining @F.

Composition can produce nested cache chains: `_at_parent` (D217) +
`_at_parent_inner_chain` (V4) — both pointing into same object,
register reuse handles efficiency.

V4 simpler approach: emit chain locals as bindings via path access
through D217 single-field-cache locals. E.g., если D217 caches
`_at_parent = @parent`, then V4 emits `_at_parent_inner_chain =
_at_parent.inner` (chain through cache rather than direct field
access).

For V4 simplicity: emit pure path-prefix bindings always referring
to original `@a.b.c` (not D217 chained). Easier composition.
