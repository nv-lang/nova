# Plan 118.5 — universal right-binding rule + `unsafe T` first-class migration

> **Создан:** 2026-06-04. **Статус:** 🆕 PLANNED.
>
> **Расширяет:** D216 V2 amend (spec/decisions/02-types.md). **Breaking
> change** для `*ro T` / `*mut T` / `*unsafe T` syntax.
>
> **Зависит от:** Plan 118 V1 (foundational FFI), Plan 118 Ф.4 V1 (typed
> read/write).

---

## Что

Унификация type-level modifier syntax под единое правило «right-binding»:
modifier применяется ко всему справа до конца type-expression или до
следующего modifier. Параллельно — `unsafe T` становится first-class type
wrapper с MaybeUninit semantic.

См. D216 V2 amend §V2.1-V2.7 для полной спецификации.

## Почему

Текущая грамматика типов имеет **два правила** одновременно:
- `ro T` parses как right-binding (`Readonly(T)` через recursive `parse_type`)
- `*ro T` / `*mut T` / `*unsafe T` parses как inline-modifier (modifier внутри
  pointer constructor)

Inconsistency дискомфортна для users (intuition диктует одно правило для
всех modifiers) и закрывает orthogonal expressiveness:
- `unsafe * T` (unsafe pointer) ↔ `* unsafe T` (valid ptr to uninit T) —
  невозможно express под current inline-modifier scheme

Migration cost линейно с FFI footprint — fix НОВЫЕ deferred plans (118.1
Ф.2-Ф.4 / 118.2 / 118.3) под новый syntax до того как accumulate'нутся
fixtures под старый.

| Декларация | Парсится | Семантика |
|---|---|---|

| `mut x T` | ✅ | binding mut, type implicit mut |
| `unsafe x T` | NEW | binding unsafe, type implicit unsafe |
| `consume x T` | NEW (после type-modifier generalization) | binding consume |
| `ro p mut * T` | NEW (V2 chain) | binding ro, pointer mut |
| `mut p * unsafe T` | NEW | binding mut, ptr to uninit T |
