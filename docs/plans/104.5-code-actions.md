## Sub-decomposition

### 104.5.1 — Quick-fix framework + machine-applicable infrastructure (~1 dev-day)

**Задача:** фундамент для всех quick-fix'ов.

**Компоненты:**
1. `diagnostic_mapping.rs` AMEND — извлекать `[E_CODE]` из начала diagnostic message → set `Diagnostic.code = Some(NumberOrString::String("E_CODE"))`.
2. `nova-lsp/src/code_actions.rs` — новый модуль: `compute_code_actions(src, diagnostics, range)`.
3. `server.rs` AMEND — вызывать `compute_code_actions` из handler `code_action`.
4. Утилиты: `extract_line_range(src, byte_off)`, `line_text(src, line)`, Levenshtein distance.

**Acceptance:** LSP клиент получает `code` в каждом diagnostic; `textDocument/codeAction` возвращает non-null для любого из ≥25 поддерживаемых E_* кодов.

## D-блок: D296 — CodeAction machine-applicable contract

**Location:** `spec/decisions/D296-code-action-contract.md`

Extends D102 (Suggestion API):

- LSP `CodeAction.kind` → `QuickFix` (primary), `Refactor` (renames), `Source.organizeImports`.
- `isPreferred: true` для machine-applicable; `false` для placeholders.
- `data: null` (no deferred resolve in V1).
- Performance budget: `textDocument/codeAction` ≤ 100ms (measured from diagnostic match to response).
- Error code extraction: `[E_CODE]` prefix in diagnostic message → `Diagnostic.code = String("E_CODE")`.
- Suggestion spans from `Diagnostic.suggestion` field used for MachineApplicable edits.
