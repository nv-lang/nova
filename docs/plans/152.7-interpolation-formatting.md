## Контекст (что есть)

`"Hello ${n}"` / `"Hello ${n:?}"` — интерполяция (D44, закрыта Plan 17 Ф.4):
- Lexer/parser → AST `InterpolatedStr { parts: [Lit | Expr{expr, spec}] }`.
- `FormatSpec::None` → `Display.@display(mut sb StringBuilder)` (D183);
  `FormatSpec::Debug` (`${e:?}`) → `Debug.@debug(mut sb StringBuilder)` (D229).
- Codegen → **StringBuilder-цепочка** (одна аллокация, pre-size estimate).
- Const-инициализатор: интерполяция запрещена (нужен runtime builder).

**Два ограничения сегодня:** (1) формат-спеков только два (`None` и `:?`) — нет
width/precision/fill/align/radix, как в Rust/Python/Kotlin/Java/JS; (2) `@display`/
`@debug` жёстко привязаны к **конкретному** `StringBuilder` → каждый `Display`-тип
импортит StringBuilder, и формат нельзя писать в произвольный sink (stdout, файл)
без промежуточной строки.

## Фаза A — сохранить через рефактор (входит в 152.0, дублируется тут как контракт)

- Реструктуризация модуля (152.0) **не ломает** интерполяцию: модульный DAG
  ацикличен (`str ↔ StringBuilder` без цикла, см. 152.0), facade держит
  `StringBuilder`/`WriteBuffer` публичными, golden `"${n}"`/`"${n:?}"`.
- **Acceptance A-part:** см. 152.0 A0.7.

## Фаза B — улучшения (D258)

### B1. Расширить формат-спеки (паритет Rust/Python/Kotlin/Java/JS)
`FormatSpec` из `{None, Debug}` → грамматика `${expr:[[fill]align][sign][#][0][width][.precision][type]]}`:
- **align/fill:** `<`/`^`/`>` + fill-char (`${x:>8}`, `${x:*^10}`).
- **width / precision:** `${x:8}`, `${pi:.2}`, `${s:.5}` (усечение).
- **sign / alt / zero:** `${n:+}`, `${n:#x}`, `${n:08}`.
- **type/radix:** `:x`/`:X`/`:o`/`:b` (radix), `:e` (экспонента), `:?` (Debug, есть).
Прецедент: Rust `format!`, Python f-string mini-language, Kotlin/Java `String.format`,
JS — слабее (только `${}`), Nova будет **сильнее JS**.

### B2. Обобщить sink форматтера (decouple Display от StringBuilder)
`@display(mut sb StringBuilder)` → `@display(mut w Write)`, где `Write` — протокол-
интерфейс (реализуют `StringBuilder`, `WriteBuffer`, stdout-writer). Даёт:
- **меньше перекрёстных ссылок** (Display-типы зависят от интерфейса `Write`, не от
  конкретного string-модуля);
- **direct-to-sink** форматирование (`print(x)` без промежуточной str-аллокации);
- прецедент: Rust `fmt::Write`/`fmt::Formatter`, Go `io.Writer`, Java `Appendable`.
- **Цена:** breaking — сигнатуры всех `@display`/`@debug` меняются (`sb` → `w`);
  массовая, но механическая миграция. Оценить отдельно (может стать 152.7.1).

### B3. Pre-size estimate через `byte_len`
Оценка ёмкости StringBuilder-цепочки — суммой `byte_len()` литералов + эвристика на
`${expr}` → меньше realloc'ов.

## Q-format-spec — РЕШЕНИЕ (закрыть в плане)

Грамматика формат-спеков — **Rust-style** mini-language
(`:[[fill]align][sign][#][0][width][.prec][type]]`), не Python/printf (ближе к
системному стилю Nova, однозначный парс). `:?` (Debug) сохраняется. Расширение —
амендмент D44/D183/D229, без смены синтаксиса `${...}`. Обобщение sink (`Write`) —
отдельное решение B2 (может быть отложено за breaking-ценой).

## Тесты

- **POS:** `"${n:8}"`/`"${n:>8}"`/`"${n:*^10}"`/`"${pi:.2}"`/`"${n:+}"`/`"${n:#x}"`/
  `"${n:08}"`/`"${s:.3}"` (усечение) дают ожидаемый вывод; `"${e:?}"` (Debug)
  не сломан; интерполяция с линзами/новым `str` (152.1) работает; (B2)
  `x.display(wb)` в `WriteBuffer` == в `StringBuilder`.
- **NEG:** невалидный спек `"${x:zz}"` → compile-error `E_BAD_FORMAT_SPEC` с fix-it;
  precision на не-поддерживающем типе → ошибка; интерполяция в const-инициализаторе →
  прежняя compile-error.

## Критерии приёмки

- **G0 (без упрощений, production-grade)** ✅ для B1 — полный Rust-style mini-language
  (fill/align/sign/`#`/zero/width/`.precision`/type), не подмножество; negative hex =
  two's-complement; alt-префикс всегда строчный (`:#X`→`0xFF`). Прогон **релизными**
  `nova`+`nova-codegen`.
- **A1.** ✅ Формат-спеки B1 реализованы (Rust-style): `t1` width/align, `t2`
  zero-pad/sign, `t3` radix (x/X/b/o + `#` + width/zero), `t4` precision (f64 + str
  усечение) — все PASS.
- **A2.** ✅ `${e:?}` (Debug) и bare `${e}` (Display) не регрессируют (D229-путь цел).
- **A3.** ✅ Невалидный спек → compile-error (`E_FORMAT_SPEC_UNKNOWN`/`E_BAD_FORMAT_SPEC`)
  с fix-it; негативный тест `n_bad_format_spec`.
- **A4.** ✅ Интерполяция совместима с `str`/линзами (152.1) и DAG (152.0) — без цикла.
