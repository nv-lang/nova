## 1. Контекст

[Plan 83.2 Ф.1](83.2-mn-default-flip.md) (2026-05-23) подготовил
инфраструктуру default-on M:N: public API `nova_runtime_auto_arm()` +
защитный auto-arm в `nova_runtime_spawn_global` / `spawn_into`
(коммиты `b72ce59b475` + `0af6e6ba482`, в main).

Активация флипа — одна строка `nova_runtime_auto_arm()` в
`emit_c.rs::emit_main_wrapper` (закомментирована). Прямая активация
вскрыла **9 pre-existing M:N багов** в supervised-drain layer,
проявлявшихся до 83.2 только при explicit `runtime.init`.

Plan 83.4 — целевая серия фиксов этих 9 багов **до уровня
production-grade паритета с Go runtime / tokio / Kotlin coroutines**.
После — флип выполняется одной строкой codegen-emit'а.

## 2. Паритет с другими M:N runtime — целевая планка

Архитектурные элементы, которые ДОЛЖНЫ быть в Nova M:N после Plan 83.4
(аналог в Go/tokio/Kotlin указан явно):

| Элемент | Go runtime | tokio | Kotlin coroutines | Node.js | **Nova цель** |
|---|---|---|---|---|---|
| Fiber atomic status state machine | `g.atomicstatus` (`_Gidle`/`_Grunnable`/`_Grunning`/…), CAS-transitions | `Header.state` bit-packed atom (NOTIFIED/RUNNING/COMPLETE/CANCELLED) | `JobState` atomic | n/a (single-thread) | `NovaFiberStatus` enum + CAS [83.4.2](83.4.2-supervised-drain-ownership.md) |
| Park с предикатом (cond-wait) | `gopark(unlockf, …)` — unlockf служит атомарным предикатом | `Notify` + `Waker` (atomic state) | `suspendCoroutine` + atomic CAS | n/a | `nova_sched_park_until(cond_fn, ctx)` [83.4.1](83.4.1-d93-park-wake-hardening.md) |
| Per-fiber state (handler/context) | `g`-struct (TLS-like per-goroutine) | `task_local!` macro, `TaskLocal<T>` | `CoroutineContext` (kotlinx) | `AsyncLocalStorage` | `NovaFiberMeta.handler_snapshot[]` в `mco_coro` user_data [83.4.2](83.4.2-supervised-drain-ownership.md) |
| Hierarchical cancellation | `context.Context` + `WithCancel(parent)` | `CancellationToken` tree | `Job` cancellation cascade | `AbortController` + `AbortSignal` | scope-tree atomic cancel + wake-all [83.4.3](83.4.3-mn-api-semantic-alignments.md) |
| Cooperative yield | `runtime.Gosched()` | `tokio::task::yield_now()` | `yield()` suspend fn | `await new Promise(r => setImmediate(r))` | `runtime.yield()` с правильной семантикой на main [83.4.3](83.4.3-mn-api-semantic-alignments.md) |
| Runtime introspection | `runtime.NumGoroutine()` + pprof | `RuntimeMetrics` (queue depths, steal counts) | `Dispatchers.IO.toString()` (debug) | `process.eventLoopUtilization()` | `fibers.slots_active()` global aggregation + queue depths [83.4.3](83.4.3-mn-api-semantic-alignments.md) |
| Data-race detection в CI | `go test -race` (TSAN) | `cargo +nightly miri test` / loom | Coroutines не имеют (Kotlin/JVM) | n/a | clang `-fsanitize=thread` на Linux в acceptance [83.4.4](83.4.4-flip-activation-and-83.2-closure.md) |
| Speedup acceptance bench | benchmark на parallel sum / parallel hash; ≥0.9× линейный | `criterion` benchmarks | `kotlinx-benchmark` | n/a | `bench/micro/parallel_*` с targets ≥0.7× linear, ≥0.85× Go [83.4.4](83.4.4-flip-activation-and-83.2-closure.md) |

**Что Nova даст ЛУЧШЕ:** algebraic effects (handler-snapshot per-fiber
— `tokio::task_local!` / `AsyncLocalStorage` это плоский per-task store,
Nova поддерживает scoped handler-with{} с явной cascade-семантикой).

**Что Nova даст НЕ ХУЖЕ:** scheduler (Plan 44.5 work-stealing деque
PPoPP 2013 — тот же подход, что Go `runqsteal`), GC (Plan 82 Ф.3 multi-
arena push-callback корректен под conservative GC — tokio/Go используют
precise GC что проще). Spawn-cost (Plan 82 Ф.5: 16-20 ns/switch — паритет
с Boost.Context).

**Где Nova продолжает быть строже Go:** настраиваемый, не безграничный
потолок fiber'ов (Plan 82 §3: 16K слотов × N workers; Go — миллионы
горутин). Снятие — arena-chaining (отдельный followup, вне 83.4).

## 5. Production-grade acceptance (umbrella)

### Корректность

- 9 регрессирующих concurrency-тестов из `[M-83.2-supervised-mn-bugs]`
  проходят под активным флипом (без `runtime.init` в теле теста).
- Полный clang `nova test` после 83.4.4 Ф.2 (активация): ≥ 1071 PASS /
  0 FAIL / 56 SKIP.
- **TSAN gate на Linux**: clang `-fsanitize=thread -fno-omit-frame-pointer`
  на `nova_tests/concurrency/*` — 0 reported races (run в [83.4.4](83.4.4-flip-activation-and-83.2-closure.md) Ф.5).

### Производительность

- **Stress/soak** (acceptance bench in [83.4.4](83.4.4-flip-activation-and-83.2-closure.md) Ф.4):
  - 10⁶ spawn × no-op fiber — без crash/leak (Boehm GC stats).
  - 10⁵ park-wake cycles в tight loop — без abort.
  - 10⁴ supervised-scopes × 100 fibers с work-stealing migration —
    без deadlock/race.
- **Context-switch** не регрессирует от Plan 82 Ф.5 baseline (16-20 ns
  arena / clang-cl + MSVC).
- **Speedup на multi-core CPU-bound** (`bench/micro/parallel_*`):
  - parallel sum / parallel hash на N=4 ядрах: **≥ 3.0× speedup** vs
    `NOVA_MAXPROCS=1` (т.е. ≥ 0.75× линейного).
  - Сравнение с Go на эквивалентной нагрузке: **≥ 0.85×** Go time
    (не хуже 15% от Go).

### Spec / документация

- spec `D-block`(s) в `spec/decisions/06-concurrency.md`:
  - D93 amendment (park-with-predicate API);
  - D-new по «M:N как default model + NOVA_MAXPROCS» (+ cross-runtime
    comparison table из §2 этого плана);
  - D-new по hierarchical cancellation token (если решим выносить
    semantics в spec, а не runtime-only).
- `docs/plans/README.md` обновлён — Plan 83.2 ✅, Plan 83.4 ✅, Plan 83 ✅.
- `[M-83.2-supervised-mn-bugs]` → ✅ ЗАКРЫТО.
- `docs/concurrency*.md` (если есть) — короткая user-facing страница про
  M:N default + escape hatch.

### Regression budget

- 0 регрессий MSVC suite (1049/16/56 baseline от Plan 82 followup).

## 6. Risk audit (обновлено в ред. 2)

| Риск | Mitigation |
|---|---|
| Park-predicate refactor затрагивает все park-сайты (sleep, blocking, channel send/recv, supervised drain) | Incremental в [83.4.1](83.4.1-d93-park-wake-hardening.md): сначала сам primitive + sleep-сайт; затем по одному сайту с targeted-тестом каждый раз |
| Handler-storage refactor ломает algebraic effects (фундамент языка) | Detail spec в [83.4.2](83.4.2-supervised-drain-ownership.md): отдельная фаза Ф.2 после Ф.1 (ownership), targeted-тест per-effect, full `nova test` после каждого effect-shift |
| Atomic status CAS под high steal rate → contention hot path | Профиль `mn_runtime_cpu_workload` до/после ([83.4.2](83.4.2-supervised-drain-ownership.md) Ф.1 acceptance); per-fiber статус aligned на cache-line (`_Alignas(NOVA_CACHELINE_SIZE)`) |
| TSAN на Linux может вскрыть исторические races НЕ из Plan 83.4 | Discover-and-document в [83.4.4](83.4.4-flip-activation-and-83.2-closure.md) Ф.5; pre-existing races документируются в `simplifications.md` как followup, НЕ блокируют Plan 83.4 |
| Speedup-target (≥0.85× Go) может не достигнуться из-за Boehm GC overhead vs Go precise | Если упрёмся ниже 0.85× — измерить вклад GC через `--gc malloc` (нет GC); если разрыв в GC — следующий план по concurrent GC если есть, иначе фиксировать в spec как известную дельту с обоснованием |
| Cancellation hierarchical tree требует ABI-changes в scope | [83.4.3](83.4.3-mn-api-semantic-alignments.md) Ф.2: оставить плоскую cancellation если ABI-impact велик; hierarchical — followup |
