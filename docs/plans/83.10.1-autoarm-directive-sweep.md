## 1. Контекст

Plan 83.10 fix (`[M-83.10-armed-user-throw-routing]`) закрыл cross-worker
USER_TYPED throw routing — 4 из 5 panic-safety tests теперь PASS под
armed M:N без `// ENV NOVA_AUTOARM=0` directive.

**Существуют ~12 других tests** с `NOVA_AUTOARM=0` directive из earlier
Plan 83.4.5.1-83.4.5.8 era. Они были добавлены когда armed M:N имел
**другие** gaps:
- `[M-83.10-armed-user-throw-routing]` — закрыт ✅.
- D93 sleep-wake race — Plan 83.4.1 ✅ (закрыт).
- supervised double-resume — Plan 83.4.5.7 ✅ (закрыт).
- handler-storage swap — Plan 83.4.5.4 ✅ (закрыт).
- main_yield uv_run — Plan 83.4.3 ✅ (закрыт).
- atomic cancel_requested — Plan 83.4.3 B5 ✅ (закрыт).

**Hypothesis:** многие `NOVA_AUTOARM=0` directives **теперь obsolete** —
закрытые gaps делают armed path correct. Mechanical sweep + verify.

## 2. Scope

**Цель:** mechanically sweep все tests с `NOVA_AUTOARM=0` directive,
verify каждый под armed M:N (без directive), classify:

**A. Obsolete** — PASSES armed после remove → remove directive.

**B. Still needed** — FAILS armed → leave directive + update comment с
актуальной причиной (которая существует сегодня, не old fixed bug).

**C. Discovered new gap** — FAILS armed и не matches existing
documented gaps → file new `[M-...]` simplifications entry.

**НЕ В SCOPE:**
- Runtime modifications.
- New test creation.
- Fixing newly discovered gaps (только documenting).

### 4.4 Per-test sweep cycle

Для каждого test с directive:

1. **Read test** (`Read nova_tests/<path>.nv`). Identify directive
   comment line.

2. **Snapshot baseline behavior** (с directive) — verify PASS standalone:
   ```bash
   cd /d/Sources/nv-lang/nova-p83-autoarm && env NOVA_GC_LIB_DIR=$NOVA_GC_LIB_DIR NOVA_GC_INCLUDE_DIR=$NOVA_GC_INCLUDE_DIR ./nova-cli/target/release/nova.exe test --gc boehm --filter <test_name>
   ```
   Если baseline FAIL — skip test (broken regardless of directive).

3. **Remove directive** — Edit'нуть `// ENV NOVA_AUTOARM=0` line к
   `// (test runs under armed M:N default — Plan 83.10 fix)`.

4. **Test armed standalone:**
   ```bash
   cd /d/Sources/nv-lang/nova-p83-autoarm && env NOVA_GC_LIB_DIR=$NOVA_GC_LIB_DIR NOVA_GC_INCLUDE_DIR=$NOVA_GC_INCLUDE_DIR ./nova-cli/target/release/nova.exe test --gc boehm --filter <test_name>
   ```

5. **Classify result:**

   **A. PASS armed → directive obsolete.** Keep removed. Update test
   header comment to mention "Validated armed M:N после Plan 83.10
   fix 2026-05-25."

   **B. FAIL armed → directive still needed.** Restore directive.
   Update comment с актуальной причиной (read error message, identify
   which armed gap remains). Possible causes:
   - `[M-83.10-nested-armed-routing]` — nested supervised inside spawn.
   - `[M-83.9-arena-pressure-1M]` — concurrent jobs только.
   - **NEW unknown gap** — file new `[M-...]` entry.

   **C. Different failure pattern** — investigate briefly, document
   hypothesis. Не fix here.

6. **Commit per-test** (per memory `feedback-commit-per-task`):
   ```bash
   git add nova_tests/<test>.nv
   git commit -m "test(plan83.10.1): <test_name> — directive obsolete после Plan 83.10 fix"
   # или
   git commit -m "test(plan83.10.1): <test_name> — keep cooperative directive [M-...]"
   ```
