## Зачем

Текущий `copy_from` ([array.h:96-100](../../compiler-codegen/nova_rt/array.h#L96)) имеет **непоследовательную** гибридную семантику:

- `src.len > dst.len` → panic «copy_from: src longer than dst»
- `src.len < dst.len` → молча копирует `src.len`, хвост dst не тронут
- `src.len == dst.len` → memcpy

Это **хуже** всех 5 эталонных языков (Go/Rust/TS/Kotlin/Java) — ни один не имеет такой гибрид «panic на длинный, тихо на короткий». Тихая truncation = silent bug factory.

Также отсутствуют:
- `extend_from(src)` — append пачкой с ростом (паритет Rust `extend_from_slice`, Go `append(dst, src...)`, Kotlin/Java `addAll(src)`)
- `insert_from(i, src)` — вставка пачкой (паритет Kotlin/Java `addAll(index, src)`, TS `splice(i, 0, ...src)`)
- `reserve(extra)` — preallocate hint (паритет Rust `Vec::reserve`)

Единственный grow-path сейчас — `for x in src { dst.push(x) }` (по элементу, виртуальный вызов). Для primitive `[]T` это в разы медленнее single bulk memcpy.

## Industry baseline + Nova edge

| Capability | Go | Rust | TS/JS | Kotlin | Java | **Nova (после 90.1)** |
|---|---|---|---|---|---|---|

| Truncation copy | ✅ `copy()` (min) | через slicing | partial | n/a | n/a | через slicing `dst[..n].copy_from(src[..n])` |

| Bulk insert at index | ❌ | ✅ `splice` | ✅ `splice` | ✅ `addAll(i,..)` | ✅ `addAll(i,..)` | ✅ `insert_from` |
| Reserve hint | ❌ | ✅ `reserve` | n/a | n/a (есть `ensureCapacity` в ArrayList) | partial | ✅ `reserve` |
| Overlap safety без UB | ✅ (memmove) | ❌ (UB / debug-panic) | ✅ | ✅ | ✅ | ✅ **always memmove** |
| View dangling предотвращение | ❌ (silent) | ✅ (borrow check, complex) | n/a | n/a | n/a | ✅ **lint at view-site** (W_VIEW_EXTEND_DETACH) |
| SIMD memcpy для primitives | ✅ | ✅ | n/a | n/a | n/a | ✅ (через C runtime + clang/gcc/msvc auto-vec) |

**Nova объективно лучше** на двух осях:
1. **Overlap-safety без borrow checker** — `copy_from` всегда memmove (как Go), но без лишней runtime-детекции.
2. **Loud view detach** — лучше «silent» (Go/TS/Kotlin/Java), мягче чем Rust borrow check (нет lifetime annotations).

**Паритет** с Rust на остальных capabilities.

## Дизайн — итоговая поверхность (4 метода + truncation через slicing)

### `dst.copy_from(src)` — strict equal-length, always memmove

```c
static void nova_array_copy_from_##T(NovaArray_##T* dst, NovaArray_##T* src) {
    if (src->len != dst->len) {
        nv_panic((nova_str){
            .ptr = "copy_from: length mismatch (use dst[..n].copy_from(src[..n]) for partial)",
            .len = ... });
    }
    memmove(dst->data, src->data, (size_t)(src->len) * sizeof(T));  // memmove не memcpy
}
```

- **Breaking change**: было `src.len > dst.len → panic` + `src.len < dst.len → silent truncation`. Стало **строгое equal-only**.
- **memmove вместо memcpy** — overlap safe by default (паритет Go, без runtime-детекции).
- Truncation use-case — через slicing: `dst[..n].copy_from(src[..n])` (Plan 96 уже работает).

### `dst.extend_from(src)` — append с ростом

```c
static void nova_array_extend_from_##T(NovaArray_##T* dst, NovaArray_##T* src) {
    int64_t needed = dst->len + src->len;
    if (needed > dst->cap) {
        int64_t new_cap = dst->cap * 2;
        if (new_cap < needed) new_cap = needed;
        T* new_data = (T*)nova_alloc((size_t)new_cap * sizeof(T));
        memcpy(new_data, dst->data, (size_t)(dst->len) * sizeof(T));
        dst->data = new_data;
        dst->cap = new_cap;
        /* view detach: lint at view-site (W_VIEW_EXTEND_DETACH) */
    }
    /* memmove (а не memcpy) — обрабатывает self-extend dst.extend_from(dst) */
    memmove(dst->data + dst->len, src->data, (size_t)(src->len) * sizeof(T));
    dst->len = needed;
}
```

- Cap-growth: **2x doubling** (паритет push в array.h:69-78); если 2x недостаточно — растёт ровно до needed.
- **memmove** для self-extend safety (snapshot не нужен — memmove корректно handles `src == dst`; правда нужно сначала закрепить `dst->len` до записи, что и сделано: src->len и dst->len snapshot'ятся в локальных через arg-eval до memmove).
- View detach происходит при realloc; lint at view-site (Ф.5).

### `dst.insert_from(i int, src)` — insert at position

```c
static void nova_array_insert_from_##T(NovaArray_##T* dst, int64_t i, NovaArray_##T* src) {
    if (i < 0 || i > dst->len) nv_panic_insert_oob(i, dst->len);
    int64_t needed = dst->len + src->len;
    int64_t tail_len = dst->len - i;
    if (needed > dst->cap) {
        /* alloc new + memcpy prefix [0..i] + reserved hole [i..i+src.len] + memcpy tail */
        int64_t new_cap = dst->cap * 2;
        if (new_cap < needed) new_cap = needed;
        T* new_data = (T*)nova_alloc((size_t)new_cap * sizeof(T));
        memcpy(new_data, dst->data, (size_t)i * sizeof(T));
        memcpy(new_data + i + src->len, dst->data + i, (size_t)tail_len * sizeof(T));
        dst->data = new_data;
        dst->cap = new_cap;
    } else {
        /* in-place: memmove tail вправо на src.len */
        memmove(dst->data + i + src->len, dst->data + i, (size_t)tail_len * sizeof(T));
    }
    /* memmove src в hole — обрабатывает overlap если src — view в dst */
    memmove(dst->data + i, src->data, (size_t)(src->len) * sizeof(T));
    dst->len = needed;
}
```

- OOB check: `i in [0, dst->len]` (включая `dst->len` — это append в конец, эквивалент `extend_from`).
- 3 случая memcpy в alloc-path (prefix / hole / tail) — но реалистично один realloc и memmove (in-place case).

### `dst.reserve(extra int)` — preallocate hint

```c
static void nova_array_reserve_##T(NovaArray_##T* dst, int64_t extra) {
    if (extra < 0) nv_panic_negative_reserve(extra);
    int64_t needed = dst->len + extra;
    if (needed > dst->cap) {
        int64_t new_cap = dst->cap * 2;
        if (new_cap < needed) new_cap = needed;
        T* new_data = (T*)nova_alloc((size_t)new_cap * sizeof(T));
        memcpy(new_data, dst->data, (size_t)(dst->len) * sizeof(T));
        dst->data = new_data;
        dst->cap = new_cap;
    }
    /* len не меняется — только cap */
}
```

- View detach происходит при realloc — тот же lint.
- Если уже хватает cap — no-op O(1).

### Truncation через slicing (no new method)

```nv
// было (под текущей truncation-OK семантикой):
let mut dst []u8 = [1, 2, 3, 4]
dst.copy_from([9, 9])  // [9, 9, 3, 4] — silent

// станет (под strict equal-only):
let mut dst []u8 = [1, 2, 3, 4]
dst[..2].copy_from([9, 9])  // explicit prefix slice, [9, 9, 3, 4]
```

Slicing `dst[..n]` уже работает (Plan 96 D144); сам делает bounds-check; composes naturally; нет лишней API surface.

## Risks + mitigations

| Риск | Mitigation |
|---|---|
| **Breaking change** в `copy_from` (silent truncation → panic) ломает unknown user code вне репо | Сейчас Nova pre-0.1 — нет stable API claim. Migration message в panic explicit указывает `dst[..n].copy_from(src[..n])` idiom — fix-it path очевиден. |
| **memmove vs memcpy** perf-loss на small copies | На современных CPU 5-15% на мелочи (<64 байт), на больших — паритет. Компилятор часто инлайнит оба одинаково. **Acceptable** — overlap safety стоит этого. Бенчмарк в Ф.4 stress test зафиксирует фактическую цифру. |
| **Self-extend `dst.extend_from(dst)`** UB через memmove на realloc-path | Snapshot `src->len` в локальной переменной до alloc — после alloc src указатель уже свежий, len старый. memmove с правильными len-границами. Test `extend_from_self.nv` покрывает. |
| **View detach lint false-positive** в legitimate scenarios | `#[allow(view_extend_detach)]` suppression syntax. Документировано в D141 amend. |
| **Insert_from in-place edge** — overlapping memmove когда `src ∩ dst[i..]` | memmove обрабатывает overlap correctly; верификация в `copy_from_overlap_via_view.nv` тесте. |
| **Cap-growth strategy** mismatch с push (если push когда-то изменится на не-2x) | extend_from использует **тот же расчёт** что push (line 71 array.h `new_cap = cap * 2`). Если push изменится — extend_from меняется в той же правке. Single source of truth в spec D141 amend. |
| **D-block conflict** | 90.1 amend'ит D141 (не новый D-block). Никаких новых номеров не нужно. |
