## 1. Контекст

`@buf.push(a).push(b).push(c)` chain через `emit_c.rs` builtin-method
dispatch (line 19474 for `push`): each chain level recursively calls
`emit_expr(obj)` к get the receiver C-string, then emits the mutation
statement using that string, и returns the same string for the next
chain level (D181 fluent-`@`-return convention).

Result: **N copies of `nova_self->buf`** в C output for depth-N chain.
Three for the WriteBuffer.@write_char pattern that triggered this work.

Field cache cannot help — AST has only **one** `Member{SelfAccess,
buf}` leaf node. The duplication is purely codegen-level string
substitution.

---

## 2. Approach

**Option C** from the workflow analysis (`w5dlb8t9w`): AST-level
lowering pre-pass adjacent to `callnorm`. Rewrites fluent chains
к Block-with-temp pattern.

## 3. Изменения

### 3.1 `compiler-codegen/src/chain_norm.rs` (NEW, ~500 LOC)

- `FLUENT_BUILTIN_METHODS: &[&str]` — hard-coded list of known-fluent
  methods (push/append/extend_from/copy_from/insert/reserve/fill/
  clear + WriteBuffer/StringBuilder write-family — 30 entries).
- `is_fluent_builtin_method(name) -> bool` — predicate.
- `normalize_chains_module(&mut Module)` — public entry-point.
- `ChainCounter` — monotonic counter for unique temp names.
- `normalize_chains_{item, fn, block, stmt, expr}` — recursive
  walkers.
- `normalize_chains_expr_children` — exhaustive ExprKind descent
  (skipping closures by construction — they're values, not chain
  positions).
- `try_extract_outer_fluent_chain(e) -> Option<FluentChain>` —
  detection: walk `Call.func.obj` chain, require each frame's method
  in fluent list AND root receiver = `Member{SelfAccess, F}`.
- `build_chain_block(chain, counter) -> Expr` — rewrite к Block с
  let-binding + each call as Stmt::Expr + trailing temp.

**Top-down detection order:** check current Expr first; if outermost
frame of a chain, wrap; else descend into children. (Bottom-up would
wrap inner chains first, breaking outer extractor's left-deep
`Call.func.obj` walk pattern.)
