# Plan 70.3 Ф.0 — Audit baseline (2026-05-19)

## Pre-fix archeology

Bug already fixed (user audit 2026-05-19 — Plan 71 follow-up annotation):
`std/runtime/string.nv:35` documents:

> **Plan 71 follow-up (2026-05-19):** signature corrected `Option[int]`
> → `Option[char]`. C-runtime эмитит codepoint (`Option[char] =
> NovaOpt_nova_int`)... потребляют как `Option[char]` (match `Some('/')`,
> equality `== Some('=')`, передача в `bcrypt_decode_char(c Option[char])`).

i.e. `str @char_at(idx int)` was declared `-> Option[int]`, returning
`Option[char]` де-факто. ~50 callers ran correctly из-за C-level collapse.
User fixed declaration; Plan 70.3 — architectural prevention.

## Collapse risk surface

При current `char → nova_int` mapping:
- `NovaOpt_nova_int` shared между всеми `Option[int]` AND `Option[char]` instances
- `NovaArray_nova_int*` shared между `[]int` AND `[]char`
- `Nova_HashMap____nova_int__<V>` shared между `Map[int, V]` AND `Map[char, V]`

В runtime неотличимы. Type-checker enforces Nova-level distinction в
DECLARATIONS, но если type-checker has bug или missing rule, silent
collapse через generic mono mangling.

## Recurrence vectors (что Plan 70.3 предотвращает)

1. **External fn signature drift** — runtime impl возвращает codepoint,
   declaration говорит int (Plan 71 follow-up bug). С distinct
   `nova_char` typedef compile-error на drift.

2. **Generic instantiation collision** — `fn f[T](x Option[T])` вызвано
   с T=int и T=char generates ONE mono instance (same name). С distinct
   typedef — два separate instances.

3. **Implicit coercion в slot passing** — `let x Option[int] = Some('a')`
   currently type-checker rejects, but если bug в matching — silent.
   Distinct C type — Cppcompiler enforces.

## Type-system baseline (post-Plan-71 fix)

Type-checker correctly distinguishes Option[char] vs Option[int] в
declared signatures. Specific test:

```nova
fn want_int(x Option[int]) -> int => match x { Some(n) => n, None => -1 }
test "type-checker rejects Option[char] in Option[int] slot" {
    let c Option[char] = Some('a')
    want_int(c)  // должно быть compile error
}
```

(Будет добавлено в `nova_tests/plan70_3/f4_int_in_char_slot_neg.nv`.)
