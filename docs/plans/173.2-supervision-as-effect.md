# Plan 173.2 — Supervision как эффект (`Supervisor` / `on_child_fail` / `Decision`)

> **Под-план [Plan 173](173-error-system-unify-harden.md)** (Ф.3 → семейство 173.0–173.3). **Создан:** 2026-06-21.
> **Статус:** 📋 PROPOSED. **Маркер:** `[M-173.2-supervision-effect]`. **Запуск:** «**выполни план 173.2**».
> **Зависит от:** [173.0](173.0-concurrency-runtime-substrate.md) (per-slot ошибки + serialized decision-loop + ctx-pinning).
> **Сквозной критерий:** «без упрощений, как для прода». **Совпадает с README**: «Erlang-style supervision = handler restart strategy».

## 1. Зачем

Сейчас парсер `supervised` берёт только `supervised(cancel: tok)`; supervisor-стратегии (08-runtime.md:168) не
реализованы. Нужна **production-grade обработка падений детей** в `spawn`/`supervised`/`parallel for`, причём
расширяемая (пользовательские политики), а не фиксированный набор.

## 2. Решения (sign-off 2026-06-21)

1. **Supervision — настоящий эффект** (в `.nv`, std/prelude — как `Fail`; §3, не в Rust):
```nova
type Supervisor effect {
    on_child_fail(idx int, err any, attempt int) -> Decision   // attempt = №попытки ЭТОГО ребёнка
}
type Decision | Restart | RestartAll | RestartRest | Escalate | Stop
```
2. **`Decision` — единственный словарь.** Стратегий-имён как отдельной сущности **нет**: «one-for-all» = хендлер,
   возвращающий `RestartAll`. Дублирования `OneForAll`/`RestartAll` нет.
3. **Без фиксированного knob `{max_restarts, period}`** — политику задаёт хендлер через `(idx, err, attempt)` +
   собственное состояние: per-child ретрай через `attempt`; глобальный circuit-breaker — свой `mut`-счётчик;
   окно `period` — через `Time`. (OTP-глобальный лимит — частный случай хендлера, не навязан.)
4. **Стратегии = хендлеры** (значения эффекта). Пример:
```nova
with Supervisor = effect Supervisor {
    on_child_fail(idx, err, attempt) => if attempt < 3 { Restart } else { Stop }  // 3 попытки на url, потом выкинуть
} {
    ro pages = parallel for url in urls { fetch(url) }
}
```
5. **Нет супервизора → дефолт-хендлер всегда `Escalate`** = сегодняшний all-or-throw (zero-cost, байт-в-байт).
   *(Каверзно: эквивалентно только при ОДНОМ падении; сегодня при нескольких теряет все-кроме-одного — документировать.)*
6. **MVP — только `Restart`/`Escalate`/`Stop`.** `RestartAll`/`RestartRest` **отложены** (нужен cancel-and-join:
   перезапуск набора обязан сперва погасить in-flight детей того набора, иначе двойная запись слота — `[M-173.2-restart-all-rest]`).
7. **Restart требует пере-читаемых входов:** в restartable-теле **запретить consume/move-захваты** (иначе attempt-2
   читает moved-from) — отдельный диагностик, либо pin входов на scope.
8. **Restart-ceiling** (высокий, конфигурируемый → `Escalate`) — liveness-backstop против бесконечного Restart на
   детерминированной ошибке (today гарантирован first-error→cancel→throw; не регрессировать в вечный hang).
9. **`on_child_fail` исполняется сериализованно, на main-flow** (субстрат 173.0), по одному падению, **до**
   уничтожения `SpawnCtx`. panic ребёнка (не throw) тоже доходит до решения (к границе fiber'а).

## 3. Фазы (после 173.0)

- **Ф.1** — `Supervisor`-эффект + `Decision` в std/prelude `.nv`; парсер/чекер видят эффект штатно.
- **Ф.2** — runtime: serialized decision-loop (173.0) зовёт ambient `Supervisor.on_child_fail`; исполнение `Escalate`/`Stop`/`Restart(single)`; per-idx `attempt`-счётчик.
- **Ф.3** — restart-ceiling + запрет consume-захвата в restartable-теле + проводка с 173.1 (`Stop` → не шлёт в канал).
- **Ф.4 (отложено)** — `RestartAll`/`RestartRest` + cancel-and-join quiesce (`[M-173.2-restart-all-rest]`).

## 4. Spec / D / Q / docs

- D-блок «supervision-as-effect: `Supervisor`/`on_child_fail(idx,err,attempt)→Decision`; стратегии = хендлеры;
  дефолт Escalate; MVP Restart/Escalate/Stop». amend 08-runtime.md (заменить нереализованный постфикс-strategy).
- Q-блок «closed handler effect-set»: `Fail` разрешён (= Escalate-with-error, guard против рекурсии handler-fails-self);
  **suspend/`Time.sleep` не должны блокировать drain-loop** (запретить или drain на отдельном fiber'е); `interrupt` запрещён.
- **Открыто:** тип `err` — `Supervisor[E]` параметризовать throw-типом scope (не `any`); `Decision::Use(value)` для map-or-default.

## 5. Тесты (pos + neg)

- **pos** `nova_tests/err173_2/`: per-url ретрай (`attempt<3 → Restart`); `Stop` (выкинуть, siblings живут); `Escalate`
  (отмена + throw наружу); глобальный circuit-breaker через `mut`-счётчик; дефолт (нет супервизора) = all-or-throw;
  panic ребёнка доходит до `on_child_fail`; restart-ceiling срабатывает на детерминированной ошибке.
- **neg:** consume-захват в restartable-теле → диагностик; `suspend`/`interrupt` в хендлере → диагностик.

## 6. Критерии приёмки

1. `Supervisor`-эффект работает на субстрате 173.0; MVP `Restart`/`Escalate`/`Stop` корректны под M:N.
2. Дефолт (нет супервизора) — байт-в-байт сегодняшний all-or-throw.
3. Ни одна ошибка ребёнка не теряется молча (per-slot retention 173.0).
4. **Без упрощений** (restart-ceiling реален, не заглушка; consume-в-restartable запрещён осознанно).
