## 1. Scope

[Plan 03.1](03.1-path-git-dependencies.md) дал `path`/`git`-зависимости
с **точным пином** (путь либо commit). 03.2 добавляет **версионные
диапазоны** и **resolver**, который выбирает конкретные версии,
удовлетворяющие всем ограничениям дерева зависимостей.

**Входит:**

- **semver + version-ranges** в `nova.toml`: `^1.2`, `~1.2.3`,
  `>=1.0, <2.0`, `1.2.*`, `*`, точная `1.2.3`.
- **git-теги как источник версий**: `{ git = "URL", version = "^1.2" }`
  → среди semver-тегов репозитория выбирается наибольший подходящий.
- **PubGrub resolver** — выбор согласованного набора версий для всего
  (транзитивного) дерева; человекочитаемое объяснение при конфликте.
- **`nova update`** с диапазонами: пере-резолв в новейшую версию в
  пределах диапазона; `nova update --precise`.
- **`nova.lock`** фиксирует resolved-версии; сборка из lock —
  детерминирована.

**НЕ входит** (следующие под-планы):

- Central registry / HTTP-протокол — [Plan 03.3](03-package-ecosystem-roadmap.md).
  03.2 резолвит версии из **git-тегов** (источник, доступный без
  registry); registry позже даёт более богатую вселенную версий через
  тот же resolver.
- `nova publish`/`search`/`info`/`audit` — [Plan 03.4](03-package-ecosystem-roadmap.md).

## 2. Текущее состояние

- ✅ `path`/`git`-deps с точным пином, межпакетный резолв, `nova.lock`,
  `git_cache` (Plan 03.1).
- ✅ `DepSource::Registry(String)` — registry-форма `dep = "1.2"`
  **парсится** (Plan 03.1 Ф.1), но не резолвится.
- ❌ Версионных диапазонов нет; `version`-поле у `git`-deps не
  поддержано; resolver'а нет — каждая зависимость пинится независимо.

## 3. Дизайн

### 3.1. semver и диапазоны

Версия — `MAJOR.MINOR.PATCH` с опциональными `-prerelease` и `+build`
(semver 2.0.0). Диапазоны:

| Синтаксис | Значение |
|---|---|
| `1.2.3` | точно `1.2.3` |
| `^1.2.3` | `>=1.2.3, <2.0.0` (caret — совместимые) |
| `~1.2.3` | `>=1.2.3, <1.3.0` (tilde — патчи) |
| `>=1.0`, `<2.0`, `>1.0`, `<=2.0` | сравнения |
| `1.2.*` / `1.*` | wildcard |
| `*` | любая |
| `>=1.0, <2.0` | пересечение (запятая = AND) |

Реализация — `compiler-codegen/src/semver.rs` (bootstrap-Rust, без
crate-зависимостей, как `manifest.rs`/`git_cache.rs`): `Version`,
`VersionReq`, парсинг, `matches`, `intersect`. Pre-release исключается
из диапазонов по умолчанию (как Cargo) — `^1.2` не берёт `2.0.0-rc1`.

### 3.2. git-теги как источник версий

`{ git = "URL", version = "^1.2" }` (взаимоисключающе с `rev`/`tag`/
`branch`): `git_cache` перечисляет теги репозитория, парсит как semver
(тег `v1.2.0` либо `1.2.0` — префикс `v` опционален), отбирает
подходящие диапазону, берёт **наибольший**. Дальше — как обычный
`git`-checkout по commit'у этого тега. `nova.lock` пишет и версию, и
commit.

### 3.3. Resolver — backtracking

Для **транзитивного** дерева недостаточно «взять max на каждую
зависимость независимо»: версия `A@1.5` может требовать `C ^1.0`, а
`A@1.6` — `C ^2.0`. Выбор версии `A` меняет ограничения на `C` —
нужен **backtracking**.

Реализация (`compiler-codegen/src/resolver.rs`) — **корректный
backtracking-резолвер**: DFS по пакетам, highest-version-first,
распространение ограничений (выбранная версия добавляет ограничения
своих deps), откат при конфликте. При неразрешимости — **диагностика**
с перечнем конфликтующих ограничений.

**Не полный PubGrub.** PubGrub (CDCL — conflict-driven clause
learning) — это backtracking-резолвер *плюс* оптимизация: обучение на
конфликтах ускоряет большие графы и даёт минимальные explanation.
Backtracking-резолвер **корректен и полон** (находит решение, если оно
есть; иначе — конфликт) — для git-tag-deps-масштаба 03.2 (малые графы)
этого достаточно. CDCL-обучение — оптимизация для registry-эры, когда
вселенная пакетов/версий станет большой (followup, 03.3+). Это
декомпозиция, не упрощение: §6.

`DependencyProvider`-трейт абстрагирует источник версий и deps —
resolver не зависит от git/registry; 03.3 (registry) подключит свой
provider без переписывания resolver'а. Источник версий в 03.2 —
git-теги объявленных репозиториев.

### 3.4. `nova.lock` и `nova update`

- `nova.lock` git-записи получают поле `version` рядом с `commit`.
- `nova update [pkg]` — пере-резолв диапазонов в новейшее в пределах
  диапазона; без аргумента — всё. `--precise pkg@X` — зафиксировать
  конкретную версию.
- Формат forward-совместим (Plan 03.1 Ф.4): новые ключи не ломают
  старый парсер.

## 4. Фазы

### Ф.1 — semver + version-ranges core ✅ (сделано)

`compiler-codegen/src/semver.rs` (bootstrap-Rust, без crate-зависимостей):

- `Version` — semver 2.0.0: `parse` (строгая) + `parse_tag` (допускает
  ведущий `v`), `Ord` с корректным prerelease-precedence (semver §11:
  числовой pre-id < алфанумерного, release > pre-release, длина хвоста);
  build-метаданные принимаются и отбрасываются.
- `VersionReq` — `AND`-список простых компараторов; `parse` всех форм
  §3.1 (caret/tilde/wildcard разворачиваются в `>=`/`<` пары; голая
  версия ≡ caret — конвенция Cargo), `matches`, `intersect`.
- Pre-release-policy: диапазон матчит pre-release-версию только если
  компаратор сам указывает pre-release того же ядра (паритет с Cargo).
- 12 unit-тестов (`semver::tests`): парсинг, ordering, caret/tilde/
  wildcard/exact/comparators, prerelease-policy, intersect, негатив.

Чистый модуль — пока ни во что не интегрирован (интеграция — Ф.2).

### Ф.2 — git-теги как источник версий ✅ (сделано)

- `manifest.rs`: `GitPin::Version(VersionReq)` — `{ git = "URL",
  version = "^1.2" }`. Пины `rev`/`tag`/`branch`/`version`
  взаимоисключающи (указано >1 → `DepSource::Invalid`); некорректный
  диапазон → `Invalid`.
- `git_cache.rs`: `select_version_tag` — `git tag --list`, парсинг
  каждого тега как semver (`Version::parse_tag`, ведущий `v`
  опционален), фильтр `VersionReq`, выбор наибольшего; не-semver теги
  пропускаются. `resolve_git_dep` для `Version`-пина: fetch (новый тег
  мог появиться upstream, как ветка) → выбор тега → commit → checkout.
- `lockfile.rs`: `pin_str` для `Version` → `version:<req>`.
- Тесты: `git_cache::resolve_by_version_range` (локальный git-репо с
  тегами `v1.0.0`/`v1.1.0`/`v1.2.0`/`v2.0.0`; `^1.0`→v1.2.0,
  `~1.1`→v1.1.0, `*`→v2.0.0; негатив `^3.0`→ошибка) + `manifest`
  parse-тесты (`version`-пин, взаимоисключение, мусорный диапазон).

### Ф.3 — Backtracking resolver ✅ (сделано)

- `resolver.rs` — `DependencyProvider`-трейт (versions + deps);
  `resolve()` — корректный backtracking-резолвер (DFS,
  highest-version-first, распространение ограничений, откат при
  конфликте; состояние по значению — откат без ручного undo);
  диагностика конфликта. 9 unit-тестов против in-memory mock
  (highest-pick, диапазон, транзитив, diamond, backtracking
  версионно-зависимый, backtracking конфликтного транзитива,
  неразрешимый конфликт, provider-ошибка, терминация на цикле).
- `git_cache.rs` — `list_versions` (semver-теги репозитория как
  вселенная версий); `ensure_db` выделен.
- `lockfile.rs` — `GitProvider` (git-backed `DependencyProvider`:
  версии = теги, deps версии = `[dependencies]` из `nova.toml` на
  теге; кэш версий и deps); `resolve_version_deps` — согласованный
  резолв версионных git-deps до `collect_dep_graph`, выбранные
  commit'ы фиксируются в `git_cache`-таблице пинов; вызывается из
  `sync`.
- E2e: `tests/version_resolve_e2e.rs` — проект → `liba ^1.0`,
  `liba@v1.1.0` → `libb ^1.0`; резолвер выбирает liba v1.1.0 +
  libb v1.0.0 (не v2.0.0), `nova.lock` фиксирует.
- CDCL-обучение (полный PubGrub) — followup registry-масштаба (§3.3).

### Ф.4 — `nova.lock` + `nova update` ✅ (сделано)

- `LockedSource::Git` получил поле `version` — resolved semver-версия
  рядом с `commit` (`None` для rev/tag/branch-пинов); render/parse.
- **Воспроизводимость:** `resolver::resolve_with_preferences` —
  предпочтительные версии из существующего `nova.lock`
  (`LockFile::git_versions`); резолвер держит зафиксированную версию,
  пока ограничения позволяют (повторный `sync` не двигает версию,
  даже если upstream появился новый тег).
- `nova update [pkg]` — снять git-пины из lock → `sync` пере-резолвит
  снятые в новейшее в пределах диапазона, прочие остаются.
- `nova update --precise NAME@VERSION` — `lockfile::update_precise`:
  добавляет точное `=VERSION`-ограничение в резолвер (выполнит либо
  упадёт с конфликтом). CLI: `cmd_update` парсит `NAME@VERSION`,
  маппит имя→URL через манифест.
- Тесты: `version_lock_repro` (lock держит 1.0.0 против upstream
  v1.5.0; `update` → 1.5.0; `--precise` → откат на 1.0.0) + unit на
  lock-таблицу/preferred.

## 5. Acceptance

- `{ git = "URL", version = "^1.2" }` собирается — выбран наибольший
  подходящий semver-тег.
- Транзитивный конфликт диапазонов → детерминированная ошибка с
  человекочитаемым объяснением.
- `nova.lock` фиксирует версию+commit; сборка из lock воспроизводима;
  `nova update` поднимает в пределах диапазона.
- 0 регрессий: `path`/точечные `git`-deps (Plan 03.1) работают как
  прежде — `version`-форма аддитивна.

## 6. Честная рамка

- **Без registry вселенная версий — git-теги.** Это не упрощение
  resolver'а: `DependencyProvider` абстрагирует источник, резолвер
  работает над любой вселенной. Registry (03.3) добавит источник, не
  переписывая resolver.
- **Backtracking-резолвер, не полный PubGrub.** Версионно-зависимые
  dep-set'ы (разные теги пакета — разные `[dependencies]`) требуют
  backtracking; «intersect-and-max» был бы некорректен. Реализованный
  backtracking-резолвер **корректен и полон**. Полный PubGrub (CDCL —
  обучение на конфликтах) — это та же backtracking-база плюс
  оптимизация скорости/explanation для **больших** графов; для
  git-tag-deps-масштаба 03.2 она избыточна. CDCL — followup
  registry-эры (03.3+). Это декомпозиция: корректность не страдает,
  откладывается лишь оптимизация — см. `simplifications.md`.
- Pre-release-версии вне диапазонов по умолчанию (паритет с Cargo) —
  осознанное решение, не пропуск.
