# Plan 70.2: LinkedList sum-type monomorphization

> **Создан:** 2026-05-18. **Статус:** proposed, не начат.
> **Приоритет:** P1. **Трудоёмкость:** ~3-5 dev-days.
> **Родительский план:** [Plan 70](70-no-silent-nova-int-fallback.md)

---

## Зачем

`LinkedList[int]` — рекурсивный sum-type:

```nova
type LinkedList[T] =
    | Nil
    | Cons(T, LinkedList[T])
```

При mono'д codegen генерируется `LinkedList.nil()` вместо
`LinkedList____nova_int.nil()` — тип теряется при monomorphization
sum-type конструкторов. Последующие вызовы методов диспатчатся в
`Nova_StringBuilder_method_append` вместо правильного метода.

**Воспроизведение:** тест `nova_tests/modules/linkedlist.nv` удалён
из batch 7 (Plan 45.B) — CC-FAIL и runtime crash.

---

## Проблема

В `emit_c.rs` monomorphization sum-type:
1. Generic конструктор `LinkedList[T].nil()` при mono с `T=int` не
   подставляет параметр в имя конструктора
2. `Cons(h, t)` destructure — `t: void*` вместо `LinkedList____nova_int*`
3. Метод `@append` диспатчится неверно — попадает в `StringBuilder`
   из-за name collision

---

## Решение

Три части:

**Ф.1** — Static constructors: `TypeName[T].variant()` при mono →
`TypeName____C_T.variant()` в generated C.

**Ф.2** — Recursive field types: в `Cons(T, LinkedList[T])` второй
элемент — рекурсивная ссылка на mono'd тип. Резолвить через
current type substitution context.

**Ф.3** — Method dispatch: при вызове метода на mono'd sum-type
проверять полное манглированное имя, не базовое.
