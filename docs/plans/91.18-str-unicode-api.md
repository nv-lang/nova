## Решения по 4 развилкам (зафиксированы 2026-06-19)

**Решение 1 — `split("")`:** ОТКАТИТЬ Б4 (паника отменяется). Пустой sep → вся строка
одним сегментом (текущее поведение), единое правило «пустой паттерн — определённый край,
не паника» для всех 9 методов (split/splitn/rsplit/rsplitn/split_terminator/find/rfind/
contains/matches/match_indices/replace/replacen). **Отменяет решение Plan 91.15 Ф.4** —
оформить как D-block amend. Причины: остальной surface уже принимает пустой паттерн
(find→Some(0), contains→true), 2 зашипленных теста (plan139_2/split_ok.nv:41,
plan91/text_methods_test.nv:122) проверяют `split("")==["whole"]`, а обоснование 91.15
«как в Go» фактически неверно (Go split("") даёт руны).

**Решение 2 — `*View→*Iter`:** Переименовать ВСЕ ТРИ (`GraphemesView`→`GraphemesIter`,
`WordsView`→`WordsIter`, `SentencesView`→`SentencesIter`) одним махом. **Явное исключение
из правила «компилятор не трогаем»:** `compiler-codegen/src/codegen/unicode_data.rs:452/713/1409`
эмитит строковые литералы `import std.unicode.{…View}` в генерируемые conformance-тесты —
их надо обновить.

**Решение 3 — `SentencesIter` ленивый:** Сделать `SentencesIter` ленивым симметрично
`WordsIter` (forward SB state-machine). `as_sentences()` станет O(1). Бонус: у sentences
НЕТ `to_titlecase`-потребителя, поэтому eager-хелперы (`sb_starts`/`sb_cats`/
`sentence_boundaries`/`sb_decode`) удаляются целиком.

**Решение 4 — `@as_str()→@into_str()`:** Переименовать consuming-финализатор StringBuilder
в `@into_str()` (Rust-идиома `into_*`). Обновить ~20 call sites.

### Nova синтаксис — критично

```
priv          = module-private (видны пирам, не видны снаружи)
priv(type)    = type-private (только методы того типа)
ro / mut      = immutable / mutable binding
@             = self;  -> @ = fluent self-return для chaining
export fn T @m() = extension-метод на T
fn f(x T)     = свободная функция (НЕ экспортируется по умолчанию)
```

- **Контракты** `requires`/`ensures` стоят **между сигнатурой и `{`** (не внутри тела).
  Проверенный `requires` элидируется в compile-time (даже в debug). Запрещён chain
  `0 <= i < n` → только `0 <= i && i < n` (E_CMP_CHAIN_UNSUPPORTED). В контрактах нельзя
  вызывать non-`#pure` методы (E2401) и нельзя tuple-выражения в `ensures`.
- **Size-accessors** (`byte_len`/`len`/`cap`/`count`) уже имеют non-negativity как
  встроенную SMT-аксиому (z3.rs:547-558, План 33.6) — **`ensures result >= 0` на них НЕ
  добавлять**, это no-op.
- **`throw E`** требует, чтобы функция объявляла эффект `Fail[E]` в сигнатуре
  (spec 04-effects.md:322). Любой sum-тип throwable, доп. bound не нужен.
- **`extern "nova" fn` без тела НЕ принимает контракты** (parse error parser/mod.rs:3104).
  `char.nv` АВТОГЕНЕРИРУЕМ (источник — runtime_registry.rs); ручные правки перезатираются.
- Стиль: tuple-binding `ro (a,b)=(x,y)`; `+=`/`-=` вместо `a=a+1`; операторы `< ==`
  вместо `.compare()/.equal()`; цепочки вызовов; **без `;`** и **без нескольких
  операторов на строке через пробел** (`term = 11 phase = 0` — тоже запрещено).

### Б3. `@rfind()` — backward scan с early exit

**Файл:** `std/runtime/string/search.nv`

```nova
export fn str @rfind(needle str) -> Option[int] {
    ro (sn, nn) = (@byte_len(), needle.byte_len())
    if nn == 0 { return Some(sn) }
    if nn > sn { return None }
    mut i = sn - nn
    while i >= 0 {
        if unsafe { RawMem.compare(@ptr + i, needle.ptr, nn) == 0 } { return Some(i) }
        i -= 1
    }
    None
}
```

**Тест** `rfind_fix.nv`: `"abcabc".rfind("b")==Some(4)`, `"abc".rfind("x")==None`,
`"abc".rfind("")==Some(3)`.

### П3. `parse_int`/`try_parse_int` — инверсия + эффект `Fail`

**Файл:** `parse.nv`. Конвенция (D77/D25, protocols.nv:126-128): bare = значение-или-throw;

```nova
/// Парсит int; бросает ParseIntError при ошибке. Для явной обработки — try_parse_int.
export fn str @parse_int(radix int = 10) -> int Fail[ParseIntError]
    requires radix >= 2 && radix <= 36
{
    match @try_parse_int(radix) {
        Ok(v)  => v
        Err(e) => throw e
    }
}

/// Recoverable: Result с явной ошибкой. Тело текущего @try_parse_int БЕЗ изменений.
/// InvalidRadix variant + runtime-проверка radix СОХРАНЯЮТСЯ (динамический radix → Err, не паника).
export fn str @try_parse_int(radix int = 10) -> Result[int, ParseIntError] { /* как сейчас */ }

/// Convenience: Option (теряет детали ошибки).
export fn str @parse_int_opt(radix int = 10) -> Option[int]

=> match @try_parse_int(radix) { Ok(v) => Some(v), Err(_) => None }
```

- Caller-migration grep: `std/` + `nova_tests/` + **`examples/`**. Известные Option-вызовы

### 3.1 Lazy `WordsIter` (words.nv)

Заменить eager-тип (buf/bounds/idx) на forward state-machine:

```nova
export type WordsIter value priv(type) {
    buf str
    pos int               // байт-офсет начала следующего сегмента
    n int                 // buf.byte_len()
    prev_imm_cat int      // непосредственно предыдущий cp (WB3/WB3c/WB3d/WB4)
    prev_eff_cat int      // последний non-ignorable (WB5+)
    prev_prev_eff_cat int // до него (WB7/WB7c/WB11)
    ri_run int            // RI parity (WB15/WB16)
}

fn WordsIter.new(buf str) -> WordsIter =>
    { buf, pos: 0, n: buf.byte_len(),
      prev_imm_cat: 0, prev_eff_cat: 0, prev_prev_eff_cat: 0, ri_run: 0 }
```

`wb_is_break_s` — flat-параметрическая версия `wb_is_break` (та же каскадная логика
WB3..WB999, но `pcat`/`ppcat`/`ncat` как параметры вместо `cats[i±k]`). Полный листинг —
см. v1 §3.2 (логика без изменений).

**Lookahead — ленивый (исправление P0 wordsiter-wb6-7b-12):** `scan_next_eff_cat` вызывать
ТОЛЬКО когда `cur` реально нуждается в right-context, иначе `ncat = 0`:
- WB6: `pcat` ∈ AHLetter && `c` ∈ {MidLetter, MidNumLetQ}
- WB7b: `pcat`==HEBREW && `c`==DQ
- WB12: `pcat`==NUMERIC && `c` ∈ {MidNum, MidNumLetQ}

Это убирает безусловный O(k) forward-scan на каждом cp (иначе @next был бы O(n·k)).

```nova
fn scan_next_eff_cat(buf str, from int, n int) -> int {
    mut j = from
    while j < n {
        ro (cp, step) = decode_at(buf, j)   // decode_at — peer из graphemes.nv
        ro cat = wbcat(cp)
        if !wb_ignorable(cat) { return cat }
        j += step
    }
    0
}

export fn WordsIter mut @next() -> Option[str] {
    if @pos >= @n { return None }
    ro start = @pos
    ro (cp0, step0) = decode_at(@buf, @pos)
    @_advance(wbcat(cp0))
    @pos += step0
    while @pos < @n {
        ro (cp, step) = decode_at(@buf, @pos)
        ro cat = wbcat(cp)
        ro ncat = if @needs_lookahead(cat) { scan_next_eff_cat(@buf, @pos + step, @n) } else { 0 }
        if wb_is_break_s(@prev_imm_cat, @prev_eff_cat, @prev_prev_eff_cat, cat, cp, ncat, @ri_run) {
            return Some(@buf[start..@pos])
        }
        @_advance(cat)
        @pos += step
    }
    Some(@buf[start..@n])
}

fn WordsIter mut @_advance(cat int) {
    @prev_imm_cat = cat
    @ri_run = wb_ri_step(cat, @ri_run)
    if !wb_ignorable(cat) {
        @prev_prev_eff_cat = @prev_eff_cat
        @prev_eff_cat = cat
    }
}

fn WordsIter @needs_lookahead(cat int) -> bool =>
    (wb_ahletter(@prev_eff_cat) && (cat == WB_MIDLETTER || wb_midnumletq(cat)))
    || (@prev_eff_cat == WB_HEBREW && cat == WB_DQ)
    || (@prev_eff_cat == WB_NUMERIC && (cat == WB_MIDNUM || wb_midnumletq(cat)))
```

Методы:
```nova
export fn WordsIter @iter() -> WordsIter => @
export fn WordsIter @is_empty() -> bool => @pos >= @n
export fn WordsIter @count() -> int {   // O(n), без precomputed bounds — задокументировать
    mut tmp = @
    mut cnt = 0
    while tmp.next() != None { cnt += 1 }
    cnt
}
export fn str @as_words() -> WordsIter => WordsIter.new(@)   // теперь O(1)
```

**Удалить** `word_boundaries`, `wb_decode`. **Оставить** `wb_is_break`, `wb_starts`,
`wb_cats` — нужны `to_titlecase`.
