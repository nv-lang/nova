### 110.9.1 — Typed `CleanupTimeoutError` throw codegen (~2-3ч)

**Что:** Заменить string-fallback throw в `nv_shield_check_deadline` на
typed `Nova_CleanupTimeoutError*` allocation + `nova_throw_typed`.

**Files:**
- `compiler-codegen/src/codegen/emit_c.rs`:
  - В preamble emit'нуть `static void _nova_throw_cleanup_timeout_impl(int duration_ms)`
    в user TU. Function allocate'ит `Nova_CleanupTimeoutError* e = nova_alloc(...)`,
    set `e->duration_ms = duration_ms`, calls `nova_throw_typed(msg, e, NOVA_TID_USER_Nova_CleanupTimeoutError)`.
  - В startup-code (после `nova_runtime_init()`): emit
    `_nova_throw_cleanup_timeout_fn = &_nova_throw_cleanup_timeout_impl;`.
- Conditional: только если `CleanupTimeoutError` в `type_id_registry`
  (всегда true когда prelude loaded, который default).

**Tests:**
- `nova_tests/plan110/cleanup_timeout_typed_throw.nv` (NEW): fixture
  с очень коротким timeout (через `WithExitTimeout` impl from 110.9.2 OR
  test-only override), body suspend'ит → throw fires, outer
  `catch e: CleanupTimeoutError { e.duration_ms }` reads typed field.

**Acceptance:**
- A110.9.1.a: codegen emit'ит impl + pointer assignment.
- A110.9.1.b: catch arm читает typed `duration_ms` field.

**Depends:** 110.9.2 (для concrete fire-test) OR test-only override
API.

### 110.9.3 — Application `register_finalizer` LIFO runtime (~½ дня)

**Что:** Application handler maintains stack of registered finalizers.
On `with Application = ... { body }` exit — fire stack in reverse order.

**Files:**
- `compiler-codegen/src/codegen/emit_c.rs`: per-`with Application` block:
  - Emit local `NovaFinalizerStack _finalizers;` (initialized empty).
  - `Application.register_finalizer(f)` dispatch — push to `_finalizers`.
  - At with-block exit (both normal + throw paths): iterate
    `_finalizers` в reverse, invoke each.
- `compiler-codegen/nova_rt/effects.h`: `NovaFinalizerStack` struct
  + push/pop/iterate helpers (small fixed-size array, grow if needed).

**Tests:**

  register A, B, C → verify execution order C, B, A.

  throw inside with-block → finalizers still fire.

**Acceptance:**
- A110.9.3.a: LIFO order verified.
- A110.9.3.b: finalizers fire on throw path.
- A110.9.3.c: D195 R8 abort/SIGKILL documented как OS limitation.

### 110.9.5 — `on_exit` strict signature check (~½ дня, рискованный)

**Что:** Currently `validate_on_exit_signature` проверяет только param
count. Spec D188 требует точную signature `(self, outcome ScopeOutcome) Fail[E] -> ()`.

**Под-задачи:**

**110.9.5.a — Fix `-> ()` parser variance (foundational):**

В AST `-> ()` представляется как `Tuple([])` ИЛИ `Named("unit")` в
зависимости от context (8 fixture'ов failed раньше из-за этого). Need
canonical form в parser.

- `compiler-codegen/src/parser/mod.rs`: при parsing return type, если
  `()` → emit canonical `Named("unit")`. Update other parse sites
  consistently. Migration audit на existing fixtures.

**110.9.5.b — Strict signature check (after .a):**

В `compiler-codegen/src/types/mod.rs validate_on_exit_signature`:
- Param 0: must be self-receiver (already checked).
- Param 1: must be `ScopeOutcome` (already checked).
- Return type: must be `()` или `Fail[E] -> ()` (NEW check).
- Effects: must contain at most `Fail[E]` (NEW check).
- Generic params: forbidden (NEW check).

**Tests:**
- `nova_tests/plan110/neg_on_exit_wrong_return.nv` (NEW).
- `nova_tests/plan110/neg_on_exit_extra_effects.nv` (NEW).
- Existing 36 fixtures NOT regress (verify post-fix).

**Acceptance:**
- A110.9.5.a: parser canonical `-> ()` representation.
- A110.9.5.b: strict signature check passes valid + rejects invalid.
- A110.9.5.c: zero regression на existing fixtures.

**Risks:**
- Parser variance fix может вызвать cascade failures на широком
  fixtures-base. Recommend running `nova test nova_tests/` (full)
  после 110.9.5.a + comparing failures vs baseline.
