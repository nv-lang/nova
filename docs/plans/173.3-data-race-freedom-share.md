# Plan 173.3 — Data-race-freedom: атрибут `#share` + capture-check + consume-в-spawn

> **Под-план [Plan 173](173-error-system-unify-harden.md)** (Ф.3 → семейство 173.0–173.3). **Создан:** 2026-06-21.
> **Статус:** 📋 PROPOSED. **Запуск:** «**выполни план 173.3**».
> **Источник:** research-свип 2026-06-21 (Rust Send/Sync, Swift 6 Sendable, Pony caps, тренд) → синтез.
> **Сквозной критерий:** «без упрощений, как для прода».

## 1. Зачем

Под M:N захват внешнего `mut` по ссылке в несколько `spawn` — **гонка** (трудноуловимый баг; в Go ловится только
рантайм-детектором). Сейчас Nova **permissive** (примеры захватывают `mut`-аккумулятор в `supervised` — работает на
коротких inline-батчах, под реальным M:N — потенциальная гонка). Цель: **гонка → compile-error**, безопасные пути
zero-cost. Мировой тренд (2024–2026) сошёлся на тип-маркере, авто-выводимом по полям, проверяемом на границе
конкуренции, с аудируемым unsafe-escape (Rust Send/Sync ≈ Swift `Sendable`) — Pony reference-caps и actor-isolation
проиграли на эргономике; Go `-race`/Kotlin/C# (рантайм/соглашение) — анти-модели.

## 2. Решения (sign-off 2026-06-21)

1. **Одна ось — `#share`** («можно ли алиасить из другого файбера»). **Send/move-ось НЕ нужна:** под общим GC move
   всегда memory-safe → его покрывает `consume`. (Rust нужны обе из-за use-after-free; нам — нет.)
2. **`#share` — АТРИБУТ типа, НЕ протокол.** Систему протоколов не трогаем; пустой протокол (маркер без метода)
   в неё не лезет (был бы истинен для всех). `#share` — маркер на типе, как `#impl`/`#forbid`.
3. **Обычные типы — авто, пишешь НИЧЕГО:** тип `#share`, если все поля `#share` (мемберуайз, как авто-derive
   `Equal`/`Hash`, Plan 126). Считается по запросу capture-check'а.
4. **«Ядовитая» база (единственная не-`#share`):** сырой мутабельный `*T`-cell. Любой тип, транзитивно его
   содержащий, не `#share` → гонки **непредставимы по построению** (аналог Rust `UnsafeCell !Sync`).
5. **`#share` пишется только на горстке sync-примитивов** (`Mutex`/`RwLock`/`Atomic`/каналы) — **audited vouch**
   автора (аналог Rust `unsafe impl Sync`): «внутри настоящая синхронизация, авто-вывод её не видит». В **их `.nv`**.
   Точное «audited/unsafe»-написание — в этом плане (под-вопрос).
6. **§3 соблюдён by construction:** компилятор хардкодит только **правило** (авто-вывод + ядовитая база +
   чтение атрибута `#share`) — **ни одного имени типа**. Пользовательский lock-free тип пишет тот же `#share`;
   `Mutex` не особый.
7. **Правило захвата** для `spawn`/`parallel for` (в чекере, §1). Тело может брать наружу **только**:
   `consume`/move (отдал во владение) · `ro` deep-immutable (никто не мутирует) · тип с `#share`. Внешний `mut`
   по ссылке (не-`#share`) → **`E_CONCURRENT_MUT_CAPTURE`**.
8. **consume-в-spawn — ЯВНЫЙ синтаксис** (новая возможность, sign-off 2026-06-21): `spawn consume c = e { … }`
   (и `spawn consume c { … }` для уже-связанного) — **зеркало** `consume c = e { body }`, но владеющий scope =
   **ребёнок** (cleanup на выходе ребёнка, не лексического блока). **Никакого молчаливого/скрытого захвата move'ов** —
   перемещение в файбер видно в коде (школа Rust `move ||`, анти-Go-неявный-захват; ложится на тезис Nova
   «видимо/ревьюится»). Чинит V1/V2-грабли (use-after-consume) by construction. Keyword'а `move` нет — используется
   `consume`. Безопасные не-move захваты (`ro`-immutable, `#share`-тип, by-value примитив) — неявные, но проверяемые
   (п.7); опасный внешний `mut` и так запрещён (см. под-вопрос §7 о полноте явности).
9. **`#share` и `consume`-move — альтернативы для РАЗНЫХ паттернов, не для одного instance.** move = отдать
   **уникальное** одному ребёнку (`#share` нерелевантен); `#share` = **алиасить одно** среди многих (Mutex/Atomic —
   их нельзя clone-move'ить). Тип может быть и `Clone`, и `#share`, но на use-site выбираешь одно. Частность: в
   десугаре сбора (173.1) Sender'ы **clone+move** → `#share` на Sender там **не используется** (достаточно `Clone`);
   настоящие `#share`-типы — `Mutex`/`RwLock`/`Atomic` (шарятся по ссылке).
9. **Поглощает Plan 118.3** `E_POINTER_CROSS_FIBER` / `#fiber_send`: ad-hoc запрет «сырой `*T` через файбер»
   становится частным случаем `#share`-проверки. (До закрытия 173.3 `#fiber_send` остаётся живым и нужным — не трогать.)
10. **Миграция (sign-off):** nv не в релизе → спеку меняем напрямую, старый небезопасный код (`mut`-захват в
    supervised) чиним/выпиливаем разом, без церемонии постепенной миграции §7 (она для «не сломать релиз»).

**Правило для пользователя одной фразой:** обычные данные → `#share` даром; `consume` — отдать уникальное мутабельное;
`Mutex[T]`/`Atomic` (сами `#share`) — шарить мутабельное (без `unsafe`); `#share` пишешь только если автор lock-free примитива.

## 3. Фазы

- **Ф.1** — атрибут `#share` (parser/attr) + авто-вывод «все поля share» (переиспользовать мемберуайз-движок Plan 126)
  + ядовитая база (`*T`-cell). Detect-режим: посчитать blast-radius по std/nova_tests.
- **Ф.2** — `unsafe #impl/#share` (audited) на sync-примитивах в их `.nv`; capture-check на границе
  spawn/supervised/parallel-for/channel-send → `E_CONCURRENT_MUT_CAPTURE`. Чинить/выпилить старый небезопасный код.
- **Ф.3** — **consume-в-spawn** (move-захват во владение ребёнка; cleanup на выходе ребёнка). Разблокирует 173.1.
- **Ф.4** — поглотить `#fiber_send`/`E_POINTER_CROSS_FIBER` (Plan 118.3) принципиальной `#share`-проверкой; ретайр старого маркера.

## 5. Тесты (pos + neg)

- **pos** `nova_tests/err173_3/`: захват `consume`-значения (move) в spawn; захват `ro`-immutable; захват `Mutex`/`Atomic`/`Sender`
  (`#share`); пользовательский тип с `#impl`-полями авто-`#share`; пользовательский lock-free тип с `unsafe #share`.
- **neg:** захват внешнего `mut` (не-`#share`) в spawn → `E_CONCURRENT_MUT_CAPTURE`; тип с `*T`-cell не `#share`;
  consume-захват, использованный после move → use-after-consume; (V1/V2 из 173.1) клон с неверным владением → диагностик.

## 6. Критерии приёмки

1. Гонка (захват `mut` ≥2 продюсерами / cross-sibling) — **compile-error**, не рантайм.
2. Безопасные пути zero-cost (move/ro/`#share` ничего не платят; платит только реальный `Mutex`/`Atomic`).
3. §3: ни одного имени типа в Rust; пользовательский sync-тип получает `#share` тем же механизмом, что `Mutex`.
4. consume-в-spawn работает (владение → ребёнок, cleanup на выходе ребёнка); закрывает V1/V2-грабли.
5. **Без упрощений** (никакого permissive-opt-out по умолчанию; ядовитая база реальна).

## 7. Открытые под-вопросы (резолв в этом плане)

1. **Захваты хендлеров** (эффект-хендлеры обходят лексический скан захватов, Q12): захваченный handler-state обязан
   быть `#share`; нужно ли эффект-row нести `#share`-обязательство. *(Наш аналог Rust-проблемы «Send-bound у async-fn» — решить заранее.)*
2. **`#share`-nameability для результата spawn/эффекта** — выразить «spawn чего угодно, чей результат/замыкание `#share`» без Rust-RTN-ретрофита.
3. **Opt-out `!share`** — тип авто-`#share`, но не должен пересекать файбер (fiber-affine handle). (Rust оставил `!Send` нестабильным годами — решить сейчас.)
4. **Точная ядовитая база** в терминах Nova (`*T`-cell) — проверить, что нет другого пути внутренней мутации в обход.
5. **`ro`/D246 ↔ `#share`:** `ro`-binding над мутабельно-алиасимым pointee — НЕ `#share`; зафиксировать, какая ось D246 кормит derive.
6. **Половинки канала** (`Sender`/`Receiver`) — возможно разное `#share`/move-обращение; подтвердить, что обе выражаются `#share` + `consume` без спец-кейса.
7. **Region/`sending`-инференс** (Swift SE-0414) — нужен ли, или `consume`/move уже покрывает 100% (вероятно да — подтвердить).
8. **Диагностика leakage:** добавление не-`#share`-поля молча снимает `#share` («почему тип перестал шариться») — сообщение + политика (это неявная часть публичного API).
9. **Полнота явности захвата spawn:** обязательны явные **только move'ы** (`spawn consume c = e { … }`), а `ro`/`#share`/
   by-value — неявные-но-проверяемые (**рекомендация**); ИЛИ полный capture-list `spawn(consume c, ro cfg) { … }` для
   ВСЕГО, что пересекает границу (максимально «no silent capture», но многословно для каждого примитива/loop-var). Решить.
