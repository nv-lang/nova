## 1. Контекст

V7 shipped IPA infrastructure: `module_write_sets` public API
computes per-method field-write-set с iterative transitive closure.

**V7 gap:** infrastructure built, but NOT integrated с V1 mut path /
V2 LICM / V3 pure / V4 chain layers. All four layers still use
conservative "any Call = barrier" rule. write_sets exposed но
unused.

V7.1 ships integration: refactor barrier checks across all 4 layers
to consult write_sets when checking a self-method call.

## 2. Design

### 2.1 IPA context propagation

Introduce `IpaCtx<'a>` struct holding:
- `write_sets: &'a HashMap<(String, String), HashSet<String>>`
- `recv_type: &'a str` — receiver type of currently analyzed fn.

Pass through call chain to barrier-checking helpers.

### 2.2 V1 mut path integration

Refactor `count_mut_prefix_reads` → `count_mut_prefix_reads_ipa`:
- Accept `Option<IpaCtx<'a>>` (None = legacy conservative path).
- `stmt_is_barrier_for_ipa(s, fname, ipa)` — same as V1 but:
  - When walking expressions, for each `Call { func: Member{
    SelfAccess, M }, args }`:
    - Look up `write_sets[(recv_type, M)]`.
    - If field `fname` ∉ write_set → NOT a barrier for this field.
    - If method unknown OR field in write_set → barrier.
  - Non-self calls (free fn, static, external) → barrier (conservative).
  - Spawn/Supervised/etc → barrier.

This means: mut field cache survives across self-method calls that
don't write that field.

### 2.3 V2 LICM integration

`collect_loop_eligible_fields` уже checks `body_has_call`. Refactor
to `body_has_invalidating_call_for(body, fname, ipa)`:
- Walk body, find Calls. For each: check IPA.
- If ALL calls в body don't write `fname` → eligible (no barrier).

### 2.4 V3 pure-cache integration

`body_has_any_self_field_write` already too coarse. V3 conservative:
если ЛЮБОЙ `@F = ...` write anywhere → skip all pure cache.

V7.1 refinement: для каждого pure-method `@M()`, look up `M`'s
write_set (pure methods have empty write_sets by definition since
they have no effects). Cache `@M()` valid while no @F = ... where
F overlaps с M's read-set.

**V7.1 simpler approach:** since `#pure` methods have `Purity::Pure`
which by D24 implies no mutation, pure-call results are invalidated
ONLY by mutations to fields that pure method reads. We don't have
explicit "reads" but can use **conservative**: pure method reads
all `@F` accessed in its body.

V7.1 ships: per-pure-method `field_read_set` inference; cache `@M()`
result valid while no `@F = ...` where F ∈ M's field_read_set.

### 2.5 V4 chain integration

For chain `@a.b.c`, root field is `a`. V4 conservative: any `@F = ...`
skips all chain caching.

V7.1 refinement: chain `@a.b.c` invalidated only by:
- Direct write to root: `@a = ...` (replaces entire subtree).
- Write to any intermediate path segment.
- Method call that writes `a` according to IPA write_set.

V7.1 ships: per-chain invalidation — chain `@a.b.c` survives writes
to `@d` (different root) AND survives self-method calls that don't
write `a`.

## 4. Acceptance criteria (A7.1.1-A7.1.10)

- **A7.1.1** mut field cache survives self-method call where callee
  doesn't write field (per IPA write_set).
- **A7.1.2** mut field cache invalidated by self-method call where
  callee writes field.
- **A7.1.3** mut field cache invalidated by non-self call
  (conservative — caller doesn't know callee semantics).
- **A7.1.4** LICM mut hoist works when loop body's calls don't
  write the field.
- **A7.1.5** pure-call cache survives writes to fields outside
  pure method's read-set.
- **A7.1.6** chain cache survives writes to unrelated root fields.
- **A7.1.7** Regression: V1-V6 fixtures all PASS — no behavior
  change for code without IPA wins.
- **A7.1.8** plan123_7_1 fixtures ≥10 (8+ positive с new IPA wins,
  2+ negative с invalidation).
- **A7.1.9** Semantic equivalence verified — NOVA_FIELD_CACHE_IPA=0
  fallback identical к V7 baseline (conservative).
- **A7.1.10** D220 amend V7.1 landed + cross-refs updated.

## 5. Risk register

- **R-7.1.1:** transitive closure incompleteness. iterative
  closure ≤10 iterations may miss deep recursive calls.
  Mitigation: if callee not in write_sets registry, treat as
  writing all (conservative).
- **R-7.1.2:** write_set staleness across modules. V7 is
  per-module; cross-module callee write_sets unknown.
  Mitigation: conservative for external/cross-module calls.
- **R-7.1.3:** pure method read-set computation. Reads include
  field accesses + method calls' reads. V7.1 conservative:
  read-set = direct `@F` accesses в body + recursive union
  through called methods (similar to write_set).
