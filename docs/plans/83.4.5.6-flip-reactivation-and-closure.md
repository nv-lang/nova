### Ф.4 — Plan 83.2 Ф.3: spec D-block + speedup benchmark (~1-2 dev-day)

#### Spec D-block

В `spec/decisions/06-concurrency.md` новый D-block «Default-on M:N
runtime»:
1. **Default model.** Compiled-бинарь M:N вкл по умолчанию;
   `nova run` (интерпретатор) — однопоточный.
2. **Worker-count resolution.** `explicit runtime.init(n>0)` >
   `NOVA_MAXPROCS` env > `uv_available_parallelism()`. Клэмп [1, 1024].
3. **Lazy worker pool.** Workers поднимаются на первом `spawn`;
   hello-world без spawn = 0 worker-потоков (Plan 83.2 §4 acceptance).
4. **Escape hatch.** `NOVA_MAXPROCS=1` — один worker, без параллелизма.
5. **Worker не делает блокирующую работу инлайн** — `Blocking`-эффект
   (Plan 83.3) обязательно `blocking{}` для FFI / syscall'ов.
6. **Spawn ordering**: НЕ специфицирован.
7. **Cancellation**: hierarchical через scope-tree (Plan 83.4.5.1) —
   parent cancel cascades в children, wake-all parked.
8. **Detach**: fire-and-forget на worker (Plan 83.4.5.2), не inline.
9. **Per-fiber state**: handler-snapshot per fiber (Plan 83.4.2 Ф.2 +
   83.4.5.4 nested inheritance) — travels с fiber'ом cross-worker.

**Cross-runtime comparison table** (скопировать из
[Plan 83.4 §2](83.4-mn-supervised-drain-hardening.md#2-паритет-с-другими-mn-runtime--целевая-планка)).

#### Speedup benchmark

`bench/micro/parallel_*`:
- `parallel_sum.nv` — `parallel for i in 1..N { sum += i*i }` CPU-bound.
- `parallel_hash.nv` — fan-out hash 10⁵ keys.
- `parallel_fib.nv` — `parallel for i in 1..40 { fib(i) }` recursive.

Замер ratio:
1. `NOVA_MAXPROCS=1` — baseline single-thread.
2. `NOVA_MAXPROCS=N` (NumCPU обычно 4-16) — multi-thread.
3. Сравнение с Go reference (опционально — `bench/reference/go_parallel_*.go`).

**Acceptance targets:**
- `parallel_sum` на N=4: ≥ 3.0× speedup vs MAXPROCS=1 (≥ 0.75× линейного).
- `parallel_sum` vs Go (если бенч есть): ≥ 0.85× Go time.
- Если ниже 0.85× Go — измерить вклад Boehm GC через `--gc malloc`;
  если разрыв в GC — задокументировать в spec.
