## Зачем

Текущий `Once` baseline (`std/runtime/sync.nv:172-214`) — двух-шаговый
`run()/done()` pattern: пользователь должен помнить вызвать `done()`,
иначе все waiters навсегда заблокированы. Это **fragile API** (M8):
забытый `done()` = silent infinite wait.

Industry-standard: **closure-form** `call_once(fn)` — single call,
exception/panic safe (RAII), невозможно misuse. Rust/Java/Kotlin —
все имеют closure form primary.

Дополнительные gaps:

| Gap | Use case | Industry |
|---|---|---|
| **`call_once(closure)`** safe API | exactly-once init, panic-safe | Rust `Once::call_once`, Kotlin `synchronized`+flag, Java `AtomicInteger`-CAS |
| **`OnceCell[T]`** value-capturing init | lazy-init с capture value | Rust `OnceCell<T>` / `once_cell` crate / Rust 1.70+ stdlib |
| **`Lazy[T]`** auto-init on first access | global-style lazy constant | Rust `Lazy<T>` (once_cell), Kotlin `by lazy {}`, Java holder pattern |
| **`OnceGuard consume` (V2)** | explicit `.commit()` vs `.abort()` (e.g., conditional init) | Plan 100.7 future |

**M8 (master):** `call_once(closure)` primary; `run()/done()` deprecated
alias с migration warning.
**M9 (master):** OnceCell + Lazy — distinct types (different APIs).

## Scope

### In scope

**`Once` (hardened):**
- **Add:** `call_once(body fn() -> ())` — closure-form, primary API. Безопасный
  даже при panic в body: cleanup транзитом через defer (mutex released,
  waiters wake с error state).
- **Add:** `is_completed() -> bool` (observability).
- **Deprecated:** existing `run() -> bool` / `done()` — keep working,
  emit `W_ONCE_RUN_DONE_DEPRECATED` через `compiler-codegen/src/diag.rs` (`Diagnostic` + `Suggestion` с `Applicability::MachineApplicable`), suggest
  `call_once(fn)`.
- **Panic-safety:** если body panics inside `call_once`, Once transitions
  to `Poisoned` state; subsequent `call_once` calls also panic с
  `OncePoisoned` error. (Unlike `run()/done()` где waiters навсегда
  заблокированы.)

**`OnceCell[T]`** (value-capturing):
- `new() -> Self` (empty state)
- `get() -> Option[T]` (returns Some если initialized; None иначе)
- `set(v T) -> bool` (returns true если this was the first set; false
  если уже initialized — value NOT replaced)
- `get_or_init(init fn() -> T) -> T` — primary API. Lazy initialization
  using init closure, exactly-once across concurrent fibers. Subsequent
  calls return cached value без re-running init.
- `take() -> Option[T]` — atomic remove value (returns Some если был
  set; resets cell back to empty). Useful для shutdown patterns.
- `is_initialized() -> bool` — observability.

**`Lazy[T]`** (auto-init wrapper):
- `Lazy.new(init fn() -> T) -> Self` — stores init closure, defers
  execution до first access.
- `force() -> T` — primary API. Triggers init if not yet, returns
  value. Idempotent (subsequent calls return cached).
- `is_forced() -> bool` — observability.
- **Difference vs OnceCell:** Lazy hides init closure (one-shot setup
  pattern: «вот функция, которая создаёт значение, дай мне его»);
  OnceCell allows set/get/get_or_init с different closures per call
  (more flexible).

### Out of scope

- **`OnceGuard consume`** (Plan 100.7-style commit/abort) — V2 sub-plan
  103.9 (gated on Plan 100.7).
- **Global Lazy** (`static LAZY: Lazy<T>` Rust pattern) — Nova
  doesn't have `static` globals; user puts `Lazy[T]` value в module-
  level binding. Plan 103.5 design works в любом scope.
- **Concurrent-init coalescing** (multiple init closures concurrently —
  only one runs, others wait) — implicit для OnceCell.get_or_init (по
  semantics), не требует separate API.
- **`Lazy[T]` с custom strategy** (sync vs none vs publication — Kotlin
  `LazyThreadSafetyMode`) — V1 sync-only; future plan если нужно.
- **OnceCell без panic-safety** (Rust `UnsyncOnceCell`) — Nova
  всегда sync (fiber-safe); single-threaded variant не нужен (sync
  primitives M:N-correct по дизайну).

## API design

### `Once` (hardened)

```nova
module runtime.sync

type OnceState {
    Fresh
  | Running
  | Done
  | Poisoned
}

/// Exactly-once execution barrier. Fiber-aware (parking).
///
/// # Primary API: `call_once(fn)` (M8)
///
/// Panic-safe: если closure panics, Once transitions to `Poisoned`;
/// subsequent `call_once` also panic с `OncePoisoned` error.
///
/// # Legacy API: `run()/done()` — deprecated since 0.2
///
/// Emit `W_ONCE_RUN_DONE_DEPRECATED`; suggest `call_once(fn)`.
/// Сохранены для backward compat (M14).
#stable(since = "0.1")
export external fn Once.new() -> Self

/// PRIMARY API (M8). Closure-form exactly-once init.
///
/// # Panic-safety
/// If `body` panics, Once → `Poisoned`; subsequent calls re-panic.
///
/// # Examples
///
/// ```nova
/// let init = Once.new()
/// init.call_once { ||
///     expensive_setup()
/// }
/// // safe: subsequent calls no-op
/// init.call_once { ||
///     never_called_again()
/// }
/// ```
#stable(since = "0.1")
export external fn Once mut @call_once(body fn() -> ())

#stable(since = "0.1")
export external fn Once @is_completed() -> bool
#stable(since = "0.1")
export external fn Once @state() -> OnceState

// ─── Deprecated API ───
#deprecated(since = "0.1", suggest = "call_once(fn)")
#stable(since = "0.1")
export external fn Once mut @run() -> bool

#deprecated(since = "0.1", suggest = "call_once(fn)")
#stable(since = "0.1")
export external fn Once mut @done()
```

### `OnceCell[T]`

```nova
/// Value-capturing one-shot initialization cell. Empty при new();
/// transitions to initialized at first `set` / `get_or_init`.
///
/// # Use case
/// Cached computed value, lazy resource initialization (db connection,
/// config snapshot).
#stable(since = "0.1")
export external fn OnceCell[T].new() -> Self

#stable(since = "0.1")
export external fn OnceCell[T] @get() -> Option[T]

#stable(since = "0.1")
export external fn OnceCell[T] mut @set(v T) -> bool

/// PRIMARY API. Initializes cell if empty (using `init` closure),
/// returns value. Concurrent calls: only one init closure runs;
/// others wait for completion + receive cached value.
///
/// # Panic-safety
/// If `init` panics, cell remains empty; next call retries (unlike
/// Once.call_once which poisons).
#stable(since = "0.1")
export external fn OnceCell[T] mut @get_or_init(init fn() -> T) -> T

/// Atomically remove value. Returns Some if was initialized;
/// cell becomes empty (re-initializable).
#stable(since = "0.1")
export external fn OnceCell[T] mut @take() -> Option[T]

#stable(since = "0.1")
export external fn OnceCell[T] @is_initialized() -> bool
```

### `Lazy[T]`

```nova
/// Auto-init wrapper. Stores init closure, deferred execution until
/// first `force()`. Result cached, idempotent.
///
/// # Vs OnceCell
/// Lazy hides init: closure provided at construction (one-shot pattern).
/// OnceCell allows different closures per get_or_init call (flexible).
/// Use Lazy для «вот значение, лениво инициализируется»; OnceCell для
/// «cache slot с different init strategies».
#stable(since = "0.1")
export external fn Lazy[T].new(init fn() -> T) -> Self

/// PRIMARY API. Triggers init if not yet done, returns cached value.
///
/// # Panic-safety
/// If init panics, Lazy → `Poisoned`; subsequent `force()` re-panic.
#stable(since = "0.1")
export external fn Lazy[T] mut @force() -> T

#stable(since = "0.1")
export external fn Lazy[T] @is_forced() -> bool
```

## Industry comparison

| Feature | Nova 103.5 | Go | Rust | Java | Kotlin |
|---|---|---|---|---|---|
| **`Once.call_once(fn)`** | ✅ primary M8 | ✅ `sync.Once.Do(f)` (closure-form only) | ✅ `Once::call_once(\|\| ...)` | ❌ user-implements via `AtomicReference` CAS | ✅ `synchronized + flag` |
| **`Once.run/done` legacy** | ⚠ deprecated alias (M14) | ❌ | ❌ | n/a | n/a |
| **`OnceCell[T]`** | ✅ | ❌ (use sync.Once + global var) | ✅ stdlib 1.70+; `once_cell` crate | ❌ (use `AtomicReference`+CAS) | ✅ `lateinit var` (different semantics) |
| **`OnceCell.get_or_init(closure)`** | ✅ | n/a | ✅ | n/a | n/a (use `by lazy {}`) |
| **`Lazy[T]`** | ✅ | ❌ | ✅ `once_cell::sync::Lazy<T>`; in stdlib `LazyLock` 1.80+ | ✅ `Lazy<T>` (interface) | ✅ `by lazy { }` |
| **`Lazy.force()`** | ✅ | n/a | ✅ `Lazy::force` | ✅ `Lazy.get()` | implicit via `val x by lazy { }` |
| **Panic-safety contract** | ✅ documented (Once → Poisoned; OnceCell → re-init OK; Lazy → Poisoned) | ⚠ implicit (function returns) | ✅ explicit (`PoisonError`) | ❌ user-responsibility | ❌ user-responsibility |
| **`take()`** на OnceCell | ✅ | n/a | ✅ | n/a | n/a |

**Nova edge:**
- Three distinct primitives (Once / OnceCell / Lazy) с clear separation
  of concerns (M9);
- Explicit Poisoned state на Once (vs Rust where Poisoned only on Mutex);
- `take()` на OnceCell — useful для shutdown patterns;
- Documentation D173 (Plan 103.7) AI-guidance: «когда что использовать»
  decision tree.

## Фазы

### Ф.0 — Baseline audit (~0.1d)

- Existing Once impl (`sync_primitives.h:391-461`) — state machine
  NEW/RUNNING/DONE, atomic transitions.
- Plan 99 method-level generic routing для `OnceCell[T]`/`Lazy[T]` — ✅ ready.
- Confirm `Option[T]` API для get/take return.

### Ф.1 — `Once.call_once(fn)` + Poisoned state (~0.3d)

- Extend `Nova_Once` struct: add `NOVA_ONCE_POISONED = 3` state.
- `call_once(body)` impl:
  - Transition NEW → RUNNING (CAS).
  - Execute body within try/catch:
    - success → DONE (release-store); wake all waiters.
    - panic → POISONED (release-store); wake all waiters с error
      throw on resume.
  - Pattern matches Rust `Once::call_once` panic-safety.
- `is_completed() / state()` getters.
- Backward compat: `run()/done()` legacy API сохранены, emit
  `W_ONCE_RUN_DONE_DEPRECATED` (lint registered).

Acceptance: 4 positive (call_once exactly-once across fibers,
state observable, deprecation warning emitted, panic poisons).

### Ф.2 — `OnceCell[T]` impl (~0.4d)

- `nova_rt/sync_primitives.h`: `Nova_OnceCell` struct (Once internal +
  value slot + value-set flag).
- `get() -> Option[T]`: load value flag (acquire); if set → return
  Some(value); else None.
- `set(v) -> bool`: CAS-style first-writer-wins; if was-empty → store
  value (release), return true; if was-set → return false (value
  unchanged).
- `get_or_init(init)`: combine Once.call_once + set:
  - If is_initialized → return cached.
  - Else use internal Once: call_once { value = init(); set_flag = true }.
  - After Once → return value (must be set).
  - Panic in `init`: cell remains empty (NOT poisoned; can retry).
- `take() -> Option[T]`: atomic test-and-clear: if was-set → return
  Some(value), clear flag; else None.
- `Nova_OnceCell_T` typedef per-T (mono'd by Plan 99 routing pattern).

Acceptance: 4 positive (get_or_init exactly-once, set returns false on
second call, take resets, panic in init retries).

### Ф.3 — `Lazy[T]` impl (~0.3d)

- `nova_rt/sync_primitives.h`: `Nova_Lazy` struct (OnceCell internal +
  init closure ref).
- `Lazy.new(init)`: store closure, OnceCell uninit.
- `force()`: `OnceCell.get_or_init(stored_init_closure)`; panic-on-init
  poisons Lazy (NOT OnceCell), subsequent force re-panic.
- `is_forced() -> bool`: delegate to OnceCell.is_initialized.

**Note:** Lazy = thin wrapper over OnceCell + saved init closure. The
poison semantics differ (Lazy panics on subsequent force after init
panic; OnceCell allows retry с new init).

Acceptance: 3 positive (force lazy, idempotent, poison on panic).

## Risks + mitigations

| Риск | Mitigation |
|---|---|
| `OnceCell[T]` через Plan 99 routing — может conflict с existing routing | Verify в Ф.2; fallback на explicit mono per-T если routing breaks |
| Recursive `get_or_init` deadlock vs panic — design decision | Ф.2 decision documented в D171; predefined behavior (recommend panic — early error) |
| Once Poisoned semantics break Once.run/done backward compat (legacy callers expecting success path) | Legacy path не triggers Poisoned (только call_once does); Ф.1 keeps state machine compatible |
| Lazy storing closure ref — GC tracking для closure captured vars | Existing closure capture mechanism (Plan 99); GC tracks; no special handling |
| `take()` race condition с concurrent `get` | CAS-based; D171 specifies linearization point |
| `set(v)` while concurrent `get_or_init` running — value clash | Document: set() bypasses get_or_init coordination; if both used, set() may interleave (last-write-wins для set; get_or_init only triggers init if empty) |
