# Plan 172.4 — Единый value-ABI + автоматический placement (by-ref / heap↔stack)

**Статус:** 📋 proposed 2026-06-21 · 🔬 design+review 2026-06-28 (multi-agent; greenlight владельца). Дизайн-нота в harvest; **HARD dep: стартует ПОСЛЕ закрытия 172.1 U.5/U.6** (единый ABI = `resolved_type_to_c` + escape-анализ, by-ref/heap↔stack без ручных аннотаций). Детальный design-lock писать при старте (после 172.1).

## 1. Цель

Компилятор обращается со всеми value-типами **единообразно** (один путь representation +
приёмник + передача) и **автоматически** выбирает передачу/размещение (by-ref vs by-value,
heap vs stack) — без явных аннотаций ([Q29](../../spec/open-questions.md): нет `ref T`). ABI/placement
**выводятся** из `ResolvedType` + escape/recursion-анализа (D315: «ABI выводится, не хранится»),
а не разбросаны по веткам codegen.

## 2. Acceptance-критерий (от владельца, 2026-06-21)

ЭТО должно компилироваться, запускаться и проходить:

```nova
type Point value { x int, y int }
fn Point mut @inc(a int) -> @ { @x += a; @y += a; @ }

mut p Point = { x: 0, y: 0 }
ro p2 = p.inc(10).inc(20)
assert(p == p2 && p.x == 30)
```

**Сейчас НЕ работает** (диагностика 2026-06-21, release nova на ветке plan-172):
1. **Fluent `mut @ -> @` на value-record** → CC-FAIL: codegen путает `NovaValue_Point` (by value)
   и `NovaValue_Point*` (by pointer) в цепочке (`initializing 'NovaValue_Point *' with
   'NovaValue_Point'`). Heap-record (`type Point {}`) тот же паттерн **проходит** → пробел
   value-record-специфичен.
2. **`==` на value-record** → CC-FAIL: `==` на C-структуре. Heap-record `==` компилится, но это
   **reference-eq** (два разных `{1,2}` → не равны). Sum-типы имеют авто-структурное `==`
   (tag+payload, [02-types.md](../../spec/decisions/02-types.md) ~:6797) — value-records НЕ имеют.

## 2.1. Подход (важно — §0/§10): НЕ патчить старый codegen, чинить ПОСТРОЕНИЕМ

Формулировка владельца (2026-06-21): **«поломано в СТАРОМ компиляторе; главное — чтобы работало
в НОВОМ»**. Эти CC-FAIL'ы — артефакт **фрагментированного** value-ABI (раздельные пути `NovaValue`
/ `NovaTuple` / heap-record — источник постоянных багов). **Запрещено** чинить их band-aid'ом в
старом codegen-пути ([`compiler-conventions.md`](../compiler-conventions.md) §10-анти-паттерн
«углублять фрагментацию»; §0 «сокращать»). Правильно: единый носитель `ResolvedType` (172.1/D315)
даёт **ОДИН** value-ABI-лоуэринг, и acceptance-кейс проходит **ПО ПОСТРОЕНИЮ** — потому что
раздельности нет. То есть **Ф.3 (единый value-ABI) — первичная работа**; fluent `mut @` (Ф.1) и
структурное `==` (Ф.2) — её **следствия/acceptance**, а не отдельные заплатки. Зависимость на 172.1
(единый носитель authoritative) — жёсткая: без него «починка» свелась бы к патчу старого пути.

## 3. Фазы

- **Ф.1 — value-record fluent `mut @ -> @` работает (acceptance).** Рассинхрон `NovaValue_X` ↔
  `NovaValue_X*` в цепочке — следствие фрагментации; устраняется **единым value-ABI-лоуэрингом
  (Ф.3)**, а НЕ точечным патчем старого `NovaValue`-пути (§2.1). Регресс-гард = acceptance-кейс.
- **Ф.2 — value-record структурное `==` (+ дизайн ref-vs-structural).** Авто-вывод структурного
  `==` для value-records (tag/поля, как sum-типы), а не `==` на C-структуре. Дизайн-вопрос (D-блок):
  value-record `==` = **структурное** (они значения); heap-record `==` = текущее reference — оставить
  или тоже структурить? → решить spec-first.
- **Ф.3 — единый value-ABI representation.** Свести `value-record` / `named tuple` / `Tuple(int,str)`
  к ОДНОМУ пути representation + приёмника. Убрать раздельность (named-tuple `ro @` by-value vs
  value-record by-pointer — [Plan 128](128-mut-receiver-abi.md)). Вектор — 172.1 U.5/U.6 (свернуть
  tuple/value-record/sum-schema дубли); 172.4 завершает на ABI-уровне.
- **Ф.4 — авто-by-ref передачи параметров.** Большие value передавать указателем автоматически,
  когда копия не наблюдаема (закрывает ЗА-аргумент Q29 без `ref T`). Foundation: `escape_analyze.rs`
  (Plan 127) уже авто-промоутит value-records — обобщить на by-ref-передачу.
- **Ф.5 — авто heap↔stack placement, в т.ч. нерекурсивный sum→стек.** Sum по умолчанию heap
  (`Nova_X*`) — правильно для **рекурсивных** (деревья/связи); **нерекурсивные/перечисления** →
  авто-свёртка в **стек-value (tagged-union)**, как билтин Option (`NovaOpt` stack) vs Result (heap),
  [Plan 62.A.bis](62.A.bis-sum-schema-registry.md). Детектор рекурсии есть (size/GC,
  [08-runtime.md](../../spec/decisions/08-runtime.md) ~:8211). Консервативно: рекурсия/неопределённость
  → heap (Plan 127 over-promote stance).
- **Ф.6 — copy-elision: RVO/NRVO + move-on-last-use для value-типов** (перф; добавлено 2026-06-26
  по итогам обсуждения с владельцем). Свежие/уникальные value-значения (результат `return`, литералы,
  временные), у которых escape-анализ (Plan 127) доказал **отсутствие второго владельца**, **строятся
  прямо в слоте-назначении** (RVO/NRVO через скрытый `sret`-указатель) либо **перемещаются на
  последнем использовании** (move-on-last-use) — вместо `memcpy`. **Прозрачно** (для `value`-типов
  копия ≡ move наблюдательно: нет идентичности; heap — moot, by-ref). **БЕЗ нового синтаксиса** (вывод,
  как Rust/Swift; НЕ `T&&`/`tmp`-маркер): едет на ОБЫЧНОМ by-value параметре, на `consume` (там даже
  move-`memcpy` не нужен), и на `return`. **Ортогонально `consume`:** `consume` = контракт владения
  (affine); это — оптимизация копий БЕЗ контракта (владелец 2026-06-26 развёл оси «владение» vs «не
  копировать лишнее»). **Обобщает D326 R7/R8** (`-> @` value-record decay = частный случай). GC проще
  C++: нет деструкторов → move = чистый перенос байтов, без moved-from-состояния (нет move-ctor).
  **Инвариант** (узкий D-блок, если понадобится): компилятор ВПРАВЕ move'ить доказанно-уникальные
  value-временные; программа НЕ зависит от идентичности копии (для value-record уже истинно).
  Маркер `[M-172.4-rvo-move-on-last-use]`. **Приоритет P3** (перф; важно для крупных value-record,
  мелкие/heap ≈ ноль); делать ПОСЛЕ Ф.3-Ф.5 (единый ABI), мерить blast-radius. **Корректность не
  затрагивает** — чистая оптимизация.

## 4. Сквозные критерии (как 172.1)

1. Соответствие [`compiler-conventions.md`](../compiler-conventions.md) §0/§1/§2 + §6 (ошибки понятны)
   + §2 (быстрый сгенерированный код — это и есть мотив Ф.4/Ф.5).
2. Без упрощений (как для прода). Behavior-change → детект-режим/замер blast-radius ДО фатала (§7).
3. Pos+neg через релизный `nova` (C-codegen). Acceptance-кейс — регресс-гард.
4. 0 регрессий против чистого бинаря; spec-first для семантики (`==`, placement) — D-блок.
