# Plan 143.2 — Leaf function-entry preempt-check elision  `[M-opt-leaf-preempt-entry-elision]`

> **Папка/ветка:** `D:\Sources\nv-lang\nova-p143` / `plan-143-leaf-preempt-elision`.

## 1. Что и зачем (два слова)
Убрать **лишний function-prologue `nova_preempt_check()`** ([emit_c.rs:14925](../../compiler-codegen/src/codegen/emit_c.rs#L14925),
Plan 44.7 — эмитится первым стейтментом КАЖДОЙ Nova-функции) на функциях, которые **провабельно достигают
safepoint без своего entry-check**. Сейчас тривиальные leaf-форвардеры (напр. `StringBuilder.append(f32)` →
`append(f64)`) несут барьер-call зря. Best-in-class: уровень HotSpot safepoint-elision / Go nosplit, точнее за
счёт whole-program монтоморфизации.

## 2. Соундность (инвариант вытеснения)
Безгранично без yield'а fiber бежит **ТОЛЬКО** через: (a) **цикл** — его back-edge check СОХРАНЁН (§2.A элидит
лишь const-small FINITE циклы); (b) **рекурсию** — цикл в call-графе. ⇒ Элидировать entry-check безопасно, если
**каждый рекурсивный цикл имеет ≥1 члена с сохранённым safepoint'ом**.

**Правило (conservative KEEP):** функция `f` СОХРАНЯЕТ entry-check ⟺ любое из:
1. `f` на call-граф **цикле** (self-recursive ИЛИ SCC>1 — взаимная рекурсия);
2. `f` делает **indirect/closure-call** (цель статически неизвестна → не доказать ацикличность);
3. `f` делает **FFI/extern-call** (C может вызвать Nova-callback → неизвестная цель);
4. `f` **address-taken** (используется как fn-value/callback → цель indirect-вызова).

**ELIDE** иначе (non-recursive, статически-разрешённые callee, не-indirect, не-address-taken).
`has_loop` для entry-check **НЕ важен** (цикл держит свой back-edge check; bounded straight-line терминируется).

## 3. Интеграция — source-level pre-pass (до эмита)
Call-граф над **source-level FnDecl-символами** (все `module.items` + импортируемые), computed ДО эмита —
полный, без on-demand-моно-проблемы. Соундно над монтоморфизацией: рекурсия/indirect/FFI/address-taken
определяются ИСХОДНИКОМ → KEEP-статус шаблона наследуется ВСЕМИ инстансами (over-approximation: если source-edge
f→g→f есть, все инстансы в цикле → KEEP; indirect → KEEP). Ни один цикл не теряется → sound.

**Алгоритм:**
1. Собрать узлы = все FnDecl (по mangled/source-символу).
2. Для каждого тела (AST-walk): direct-call → ребро f→callee (резолв по имени/методу); call где func — не
   статически-резолвимый fn-символ (closure/fn-ptr) → `makes_indirect`; extern/FFI-callee → `makes_ffi`.
3. Отдельный скан: fn, использованная как VALUE (не в call-позиции func) → `address_taken`.
4. SCC (Tarjan) → cycle-члены. `KEEP = {cycle} ∪ {makes_indirect} ∪ {makes_ffi} ∪ {address_taken}`.
5. В `emit_fn` prologue (14925): эмитить `nova_preempt_check()` ТОЛЬКО если fn ∈ KEEP.
**Conservative default:** любой неразрешённый callee / сомнение → KEEP (никогда не элидим под вопросом).

## 5. Критерии приёмки
1. ✅ **Позитив:** leaf/forwarder (no-loop, non-recursive, resolved callee, не-address-taken) → entry-check
   ЭЛИДИРОВАН (проверка в сгенерённом C; `StringBuilder.append(f32)` — конкретный кейс).
2. ✅ **Негатив/KEEP:** прямая рекурсия, взаимная рекурсия (SCC), indirect/closure-call, FFI-call, address-taken
   fn → entry-check СОХРАНЁН.
3. ✅ **Соундность:** ни один рекурсивный цикл не остаётся без safepoint'а (анализ + stress: рекурсивный fiber
   вытесняется/yield'ит). Conservative над моно.
4. ✅ **Регрессия:** 0 new fail (Vec/collection/concurrency слайс; pre-existing подтверждены на main).
5. ✅ **Тесты через релизный nova** (build + grep C).
6. ✅ **«Без упрощений как для прода» — ОБЯЗАТЕЛЬНЫЙ критерий:** conservative KEEP при любом сомнении; sound над
   монтоморфизацией; никаких shortcut'ов, маскирующих недоказанные случаи.
