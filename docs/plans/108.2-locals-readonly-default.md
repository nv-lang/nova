# Plan 108.2 — Локальные `let` без `mut` = read-only (enforcement D36)

> **Sub-plan Plan 108** (Plan 108.1 уже flip'нул params).  Plan 108.2 —
> finally enforce D36 для local bindings: `let x = ...` immutable,
> `let mut x = ...` mutable.  Согласует семантику с params (Plan 108.1).

## TL;DR

Спецификация D36 уже говорит: «`let mut acc`/`acc.field = ...` ✓, `let acc`/
`acc.field = 100` ✗ ОШИБКА».  Но компилятор это **не enforce'ит**.  Plan
108.2 закрывает gap: mut-метод/field-assign на `let`-binding без `mut` →
`E_LOCAL_NOT_MUT`.

```nova
// Было (V1 — D36 spec, но no enforcement):
let b = Box.new(1)
b.value = 99                  // компилировалось, silent mutability ✗
b.push(2)                     // тоже OK ✗

// Стало (Plan 108.2 — enforcement):
let b = Box.new(1)
b.value = 99                  // ✗ E_LOCAL_NOT_MUT
b.push(2)                     // ✗ E_LOCAL_NOT_MUT
let mut b2 = Box.new(1)
b2.value = 99                 // ✓
```

### Ф.2 — Type-checker enforcement (~2 часа)

В `compiler-codegen/src/types/mod.rs`:

1. `ConsumeCtx.local_mut: HashMap<String, bool>` — отслеживать is_mut
   для каждого local binding.  Заполняется при processing `Stmt::Let`.
2. При вызове mut-метода на Ident:
   - Сначала проверить `param_mut` (Plan 108.1).
   - Затем `local_mut`.  Если is_mut=false и метод mut → `E_LOCAL_NOT_MUT`
     + machine-applicable Suggestion («add `mut` после `let`»).
3. Field assignment (`b.field = ...`) — то же check.
4. Index assignment (`b[i] = ...`) — то же.
5. `consume X = ...` binding — implicit mut (как params).
