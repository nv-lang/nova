## 1. Контекст

V1 D217 mut-field caching работает на **first-region only**: cache
local emit'ится at body prefix, валиден до first barrier (write OR
call), reads после boundary остаются как `@<F>`.

Реалистичный паттерн:

```nova
fn Counter mut @cycle(target int) -> int {
    ro a = @x          // cached (region A)
    ro b = @x          // cached (region A)
    @x = target        // ← barrier
    ro c = @x          // NOT cached (V1 missed)
    ro d = @x          // NOT cached (V1 missed)
    a + b + c + d
}
```

V1 кеширует только region A (2 reads), а region B остаётся без
оптимизации, хотя там тоже ≥ threshold reads. Пользовательский pattern
(WriteBuffer / Counter-style mutating methods) demonstrates that
second+ regions достаточно частый случай.

V1.1 закрывает гэп: split body на regions by barriers, emit cache
local per region с reads ≥ threshold.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**Новые public-private structs (~30 LOC):**

```rust
#[derive(Debug, Clone)]
struct MutRegion {
    start: usize,                // stmts index inclusive
    end: usize,                  // stmts index exclusive
    reads: usize,
    first_span: crate::diag::Span,
    trailing_included: bool,     // covers Block.trailing too
}

#[derive(Debug)]
struct MutRegionTarget {
    fname: String,
    region: MutRegion,
    region_idx: usize,           // 0 = first region per field
    local_name: String,          // _at_<F> or _at_<F>_r<N>
}
```

**Новый helper `find_mut_regions_with_ipa` (~80 LOC):** walks
`Block.stmts`, splits на barrier boundaries, returns `Vec<MutRegion>`.
Trailing handled separately — extends current region or closes без
trailing if itself a barrier.

**Новый helper `stmt_span` (~20 LOC):** defensive span extraction
covering all `Stmt` variants (used для cache-let placement).

**Modified `cache_fn_with_ipa` (~60 LOC):** replaces flat
`mut_candidates` с per-region targets. Falls back на V1 first-region
logic для FnBody::Expr / External.

**Modified `rewrite_fn_body_split_with_ipa` (~80 LOC):** per-region
read rewrite + per-region let insertion. Two phases:
1. Non-prefix groups (region.start > 0) in descending-`start` order
   so insertion preserves untouched indices.
2. Prefix bucket — ro fields + first-region mut lets prepended together.

**New helper `build_at_field_let` (~15 LOC):** factor out
`let <local> = @<field>` Stmt construction (used by both phases).

**Modified `analyze_fn_for_explain` (~5 LOC):** scan ALL top-level
`_at_*` lets (V1 broke at first non-`_at_*` stmt; V1.1 continues).

## 4. Не в скоупе

- **Nested regions** в if/while/match-arms — currently V1.1 covers
  только top-level `Block.stmts`. Nested expressions внутри single
  stmt treated atomically. V1.2 followup.
- **Callee-non-self-mutation IPA** (V7.5) — fluent-builder pattern
  `@buf.push().push().push()` где push doesn't mutate outer self's
  field. Cross-type IPA out of V1 family scope.
- **Pure-call cache integration** — V3 pure-cache already handles
  pure-method results separately; V1.1 ortho.
