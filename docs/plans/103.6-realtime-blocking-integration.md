## –Ч–∞—З–µ–Љ

D64 (`realtime { }`) –≥–∞—А–∞–љ—В–Є—А—Г–µ—В non-suspension execution –≤ block вАФ
–і–ї—П real-time embedded / GC-pause-free hot loops. D50 ¬І4 (Plan 83.3)
`blocking { }` вАФ leaf FFI/syscall offload. Sync primitives –Є–Љ–µ—О—В
**mixed contract:**

| –Ь–µ—В–Њ–і | –Ь–Њ–ґ–µ—В park? | Allowed –≤ `realtime { }`? | Allowed –≤ `blocking { }`? |
|---|---|---|---|
| **AtomicX.load/store/swap/CAS/fetch_*** | вЭМ leaf | вЬЕ | вЬЕ |
| **fence(MemOrdering)** | вЭМ leaf | вЬЕ | вЬЕ |
| **Mutex.try_lock / try_lock_for** | вЭМ no-wait –Є–ї–Є timer | вЬЕ (try_lock); вЪ† try_lock_for –Є—Б–њ–Њ–ї—М–Ј—Г–µ—В libuv timer (NO –≤ realtime) | вЪ† try_lock_for NO; try_lock вАФ leaf-like, but mu.lock –≤–љ—Г—В—А–Є –µ—Й—С leaf |
| **Mutex.lock / unlock / with_lock** | вЬЕ park possible | вЭМ | вЪ† unlock вАФ leaf; lock вАФ park possible вЖТ NO |
| **RwLock.read / write / with_read / with_write** | вЬЕ park | вЭМ | вЭМ |
| **ReentrantMutex.lock** | вЬЕ park | вЭМ | вЭМ |
| **Semaphore.acquire / with_permit** | вЬЕ park | вЭМ | вЭМ |
| **Semaphore.try_acquire / release** | вЭМ leaf-like | вЬЕ try; вЪ† release wakes waiters вЖТ NO –≤ strict realtime | вЬЕ try; release вЪ† |
| **Barrier.wait** | вЬЕ park | вЭМ | вЭМ |
| **CountDownLatch.await / count_down (last one wakes)** | вЪ† await parks; count_down only wakes –µ—Б–ї–Є reaches 0 | await вЭМ; count_down вЪ† –µ—Б–ї–Є –Љ–Њ–ґ–µ—В trigger wake | await вЭМ |
| **Condvar.wait / wait_for / wait_until / notify_one / notify_all** | wait parks; notify wakes | wait вЭМ; notify вЪ† | wait вЭМ; notify вЪ† |
| **Once.call_once / run** | вЬЕ park (–µ—Б–ї–Є –і—А—Г–≥–Њ–є fiber running) | вЭМ | вЭМ |
| **Once.done** | вЭМ leaf wake | вЪ† wakes waiters | вЪ† |
| **OnceCell.get / set / take / is_initialized** | вЭМ leaf | вЬЕ | вЬЕ |
| **OnceCell.get_or_init** | вЬЕ park (–µ—Б–ї–Є init concurrent) | вЭМ | вЭМ |
| **Lazy.force** | вЬЕ park | вЭМ | вЭМ |

–С–µ–Ј type-checker enforcement –њ–Њ–ї—М–Ј–Њ–≤–∞—В–µ–ї—М –њ–Є—И–µ—В `realtime { mutex.lock() }`
вЖТ compiles, runtime –Љ–Њ–ґ–µ—В suspend вЖТ violates D64 guarantee. Plan 103.6
fixes —З–µ—А–µ–Ј **compile-time errors**.

**M12 (master):** type-checker enforce; runtime-only check (Go/Rust/Java

## Scope

### In scope

- **Type-checker rules** –≤ `compiler-codegen/src/types/check_realtime.rs`
  (–Є–ї–Є extend existing Plan 16 module):
  - –Я—А–Є –њ—А–Њ–≤–µ—А–Ї–µ body –≤–љ—Г—В—А–Є `realtime { ... }` block, walk method
    calls; reject –ї—О–±—Л–µ –Є–Ј **forbidden list** (table above) —Б
    proper error code.
  - Same –і–ї—П `blocking { ... }` body вАФ narrower forbidden list (—В–Њ–ї—М–Ї–Њ
    park-ing, –љ–µ leaf-wake).
- **Method annotations:** –і–Њ–±–∞–≤–Є—В—М marker –≤ `std/runtime/sync.nv`
  –і–ї—П –Ї–∞–ґ–і–Њ–≥–Њ external fn:
  - `#realtime_safe` вАФ leaf method, allowed –≤ realtime/blocking.
  - `#parks` вАФ –Љ–Њ–ґ–µ—В park, forbidden –≤ realtime/blocking.
  - `#wakes` вАФ leaf wake (no park self, –љ–Њ wakes others); contextual
    decision: realtime forbids (wake –Љ–Њ–ґ–µ—В interact —Б scheduler),
    blocking allows (wake вАФ operation –љ–∞ shared state).
  - Default (no annotation) вЖТ conservative `#parks` assumed.
- **6 error codes** (`compiler-codegen/src/diag.rs` (`Diagnostic` + `Suggestion` + `Applicability`), machine-applicable suggestions):
  - `E_REALTIME_SYNC_PARK`: ¬Ђ`{method}` –Љ–Њ–ґ–µ—В –њ—А–Є–Њ—Б—В–∞–љ–Њ–≤–Є—В—М fiber;
    –Ј–∞–њ—А–µ—Й–µ–љ–Њ –≤ `realtime { }`. Suggest: use `try_*` variant¬ї
  - `E_REALTIME_SYNC_WAKE`: ¬Ђ`{method}` –Љ–Њ–ґ–µ—В —А–∞–Ј–±—Г–і–Є—В—М waiter (—В–Њ–ґ–µ
    –Ј–∞–њ—А–µ—Й–µ–љ–Њ –≤ realtime); suggest: defer wake outside realtime block¬ї
  - `E_BLOCKING_SYNC_PARK`: ¬Ђ`{method}` –Љ–Њ–ґ–µ—В park fiber; –≤ `blocking { }`
    leaf-execution only вАФ –њ–µ—А–µ–Љ–µ—Б—В–Є—В—М –≤–љ–µ –±–ї–Њ–Ї–∞¬ї
  - `E_REALTIME_NESTED_SYNC_VIA_FN`: ¬Ђ—Д—Г–љ–Ї—Ж–Є—П `{name}` –Є—Б–њ–Њ–ї—М–Ј—Г–µ—В
    park-ing sync; –љ–µ–ї—М–Ј—П –≤—Л–Ј–≤–∞—В—М –Є–Ј realtime вАФ –Њ—В–Љ–µ—В–Є—В—М `#parks` –Є–ї–Є
    refactor¬ї
  - `W_REALTIME_TRY_LOCK_FOR_TIMER`: ¬Ђ`try_lock_for(Duration)` –Є—Б–њ–Њ–ї—М–Ј—Г–µ—В
    libuv timer (–њ–Њ—В–µ–љ—Ж–Є–∞–ї—М–љ–Њ suspend); –µ—Б–ї–Є duration = 0 вАФ use
    `try_lock()` instead¬ї
  - `W_BLOCKING_NOTIFY_RISK`: ¬Ђ`notify_*` –≤ `blocking { }` –Љ–Њ–ґ–µ—В wake
    fiber –љ–∞ blocking thread (M:N safety risk); use AtomicBool flag +
    fence pattern¬ї

  blocking √Ч method.
- **Test suite:** 12+ negative tests (one per error code √Ч variations).
