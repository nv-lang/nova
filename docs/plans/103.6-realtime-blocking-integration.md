# Plan 103.6 — `realtime { }` / `blocking { }` integration для sync primitives

> **Статус:** 📋 proposed 2026-05-25, не начат
> **Приоритет:** P1 (closure для D64 contract на sync primitives)
> **Оценка:** ~1.5 dev-day
> **Зависимости:** Plan 103.2-103.5 ✅ (нужны типы для enforcement); Plan 16 ✅ (D63/D64 capability enforcement infrastructure); Plan 83.3 ✅ (Blocking effect + nogc enforcement в blocking-body)
> **Master:** [Plan 103](103-sync-primitives-spec-formalization.md). Design M12 применимо.

## Зачем

D64 (`realtime { }`) гарантирует non-suspension execution в block —
для real-time embedded / GC-pause-free hot loops. D50 §4 (Plan 83.3)
`blocking { }` — leaf FFI/syscall offload. Sync primitives имеют
**mixed contract:**

| Метод | Может park? | Allowed в `realtime { }`? | Allowed в `blocking { }`? |
|---|---|---|---|
| **AtomicX.load/store/swap/CAS/fetch_*** | ❌ leaf | ✅ | ✅ |
| **fence(Ordering)** | ❌ leaf | ✅ | ✅ |
| **Mutex.try_lock / try_lock_for** | ❌ no-wait или timer | ✅ (try_lock); ⚠ try_lock_for использует libuv timer (NO в realtime) | ⚠ try_lock_for NO; try_lock — leaf-like, but mu.lock внутри ещё leaf |
| **Mutex.lock / unlock / with_lock** | ✅ park possible | ❌ | ⚠ unlock — leaf; lock — park possible → NO |
| **RwLock.read / write / with_read / with_write** | ✅ park | ❌ | ❌ |
| **ReentrantMutex.lock** | ✅ park | ❌ | ❌ |
| **Semaphore.acquire / with_permit** | ✅ park | ❌ | ❌ |
| **Semaphore.try_acquire / release** | ❌ leaf-like | ✅ try; ⚠ release wakes waiters → NO в strict realtime | ✅ try; release ⚠ |
| **Barrier.wait** | ✅ park | ❌ | ❌ |
| **CountDownLatch.await / count_down (last one wakes)** | ⚠ await parks; count_down only wakes если reaches 0 | await ❌; count_down ⚠ если может trigger wake | await ❌ |
| **Condvar.wait / wait_for / wait_until / notify_one / notify_all** | wait parks; notify wakes | wait ❌; notify ⚠ | wait ❌; notify ⚠ |
| **Once.call_once / run** | ✅ park (если другой fiber running) | ❌ | ❌ |
| **Once.done** | ❌ leaf wake | ⚠ wakes waiters | ⚠ |
| **OnceCell.get / set / take / is_initialized** | ❌ leaf | ✅ | ✅ |
| **OnceCell.get_or_init** | ✅ park (если init concurrent) | ❌ | ❌ |
| **Lazy.force** | ✅ park | ❌ | ❌ |

Без type-checker enforcement пользователь пишет `realtime { mutex.lock() }`
→ compiles, runtime может suspend → violates D64 guarantee. Plan 103.6
fixes через **compile-time errors**.

**M12 (master):** type-checker enforce; runtime-only check (Go/Rust/Java
не делают этого вообще) недостаточно.

## Scope

### In scope

- **Type-checker rules** в `compiler-codegen/src/types/check_realtime.rs`
  (или extend existing Plan 16 module):
  - При проверке body внутри `realtime { ... }` block, walk method
    calls; reject любые из **forbidden list** (table above) с
    proper error code.
  - Same для `blocking { ... }` body — narrower forbidden list (только
    park-ing, не leaf-wake).
- **Method annotations:** добавить marker в `std/runtime/sync.nv`
  для каждого external fn:
  - `#realtime_safe` — leaf method, allowed в realtime/blocking.
  - `#parks` — может park, forbidden в realtime/blocking.
  - `#wakes` — leaf wake (no park self, но wakes others); contextual
    decision: realtime forbids (wake может interact с scheduler),
    blocking allows (wake — operation на shared state).
  - Default (no annotation) → conservative `#parks` assumed.
- **6 error codes** (Plan 50 D102 format, machine-applicable suggestions):
  - `E_REALTIME_SYNC_PARK`: «`{method}` может приостановить fiber;
    запрещено в `realtime { }`. Suggest: use `try_*` variant»
  - `E_REALTIME_SYNC_WAKE`: «`{method}` может разбудить waiter (тоже
    запрещено в realtime); suggest: defer wake outside realtime block»
  - `E_BLOCKING_SYNC_PARK`: «`{method}` может park fiber; в `blocking { }`
    leaf-execution only — переместить вне блока»
  - `E_REALTIME_NESTED_SYNC_VIA_FN`: «функция `{name}` использует
    park-ing sync; нельзя вызвать из realtime — отметить `#parks` или
    refactor»
  - `W_REALTIME_TRY_LOCK_FOR_TIMER`: «`try_lock_for(Duration)` использует
    libuv timer (потенциально suspend); если duration = 0 — use
    `try_lock()` instead»
  - `W_BLOCKING_NOTIFY_RISK`: «`notify_*` в `blocking { }` может wake
    fiber на blocking thread (M:N safety risk); use AtomicBool flag +
    fence pattern»
- **Spec contract** in D172 (Plan 103.7): explicit matrix realtime/
  blocking × method.
- **Test suite:** 12+ negative tests (one per error code × variations).

### Out of scope

- **Refactoring sync примитивов** на realtime-friendly (lock-free)
  alternative — beyond scope; Plan 103.6 just enforces existing
  semantics correctly.
- **Runtime check fallback** (если type-checker missed something) —
  reasonable assumption: nogc/realtime enforcement в bootstrap
  достаточно strict для catch'а; runtime check добавлен в Plan 16,
  reused.
- **Cross-function effect analysis** (whole-program — функция X
  использует park-ing → caller в realtime forbidden) — partial в Plan
  16; full inference — отдельный large plan (not 103.6).

## API design — annotations

Добавить в `std/runtime/sync.nv` (примеры; полный sweep — Ф.1):

```nova
// AtomicI64
#realtime_safe
#stable(since = "0.2")
export external fn AtomicI64 @load(ord Ordering) -> int64
// ... все atomic ops — #realtime_safe

// Mutex
#parks
#stable(since = "0.1")
export external fn Mutex mut @lock()

#realtime_safe
#stable(since = "0.1")
export external fn Mutex mut @try_lock() -> bool

#parks  // libuv timer = potential suspend
#stable(since = "0.2")
export external fn Mutex mut @try_lock_for(timeout Duration) -> bool

#wakes  // leaf wake (no park self)
#stable(since = "0.1")
export external fn Mutex mut @unlock()
```

Annotation parser: extend existing #-attr lexer (Plan 33.x уже
парсит `#opaque`/`#fuel(N)`/`#stable(since)` etc.).

### Ф.1 — Annotation sweep на sync.nv (~0.3d)

- Walk all external fn declarations в `std/runtime/sync.nv` + new ones
  added by 103.2-103.5.
- Annotate каждый: `#realtime_safe` / `#parks` / `#wakes`.
- Default fallback: если annotation missing → conservative `#parks`
  (errs on safe side, type-checker rejects in realtime/blocking).

Acceptance: ≥50 declarations annotated (12 atomics × 13 ops + Mutex
9 + RwLock 12 + Semaphore 8 + Barrier 6 + CountDownLatch 5 + Condvar 7
+ Once 5 + OnceCell 6 + Lazy 4 = ~85 annotations).

### Ф.2 — Type-checker enforcement (~0.5d)

- Extend `check_realtime.rs` / `check_blocking.rs` (Plan 16/83.3 modules):
  - On entering `realtime { }` / `blocking { }` body, push capability
    flag.
  - On method call resolution: check if method has `#parks` / `#wakes`
    annotation.
  - Reject с appropriate error code if conflict.
  - Propagate through user-defined functions: if function calls
    `#parks` method, function inherits `#parks` (conservative); if
    function attempts use in realtime → `E_REALTIME_NESTED_SYNC_VIA_FN`.
- Error code registration (6 codes; Plan 50 D102 format).

Acceptance: type-checker correctly rejects all forbidden combinations
(test matrix Ф.4).

### Ф.3 — Lint warnings (~0.2d)

- `W_REALTIME_TRY_LOCK_FOR_TIMER`: detect `try_lock_for(Duration.zero())`
  pattern, suggest plain `try_lock()`.
- `W_BLOCKING_NOTIFY_RISK`: detect `notify_*` calls within `blocking { }`.

Acceptance: 2 lint tests fire correctly.

## Risks + mitigations

| Риск | Mitigation |
|---|---|
| Conservative annotation (`#parks` default) ломает legitimate use case (leaf-method without annotation) | Sweep Ф.1 explicit для всех methods; lint `W_MISSING_SYNC_ANNOTATION` помечает new declarations |
| Cross-function inference incomplete — false negatives | Plan 16 уже имеет partial; full inference — out of scope; document'ировано в D172 что explicit annotation recommended |
| User-fn `#parks` annotation manually added — tedious | LSP quick-fix: detect call к `#parks` method из non-annotated fn, suggest add `#parks`. Plan 50 D102 integration. Optional V1. |
| `notify_one` в `blocking { }` — wake on blocking-thread может interact с GC | Conservative: `W_BLOCKING_NOTIFY_RISK` warning; full prohibition — отдельный future plan if needed |
| Existing tests без realtime/blocking wrapper — should still PASS | Smoke test full suite after Ф.2; if any regression — annotation bug |
