# Plan 70.1: Module alias resolution в codegen — `import X as th` + `th.func()`

> **Создан:** 2026-05-18. **Статус:** proposed, не начат.
> **Приоритет:** P1. **Трудоёмкость:** ~1-2 dev-days.
> **Родительский план:** [Plan 70](70-no-silent-nova-int-fallback.md)

---

## Зачем

`import std.testing.handlers as th` + `th.fixed_ms(100)` — парсится
корректно, но в C codegen псевдоним `th` не раскрывается в путь
модуля. Генерируется `th` как необъявленный идентификатор → CC-FAIL.

**Воспроизведение:** тест `nova_tests/modules/snowflake.nv` удалён из
batch 7 (Plan 45.B) именно из-за этого бага. Также затрагивает
`std/testing/handlers.nv` и любой код с `import X as alias`.

---

## Проблема

В `emit_c.rs` при резолюции вызова `alias.func()`:
- Парсер сохраняет псевдоним (`th`) в AST как `Ident`
- Codegen при emit Call не ищет псевдоним в таблице импортов
- Эмитит `th` напрямую → `th` undeclared в C

---

## Решение

При emit `Call { func: Member { obj: Ident("th"), name: "func" } }`:
1. Проверить таблицу import aliases — есть ли `th` → `std.testing.handlers`
2. Раскрыть в полный путь модуля для C-name mangling
3. Эмитить корректное `Nova_testing_handlers_func(...)`

---

## Тесты
