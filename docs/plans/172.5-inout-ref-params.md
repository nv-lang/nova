# Plan 172.5 — In-out `ref`-параметры (safe by-ref borrow) + формализация `@`/`-> @`

**Статус:** 📋 proposed 2026-06-26
**Umbrella:** [Plan 172 — Переработка компилятора: типовая система](172-compiler-rework.md).
**D-блок:** [D326](../../spec/decisions/02-types.md#d326) (нормативный дизайн; amends [Q29](../../spec/open-questions.md), D132, D228; adopt D181/D184; bounds D157/D246-P10).
**Источник:** owner-решение 2026-06-26 — переоткрыл [Q29](../../spec/open-questions.md) в **param-mode** форме (отвергнутый 2026-06-21 `ref`-ТИП остаётся отвергнут).
**Deps:** **172.4** (единый value-ABI + авто-placement: `ro ref` авто, `@`/`-> @`, heap↔stack — реализует ЧАСТЬ дизайна; 172.5 кладёт сверху только explicit `mut ref` + эксклюзивность + chain-mode-gating) → **172.1** (ResolvedType authoritative, D315). **Старт ПОСЛЕ 172.4** (или синхронно — `@`/`-> @` общие).
**Приоритет:** P2 (язык-фича; перф/эргономика — §2). Узкая, без лайфтаймов.

---

## 1. Цель

Ввести `ref` как **режим передачи параметра** (Swift `inout` / C# `in`+`ref`), НЕ тип: безопасная in-place мутация caller-значения (`mut ref`) и zero-copy чтение больших стек-значений (`ro ref`, авто) — без сырых указателей, без лайфтаймов. Формализовать `@`-ресивер как частный случай (`mut @ ≡ mut ref @`, `-> @ ≡ ref @`). Полный нормативный дизайн — [D326](../../spec/decisions/02-types.md#d326) (R1-R12).

**Что закрывает (дыра `mut @`-only из Q29):** out-параметры, мутация не-ресивера, несколько mut-аргументов (`swap(mut ref a, mut ref b)`). **Для одиночной мутации канон остаётся `mut @`** — `mut ref` узкий инструмент, не «способ менять любой параметр».

## 2. Дизайн (из D326 — здесь только карта)

| Ось | Правило (D326) |
|---|---|
| Форма | `ro ref a T` (авто/невидимо — это 172.4) · `mut ref a T` (явный, единственный user-facing) |
| Call-site | `inc(ref x)` (короткий `ref`-маркер; `ro ref` маркера нет) |
| `@` ABI | `mut @` всегда by-ptr (любой размер); `ro @` size-discretionary |
| `-> @` | режим = режим ресивера (D181); цепочка гейтит (`c.peek().bump()` → `E_RECEIVER_BINDING_NOT_MUT`); consume `@ -> @` → `E_CONSUME_RECEIVER_RETURNS_AT` |
| `-> @` биндинг | heap/consume/builder → **алиас** (D131/D132); value-record → **копия** (D246 оракул) — НЕ blanket-copy |
| value-`@` escape | decay by-value при bind/return/store (R8) — не висячий ptr |
| Эксклюзивность | УЗКАЯ: same-call same-root prefix-overlap → `E_REF_ALIAS_OVERLAP`; **НЕ Rust/Swift-гарантия** (GC, D157/D246-P10) |
| Эскейп | `ref` не хранить/захватывать/возвращать (кроме `@`) — sync-call lifetime |
| FFI | граница берёт сырой `*`/`*mut`, никогда `ref` |
| Дженерики | `fn f[T](mut ref x T)` ок; `[T consume]`-bound несовместим; `mut ref`≠consume |

## 3. Фазы

- **Ф.0 — Спека (готово).** D326 + amend Q29 (✅ 2026-06-26). Acceptance-кейсы (§4) согласованы.
- **Ф.1 — Parser.** `ro ref`/`mut ref` на параметре; call-site `ref <place>`. Реджект `ref` вне param-позиции (`ref T`-локал/поле/возврат/коллекция → `E_REF_NOT_A_TYPE`). `consume @ -> @` → `E_CONSUME_RECEIVER_RETURNS_AT`.
- **Ф.2 — Checker: эксклюзивность.** Place-overlap отношение (per-pair prefix, НОВОЕ — рядом с consume place-парсингом, не решёткой) → `E_REF_ALIAS_OVERLAP` на same-call same-root. Index-disjoint: только int-литералы (V1, без SMT). Borrow ≠ move (D131/D133 не трогаем).
- **Ф.3 — Checker: `-> @` mode + chain-gating.** Режим возврата `@` = режим ресивера (D181); `mut @` на `ro ref @`-цепочке → `E_RECEIVER_BINDING_NOT_MUT`. Биндинг `-> @`: класс-split (R7) — heap/consume алиас, value-record копия (D246 оракул, amend D132 cross-ref).
- **Ф.4 — Codegen / ABI.** `mut ref` param → C-указатель (как `@`-ресивер, R5). value-`@` escape-decay (R8). `ro ref` авто — это **172.4** (не дублировать; cross-ref). FFI-граница пинит raw (R11).
- **Ф.5 — Тесты.** pos: `swap(mut ref a, mut ref b)`, out-param, `f(mut ref x.a, mut ref x.b)`, fluent `mut @ -> @`-цепочка. neg: `f(mut ref x, mut ref x)`/`f(mut ref x.a, mut ref x)` → `E_REF_ALIAS_OVERLAP`; `c.peek().bump()` → `E_RECEIVER_BINDING_NOT_MUT`; `ref T`-локал → `E_REF_NOT_A_TYPE`; ref в spawn/поле → escape-ошибка. **Acceptance-гейт §4**.
- **Ф.6 — Закрытие.** Полный nova test (батчи), backlog/simplifications/project-creation, перенумерация D326 при коллизии, отметить `[M-181-...]`-fixture как 172.5-гейт.

## 4. Acceptance-критерии

```nova
// (1) in-out: мутация caller-значения через не-ресивер-параметр
fn inc(mut ref a int) { a = a + 1 }
mut x = 5
inc(ref x)
assert(x == 6)

// (2) несколько mut-аргументов (mut @ не выражает)
fn swap(mut ref a int, mut ref b int) { ro t = a; a = b; b = t }
mut p = 1; mut q = 2
swap(ref p, ref q)
assert(p == 2 && q == 1)

// (3) разные поля — ок; одно место дважды — ошибка
fn f(mut ref m int, mut ref n int) { ... }
mut s = Pair{a: 0, b: 0}
f(ref s.a, ref s.b)        // OK
// f(ref s.a, ref s)       // E_REF_ALIAS_OVERLAP

// (4) fluent value-record (общий с 172.4) + chain-gating
type P value { x int }
fn P mut @inc() -> @ { @x += 1; @ }
ro p2 = P{x: 0}.inc().inc()   // value-record: биндинг = копия (R7b)
assert(p2.x == 2)
// c.peek().bump()             // E_RECEIVER_BINDING_NOT_MUT (peek ro, bump mut)
```
- **[M-181-ifexpr-value-materialize-codegen]-fixture** (fluent `-> @`-хвост в if-цепочке, base64 `decode_with`) — **должна и дальше компилиться** (R7/R8 не регрессят R3-фикс `836befcb`).
- **Без упрощений, как для прода** (обязательный критерий).

## 5. Риски / открытые

- **Эксклюзивность-скоуп (R9).** Узкая — не заявлять Rust/Swift-гарантию; явно оговорить «sound-под-GC везде ещё» (D157/D246-P10). Риск over-claim → ложная безопасность. Mitigation: фикстуры на «aliased-mut через указатель/heap НЕ ловится — by design».
- **`-> @` биндинг alias-vs-copy (R7).** Главная поправка дизайна: НЕ blanket-copy. Amend D132 (alias-гарантия load-bearing для consume-чекера). Риск: копия heap-builder'а раздвоит хэндлы → потеря use-after-consume.
- **value-record `-> @` mid-chain in-place** (`mut y = x.foo()` same-slot вместо copy при доказанном escape) — **отложено** (`[M-172.5-value-at-inplace-bind]`, перф-only, гейт на Q-value-abi-auto-placement). V1: всегда copy-on-bind для value-record + опц. lint `W_FLUENT_VALUE_RECORD_COPY`.
- **Index-disjoint i≠j** — V1 literal-only; SMT-прувер не вводим (нет в спеке).
