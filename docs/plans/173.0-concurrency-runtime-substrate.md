## 1. Зачем (проверено по коду)

Дизайн supervision-as-effect (173.2) и сбора (173.1) опирается на свойства, которых **shipped-рантайм не даёт**:

| # | Факт в коде | Почему ломает |
|---|---|---|
| 1 | Ошибка детей схлопывается в **один слот** `first_error_atomic` CAS (`fibers.h:1736`, USER-over-CANCEL) | `on_child_fail` не может быть вызван N раз на N падений — 2 из 3 ошибок теряются как CANCEL |
| 2 | `SpawnCtx` и захваты GC-освобождаются при смерти fiber'а (`q->fiber_ctx[i]=NULL`) | `Restart` нечего перезапускать; хендлеру негде взять входы |
| 3 | Нет стека/кадра, чтобы легально запустить Nova-хендлер в момент падения (fiber размотан, worker вернулся в pickup-loop) | `on_child_fail` (Nova-код) физически негде исполнить |
| 4 | Multi-worker supervised drain — известная нерешённая гонка (Plan 83.4.x/83.11); фича тестируется только при `NOVA_MAXPROCS=1` | claims 173.x неверифицируемы на реальном таргете |

## 2. Решения

1. **Per-slot retention ошибок:** заменить одно-слотовый `first_error_atomic` на **`child_error[idx]`-массив**
   (расширить существующий `fiber_error[]`, `fibers.h:198/1688`) — каждое падение хранится по индексу ребёнка.
2. **Сериализованный failure-decision loop на main-flow:** место, где по одному обрабатываются retained-падения
   **до** уничтожения `SpawnCtx` упавшего; с push'ем fail-frame на call-site (чтобы решение могло само throw'ить).
   Это hook, на котором 173.2 запускает `on_child_fail` (здесь — только инфраструктура, без самого эффекта).
3. **Пиннинг `SpawnCtx` на время scope**, когда присутствует супервизор/рестарт (расширить `ctx_pins`) — чтобы
   входы для рестарта/решения были живы.
4. **Закрыть multi-worker supervised-drain гонку** (координация с Plan 83.4.x/83.11): supervised корректен под
   реальным M:N, без костыля `NOVA_MAXPROCS=1`.

## 3. Фазы

- **Ф.1 — drain-race:** закрыть multi-worker supervised drain (Plan 83.11 `[M-83.11-grow-vs-wake-race]` и смежные);
  снять зависимость тестов от `NOVA_MAXPROCS=1` для supervised. Гард-фикстуры под реальным M:N.
- **Ф.2 — per-slot `child_error[]`:** retention падений по индексу; API чтения «список (idx, err) этого scope».
- **Ф.3 — serialized decision-loop + ctx-pinning:** main-flow цикл обработки retained-падений до освобождения ctx;
  pinning ctx на scope. Hook-точка для 173.2 (пока вызывает default-политику = Escalate-all-or-throw = сегодня).

## 5. Тесты (pos + neg)

- **pos** `nova_tests/err173_0/`: N детей падают **одновременно** разными ошибками → все N retained (не схлопнуты);
  supervised под `NOVA_MAXPROCS>1` стабилен (стресс); decision-loop вызывается по разу на падение; ctx жив до решения.
- **neg/гард:** регресс-гард на закрытую drain-гонку (воспроизведение исходного зависания/AV под M:N).

## 6. Критерии приёмки

1. supervised корректен под реальным M:N (нет зависаний/AV; `NOVA_MAXPROCS=1` больше не требуется).
2. Падения детей retained per-slot (не схлопываются в одно).
3. Существует main-flow hook исполнить решение на падение **до** освобождения `SpawnCtx`.
4. Полный регресс зелёный; верификация против чистого бинаря (§7). **Без упрощений.**
