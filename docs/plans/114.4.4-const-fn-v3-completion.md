## Зачем

После Plan 114.4.3 V2 const fn покрывает main-path comptime — но 9
followups остались для full-feature parity:

**Trivial / UX (Ф.1-Ф.2):**
- Configurable evaluation depth (для deep recursion).
- Friendly error messages для runtime-let / HOF cases.

**V2.1 (Ф.3-Ф.4):**
- Loops в body (для accumulator patterns).
- Record/sum patterns в match (для structured comptime data).

**V3 (Ф.5-Ф.8):**
- Type reflection (sizeof[T]/align_of[T]).
- Real first-class через runtime trampoline (HOF passes).
- Closure-returning const fn (builder/curry patterns).
- True per-const-arg monomorphization (Rust const generics-style).

## Дизайн

### Ф.1 — `#fn_eval_max_depth(N)` attribute (`[M-114.4.3-configurable-depth]`)

Per-fn override evaluator depth limit. Default 256, configurable до N.

Parser:
- Расширить doc_attrs collection в `parse_fn` на `#fn_eval_max_depth(N)`.
- Store as `Option<u32>` в FnDecl.

Evaluator:
- При eval_call для fn с custom depth — use override вместо MAX_EVAL_DEPTH.

### Ф.2 — Friendly errors (`[M-114.4.3-runtime-let-enforcement]` + `[M-114.4.3-friendly-hof-error]`)

**Runtime-let enforcement:**
- Walker `validate_const_fn_uses`: walk Stmt::Let, если value resolves к
  const fn name → `E_CONST_FN_FIRST_CLASS` "use `const ALIAS = const_fn`
  for compile-time alias или `|x| const_fn(x)` lambda для runtime HOF".

**HOF friendly error:**
- Walker detects: Call arg где argument expr = Ident matching const fn.
  Error message `E_CONST_FN_FIRST_CLASS_RUNTIME_HOF` "const fn `X` не
  имеет runtime symbol — wrap в lambda: `|args| {X}(args)`".

### Ф.3 — Loops (`[M-114.4.3-loops]`)

Allow `for`/`while`/`loop` в const fn body:

Body checker:
- Убрать reject. Allow с recursive sub-validation cond/iter/body.

Evaluator:
- `for x in iter { body }`: evaluate iter → range/array; iterate, bind
  x в env, eval body. Watch для loop-break.
- `while cond { body }`: eval cond → bool; loop with depth-budget.
- `loop { body }`: pure inf loop — require break/return.
- Termination check: iteration counter с MAX_LOOP_ITERATIONS=10_000
  default; configurable через `#fn_eval_max_iterations(N)`.
- `break`/`continue` — handled через control flow.

### Ф.4 — Record/sum patterns (`[M-114.4.3-pattern-record-sum]`)

Extend match pattern V2.0 subset → V2.1:
- `Variant(p1, p2)` — sum-type destructuring.
- `Variant { field: pat }` — record-with-fields.
- `(a, b)` — tuple destructuring (если const tuples появятся).

Requires ConstValue extension — add `Record(HashMap<String, ConstValue>)`
+ `Variant(String, Vec<ConstValue>)` variants.

Reality: most cases не нужно — V2.0 literal/wildcard покрывает 90%.
V2.1 — для structured const values (когда Plan 120 named tuples
интегрируется).

### Ф.5 — Type reflection (`[M-114.4.3-t-reflection]`)

Add intrinsics:
- `sizeof[T]()` → const int — byte size of T per backend ABI.
- `align_of[T]()` → const int — alignment of T.
- `T.const_FIELD` — read assoc const of T (Plan 114.4.1 D200).

Compiler integration:
- Type system must provide TypeInfo trait — size + align lookup.
- For primitive types — hardcoded sizes (int=8, char=4, bool=1, etc.).
- For records — sum of fields + padding.
- For generic types — per-monomorphization key.

Evaluator handles `Call(Ident("sizeof"), TurboFish<T>)` → ConstValue::Int.

### Ф.6 — Runtime HOF (`[M-114.4.3-runtime-hof]`)

Generate runtime trampoline для const fn используемой в runtime context.

Detection: walker finds Call site где const fn passed as argument к
runtime fn. Trigger trampoline emission.

Codegen extension:
- Generate `_const_fn_X_trampoline(runtime_args...)` wrapper.
- Trampoline body calls const fn with const args extracted from closure
  capture (если есть).
- For pure const fn (no const params): just generate runtime fn wrapper.

ABI: trampoline = regular function pointer compatible с runtime HOF.

### Ф.7 — Closure-returning const fn (`[M-114.4.3-closure-from-const-fn]`)

Allow `const fn make_adder(const n int) -> const (int -> int) =>
|x| x + n`.

Body checker:
- Accept closure literal в const fn body.
- Closure captures must be const-eligible.

Codegen:
- Generate specialized closure per (fn, const_args) combination.
- Closure становится regular runtime closure с baked-in const captures.

### Ф.8 — Mono specialization (`[M-114.4.3-mono-specialization]`)

True per-const-arg monomorphization для mixed const fns.

Replace V2.0 "mixed fn stays runtime, const params informational" с:
- Per call site с different const arg → отдельная C fn `Nova_fn_X_cst_HASH`.
- Const params в body get substituted с literal values pre-codegen.
- Naming: hash of (const_arg_values) → unique mangle.

Integration с Plan 70.5 mono pipeline:
- Reuse mono cache для (fn_name, const_args, generic_T) tuples.
- Dead specs dropped per existing pipeline.
