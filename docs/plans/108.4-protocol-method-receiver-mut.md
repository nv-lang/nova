## Зачем

После Plan 108.1/108.2/108.3 (default-ro для params/locals/loops/patterns),
**protocol method declarations остались единственной позицией с implicit
default-mut**. Это нарушает consistency contract:

**Текущее состояние** ([std/prelude/collections.nv:107-114](../../std/prelude/collections.nv#L107-L114)):
```nova
export type Iterable[T] protocol {
    next() -> Option[T]               // ← НЕТ mut, parser не поддерживает
}

// Implementers вводят mut в free-form, type-checker не enforce'ит match:
fn Counter mut @next() -> Option[int] { @cur += 1; ... }       // mut receiver
fn RangeIter mut @next() -> Option[int] { ... }                // mut receiver
fn HashMapIter[K,V] mut @next() -> Option[(K, V)] { ... }      // mut receiver
```

Spec comment в `collections.nv:107`: «**bootstrap parser НЕ поддерживает
`mut`-модификатор перед именем метода в protocol-declaration**. Receiver
mutability подразумевается use-case'ом и объявляется на стороне implementer'а».

**3 проблемы текущего «honour system»:**

1. **Type-checker не enforce'ит mut consistency между protocol и impl.**
   Protocol объявляет `next() -> Option[T]` (без mut → должен быть ro
   receiver post-108.x). Implementer пишет `fn Counter mut @next()` (mut
   receiver). **Mismatch silently accepted** — нарушает contract.

2. **`#impl(Iterable)` annotation (Plan 91.9 D186) impotent.** Если user
   пишет `#impl(Iterable) type Counter { ... }` + `fn Counter @next() ->
   Option[int]` (ro receiver), then iterator не работает (нельзя advance
   cursor) — но type-checker accepts. Bug surfaces only at runtime via
   silent infinite loop / non-progress.

3. **Inconsistency с остальными default-ro positions.** После Plan 108.x:
   - param без mut → ro (`E_PARAM_NOT_MUT`)
   - local без mut → ro (`E_LOCAL_NOT_MUT`)
   - loop-var без mut → ro
   - pattern binding без mut → ro
   - **protocol method без mut → ???** ← undefined, parser не принимает modifier

   Pre-108.x: «default-mut everywhere» был consistent. Post-108.x: «default-ro
   everywhere except protocol methods» — это **broken consistency**.

**Plan 108.4 closes the gap:**
- Parser: разрешить `mut`/`ro`/`consume` prefix в protocol method declarations.
- Default = `ro` (consistent с Plan 108.x family).
- Type-checker: enforce impl receiver matches protocol declaration.
- Stdlib migration: добавить explicit `mut` к `Iterable.next`, etc — fix всё
  protocol declarations.

## Дизайн

### Grammar — protocol method receiver prefix

```nova
type Iterable[T] protocol {
    mut next() -> Option[T]                       // mut receiver (default-mut method)
    len() -> int                                   // ro receiver (default)
    is_empty() -> bool                             // ro
}

type Closeable protocol {
    consume close() -> Result[(), Error]          // consume receiver
}

type Comparable[T] protocol {
    compare(other ro T) -> int                    // ro receiver, ro param
}

type Writer protocol {
    mut write(data ro []u8) -> Result[int, IoError]
    mut flush() -> Result[(), IoError]
    consume close() -> Result[(), IoError]        // consume — закрывает permanently
}

type Hashable protocol {
    hash() -> u64                                  // ro — детерминированный hash
}
```

**Правила:**

1. **Default = `ro`** (consistent с param/local/loop-var default per Plan 108.x).
2. **Explicit prefix** `mut`/`ro`/`consume` перед method name определяет
   receiver mutability **для всех implementers**.
3. **`ro` redundant** (default), но allowed для documentation clarity:
   `ro len() -> int` ≡ `len() -> int`.
4. **`mut`/`consume` ≠ `ro`** — implementer должен match.
5. **Modifier-conflicts** (parse error):
   - `mut ro foo()` — `E_PROTO_METHOD_MOD_CONFLICT`
   - `mut consume foo()` — `E_PROTO_METHOD_MOD_CONFLICT`
   - `consume ro foo()` — `E_PROTO_METHOD_MOD_CONFLICT`

### Implementer enforcement

```nova
type Counter { mut cur int, end int }

// ✓ MATCH: protocol mut → impl mut
fn Counter mut @next() -> Option[int] {
    if @cur >= @end { None } else { ro v = @cur; @cur += 1; Some(v) }
}

// ✗ MISMATCH: protocol mut → impl ro
fn Counter @next() -> Option[int] { ... }
// → E_PROTO_IMPL_RO_FOR_MUT:
//   "Protocol Iterable.next() declares mut receiver; implementation must
//    also use mut receiver. Change to `fn Counter mut @next()`."

// ✓ MATCH: protocol ro → impl ro
fn Counter @len() -> int => @end - @cur

// ✗ MISMATCH: protocol ro → impl mut
fn Counter mut @len() -> int { ... }
// → E_PROTO_IMPL_MUT_FOR_RO:
//   "Protocol Iterable.len() declares ro receiver; implementation cannot
//    use mut. Change to `fn Counter @len()` or remove mutation from body."

// ✓ MATCH: protocol consume → impl consume
fn FileHandle consume @close() -> Result[(), IoError] { ... }

// ✗ MISMATCH: protocol consume → impl mut
fn FileHandle mut @close() -> Result[(), IoError] { ... }
// → E_PROTO_IMPL_MUT_FOR_CONSUME
```

### Default-ro «structural conformance»

После Plan 108.4 protocol-method **без prefix** = ro receiver. Implementer
**также без prefix** = ro receiver. **Match implicit'ом — OK**:

```nova
type Hashable protocol {
    hash() -> u64                                  // ro implicit
}

type MyId { id u64 }

fn MyId @hash() -> u64 => @id                     // ro implicit — matches
```

Для backwards-compat (pre-108.x когда default был mut implicit'ом) — Plan
108.4 включает stdlib migration sweep (Ф.3) — все existing protocol decls
получают explicit `mut`/`ro` где applicable.

### Cross-modifier с `#impl(P)` (Plan 91.9 D186)

`#impl(P)` annotation forces explicit protocol-conformance check. Plan 108.4
делает этот check **stricter** — теперь enforce'ит и receiver mutability:

```nova
#impl(Iterable[int])
type Counter { mut cur int, end int }

fn Counter @next() -> Option[int] { ... }         // ← E_PROTO_IMPL_RO_FOR_MUT
                                                   // (Iterable.next is mut)
```

Без `#impl(P)` — structural duck-typing (D58) — Plan 108.4 enforce'ит при
**use-site** (for-in loop, generic bound `[T Iterable[U]]`).

### Сравнение с mainstream

| Язык | Receiver mutability в protocol/trait/interface |
|---|---|
| Rust | `trait Iterator { fn next(&mut self) -> Option<Self::Item> }` — explicit `&mut self` |
| Go | `interface { Next() *T }` — implicit pointer mut (no shared ref); receiver type decides |
| Swift | `protocol IteratorProtocol { mutating func next() -> Element? }` — `mutating` keyword |
| Kotlin | `interface Iterator<T> { fun next(): T }` — Kotlin permits any side effect (no mut tracking) |
| Java | `interface Iterator<T> { T next(); }` — same as Kotlin, no static mutability |
| Haskell | type classes — pure (immutable); state-mut via monads |
| **Nova V2 (Plan 108.4)** | `protocol { mut next() -> Option[T] }` — explicit `mut`/`ro`/`consume`, **enforced** | **best-in-class** (Swift-tier + consume) |

Nova matches Swift `mutating` (explicit + enforced) и расширяет на `consume`
(linear-types) — unique combination.

## Грамматика

Текущая (фрагмент `compiler-codegen/src/parser/mod.rs:2244+` `parse_effect_methods`):

```ebnf
proto_method_decl ::= IDENT "(" param_list? ")" effect_list? ("->" type)?
```

Новая (после Plan 108.4):

```ebnf
proto_method_decl ::= ("mut" | "ro" | "consume")? IDENT "(" param_list? ")"
                      effect_list? ("->" type)?
```

**Tokenizer:** без изменений (`KW_MUT`/`KW_RO`/`KW_CONSUME` уже keyword tokens).

**Parser:** `parse_effect_methods` (в `compiler-codegen/src/parser/mod.rs`) —
добавить optional consume/mut/ro lookahead перед `IDENT`.

**AST:** `ProtoMethodDecl` struct получает field `receiver_mut:
ReceiverMutability` (enum `Ro | Mut | Consume`); default = `Ro`.

**Type-checker:** при resolving protocol implementation — compare
`receiver_mut` декларации vs `receiver_mut` каждого impl-метода;
mismatch → один из:
- `E_PROTO_IMPL_RO_FOR_MUT` — protocol mut, impl ro
- `E_PROTO_IMPL_MUT_FOR_RO` — protocol ro, impl mut
- `E_PROTO_IMPL_MUT_FOR_CONSUME` — protocol consume, impl mut/ro
- `E_PROTO_IMPL_CONSUME_FOR_MUT` — protocol mut/ro, impl consume

Plus parse-time:
- `E_PROTO_METHOD_MOD_CONFLICT` — multiple modifiers (`mut ro foo()`).

## D-block changes

### D209 (NEW) — Protocol method receiver mutability

**Локация:** `spec/decisions/04-effects.md` (рядом с protocol-related blocks).

**Что.** Protocol method declarations carry explicit receiver mutability
prefix (`mut`/`ro`/`consume`). Default = `ro` (consistent с Plan 108.x
family). Implementers обязаны match.

**Grammar:**
```ebnf
proto_method_decl ::= ("mut" | "ro" | "consume")? IDENT "(" param_list? ")"
                      effect_list? ("->" type)?
```

**Default:** `ro` (без prefix'а).

**Enforcement:**
- `#impl(P)` annotation (D186) — declares-conformance: type-checker matches
  every method's receiver_mut.
- Structural conformance (D58) — at use-site (for-in / generic bound) —
  same check.

**Errors:**
- `E_PROTO_METHOD_MOD_CONFLICT` — parse-time (multiple modifiers).
- `E_PROTO_IMPL_RO_FOR_MUT` — protocol mut, impl ro.
- `E_PROTO_IMPL_MUT_FOR_RO` — protocol ro, impl mut.
- `E_PROTO_IMPL_MUT_FOR_CONSUME` / `E_PROTO_IMPL_CONSUME_FOR_MUT` —
  consume mismatches.

**Migration:** stdlib protocols (Iterable, Comparable, Hashable, Closeable,
etc) — Ф.3 sweep добавляет explicit prefixes. Existing implementers
match-able since pre-108.4 «honour system» convention уже использовал
correct mut на side implementer'а (per `collections.nv:107-114` note).

**Cross-ref:**
- D58 amend (Iterable signature stay `mut next()`).
- D72 amend (`T Iterable[X]` bound — mut consistency).
- D186 amend (`#impl(P)` annotation checks receiver_mut).
- Plan 108.1/108.2/108.3 — consistency story (default-ro everywhere).

### D58 amend — Iterable structural protocol signature

**Локация:** `spec/decisions/03-syntax.md`.

Old: «`Iterable[T]` = duck-typing на `next() -> Option[T]`».
New: «`Iterable[T]` = duck-typing на `mut next() -> Option[T]`». Use-site
checker enforce'ит mut consistency.

### D72 amend — `T Iter` bound usage

**Локация:** `spec/decisions/03-syntax.md` (где определён D72).

Add note: «когда type участвует как bound `[T Iterable[U]]`, type-checker
проверяет receiver_mut consistency для всех protocol methods. Mismatch →
`E_PROTO_IMPL_*` errors».

### D186 amend — `#impl(P)` annotation enforcement

**Локация:** `spec/decisions/02-types.md` (где определён D186).

Add note: «`#impl(P)` annotation теперь checks receiver_mut в дополнение
к method signature. Несовпадение receiver mut между protocol и impl →
`E_PROTO_IMPL_*` errors».
