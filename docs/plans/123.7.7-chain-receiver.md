## 1. Контекст

V7.5 IPA refinement обрабатывает `@F.method()` (direct field-receiver
call). Chain receivers like `@a.b.method()` / `@a.b.c.method()`
falled through к conservative-invalidate.

By the same self-type reasoning as V7.5: callees invoked through
self-rooted chain cannot reach SIBLING fields of self. So sibling
caches survive these calls.

V7.7 extends V7.5 К detect chain receivers of arbitrary depth.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**New helper `call_recv_self_chain` (~28 LOC):**

```rust
fn call_recv_self_chain(obj: &Expr) -> Option<Vec<String>> {
    let mut segments = Vec::new();
    let mut cur = obj;
    loop {
        match &cur.kind {
            ExprKind::Member { obj: inner, name } => {
                segments.push(name.clone());
                cur = inner;
            }
            ExprKind::SelfAccess => break,
            _ => return None,
        }
    }
    if segments.is_empty() { return None; }
    segments.reverse();
    Some(segments)
}
```

**Refined dispatch в `expr_contains_invalidating_call_for` (~12 LOC):**
