## 1. Контекст и root cause

Per `[M-82-linux-arena-dealloc]` (simplifications.md, enriched
2026-05-26 Plan 82.1):

**Проблема:** `nova_fiber_dealloc` в POSIX-пути (`fiber_arena.c`)
использует thread-local `_t_arena` для поиска арены-владельца. Под
M:N runtime (armed, work-stealing):

- `nova_runtime_spawn_global` создаёт `mco_coro` на потоке-спаунере
  (main или worker A) → `nova_fiber_alloc` берёт слот из A's TLS arena.
- Fiber push'ится в deque worker'а B (round-robin) или steal'ится
  worker'ом C.
- При завершении fiber'а: `mco_destroy` на B/C → `nova_fiber_dealloc`
  проверяет `_t_arena` B/C — ptr принадлежит арене A → **warning +
  slot leak**.

**Почему Windows не затронут:**
`fiber_arena_win.c` имеет глобальный список арен `_nova_fw_arena_list`.
`nova_fiber_dealloc` на Windows ищет арену по адресному диапазону,
не по TLS. Комментарий `fiber_arena_win.c:27-31`:
> `cross-thread dealloc (work-stealing: fiber мигрировал A→B) находит
> арену-владельца ПО АДРЕСУ, не по TLS`

**Этот план:** портировать Windows-механизм на POSIX.

## 3. Design

### 3.1 Global arena list

```c
/* Append-only intrusive linked list of all POSIX arenas.
 * Added in nova_fiber_arena_init per thread, never removed.
 * Next pointer stored inline в NovaFiberArena struct (extend it).
 * List head: _nova_posix_arena_list_head (global, mutex-guarded
 * for add-only; reads are lock-free via atomic load). */

struct NovaFiberArena {
    char*    base;
    size_t   virtual_size;
    size_t   slot_size;
    size_t   slot_count;
    size_t   slots_active;  /* atomic — cross-thread dealloc mutates */
    size_t   high_water;
    uint64_t free_bits[NOVA_FIBER_BITMAP_WORDS];  /* atomic ops для cross-thread */
    /* Plan 82.2: intrusive list node для global arena registry. */
    struct NovaFiberArena* next_arena;  /* NULL = list tail */
};

static NovaFiberArena* _nova_arena_list_head = NULL;
static pthread_mutex_t _nova_arena_list_mu   = PTHREAD_MUTEX_INITIALIZER;
```

### 3.2 Registration at init

В `nova_fiber_arena_init()`, после установки `_t_arena.base`:

```c
/* Register в global list для cross-thread dealloc dispatch. */
pthread_mutex_lock(&_nova_arena_list_mu);
_t_arena.next_arena = _nova_arena_list_head;
_nova_arena_list_head = &_t_arena;
pthread_mutex_unlock(&_nova_arena_list_mu);
```

List modification — редкое событие (per worker init), mutex overhead
не критичен.

### 3.3 Address-based lookup

```c
/* Find arena that contains ptr. O(N_arenas) — N_arenas = N_workers + 1 (main).
 * Lock-free read: list is append-only, _nova_arena_list_head set once
 * per arena, traversal safe without lock. */
static NovaFiberArena* _nova_find_arena_for(const char* p) {
    NovaFiberArena* a = _nova_arena_list_head;
    while (a) {
        if (a->base && p >= a->base + NOVA_FIBER_GUARD_SIZE
                     && p <  a->base + a->virtual_size) {
            return a;
        }
        a = a->next_arena;
    }
    return NULL;
}
```

### 3.4 Updated `nova_fiber_dealloc`

```c
void nova_fiber_dealloc(void* ptr, size_t size, void* allocator_data) {
    (void)size; (void)allocator_data;
    if (!ptr) return;

    char* p = (char*)ptr;

    /* Fast path: ptr in current thread's arena (normal case). */
    NovaFiberArena* a = &_t_arena;
    if (!a->base ||
        p < a->base + NOVA_FIBER_GUARD_SIZE ||
        p >= a->base + a->virtual_size) {
        /* Slow path: cross-thread dealloc — find owner by address. */
        a = _nova_find_arena_for(p);
        if (!a) {
            fprintf(stderr, "nova: fiber_dealloc ptr outside all arenas (%p)\n", ptr);
            return;
        }
    }

    /* Reverse ptr → slot index using owning arena's layout. */
    size_t offset = (size_t)(p - a->base - NOVA_FIBER_GUARD_SIZE);
    size_t slot   = offset / a->slot_size;
    if (slot >= a->slot_count) {
        fprintf(stderr, "nova: fiber_dealloc slot index out of range\n");
        return;
    }

    /* Clear slot bit. Cross-thread: use atomic AND. Same-thread: plain OK.
     * _arena_mark_slot_free_atomic handles both paths. */
    _arena_mark_slot_free_atomic(a, slot);
    __atomic_fetch_sub(&a->slots_active, 1, __ATOMIC_RELAXED);

    /* MADV_DONTNEED batch: only on owning thread (own arena idle check).
     * Cross-thread dealloc skips this — owning thread handles on next alloc
     * or idle path. */
    if (a == &_t_arena && a->slots_active == 0 && a->high_water > 0) {
#ifdef MADV_DONTNEED
        char* range_base = a->base + NOVA_FIBER_GUARD_SIZE;
        size_t range_size = a->high_water * a->slot_size - NOVA_FIBER_GUARD_SIZE;
        madvise(range_base, range_size, MADV_DONTNEED);
#endif
    }
}
```

### 3.5 Atomic bitmap operations

`slots_active` и `free_bits` — нужны атомарные операции при cross-thread
clear. На gcc/clang:

```c
static void _arena_mark_slot_free_atomic(NovaFiberArena* a, size_t slot) {
    size_t w = slot / 64;
    size_t b = slot % 64;
    uint64_t mask = ~(1ULL << b);
    __atomic_fetch_and(&a->free_bits[w], mask, __ATOMIC_RELEASE);
}
```

`slots_active` — `__atomic_fetch_sub` (RELAXED достаточно для counter;
correctness не критична для cross-thread — используется только для
MADV_DONTNEED trigger на own thread).

Для аллокации (only on own thread): оставить `_arena_mark_slot_used`
как plain operation (или `__atomic_fetch_or` для safety).

### 3.6 Cleanup consideration

При `_arena_thread_exit_cleanup` (pthread_key callback): arena может быть
в global list. Список — append-only, никогда не удаляем ноды (арены
переживают поток для cross-thread dealloc). Но `munmap` в cleanup
освобождает физическую память — после этого cross-thread dealloc на
этот arena опасен.

**Solution:** в `_arena_thread_exit_cleanup` — clear `a->base = NULL`
ПОСЛЕ `munmap`. `_nova_find_arena_for` проверяет `a->base != NULL` →
skip dead arenas.

Ordering: `munmap` → `a->base = NULL` (release store), `_nova_find_arena_for`
loads `a->base` (acquire). Safe under acquire/release ordering.

```c
static void _arena_thread_exit_cleanup(void* arg) {
    NovaFiberArena* a = (NovaFiberArena*)arg;
    if (!a || !a->base) return;
    /* ... GC_remove_roots ... */
    munmap(a->base, a->virtual_size);
    /* Plan 82.2: atomic NULL после munmap — cross-thread dealloc
     * видит dead arena через _nova_find_arena_for → skip. */
    __atomic_store_n(&a->base, NULL, __ATOMIC_RELEASE);
    /* Don't memset — a->next_arena must remain valid for list traversal. */
    a->virtual_size   = 0;
    a->slots_active   = 0;
    a->high_water     = 0;
    memset(a->free_bits, 0, sizeof(a->free_bits));
}
```

**NB:** не делать `memset(a, 0)` — уничтожит `next_arena` pointer в list.

## 5. Risk

| Risk | Mitigation |
|---|---|
| `_nova_find_arena_for` traversal under ABA / partial init | Arena added to list only AFTER fully initialized (`base` set) |
| `munmap` race: cross-thread dealloc after cleanup | `base = NULL` after munmap; `_nova_find_arena_for` skips NULL-base arenas |
| `free_bits` atomic vs plain ABI mismatch | Use `__atomic_fetch_and/or` consistently; C11 `_Atomic` alternative for stricter guarantee |
| SIGSEGV in `_arena_thread_exit_cleanup` after `memset` | Change to targeted field zero'ing (保留 `next_arena`) |
| macOS: `MADV_DONTNEED` semantics differ (returns pages to OS but may re-demand) | Behavior unchanged; MADV_DONTNEED only triggered from own thread (unchanged) |
