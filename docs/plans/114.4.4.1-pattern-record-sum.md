# Plan 114.4.4.1 — Record/sum patterns в const fn match (V2.1 extension)

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.4 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — niche feature. Most V2.0 use-cases covered by
>   literal + wildcard + ident patterns.
> **Оценка:** ~½-1 dev-day.
> **Зависимости:** Plan 114.4.3 V2 (match pattern V2.0 baseline).
>
> **Why extracted:** ConstValue currently supports только primitive
>   variants (Int/Float/Bool/Str/Char/Unit). Adding `Record(HashMap)` +
>   `Variant(name, args)` для destructuring требует evaluator extension.
>   Most const fn use-cases работают с primitives — record/sum
>   destructuring polish feature, не blocking V3 ship.

## Scope

Extend match pattern V2.0 → V2.1:
- `Variant(p1, p2)` — sum-type destructuring.
- `Variant { field: pat }` — record-with-fields.
- `(a, b)` — tuple destructuring.

Requires:
- ConstValue::Record(HashMap<String, ConstValue>).
- ConstValue::Variant(String, Vec<ConstValue>).
- ConstValue::Tuple(Vec<ConstValue>).
- match_const_pattern extension.
- Body checker: pattern_record/variant/tuple accept.

## Tests

- Positive: `match Point(x, y) { Point(a, b) => a + b }`.
- Negative: malformed pattern.

## Acceptance

A1: Variant/Tuple/Record patterns parse + evaluate.

См. Plan 114.4.4 closure для контекста.
