# Plan 140.1 — Contract & assert diagnostics (short location-first format + custom message)

> **Создан:** 2026-06-12.  **Статус:** 📋 PLANNED (gated на Plan 140).
> **Эстимат:** ~1.5–2 dev-day.  **Model:** Opus + Thinking ON.
> **Зависит от:** Plan 140 (contracts enforced in release — формат/сообщение проходят через
> `nova_contract_violation`, который после 140 срабатывает и в release).
> **Закрывает:** `[M-140-contract-message]` (backlog) + verbose-формат + отсутствие локации у assert.

---

## Проблема

1. **Многословный префикс контрактов** (`contracts.h`):
   `contract requires failed in `withdraw`: amount > 0 at bank.nv:42` — длинно («contract» + «in `fn`» избыточны).
2. **`assert`/`debug_assert` без локации** (`effects.h:469`): `assertion failed: a > 0` — нет `file:line`, не найти где упало; и «assertion» длинно.
3. **Нет кастомного сообщения на контрактах.** `assert(cond, "msg")` (D84) — есть; `requires`/`ensures`/`invariant` — нет.

---

## Решение

### A. Единый короткий формат (location-first) — для контрактов И assert
```
<file>:<line>: <kind> failed: <expr>
```
где `<kind>` ∈ `requires` / `ensures` / `invariant` / `assert`. Примеры:
```
bank.nv:42: requires failed: amount > 0
bank.nv:50: ensures failed: result >= 0
vec.nv:12: invariant failed: size <= cap
foo.nv:10: assert failed: a > 0
```
- Убраны «contract» и «in `fn`» (kind-слово подразумевает контракт; `file:line` локализует).
- `assertion` → `assert`; добавлен `file:line`.
- Location-first → терминал/IDE делают `file:line` кликабельным (jump-to-line).
- `debug_assert` → тот же `assert failed` (различие — только debug-only-гейт, не текст).

### B. Опциональное пользовательское сообщение на контрактах
```nova
requires amount > 0, "amount must be positive"
```
→ сообщение показывается + выражение в скобках:
```
bank.nv:42: requires failed: amount must be positive (amount > 0)
```
Без сообщения — формат A.

**`assert`/`debug_assert` с сообщением — тот же принцип** (`assert(a > 0, "must be positive")`):
```
foo.nv:10: assert failed: must be positive (a > 0)
```

**LOC (`<file>:<line>:`) ВСЕГДА авто-добавляется в начало** — и контракты, и assert, с сообщением и без.
Пользователь НЕ включает локацию в своё сообщение (как и `nova_str` exprs) — она проставляется
автоматически из `__FILE__`/`__LINE__` на codegen-сайте. Унифицирует/расширяет `assert(cond, "msg")` (D84).
