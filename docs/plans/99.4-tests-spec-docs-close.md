## Что делаем

### Ф.1 — Comprehensive positive tests (~0.2 д)

`nova_tests/plan99/`:

**Option.map** — `option_map_typed.nv`:
- int→int (identity), int→str (T≠U primitive→primitive).
- str→char, char→int, int→bool.
- User-record→int (non-primitive → primitive).
- int→User-record (primitive → non-primitive).
- Chain: `o.map(|x| x*2).map(|y| str.from(y))` — double mono.
- В for-in над `[]Option[int]` (хотя Plan 98 known-limit для
  `[]Option[T]` inference — может требовать turbofish).

**Option.unwrap_or_else** — `option_unwrap_or_else_lazy.nv`:
- Lazy default invocation: closure with side-effect counter, verify
  не вызывается на Some.
- Default invoked on None.
- Capture: closure captures outer variable.

**Option.ok_or** — `option_ok_or_to_result.nv`:
- Some(int) + err(str) → Ok-Result.
- None + err(str) → Err-Result.
- Some(int) + err(int) — разные E.
- Chain: `o.ok_or(err).map(|x| x*2)` — Option→Result→Result mono pipeline.

**Result.map** — `result_map_ok_transform.nv`:
- Ok(int).map(|x| str.from(x)) → Ok(str). Err passes through.
- Composition `r.map(...).map_err(...)` — both transforms compose.

**Result.map_err** — `result_map_err_transform.nv`:
- Err(str).map_err(|e| str.len(e)) → Err(int). Ok passes through.

**Result.unwrap_or_else** — `result_unwrap_or_else_recovery.nv`:
- Ok(v) → v (closure not called).
- Err(e) → f(e) recovery.
