# Plan 85.2 — Полное тестовое покрытие `ReadBuffer` и `WriteBuffer`

> **Статус:** ✅ ЗАКРЫТ 2026-05-22 (Ф.1–Ф.5)
> **Приоритет:** P1 | **Оценка:** ~1.5 дня
> **Родительский план:** [Plan 85](85-builtin-types-test-coverage.md)
> **Зависимости:** нет (standalone)
>
> **Итог:** каталог `nova_tests/buffers/` — 11 файлов (10 позитивных +
> 1 негативный). Все 44 метода ReadBuffer и 31 метод WriteBuffer
> покрыты, включая round-trip всех числовых типов (LE+BE), char/str,
> bytes. Codegen-багов не найдено.
>
> **Уточнения по факту API:**
> - WriteBuffer `@into()` — НЕ `consume` (в отличие от StringBuilder);
>   декларация `std/runtime/write_buffer.nv` — просто `@into() -> []u8`.
> - Fail-form OOB (`read_byte()` за границей) внутри `test`-блока
>   перехватывается test-harness'ом → тест FAIL, не процесс-panic.
>   Escape до runtime-panic требует `fn main` (см. `neg_read_oob.nv`).
> - `try_read_f64_*` Ok-payload в bootstrap несёт int-биты; распаковка
>   через `f64.from_bits(r.unwrap_or(0))` (см. `read_floats.nv`).

## Ключевые тесты — round-trip

```nova
test "i64 BE round-trip" {
    let wb = WriteBuffer.new()
    wb.write_i64_be(-1_000_000)
    let rb = ReadBuffer.from(wb.into())
    assert(rb.read_i64_be() == -1_000_000)
}

test "f64 LE round-trip" {
    let wb = WriteBuffer.new()
    wb.write_f64_le(3.14159)
    let rb = ReadBuffer.from(wb.into())
    assert(rb.read_f64_le() == 3.14159)
}

test "char round-trip" {
    let wb = WriteBuffer.new()
    wb.write_char('Я')
    let rb = ReadBuffer.from(wb.into())
    assert(rb.read_char() == 'Я')
}
```
