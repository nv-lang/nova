## Два изменения

### 1. Атрибуты ПЕРЕД `module` (вариант C)

**Сейчас:** `#cfg`/`#forbid`/`#doc` идут **после** `module`:
```nova
module net.tls
#forbid Net
#cfg(target_os = "windows")
```

**Цель:** атрибуты **перед** `module`:
```nova
#forbid Net
#cfg(target_os = "windows")
module net.tls
```

**Почему:**
- **Консистентность с item-level.** `#cfg`/`#realtime`/`#pure` идут
  перед `fn`. Одно правило «атрибут перед аннотируемым» вместо двух.
- **`#cfg` семантически — гейт.** «Существует ли файл вообще» решается
  ДО чтения `module`. Читать гейт после объявления — как `if` после тела.
- **AI-first signal ordering.** LLM/человек видит условия файла сразу
  при открытии, до `module`/`import`/items.
- **Rust-familiar.** `#[cfg]` / `#[derive]` перед `mod`/`fn`.

**Парсинг:** `module`-декларация — обязательный разделитель. Всё до неё
= file-атрибуты, всё после (перед item) = item-атрибуты. Lookahead не
нужен.

### 2. `#cfg` операторы `|| && !` вместо `any/all/not`

**Сейчас (Plan 42.14 Ф.1):** функц-форма `any(...)`, `all(...)`, `not(...)`.

**Цель:** Go/C-стиль булевы операторы:
```nova
#cfg(target_os = "linux" || target_os = "macos")
#cfg(feature = "tls" && !target_os = "windows")
#cfg((feature = "a" || feature = "b") && target_os = "linux")
```

**Грамматика** (precedence: `!` > `&&` > `||`, как C/Rust/Go):
```
cfg_expr := cfg_or
cfg_or   := cfg_and ('||' cfg_and)*
cfg_and  := cfg_not ('&&' cfg_not)*
cfg_not  := '!' cfg_not | cfg_atom
cfg_atom := '(' cfg_expr ')' | key '=' string
```

**AST не меняется.** `CfgPredicate::{Any,All,Not}` — internal имена.
Парсер схлопывает `a || b || c` → `Any([a,b,c])`, `a && b` → `All`,
`!a` → `Not`. `eval_cfg_predicate` не трогается.

**Токены:** `||`, `&&`, `!` уже есть в лексере Nova (булевы операторы
языка) — новых токенов не нужно.

## Phases

### Ф.1 — `#cfg` операторы `|| && !`

- `parser/mod.rs::parse_cfg_predicate` — переписать с функц-формы на
  recursive-descent с precedence (`cfg_or`/`cfg_and`/`cfg_not`/`cfg_atom`).
- AST `CfgPredicate` — без изменений.
- `eval_cfg_predicate` — без изменений.

### Ф.2 — module-attrs перед `module`

- `parser/mod.rs::parse_module` — собрать `#cfg`/`#forbid`/`#doc` цикл
  **до** парсинга `module` декларации (сейчас — после).
- Грамматика файла:
  `file := module_attr* module_decl import* (item_attr* item)*`

  выше `module` в существующих файлах.

- `#cfg(any/all/not)` → операторы в nova_tests/cfg/*.

## Что НЕ входит

- `#cfg` в expression position — по-прежнему нет.
- Inner/outer attribute различие (`#` vs `#!`) — Nova минимализм.
- Новые токены — `|| && !` уже в лексере.
