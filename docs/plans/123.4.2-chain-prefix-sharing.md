## 1. Контекст

V4 / V4.1 emit per-chain `_at_<full>_chain = @<full>` лет. Two chains
sharing prefix `@a.b` walk it twice. Telemetry baseline показала что
до 40% chains в practical Nova code share length-2 prefix → V4.2
adds prefix extraction.

---

## 2. Изменения

### 2.1 field_cache.rs `chain_cache_fn_impl`

Добавлен этап compute_chain_prefix_sharing после name generation:
- Group `to_cache` by `path[..2]`.
- For groups с ≥2 chains, alloc collision-safe local name
  `_at_<a>_<b>_pre`, store в prefix_map.
- Budget: prefix lets count toward `cfg.max_per_fn`.

Эмит этап:
- Prefix lets first (sorted by prefix key).
- Per-chain lets — `find_chain_shared_prefix` lookup:
  - Hit → `build_chain_from_ident(prefix_local, tail, span)`.
  - Miss → `build_chain_expr(path, span)` (unchanged V4 behavior).

### 2.2 Tests (field_cache::tests)

2 new unit tests + 1 runtime fixture:
- `v42_shared_prefix_emitted_when_two_chains_share_prefix` —
  expected `_at_a_b_pre + _at_a_b_c_chain + _at_a_b_d_chain` lets.
- `v42_no_shared_prefix_when_single_chain` — no `_pre` let.
- Runtime fixture `nova_tests/plan123_4_2/v42_chain_prefix_sharing_ok.nv`:
  `@inner.middle.y.c + .c + .d + .d` semantic preservation.

## 4. Не в скоупе

- Length-3+ prefix sharing — recursive common-ancestor algorithm,
  V4.3 followup. V4.2 minimal addition: 2-deep only.
- Cross-fn prefix sharing (module-level caching of `@a.b` if
  multiple methods use it) — Plan 123.8 territory.
- Mut field as prefix root — V4 V4.1 already skips when root in
  body_writes; V4.2 inherits that constraint.
