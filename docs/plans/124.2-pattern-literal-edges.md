## 1. Scope

Plan 124.1 закрыл **basic-case** `Pattern::Record` в `Stmt::Let` и
`ExprKind::RecordLit` (named, non-spread). 124.2 расширяет coverage
на:

1. **Match arms** — `match acc { Account { money, ... } => ... }`.
2. **IfLet** — `if let Account { money, .. } = acc { ... }`.
3. **WhileLet** — `while let Some(Account { money }) = next() { ... }`
   (вложенно через variant pattern; на V1 ограничиваем top-level Record).
4. **For-loop** — `for Account { money, .. } in accounts { ... }`.
5. **ParallelFor** — same as For.
6. **Nested Pattern::Record** — `User { addr: Address { street } } = u`.
7. **Record literal `..spread`** — `Account { ...other }` outside type
   leaks priv через spread (new code `E_PRIV_FIELD_INIT_SPREAD`).
8. **Rest pattern `{ priv_f, .. }`** — `..` НЕ produces a binding, и не
   leaks priv. Документируется в D221 §3.

Out of scope (other sub-plans):
- Generic types (124.3)
- Named tuple positional priv access (124.4)
- LSP/doc surface (124.5)
- Friend escape hatches (124.6)
- Type-level priv flip (124.7)

## 2. Design

### 2.1 Helper extraction

Существующая логика в `f1_stmt Stmt::Let` (lines ~3134-3170 в
`compiler-codegen/src/types/mod.rs`) копируется в helper method:

```rust
fn check_priv_pattern(
    &self,
    pattern: &Pattern,
    scrutinee_ty: Option<&TypeRef>,
    errors: &mut Vec<Diagnostic>,
);
```

Helper:
- Resolves type via `pattern.type_path` приоритетно (explicit), fallback на
  `scrutinee_ty`.
- Если type не resolved → no-op (safe; reported elsewhere).
- Если type = Record И outside type-method scope → проверяет каждое
  RecordPatternField на `priv_field`.
- Рекурсивно обходит sub-patterns: для каждого `pf.pattern: Option<Pattern>`
  с resolved sub-field type выполняет `check_priv_pattern(sub, sub_ty)`.
- Pattern::Or → recurses в каждый alternative с тем же `scrutinee_ty`.
- Pattern::Binding → recurses в inner.
- Остальные variants — no-op (Variant/Array/Tuple/Wildcard/Ident/Literal не
  имеют priv-field semantics в этом sub-plan).

### 2.2 Hook sites

| Site | Scrutinee ty source |
|---|---|
| `f1_stmt Stmt::Let` | `infer_expr_type(&d.value, scope)` |
| `f1_expr ExprKind::Match` (each arm) | `infer_expr_type(scrutinee, scope)` |
| `f1_expr ExprKind::IfLet` | `infer_expr_type(scrutinee, scope)` |
| `f1_expr ExprKind::WhileLet` | `infer_expr_type(scrutinee, scope)` |
| `f1_expr ExprKind::For` | `elem_type.clone().or_else(infer_iter_elem)` |
| `f1_expr ExprKind::ParallelFor` | same as For |

### 2.3 Spread in RecordLit (`E_PRIV_FIELD_INIT_SPREAD`)

`Account { ...other }` outside Account-method:
- spread implicitly inits ALL fields including priv (since spread copies)
- emit `E_PRIV_FIELD_INIT_SPREAD` (NEW code) на spread field span
- hint: «use factory method `Account.new(...)` or named fields excluding priv»

Inside Account-method — spread allowed (recv = Account → allowed).

### 2.4 Rest pattern `{ priv_f, .. }` semantics

`{ priv_f, .. }` outside type:
- `priv_f` named explicitly → still fires `E_PRIV_FIELD_PATTERN`
- `..` (Pattern::Record.rest = true) НЕ produces a binding → no leak

D221 §3 формализует: `..` = «ignore remaining fields, no bindings».
