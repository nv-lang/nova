## Зачем

### Нужно для двух классов use cases:

1. **Современные редакторы (Helix, Zed, modern Neovim, GitHub web)** — все они используют tree-sitter как единственный механизм подсветки + structural ops. Без grammar Nova:
   - Не виден в Zed marketplace.
   - Не открывается в Helix runtime (нет `runtime/grammars/nova/`).
   - GitHub web показывает `.nv` файлы plaintext (linguist требует tree-sitter detection для color stats).
   - Neovim 0.5+ users получают только legacy regex-syntax (handcrafted, drift relative to lexer).
2. **Tree-aware operations** — на base tree-sitter parse tree:
   - Structural search (Helix `:tree-sitter` queries).
   - Block-level navigation (`]m` / `[m` next-fn).
   - Folding regions из parse tree (точнее indent-based).
   - Selection expansion по AST node boundaries (Helix `mh`/`ml`, Zed `Ctrl+W`).

### Дополнительные benefits:

- **Drift-detection с компилятором:** test corpus в Ф.5 запускается в CI; если Nova lexer/parser добавляет keyword/production без обновления tree-sitter — CI fail.
- **Reuse в LSP:** опционально nova-lsp (sub-plan 104.0+) может использовать tree-sitter для быстрого incremental highlight tokens (semantic-tokens fallback) — но **не первичный путь**, master M-11 говорит «reuse compiler-codegen for LSP».

### НЕ нужно для:

- Type-checking (LSP делает через compiler-codegen).
- Error recovery в LSP (tree-sitter имеет fault-tolerant parser, но fixture с ошибками — corner-case; LSP полагается на compiler-codegen парсер).
- Code generation / refactoring (rust-analyzer scope; LSP делает через compiler-codegen AST с spans).

## Scope

### In scope

- **`grammar.js`** — full Nova grammar, mirror compiler-codegen productions:
  - Lexical: identifiers, numbers (hex/binary/octal/decimal/float с `_`), strings (double-quoted + interp `${...}` + triple-quoted), char literals, comments (`//`/`///`/`/* */`).
  - Top-level: `module`, `import`, declarations (`fn`/`type`/`effect`/`alias`/`const`/`protocol`/`test`).
  - Function: signatures, generics `[T]` / `[T: Bound]`, params, return type, body, where-clauses, requires/ensures contracts.
  - Type: sum-type `| Variant`, record-type `{ field type }`, alias, protocol-decl.
  - Expressions: primary (literal/ident/paren/path), call, method call (`.`), field access (`.`), indexing `[]`, binary ops, unary ops, lambda `fn() => expr`, `match`, `if`/`else`, blocks, `let`/`mut`/`const`.
  - Statements: assignment, control-flow (`for`/`while`/`loop`/`break`/`continue`/`return`), `throw`, `defer`/`errdefer`, `spawn`/`supervised`/`parallel`/`detach`/`blocking`, contracts.
  - Operators precedence — mirror parser's Pratt-style.
- **Queries (`queries/*.scm`)**:
  - `highlights.scm` — semantic highlighting (keyword/type/function/variable/string/number/comment/etc.).
  - `folds.scm` — folding regions (fn body, type body, comment blocks, match arms).
  - `indents.scm` — indentation rules (block-open/close, multi-line expr continuation).
  - `injections.scm` — string interpolation `${...}` (re-injects Nova), doc-comment markdown injection (`///` → markdown), raw-string `` json`...` `` (JSON injection — Plan 26 syntax).
  - `locals.scm` — scope tracking для local vars/params (улучшает rainbow-highlighting в editors что поддерживают).
- **Test corpus (`test/corpus/*.txt`)** — формат `tree-sitter test`:
  - Smoke tests: hello-world, fn-decl, type-decl per kind.
  - Per-construct: keywords (47 lexer + 4 contextual), expressions, statements, declarations, error-recovery snippets.
  - Stretch: corpus самплы из `examples/` directory (real-world Nova code).
- **Project skeleton (`tree-sitter-nova/` separate repo content prepared в `tools/tree-sitter-nova/` для миграции)**:
  - `package.json` (npm package для `tree-sitter` CLI).
  - `binding.gyp` (Node-binding для native parser load).
  - `bindings/` (Node/Rust bindings stub — generated by `tree-sitter generate`).
  - `LICENSE` (MIT — match `nv-lang/nova`).
  - `README.md` (install + usage docs).
- **Distribution prep**:
  - GitHub release tag spec `tree-sitter-nova-v0.1.0` (initial).
  - Helix runtime PR draft (config + queries — `runtime/grammars/nova/` layout).
  - Zed extension config (`extensions/nova/extension.toml`).
  - `nvim-treesitter` parser-config snippet.

### Out of scope

- Auto-derive grammar from compiler-codegen parser (tree-sitter JS DSL — manual).
- Full semantic analysis (type-check, effect inference) — LSP scope, не tree-sitter.
- Bundling tree-sitter binaries в Nova distribution — editor handles это.
- Helix/Zed/Neovim PR submissions — Plan 104.8 (editor packaging) обрабатывает submission flow. Этот sub-plan готовит artefacts (queries + grammar + docs).
- Final separate repo creation (user-action; см. Q-104.7-1) — этот sub-plan keeps content в `tools/tree-sitter-nova/` для удобства Ф.0-Ф.5; Ф.6 готовит миграционный gist.
- Performance benchmark grammar parse speed — defer к Ф.6 если CI signals.
