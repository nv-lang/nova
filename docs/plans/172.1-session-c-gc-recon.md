   `GC_add_roots(base, base+new_high*slot_size)` (fiber_arena.c:490) — границы ПРЯМО зависят от
   `slot_size`(=layout). Перетряхивание mono-set сдвигает где живые указатели vs pointer-shaped мусор +
   границы root-range → layout-sensitive Heisenbug (false-retention / missed-root→premature-collect→UAF)
   детерминированно становится SEGV. Триггер: агрегированный `plan154` sibling `test_9` (inline-ripple →
   `[]T`/`Vec` spelling-flip → ранний Vec-mono). Класс тот же, что 83.11.
