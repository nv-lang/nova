# Plan 91.9 — `#impl(Protocol1 + Protocol2)` annotation (D186)

> **Подплан Plan 91** ([91-stdlib-mvp-for-0.1.md](91-stdlib-mvp-for-0.1.md)).
> Статус: 🔴 OPEN.
> Зависимости: **Plan 91.8a** (canonical protocols).

## Мотивация

Сейчас Nova использует **structural typing** для protocols (D58): тип satisfies
`Comparable` если у него есть `@compare(other Self) -> int`. Никаких явных
импортов или annotations.

**Проблемы structural typing:**

1. **Нет compile-time guarantee:** type-author не знает, какие protocols он
   реализовал. Если случайно забыл `@equals` — узнаёт только когда другой
   код пытается использовать в `[T Equatable]` bound.
2. **`nova doc` неполный:** инструмент не может уверенно сказать «этот тип
   implements X». Нужен auto-scan по методам с risk of misclassification.
3. **Refactoring fragile:** rename `@compare` → `@cmp` тихо ломает Comparable
   satisfaction; error появляется только в use-site.

**Решение:** opt-in декоратор `#impl(...)` перед type declaration.

## Scope

### Синтаксис

```nova
#impl(Comparable + Hashable + Printable)
type MyDate { y int, m int, d int }

fn MyDate @compare(other Self) -> int { ... }
fn MyDate @hash() -> u64 { ... }
fn MyDate @fmt(sb StringBuilder) -> () { ... }
// `equals` auto-derived из Comparable default body (Plan 91.8a)
```

### Семантика

- **Verification mode (compile-time check):** компилятор проверяет, что type
  имеет все методы каждого protocol в списке. Если нет — clear error с
  списком missing methods.
- **Use-site остаётся structural:** для `[T Comparable]` bound type-checker
  по-прежнему проверяет наличие методов structurally. `#impl` НЕ требуется
  чтобы использовать как Comparable.
- **`nova doc` improvement:** прямо показывает «implements Comparable, Hashable, Printable».
- **Multiple protocols:** через `+` (consistent с multi-bound `[T A + B]`, Plan 101.3).

### Что НЕ делает

- НЕ меняет use-site (остаётся structural — D58 preserved).
- НЕ обязателен (opt-in).
- НЕ создаёт nominal typing.

### Syntax position

`#impl(...)` ставится **перед** `type T` (как другие декораторы `#stable`,
`#auto_*`):

```nova
#stable(since = "0.1")
#impl(Comparable + Hashable)
type UserId { value u64 }
```

Порядок декораторов: `#impl` рядом с `#stable`; order arbitrary.

## Spec / D-block

### D186 — `#impl(...)` annotation для protocol verification

**Файл:** `spec/decisions/02-types.md`.

**Content:**

1. Rationale: opt-in compile-time verification, `nova doc` improvement, refactoring safety.
2. Syntax: `#impl(P1 + P2 + ...)` перед type declaration.
3. Семантика:
   - verification mode (errors at type-site, не use-site)
   - structural use-site preserved
   - multi-protocol через `+` (consistent с multi-bound)
4. Error format: E_IMPL_MISSING_METHODS с listing missing methods и protocol сигнатур.
5. Compatibility: opt-in, existing types не нуждаются в migration.
6. Связь: D58 (structural protocols), D72 (bounds), D183 (canonical protocols), Plan 101.3 (multi-bound `+` syntax).

## Открытые вопросы

1. **Q1:** Embedded protocols (`Comparable use Equatable`) — нужно ли explicit повторение `#impl(Equatable)` или embed достаточен? **Default:** embed достаточен — `#impl(Comparable)` automatically verifies Equatable methods (через `use`).
2. **Q2:** Multiple `#impl` annotations разрешены: `#impl(A) #impl(B)` или только single? **Default:** single annotation с `+` syntax; multiple — error для clarity.
3. **Q3:** `#impl()` пустой список — что делает? **Default:** parse error.
4. **Q4:** Protocols с generic params (`#impl(Iterable[int])`) — supported? **Default:** да, generics passed through.
