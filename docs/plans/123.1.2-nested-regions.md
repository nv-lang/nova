## 1. Контекст

V1.1 splits FN body's **top-level** stmts на barrier boundaries.
Когда top-level stmt сам contains a barrier (e.g. `if cond { ... @x =
99 ... }`), V1.1 treats whole stmt as barrier и skips its body.
Realistic pattern:

```nova
fn Counter mut @nested_cycle(cond bool, target int) -> int {
    mut acc = 0
    if cond {
        ro a = @x        // NOT cached by V1.1
        ro b = @x        // NOT cached by V1.1
        @x = target      // barrier (nested)
        ro c = @x        // NOT cached by V1.1
        ro d = @x        // NOT cached by V1.1
        acc = a + b + c + d
    }
    acc
}
```

V1.1 fires no cache (outer regions are empty or below-threshold). The
read-heavy nested then-block goes unoptimized — wasted opportunity.

V1.2 closes the gap: recursive nested-block analysis, allocating
unique `_at_<F>_n<N>` cache locals per qualifying nested region.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**Refactor:** `find_mut_regions_with_ipa(body: &FnBody, ...)` →
delegates к new `find_mut_regions_in_block(b: &Block, ...)` — block-
level extraction. V1.1 wraps via FnBody → Block.

**New helpers (~330 LOC):**

- `alloc_nested_region_local(fname, &mut seq, &mut local_names) ->
  String` — allocates `_at_<F>_n<N>` collision-safe name. Counter
  `seq` is session-monotonic (per cache_fn_with_ipa call).
- `walk_nested_blocks_for_mut_field(top_block, fname, ...)` — entry
  point. Walks top block's stmts/trailing descending into nested
  expressions.
- `process_nested_block_for_mut_field(block, fname, ...)` — per-block
  region analysis + target allocation + read rewrite + let injection.
  **Bottom-up**: descends into nested children FIRST (Phase A), then
  processes this block (Phase B). Bottom-up prevents outer-rewriter
  double-rewrite по already-inserted nested lets.
- `descend_stmt_for_nested(s, fname, ...)` — walks every Stmt variant.
- `descend_expr_for_nested(e, fname, ...)` — exhaustive ExprKind walk.
  Recognizes ALL nested-block-containing variants: If/IfLet/While/
  WhileLet/Match (arms)/For/ParallelFor/Loop/With/Forbid/Realtime/
  Detach/Blocking/Supervised/Block-Expr/Trailing-Block/-Fn/-Legacy.
  Closures (`Lambda/ClosureLight/ClosureFull/HandlerLit/ProtocolLit`)
  explicitly skipped — V1 closure exclusion inherited.
- `descend_else_for_nested(eb, fname, ...)` — handles `ElseBranch`
  enum (`Block(Block)` или `If(Box<Expr>)`).

**Integration в `cache_fn_with_ipa`:**

1. Original V1.1 sequence unchanged (analysis + ro/mut candidates +
   name allocation + `rewrite_fn_body_split_with_ipa`).
2. Early-return condition refined: skip when ALSO no mut-field is
   read anywhere (else V1.2 may have nested work).
3. After Phase 1 rewrite: build `mut_field_names_for_nested` list of
   mut fields с reads > 0 (excluding closure-captured), iterate
   alphabetical order calling `walk_nested_blocks_for_mut_field`.
4. Shared budget: `cfg.max_per_fn − total_caches` remaining slot.
