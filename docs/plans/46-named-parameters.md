## Зачем

D102 вводит две связанные возможности:

1. **Значения параметров по умолчанию** — `fn connect(host str, port int = 8080)`.
2. **Именованные аргументы на вызове** — `connect("h", port: 9000)`.

Мотивация (см. D102 §«Почему»): self-documenting call-sites вместо
позиционных `bool`-флагов (классическая ошибка LLM-генерации); снятие
builder/option-struct boilerplate для простых случаев; и — конкретно —
включение синтаксиса `supervised(cancel: tok)` ([D75](../../spec/decisions/06-concurrency.md#d75-supervisedcancel-tok--структурная-отмена-с-внешним-токеном)).

Без упрощений: фича вводится как **общая** возможность языка для всех
функций и `@`-методов, не точечный спецсинтаксис.

---

## Архитектурное решение

Опорные правила — из D102. Ключевые инварианты для реализации:

- **Объявление:** параметры с дефолтом идут строго после параметров без
  дефолта. Default-выражение вычисляется на **месте вызова**, каждый
  вызов заново, и может ссылаться на предшествующие параметры и
  module-level `const`.
- **Вызов:** позиционные аргументы первыми (bind слева направо), затем
  именованные (переставимы). Именованный перед позиционным — ошибка.
  Каждый параметр связывается ровно один раз.
- **Variadic** ([D69](../../spec/decisions/03-syntax.md#d69-variadic-параметры-через-items-t))
  остаётся последним, без дефолта; именованные аргументы — только для
  параметров до variadic.
- **Trailing-форма** ([D43](../../spec/decisions/03-syntax.md#d43)) связывает
  последний функциональный параметр; нельзя одновременно trailing-формой
  и именованным аргументом.
- **Overloading в Nova отсутствует** — разрешение «имя → параметр»
  однозначно, без type-directed resolution.

Codegen-стратегия: генерируемый C-вызов **всегда позиционный**.
Компилятор на месте вызова раскладывает позиционные + именованные
аргументы в порядок параметров callee и вставляет вычисление
default-выражений для опущенных параметров (left-to-right, как temps).

## Фазы

### Ф.0 — AST + парсер

- `Param` получает поле `default: Option<Expr>`.
- Аргумент вызова: `CallArg { name: Option<Ident>, value: Expr }` (или
  enum `Positional | Named`). Затрагивает `ExprKind::Call`,
  method-call, конструкторы — единая структура аргументов.
- `parse_params`: после `ident type` — опциональный `= expr`. Diagnostic
  «default-параметр перед обязательным» (правило 1 объявления).
- `parse_call_args`: распознать `ident ':' expr` как именованный
  аргумент; diagnostic «позиционный после именованного».
- Дисамбигуация: внутри `(...)` `ident ':' expr` — всегда именованный
  аргумент; коллизии с record-литералом нет (record — `Имя { ... }`).

### Ф.1 — Name-resolution / type-checker

- Argument-binding pass: для каждого `Call` сопоставить позиционные +
  именованные аргументы с параметрами callee. Diagnostics:
  - именованный аргумент с несуществующим именем параметра;
  - параметр связан дважды (позиционно и по имени);
  - обязательный параметр (без дефолта) не связан;
  - позиционный аргумент после именованного.
- Default-выражение type-check'ается против типа параметра в scope,
  где видны **предшествующие** параметры и module-level `const`.
- Результат binding-pass'а сохраняется в типизированном AST как
  нормализованный позиционный список (с маркерами «использовать
  дефолт») — codegen не пересопоставляет.

### Ф.2 — Codegen (`emit_c.rs`)

- На месте вызова: разложить аргументы в позиционный порядок callee.
- Для опущенных параметров — эмитить вычисление default-выражения как
  локальный temp перед вызовом, left-to-right; default может ссылаться
  на ранее вычисленные аргументы/параметры.
- Именованные аргументы, переданные не по порядку, — эмитить temps в
  source-order вычисления, переставить в call-order (сохранить порядок
  side-эффектов = source-order аргументов).
- Сгенерированный C-вызов — чисто позиционный.

### Ф.3 — Взаимодействие с variadic и trailing-формой

- Variadic + дефолты: параметры до `...items` могут иметь дефолты;
  `...items` — без дефолта; именованных аргументов после variadic нет.
- Trailing-block / trailing-fn (D43): trailing-форма заполняет последний
  функциональный параметр. Diagnostic, если тот же параметр передан и
  trailing-формой, и именованным аргументом.
- `@`-методы и protocol-методы: именованные аргументы работают
  единообразно (receiver — позиция 0, дефолты/имена для остальных).
- `builtins.nv` / external-функции: дефолты допустимы в декларации
  external-функции; binding-pass работает так же.

### Ф.4 — Тесты

`nova_tests/named_params/` — positive:
- все позиционно; все именованно; смешанно (часть поз. + часть им.);
- именованные переставлены; опущенный дефолтный параметр;
- default-выражение ссылается на предшествующий параметр
  (`fn slice(xs []int, from int = 0, to int = xs.len())`);
- именованный аргумент в method-call и конструкторе;
- именованные аргументы перед variadic; trailing-fn + именованные для
  не-последних параметров;
- порядок side-эффектов при переставленных именованных аргументах
  (observable — например через push в массив).

`nova_tests/.../negative_*` — `EXPECT_COMPILE_ERROR`:
- default-параметр объявлен перед обязательным;
- позиционный аргумент после именованного;
- параметр связан дважды;
- обязательный параметр не передан;
- именованный аргумент с несуществующим именем;
- именованный аргумент для variadic-параметра;
- параметр передан и trailing-формой, и именованным аргументом.

## Что НЕ входит

- **Spread аргументов в вызов** (`f(...record)`, `f(..array)`) — отвергнут
  в D102 §«Что отвергнуто».
- **Обязательно-именованные параметры** (Swift `func f(name x:)`) —
  отвергнуто; любой параметр и позиционный, и именуемый.
- **Перегрузка функций** — в Nova отсутствует, вне scope.
- **Реализация `supervised(cancel:)`** — это Plan 47.

## Acceptance criteria

- [ ] Объявление с дефолтами парсится; default-перед-обязательным —
      compile error.
- [ ] Именованные аргументы парсятся; позиционный-после-именованного —
      compile error.
- [ ] Binding-pass ловит: двойную привязку, отсутствие обязательного,
      несуществующее имя.
- [ ] Default-выражение вычисляется на месте вызова, видит
      предшествующие параметры.
- [ ] Codegen: переставленные именованные аргументы дают корректный
      позиционный C-вызов с source-order side-эффектами.
- [ ] Variadic + trailing-форма: правила взаимодействия enforced.
- [ ] Все positive + negative тесты `nova_tests/named_params/` PASS.
- [ ] Полный regression `nova test` без новых FAIL (release-сборка).
