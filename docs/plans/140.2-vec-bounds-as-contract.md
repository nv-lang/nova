# Plan 140.2 — Vec `@index` bounds as elidable contract (`[M-140-bounds-as-contract]`)

> **Создан:** 2026-06-13.  **Статус:** 📋 IN PROGRESS.  **Приоритет:** P2.
> **Закрывает:** `[M-140-bounds-as-contract]`.  **Зависит от:** Plan 140 (enforce-in-release) +
> Plan 150 (`&&`-форма; вакуумная `0<=i<@len` теперь compile-error).
> **Решение автора (2026-06-13):** вариант 1 — de-inline `v[i]` → контракт на `@index` + переиспользовать
> элизию Plan 140 (proven → zero-cost, unproven → runtime abort). Rust-style bounds check.

## Контекст / находка
`Vec @index(i int)` (vec_owned.nv:404) и `mut @index(i int, val)` (433) **уже** bounds-check'ают вручную
(`if i<0 || i>=@len { panic }`) — OOB безопасен, но проверка **always-on**. Кодген `v[i]` **инлайнит**
доступ ([emit_c.rs:18920](compiler-codegen/src/codegen/emit_c.rs#L18920)) с `nv_panic_index_oob` —
**мимо** контрактной машины Plan 140 (`proven_contracts` / `contracts_elided_here`). Поэтому даже
доказуемый индекс (`for i in 0..v.len()`) платит за проверку.

**Цель:** выразить bounds как контракт `requires 0 <= i && i < @len` на `@index` и направить `v[i]`
через метод, чтобы Plan 140 элидировал проверку при доказуемом индексе (zero-cost), оставляя runtime
abort при недоказуемом (safe). Семантика OOB не меняется (был panic → стал contract-violation abort).

## Ф.1 — stdlib: контракт на `@index` (~0.25d)
- `Vec[T] @index(i int) -> T` (vec_owned.nv:404): убрать ручной `if i<0||i>=@len { panic }`, добавить
  `requires 0 <= i && i < @len`; тело → `unsafe { @data[i] }`.
- `Vec[T] mut @index(i int, val T) -> ()` (433): аналогично — `requires 0 <= i && i < @len` + тело
  `unsafe { @data[i] = val }`.
- Range `@index(r Range)` (771): отдельный контракт (`requires 0 <= r.start && r.end <= @len && r.start <= r.end`)
  — оценить отдельно (slice имеет свой inline-check); MVP — оставить как есть, отметить followup если не входит.
- **Commit:** `feat(plan140.2 Ф.1): requires 0<=i && i<@len on Vec @index/mut @index`

## Ф.2 — codegen: route `v[i]` / `v[i]=val` через метод (~0.5d, риск)
- READ `v[i]` (emit_c.rs Index-arm ~18920 + другие Index-сайты 8420/14257): для `Nova_Vec____<T>`
  заменить inline statement-expr на вызов монолизированного `@index` (reuse `vec_method_call` /
  `register_vec_mono_method`). Монолизация триггерится самим вызовом.
- WRITE `v[i]=val` (Stmt::Assign ~16053): route через `mut @index(i,val)` (или сохранить inline, но
  emit контракт-проверку с элизией — оценить, что проще/безопаснее).
- **Per-fix:** после каждого сайта — targeted-фикстура + сборка; **basics ОБЯЗАТЕЛЬНО** (урок 139.2).
- **Commit:** `feat(plan140.2 Ф.2): route v[i] through contracted @index (Plan 140 elision)`

## Ф.3 — Тесты pos+neg (~0.25d, И C-codegen И interp, релизный nova)
**Позитивные** (`nova_tests/plan140_2/`):
- in-bounds `v[i]` / `v[i]=x` работают; `for i in 0..v.len() { v[i] }` (proven — должно элидироваться).

- OOB `v[i]=x` → abort.
- **Commit:** `test(plan140.2): bounds-contract pos/neg + OOB-trap`

## Критерии приёмки
- **B1** — `Vec @index`/`mut @index` несут `requires 0 <= i && i < @len`; ручной `if...panic` убран.
- **B2** — `v[i]` / `v[i]=val` направлены через контрактный метод (или inline с контракт-элизией).
- **B3** — OOB `v[i]` (i<0 или i≥len) → runtime abort в **debug И release** (Plan 140 enforce); сообщение
  внятное (D140.1 формат / index+len).
- **B4** — доказуемый индекс (`for i in 0..v.len()`) — проверка **элидируется** (proven via Plan 140;
  спот-чек сгенерированного C: нет bounds-branch).
- **B5** — `@get`/`@first`/`@last` (Option) — поведение не изменилось (graceful None на OOB).
- **B6** — pos+neg фикстуры релизным nova (C-codegen + interp); **0 регрессий** (basics + широкий sweep);
  perf-sanity: unproven `v[i]` не регрессит (метод заинлайнен).
- **B7** — spec D-блок + docs + backlog closed.
