auto-armed at main() start.

**Фикс:** test-кейс «runtime not initialized by default» удалить либо
переименовать в «runtime auto-armed at main start» с инвертированным
assert'ом. Сам сценарий «зачем оно проверяется» — устарел.

#### C2 — `mn_maxprocs_getter`

Аналогичная правка ассерта в одном из 3 тест-кейсов.

**Файлы:** `nova_tests/concurrency/mn_runtime_smoke.nv`,
`nova_tests/concurrency/mn_maxprocs_getter.nv`.

### Ф.3 — Plan 83.2 Ф.2: sweep + escape hatch (~1 day)

Соответствует [Plan 83.2 §3 Ф.2](83.2-mn-default-flip.md).

- **Sweep `runtime.init(n)`** в `std/` (если есть обязательные
  вызовы — Plan 83.2 audit показал, что в std нет, только в doc-
  комментариях). Перепроверить.
- **Test-suite review:** mn_* тесты тестируют API `runtime.init`
  напрямую — оставить. Прочие, что начинались с `runtime.init(N)` для
  «активации M:N» — убрать вызов, обновить doc-комментарий ("теперь
  M:N по умолчанию").
- **Escape hatch:** добавить smoke-тест
  `nova_tests/concurrency/mn_escape_hatch_single.nv` под
  `// ENV NOVA_MAXPROCS=1` — supervised{spawn} compiled-binary
  выполняется с 1 worker'ом.
- **Документация:** `docs/concurrency*.md` (если есть) + README
  обновить: «M:N по умолчанию; для детерминистики `NOVA_MAXPROCS=1`».

### Ф.4 — Plan 83.2 Ф.3: spec + speedup benchmark (~1-2 day)

Соответствует [Plan 83.2 §3 Ф.3](83.2-mn-default-flip.md).

- **Spec D-block** в `spec/decisions/06-concurrency.md`:
  - «M:N — дефолтная модель исполнения для compiled-бинарей».
  - `nova run` (интерпретатор) — однопоточный.
  - Порядок разрешения `NOVA_MAXPROCS`.
  - Правило «worker не делает блокирующую работу инлайн» (ссылка на
    [Plan 83.3](83.3-blocking-effect-threadpool.md)).
- **Speedup benchmark** в `bench/micro/`: CPU-bound fan-out (например,
  parallel sum / parallel hash) с замером single-thread (`NOVA_MAXPROCS=1`)
  vs multi-thread. Замер speedup ratio на N ядрах. Ожидание: linear
  scaling до количества ядер на CPU-bound нагрузке.
- **`docs/plans/README.md`:** Plan 83.2 → ✅ ЗАКРЫТО; Plan 83.4 → ✅
  ЗАКРЫТО; Plan 83 roadmap → 83.1+83.2+83.3 ✅.
