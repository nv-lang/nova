#### Escape hatch test

Создать `nova_tests/concurrency/mn_escape_hatch_single.nv`:
```nova
// Plan 83.2 Ф.2: NOVA_MAXPROCS=1 — escape hatch для детерминистики.
// ENV NOVA_MAXPROCS=1

module concurrency.mn_escape_hatch_single

test "NOVA_MAXPROCS=1 → exactly 1 worker after first spawn" {
    // Без runtime.init: auto-arm на первом spawn.
    mut x = 0
    supervised { spawn { x = 99 } }
    assert(x == 99)
    assert(runtime.worker_count() == 1)
}

test "fibers serialize on single worker (no race on shared var)" {
    mut sum = 0
    supervised {
        for i in 1..11 { spawn { sum = sum + i } }
    }
    assert(sum == 55)  // нет race с 1 worker'ом
    assert(runtime.worker_count() == 1)
}
```

### Ф.4 — Plan 83.2 Ф.3: speedup-бенчмарк (~1-2 dev-day)

#### Канонические workloads в `bench/micro/parallel_*`

1. **`parallel_sum.nv`** — `supervised { parallel for i in 1..N { sum += i*i } }`
   с `N` = 10⁶, замер ns per iteration. CPU-bound, минимальная
   синхронизация.
2. **`parallel_hash.nv`** — fan-out hash 10⁵ keys в worker'ы, join
   результаты. CPU+memory bandwidth.
3. **`parallel_fib.nv`** — `parallel for i in 1..40 { fib(i) }` — pure
   CPU recursive.

#### Замер speedup

Каждый бенч прогоняется ТРИ раза:
1. `NOVA_MAXPROCS=1` (baseline single-thread).
2. `NOVA_MAXPROCS=N` (где N = NumCPU, обычно 4-16).
3. Сравнение с Go reference (`bench/reference/go_parallel_sum.go` —
   эквивалент на горутинах + `runtime.GOMAXPROCS(N)`).

#### Acceptance targets

- `parallel_sum` на N=4: **≥ 3.0× speedup** vs `NOVA_MAXPROCS=1`
  (≥ 0.75× линейного).
- `parallel_sum` vs Go: **≥ 0.85× Go time** (не хуже 15% от Go).
- Если ниже 0.85× — измерить вклад Boehm GC через `--gc malloc` (нет
  GC, leak). Если разрыв в GC — fixate в spec как известная дельта
  (concurrent GC — следующий план Plan 27.2 если будет).

### Ф.5 — Stress/soak + TSAN gate (~1-2 dev-day)

#### Stress harnesses в `83.4.4-artifacts/` (standalone C)

1. **`stress_spawn_1m.c`**: 10⁶ no-op spawn (10× `spawn { }`,
   100k repetitions), Boehm GC stats до/после — нет leak'а.
   Watchdog 60s.
2. **`stress_park_wake_100k.c`**: 10⁵ park-wake cycles с timer'ом
   1 ms (через `nova_sched_park_until`) — без abort. Watchdog 30s.
3. **`stress_supervised_migration.c`**: 10⁴ supervised-scopes × 100
   fibers, форсируем work-stealing migration через uneven workload —
   без deadlock/race. Watchdog 60s.

Все harnesses интегрировать в `83.4.4-artifacts/build.ps1` +
`build.sh` для CI-готовности.

#### TSAN gate (Linux only)

clang `-fsanitize=thread -fno-omit-frame-pointer` на
`nova_tests/concurrency/*` (sub-set до полной suite — sleep_*,
supervised_*, cancel_*, parallel_*, mn_*).

Acceptance: 0 reported races в park/wake-пути, в supervised drain,
в handler-storage save/restore, в work-stealing steal.

**Pre-existing races** (если выявятся вне scope Plan 83.4) →
зафиксировать в `simplifications.md` как followup-маркер, **не**
блокирует closure.

### Ф.6 — Spec D-block + закрытие (~1 dev-day)

#### Spec в `spec/decisions/06-concurrency.md`

Новый D-block (нумерация по конвенции — следующий доступный, e.g.
**D145**) «M:N runtime — default model + NOVA_MAXPROCS resolution»:

1. **Default model.** Compiled-бинарь M:N включён по умолчанию.
   `nova run` (интерпретатор) — однопоточный.
2. **Worker-count resolution order** (из Plan 83.1):
   `explicit runtime.init(n>0)` > `NOVA_MAXPROCS` env > `uv_available_parallelism`.
   Клэмп [1, 1024].
3. **Lazy worker pool.** Workers поднимаются лениво на первом
   `spawn`; hello-world без spawn — 0 worker'ов.
4. **Escape hatch.** `NOVA_MAXPROCS=1` — один worker, без
   параллелизма (НЕ даёт scheduling-детерминизм; убирает только
   inter-worker race).
5. **Worker не делает блокирующую работу инлайн** — `Blocking`-эффект
   (Plan 83.3) обязательно обернуть `blocking{}` для FFI / syscall'ов
   вне libuv async.
6. **Spawn ordering**: НЕ специфицирован (work-stealing scheduler;
   тесты на ordering должны быть set-equality или под
   `NOVA_MAXPROCS=1`).
7. **Cancellation**: hierarchical через scope-tree (см. [83.4.3](83.4.3-mn-api-semantic-alignments.md)),
   cooperative — fiber observes cancel на следующем park-check либо
   user-side `runtime.cancelled()` poll.
8. **Per-fiber state**: handler-snapshot + parent_scope per fiber
   (см. [83.4.2](83.4.2-supervised-drain-ownership.md)); persists across worker migration.
