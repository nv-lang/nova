## 1. Bugs + симптомы

**Тесты:** `per_fiber_handlers`, `time_handler`.

**Симптом под активным flip** (несмотря на Plan 83.4.2 Ф.2 handler-
storage save/restore на worker):
- `per_fiber_handlers`: один из sub-тестов всё ещё падает —
  предположительно nested `with X = h { spawn { with X = h2 { … } } }`
  edge case.
- `time_handler`: специфический corner case для `Time` effect handler
  (e.g. Time.sleep вызывается между save/restore'ами).

## 2. Root cause hypothesis

Plan 83.4.2 Ф.2 добавил **basic** handler-snapshot save/restore на
worker (через `nova_effect_snapshot_save`/`_restore` вокруг
`mco_resume` в `_worker_main`). Это покрывает:
- single fiber, один `with X = h { … }` блок — корректно.
- два spawned fiber'а с разными handler'ами — каждый видит свой.

НЕ покрывает:
- **Nested with в spawn**: `with X = h1 { spawn { with X = h2 { foo() } } }`.
  Parent fiber's snapshot захватывается на spawn-time (h1).
  Spawned fiber resume'ится → restore'ит snapshot (h1) → executes
  inner `with X = h2` → push'ит h2 в TLS → call foo() (видит h2) →
  pop h2 → возвращается. Save → snapshot теперь имеет h1 (correct).
  **Но**: если spawned fiber yield'ит ВНУТРИ inner `with` (e.g.
  `Time.sleep` внутри `with X = h2`), save фиксирует TLS state с
  h2 на top. Restore позже — h2 правильно. Pop происходит после
  resume. OK?

  Hmm, must verify. Корнер: если `Time.sleep` внутри `with X = h2`
  под M:N, worker park'ает fiber → save snapshot (с h2) →
  unparks → restore (с h2) → продолжает inner-block, pop'ит h2.
  Должно работать.

- **Спавн from inside with блока** на parent fiber: parent's TLS
  имеет h1 (from `with X = h1`). Spawn захватывает snapshot. Spawned
  fiber начинает с h1. Parent yield'ит (например в Time.sleep) →
  worker save'ит parent's snapshot с h1 → restore при wake'е. OK.
  **Но**: если spawned fiber меняет TLS через свой `with X = h2`,
  pop'ит h2 ПОСЛЕ scope-end. Если scope-end приходит ВО ВРЕМЯ
  spawn'ed yield (impossible by Nova semantics — `with`-block
  определён в Nova syntax как scope) — нет проблем. ОК?

Hypothesis: corner случай — когда `with X = h2` внутри spawn вызывает
`Time.sleep` → spawned fiber park'ится → worker B steal'ит его → B
restore'ит snapshot — snapshot захвачен с h2 на top, restore делает
`_nova_handler_X = h2`. OK.

Реальное проявление — нужна Ф.0 investigation (запустить тест под
flip и посмотреть EXACT FAIL output).

## 3. Cross-runtime reference

### tokio `task_local!`

```rust
task_local! {
    static FOO: u32;
}

FOO.scope(42, async {
    spawn(async { FOO.try_with(|v| println!("{}", v)) }); // inherits 42
}).await;
```

- `scope(value, future).await` — устанавливает TLS на время future.
- Spawned tasks НЕ inherit by default! `FOO.try_with` returns Error.
- Чтобы inherit — explicit propagation `FOO.scope(value, future)`
  при spawn.

### Node `AsyncLocalStorage`

```js
const als = new AsyncLocalStorage();
als.run({ user: 'alice' }, () => {
    setImmediate(() => {
        als.getStore();  // { user: 'alice' } — inherited
    });
});
```

- Snapshot **inherited** через `setImmediate`, `process.nextTick`,
  Promise chains.
- Снапшот immutable — `als.run` создаёт new context (fork-on-set).

### Kotlin `CoroutineContext.Element`

```kotlin
val ctx = MyKey + parentContext
launch(ctx) {
    val v = coroutineContext[MyKey]  // inherited
}
```

- Context elements — immutable, fork-on-update.
- Inheritance automatic в child coroutines.

### Nova algebraic effects

`with X = h { body }` — body видит h как handler для X. Если body
spawn'ит fiber — fiber должен (semantically D80) видеть h тоже (как
Node AsyncLocalStorage), пока не push'ит свой `with X = h2`.

Текущая Plan 83.4.2 Ф.2 implementation: snapshot захватывается на
**worker resume time** из home scope's `fiber_effect_snapshot[]`. Этот
snapshot обновляется на save'е после yield. Init snapshot на
**spawn-time** должен взять current TLS (от parent fiber'а where it
spawn happens).

**Hypothesis:** на spawn-time `fiber_effect_snapshot[i]` init'ится
`nova_alloc(NovaEffectSnapshot)` → пустой snapshot, не наследует
parent. Spawn fiber видит EMPTY effect snapshot → first `with X = h2`
PUSH'ит h2 на TOP пустого stack → fine. Но если spawn'ed fiber делает
`raise X` БЕЗ собственного `with`, ищет handler — нет (snapshot
empty) → fall to outer (parent's) handler? Сейчас вернётся к TLS state
если snapshot не активирован, либо empty snapshot если restore'нут.

**Likely bug:** spawn-time init не наследует parent TLS handler-state.
Spawn'ed fiber начинает с пустой handler-stack, не наследует parent's
`with X = h1`. Test:
```nova
with X = handler1 {
    spawn {
        raise X  // expected: handler1 called; actual: no handler → panic
    }
}
```

Если это и есть bug — fix: spawn captures parent's current TLS snapshot
into spawned ctx's `effect_snapshot[i]`.

**Confirmation TBD в Ф.0**.

## 4. Дизайн решения

### Ф.1 — Diagnose per_fiber_handlers + time_handler (~1 dev-day)

Run флип-active build на этих тестах, capture exact fail output.
Для каждого fail:
1. Что test пытается проверить (semantic).
2. Какой path: nested with / spawn-inherit / yield-in-with / etc.
3. Где сейчас break.

### Ф.2 — Spawn-time handler inheritance (likely fix) (~1-2 dev-day)

Если diagnosis confirms «spawned fiber не наследует parent's TLS
handler-state»:

```c
/* В nova_fiber_spawn_into / nova_runtime_spawn_into: */
/* После alloc'а NovaEffectSnapshot: */
nova_effect_snapshot_save(snapshot);  /* captures CURRENT TLS state
                                       * (parent fiber's handler-stack) */
q->fiber_effect_snapshot[slot] = snapshot;
```

Spawn'ed fiber на первом resume → worker restore'ит snapshot →
видит parent's handler-stack. С semantics tokio: explicit inheritance
default; не нужно `with X = h2 { spawn { … } }` шаблон чтобы inherit.

### Ф.3 — Time-handler specifics (~0.5 dev-day)

`time_handler` test — investigate отдельно. Если Time-effect
handler-snapshot имеет specifics (e.g. `Time.now()` вызывается
ДО handler-resolve где snapshot ещё empty), нужен init-from-TLS.

Likely same root cause as per_fiber_handlers (snapshot init не
наследует) → один fix покрывает оба.

## 5. Acceptance

- `per_fiber_handlers`, `time_handler` PASS под active flip.
- Spawned fiber видит parent's `with X = h` handler-state by default.
- 0 регрессий existing effects suite.

## 6. Risk

- **Handler-inheritance ABI** — изменение init snapshot policy
  может surprise existing user-code, который полагался на «spawned
  fiber starts with empty handlers». Mitigation: проверить existing
  tests; если регресс — обсудить semantics в D80 amend (parent-
  inherit default ≠ existing).
- **Snapshot capture cost on spawn** — `nova_effect_snapshot_save`
  на spawn-time добавляет overhead на каждый spawn. Mitigation:
  cost — fixed K = NOVA_MAX_EFFECTS pointer copies; small.
