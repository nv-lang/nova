### In scope (D173 AI-first guidance — special content)

Decision tree «когда что использовать»:

```
Нужно ли разделить mutable state между fiber'ами?
│
├── НЕТ → не нужен sync. Используй channel + actor pattern (D79).
│
├── ДА, exactly-once init —
│        ┌── stateless action (no value)         → Once.call_once(fn)  [D171]
│        ├── value-capturing (return T)          → OnceCell[T].get_or_init(fn)
│        └── auto-init on first access (wrap T)  → Lazy[T] with .force()
│
├── ДА, counter / numeric stat —
│        ┌── single int                         → AtomicI64.fetch_add(delta, Ordering.Relaxed)
│        ├── max/min tracking                   → AtomicI64.fetch_max(value, Relaxed)
│        └── bitset / flags                     → AtomicU32.fetch_or / fetch_and
│
├── ДА, one-shot ownership ("first wins") —
│        ┌── bool flag                          → AtomicBool.swap(true) returns false → winner
│        └── pointer ("first to publish")       → AtomicPtr.compare_exchange(null, ptr)
│
├── ДА, exclusive access to complex state —
│        ┌── short critical section, simple     → Mutex + with_lock(fn)  [D169]
│        ├── read-heavy, occasional writes      → RwLock + with_read(fn) / with_write(fn)
│        └── recursive callbacks                → ReentrantMutex (opt-in, recommend regular Mutex first)
│
├── ДА, bounded concurrency / rate limit       → Semaphore + with_permit(fn)  [D170]
│
├── ДА, N-party rendezvous (epoch sync)        → Barrier (reusable) / CountDownLatch (one-shot)
│
└── ДА, "wait until predicate" —
         → Mutex + Condvar + wait_until(m, predicate)  [D170]
```

**5 canonical patterns** (with anti-patterns):

1. **Counter** — `AtomicI64.fetch_add(1, Ordering.Relaxed)`.
   - Anti: `Mutex.with_lock { counter = counter + 1 }` (overkill,
     waste of contention).

2. **One-shot init** — `Once.call_once { ... }` или
   `Lazy.new(fn).force()`.
   - Anti: `if !flag { mutex.lock(); if !flag { init(); flag = true };
     mutex.unlock() }` (DCL pattern, error-prone).

3. **Producer-consumer bounded buffer** — `Mutex + Condvar` predicate-
   loop:
   ```nova
   // producer
   m.with_lock { ||
       while buffer.is_full() {
           not_full.wait(m)
       }
       buffer.push(item)
       not_empty.notify_one()
   }
   ```
   - Anti: `Channel[T]` если backpressure нативный (channel ❤ better
     option in 90% cases).

4. **Read-heavy snapshot** — `RwLock.with_read { ... }`.
   - Anti: `Mutex` (всех writers и readers serializes).

5. **Rate-limited workers (N parallel)** — `Semaphore.new(N) +
   with_permit { work() }`.
   - Anti: `Channel[unit].new(N)` token channel (works but verbose).

### D167 — Memory ordering & happens-before

```markdown
## D167. Memory ordering & happens-before между fiber'ами (Plan 103.1)

### Что
- `Ordering` enum: `Relaxed | Acquire | Release | AcqRel | SeqCst`
- `fence(Ordering)` module function
- Default ordering для simple-overload methods — `SeqCst` (M1)
- Compile-time validation invalid ordering на load/store

### Правило
[Verbatim signatures из 103.1]

### Memory model contract
- Bootstrap-runtime (single-threaded fiber pool): orderings reduce к
  sequenced-before; все варианты semantically equivalent.
- M:N runtime (Plan 23, Plan 83.4.5.x): полный memory model по
  C11 `__atomic_*` семантике; happens-before между fiber'ами через
  paired Acquire/Release.
- SeqCst: total order на всех SeqCst ops.

### Связь
- D14, D50, D71 — fiber runtime подложка
- D79 — channel send/recv → happens-before (Go-style)
- Plan 103.1 — реализация
- Plan 103.2 — uses Ordering на atomic ops

### Что отвергнуто
- `Ordering::Consume` (C++ rarely used, ill-specified, deprecated)
```

## D168. Atomic primitives API contract (Plan 103.2)

### Что
- 12 sized atomic типов: AtomicI8/I16/I32/I64, AtomicU8/U16/U32/U64,
  AtomicIsize/Usize, AtomicBool, AtomicPtr[T]
- 13 operations matrix: load, store, swap, compare_exchange,
  compare_exchange_weak, fetch_{add, sub, or, and, xor, max, min, nand}

- Plan 103.2 реализация
