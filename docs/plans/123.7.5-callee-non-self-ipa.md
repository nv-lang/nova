## 1. Контекст

V7.1 IPA refines `@method()` (direct self-method) barrier detection:
calls whose write-set excludes `fname` don't invalidate the cache.
But V7.1 conservatively invalidates для ANY non-direct-self receiver
— including the very common `@F.method()` pattern.

Realistic scenario:

```nova
type Buf { mut count int, mut arr []int }
fn Buf mut @track(v int) -> int {
    ro a = @count          // V1.1 region A
    ro b = @count
    @arr.push(v)           // ← V7.1 treats as barrier (conservative)
    ro c = @count          // V1.1 starts new region after barrier
    ro d = @count
    a + b + c + d
}
```

V7.1 sees `@arr.push(v)` as a non-direct-self call → conservative
invalidate → @count cache split into two regions (only one usable
per V1 first-region rules). With V1.1, both regions cached
separately. But this is **wasteful**: `@arr.push()` operates on
`@arr` field's value, by Nova's type system cannot reach `@count`
field of `self`. The cache could span across the call.

V7.5 refines this: для `@F.method()` calls, sibling field caches
(`fname != F`) survive.

## 2. Workflow-driven scope correction

Pre-V7.5 docs claimed "V7.5 closes WriteBuffer.@write_char chain
pattern". **This was wrong.** Workflow w5dlb8t9w root_cause agent
revealed:

1. AST для `@buf.push(a).push(b).push(c)` has **exactly 1**
   `Member { obj: SelfAccess, name: "buf" }` node. Parser at
   compiler-codegen/src/parser/mod.rs:5969-5975 constructs left-deep
   nested Call/Member tree без duplicating leaf.

2. Three `nova_self->buf` in C output come from codegen recursive
   `emit_expr(obj)` propagation в `emit_c.rs:19567` (the `"push"`
   builtin handler). Each chain level appends a
   `nova_array_push_*(obj_c, ...)` statement via `self.line(...)`,
   each independently substituting the receiver C-string. Plus
   `return Ok(obj_c)` at line 19593 propagates the same string
   upward (Plan 91.7 / D181 fluent-`@`-return).

This is purely a **codegen** string-substitution propagation issue,
not an AST или IPA issue. Field-cache (V1/V1.1/V1.2 mut/ro) can't
help because there's only 1 AST leaf to cache.

V7.5 IPA refinement остается **valuable** for its actual scope:
sibling-field cache survival. But a separate marker
`[M-123.4.4-codegen-fluent-chain-root-temp]` is spawned для the
codegen-level fix to WriteBuffer chain duplicate.

## 3. Изменения

### 3.1 `compiler-codegen/src/field_cache.rs`

**New helper (~12 LOC):**

```rust
fn call_recv_self_field(obj: &Expr) -> Option<&str> {
    if let ExprKind::Member { obj: inner, name } = &obj.kind {
        if matches!(inner.kind, ExprKind::SelfAccess) {
            return Some(name.as_str());
        }
    }
    None
}
```

**Refined `expr_contains_invalidating_call_for`** (~10 LOC delta):

```rust
} else if let Some(recv_field) = call_recv_self_field(obj) {
    // V7.5: @F.method() — sibling-field cache safe.
    if fname == recv_field {
        true  // conservative для own field
    } else {
        false  // sibling-safe
    }
} else {
    true  // var.method() / chain — conservative
}
```

## 5. Не в скоупе

- **Same-field refinement** — `@F.method()` invalidation of `@F`
  itself remains conservative. V7.6 (future) с TypeDecl integration
  could distinguish reference-vs-value types и relax further.
- **Chain receivers** (`@a.b.method()`) — V7.5 deliberately scoped к
  direct `@F.method()`. V7.7 (future) extension.
- **WriteBuffer chain duplicate** — codegen issue, separate marker
  `[M-123.4.4-codegen-fluent-chain-root-temp]`.
