# Plan 85.4 — Полное тестовое покрытие протоколов сравнения и отображения

> **Статус:** ✅ ЗАКРЫТ 2026-05-22 (Ф.1–Ф.3)
> **Приоритет:** P1 | **Оценка:** ~1 день
> **Родительский план:** [Plan 85](85-builtin-types-test-coverage.md)
> **Зависимости:** нет (standalone); `Display` требует `StringBuilder` (Plan 85.1 опционально первым)
>
> **Итог:** каталог `nova_tests/protocols/comparison/` — 7 файлов
> (6 позитивных + 1 негативный). Hashable / Equatable / Comparable /
> Display + Ordering + generic-bounds.
>
> **Найдены и исправлены 3 codegen-бага:**
> 1. `char.hash()` не находил builtin (`nova_char` отсутствовал в
>    `prim_builtin_method`) → mis-dispatch на user-метод `hash` →
>    **segfault** (nova_char передавался как receiver-pointer).
> 2. `==` / `!=` на типе с user-defined `@eq` эмитил `->tag`-сравнение
>    (D46 dispatch на `@eq` игнорировался) → invalid C для record-типов.
> 3. `apply_type_subst_to_ref` возвращал `void*` для concrete
>    non-primitive return-типа (`-> Ordering`) → call-site mis-typing.
>
> **Корректировки синтаксиса плана:** примеры были в Rust-стиле
> (`impl Hashable for Color { fn hash }`, `;` в полях record) — в Nova
> протоколы структурные, методы `fn T @hash()`, поля через запятую.
> Multi-bound — не `T A + B`, а именованный/анонимный protocol;
> bootstrap-парсер поддерживает named (анонимный inline в bound-позиции
> пока нет).

## Протоколы (std/prelude/protocols.nv)

```nova
type Hashable protocol {
    hash() -> u64
}

type Equatable protocol {
    eq(other Self) -> bool
}

type Comparable protocol {
    cmp(other Self) -> Ordering   // Ordering = Less | Equal | Greater
}

type Display protocol {
    fmt(sb StringBuilder) -> ()
}
```

## Примеры тестов

```nova
// hashable.nv
type Color { r u8; g u8; b u8 }
impl Hashable for Color {
    fn hash() -> u64 {
        mut h u64 = 14695981039346656037 as u64
        h = h ^ (@r as u64); h = h * 1099511628211 as u64
        h = h ^ (@g as u64); h = h * 1099511628211 as u64
        h = h ^ (@b as u64); h = h * 1099511628211 as u64
        h
    }
}

test "Color.hash determinism" {
    ro c = Color { r: 255 as u8, g: 0 as u8, b: 128 as u8 }
    assert(c.hash() == c.hash())
}

test "different Colors have different hash" {
    ro red = Color { r: 255 as u8, g: 0 as u8, b: 0 as u8 }
    ro blue = Color { r: 0 as u8, g: 0 as u8, b: 255 as u8 }
    assert(red.hash() != blue.hash())
}

// comparable.nv
type Priority { level int }
impl Comparable for Priority {
    fn cmp(other Priority) -> Ordering {
        if @level < other.level { Less }
        else if @level == other.level { Equal }
        else { Greater }
    }
}

test "Priority Less" {
    ro a = Priority { level: 1 }
    ro b = Priority { level: 5 }
    match a.cmp(b) {
        Less => assert(true)
        _ => assert(false)
    }
}

// display.nv
type Point { x int; y int }
impl Display for Point {
    fn fmt(sb StringBuilder) -> () {
        sb.append("(")
        sb.append(str.from(@x))
        sb.append(", ")
        sb.append(str.from(@y))
        sb.append(")")
    }
}

test "Point Display" {
    ro p = Point { x: 3, y: 4 }
    ro sb = StringBuilder.new()
    p.fmt(sb)
    assert(sb.into() == "(3, 4)")
}
```
