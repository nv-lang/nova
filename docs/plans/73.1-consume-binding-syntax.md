## Зачем

Plan 73 (D131) ввёл `consume` keyword **на receivers и params**:
```nova
fn StringBuilder consume @into() -> str       // consume receiver
fn drain(consume t Token) -> int               // consume param
```

Но **на let-binding** consume-обязательство **невидимо**:
```nova
let sb = StringBuilder.new()    // ← неявно: sb имеет consume-obligation
sb.into()                        // ← consume happens silently
```

Это создаёт **3 проблемы:**

1. **Невидимость ownership** на binding site — reviewer не видит что переменная
   будет потреблена. В Rust `let x = ...` тоже implicit move/copy, но borrow
   checker делает это безопасным через lifetimes. Nova lifetime-free → нужна
   syntactic visibility.

2. **Dangling view problem.** Сейчас можно написать:
   ```nova
   let sb = StringBuilder.new()
   let twin = sb           // ← view-borrow? alias? move?
   sb.into()               // ← sb потреблён
   twin.append("late")     // ← dangling — UB
   ```
   Текущий type-checker (Plan 73 D131 + Plan 100.3 D157) частично ловит это,
   но семантика `let twin = sb` неоднозначна.

3. **Inconsistency с D157 view-default на params.** D157 говорит: «не-consume
   param = view-borrow». Но `let X = consume_var` внутри тела — что? Move?
   Alias? Borrow без lifetime?

## Цель

Сделать ownership/borrow **explicit на каждом binding site** через:
1. **`consume X = expr`** — ownership-binding, X владеет
2. **`let X = expr`** — value-binding, разрешён ТОЛЬКО для не-consume-обязательных RHS
3. **Запретить view-binding внутри тела функции** — view существует ТОЛЬКО как
   function param (D157 carry-over, расширенный на reject в-теле)

## Правила (полный сет)

### Правило 1 — `consume` keyword обязателен на binding ownership-instance

`consume X = expr` требуется когда `expr` возвращает **consume-обязательную**
инстанс. Без keyword'а → ошибка.

```nova
type Token consume { val int }
fn Token.new(v int) -> Token => { val: v }

// ❌ ОШИБКА E_CONSUME_KEYWORD_MISSING
let tok = Token.new(7)

// ✅ OK
consume tok = Token.new(7)
```

**Когда RHS считается consume-обязательным:**
- Constructor consume-type'а (D133): `Token.new(...)`
- Функция возвращающая consume-type: `fn open_file() -> File consume`
- Member access consume-поля: `let x = self.consume_field` (если self — consume-aware)
- Возврат consume-метода: TBD — обычно methods consume self не возвращают instance, но edge case

**Когда RHS НЕ consume-обязательный (regular `let`):**
- Primitive: `let n = 42`
- String literal: `let s = "hi"`
- Regular record: `let r = Point { x: 1, y: 2 }` (если Point не consume)
- Non-consume method call: `let len = sb.len()` (len returns int)
- View-borrow result (внутри fn): не возникает потому что views forbidden in body
- View-borrow param (внутри fn получивший view): `let view_param_result = ...`

### Правило 2 — view-binding в теле fn запрещён

```nova
consume sb = StringBuilder.new()
sb.append("hi")

// ❌ ОШИБКА E_VIEW_BINDING_FORBIDDEN
let twin = sb

// ✅ OK — move ownership
consume twin = sb
// sb теперь dead; twin владеет

// ✅ OK — передать как view через function
fn read_len(view T) -> int => view.len()
let n = read_len(sb)  // sb остаётся Live; read_len получает view-param
```

**Error message (E_VIEW_BINDING_FORBIDDEN):**
```
error[E_VIEW_BINDING_FORBIDDEN]: view-binding на consume-обязательную переменную
запрещён внутри тела функции.
  --> file.nv:42:5
   |
42 |     let twin = sb
   |     ^^^^^^^^^^^^^
   |
   = note: view-bindings разрешены ТОЛЬКО как function-параметры (D157).
   = note: dangling-view risk: если `sb` потом потреблён, `twin` указывает в "никуда".
   = suggestion: для transfer ownership используй:
                 consume twin = sb
   = suggestion: для view используй function call:
                 fn use_view(view T) -> R => …
                 let r = use_view(sb)  // sb остаётся Live
```

### Правило 3 — `consume X = consume_var` = move

```nova
consume sb = StringBuilder.new()
consume sb2 = sb        // ← move: sb dead, sb2 owns
// sb.append("late")    // ← ERROR D131-use-after-consume
```

### Правило 4 — `consume mut X = expr` запрещён (уже)

Существующий reject в [parser/mod.rs:3311](compiler-codegen/src/parser/mod.rs#L3311) сохраняется.

### Правило 5 — view как параметр сохраняет lifetime caller'а

```nova
fn read_len(view sb T) -> int => sb.len()

fn main() {
    consume sb = StringBuilder.new()
    let n = read_len(sb)      // view-borrow на duration of call
    sb.append("more")          // OK — sb всё ещё Live (view вернулся)
    consume v = sb.into()      // OK — consume в конце
}
```

Safe by construction: view живёт только время вызова; owner (`sb`) переживает
вызов by stack semantics.

### Правило 6 — consume obligation в-scope check (carry from D131)

Если переменная объявлена `consume X = …`, она ОБЯЗАНА быть consumed до конца
scope'а. Это уже D131; здесь только syntactically более visible.

```nova
fn must_consume() {
    consume tok = Token.new(7)
    // ❌ ОШИБКА D131-forgot-to-consume — tok не потреблён до конца fn
}
```

## Scope

### In scope

- **Type-checker enforcement** в `compiler-codegen/src/types/mod.rs`:
  - Trackingлос consume-obligated RHS expressions
  - При `let X = consume_obligated_expr` → emit `E_CONSUME_KEYWORD_MISSING` с
    machine-applicable suggestion (`consume X = ...`)
  - При `let X = consume_var` → emit `E_VIEW_BINDING_FORBIDDEN`
  - При `consume X = expr` где expr НЕ consume-obligated → emit warning
    `W_CONSUME_KEYWORD_UNNECESSARY` (optional, Ф.6)
- **Error codes** (`compiler-codegen/src/diag.rs`):
  - `E_CONSUME_KEYWORD_MISSING`
  - `E_VIEW_BINDING_FORBIDDEN`
  - `W_CONSUME_KEYWORD_UNNECESSARY` (optional)
- **Spec amendment** D131 в `spec/decisions/05-memory.md` (или новый D175)
- **Тестовое покрытие** — миграция всех plan73/ тестов + добавление neg-tests
  для новых ошибок
- **Stdlib sweep** — все `let X = …` где X consume-obligated → `consume X = …`
  - Особенно sync.nv (после Plan 103.9 — там много consume guards)
  - stdlib `with_lock(fn)`, `with_permit(fn)` helpers
- **plan100_/plan103_9 tests sweep** — migrate `let` → `consume` где применимо

### Out of scope

- **Полная linear-type semantic** (must-consume = ровно 1 раз) — это Plan 80
- **Cross-function inference** — пользователь annot'ит каждое binding
- **Auto-migration tool** (`nova consume-bind-migrate`) — Plan 100.7 territory;
  можно extend существующий `nova consume-migrate`
- **Closure capture rules** — view-by-default closure capture (D157) сохраняется;
  если closure capture'ит consume-obligated var, рассматривается как consume-closure
  (Plan 100.3)

## Industry comparison

| Аспект | Nova 73.1 | Rust | Go | TypeScript | Swift |
|---|---|---|---|---|---|
| Explicit ownership на binding | ✅ `consume X = …` | ⚠ implicit move (`let x = y`) | ❌ GC | ❌ GC | ⚠ implicit move на `consuming func` |
| View-binding в теле | ❌ запрещён (no lifetime) | ✅ `&x` + lifetime | n/a | n/a | ✅ borrow с lifetime |
| Lifetime аннотации | ❌ не требуются | ✅ `<'a>` | n/a | n/a | partial (~implicit) |
| Dangling view возможен | ❌ by construction | ⚠ требует borrow-checker | n/a | n/a | ⚠ Swift exclusivity |
| Visible move on assignment | ✅ keyword | ❌ silent | n/a | n/a | ⚠ implicit |

**Nova edge:**
- Visible ownership transfer на каждом binding (unique)
- Zero lifetime annotations (vs Rust)
- No dangling view by construction (vs Rust borrow-checker complexity)
- Безопасность achieved через restriction (view only as param) вместо lifetime tracking
