## Зачем

Plan 114.4 D200 (draft) декларирует associated constants — константы
прикреплённые к типу, accessible через namespace `Type.NAME`, без места
в instance layout. Java `static final`, Rust `impl T { const X }`,
Kotlin `companion const val`.

**Use cases:**
- Version numbers, protocol identifiers: `Config.VERSION`,
  `Protocol.MAGIC_BYTES`.
- Capacity/size limits: `Buffer.DEFAULT_CAPACITY`.
- Math constants: `Circle.PI`, `Complex.UNIT_IMAGINARY`.
- Error/status codes: `Response.OK_STATUS`.
- Per-mono sizes для generic containers: `Box[int].SIZE`,
  `Pair[T,U].TOTAL`.

**Что сейчас (post-Plan-114.4):**
- `const X = expr` валиден только на module-level + scope-local.
- Record/sum/generic-type assoc const — не существует.
- Workaround — module-level `const TYPE_X_MAX = …`, ломает namespacing.

**Что хочется (Plan 114.4.1 close):**
- `type Config { const VERSION int = 2; name str }` — assoc const.
- `Config.VERSION` namespace access (2 без instance).
- `instance.VERSION` → `E_CONST_INSTANCE_ACCESS`.
- Sum-type assoc const: `type Status = A | B { const VERSION = 2 }`.
- Generic T-independent: `type Box[T] { const TAG = 0 }`.
- Generic T-dependent: `type Box[T] { const SIZE = sizeof(T) }` с
  per-monomorphization symbol emit (`Box_int_SIZE`, `Box_str_SIZE`).

## Дизайн

Полный дизайн — в [D200](../../spec/decisions/02-types.md#d200)
(draft, добавлен в Plan 114.4 commit `bbe95581a1d`).

### Ф.1 — Parser + checker (non-generic + T-independent)

#### Parser

- `parse_record_fields` расширить: принимать `const NAME TYPE = expr`
  внутри `type X { ... }` body. Wrap как **новый** AST: `RecordField`
  получает enum-tag (или отдельный `RecordAssocConst { name, ty, value }`).
- Modifier-conflicts на parse-time:
  - `mut const` / `const mut` → `E_CONST_MUT_CONFLICT`.
  - `ro const` / `const ro` → `E_CONST_RO_REDUNDANT`.
  - `consume const` → `E_CONST_CONSUME_CONFLICT`.
- `export const` field — accepted (publicly accessible).

#### Checker

- Constexpr-eligibility — reuse `check_const_constexpr` (Plan 114.4 Ф.1).
- `Type.NAME` namespace access — новый path в name-resolver:
  - При `parse_expr` встретили `Path(["Type", "NAME"])` (или `Member`
    on `Type` identifier) — резолвить через type's const-table.
  - Type's const-table — новый field в `TypeDecl` (или `ModuleEnv.types`
    extended).
- `instance.NAME` → `E_CONST_INSTANCE_ACCESS` с hint «use `Type.NAME`».
- Record literal без указания assoc const fields:
  - Указание `Config { VERSION: 5, … }` → `E_CONST_FIELD_IN_LITERAL`.
- `sizeof(Type)` НЕ включает assoc const (zero storage).

#### Codegen non-generic + T-independent

- Emit top-level `static const T Type_FieldName = literal;` в .rodata
  (вне struct definition).
- `Type.FieldName` resolution в Nova → C-symbol `Type_FieldName`.
- `export const` field — public C-symbol visibility.
- Record struct definition НЕ содержит assoc-const-fields (zero storage).

### Ф.2 — Sum-type assoc const

- Parser: расширить sum-type body парсинг на `const` field.
- Checker: assoc const на sum-type-level (не per-variant).
  - Per-variant const (`Active { const X = 1 }`) — out-of-scope V1,
    followup `[M-114.4.1-per-variant-const]`.
- Codegen: emit `static const T SumType_FieldName = literal;` —
  identical к record-field.
- Resolution `SumType.NAME` — same path в name-resolver.

### Ф.3 — Generic T-dependent per-monomorphization codegen (safety-hatched)

> **Safety hatch:** decision point в конце Ф.3.2 smoke на minimal
> example. Если per-mono integration требует deep refactor of mono
> pipeline → extract в Plan 114.4.1.1, Plan 114.4.1 шипится с Ф.1+Ф.2.

#### T-independent (RHS не reference'ит generic params)

```nova
type Box[T] {
    const TAG int = 0
    value T
}
Box.TAG                                  // ✓ emit single symbol Box_TAG
```

Single symbol emit (как non-generic). Checker classifies — если RHS не
содержит generic param refs, treat as non-generic.

#### T-dependent (RHS reference'ит generic param)

```nova
type Box[T] {
    const SIZE int = sizeof(T)
    value T
}
Box[int].SIZE                            // ✓ 8 — per-mono Box_int_SIZE
Box[str].SIZE                            // ✓ 16 — per-mono Box_str_SIZE
Box.SIZE                                 // ✗ E_GENERIC_CONST_REQUIRES_INSTANTIATION

type Pair[T, U] {
    const TOTAL int = sizeof(T) + sizeof(U)
    first T
    second U
}
Pair[int, str].TOTAL                     // ✓ per-(T,U)-mono Pair_int_str_TOTAL
```

**Reuse Plan 70.5 monomorphization pipeline:**
- На каждой monomorphization `Box[int]` (создаётся при first reference) —
  emit assoc const RHS evaluated с T=int. Result → `static const int
  Box_int_SIZE = 8;`.
- Naming convention: `Type_<mono_args>_FieldName` (coherent с existing
  generic-fn mono naming).
- `Box.SIZE` без instantiation → `E_GENERIC_CONST_REQUIRES_INSTANTIATION`
  с hint «depends on T; use `Box[T].SIZE`».

**Allowed в T-dependent RHS (V1):**
- `sizeof(T)` где T — generic param.
- Арифметика над `sizeof(T_i)` и literals.
- Ссылки на T-independent `const` через `Type.CONST`.

**НЕ allowed в V1** (followups):
- `T.METHOD()` calls — runtime, `[M-114.4.1-t-method-in-const]`.
- `const fn` calls с generic args — `[M-114.4.1-generic-const-fn]`
  (требует generic const fn monomorphization).
- Recursive type references (`Tree[T] { const X = sizeof(Tree[T]) }`) →
  `E_GENERIC_CONST_CYCLE`.

**Cyclic detection:**
- При each per-mono evaluation, track parent chain.
- Если encountering already-in-progress mono — `E_GENERIC_CONST_CYCLE`.

## Tests (T2 series carried-over from Plan 114.4)

### Positive

- **T2.1** Assoc const: `type Config { const VERSION int = 2; name str }`;
  `Config.VERSION == 2`; emit `static const int Config_VERSION = 2;`.
- **T2.2** `sizeof(Config) == sizeof(name_only)` (assoc const не в layout).
- **T2.3** Record literal без assoc fields type-checks:
  `ro c = Config { name: "alice" }`.
- **T2.4** `export const VERSION int = 2` cross-module access.
- **T2.5** Sum-type: `type Status = A | B { const VERSION = 2 }`;
  `Status.VERSION == 2`; emit single `Status_VERSION`.
- **T2.6** Generic T-independent: `type Box[T] { const TAG = 0 }`;
  `Box.TAG == 0`; emit single `Box_TAG`.
- **T2.7** Generic T-dependent: `type Box[T] { const SIZE = sizeof(T) }`;
  `Box[int].SIZE == 8`; `Box[str].SIZE == 16`; per-mono `Box_int_SIZE`,
  `Box_str_SIZE`.
- **T2.8** Generic T-dependent с арифметикой: `const HEADER = sizeof(T)
  + 8`; `Box[int].HEADER == 16`.
- **T2.9** Cross-T-param: `type Pair[T,U] { const TOTAL = sizeof(T)
  + sizeof(U) }`; `Pair[int,str].TOTAL` per-(T,U)-mono.
- **T2.10** Doc-gen `nova doc` renders type page с "Associated
  constants" section.

### Negative (EXPECT_COMPILE_ERROR)

- **T2-NEG.1** `c.VERSION` → `E_CONST_INSTANCE_ACCESS`.
- **T2-NEG.2** `Config { VERSION: 5, … }` → `E_CONST_FIELD_IN_LITERAL`.
- **T2-NEG.3** `type T { mut const X = 5 }` → `E_CONST_MUT_CONFLICT`.
- **T2-NEG.4** `type T { ro const X = 5 }` → `E_CONST_RO_REDUNDANT`.
- **T2-NEG.5** `Box.SIZE` (T-dependent без instantiation) →
  `E_GENERIC_CONST_REQUIRES_INSTANTIATION`.
- **T2-NEG.6** Cyclic `type Tree[T] { const SIZE = sizeof(T) +
  sizeof(Tree[T]) }` → `E_GENERIC_CONST_CYCLE`.
- **T2-NEG.7** Non-constexpr RHS в assoc const → `E_CONST_NOT_CONSTEXPR`
  (reuse Plan 114.4 Ф.1 check).
