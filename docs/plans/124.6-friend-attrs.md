## 1. Мотивация

D220 устанавливает **strict type-method-only** scope для priv-field access —
строже, чем в Go (package-wide), Rust (`pub(crate)` crate-wide) или Kotlin
(`internal` module-wide). Это необходимое свойство инварианта, но в ряде
production scenarios нужна **controlled relaxation**:

1. **Unit-тесты.** Тест должен верифицировать внутреннее состояние (balance,
   cursor position, cache size) без публикации getter'а в public API.
2. **Sibling helper types.** `Account` + `Bank` audit utilities должны
   координировать доступ без избыточного boilerplate.

Обе потребности покрываются двумя **явными, syntactically-marked opt-in
escape hatches** (D224). Никаких implicit relaxation.

## 2. Design (D224 amended, 2026-06-16)

> AMENDED 2026-06-16: дизайн изменён по сравнению с оригиналом 2026-06-02.
> Полная нормативная спецификация — в D224 (spec/decisions/02-types.md §D224).

**Что изменилось:** оригинальный дизайн имел `#test_access(T)` на `fn`-уровне
и `#visible_to(Y)` на `field`-уровне. Новый дизайн переходит на три механизма:
(1) implicit same-module, (2) `#[test_access]` на test-block, (3) `#[pub_to]` на TypeDecl.

### 2.1 Implicit same-module test access (Rule 2, D224 §2)

Любой `test "..." { }` блок в том же **модуле** (папке), что и тип T,
автоматически имеет priv(type) access к T. Атрибут не нужен.

"Тот же модуль" = одна папка (Nova module = folder = все .nv файлы как peers).
Реализован через `TestBlockGuard` (`in_test_block = true`) + сравнение путей модулей.

### 2.2 `#[test_access(TypeA, TypeB)]` — per-test-block cross-module opt-in

Attribute перед `test "..." { }` блоком: весь блок получает priv-field access к
перечисленным типам из другого модуля.

```nova
#[test_access(Account)]
test "cross module balance check" {
    ro acc = Account { balance: 0.0 }
    assert(acc.balance == 0.0)     // OK — test_access grant
}
```

Scope: только тело помеченного test-блока. Парсер: `parse_test_access_attr()` в
`parse_item()` → `TestDecl.test_access: Vec<String>`.

### 2.3 `#[pub_to(TypeY, TypeZ)]` — type-level friend declaration

Attribute на TypeDecl: методы перечисленных типов получают priv access ко
**всем** priv-полям данного типа.

```nova
#[pub_to(Bank)]
export type Account {
    priv(type) mut balance f64
}

export fn Bank @audit_account(a Account) -> f64 =>
    a.balance        // OK — Bank в pub_to Account
```

Тип-уровень (не per-field): все priv поля Account открыты для Bank.
Парсер: `parse_type_attrs()` → 4-tuple включая `pub_to: Vec<String>` →
`TypeDecl.pub_to: Vec<String>`. Checker: `type_pub_to: HashMap<String, Vec<String>>`.

### 2.4 Combined access predicate (4 rules, D224 §4)

priv-field access разрешён, если хотя бы одно из:

1. `current_recv_type == tname` — canonical type-method scope (D220).
2. `in_test_block && same_module(current_module, type_defining_module[tname])` — implicit same-module.
3a. `in_test_block && tname ∈ test_block_test_access` — `#[test_access]` cross-module.
3b. `tname ∈ current_fn_test_access` — legacy fn-level (preserved).
4. `current_recv_type ∈ type_pub_to[tname]` — `#[pub_to]` friend grant.

Реализация: `priv_access_allowed_base` в `TypeCheckCtx`.
