## 1. Контекст

V7.1 §4 documented thread-local IPA plumbing as an interim measure:
`LICM_WRITE_SETS` / `PURE_IPA_CTX` / `CHAIN_IPA_CTX` thread_local
slots set by `*_with_ipa` wrappers, snapshotted by inner barrier
helpers. Simpler patch but data flow opaque + incompatible с future
multi-threaded compilation.

---

## 2. Изменения

### 2.1 Type signature additions

Functions updated to accept `ipa: Option<IpaCtx<'_>>` param:

- `licm_fn_impl` (new; `licm_fn` retains old 3-arg sig, delegates с `None`)
- `licm_block` / `licm_stmt` / `licm_expr` / `process_loop`
- `collect_loop_eligible_fields`
- `pure_cache_fn_impl` (new; `pure_cache_fn` delegates с `None`)
- `chain_cache_fn_impl` (new; `chain_cache_fn` delegates с `None`)

### 2.2 Wrapper restructure

`licm_fn_with_ipa` / `pure_cache_fn_with_ipa` / `chain_cache_fn_with_ipa`
теперь:
1. Call `recv_type_for_ipa(f, cfg, write_sets) -> Option<String>` —
   returns None when IPA disabled / write_sets empty / no receiver.
2. None-branch → call `*_impl(f, ..., None)` (V1 fallback).
3. Some-branch → construct `IpaCtx<'_>` borrowing the local
   `recv_type: String` + write_sets/read_sets, call `*_impl(f, ...,
   Some(ipa))`.

The local `recv_type: String` outlives the `*_impl` call, decoupling
from `&mut f` aliasing concerns.

### 2.3 thread_local!{} block removed

`LICM_WRITE_SETS` / `PURE_IPA_CTX` / `CHAIN_IPA_CTX` deleted from
`field_cache.rs`. All `*.with(|x| x.borrow().clone())` snapshot calls
replaced с direct parameter access.

### 2.4 Internal helpers updated

`collect_loop_eligible_fields` body changes:
```rust
// before (V7.1):
let licm_ipa = LICM_WRITE_SETS.with(|ws| ws.borrow().clone());
// ...
match &licm_ipa { Some((rt, ws)) => /* build IpaCtx */, None => ... }

// after (V7.2):
match ipa {
    Some(ipa) => block_contains_invalidating_call_for(body, fname, Some(ipa)),
    None => block_contains_call(body),
}
```

`pure_cache_fn_impl` / `chain_cache_fn_impl` preserve their local
`pure_ipa` / `chain_ipa` tuple variables for minimal churn — they
populate them from the explicit `ipa` parameter (cloning the
relevant HashMap once), rather than from a thread-local snapshot.

## 4. Risk assessment

`recv_type` String clone per `*_with_ipa` invocation = one heap
allocation per method. ~50 method per typical module = ≤50 short
allocations — negligible.

Aliasing risk avoided by holding `recv_type: String` in wrapper's
stack frame between IpaCtx construction и `*_impl(&mut f)` call.
Rust borrow checker validates.

---

## 5. Future readiness

Multi-threaded compilation: explicit parameter threading is
race-free by construction (no shared mutable state across thread
boundaries). With thread_local, parallel module compilation would
risk RefCell panic or panic-poisoned state across thread pool reuse.
