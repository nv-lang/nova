## Design refinement 2026-06-03 (user discussion finalization)

### NO Slice[T] type — use existing `[]T` design

**D144 single-type philosophy:** "`[]T` — один тип для owner и view".
`arr[a..b]` уже возвращает `[]T` view (24-byte header `NovaArray_T*` с
adjusted `data + from`, `len = cap = to - from`). Plan 96 enforces.

**Conflict:** introducing `type Slice[T]` separate type contradicts D144.

**Resolution:** **add 1 method to `[]T`** для FFI access к internal pointer:

```nova
// std/collections/vec.nv — extend built-in []T methods
fn []T @as_ptr() -> *T                  // NEW — ro view of internal data ptr (FFI read)
fn []T mut @as_mut_ptr() -> *mut T      // NEW — mut view (FFI write, requires mut binding)
fn []T @len() -> int                    // existing (D32)
fn []T @cap() -> int                    // existing (D32)
```

**Naming `@as_ptr` / `@as_mut_ptr`** (not bare `@ptr` / `@ptr_mut`):
- Avoid visual confusion с `ptr` primitive type (Plan 115 D214)
- Match `as_X` view convention (zero-copy view, lifetime-aware)
- Rust precedent: `Vec::as_ptr()` / `Vec::as_mut_ptr()`

**FFI usage** (canonical pattern read(fd, buf, n)):
```nova
external fn read(fd int, buf *mut u8, n usize) -> isize

unsafe {
    ro view = buffer[0..1024]                                 // []u8 view (D144)
    ro bytes_read = read(fd, view.as_ptr() as *mut u8, view.len() as usize)
}
```

Никакого нового slice-type. Zero new types в Plan 118.2 core scope.

### `[]T @as_ptr()` / `@as_mut_ptr()` semantics

- Returns internal data pointer (potentially offset for view subslice per D144)
- `@as_ptr() -> *T` (≡ `*ro T` per D216 §1) — readonly access
- `@as_mut_ptr() -> *mut T` — mutable access (requires `mut` binding на receiver)

**Decision (V1):** Separate methods для clear intent:
  ```nova
  fn []T @as_ptr() -> *T                  // ro access (read FFI)
  fn []T mut @as_mut_ptr() -> *mut T      // mut access (write FFI — requires mut binding)
  ```

**`@as_ptr` itself unsafe?** NO — pointer dereference is unsafe, но getting the
address is read-of-header-field (no UB). Caller wraps subsequent use в
`unsafe { ... }` automatically per E_UNSAFE_REQUIRED.

### NO ManuallyDrop — not needed в Nova

**Rust ManuallyDrop's role:** suppress automatic Drop on a type that has `Drop` impl.

**Nova doesn't have implicit Drop:**
- Records (heap, GC-managed) — no destructor; GC handles
- `consume X = init() { body }` scope — **opt-in** через Consumable protocol;
  if you don't use consume scope, no on_exit runs
- Plain binding `ro h = init()` — no scope-bound cleanup

**Skip-destructor in Nova = just don't use consume scope:**
```nova
// Nova "ManuallyDrop" equivalent — just regular binding:
unsafe {
    ro handle = open_file()         // raw handle, NO Consumable on_exit registered
    c_takes_ownership(handle)        // C side now owns
    // handle dropped из scope, no cleanup runs — ownership truly transferred
}
```

**Verdict:** `ManuallyDrop[T]` type не needed. Plan 118.2 scope reduced.

### MaybeUninit[T] — confirmed (only new type в Plan 118.2)

Type-safe uninitialized storage для performance-critical bulk-init patterns:

```nova
// std/ffi/maybe_uninit.nv
type MaybeUninit[T] priv {
    slot *unsafe T                   // raw bytes slot, NOT initialized
}

fn[T] MaybeUninit[T].uninit() -> MaybeUninit[T]      // alloc, no init (perf win)
fn[T] MaybeUninit[T].zeroed() -> MaybeUninit[T]      // alloc + memset(0)
fn[T] MaybeUninit[T] mut @write(v T) -> ()           // initialize slot

#unsafe
fn[T] MaybeUninit[T] @assume_init() -> T             // user-promise it's init
```

**Use cases:**
- Bulk-init array без zero-pre-fill: `MaybeUninit.uninit[Point](1000)`
- `read()` into uninit buffer: libc reads bytes, нам не нужен zero-init
- Avoiding default-construction cost для expensive `T`

## Зачем

После Plan 118 core у нас есть typed `*T` family. FFI to bulk buffer APIs
вынуждает pass `(*T, usize)` pair manually:

```nova
// Without slice fat-pointer:
external fn process_image(data *ro u8, len usize) -> i64
unsafe {
    process_image(buf_ptr, 1024)            // 2 args, easy to mismatch
}

// With slice fat-pointer:
external fn process_image(data *ro [u8]) -> i64
unsafe {
    process_image(slice)                    // 1 arg, type-safe
}
```

**Uninitialized memory** — typical FFI pattern (caller provides buffer для
output). Plan 118 core имеет `Option[*T] = None + init` для null-init
pattern. Но для **arrays + structs** нужен `MaybeUninit[T]`:

```nova
// Without MaybeUninit:
mut buf [u8] = [0; 1024]                    // forced zero-init (wasteful)
unsafe { read_into(buf.as_mut_ptr_unsafe(), 1024) }

// With MaybeUninit:
mut buf MaybeUninit[[u8; 1024]] = uninit()  // no zero-init
unsafe {
    read_into(buf.as_mut_ptr(), 1024)
    ro init_buf = buf.assume_init()          // claims initialization
}
```

**`ManuallyDrop[T]`** — для **ownership transfer FFI**: Nova вызывает C function
которая takes ownership (e.g., `sqlite3_bind_text(stmt, _, ptr, _, free)`); Nova
side must NOT run destructor when value goes out of scope (C will free).

```nova
unsafe {
    ro buf = String.new("hello")
    ro md = ManuallyDrop.new(buf)            // wraps; destructor skipped
    sqlite3_bind_text(stmt, 1, md.ptr(), -1, sqlite_free)
    //                              ^^^^^^^ C takes ownership; Nova не calls destructor
}
```

### Mainstream comparison

| Язык | Slice fat-pointer | Uninitialized storage | Manually-skip destructor |
|---|---|---|---|
| **Rust** | `&[T]` / `&mut [T]` (built-in) | `MaybeUninit<T>` (std::mem) | `ManuallyDrop<T>` (std::mem) |
| **Zig** | `[]T` (slice = `{ptr, len}`) | `undefined` value | wrap в `*T` (no auto-destructor anyway) |
| **C#** | `Span<T>` / `ReadOnlySpan<T>` | `stackalloc T[N]` (uninitialized) | `GC.SuppressFinalize` |
| **Swift** | `UnsafeBufferPointer<T>` | `UnsafePointer.allocate(...)` (uninit) | manual `deinitialize` |
| **D** | `T[]` (slice = `{ptr, len}`) | `void[]` uninitialized | `core.lifetime.copyEmplace` patterns |
| **Go** | `[]T` (slice = `{ptr, len, cap}`) | `make([]T, N)` (zero-init by default) | finalizers (no skip API) |
| **C** | `(T*, size_t)` pair (manual) | declared без init | NA |
| **Nova V1** (Plan 118 core) | manual `(*T, usize)` pair | `Option[*T] = None + init` (limited) | NA |
| **Nova V2** (Plan 118.2) | `*[T]` fat-pointer (ptr + len) | `MaybeUninit[T]` + `uninit()` builtin | `ManuallyDrop[T]` wrapper |

## Дизайн

### 1. Slice fat-pointer (`*[T]` family)

```nova
*[T]            // ro slice fat-pointer (default); short for *ro [T]
*ro [T]         // explicit readonly
*mut [T]        // mutable slice
*unsafe [T]     // unsafe variant (after arithmetic on inner ptr)
```

**Layout:** 2-word struct (16 bytes on 64-bit):
```c
typedef struct nova_slice_T {
    const T* ptr;
    size_t   len;
} nova_slice_T;
```

**ABI compatibility:** matches C convention `(T*, size_t)` pair — direct FFI
when struct passing supported (Sys V passes ≤16-byte structs в registers).

**Methods:**
- `(*[T]).as_ptr() -> *ro T` — raw pointer (first element)
- `(*mut [T]).as_mut_ptr() -> *mut T` — mut raw pointer
- `(*[T]).len() -> usize` — element count
- `(*[T])[i usize] -> T` — indexed read (bounds-checked in safe mode;
  unsafe-unchecked variant `.get_unchecked(i)`)
- `(*mut [T])[i usize] = v` — indexed write
- `(*[T]).sub(start usize, end usize) -> *[T]` — sub-slice (no copy)

**Construction:**

```nova
unsafe {
    // From raw ptr + len
    ro slice = slice_from_raw_parts(ptr, 1024)         // *ro [u8]

    // From []T array (in unsafe — array may relocate)
    ro arr = [1, 2, 3]
    ro slice = (&arr).as_slice_unsafe()                 // *ro [i32]
}
```

**Why typed slice (vs (*T, usize) pair):**
1. **Type safety:** compiler enforces ptr+len pair (no mismatch)
2. **Single arg:** FFI signatures cleaner
3. **Bounds tracking:** `.len()` accessible from pointer; bounds checking possible
4. **ABI compatible:** same layout as C convention (verified Ф.5 ABI snapshot)

### 2. `MaybeUninit[T]`

```nova
type MaybeUninit[T]                                     // opaque wrapper

// Construction
mut buf MaybeUninit[Account] = uninit()                 // no init
mut arr MaybeUninit[[u8; 1024]] = uninit()              // 1024 bytes uninit

// Initialization (typed write to pointer)
unsafe {
    buf.as_mut_ptr().write(Account { name: "...", age: 30 })
}

// Claim init (transition to initialized T)
ro init_buf Account = unsafe { buf.assume_init() }
```

**API:**
- `uninit[T]() -> MaybeUninit[T]` — uninitialized storage of size sizeof(T)
- `(MaybeUninit[T]).as_ptr() -> *ro T` — read pointer (UB if not init)
- `(MaybeUninit[T]).as_mut_ptr() -> *mut T` — write pointer (для init)
- `(MaybeUninit[T]) consume @assume_init() -> T` — claims init; consumes wrapper;
  returns T (unsafe — UB if not fully init)
- `(MaybeUninit[T]).write(v T) -> T` — convenience init + return inner T
- `(MaybeUninit[T]).zeroed() -> MaybeUninit[T]` — zero-init helper

**Storage:** `MaybeUninit[T]` = `T`-sized stack/heap allocation (depending
on T), NO initial value. Compiler tracks "uninit" state (no auto-deref reads
allowed).

**Use cases:**
- FFI out-params: caller provides buffer, callee fills
- Performance: avoid double-init для read-then-overwrite patterns
- Generic uninit storage: collection slots, allocator chunks

### 3. `ManuallyDrop[T]`

```nova
type ManuallyDrop[T] {
    ro value T
}

// Wrap (skip destructor on drop)
unsafe {
    ro buf = String.new("hello")
    ro md = ManuallyDrop.new(buf)            // T moved into wrapper; no drop

    sqlite_bind_text(stmt, 1, md.ptr_as_unsafe(), -1, sqlite_free)
    // C takes ownership of pointer; Nova-side md goes out of scope without
    // running String destructor (C will call sqlite_free)
}
```

**API:**
- `ManuallyDrop.new(v T) -> ManuallyDrop[T]` — wrap, suppress destructor
- `(ManuallyDrop[T]).get() -> *ro T` — pointer to inner value
- `(ManuallyDrop[T]).get_mut() -> *mut T` — mut pointer
- `(ManuallyDrop[T]) consume @drop() -> ()` — explicitly run destructor (rare)
- `(ManuallyDrop[T]) consume @into_inner() -> T` — unwrap; destructor re-armed

**Interaction with consume types (D131):** ManuallyDrop wraps consume types
по-особому — destructor может не быть available на T (e.g., wrapped raw ptr).
Detail: compiler tracks ManuallyDrop'd values отдельно от regular consume
analysis.

### 4. Array → slice coercion

```nova
ro arr = [1, 2, 3, 4, 5]                                // []i32, GC-tracked heap

// Coercion in unsafe (array buffer may relocate via GC compaction):
unsafe {
    ro slice = (&arr).as_slice_unsafe()                  // *ro [i32]
    // Use slice immediately — no GC trigger between coercion and use
    process(slice)
}
```

**Why unsafe:** GC compaction может move array buffer; slice'd pointer
становится dangling. Honor-system: user обещает no GC trigger в unsafe block
(per Plan 118 core §16).

**Diagnostic:** `(*[T]).as_ptr()` outside unsafe — `E_UNSAFE_REQUIRED`.

### 5. Slice indexing

```nova
unsafe {
    ro slice *ro [u8] = ...
    ro byte = slice[5]                                   // safe-checked read
    ro byte_uc = slice.get_unchecked(5)                  // unsafe-unchecked
}
```

**Safe (bounds-checked):** `slice[i]` — emits bounds check; panic if `i >= len`.

**Unchecked:** `.get_unchecked(i)` / `.set_unchecked(i, v)` — skips bounds
check; UB if `i >= len`. Use cases: hot loops where bounds verified externally.

### 6. Sub-slice

```nova
unsafe {
    ro slice *ro [u8] = ...
    ro sub = slice.sub(10, 20)                           // *ro [u8] of length 10
    // sub points to slice[10..20], length 10, shares storage
}
```

**Codegen:** computes `ptr + start * sizeof(T)` + `len = end - start`;
no copy.

**Bounds:** `start <= end <= slice.len()`; panic if violated.

- Opaque wrapper для uninitialized typed storage

- Methods: as_ptr, as_mut_ptr, assume_init (consume), write, zeroed
- Init-flow analysis prevents reads from uninit storage

#### §3. `ManuallyDrop[T]`

- Wrapper that skips destructor on drop
- `ManuallyDrop.new(v T) -> ManuallyDrop[T]`
- Methods: get, get_mut, drop (explicit), into_inner
- For ownership-transfer FFI patterns

**Cross-ref:** D216 (typed pointer foundations), D217 (memory primitives), D52
(newtype patterns), D131 (consume types).

## Migration impact

### Plan 115 buffer-pair FFI simplification

```nova
// Before (Plan 115 V1):
external fn process_image(data *ro u8, len usize) -> i64
unsafe { process_image(buf_ptr, 1024) }

// After (Plan 118.2):
external fn process_image(data *ro [u8]) -> i64
unsafe { process_image(slice) }
```

**LOC reduction:** typical buffer FFI loses 1 arg + bounds-tracking
manual code.

### Uninitialized buffer pattern

```nova
// Before:
mut buf [u8] = [0; 1024]                    // forced zero-init (~1µs for large bufs)
unsafe { read_into(buf.as_mut_ptr_unsafe(), 1024) }

// After:
mut buf MaybeUninit[[u8; 1024]] = uninit()
unsafe {
    read_into(buf.as_mut_ptr(), 1024)
    ro init_buf = buf.assume_init()
}
```

**Perf:** eliminates zero-init pass для large buffers (visible savings
для 1MB+ reads).

## Compiler error/warning codes

### New errors

- `E_MAYBEUNINIT_READ_BEFORE_INIT` — read MaybeUninit без prior write/zeroed
- `E_SLICE_INDEX_OUT_OF_BOUNDS` — runtime panic для `slice[i]` (i >= len)
- (existing `E_UNSAFE_REQUIRED`, `E_USE_AFTER_CONSUME` re-used)

- `W_MANUALLYDROP_PRIMITIVE_NO_OP` — `ManuallyDrop.new(primitive)` is no-op
  (primitives don't have destructors)
- `W_SLICE_BOUNDS_STATICALLY_OOB` — compile-time-detectable slice OOB access
