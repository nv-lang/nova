## 1. Контекст

V6.2 emit'нул static **cycles-saved estimate** через
`cpu_savings_estimate(report)` (heuristic: 4/40/8 cycle weights). Эта
оценка — proxy под CPU-time regression detection. Без cross-validation
против real-program wallclock невозможно судить, насколько estimate
коррелирует с реальным speedup'ом на корпусе real-world программ.

V6.2.1 закрывает gap: dedicated subcommand `nova bench field-cache`
builds каждый input twice (`NOVA_FIELD_CACHE=1` vs `=0`), runs N
samples per variant, reports median wallclock и speedup рядом с V6.2
static estimate.

## 2. Изменения

### 2.1 `nova-cli/src/bench/field_cache_wallclock.rs` (NEW, ~410 LOC)

- `FieldCacheWallclockOpts` / `WallclockEntry` / `StaticLayerBreakdown`
  structs.
- `run(opts) -> Result<i32>` — main entry: discover files →
  per-file measurement → terminal table → optional JSON + gate.
- `measure_one` — двойной subprocess `nova build` (ON + OFF), N
  interleaved samples, median computation.
- `build_one` — subprocess wrapper, explicit `NOVA_FIELD_CACHE=1|0`
  override (предотвращает leak inherited env в одну ветку).
- `run_one` — exe spawn + `Instant::now()` wallclock.
- `median` — sort + middle-index, overflow-safe average для even-sized.
- `geomean_speedup_pct` — log-domain geomean of `(1 + speedup_i/100)`,
  filters non-OK entries (Hennessy & Patterson §1.10).
- `static_estimate` — in-process reuse of V5 `analyze_module` +
  V6.2 `cpu_savings_estimate` pipeline.
- `has_fn_main` — cheap text-scan, matches `fn main(`/`fn main ` only.
- `discover_files` / `walk` — recursive .nv discovery, skips
  `_module.nv` / hidden dirs.
- `print_table` / `build_json` / `fmt_ns` / `truncate` — output
  formatters.

### 2.2 `nova-cli/src/bench/mod.rs`

- New `pub mod field_cache_wallclock;` declaration.

### 2.3 `nova-cli/src/main.rs`

  gate_regression_pp, skip_failed }` variant (~50 LOC clap args).

### 2.4 `bench/field_cache/` (NEW directory, 2 fixtures)

- `02_chain_heavy.nv` — three chain accesses `@inner.m.{x,y,z}.v`
  share `@inner.m` prefix; demonstrates V4 chain + V4.2 prefix sharing.
