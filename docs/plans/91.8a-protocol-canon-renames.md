## Мотивация

Текущие protocols в Nova несогласованы:
- `Iter[T]` — без `-able` suffix (исключение из конвенции `Equatable`/`Comparable`/`Hashable`).
- `Display` — noun-style (Rust-наследие), не `-able`.
- `Equatable.eq()` — короткое имя; `equals` явнее.
- `Comparable.cmp() -> Ordering` использует sum-type, при этом `str.compare() -> int` (D178) использует int. Два пути сравнения = D9 violation.
- Default method bodies в protocols не поддержаны → каждый implementer пишет boilerplate `equals` рядом с `compare`.

### Default bodies в protocols (новая фича)

Правило для protocol-методов:
- **Метод без тела** = abstract (тип ОБЯЗАН реализовать)
- **Метод с телом** (`=> expr` или `{...}`) = default; type-implementer может override

```nova
type Comparable protocol {
    use Equatable
    compare(other Self) -> int
    equals(other Self) -> bool => @compare(other) == 0    // default body
}
```

## Spec / D-block

### D183 — Canonical comparison protocols + default bodies

**Файл:** `spec/decisions/02-types.md` (main nova repo, branch main).

**Title:** `D183. Canonical protocols: Iterable/Equatable/Comparable/Hashable/Printable + default method bodies`

**Content:**

1. Rename rationale (`-able` convention, `int` vs `Ordering`).
2. Final canonical decls (см. выше).
3. Default method bodies в protocols:
   - правило body-присутствие = default
   - семантика: substitute `Self` = type-being-checked; emit synthetic function `Nova_T_method_X` если type не задал явно
   - override allowed (implementer can replace для perf)
4. Removal: `Ordering`, `Less`, `Equal`, `Greater` deprecated and removed.
5. Migration guide: `eq` → `equals`, `cmp` → `compare`, match Ordering → if < 0 / > 0.
6. Связь: D178 (`str.compare -> int`), D131 (fluent), D39 (`use` embed), D58 (protocol structural), D109 (split policy).
