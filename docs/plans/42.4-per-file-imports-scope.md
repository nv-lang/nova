> **Создан 2026-05-14** — вынесен из [Plan 42](42-folder-modules.md)
> sub-plan 42.4 в отдельный файл для clarity scope и multi-commit
> работы. Реализует **правило C** из production audit Plan 42.

## Контекст

Plan 42 MVP закрыл folder-modules (peers share namespace declarations).
Bootstrap implementation делает **flat merge** imports: при resolve
folder-module, все peer-файлы парсятся, items объединяются, **imports
peer'а A видны peer'у B**. Это нарушает правило C production audit:

> Peers share **declarations** (functions, types, constants), но
> **не imports**. Каждый peer-файл имеет свои `import` statements,
> видимые **только в нём**.

Сейчас:
```
admin/
├── users.nv:    import std.text.regex.{Re}      // нужен здесь
└── audit.nv:    (no Re import — но компилятор позволяет использовать Re)
```

Должно быть: `audit.nv` ссылается на `Re` → compile error
«unresolved name: did you forget `import std.text.regex.{Re}`?».

### Текущее состояние

В bootstrap **никакого** теста, демонстрирующего violation, не существует.
std/* и nova_tests/* не используют folder-modules с cross-peer
imports. Spec gap **не виден** в регрессии. 42.4 — preemptive correctness
fix перед production-scale stdlib (Plan 18 P0: networking, crypto).

## Архитектурное решение

**Additive AST refactor** (не разрушающий):

```rust
pub struct Module {
    pub name: Vec<String>,
    pub imports: Vec<Import>,    // сохраняем для backward compat
    pub items: Vec<Item>,        // flat — для codegen/interp/verify
    pub files: Vec<SourceFile>,  // NEW — per-peer attribution
    pub attrs: Vec<ModuleAttr>,
    pub span: Span,
}

pub struct SourceFile {
    pub path: PathBuf,
    pub file_id: FileId,
    pub imports: Vec<Import>,    // imports именно этого peer'а
    pub items: Vec<Item>,        // items declared HERE
    pub imported_items: Vec<Item>,  // items pulled via этого peer's imports
}
```

**Подход:**

1. **AST additive:** новое поле `Module.files` сосуществует с
   `Module.items` (flat). Codegen/interp/verify читают flat — не
   меняются (backward compat).
2. **FileId activation:** активировать `FileRegistry` (сейчас dead
   infrastructure). Каждому peer-файлу — unique FileId.
3. **Span propagation:** walker переписывает все Span'ы AST peer'а с
   unique FileId перед merge.
4. **Type-checker per-peer scope:** при walking item, используется
   FileId из item.span → lookup peer's imports → name resolution в
   peer scope (shared declarations + own imports + builtins).

Backward compat: existing test corpus continues работать. Codegen
видит flat `module.items`. Только type-checker делает stricter
name resolution.

## Phases

### Ф.1 — AST struct + FileRegistry activation

- `compiler-codegen/src/ast/mod.rs`: `SourceFile` struct,
  `Module.files: Vec<SourceFile>`.
- `compiler-codegen/src/imports.rs`: при resolve, populate
  `Module.files`. Каждый peer file → unique FileId через
  `FileRegistry.register()`.
- Single-file entry: `Module.files = vec![SourceFile { ... }]` (один elem).
- `module.items` remains flat — содержит **всё** (defined + imported)
  для backward compat.

**Acceptance:** compiles, regression PASS. No behavior change (type-
checker ещё не использует `files`).

### Ф.2 — Span walker для FileId propagation

- `compiler-codegen/src/span_walker.rs` (new): walker над AST, который
  переписывает `Span.file_id` на all nodes (Module, Item, Expr, Stmt,
  Pattern, TypeRef, Param, etc.).
- Используется в imports.rs при parsing peer file — после parse,
  walker assigns peer's FileId.

**Acceptance:** items в Module.files[i].items имеют consistent
`span.file_id == files[i].file_id`. Verified через debug assertion.

### Ф.3 — Type-checker per-peer name resolution

- `compiler-codegen/src/types/mod.rs::check_module`:
  - Build per-peer name scope: `peer_scopes: HashMap<FileId, HashSet<String>>`.
    Каждый peer scope = builtins + shared declarations (all peers' items
    by name) + this peer's imports.
  - При walking item, lookup `item.span.file_id` → resolve names
    against `peer_scopes[file_id]`.
  - Cross-peer imported name use → error «name X imported by peer
    `<other_peer.nv>`, not by this peer; add explicit import».
- Backward compat для single-file: одна запись в peer_scopes (MAIN_FILE_ID).

**Acceptance:** new tests (Ф.4) PASS, существующие 274 PASS.

## Acceptance criteria

- AST `Module.files: Vec<SourceFile>` populated correctly для folder-
  modules (N elements) и single-file (1 element).
- Each peer's `imports` доступны только в peer's items.
- Cross-peer import use → clear compile error с указанием importing peer.
- Shared declarations (functions, types, constants) видны всем peers —
  **не сломано**.
- Existing 274 PASS — без регрессий.
- Backward compat: codegen/interp/verify не затронуты (продолжают
  читать flat `module.items`).

## Risks / Trade-offs

- **Span walker depth.** AST имеет много вложенностей (Expr →
  ExprKind → recursive variants, Stmt, Pattern, TypeRef, Param,
  Contract, FrameTarget, etc.). Walker должен handle each. Risk:
  пропустить узел → wrong file_id → ложные ошибки. Mitigation:
  exhaustive match в Rust enums + tests verifying file_id propagation.
- **Backward compat для existing tests.** Single-file modules
  должны работать как раньше. Module.files = [1 elem] с MAIN_FILE_ID.
  Type-checker treats `peer_scopes[MAIN_FILE_ID]` как все imports +
  builtins → identical to current behavior.
- **Performance.** HashMap lookup на каждое expression name —
  small overhead. Bootstrap не критичен. Production-grade — может
  потребовать FlatMap / pre-built per-peer scope cached.
- **Codegen flat items dedup.** Если peer A и peer B оба импортируют
  module M, items M в module.items сейчас merged через `visited` set
  — appear once. С `Module.files`, каждый peer's `imported_items` will
  contain M's items. Flat `module.items` должен dedup. Mitigation:
  imports.rs визуально dedup'ит при flat copy.

## Что НЕ входит

- **Visibility enforcement** (`pub(crate)`, `pub(super)`) — отвергнут
  D5. `internal/` directory convention (Plan 42 правило H) уже работает.
- **Codegen refactor** — flat `module.items` сохраняется для backward
  compat. Codegen не знает о peers.
- **Per-peer attributes** (peers могут иметь свои `#forbid`) — уже
  работает через Plan 42.1 file-level scope.
- **Selective imports validation** (`import X.{A, B}` — что A, B
  exported из X) — separate Plan 35.A R26 sub-plan.
