## 1. Контекст

V1.1 outer region scanner uses single-iteration read counts. Loop
bodies execute N times — real read cost is N × syntactic_count.
V1.1 без weighting under-promotes caching for fns with hot-loop-only
reads.

V2 LICM hoists loop-invariant `@F` reads (D218) but has its own
threshold + barrier rules. V1.1 outer cache + V2 LICM hoist don't
share a cost model.

V2.1 introduces loop-iteration weighting in V1.1 outer counter using
`NOVA_FC_LOOP_ITERS` (default 8 — matches V6.2 cycle-estimate weight).

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**New env-driven weight (~10 LOC):**

```rust
fn v2_1_loop_iters_weight() -> usize {
    std::env::var("NOVA_FC_LOOP_ITERS")
        .ok().and_then(|s| s.parse::<usize>().ok())
        .filter(|&n| n > 0).unwrap_or(8)
}
```

**New weighted counter family (~230 LOC):**

- `count_field_reads_in_expr_weighted(e, fname, loop_mult)` — mirror
  of existing `count_field_reads_in_expr` with multiplier propagation.
  At `While`/`For`/`ParallelFor`/`WhileLet`/`Loop` body descent,
  multiplies `loop_mult` by `v2_1_loop_iters_weight()` (saturating).
- `count_field_reads_in_stmt_weighted` + `count_field_reads_in_block_
  weighted` — recursive helpers.
- Caller seeds `loop_mult = 1` at the top of a Block.

**`find_mut_regions_in_block` modified (~3 LOC):**

```rust
let in_stmt = count_field_reads_in_stmt_weighted(s, fname, 1);
...
let trail_reads = count_field_reads_in_expr_weighted(t, fname, 1);
```
