## Идея

Для FFI/JSON/протоколов нужны конверсии в/из UTF-16 (Windows API, JS-interop, JSON
`\uXXXX`) и доступ к сырым codepoint'ам. Дополняет байтовый слой (`from_bytes_*`/
`to_bytes`) и codepoint-линзу (`as_chars`).

## D255. UTF-16/32 interop

- **`@encode_utf16() -> []u16`** — UTF-16 code units (surrogate-пары для cp > 0xFFFF).
- **`str.from_utf16(units []u16) -> Result[str, Utf16Error]`** — decode UTF-16
  (валидирует surrogate-пары; lone surrogate → `Err`).
- **`@code_points() -> []int`** — raw int codepoints (без `char`-обёртки) — для
  низкоуровневых протоколов/таблиц.
- **surrogate-помощники:** `is_high_surrogate`/`is_low_surrogate`/`decode_surrogate_pair`
  (в `std/encoding/utf16`).

Размещение: `std/encoding/utf16.nv` (модуль `encoding.utf16`), методы `str` —
делегаты в него или прямые.

## Тесты

- **POS:** roundtrip `s.encode_utf16()` → `from_utf16()` == `s` (ASCII, BMP,
  supplementary — эмодзи 😀 = surrogate-пара); `"😀".encode_utf16().len()==2`;
  `code_points("a😀")==[0x61, 0x1F600]`.
- **NEG:** lone high/low surrogate в `from_utf16` → `Err(Utf16Error)`;
  усечённая surrogate-пара → `Err`.

## Критерии приёмки

- **A1.** `encode_utf16`/`from_utf16` — корректный roundtrip на ASCII/BMP/
  supplementary (surrogate-пары).
- **A2.** `from_utf16` отвергает lone/усечённые surrogate → `Err(Utf16Error)`
  (инвариант R-UTF8 сохраняется на выходе).
- **A3.** `code_points()` даёт корректные int-codepoint'ы (== `as_chars()` значения
  как int).
- **A4.** Размещено в `std/encoding/utf16`; `str`-методы — тонкие делегаты.
- **A5.** Полный `nova test` без новых FAIL; plan152_6-фикстуры PASS.

## Связь
