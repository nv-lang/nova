## Мотивация

Сейчас весь `str` — один файл [string.nv](../../std/runtime/string.nv) (~766 строк),
и часть buffer-building (`@trim`/`@to_lower`/`@to_upper`/`@concat`) реализована
**push-loop-копипастом** по `[]u8` в обход `StringBuilder` (исторически — из-за
import-цикла `prelude → … → string_builder`). Цель: разнести по файлам,
переиспользовать `RawMem` по максимуму, устранить копипаст через **internal-модуль**.

## Scope

Папка `std/runtime/string/` — **один модуль `runtime.string`**, размазанный по
co-equal файлам (AMEND F4; все файлы объявляют `module runtime.string`):

| Файл | Модуль | Содержимое |
|---|---|---|
| `core.nv` | `runtime.string` | `len`/`byte_len`/`char_len`/`byte_at`/`is_empty`/`as_bytes`/`to_bytes`/`to_chars`/identity (`compare`) + `from_bytes_*`/`alloc_copy`/helpers (`cp_to_char`/`validate_utf8`/`is_cont`) |
| `search.nv` | `runtime.string` | `find`/`rfind`/`contains`/`starts_with`/`ends_with`/`split`/`sub_view`/(152.2 `match_indices`/split-family) |
| `transform.nv` | `runtime.string` | `trim*`/case(ASCII)/`replace`/(152.2 `replacen`)/`pad*`/`repeat`/`concat`/`plus` |
| `parse.nv` | `runtime.string` | `parse_int`/`try_parse_int`/`ParseIntError` |
| `chars.nv` | `runtime.string` | `char_at`/(152.1 `CharsView`/`CharIndicesView` + char-курсор) |
| ~~`string_buffer.nv`~~ (D-R5: НЕ вводится) | — | **`_buffer` ≡ `Vec[u8]`** (уже RawMem-буфер, Plan 131): `@append`=`RawMem.copy` memmove + `from_bytes_unchecked_steal`. Отдельный `StrBuf` дублировал бы Vec (нарушает DRY/минимализм). `StringBuilder` уже обёртка над `{mut buf []u8}` → Ф.4 де-факто. См. [findings D-R5](152-findings.md). |

> Нет `string.nv` и нет facade (см. AMEND/F4). `import std.runtime.string.{X}` и prelude
> `export import std.runtime.string.{…}` работают без изменений — модуль `runtime.string`
> существует (как папка). `_buffer` — sibling `runtime.string_buffer` (не submodule
> папки, чтобы не попасть в публичную поверхность `runtime.string`; экспортируется для
> `string_builder`, в prelude НЕ реэкспортируется).

- **RawMem-max:** свести `alloc_copy`/`from_bytes_*` и все push-loop'ы в `_buffer` на
  прямые `RawMem.alloc`/`copy_nonoverlapping`/`fill`/`compare` (один memcpy-проход,
  меньше bounds-check'ов). str-методы строят результат через `_buffer`.
- **Cycle-break:** `string_buffer` (`#no_prelude`) зависит только от `RawMem` + ptr-ops →
  вне prelude-цикла. И str-`transform`, и `StringBuilder` строятся на `string_buffer`
  (`StringBuilder` = тонкий публичный `consume`-wrapper над `_buffer`).
- **Internal-видимость:** ключевого `internal` нет → `_`-naming + НЕ-реэкспорт в prelude
  (`runtime.string_buffer` не в prelude facade). Module-private хелперы внутри папки —
  через отсутствие `export` (видны всем co-equal файлам модуля `runtime.string`).
- **Чистка реестра (D-R2, согласовано с автором).** Удалить вестигиальные **Nova-body**
  str-записи из `compiler-codegen/src/codegen/runtime_registry.rs` (`len`/`byte_len`/
  `char_len`/`byte_at`/`is_empty`/`starts_with`/`ends_with`/`contains`/`find`/`rfind`/
  `char_at`/`trim`/`to_lower`/`to_upper`/`concat`/`plus`/`to_bytes`/`as_bytes`/`to_chars`/
  `split`/`compare`/`parse_int`/`pad_left`/`pad_right`/`repeat`/`replace`) — они НЕ драйвят
  резолв (F2: метод `get` без записи в реестре резолвится из `.nv`). **Оставить:** `eq`/
  `lt`/`le`/`gt`/`ge`/`hash` (бэкают хардкод операторов `emit_c.rs:17302` + DoS-seed). Guard
  — полный `nova test` без новых FAIL. Декомиссия операторного хардкода → 152.5a (D-R4).
- **Интерполяция/форматтер — сохранить без цикла (критично).** Интерполяция `"${e}"`/
  `"${e:?}"` (D44) десугарится codegen'ом в **StringBuilder-цепочку** (`@display(sb)` /
  `@debug(sb)`, D183/D229). `Display.@display(mut sb StringBuilder)` пишет в
  `StringBuilder` напрямую → есть взаимозависимость `str ↔ StringBuilder`
  (`StringBuilder.append(str)`/`as_str()->str` ⊥ `str.@display`/`pad` нужен builder).
  Реструктуризация обязана оставить модульный граф **ацикличным DAG**:
  ```
  _buffer (RawMem, leaf, #no_prelude)
      ↑                ↑
  str.core ────────► StringBuilder        (StringBuilder зависит от str.core.as_bytes + _buffer)
      ↑                ↑
  str.{transform,search,chars,parse}      (строят через _buffer, НЕ через StringBuilder, где можно)
      ↑
  str.@display / Display-impl типов ─► StringBuilder.append   (leaf-ward, без обратного ребра)
  ```
  Facade обязан держать публичными `StringBuilder` и `WriteBuffer` (codegen
  интерполяции и `Display`-impl'ы их резолвят). `pad_*`/interpolation продолжают
  работать; никаких новых перекрёстных импортов str↔StringBuilder сверх DAG.

## Фазы

- **Ф.0.0 — Валидация (до рефактора).** Подтвердить, что методы типа `str` могут жить
  в **нескольких** модулях (`core`+`search`+…) с type-method-privacy к `@ptr`/`@len`.
  Прецедент: Plan 139.1 (decl `core.nv` ⊥ методы `string.nv` = 2 модуля). Нужно
  подтвердить N>2 и резолв из любого. **Fallback** при ограничении резолвера: один
  `methods.nv` + internal `_buffer` (дробление слабее, цель «RawMem + без копипаста»
  достигается).
  **РЕЗУЛЬТАТ Ф.0.0 (2026-06-13): PASS** через C-codegen `nova test`. Подтверждено:
  (1) папка co-equal файлов = один модуль (`string/core.nv`+`search.nv`, оба
  `module runtime.string`, методы из обоих резолвятся); (2) str-метод в стороннем
  модуле читает priv `@ptr`/`@len`; (3) `export import` facade работает; (4) резолв из
  `.nv` без реестра (метод `get`, 0 записей, работает без импорта). F4: file+folder
  одного имени запрещён → co-equal-папка вместо facade-файла.
- **Ф.2 — Разнести методы** по co-equal файлам `string/{core,search,transform,parse,
  chars}.nv` (все `module runtime.string`); удалить `std/runtime/string.nv`. **Чисто
  механический move** — golden-тривиально (поведение не меняется). Делается ПЕРВЫМ
  после Ф.0.0 (низкий риск), отдельный коммит-чекпойнт.
- **Ф.2.5 — Чистка реестра (D-R2).** Удалить вестигиальные Nova-body str-записи из
  `runtime_registry.rs` (список — в Scope). Пересобрать компилятор. Guard: полный
  `nova test` без новых FAIL. Отдельный коммит.
- **Ф.1 — `string_buffer`.** Реализовать byte-builder на `RawMem` (модуль
  `runtime.string_buffer`, `#no_prelude`); перенести `alloc_copy`/`from_bytes_*`-логику.
- **Ф.3 — Переписать transform** на `_buffer` (убрать push-loop-копипаст). Golden
  байт-эквивалентность.
- **Ф.4 — `StringBuilder` на `_buffer`** (тонкий wrapper; убрать дублирование).
- **Ф.5 — Тесты + golden** + чистка.
- **Ф.6 — Doc + close.** `docs/strings-internals.md`; project-creation/simplifications.

> **Порядок исполнения (после Ф.0.0 ✅):** Ф.2 (move) → Ф.2.5 (registry cleanup) →
> Ф.1 (`string_buffer`) → Ф.3 (transform на buffer) → Ф.4 (StringBuilder) → Ф.5 → Ф.6.
> Каждая — коммит-чекпойнт (rate-limit устойчивость).

## Тесты

- **POS:** facade + подмодули компилируются; существующие `import
  std.runtime.string.{...}` работают; `_buffer` строит str из RawMem
  (alloc+memcpy+NUL) корректно; `StringBuilder` и str-`transform` на общем `_buffer`
  дают идентичный результат; **golden** — `trim`/`to_lower`/`to_upper`/`concat`
  байт-в-байт как до рефактора.
- **NEG:** prelude-цикл отсутствует (сборка prelude не висит/не падает); прямой
  import `_buffer` из prelude-пути запрещён конвенцией (grep-аудит).

## Критерии приёмки

- **A0.1** Папка `runtime/string/` + facade компилируется; существующие импорты целы.
- **A0.2** Ноль push-loop-копипастов alloc/grow/NUL вне `_buffer` (grep-аудит).
- **A0.3** Все buffer-аллокации `str` идут через `RawMem`-`_buffer`.
- **A0.4** Нет prelude-цикла; сборка prelude стабильна.
- **A0.5** Golden байт-эквивалентность `trim`/case/`concat` до/после.
- **A0.6** `StringBuilder` без дублирования builder-логики (на `_buffer`).
- **A0.7** **Интерполяция работает без регресса:** `"${n}"`/`"${n:?}"` компилируется
  и даёт прежний результат (golden); модульный граф ацикличен (нет цикла
  `str↔StringBuilder`); `StringBuilder`/`WriteBuffer` публичны через facade.
- **A0.8** Полный `nova test` без новых FAIL vs baseline (вкл. существующие
  `string_interpolation`-фикстуры).
