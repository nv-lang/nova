## Мотивация

Сейчас весь `str` — один файл [string.nv](../../std/runtime/string.nv) (~766 строк),
и часть buffer-building (`@trim`/`@to_lower`/`@to_upper`/`@concat`) реализована
**push-loop-копипастом** по `[]u8` в обход `StringBuilder` (исторически — из-за
import-цикла `prelude → … → string_builder`). Цель: разнести по файлам,
переиспользовать `RawMem` по максимуму, устранить копипаст через **internal-модуль**.

## Scope

Папка `std/runtime/string/` (module-per-file, как `std/collections/`):

| Файл | Модуль | Содержимое |
|---|---|---|
| `core.nv` | `runtime.string.core` | `byte_len`/`as_bytes`/`as_chars`/`slice`/`is_empty` + identity (`eq`/`hash`/`clone`/`compare`) |
| `search.nv` | `runtime.string.search` | `find`/`rfind`/`contains`/`starts_with`/`ends_with`/split-семейство/`match_indices` |
| `transform.nv` | `runtime.string.transform` | `trim*`/case(ASCII)/`replace`/`replacen`/`pad*`/`repeat`/`concat` |
| `parse.nv` | `runtime.string.parse` | `parse_int`/`try_parse_int` |
| `chars.nv` | `runtime.string.chars` | `CharsView`/`CharIndicesView` + char-курсор |
| `_buffer.nv` | `runtime.string._buffer` (**internal**, `#no_prelude`) | byte-builder на `RawMem`: `alloc`/grow/`copy_nonoverlapping`/`fill` + NUL-term + `finish()->str` |
| `string.nv` (facade) | `runtime.string` | `export import` подмодулей — сохраняет `import std.runtime.string.{...}` + prelude-wiring |

- **RawMem-max:** свести `alloc_copy`/`from_bytes_*` и все push-loop'ы в `_buffer` на
  прямые `RawMem.alloc`/`copy_nonoverlapping`/`fill`/`compare` (один memcpy-проход,
  меньше bounds-check'ов). str-методы строят результат через `_buffer`.
- **Cycle-break:** `_buffer` (`#no_prelude`) зависит только от `RawMem` + ptr-ops →
  вне prelude-цикла. И str-`transform`, и `StringBuilder` строятся на `_buffer`
  (`StringBuilder` = тонкий публичный `consume`-wrapper над `_buffer`).
- **Internal-видимость:** ключевого `internal` нет → конвенция `_`-префикс; facade
  не реэкспортит `_buffer` (см. umbrella Q-module-internal-visibility, ЗАКРЫТО).
- **Интерполяция/форматтер — сохранить без цикла (критично).** Интерполяция `"${e}"`/
  `"${e:?}"` (D44) десугарится codegen'ом в **StringBuilder-цепочку** (`@display(sb)` /
  `@debug(sb)`, D183/D229). `Display.@display(mut sb StringBuilder)` пишет в
  `StringBuilder` напрямую → есть взаимозависимость `str ↔ StringBuilder`
  (`StringBuilder.append(str)`/`as_str()->str` ⊥ `str.@display`/`pad` нужен builder).
  Реструктуризация обязана оставить модульный граф **ацикличным DAG**:
  ```
  _buffer (RawMem, leaf, #no_prelude)
      ↑                ↑
  str.core ────────► StringBuilder        (StringBuilder зависит от str.core.as_bytes + _buffer)
      ↑                ↑
  str.{transform,search,chars,parse}      (строят через _buffer, НЕ через StringBuilder, где можно)
      ↑
  str.@display / Display-impl типов ─► StringBuilder.append   (leaf-ward, без обратного ребра)
  ```
  Facade обязан держать публичными `StringBuilder` и `WriteBuffer` (codegen
  интерполяции и `Display`-impl'ы их резолвят). `pad_*`/interpolation продолжают
  работать; никаких новых перекрёстных импортов str↔StringBuilder сверх DAG.

## Фазы

- **Ф.0.0 — Валидация (до рефактора).** Подтвердить, что методы типа `str` могут жить
  в **нескольких** модулях (`core`+`search`+…) с type-method-privacy к `@ptr`/`@len`.
  Прецедент: Plan 139.1 (decl `core.nv` ⊥ методы `string.nv` = 2 модуля). Нужно
  подтвердить N>2 и резолв из любого. **Fallback** при ограничении резолвера: один
  `methods.nv` + internal `_buffer` (дробление слабее, цель «RawMem + без копипаста»
  достигается).
- **Ф.1 — `_buffer`.** Реализовать byte-builder на `RawMem`; перенести в него
  `alloc_copy`/`from_bytes_*`-логику.
- **Ф.2 — Разнести методы** по `core`/`search`/`transform`/`parse`/`chars` + facade.
- **Ф.3 — Переписать transform** на `_buffer` (убрать push-loop-копипаст).
- **Ф.4 — `StringBuilder` на `_buffer`** (тонкий wrapper; убрать дублирование).
- **Ф.5 — Тесты + golden** + чистка.
- **Ф.6 — Doc + close.** `docs/strings-internals.md`; project-creation/simplifications.

## Тесты

- **POS:** facade + подмодули компилируются; существующие `import
  std.runtime.string.{...}` работают; `_buffer` строит str из RawMem
  (alloc+memcpy+NUL) корректно; `StringBuilder` и str-`transform` на общем `_buffer`
  дают идентичный результат; **golden** — `trim`/`to_lower`/`to_upper`/`concat`
  байт-в-байт как до рефактора.
- **NEG:** prelude-цикл отсутствует (сборка prelude не висит/не падает); прямой
  import `_buffer` из prelude-пути запрещён конвенцией (grep-аудит).

## Критерии приёмки

- **A0.1** Папка `runtime/string/` + facade компилируется; существующие импорты целы.
- **A0.2** Ноль push-loop-копипастов alloc/grow/NUL вне `_buffer` (grep-аудит).
- **A0.3** Все buffer-аллокации `str` идут через `RawMem`-`_buffer`.
- **A0.4** Нет prelude-цикла; сборка prelude стабильна.
- **A0.5** Golden байт-эквивалентность `trim`/case/`concat` до/после.
- **A0.6** `StringBuilder` без дублирования builder-логики (на `_buffer`).
- **A0.7** **Интерполяция работает без регресса:** `"${n}"`/`"${n:?}"` компилируется
  и даёт прежний результат (golden); модульный граф ацикличен (нет цикла
  `str↔StringBuilder`); `StringBuilder`/`WriteBuffer` публичны через facade.
- **A0.8** Полный `nova test` без новых FAIL vs baseline (вкл. существующие
  `string_interpolation`-фикстуры).
