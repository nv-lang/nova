## 2. Design

### 2.1 CLI flags

Add to nova-cli Cmd::Test / Cmd::Build / Cmd::Compile:
- `--field-cache-threshold <N>` — D217 V1 threshold.
- `--field-cache-licm-threshold <N>`.
- `--field-cache-pure-threshold <N>`.
- `--field-cache-chain-threshold <N>`.
- `--field-cache-max <N>` — per-fn cap.
- `--field-cache-chain-depth <N>` — chain max depth.
- `--no-field-cache` (alias `--field-cache=0`).
- `--no-field-cache-licm`, etc.

Flags translate to FieldCacheConfig and override env vars.

### 2.2 Telemetry mode

`nova check --telemetry-cache` (V6):
- Walks files; analyze_module per file.
- Aggregates:
  - Total methods scanned.
  - % methods with ≥1 cache.
  - Median caches/method (over affected).
  - p99 caches/method.
  - Per-layer breakdown (D217 / D218 / D219 / chain).
- Emit JSON или human-readable summary.

### 2.3 Migration guide

`docs/migration/123-field-cache.md` — production teams:
- Default-ON rollout strategy.
- How to detect regressions (NOVA_FIELD_CACHE=0 differential).
- Per-build opt-out via CLI flag.
- Edition pinning if needed.
