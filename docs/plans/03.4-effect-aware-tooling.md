## 1. Scope

Plan 03 §4 — **Nova-уникальный угол**: менеджер пакетов языка с
эффект-системой умеет то, чего не может ни Cargo, ни Go. Этот план
реализует **автономно-кодируемый** срез:

- **effect-surface** — агрегированный effect-row публичного API пакета:
  «`foo` использует `Net`, `Fs`».
- **`nova info <pkg>`** — показывает effect-surface зависимости/пакета.
- **effect-diff** — сравнение effect-surface двух версий: minor-bump,
  добавивший `Net` в ранее чистый API, — **supply-chain red flag**.
- **capability-confined deps** — `[dependencies] X = { …, forbid = [...] }`:
  компилятор enforce'ит, что effect-surface зависимости не выходит за
  разрешённое.

**НЕ входит** (registry-gated — нужен [Plan 03.3](03-package-ecosystem-roadmap.md)):
`nova publish` / `search` / `info`-из-registry; `nova audit` (нужна
внешняя OSV-БД advisory). `nova add`/`update` — уже сделаны (03.1/03.2).

## 3. Дизайн

### 3.1. effect-surface

Effect-surface пакета = **объединение** effect-row всех **публичных**
(`export`) функций его API. Источник — `doc`-инфраструктура:
`doc::build_workspace` строит `DocTree`, у каждого модуля —
`effect_matrix: Vec<EffectMatrixEntry>` (`fn_name` + `effects`).
`strip_private` оставляет только exported. Агрегация — union эффектов
по всем модулям, сортировка, дедуп.

`nova info` показывает: имя пакета, агрегированную effect-surface,
разбивку «эффект → какие функции его вносят» (для аудита — видно
*откуда* `Net`).

### 3.2. effect-diff

Две effect-surface (A и B) → `added` / `removed` эффекты. Применения:
`nova info <path-A> --diff <path-B>`; для git-зависимости — diff между
двумя тегами (через `git_cache` checkout). Ненулевой `added` —
сигнал для ревью (CI может падать).

### 3.3. capability-confined deps

`[dependencies] foo = { git = "...", forbid = ["Net", "Fs"] }`.
При сборке: effect-surface зависимости `foo` пересекается с `forbid` →
непусто → **compile error** («зависимость `foo` использует запрещённый
`Net`»). Граница в типах, не в рантайме (сильнее Deno-permissions).
Поле `forbid` парсится в `manifest.rs` (Dependency).

## 4. Фазы

### Ф.1 — `nova info` + effect-surface ✅ (сделано)

- `compiler-codegen/src/effect_surface.rs` — `EffectSurface` +
  `compute(&DocTree)`: агрегирует `effect_matrix` всех модулей →
  объединение эффектов публичного API + разбивка «эффект → функции».
  (`diff` тоже здесь — для Ф.2.)
- CLI `nova info <PATH|dep-name>` (`cmd_info`): резолв цели —
  путь (`.nv` / каталог) либо имя зависимости из `[dependencies]`
  (path → каталог, git → checkout через `git_cache`); парс модулей →
  `doc::build_workspace` → `strip_private` → `compute`; печать
  human + `--format json`.
- Тесты: 4 unit (`effect_surface::tests` — агрегация, исключение
  приватных, pure-пакет, diff). End-to-end проверка `nova info` на
  пакете с `Net`/`Fs`-функциями.

### Ф.2 — effect-diff ✅ (сделано)

- `nova info <A> --diff <B>` — added/removed эффекты между effect-surface
  двух пакетов/версий (B — база). `info_surface` выделен в helper —
  переиспользуется для обеих целей; цель `--diff` резолвится так же
  (путь / dep-name / git-checkout).
- `--fail-on-new` — ненулевой exit-код при появлении новых эффектов
  (CI-gate против supply-chain: новая сетевая/FS-активность в
  patch/minor-релизе → падение CI).
- Вывод human + `--format json`.
- Тесты: `effect_surface::diff` unit (added/removed/идентичность) +
  end-to-end проверка `nova info --diff` (+ `--fail-on-new` exit-код).

### Ф.3 — capability-confined deps (`forbid`) ✅ (сделано)

- `manifest.rs`: поле `Dependency.forbid: Vec<String>`;
  `parse_inline_table` сделан bracket-aware (запятая внутри `[...]` не
  рвёт поля); `parse_dep_forbid` парсит `forbid = ["Net", "Fs"]`.
- `effect_surface.rs`: `surface_of_package(dir)` — effect-surface по
  каталогу; `check_forbidden(entry)` — для каждой зависимости с
  `forbid` вычисляет её effect-surface и падает, если запрещённый
  эффект присутствует (точное совпадение либо параметризованный —
  `forbid=["Fail"]` ловит `Fail[E]`). Ошибка указывает эффект и
  функции, которые его вносят.
- `cmd_build`: `check_forbidden` вызывается после `lockfile::sync` —
  нарушение валит сборку рано, до резолва импортов. Граница в типах,
  не в рантайме (сильнее Deno-permissions).
- Тесты: 4 unit (`parse_dep_forbid`, `check_forbidden` violation/ok) +
  end-to-end проверка `nova build` с `forbid`-нарушением → ошибка.

### Ф.4 — Тесты + spec + docs (~1 день)

Сквозные тесты; spec — новое D-решение про effect-surface/effect-diff/
capability-confined deps; `docs/nova-cli*` (`nova info`).

## 5. Acceptance

- `nova info <pkg>` печатает агрегированную effect-surface публичного
  API с разбивкой «эффект → функции».
- `nova info --diff` показывает added/removed эффекты между версиями.
- Зависимость с `forbid`, нарушающая границу, → детерминированная
  compile-ошибка.
- 0 регрессий.

## 6. Честная рамка

- `nova publish`/`search`/`audit` — **сознательно не здесь**: нужны
  registry (03.3) и внешняя OSV-БД. Это декомпозиция, не упрощение.
- effect-surface строится из **объявленных** эффектов публичных
  сигнатур (D28: public fn объявляет эффекты явно) — точна by
  construction, без межпроцедурного анализа.
