## 1. Scope

Центральный реестр пакетов: HTTP-протокол публикации/скачивания +
production security-модель (Plan 03 §5):

- HTTP registry-протокол (как `crates.io` index / Go module proxy).
- **Content-addressed immutable releases** — каждая версия = sha256;
  опубликованную нельзя подменить (никакого npm-`unpublish`).
- **Подпись релизов + transparency log** (Go sumdb / Sigstore-стиль) —
  скомпрометированный реестр не может тихо подменить пакет.
- registry-резолв версий через существующий резолвер (03.2).
- `registry`-конфиг (GOPROXY-стиль) — mirror / proxy / offline /
  air-gapped.

**НЕ входит:** хостинг `pkg.nv-lang.org` (сервер, CDN, UI, policy) —
[Plan 03.5](03-package-ecosystem-roadmap.md). 03.3 — **протокол и
клиент**; 03.5 — инфраструктура. `nova publish`/`search`/`info`-из-
registry — [Plan 03.4](03.4-effect-aware-tooling.md) §6 (разблокируется
этим планом).

## 2. Задел — что уже готово (03.1 / 03.2 / 03.4)

Реестр **встаёт поверх** уже построенного — почти ничего не переписывая.
Это зафиксировано здесь, чтобы знание не потерялось до старта 03.3:

- **`DependencyProvider`-трейт** (`compiler-codegen/src/resolver.rs`,
  Plan 03.2 Ф.3) — резолвер версий **не зависит** от источника.
  Registry = ещё одна реализация `DependencyProvider`:
  `available_versions` → registry-index, `dependencies` → метаданные
  пакета из registry. **Backtracking-резолвер переписывать не нужно** —
  он работает над любой «вселенной версий».
- **`nova.lock` forward-compat** (Plan 03.1 Ф.4) — парсер игнорирует
  неизвестные ключи. Registry-записи (`source = "registry"`, `hash` —
  sha256 + log-inclusion-proof) добавляются **без format-break**;
  формат `nova.lock` уже это закладывал.
- **`DepSource::Registry(version)`** (Plan 03.1 Ф.1) — registry-форма
  `dep = "1.2"` в `[dependencies]` **уже парсится**; 03.3 добавляет ей
  только резолюцию.
- **`git_cache` cache-раскладка** (Plan 03.2 Ф.2) — модель
  `db/<id>` + `co/<id>/<commit>` напрямую переносится на
  content-addressed registry-тарболы (`<sha256>` вместо `<commit>`).
  offline-режим (`NOVA_OFFLINE`) уже есть.
- **effect-surface** (`effect_surface.rs`, Plan 03.4) — реестр может
  хранить агрегированную effect-surface пакета в метаданных:
  `nova info <pkg>` и effect-diff работают **до скачивания**;
  effect-diff на уровне registry — supply-chain-сигнал из Plan 03 §4.

## 3. Открытые вопросы (на старт 03.3)

- Точный HTTP API (sparse index как у Cargo 2023+ vs git-index).
- Формат transparency log + хранение inclusion-proof в `nova.lock`.
- Схема подписи (minisign / Sigstore / собственная — с учётом
  no-crate-deps-политики bootstrap-компилятора).
- Аутентификация публикации, namespace-резервирование, typosquatting-
  policy.
