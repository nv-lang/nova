## 1. Контекст

V3.1 закрыл cached `@<m>(literal1, ...)` для scalar literals only.
Real-world fixtures part of telemetry baseline'а показали что 30%+
calls используют structural args — tuples (`(x, y)` коэффициенты),
record literals (`Color { r: 0, g: 0, b: 255 }`). V3.2 расширяет
canonical encoder.

---

## 2. Изменения

### 2.1 field_cache.rs `canonical_literal_repr`

Добавлены 2 new branches:
- `TupleLit(items)` → `T<N>{<repr1>;<repr2>;...}` (positional, recursion).
- `RecordLit { type_name, fields, inferred_map_v }` → `R<TypeName?>{name:repr;...}` с fields отсортированными alphabetically.

Rejection paths:
- `inferred_map_v.is_some()` → None (D55 map-coercion).
- Any `RecordLitField::is_spread = true` → None.
- Any field без `value` (shorthand pun) → None.
- Recursion: nested non-literal → bubbled-up None.

### 2.2 Tests

7 new unit tests в field_cache::tests:
- `v32_tuple_literal_repr_canonical`
- `v32_nested_tuple_literal_repr`
- `v32_record_literal_sorted_canonical`
- `v32_record_literal_spread_rejected`
- `v32_record_literal_pun_shorthand_rejected`
- `v32_record_literal_with_non_literal_arg_rejected`
- `v32_inferred_map_v_record_rejected`

1 new runtime fixture (plan123_3_2):
- `v32_tuple_literal_args_ok.nv` — `@sum_pair_with((1,2), 100)` cached.
