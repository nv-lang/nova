# Plan 99.2 — Contextual variant constructors

> **Sub-plan Plan 99 (master).** Bare `Some(v)`/`None`/`Ok(v)`/`Err(e)`
> (Ident-form) в expression-position учитывают `current_fn_return_ty`
> для выбора mono-репрезентации. Независимый от Plan 99.1.
> **Статус:** 📋 proposed 2026-05-23.
> **Приоритет:** P3.
> **Оценка:** ~0.5 dev-day.
> **Зависимости:** [Plan 95](95-builtin-sum-method-mono.md) ✅,
> [Plan 95.bis](95.bis-option-result-pure-methods-nova-body.md) ✅.
> **Master:** [Plan 99](99-option-result-closure-applying-methods-nova-body.md).

## Цель

`Some(v)` / `None` / `Ok(v)` / `Err(e)` в Ident-form внутри функции
с return-type `-> Option[X]` или `-> Result[Y, Z]` эмитят
**typed compound literal** для mono'd варианта,
а не legacy hardcoded `nova_make_<Type>_<Variant>` (который всегда
возвращает `NovaOpt_nova_int` / `Nova_Result*`).

Plan 99.2 нужен потому что:
- `nova_make_Option_Some(v)` в `array.h:276` hardcodes `nova_int → NovaOpt_nova_int`.
- Для `map[U] -> Option[U]` body `Some(f(v))` должен дать `NovaOpt_<U>`,
  не `NovaOpt_nova_int`.

## Где сейчас

| Variant | Path form | Bare Ident form |
|---|---|---|
| `Some(v)` | ✅ `:17251` typed compound + `register_novaopt_decl` | ❌ `:14060` → `nova_make_Option_Some` (hardcoded `NovaOpt_nova_int`) |
| `None` | ✅ `:17264` использует `current_fn_return_ty` | ⚠️ Bare `None` — нужно проверить ветку |
| `Ok(v)` | через `find_variant_compat` → `:14362-14378` | Same path |
| `Err(e)` | через `find_variant_compat` → `:14362-14378` | Same path |

Bare `Ok(v)`/`Err(e)` (`:14362-14378`) делают best-effort:
- `Ok(v)`: ok-type из arg, err-type `:= nova_str` (default).
- `Err(e)`: ok-type `:= nova_int` (default), err-type из arg.
- Resolution через `result_repr_c_type` + `result_ctor_name`.
**Defaults сломаны** для не-(int,str) Result.

## Что делаем

### Ф.1 — `Some(v)` bare ident-form (~0.15 д)

**Где:** `emit_c.rs:14340-14389` (внутри `infer_func_c_name`'s callee
для `find_variant_compat` matched Ident).

**Логика:**
```rust
if type_name == "Option" && name == "Some" {
    // Plan 99.2: contextual Option-type из current_fn_return_ty.
    let ret_ty = self.current_fn_return_ty.as_ref();
    if let Some(rt) = ret_ty {
        if let Some(sani) = rt.strip_prefix("NovaOpt_") {
            // Typed compound literal — параллель `Option.Some(v)` path form.
            // Caller emit_call после получит этот name; нужно вернуть
            // construction template. Альтернатива: специальный pre-emit.
            // ...
        }
    }
    // Fallback: legacy nova_make_Option_Some (current behavior).
}
```

**Реализация-уточнение:** `infer_func_c_name` возвращает имя C-функции
для вызова. Для compound literal нужен **inline emit**, не имя.
Альтернатива: рефакторить так чтобы `Some(v)` обрабатывался ВНЕ
`infer_func_c_name` — на уровне `emit_call` (или раньше) — и эмитил
compound literal напрямую. Параллель `:17251` (Option.Some path form
в `infer_path_call` или соответствующем месте).

**Practical approach:** добавить early-detect в `emit_call`:
```rust
// emit_call entry: Plan 99.2 — bare Some/None contextual.
if let ExprKind::Ident(name) = &func.kind {
    if (name == "Some" || name == "None") {
        if let Some(rt) = &self.current_fn_return_ty {
            if let Some(sani) = rt.strip_prefix("NovaOpt_") {
                // Emit compound literal for Some(v)/None matching context.
                ...
                return Ok(format!("(NovaOpt_{}){{ ... }}", sani));
            }
        }
    }
}
// Fallback: existing find_variant_compat path.
```

### Ф.3 — `Ok(v)`/`Err(e)` bare (~0.15 д)

**Где:** `emit_c.rs:14362-14378`.

**Логика:**
```rust
if type_name == "Result" && (name == "Ok" || name == "Err") {
    // Plan 99.2: contextual Result-type из current_fn_return_ty.
    if let Some(rt) = &self.current_fn_return_ty {
        if Self::is_result_like(rt) {
            if let Some((ok_c, err_c)) = self.novares_ok_err(rt) {
                // Use contextual (ok_c, err_c) instead of arg-defaults.
                let res_c = self.result_repr_c_type(&ok_c, &err_c);
                return self.result_ctor_name(&res_c, name);
            }
        }
    }
    // Fallback: existing arg-default behavior.
}
```

### Ф.4 — Тесты (~0.15 д)

`nova_tests/plan99_2_ctor/`:
- `option_ctor_in_body.nv`:
  ```nova
  fn make_some_int() -> Option[int] => Some(42)  // → NovaOpt_nova_int
  fn make_some_str() -> Option[str] => Some("x") // → NovaOpt_nova_str
  fn make_none_str() -> Option[str] => None      // → NovaOpt_nova_str
  fn make_none_user() -> Option[User] => None    // → NovaOpt_Nova_User_p

  test "Some(int) → NovaOpt_int" { let o = make_some_int(); assert(o.unwrap_or(0) == 42) }
  test "Some(str) → NovaOpt_str" { let o = make_some_str(); assert(o.unwrap_or("") == "x") }
  test "None[str] context" { let o = make_none_str(); assert(o.is_none()) }
  test "None[User] context" { let o = make_none_user(); assert(o.is_none()) }
  ```

  ```nova
  fn make_ok_user() -> Result[User, str] => Ok(User{ id: 1, name: "A" })
  ```

- **Negative**: `Some` в функции без Option return → fallback должен

## Acceptance criteria

- [ ] `Some(v)` bare в `-> Option[X]` контексте — typed compound
      literal `NovaOpt_<X>`.
- [ ] `None` bare в `-> Option[X]` контексте — same.
- [ ] `Ok(v)`/`Err(e)` bare в `-> Result[T,E]` контексте — typed mono
      ctor `NovaRes_<T>_<E>` (не legacy default).
- [ ] Fallback (без contextual return-type) — legacy behavior preserved
      (backward-compat).
- [ ] 0 regressions.

## Risk + mitigation

**R1.** Изменение `Some(v)`/`Ok(v)` bare-form может затронуть legacy
код, который полагается на legacy ctor types. Mitigation: change
только когда `current_fn_return_ty` matches Option/Result; иначе
legacy path сохраняется. Verify через `plan62`/`plan59` regression.

**R2.** `current_fn_return_ty` может быть None в некоторых contexts
(top-level let, REPL и т.п.) — fallback на legacy.
