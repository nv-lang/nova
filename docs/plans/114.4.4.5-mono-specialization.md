# Plan 114.4.4.5 — True per-const-arg monomorphization

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.8 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — performance optimization.
> **Оценка:** ~1-2 dev-day.
> **Зависимости:** Plan 114.4.3 V2 mixed-args + Plan 70.5 mono pipeline.
>
> **Why extracted:** Pragmatic V3.0 в Plan 114.4.3 ship'нул mixed fn как
>   runtime fn где const params — informational. True monomorphization
>   (per-const-arg specialisation как Rust const generics) требует deep
>   integration с Plan 70.5 pipeline.

## Scope

Replace V3.0 "mixed fn stays runtime, const params informational" с:
- Per unique const-arg combination → отдельная C fn `Nova_fn_X_cst_HASH`.
- Const params в body get substituted с literal values pre-codegen.
- Mono cache extension: (fn_name, const_args, generic_T) tuple keys.

## Tests

- Positive: 3 call sites с different const args → 3 distinct C fns в output.
- Positive: 100 identical call sites → 1 specialization (deduped).

## Acceptance

A1: Per-const-arg specialization observable through codegen output.
A2: Dead specializations dropped per existing mono cache logic.
A3: Const param value visible at runtime (compiler embeds literal).

### Implementation

- New module `compiler-codegen/src/const_fn_mono.rs` (~440 LOC).
- `specialize_mixed_const_fns(module)` entry point.
- AST walker recursively traverses all Expr/Stmt/Block/Item nodes.
- For each Call site of mixed const fn:
  - Extract const args as ConstValue (via try_literal_to_value).
  - Generate spec name `<orig>__cst_<idx>` (sequential counter).
  - Reuse existing spec if (fn_name, const_args) already specialized.
  - Rewrite Call: replace func ident name + drop const args.
- After walk: clone FnDecl per spec, substitute const param idents в
  body с literal exprs, drop const params from signature.
- Inserted specialized FnDecls в module.items.
