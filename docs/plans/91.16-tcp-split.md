# Plan 91.16 — TcpStream split: TcpReadHalf + TcpWriteHalf

STATUS: CLOSED (branch plan-91-net, 2026-06-17)

## Motivation
TcpStream is a consume value. Two fibers cannot hold it simultaneously.
split() creates two half-types sharing one C handle via atomic refcount.

## Implementation
- C: NovaRt_TcpReadHalf / NovaRt_TcpWriteHalf structs (pointer to shared stream)
  Separate read_scope/read_slot + write_scope/write_slot to avoid TOCTOU.
  _Atomic int split_refcount: split() sets 2, each close() decrements, socket closed at 0.
- Effect: TcpNet extended with split_stream + 14 read_half_*/write_half_* ops + write_all.
- Nova: TcpReadHalf/TcpWriteHalf consume value types in std/net/tcp.nv.
- Mock: all new ops covered in mock.nv.
- Tests: nova_tests/plan91_16/ (3 PASS: mock, echo_slow, after-split-neg).

## D-blocks
D301: TcpStream.split() spec (spec/decisions/04-effects.md)

## V1 limitations
- write_all is C-backed (not a Nova loop): vtable OK-type erasure made a Nova-level while loop
  fail at codegen (nova_str where Nova_NetError_p expected).
- Only 1 negative test (use-after-split): consume (rd, wr) = s.split() is a parse error;
  double-close-of-half and read-after-close-of-half are not compile-time errors in V1.
  Runtime refcount protects. Followup: [M-91.16-tuple-consume-binding].
