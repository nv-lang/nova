# Plan 159.1 — Method-reachability DCE: точность + звучность-как-МОДЕЛЬ

> Под-план [159 (Reachability-based codegen)](159-reachability-codegen.md).
> Соответствует [compiler-conventions.md](../compiler-conventions.md): **§0/§1** (leak = §1-нарушение,
> C ловит резолвнутое-но-недоступное), **§3** (single-window реестр, не россыпь hardcode + врезка
> «видимость ≠ доступность»), **§4** (gaps под `[M-*]`), **§5** (D-блок), **§6** (explain-tool),
> **§7** (detect-режим / kill-switch / baseline-vs-чистый-бинарь / over-keep), **§8** (pos+neg на
> каждый desugar).

## 0. Коррекция первичного понимания (урок §7.7)

Первый черновик этого плана описывал **pre-159** состояние из устаревших doc-комментов
(«method-DCE нет / методы держатся per-receiver-type / DCE off без main»). **Адверсариальная
проверка кода это опровергла** — и опровергла два «P0-разрыва» первичного синтеза:

- **«vtable-membership не моделируется»** → опровергнуто юнит-тестом `protocol_dispatch_keeps_method_via_selector`
  (`emit_c.rs:39428`): protocol-dispatch `g.greet()` **уже выживает** (коэрция называет тип +
  селектор пишется → метод не в `dead_method_keys`). Реальный остаток узок.
- **«`@method:<name>` инертен»** → опровергнуто: он **используется** в `imports.rs:441`. Переключить
  firing метода на него = **регресс** module-qualified вызовов (`mod.func()` пишется как bare-name,
  не `@method:`).

**Урок §7.7 (зафиксирован):** проверяй заявленный разрыв ПЕРЕД планированием работы по нему —
«X выглядит неохваченным» ≠ «X неохвачен». Реальные разрывы — другие и **уже**, чем казалось.

## 1. Текущее состояние (факт, по коду — §7.1)

- **Per-method DCE РЕАЛИЗОВАН** (Plan 159 Ф.2): метод → свободная C-fn `Nova_<T>_method_<m>(self,…)`;
  собирается в methods-list `(тип, имя, refs)`; «выстреливает» (тело+fwd живут) ⟺ `reachable(тип)
  ∧ reachable(имя)` — intersection + фикспоинт (`emit_c.rs:442`, `420-465`). Never-fired →
  `dead_method_keys`; fwd+body skip (`3091-3100`, mirror `8942-8949`).
- **Гранулярность COARSE-BY-NAME** (`[M-159-method-pruning]`, P3): ключ на bare type-name + bare
  selector → любая достижимая коллизия (локал/поле/const/overload/одноимённый метод другого типа)
  over-keep'ит. Точный `@method:<name>` собран (`lints.rs:1166`), но в firing не консультируется.
- **Корни:** `main` ∪ имена unconditional-items (generic fns, externals, **tests, benches**,
  type-decls). НО tests/benches — корни **только при наличии `main`**: `!has_main` → `return
  DeadDecls::default()` = DCE OFF целиком (`322-327`). Это **намеренно** для negative
  `EXPECT_CC_ERROR`-фикстур (их main-less ошибочные тела ОБЯЗАНЫ дойти до C, `comment 200-202`).
- **Generics исключены** из DCE-кандидатуры (lazy mono-worklist, Plan 48).
- **Vtable** (Plan 72 P3-B): эмитится demand-driven; обычный путь выживает по construction
  (тест выше). Узкий остаток — слот, чьё имя-селектор нигде не появляется (witness/супертрейт/
  drop-через-vtable).
- **Звучность ЭМЕРДЖЕНТНА:** desugar-селекторы захардкожены россыпью в `collect_used_names`
  (`lints.rs:1198-1370`: binary→equal/eq/compare/concat, index→index, for/pfor→next/iter,
  interp→StringBuilder/…). **Over-prune уже стрелял** (`Contract.message_expr`, фикс `b26d1cab`) —
  эмпирическое доказательство хрупкости hand-seed.

## 2. Цель

Не «добавить method-DCE» (он есть), а: **(1)** закрыть РЕАЛЬНЫЕ over-prune дыры; **(2)** обобщить
якорь **по категории цели** без поломки negative-фикстур; **(3)** превратить звучность из
**эмерджентной** (hand-seed) в **моделируемую** (§3 single-window реестр + §7 over-keep инвариант);
**(4)** опц. догнать Go по точности (arity-ключ).

**Граница (§0/§1):** метод-DCE = **эмиссионный слой** (не эмитить unused → ссылка на недоступный
callee не появляется → CC-FAIL исчезает). **USED-половина** (вызванный примитив-метод под
селективным `#prelude`) — **отдельный слой доступности** (C-helper-лоуэринг / слои stdlib), см.
конвенция §3 «видимость ≠ доступность». Не путать слои.

## 3. Фазы (каждая под §7)

### Ф.0 — Карта рёбер + категоризация целей (§7.1/§7.2 — фундамент)
- Перечислить ВСЕ способы оживления метода + точку в коде + покрыт/нет: прямой `x.m()`;
  vtable-construction (коэрция T→protocol); операторы (`[]`/`==`/`+`/`<`/`hash`); `for-in`→`@iter/@next`;
  **drop/`on_exit`/finalizer** (scope-exit, имя не названо); **auto-into/From-on-`?`** (неявная
  коэрция вне interp); **method-value/closure-capture** (operand-position); **FFI-callback**
  (Nova-fn→C function pointer); cross-module/separate-compilation.
- Категоризировать build-target: `Executable(main) | TestRunner | BenchRunner | Library |
  NegativeCCError`. **Артефакт:** таблица «способ → ребро» + «цель → root-политика». Непокрытое
  ребро → `[M-*]` (§4), не молчаливый пропуск.

### Ф.1 — P0 точечные over-prune дыры (наивысший severity)
- **`on_exit`/drop-селектор seed** в `ConsumeScope`-walk (`lints.rs:~1107`) + **рекурсивно по
  droppable-полям** (структурный обход type-graph, НЕ AST — `collect_used_names` принципиально не
  видит scope-exit drop без него). Закрывает наивысший over-prune (finalizer достижим чисто
  scope-exit). Конкретная готовая точка для seed уже есть.
- **contract `message_expr`** — регресс-гард (фикс `b26d1cab`) + завести в реестр Ф.3.
- §8: pos-фикстура (метод жив ТОЛЬКО через desugar → выживает под DCE-ON) + kill-switch parity.

### Ф.2 — Обобщить якорь ПО КАТЕГОРИИ (НЕ снимать kill-switch вслепую)
- Root-политика per build-target (из Ф.0): `TestRunner` → roots = test/bench-fns **без** main;
  `Library` → roots = pub/extern surface; `NegativeCCError` → **DCE ВЫКЛЮЧЕН** (мёртвое ошибочное
  тело обязано дойти до C); `Executable` → main. **Закрывает leak в тест-файлах** (test-блоки =
  корни → unused `find_first_panic` срезан → нет ссылки на недоступный `starts_with`).
- §7.5: baseline-vs-**чистый** бинарь (kill-switch `NOVA_REACH_DCE=0` на ТОМ ЖЕ бинаре). §8:
  regression-guard на КАЖДУЮ категорию — особо `EXPECT_CC_ERROR` не должен ложно-PASS (критика:
  единый «no-main → roots=tests» срезал бы ошибочное тело → негатив перестал ловить ошибку).

### Ф.3 — Единый `DesugarSelector`-реестр (§3 single-window; звучность → MODELED)
- ОДНА таблица keyed by desugar-kind, консультируемая И эмиттером десугара, И `collect_expr` —
  вместо россыпи hand-seed (§3: «параллельные пути под одно и то же = повторяющийся источник
  багов»). Забыл seed → **compile-time mismatch**, не latent over-prune.
- §7: внедрять как **НАДМНОЖЕСТВО** всех текущих seed-точек (binary/index/for/pfor/interp/contract
  + on_exit/auto-into из Ф.1) + **assert на полноту** + пер-desugar регресс-гард ДО рефактора.
  Иначе миграция сама потеряет seed-точку → регресс. Превращает звучность из emergent в modeled.

### Ф.4 — Точность ключа `(type, name, arity)` (опц., догнать Go — §7.3 калибровка)
- methods-vector + `dead_method_keys` по `(type, name, param_count)`; call-site seed несёт арность
  (`argc` уже известен из `args.len()`). Закрывает overload/multi-param edge (memory
  `feedback-nova-tests`: арность/multi-param — обязательный edge-тест). **bare-name остаётся**
  fallback (критика: наивный переход на `@method:` ломает module-qualified) — arity лишь уточняет.

### Ф.5 — Explain-tool (§6 диагностика)
- `NOVA_REACH_EXPLAIN=<Type.method>` → why-live/why-dead путь (root → рёбра → fired/not). Дёшево
  поверх worklist (логировать рёбра при вставке). Для отладки over-prune (Пробл.2) и
  availability-leak (Пробл.1) — §6 «понятная диагностика, не крипто».

### Ф.6 — Spec / docs / conventions (§5)
- **D-блок** (расширить **D283** на method-reachability: per-`(type,name,arity)`, категории целей,
  witness/desugar/drop рёбра, over-keep инвариант G0).
- `[M-*]` на остаточные непокрытые рёбра (§4) + backlog.
- Кандидат во врезку `compiler-conventions.md` §1/§7: «звучность DCE — MODELED (реестр + over-keep),
  не emergent (hand-seed)» — по согласованию (governance).

## 4. НЕ делать (критика опровергла — анти-§7.7)

- ❌ **wire `@method:` в firing как ЗАМЕНУ bare-name** — регресс module-qualified (`mod.func()`);
  `@method:` не инертен (`imports.rs:441`). Только как ДОП-условие + bare-name fallback.
- ❌ **vtable-edge как ЗАМЕНА** bare-name пути — может СНИЗИТЬ звучность: vtable demand-driven во
  время codegen, `compute_dead_decls` — ДО; неполный предпроход обнаружения коэрций → метод prune,
  thunk эмитнут → undeclared. Текущий путь (имя-селектор) случайно надёжнее. Дополнять, не заменять.
- ❌ **снять `!has_main`** без категоризации целей — ложный PASS на `EXPECT_CC_ERROR`.

## 5. Связь с Проблемой 1 (availability) — §0/§1 + конвенция §3

Метод-DCE = эмиссионный слой (unused-половина leak'а). **USED-половина** (вызванный примитив-метод
под `#prelude(partial)`) — **отдельный слой доступности**: C-helper-лоуэринг (`str.starts_with →
nova_str_starts_with`) ИЛИ слои stdlib (`core⊂runtime⊂…`). Конвенция §3 «видимость ≠ доступность» —
это availability-слой; метод-DCE — emission-слой. Критика: организационные правки stdlib (слои,
полные депы) меняют **COMPILED**, но не корень «чекер РЕЗОЛВИТ глобально vs COMPILED» — для корня
нужен `resolved ⊆ emittable` инвариант (отдельный шаг, warn-режим сначала).

## 6. Звучность (§7 G0) — честно, без overclaim (§10)

Цель — звучность **MODELED**, не emergent. Сейчас держится на hand-seed (доказанно хрупко:
`b26d1cab`). **До Ф.3 Nova по «sound by construction» ХУЖЕ Zig/Rust**, не лучше. Претензия «лучше
всех» (sound method-DCE без escape-hatch — AOT+mono+нет рефлексии) обоснована **только ПОСЛЕ** Ф.3 +
`resolved⊆emittable` ассерт, и требует заморозки reflection-контракта до **Plan 172.8 (Any/downcast)**
(иначе closed-world теряется). Без эмпирики «лучше» не заявляем (§10).

## 7. Acceptance / Deps / Priority

- **Acceptance:** 0 новых FAIL полного регресса (§7.6) + per-desugar pos-guards PASS + byte-identical
  при kill-switch `=0` + `EXPECT_CC_ERROR` не ложно-PASS + on_exit/drop over-prune закрыт + измеренный
  shrink `.c` (§2).
- **Deps:** Plan 159 (Ф.1-2 движок, D283, kill-switch `NOVA_REACH_DCE`), Plan 72 P3-B (vtable),
  конвенция §3 (availability-слой — отдельно). Не блокирует C-helper-заглушку used-половины.
- **Priority:** Ф.1 (on_exit) **P0**; Ф.2 (категоризация → закрывает leak) **P0**; Ф.3 (реестр —
  звучность modeled) **P1**; Ф.4/Ф.5 **P2**; Ф.6 spec вместе с реализацией.
