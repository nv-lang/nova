# Plan 159.1 — Method-level reachability DCE («Ф.7.2-methods»)

> Под-план [159 (Reachability-based codegen)](159-reachability-codegen.md). Реализует
> именованный в коде инкремент **«Ф.7.2-methods»** (`emit_c.rs:154`, заметка Plan 81 Ф.7).

## Зачем

**Цель:** распространить reachability-DCE на **методы** — не эмитить в C тело+forward-decl
мономорфного метода, недостижимого от корней, **по-методно** (а не «весь тип-приёмник
сразу»). Метод codegen опускает в свободную C-функцию `Nova_T_method_m(T self, …)` — `@`/`self`
просто первый параметр, поэтому метод прунится тем же reachability-механизмом, что и свободная
функция; единственное отличие — **динамический диспатч** (vtable), который надо смоделировать
рёбрами достижимости.

**Что чинит (две независимые выгоды):**

1. **Класс leak'ов «unused метод тянет недоступное».** `MultiError @find_first_panic` использует
   `str.starts_with`; под `#prelude(core, runtime, errors)` модуль `string` не подтянут. Сейчас
   метод эмитится (хоть не вызван) → ссылка на `Nova_str_method_starts_with` повисает →
   `CC-FAIL` (struct-tag/undeclared). Если **неиспользуемый** метод не эмитить — ссылка исчезает,
   leak закрыт. Прямая причина текущего leak'а: fixture `plan107/prelude_multi_attr` — тест-файл
   **без `fn main`**, а DCE без main выключен (см. ниже) → `find_first_panic` эмитится.
2. **Perf (§2).** Меньше `.c` (как Ф.1 для свободных fn: 10606→2494 строк, ~4.25×↓ на unicode),
   быстрее C-компиляция, меньше работы codegen.

**§0/§1 связь:** «резолвнутый-но-неиспользуемый метод не должен форсить эмиссию своих недоступных
callee'в». Дополняет конвенцию [«видимость (`#prelude`) ≠ доступность»](../compiler-conventions.md)
(§3): метод-DCE закрывает **unused-половину** (метод не вызван); **used-половину** (вызванный
примитив-метод под селективным prelude) закрывает доступность — C-helper-лоуэринг
(`str.starts_with → nova_str_starts_with`) или split string-minimal. Это **не альтернативы, а два
слоя** (эмиссия vs доступность).

## Текущее состояние (по коду `emit_c.rs`)

- **Свободные fn + const/ro-globals** прунятся (Plan 159 Ф.1, `compute_dead_decls`).
- **Методы** — частично: `dead_method_keys` ((receiver, method)) существует, НО:
  - **Грубо — якорь на ТИП-приёмник, не на метод.** «if `T` named ANYWHERE reachable, ALL of
    `T`'s methods are kept» (`emit_c.rs:244`). Назвали тип → сохранили **все** его методы.
  - **Тела методов эмитятся безусловно** в Ф.1 («still out of scope for body-level method DCE»,
    `emit_c.rs:235`); `dead_method_keys` срезает только при недостижимости **тип-приёмника**.
  - **Без `main` — DCE целиком выключен** (`compute_dead_decls_with:302` → `return default`):
    «не executable → keep everything». Тест-файлы (нет `fn main`; harness синтезирует main позже)
    → метод-DCE no-op → unused-методы эмитятся. **Это причина leak'а в тестах.**
- **Переиспользуемая инфраструктура:**
  - Plan 159 Ф.2 — indirect reachability (протокол/замыкания) для свободных fn уже есть.
  - Plan 72 P3-B — vtable-инстансы эмитятся: `emitted_vtable_instances: HashSet<(vtable_struct,
    concrete_mangled)>`, `protocol_var_vtable` (var → `__vt_x`). Это и есть рёбра динамического
    диспатча.

## Звучность (G0 — главный инвариант: over-keep, НИКОГДА over-prune)

Метод срезается **только** если доказуемо недостижим: (а) не вызван напрямую, И (б) не в одном
живом vtable-инстансе, И (в) не вызван через десугар/оператор/drop. Любая неоднозначность →
**сохранить**. Over-prune мономорфного метода = dangling C-ссылка ИЛИ отсутствующее тело при
рантайм-вызове = **сломанный бинарь** (хуже, чем лишний код). Поэтому **Ф.0 (карта рёбер) —
фундамент**: пропущенный способ вызова = over-prune.

### Опасные рёбра (каждое ОБЯЗАНО быть смоделировано — иначе over-prune)

| Способ, которым метод становится живым | Риск, если забыть |
|---|---|
| Прямой `x.m(…)`, `x: T` | (базовый static edge) |
| **Протокол-динамический** `p.m()`, `p: dyn P` → vtable | главный — метод жив через слот, прямого ребра нет |
| **Операторные методы**: `@index`/`@get` (`a[i]`), `@eq`/`@equal` (`==`), `@compare` (`<`), `@plus`/`@add` (`+`), `@hash` | вызов синтаксисом, не `.m()` |
| **Итератор-протокол**: `for x in xs` → `@iter`/`@next`/`@as_chars` | десугар-вызов |
| **Конверсии**: `into()`/`from()`, `as`-cast методы, `str.from(char)`, `char.into()` | косвенный вызов |
| **Drop/consume/defer**: drop-методы на consume-типах / scope-exit | codegen-вызов, не в AST |
| Замыкания / first-class method-values / method-references | захват без call-site |
| Default-методы протоколов; trait-bound методы в дженериках | дженерики уже lazy-mono — но default-тела проверить |
| Cross-module вызов | Nova не имеет C-ABI export → merge в executable, как у свободных fn |

## Декомпозиция

### Ф.0 — Карта рёбер достижимости методов (звучностный фундамент)
Перечислить **каждый** способ, которым мономорфный метод становится reachable в лоуэринге
(таблица выше — стартовая, дополнить по коду). Для каждого: где в `emit_c.rs` он эмитится и как
добавить ребро/корень. Особо: все vtable-instance emission sites (`emitted_vtable_instances`),
операторный диспатч, `for-in` десугар, drop/defer, into/from. **Артефакт:** таблица «способ →
ребро», ревью на полноту. Непокрытый способ = known over-prune-риск (помечается, gate на закрытие).
*Это самый важный шаг — §7.1/§7.2.*

### Ф.1 — Vtable-membership как рёбра достижимости
Каждый **достижимый** vtable-инстанс `(P, T)` добавляет корни `(T, m)` для всех слотов-методов `m`
протокола `P`. Переиспользовать Plan 159 Ф.2 (indirect protocol-reachability для свободных fn уже
моделирует «живой vtable → его callee'и»). Консервативно: vtable-инстанс построен в живом коде →
все его слот-методы живы.

### Ф.2 — Операторные/десугар/drop-методы как корни
Из карты Ф.0: когда оператор (`[]`/`==`/`+`/`<`), `for-in`, конверсия (`into`/`from`/`as`) или
drop/defer на типе `T` достижимы → соответствующий метод `(T, op_method)` — корень. Реализовать
через `collect_used_names`-расширение (распознавать операторные/десугар-позиции, не только
`Ident`/`Member`).

### Ф.3 — Per-method reachability в `compute_dead_decls`
Кандидаты = мономорфные non-external методы (ключ — mangled `Nova_T_method_m` или `(T, m)`).
Заменить грубый per-receiver-type якорь (`emit_c.rs:244`) на **per-method** замыкание над рёбрами
Ф.1+Ф.2+прямые вызовы. Дед-метод = кандидат вне reachable-множества. Over-keep при коллизии имён.
Дженерик-методы — out of scope (уже lazy через mono-worklist, как generic free fns).

### Ф.4 — Обобщить ЯКОРЬ: корни = `main` ∪ tests ∪ benches ∪ lib-exports (НЕ только `main`)
**Не «тест-режим», а общее правило.** Сейчас `if !has_main { return default }` (`:302`) —
слишком грубо: бросает DCE целиком, хотя корни есть и без `main`. Якорь обобщается:
- **executable** → корень `main`;
- **тест-компиляция** → корни = `test`/`bench`-блоки (их зовёт harness). Тесты УЖЕ перечислены
  корнями для free-fn DCE (`:162`), но `!has_main`-гард срабатывает раньше и перечёркивает их;
- **библиотека** (no main, есть `export`) → корни = экспортируемый API.

«Keep all» оставить **только** когда корней реально НЕТ — negative `EXPECT_CC_ERROR`-фикстура с
голыми ошибочными декларациями (их тела обязаны дойти до C), `nova check` без entry. Т.е.
`has_roots = has_main || has_tests || has_benches || has_lib_exports`; `!has_roots → keep all`.

**Важно — это чинит и СУЩЕСТВУЮЩИЙ free-fn DCE:** в тест-файлах он сейчас тоже ничего не прунит
из-за того же `!has_main`-гарда. Обобщение якоря — отдельный фундаментный шаг, выгодный обоим
(free-fn + method) DCE; делается до/вместе с per-method (Ф.3). **Закрывает `prelude_multi_attr`**
(тест-блоки как корни → `find_first_panic` недостижим → срезан).

### Ф.5 — Kill-switch + detect-mode blast-radius (§7.2)
Под-флаг (`NOVA_REACH_DCE_METHODS`, либо расширить `NOVA_REACH_DCE`). Detect-mode: счётчик
методов-кандидатов на прун + проверка, что ни один реально-достижимый не попал в dead (сверка с
поведением). Выключенный флаг → **byte-identical** прежнему выходу. Замер: сколько методов
срезается на std/nova_tests.

### Ф.6 — Верификация (§7.5/§7.6) + guard-фикстуры
- **Полный регресс** (не сэмпл) + **baseline-vs-чистый-бинарь** (kill-switch на ТОМ ЖЕ бинаре).
- **Guard-фикстура на КАЖДОЕ опасное ребро** (Ф.0): protocol-dyn-dispatch, оператор (`[]`/`==`/`+`),
  `for-in`, drop/defer, `into`/`from`, closure-captured method — каждая проверяет, что метод НЕ
  over-prune'ится (вызывается и работает).
- `plan107/prelude_multi_attr` → **PASS** (`find_first_panic` срезан, leak закрыт).
- **Perf-замер:** `.c` size + compile time до/после (§2, как Ф.1 указала 4.25×↓).
- Acceptance gate: **0 новых FAIL** полного регресса + все guard PASS + byte-identical при
  выключенном флаге + измеренный shrink.

### Ф.7 — Spec/docs/error-index
- **D-блок** (расширить **D283** reachability-codegen на method-level): корни, рёбра (vtable/
  оператор/desugar/drop), per-method гранулярность, over-keep-инвариант.
- Закрыть unused-половину `[M-172.1-U7-prelude-strmethod-availability]`; **явно** отметить, что
  used-половина (вызванный примитив-метод под селективным prelude) остаётся за доступностью
  (C-helper / split string-minimal) — отдельная запись.
- Sync README-плана + simplifications/conventions при необходимости.

## Acceptance (умбрелла)

Метод-DCE включён по умолчанию (kill-switch для отката); полный регресс **0 новых FAIL**; все
guard-фикстуры опасных рёбер PASS; `prelude_multi_attr` PASS; byte-identical при `=0`; измеренный
shrink `.c`; D283 расширен.

## Риски

- **Over-prune = unsound = сломанный бинарь** (главный). Mitigation: карта Ф.0 (полнота рёбер) +
  conservative over-keep + guard-фикстура на каждое ребро + detect-mode перед включением.
- **Protocol-dispatch консервативность** (заметка Plan 81 Ф.7) — Ф.1 решает через vtable-membership.
- **Операторные/desugar/drop-вызовы** не видны как `Member`-селекторы — Ф.2 расширяет
  `collect_used_names`; пропуск = over-prune.

## Deps / связь

- **Plan 159** (Ф.1 reachability-ядро + Ф.2 indirect/protocol для свободных fn + kill-switch +
  D283) — база.
- **Plan 72 P3-B** (vtable-инстансы `emitted_vtable_instances`) — рёбра динамического диспатча.
- **Plan 81 Ф.7.2-methods** — этот инкремент (lineage).
- **Конвенция «видимость ≠ доступность»** (`compiler-conventions.md` §3) — метод-DCE = эмиссионный
  слой; доступность (used-половина) = отдельный слой (C-helper/split).
- **Не блокирует** немедленную заглушку leak'а через C-helper (route `str.starts_with` →
  `nova_str_starts_with`) — она независима и закрывает used-половину.

## Размер / приоритет

Звучностный компилятор-инкремент с careful-картой рёбер. ~Ф.0-Ф.7, оценка средняя (основная
стоимость — Ф.0 карта + Ф.6 guard-фикстуры на каждое ребро). **P2** (perf + leak-класс; не
блокирует прод — есть C-helper-заглушка). Gate: после Ф.0-карты решить объём Ф.1-Ф.2 по реальному
числу рёбер.
