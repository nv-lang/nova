> «с чистого листа» (3 параллельных агента + прямая верификация),

## Контекст

Аудит Plan 42 (folder-modules) выявил, что claim «production-grade,
no simplifications» нарушен в нескольких местах. Найдено:

1. **Архитектурный gap.** Per-peer FileId изоляция (Rule C) реально
   работает **только для импортированных** folder-modules. Entry-модуль
   регистрируется как один `PeerFile` с `MAIN_FILE_ID`
   ([imports.rs:69-86](../../compiler-codegen/src/imports.rs)) —
   уникальные `file_id` раздаются только peer'ам импортированных
   folder-modules. `span_walker.rs` (предписан Plan 42.04 Ф.2) — **не
   создан**; doc-comment `ast/mod.rs` до сих пор говорит «placeholder».
   `[M10]`/`[M12]` помечены RESOLVED без квалификации.
2. **Rule H — документация противоречит коду.** Plan 42.14 Ф.5 заявляет
   фикс Rule H (сравнение по declared module name) — его нет. Код
   сравнивает строки import-path. Plan 42.13 честен.
3. **Shipped dead code.** `resolve_import_path` (неиспользуем),
   `peer_canons` (vestigial), `let _ = imp.items.is_some()` (no-op),
   `Box::leak` в `current_target_os()` (неограниченная утечка).
4. **Doc drift.** 42.04 описывает `SourceFile`/`imported_items:
   Vec<Item>` — реальность `PeerFile`/`imported_item_names:
   HashSet<String>`. 42.09 говорит rename не работает — работает.
   main-42 acceptance criteria не отмечены, «42.A/42.B» naming.
5. **Три копипаст-сканера** `module`-строки (`read_module_decl`,
   `is_folder_module_peer`, `is_folder_module_dir`) — дрейфуют, нет
   обработки блок-комментариев.
6. **Слабые тесты.** `folder_module_with_test` не дёргает `_test` peer.
   `feature_gate`/`target_os_module` не проверяют invisibility. Нет
   `folder_with_single_file`, negative `inconsistent_decl`.
7. **Module-attrs после `module`** — misleading error вместо чёткого
   отказа.
8. **re-export rename consumer-side** — visibility check не смотрит
   `it.alias`.

**Что подтвердилось хорошим:** Rule D (2-pass codegen) реально работает
через глобальный forward-decl pass; callnorm во всех codegen-путях;
`#cfg` eval, ambiguous-диагностика, selective filter, re-export
propagation — корректны.

## Фазы (по возрастанию риска, commit после каждой)

### Ф.1 — Honesty: doc drift fix (docs only, zero risk)

- `42.04-per-file-imports-scope.md` — `SourceFile`→`PeerFile`,
  `imported_items`→`imported_item_names`, отметить что `span_walker.rs`
  не создан (FileId через `parse_with_file_id`).
- `42.09-re-export.md` — rename работает end-to-end, обновить статус.
- `42.14-backlog-closure.md` Ф.5 — честно: Rule H **не** был
  переведён на declared-name (это сделает Ф.4 ЗДЕСЬ).
- `42-folder-modules.md` — отметить acceptance criteria, мигрировать
  «42.A/42.B»→цифры, sub-plan 42.5 (2-pass) — отметить satisfied via
  global forward-decl pass.
- `simplifications.md` — `[M10]`/`[M12]` квалифицировать «resolved для
  импортированных folder-modules»; `[M11]` — отметить block-comment
  edge (закроется Ф.3).

### Ф.2 — Dead code cleanup (low risk)

- Удалить `resolve_import_path` (imports.rs — полностью неиспользуем,
  superseded `resolve_module_paths`).
- Удалить vestigial `peer_canons` accumulator → заменить на прямой
  `PathBuf` где нужен `PeerFile.path`.
- Удалить no-op `let _ = imp.items.is_some();` и `let _ = &peer_canons;`.
- `current_target_os()` — убрать `Box::leak`, вернуть `String` (или
  `&'static str` через match без env-var leak).

### Ф.3 — Consolidate module-line scanners (low-medium risk)

- Один хелпер `scan_module_decl(src) -> Option<Vec<String>>` —
  пропускает blank / `//` / `/* */` / `#`-attr строки, читает
  `module a.b`.
- `read_module_decl`, `is_folder_module_peer`, `is_folder_module_dir`
  переводятся на него.
- Закрывает block-comment edge из `[M11]`.

### Ф.4 — Rule H actual fix (medium risk)

- `internal/` boundary проверяется по **declared module name** peer'ов
  `internal/` (D29 rev-3.1 — `owner.internal.target`), не по import-path
  строке. Importing module's declared name должен иметь `owner` prefix.
- Снимает re-export bypass.

### Ф.5 — Module-attrs-after-`module` clean rejection (low-medium risk)

- Парсер: `#forbid`/`#cfg`/`#doc` после `module` declaration → чёткая
  ошибка «module-level attribute must precede `module` declaration».

### Ф.6 — re-export rename consumer-side fix (low-medium risk)

- Visibility predicate в `resolve_one` смотрит и `it.name`, и
  `it.alias` — consumer импортирующий уже-renamed имя видит его.

### Ф.7 — Test strengthening (low risk, additive)

- `folder_module_with_test` — `_use.nv` должен пытаться вызвать символ
  из `_test` peer (доказать test-only isolation в build mode).
- `feature_gate` / `target_os_module` — assert invisibility gated
  items (negative-побратимы или inline EXPECT в отдельных файлах).
- Добавить `folder_with_single_file` (правило E) positive.
- Добавить negative `folder_inconsistent_decl` (peers с разными `module`).

### Ф.8 — Entry-folder-module isolation (high risk — implement или honest-defer)

- Investigate: может ли `resolve_imports_inline_ex` детектить, что
  `entry_path` — peer folder-module, и собрать siblings с distinct
  `file_id` + `is_entry_module: true`.
- Если clean и регрессия чистая — реализовать.
- Если требует test-runner changes / ломает >5 тестов нетривиально —
  honest-defer: задокументировать `[M-entry-folder-module]` в
  simplifications.md (не reachable сейчас — test_runner/nova-cli всегда
  используют single-file либо `_use.nv` entry).

### Ф.9 — Regression + docs + commit

- Full `nova test` (release) — без новых FAIL.
- project-creation.txt + simplifications.md финальные записи.
- Каждая фаза — отдельный commit.
