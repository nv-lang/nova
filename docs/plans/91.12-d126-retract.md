## Зачем

D126 (`external type X` syntax, Plan 62.D.bis) — bridge bootstrap до полноценной
typed runtime support. Изначально 5 stdlib типов через D126:
  - `WriteBuffer`, `ReadBuffer` в std/prelude/collections.nv
  - `OnceCell[T]`, `Lazy[T]`, `Condvar` в std/runtime/sync.nv

С Plan 109 (D179) StringBuilder ушёл с external type на pure Nova consume
record (`type StringBuilder consume { mut buf []u8 }`). Аналогичный паттерн
работает для WriteBuffer/ReadBuffer — они только обёртки над `[]u8` с cursor
position'ом (ReadBuffer) / append-semantics'ой (WriteBuffer). C runtime
(`nova_rt/{write,read}_buffer.h`) — это ~30+60 inline C functions с macros
для per-width LE/BE encoding, которые делают одно и то же на Nova: bit-shifts
+ push/extend в `mut buf []u8`.

Sync types (OnceCell/Lazy/Condvar) — другой случай: они требуют atomic state,
park/wake fiber primitives, mutex-locking. Pure-Nova реализация = большой
followup (Plan 91.12.x or 103.x extension). В этом Plan 91.12 v1 они НЕ
мигрируются — остаются как external type.

## Дизайн

### WriteBuffer (std/runtime/write_buffer.nv)

```nova
#no_prelude
module runtime.write_buffer

export type WriteBuffer {
    mut buf []u8
}

const INITIAL_CAPACITY = 16

export fn WriteBuffer.new() -> Self => WriteBuffer.with_capacity(INITIAL_CAPACITY)
export fn WriteBuffer.with_capacity(n int) -> Self => { buf: []u8.with_capacity(n) }
export fn WriteBuffer.from(b []u8) -> Self { ... }

// Query: @len, @capacity, @clone
// Writes: @write_byte/bytes/zero/char/str + 18 numeric × LE/BE
// Finalize: consume @into -> []u8
```

**Дизайн-решения:**

1. **WriteBuffer — НЕ consume type** (в отличие от StringBuilder). Только
   `consume @into()` метод consume. Reason: backward-compat с predecessor
   C runtime, где `let mut wb = WriteBuffer.new()` был стандартным паттерном.
   Делать тип consume → требовать `consume wb = ...` биндинг для всех
   существующих fixtures (~30 файлов). Менее ergonomic для callers.
2. **Numeric writes** — bit-shift + truncating `as u8` cast. Pattern:
   `@buf.push(x as u8).push((x >> 8) as u8)...` (LE), reversed for BE.
   `int` (i64) bit-pattern preserved через unsigned/signed casts;
   `as u8` truncates lowest byte regardless of source sign.
3. **f32/f64** — IEEE 754 reinterpret через `to_bits()` → u32/u64 →
   соответствующий integer write.
4. **`@write_zero(n)`** — bulk-zero append через `reserve(n) + push 0 × n`
   loop. Pure Nova не имеет `extend_filled(n, v)` primitive — followup
   [M-91.12-append-filled].

### ReadBuffer (std/runtime/read_buffer.nv)

```nova
#no_prelude
module runtime.read_buffer

export type ReadBuffer {
    ro data []u8
    mut pos int
}

export fn ReadBuffer.from(b []u8) -> Self => { data: b, pos: 0 }
// @position, @remaining, @has_remaining, @remaining_bytes
// 22 reads: @read_X / @try_read_X pair × { byte, bytes, u8, i8, u16/32/64 LE/BE × {u,i},
//                                            f32/f64 LE/BE, char, str }
```

**Дизайн-решения:**

1. **ReadBuffer — non-consume cursor view**: `{ ro data, mut pos }`. Read-many,
   no ownership transfer.
2. **Pattern: try_X primary, read_X derived**:
   ```nova
   export fn ReadBuffer mut @try_read_u32_le() -> Result[u32, ReadBufferError] { ... }
   export fn ReadBuffer mut @read_u32_le() Fail[ReadBufferError] -> u32
       => @try_read_u32_le()!!
   ```
   Single source of truth, !! converts Err→throw.
3. **Sign extension** для signed reads (i16/i32) — explicit `if raw >= MSB { raw - 2*MSB } else { raw }`
   pattern.
4. **UTF-8 char/str** — helper `@_decode_utf8_at(p) -> Result[(int, int), _]`
   (codepoint + consumed_bytes). Pattern взят из `_nova_rb_decode_utf8_one`
   в old C runtime. Invalid UTF-8 → UnexpectedEnd (bootstrap limitation:
   ReadBufferError имеет только UnexpectedEnd variant).
5. **`char.try_from(cp)`** — D54 запрет `int as char`, использован
   `match char.try_from { Ok(c) => Ok(c), Err(_) => Err(UnexpectedEnd) }`
   (Err unreachable т.к. decoder уже validate'нул UTF-8, но нужно для
   well-typedness).

### Codegen bug fixes (incidental, surfaced by Plan 91.12)

**Bug 1 — `Result[T, E].unwrap_or` / `.unwrap` для non-int T**

Старый C runtime хранил Result Ok payload как `nova_int` slot (boxing
для non-int T через bit-cast). При unwrap, codegen делал union bit-pun
nova_int → target type. Для typed mono `NovaRes_<T>_<E>*` storage уже
typed (`struct { nova_f64 _0; } Ok`); bit-pun становится wrong (interpret
nova_f64 value as nova_int, then back as f64 — garbage).

Fix: detect typed mono prefix `NovaRes_*` в `obj_ty` → direct access
`payload.Ok._0` без cast_from_nova_int. Legacy bootstrap `Nova_Result*`
сохраняет старый bit-pun path. Affected functions:
- `emit_expr` Result.unwrap_or inline path (~line 18020).
- `emit_expr` Result.unwrap inline path (~line 18210).

**Bug 2 — Module-private `const` C-name collision**

`const X = ...` в разных модулях с одинаковым именем коллидит в global
symbol table (codegen эмитит `static const TYPE X = ...` без module-path
mangling). Surface: `INITIAL_CAPACITY = 16` в string_builder.nv и
write_buffer.nv → redefinition error.

Fix: `private_const_c_names: HashMap<(FileId, String), String>` — pre-pass
по `module.peer_files`, для каждого `!c.is_export` const'а:
`Nova_const_<modpath>_<name>` mangled C-symbol. Decl emission использует
mangled. Use-site Ident resolution: lookup по (expr.span.file_id, name);
fallback на single-candidate match если span attribution не дала результата.

Followup [M-91.12-const-resolution-via-types] — для production-grade
collision disambiguation независимо от span fidelity (требует Ident → ConstDecl
resolution через type-checker).
