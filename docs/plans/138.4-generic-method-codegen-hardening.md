## Фазы (атомарные; каждая: fix codegen → re-attempt cleanup → verify-or-document)

### Ф.1 — G-C: primitive-T generic `.clone()` dispatch (~0.5-1d)

Самый ценный (user подтвердил deep-clone). Проблема: в deep `Vec[T].clone()` тело
`@data[i].clone()` — для primitive T (int/bool/f64/...) у элемента нет user-`@clone`,
mono мис-диспатчит → crash. Fix: `.clone()` на primitive T = **identity-copy** (built-in:
primitives и `value`-типы без user-clone → bitwise self). Generic `.clone()` call должен
резолвить: user-`@clone` если есть, иначе identity для primitive/POD.

Re-attempt: восстановить deep `Vec[T Clone] @clone()` (138.3 Ф.2 форма) + HashMap/Set deep
(138.3 Ф.3). Test plan138_3 t1 (Vec[Point] deep — мутация копии ≠ оригинал) + Vec[int] (identity).
**Commit:** `fix(plan138.4 Ф.1 G-C): primitive-T generic .clone() = identity; unblock deep collection clone`

### Ф.2 — G-A: full-signature overload mangling (generic-type methods) (~0.5-1d)

Проблема: mangling generic-type методов не дизамбигуирует по сигнатуре → read `@index(i)->T`
и write `mut @index(i,val)` коллизят в один C-символ. Fix: mangle generic-method C-name по
**полной сигнатуре** (arity + param-c-types + recv_mutable), на template-fwd-decl И
`register_vec_mono_method` (emit_c.rs:2566-2607) — синхронно с multi-overload mangler.

Re-attempt: `[M-138-mutindex-rename]` `@index_set` → `mut @index` (3-way overload: read scalar,
read range, write). Test plan138_1/plan138_2.
**Commit:** `fix(plan138.4 Ф.2 G-A): full-signature mangling for generic-method overloads; unblock mut @index`

### Ф.3 — G-B: recv-mut overload return-type tiebreak в infer (~0.5d)

Проблема: `infer_expr_c_type` для method-call (emit_c.rs ~30248-30571) фильтрует overload только
по param-types → return-тип всегда первого-объявленного. Fix: tiebreak return-overload по
receiver mutability (использовать `is_obj_mutable`, как mangle_fn 9723).
**Caveat (footgun):** даже с fixed infer, getmut-rename даёт mut-recv-dispatch footgun
(`mut v; v.get(1)` read → mut-overload). Поэтому Ф.3 = **починить инфраструктуру** (полезно
для differing-return-type overloads вообще); но getmut-rename САМ — **отдельное решение**:
рекомендация оставить `@get_mut` distinct (Rust precedent), т.к. footgun реален. Re-attempt
getmut ТОЛЬКО если footgun решаем (напр. mut-recv read дизамбигуируется явно).
**Commit:** `fix(plan138.4 Ф.3 G-B): infer return-type tiebreak by recv-mut for differing-return overloads`

### Ф.4 — G-D: generic-pointee type preservation под right-binding (`*T`) (~0.5-1d)

Проблема: `mut data *T` в generic-поле → pointee `Nova_any*` (T теряется при mono subst).
Fix: V2 §V2.1 right-binding в codegen — `Mut(Pointer(T))` сохраняет generic T в pointee при
мономорфизации (TypeRef::Mut transparent + pointer-pointee subst). Это часть
`[M-118.5-right-binding-migration]`.

Re-attempt: `[M-138.2-vec-type-priv]` part-b `*mut T` → `*T`. Test plan138_*.
**Commit:** `fix(plan138.4 Ф.4 G-D): preserve generic pointee type under right-binding; enable *T`

### Ф.5 — Docs + close

Markers DONE/BLOCKED + commits; spec D216/D230 amends если impl догнал prose; simplifications/
project-creation/discussion-log/memory; push.

## Addendum Ф.6 — G-E: `Self` как ВЛОЖЕННЫЙ generic type-arg (2026-06-15)

> **Статус:** ✅ ЗАКРЫТО (return+param позиции на mono value-инстансах). **Ветка:** `plan-self-nested-generic` (merged main `fc4568c4`).
> **Триггер:** Plan 153.2-Z (zero-cost lazy, `std/collections/vec_iter_zc.nv`) — попытка заменить
> повтор receiver-типа в return адаптер-на-адаптер методах (`-> FilterIter[MapIter[I,T,U], U]`)
> на идиоматичный `Self` (`-> FilterIter[Self, U]`) вскрыла **5-й gap того же домена**, что Ф.1-Ф.4:
> codegen-инфраструктура generic-метод mono НЕ биндила `Self` в `current_type_subst`.

**Gap G-E** | блокировал `[M-138.2-self-in-param]` (расширение исходного param-only маркера) | mono value-generic метод с `Self` в качестве вложенного type-arg Named-типа (`-> MapIter[Self, U, V]` / `g FilterIter[Self, U]`) лоуэрился в расходящееся C-имя | ✅ DONE `60a875a0`.

**Root cause** (эмпирически, как и Ф.1-Ф.4 — отличался от guess'а): call-site return-inference биндил
`Self` *value-aware* (без trailing `*`), но `register_mono_method_instance` (fwd-decl, emit_c.rs ~13946)
и `emit_monomorphized_method` (body, ~14030) строили `current_type_subst` **только из receiver-generics
— без записи `Self`**. → вложенный `Self` промахивался мимо early-lookup (`type_ref_to_c:5265`) и падал
в `"Self"`-арм → POINTER-форма → spurious trailing `*` → `_p` в mangle → fwd-decl ≠ call-site ≠ body
(три разных mono-имени одного метода).

**Fix:** в ОБОИХ местах после установки `current_receiver_type` биндить
`Self → value_aware_generic_c_type("Nova_{recv}*")` в `current_type_subst` через `.entry().or_insert()`
(no-clobber), guard `recv_type.contains("____")` (только mono-инстансы; `value_aware_*` оставляет
heap-generic/non-value формы как есть → top-level heap `-> Self` НЕ затронут). Verified в сген. C:
fwd-decl/call-site/body делят один mono-символ; spurious `_p` = 0.

**Тесты** (`nova_tests/self_nested/`, проверены релизным `nova` через C-codegen):
- `repro.nv` — позитив: вложенный `Self` в RETURN (`-> MapIter[Self,U,V]`) компилируется+рантаймит.
- `repro_param.nv` — позитив: вложенный `Self` в PARAM (`g FilterIter[Self,U]`).
- `repro_explicit.nv` — control: explicit-форма (без `Self`) даёт тот же результат.
- `repro_control.nv` — no-regression guard: heap-generic top-level `-> Self`-chains (НЕ mono value) не сломаны.
- `self_no_receiver_fail.nv` — **негатив** (`// EXPECT_COMPILE_ERROR`): голый `Self` в свободной функции
  (нет receiver'а) отвергается `[E7001] Self type used outside receiver` — биндинг `Self` корректно gated
  на наличие receiver'а (фикс НЕ инжектит `Self` универсально).
- `self_nested_arity_fail.nv` — **негатив**: `MapIt[Self]` (host generic ждёт 3 type-arg, дан 1) отвергается
  `[E7310] expects 3 type arguments` — Self-subst НЕ обходит проверку арности host-типа.

**Stdlib:** `std/collections/vec_iter_zc.nv` — 8 adapter-on-adapter методов (MapIter/FilterIter/FilterMapIter)
переведены на `Self` (`refactor` `9f614e9b`).

**Spec:** [02-types.md → D66 §«Self как вложенный generic type-arg»](../../spec/decisions/02-types.md).

**Residuals** (вне покрытия фикса, трекаются `[M-138.2-self-in-param]` 🟡 ЧАСТИЧНО):
1. `Self` == single-param generic (`VecIter[T]`), использованный как type-arg ВНУТРИ multi-param адаптера
   (`MapIter[Self,T,U]`) — chain-ENTRY методы VecIter ломают codegen (`E_PRIMITIVE_NO_PROTOCOL_METHOD`,
   receiver мис-резолвится в `int`); потому VecIter-entry оставлены explicit (latent limitation).
2. top-level `other Self` param на ERASED-stub heap-generic (отд. code-path `emit_generic_method_erased`)
   + generic-static-method param-subst (`Vec[T].from` `[]T`-stub).
