## Фазы (атомарные; каждая: fix codegen → re-attempt cleanup → verify-or-document)

### Ф.1 — G-C: primitive-T generic `.clone()` dispatch (~0.5-1d)

Самый ценный (user подтвердил deep-clone). Проблема: в deep `Vec[T].clone()` тело
`@data[i].clone()` — для primitive T (int/bool/f64/...) у элемента нет user-`@clone`,
mono мис-диспатчит → crash. Fix: `.clone()` на primitive T = **identity-copy** (built-in:
primitives и `value`-типы без user-clone → bitwise self). Generic `.clone()` call должен
резолвить: user-`@clone` если есть, иначе identity для primitive/POD.

Re-attempt: восстановить deep `Vec[T Clone] @clone()` (138.3 Ф.2 форма) + HashMap/Set deep
(138.3 Ф.3). Test plan138_3 t1 (Vec[Point] deep — мутация копии ≠ оригинал) + Vec[int] (identity).
**Commit:** `fix(plan138.4 Ф.1 G-C): primitive-T generic .clone() = identity; unblock deep collection clone`

### Ф.2 — G-A: full-signature overload mangling (generic-type methods) (~0.5-1d)

Проблема: mangling generic-type методов не дизамбигуирует по сигнатуре → read `@index(i)->T`
и write `mut @index(i,val)` коллизят в один C-символ. Fix: mangle generic-method C-name по
**полной сигнатуре** (arity + param-c-types + recv_mutable), на template-fwd-decl И
`register_vec_mono_method` (emit_c.rs:2566-2607) — синхронно с multi-overload mangler.

Re-attempt: `[M-138-mutindex-rename]` `@index_set` → `mut @index` (3-way overload: read scalar,
read range, write). Test plan138_1/plan138_2.
**Commit:** `fix(plan138.4 Ф.2 G-A): full-signature mangling for generic-method overloads; unblock mut @index`

### Ф.3 — G-B: recv-mut overload return-type tiebreak в infer (~0.5d)

Проблема: `infer_expr_c_type` для method-call (emit_c.rs ~30248-30571) фильтрует overload только
по param-types → return-тип всегда первого-объявленного. Fix: tiebreak return-overload по
receiver mutability (использовать `is_obj_mutable`, как mangle_fn 9723).
**Caveat (footgun):** даже с fixed infer, getmut-rename даёт mut-recv-dispatch footgun
(`mut v; v.get(1)` read → mut-overload). Поэтому Ф.3 = **починить инфраструктуру** (полезно
для differing-return-type overloads вообще); но getmut-rename САМ — **отдельное решение**:
рекомендация оставить `@get_mut` distinct (Rust precedent), т.к. footgun реален. Re-attempt
getmut ТОЛЬКО если footgun решаем (напр. mut-recv read дизамбигуируется явно).
**Commit:** `fix(plan138.4 Ф.3 G-B): infer return-type tiebreak by recv-mut for differing-return overloads`

### Ф.4 — G-D: generic-pointee type preservation под right-binding (`*T`) (~0.5-1d)

Проблема: `mut data *T` в generic-поле → pointee `Nova_any*` (T теряется при mono subst).
Fix: V2 §V2.1 right-binding в codegen — `Mut(Pointer(T))` сохраняет generic T в pointee при
мономорфизации (TypeRef::Mut transparent + pointer-pointee subst). Это часть
`[M-118.5-right-binding-migration]`.

Re-attempt: `[M-138.2-vec-type-priv]` part-b `*mut T` → `*T`. Test plan138_*.
**Commit:** `fix(plan138.4 Ф.4 G-D): preserve generic pointee type under right-binding; enable *T`

### Ф.5 — Docs + close

Markers DONE/BLOCKED + commits; spec D216/D230 amends если impl догнал prose; simplifications/
project-creation/discussion-log/memory; push.

## Итог (2026-06-11)

**Все 4 codegen-gap закрыты, 0 регрессий.** Ключевые архитектурные находки (root causes
отличались от prose-уровневых guess'ов плана — проверены эмпирически, не теоретизированы):

- **Ф.1 G-C** (`88432dd6f02` + `363f4b53788`): мис-диспатч НЕ в `register_vec_mono_method`, а
  в single-key last-wins `method_receivers["clone"]` instance-fallback (emit_c.rs ~22159, lookup
  ТОЛЬКО по имени метода, игнорируя receiver-тип). Fix: `PrimBuiltin::Identity` для primitive-C-типов
  + record/heap identity-clone arm раньше fallback'а; зеркало в `infer_expr_c_type`. Deep Vec/HashMap/Set
  clone восстановлены (per-element `.clone()`); plan138_3 теперь exercises РЕАЛЬНЫЙ deep clone.
- **Ф.2 G-A** (`14e6da56175`): collision в ERASED-template пути (`Nova_Vec_method_*`), НЕ mono. Root:
  `mangle_fn` `want_params` via `type_ref_to_c` (bare T → `nova_int`) расходился с registry pre-pass
  `erased_type_ref_c` (bare T → `void*`). Fix: helper `mangle_want_params(recv, f)` реплицирует erasure
  точно. 3 `@index` overload'а теперь distinct C-символы. Backward-compat: single-overload методы
  сохраняют base C-name → zero mass regression.
- **Ф.3 G-B** (`7c82ff2061c`): widened `recv_and_method` tuple до 4-эл (+ call-site recv-mutability via
  `is_obj_mutable`); multi-overload return-pool tiebreak по `recv_mutable` BEFORE `pool.first()`, guarded
  `pool.len()>1`. INFRA-готовность; getmut остаётся distinct (footgun: на mut receiver plain read
  `v.get(1)` диспатчил бы mut-overload).
- **Ф.4 G-D** (`38360c30d80`): root НЕ generic-T → Nova_any (subst работал), а binding-mut `const`-pointee
  для DEAD prelude-induced `Vec[any]` инстанса (из `...items []any` в print/println) ломал И writable store
  И forward-typedef (`const Nova_any` не проходил `starts_with("Nova_")`). Fix: `field_type_with_binding_mut`
  + `promote_pointer_pointee_mut` — для mut-поля `*T ≡ *mut T` (D33 §2 / D216 §V2.6), applied на bare-template
  И mono-path. `mut data *mut T` → `mut data *T` в vec_owned.nv.

Phase commits: `88432dd6f02` + `363f4b53788` (Ф.1), `14e6da56175` (Ф.2), `7c82ff2061c` (Ф.3),
