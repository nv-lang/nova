# Plan 114.4.4.3 — Runtime HOF через trampoline для const fn

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.6 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — нужно для real first-class const fn (`map(arr, const_fn)`).
> **Оценка:** ~1-2 dev-day.
> **Зависимости:** Plan 114.4.3 V2 (first-class alias baseline).
>
> **Why extracted:** Runtime trampoline generation требует ABI design —
>   как const fn без runtime symbol становится function pointer. Workaround
>   через manual lambda wrapping работает (Plan 114.4.4 Ф.2 friendly
>   error suggests это).

## Scope

Generate runtime trampoline когда const fn используется в runtime
context:
- `arr.map(my_const_fn)` — detect const fn passed as HOF arg.
- Codegen `_const_fn_X_trampoline(runtime_args...)` wrapper.
- Wrapper calls const fn с constants extracted from closure capture.

## Tests

- Positive: arr.map(my_const_fn) — works через automatic trampoline.
- Positive: filter с const fn predicate.

## Acceptance

A1: Const fn passed to runtime HOF works (no manual lambda needed).
A2: Trampoline emitted only when needed (dead const fn not emitted).
