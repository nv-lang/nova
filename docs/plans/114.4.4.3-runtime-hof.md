# Plan 114.4.4.3 — Runtime HOF через trampoline для const fn

> **Создан 2026-06-01** — extracted from Plan 114.4.4 Ф.6 safety hatch.
> **Статус:** 🆕 PLANNED.
> **Приоритет:** P3 — нужно для real first-class const fn (`map(arr, const_fn)`).
> **Оценка:** ~1-2 dev-day.
> **Зависимости:** Plan 114.4.3 V2 (first-class alias baseline).
>
> **Why extracted:** Runtime trampoline generation требует ABI design —
>   как const fn без runtime symbol становится function pointer. Workaround
>   через manual lambda wrapping работает (Plan 114.4.4 Ф.2 friendly
>   error suggests это).

## Scope

Generate runtime trampoline когда const fn используется в runtime
context:
- `arr.map(my_const_fn)` — detect const fn passed as HOF arg.
- Codegen `_const_fn_X_trampoline(runtime_args...)` wrapper.
- Wrapper calls const fn с constants extracted from closure capture.

## Tests

- Positive: arr.map(my_const_fn) — works через automatic trampoline.
- Positive: filter с const fn predicate.

## Acceptance

A1: Const fn passed to runtime HOF works (no manual lambda needed).
A2: Trampoline emitted only when needed (dead const fn not emitted).

### Implementation

- New module `compiler-codegen/src/const_fn_trampoline.rs` (~660 LOC).
- Entry `generate_const_fn_trampolines(module, const_fn_decls, aliases)`.
- 4 steps:
  1. **Collect seeds**: walker over module finds Idents к const fn names
     в non-callee positions (Call.args, Let.value, Assign.value,
     Return.value). Alias resolution: ALIAS → original const fn name.
  2. **Transitive closure**: для каждого seed walk body, find Call
     targets к другим const fn, transitively add to trampoline set.
  3. **Generate trampolines**: clone FnDecl, rename `<name>__trampoline`,
     demote `is_const`/`return_is_const` → false. Walk body: substitute
     `size_of[T]()`/`align_of[T]()` с literal Int; rewrite Call к other
     trampoline-set members на `<other>__trampoline`. Generic const fn
     rejected с `E_CONST_FN_TRAMPOLINE_GENERIC`.
  4. **Rewrite first-class Idents**: replace `Ident("name")` (after alias
     resolution) в non-callee positions с `Ident("name__trampoline")`.

### Pipeline integration (const_fn_eval.rs:rewrite_const_fn_calls)

- Trampoline pass invoked AFTER main walker (inline + intrinsics),
  BEFORE validate + retain.
- `validate_const_fn_runtime_uses` accepts `trampoline_set` param и skip'ает
  names в нём (плюс aliases ссылающиеся на них) — иначе friendly errors
  V3 продолжали бы fail'ить.
- Retain step drops оригинальные fully-const fn names; trampoline names
  не совпадают с фильтром и переживают retain.
