## ⚠ Discovered runtime bug: [M-83.9-arena-pressure-1M]

**Под jobs=16 concurrent test runner**, fiber arena slot stack overflow
~slot 27-151 sporadic для tests с N ≥ 100K (либо длительной workload).

**Repro:** `nova test --gc boehm --filter plan83_stress_armed` (без
`-j 1`) → spawn_stress + memory_bounded RUN-FAIL.

**Standalone (--filter on single test):** all PASS.

**Hypothesis:**
1. Multi-process Boehm GC global lock contention scales nonlinearly when
   multiple `nova test` binaries run параллельно.
2. Fiber arena slot stack pressure — minicoro stack pages могут share
   Boehm reserved memory регион across processes (Windows VirtualAlloc
   commit pages).
3. Test runner subprocess startup overhead under 16 simultaneous
   processes (libuv handle init, GC_INIT, fiber arena init) saturates
   I/O / kernel resources.

**Не блокер для production armed M:N** — single-process workloads pass
к 10⁶+. Это test-runner stress.

**V2 followup options:**
- Test directive `// SERIAL_ONLY` + test runner respects (no concurrent
  scheduling).
- Group в `plan83_stress_armed_serial/` dir с runner-side rule.
- Investigate Boehm cross-process behavior.

Скорее всего стрелять reduces concurrent risk: либо increase `--jobs 1`
для stress directory, либо separate stress run в CI.

## 1. Проблема

Plan 83.4.5.6 §6.4 acceptance criteria:
- **10⁶ spawn** без leak/crash/deadlock
- **10⁵ park-wake** cycles
- **10⁴ cancel** cycles
- TSAN gate (Linux) — 0 races

**Реально landed (Plan 83.4.5.6 partial closure):**
- 10³ spawn (1000) — **3 порядка меньше**
- 10² park-wake (100) — **3 порядка меньше**
- 10¹ cancel (10) — **3 порядка меньше**
- **И все идут с `// ENV NOVA_AUTOARM=0` (cooperative mode), НЕ armed.**

Это покрывает correctness для cooperative path, но **production default
— armed M:N**. Validation production-default'а отсутствует.

Причина перевода тестов в cooperative — performance overhead (180s
timeout на 10K spawn под Windows armed). Plan 83.5+83.6 закрывают этот
gap → можно validate под armed.

### Positive — основной набор:

**`spawn_stress_1M.nv`** (10⁶ no-op spawn, ≤120s acceptance):
```nv
module plan83_stress_armed.spawn_stress_1M

// Plan 83.9: armed M:N production stress.
// 10⁶ spawn без NOVA_AUTOARM=0 directive — production default.
// Acceptance: complete <120s, no leak, no crash, no deadlock.
```

**`park_wake_stress_100k.nv`** (10⁵ park-wake cycles):
```nv
test "100K park-wake via channel ping-pong" {
    use std::channels::*
    let n = 100_000
    let (tx, rx) = chan[int].make()
    let mut received = 0
    supervised {
        spawn {
            let mut i = 0
            while i < n {
                let v = rx.recv()
                received += 1
                i = i + 1
            }
        }
        let mut i = 0
        while i < n {
            tx.send(i)
            i = i + 1
        }
    }
    assert(received == n)
}
```

**`cancel_stress_10k.nv`** (10⁴ supervised cancel cycles):
```nv
use std::runtime::cancel

test "10K supervised cancel cycles — no leak no race" {
    use std::runtime::gc
    let heap_before = gc.heap_size()

    let mut i = 0
    while i < 10_000 {
        let tok: CancelToken[Timeout] = cancel.token()
        supervised(cancel: tok) {
            spawn { /* short body */ }
            // Cancel mid-flight to exercise wake-all path.
            if i mod 2 == 0 {
                tok.cancel()
            }
        }
        i = i + 1
    }

    gc.collect()
    let heap_after = gc.heap_size()
    // No leak per cancel cycle.
    assert(heap_after - heap_before < 1024 * 1024)  // <1MB growth bound
}
```

bounded:
```nv
test "10K cycles of {spawn,park,cancel} — heap bounded" {
    use std::runtime::gc
    let max_growth = 0
    let baseline = gc.heap_size()
    let mut i = 0
    while i < 10_000 {
        supervised {
            spawn { }
            spawn { }
            spawn { }
        }
        if i mod 1000 == 0 {
            gc.collect()
            let growth = gc.heap_size() - baseline
            if growth > max_growth { max_growth = growth }
        }
        i = i + 1
    }
    // Bound: pool capacity + overhead = ~10MB.
    assert(max_growth < 10 * 1024 * 1024)
}
```

## 5. Риски

- **Slow runners (CI)** — установить generous EXPECT_TIMEOUT_MS, опционально
  отметить тесты как nightly-only через test-runner directive.
- **macOS resource limits** — может быть низкий ulimit -n. Use stress
  tests requiring `< 100 concurrent fibers` для consistent CI.
- **Memory growth on Windows** — fiber arena 8MB × pool — heap отметка
  может быть >10MB; tune acceptance per platform.

## 6. Зависимости

Без Plan 83.5 + 83.6 эти тесты будут timeout (current armed spawn-cost
~10-100µs × 10⁶ = 10-100s минимум, не считая park/wake/cancel
overhead). Поэтому 83.9 GATED на 83.5+83.6 готовностью.
