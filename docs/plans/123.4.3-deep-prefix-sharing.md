## 1. Контекст

V4 / V4.1 эмитили per-chain `_at_<full>_chain = @<full>` lets — каждый
chain независимо walk'ит root. V4.2 добавил length-2 prefix sharing:
группы ≥2 chains, share длину-2 prefix `@<a>.<b>`, получают shared
`_at_<a>_<b>_pre`, и per-chain lets начинают через `<prefix-local>.tail`.

V4.3 расширяет sharing на **length 3+** через iterative deepening +
**parent prefix chaining** — deeper prefix references shallower
parent, давая O(depth) sharing depth instead of O(2). Реалистичные
Nova-модули с deep record-вложенностью (`Mid → Inner → Leaf` chains)
получают существенную экономию.

## 2. Изменения

### 2.1 `compiler-codegen/src/field_cache.rs`

**Новый struct:**

```rust
#[derive(Debug, Clone)]
pub(crate) struct PrefixInfo {
    pub(crate) name: String,
    pub(crate) span: crate::diag::Span,
    pub(crate) parent: Option<Vec<String>>,  // parent prefix-path
}
```

`parent: None` → value-выражение = `@<full prefix path>` (V4.2 case);
`parent: Some(p)` → value = `<parent_local>.<remaining_segments>` chain.

**Replaced function `compute_chain_prefix_sharing`** (~60 LOC):
итеративный deepening по prefix-длине L = 2..max_path_len.

**Replaced helper `find_chain_shared_prefix`** (~15 LOC):
walks `(2..path.len()).rev()` — picks LONGEST covering prefix, NOT
just length-2.

**New helper `find_longest_existing_parent`** (~10 LOC):
walks `(2..prefix.len()).rev()` — для deeper prefix находит shallower
prefix в `out`.

**Emission loop** (`chain_cache_fn_impl`, ~20 LOC):
- Prefix lets sort key изменён с `lex` на `(len, lex)` — shorter-first.
- Prefix value builds через `build_chain_from_ident(parent_local, tail, span)`
  если `parent: Some` else через `build_chain_expr(path, span)`.

### 2.2 Unit tests (compiler-codegen/src/field_cache.rs)

Добавлено 6 unit tests (`v43_*`):

- `v43_length3_shared_prefix_emitted` — два @a.b.c.x + @a.b.c.y →
  `_at_a_b_pre` + `_at_a_b_c_pre`.
- `v43_length3_prefix_references_length2_parent` — verifies
  `_at_a_b_c_pre` value-expr = `Member { Ident("_at_a_b_pre"), "c" }`.
- `v43_chain_let_uses_longest_prefix_cover` — `_at_a_b_c_x_chain`
  value = `Member { Ident("_at_a_b_c_pre"), "x" }` (NOT
  `Member { Member { Ident("_at_a_b_pre"), "c" }, "x" }`).
- `v43_no_deep_prefix_when_only_one_chain_at_depth3` — три chains
  через distinct middle segments → ТОЛЬКО length-2 prefix emitted.
- `v43_length4_prefix_chains_through_parents` — @a.b.c.d.e +
  @a.b.c.d.f с chain_max_depth=6 → length-2 → length-3 → length-4
  prefix chain.
- `v43_emission_order_shorter_first` — verifies position(length-2) <
  position(length-3) < position(length-4) в `b.stmts`.
