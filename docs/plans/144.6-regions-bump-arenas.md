## Цель (ПЕРЕСМОТРЕНА — §7.6 moving-вердикт)
**General moving в compile-to-C НЕ строить** (прецеденты Mercury/Bigloo/LLVM-ShadowStackGC +
interior-`str.ptr` fixup-налог; категория односторонняя). Вместо — **regions / bump-арены** для
locality / throughput / анти-фрагментации **без fixup-налога**.

Moving — только узкое **pinned, НЕ-interior** подмножество, и только если конкретная workload
**докажет** нужду. Codegen reload (инвариант §7.3 #3) + base+offset fixup (Q7) — лишь для этого
подмножества. Представление base+offset заложено в ABI ещё на [144.0.1](144.0.1-gate-frame-abi-object-start.md)
(чтобы не запереть moving навсегда), но коллектор остаётся non-moving.

## Acceptance criteria
- Regions/bump-арена даёт compaction-подобный выигрыш **без pointer-fixup**.
- *(опц.)* pinned non-interior copying-nursery зелёный — **только на доказанной workload**.
- «без упрощений как для прода» — обязательный критерий.
