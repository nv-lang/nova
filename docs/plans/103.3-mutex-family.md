## –Ч–∞—З–µ–Љ

–Ґ–µ–Ї—Г—Й–Є–є `Mutex` baseline (`std/runtime/sync.nv:92-131`) вАФ minimal:
`lock/unlock/try_lock`, fair FIFO, NOT reentrant. **Industry gaps:**

| Gap | Impact |
|---|---|
| **`try_lock_for(Duration)`** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | –Э–µ–≤–Њ–Ј–Љ–Њ–ґ–љ–Њ write–∞ bounded-wait pattern (deadline-driven) |
| **`is_locked()`** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | –Э–µ–≤–Њ–Ј–Љ–Њ–ґ–љ–∞ observability (debugging, metrics) |
| **`with_lock(fn)` helper** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | –Я—А–Є–љ—Г–ґ–і–∞–µ—В –Ї `lock()/unlock()`; –Ј–∞–±—Л—В—Л–є unlock вАФ runtime deadlock; `defer m.unlock()` –њ–Њ–Љ–Њ–≥–∞–µ—В –љ–Њ verbose |
| **Fairness mode opt-in** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | Always fair FIFO вАФ –Њ–±–µ—Б–њ–µ—З–Є–≤–∞–µ—В predictability, –љ–Њ perf-critical pattern (e.g., spin-lock-like contention) –љ–µ –Є–Љ–µ–µ—В escape |
| **`RwLock`** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | Read-heavy patterns force Mutex bottleneck |
| **`ReentrantMutex`** –Њ—В—Б—Г—В—Б—В–≤—Г–µ—В | Legacy-style recursive locking –љ–µ–≤–Њ–Ј–Љ–Њ–ґ–µ–љ (use case: migration –Є–Ј codebase'–Њ–≤ —Б reentrant Mutex –њ—А–µ–і–њ–Њ–ї–Њ–ґ–µ–љ–Є–µ–Љ) |

**M15 (master):** V1 ships –±–µ–Ј consume-guards; helpers `with_lock(fn)`,
`with_read(fn)`, `with_write(fn)` вАФ preferred pattern. V2 (Plan 103.9)
–њ–µ—А–µ–≤–Њ–і–Є—В –љ–∞ `MutexGuard consume`.

## Scope

### In scope

**`Mutex` (hardened):**
- Existing: `new()`, `lock()`, `unlock()`, `try_lock()` вАФ preserved.
- **Add:** `try_lock_for(timeout Duration) -> bool` (returns true if
  acquired within timeout).
- **Add:** `is_locked() -> bool` (best-effort observability; NOT
  reliable –і–ї—П CAS-test).
- **Add:** `with_lock[R](body fn() -> R) -> R` helper вАФ closure form,
  defer-based release. **Preferred pattern –≤ V1** (M15).
- **Add:** unfair-mode opt-in constructor `Mutex.new_unfair() -> Self`
  (–і–ї—П perf-critical contention; documented trade-off).
- **Documented invariants:** still NOT reentrant; double-unlock = debug
  panic (NOVA_SYNC_ASSERT –≤ sync_primitives.h:236).

**`RwLock`:**
- `new() -> Self`
- `read()` / `try_read()` / `try_read_for(timeout Duration) -> bool` /
  `read_unlock()`
- `write()` / `try_write()` / `try_write_for(timeout Duration) -> bool` /
  `write_unlock()`
- `with_read[R](body fn() -> R) -> R` helper
- `with_write[R](body fn() -> R) -> R` helper
- **Fairness:** writer-priority default (M7) вАФ escape writer-starvation;
  `RwLock.new_reader_priority()` opt-in –і–ї—П read-heavy workloads.
- Internal state: read counter (AtomicI32) + writer flag (AtomicBool) +
  writer-waiting flag (AtomicBool); waiter list –њ–Њ–і guard mutex.

**`ReentrantMutex` (opt-in):**
- `new() -> Self`
- `lock()` / `unlock()` / `try_lock()` / `try_lock_for(...)` вАФ same surface
  as Mutex
- Reentrant logic: owner-fiber-id tracking (AtomicU64) + lock-count
  (AtomicI32). lock() –Њ—В —В–Њ–≥–Њ –ґ–µ fiber'–∞ вАФ incr count; unlock() вАФ decr,
  release –Ї–Њ–≥–і–∞ count = 0.
- `with_lock` helper.
- **WARN –љ–∞ mix:** `ReentrantMutex` + `Condvar` (Plan 103.4) =
  surprising semantics (Java has historical pitfalls). Documented:
  `Condvar.wait()` releases lock entirely (count вЖТ 0), wake re-acquires
  —Б count = 1. AI-friendly diagnostic `W_REENTRANT_CONDVAR_RECOMMEND`.

### Out of scope

- `MutexGuard consume` вАФ V2 sub-plan 103.9 (gated Plan 100.7).
- Mutex `data-carrying` (Rust `Mutex<T>`) вАФ M4 reject.
- Mutex poisoning (Rust `LockResult`) вАФ M5 reject.
- `RwLock` upgradeable read вЖТ write вАФ –Њ—В–і–µ–ї—М–љ—Л–є future plan (Rust
  parking_lot has it; Nova v1 –±–µ–Ј, complex semantics).
- `Mutex.lock_async` (cancellable lock) вАФ handled —З–µ—А–µ–Ј `cancel_scope`
  pattern —Г–ґ–µ (mutex park —З–µ—А–µ–Ј nova_sched, cancellation propagates).

## API design

### `Mutex` (extended)

```nova
module runtime.sync

/// Fair FIFO fiber-aware mutex. NOT reentrant.
#stable(since = "0.1")
export external fn Mutex.new() -> Self

/// Unfair (LIFO-leaning, perf-friendly) opt-in variant.
/// Use case: high-contention short critical sections where FIFO ordering
/// not required. Trade-off: starvation possible вАФ pick only –њ–Њ—Б–ї–µ benchmark.
#stable(since = "0.1")
export external fn Mutex.new_unfair() -> Self

#stable(since = "0.1")
export external fn Mutex mut @lock()
#stable(since = "0.1")
export external fn Mutex mut @unlock()
#stable(since = "0.1")
export external fn Mutex mut @try_lock() -> bool

/// Try to acquire within timeout. Returns true if acquired, false if
/// timeout expired. Uses Plan 22 sleep/timer infrastructure.
#stable(since = "0.1")
export external fn Mutex mut @try_lock_for(timeout Duration) -> bool

/// Best-effort observability. NOT for CAS-style test-and-set вАФ
/// state –Љ–Њ–ґ–µ—В measurement вЖТ action race.
#stable(since = "0.1")
export external fn Mutex @is_locked() -> bool

/// PREFERRED PATTERN (M15 V1). Closure-form lock/unlock —Б defer-based
/// release вАФ –љ–Є–Ї–Њ–≥–і–∞ –љ–µ –Ј–∞–±—Л–≤–∞–µ—В—Б—П unlock –і–∞–ґ–µ –њ—А–Є panic –≤ body.
///
/// V2 (Plan 103.9): —Н—В–Њ—В helper —Б—В–∞–љ–Њ–≤–Є—В—Б—П thin wrapper –љ–∞–і
/// `MutexGuard consume` API; user code –љ–µ –Љ–µ–љ—П–µ—В—Б—П.
///
/// # Examples
///
/// ```nova
/// let m = Mutex.new()
/// let result = m.with_lock { ||
///     compute_under_lock()
/// }
/// ```
#stable(since = "0.1")
export fn Mutex mut @with_lock[R](body fn() -> R) -> R {
    self.lock()
    defer self.unlock()
    body()
}
```

### `ReentrantMutex`

```nova
/// Reentrant mutex: same fiber can lock multiple times –±–µ–Ј deadlock.
/// Tracks owner-fiber-id + recursion count.
///
/// **Use case:** legacy migration, callbacks-into-locked-context patterns.
/// **Recommended default**: regular `Mutex` (deadlock-detection early).
///

/// (count вЖТ 0), wake re-acquires —Б count = 1 вАФ original recursion

#stable(since = "0.1")
export external fn ReentrantMutex mut @lock()
#stable(since = "0.1")
export external fn ReentrantMutex mut @unlock()
#stable(since = "0.1")
export external fn ReentrantMutex mut @try_lock() -> bool
#stable(since = "0.1")
export external fn ReentrantMutex mut @try_lock_for(timeout Duration) -> bool

/// Returns recursion depth –і–ї—П current fiber, 0 –µ—Б–ї–Є –љ–µ locked —Н—В–Є–Љ fiber'–Њ–Љ.
#stable(since = "0.1")
export external fn ReentrantMutex @lock_count() -> int

#stable(since = "0.1")
export fn ReentrantMutex mut @with_lock[R](body fn() -> R) -> R {
    self.lock()
    defer self.unlock()
    body()
}
```

### –§.2 вАФ RwLock impl (~0.6d)

- `nova_rt/sync_primitives.h`: `Nova_RwLock` struct (mu, read_count,
  write_locked flag, write_waiting flag, waiter lists).
- Algorithm (writer-priority):
  - `read()`: if write_locked or write_waiting вЖТ park reader; else
    incr read_count.
  - `write()`: set write_waiting=true, wait for read_count=0 and
    write_locked=false вЖТ set write_locked=true.
  - `read_unlock()`: decr read_count; if 0 and write_waiting вЖТ wake
    one writer.
  - `write_unlock()`: write_locked=false; if write_waiters вЖТ wake one
    writer; else wake all readers.
- `reader_priority` flag: skip write_waiting check on read() path.
- `try_read`/`try_write`/`try_*_for(Duration)`: short-circuit –±–µ–Ј park.
- `with_read[R]` / `with_write[R]` Nova-body helpers.

Acceptance: 6 positive tests (multiple readers, single writer mutual
exclusion, writer-priority, reader-priority opt-in, try_read timeout).
