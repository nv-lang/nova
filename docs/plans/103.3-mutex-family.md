## ⚠ Discovery from Plan 103.5 (apply at impl time)

**NOVA_SYNC_ASSERT — no-op в Dev mode** (`compiler-codegen/nova_rt/
sync_primitives.h:43-53`):

```c
#ifdef NOVA_DEBUG
#  define NOVA_SYNC_ASSERT(cond, msg) ...abort()...
#else
#  define NOVA_SYNC_ASSERT(cond, msg) ((void)0)   // NO-OP в Dev!
#endif
```

Plan 103.5 обнаружил: `Nova_Once_method_done` использовал
`NOVA_SYNC_ASSERT(o->state == NOVA_ONCE_RUNNING, ...)` → silent no-op
в Dev → undefined behavior при misuse. **Fix в 103.5:** unconditional
state check через `Nova_Fail_fail + nova_throw`, независимо от

**Применить тот же pattern в Plan 103.3:**
- **`Nova_Mutex_method_unlock` (sync_primitives.h:236):**
  `NOVA_SYNC_ASSERT(m->locked, "Mutex.unlock() called on unlocked mutex")`

  owner mismatch + lock_count > 0 invariants — unconditional check.
- **`Nova_RwLock_method_read_unlock` / `write_unlock` (новые, Ф.2):**
  lock-kind mismatch + reader_count > 0 / write_locked — unconditional.

**Acceptance criterion (новый):** все state invariants в Plan 103.3

## Зачем

Текущий `Mutex` baseline (`std/runtime/sync.nv:92-131`) — minimal:
`lock/unlock/try_lock`, fair FIFO, NOT reentrant. **Industry gaps:**

| Gap | Impact |
|---|---|
| **`try_lock_for(Duration)`** отсутствует | Невозможно writeа bounded-wait pattern (deadline-driven) |
| **`is_locked()`** отсутствует | Невозможна observability (debugging, metrics) |
| **`with_lock(fn)` helper** отсутствует | Принуждает к `lock()/unlock()`; забытый unlock — runtime deadlock; `defer m.unlock()` помогает но verbose |
| **Fairness mode opt-in** отсутствует | Always fair FIFO — обеспечивает predictability, но perf-critical pattern (e.g., spin-lock-like contention) не имеет escape |
| **`RwLock`** отсутствует | Read-heavy patterns force Mutex bottleneck |
| **`ReentrantMutex`** отсутствует | Legacy-style recursive locking невозможен (use case: migration из codebase'ов с reentrant Mutex предположением) |

**M15 (master):** V1 ships без consume-guards; helpers `with_lock(fn)`,
`with_read(fn)`, `with_write(fn)` — preferred pattern. V2 (Plan 103.9)
переводит на `MutexGuard consume`.

## Scope

### In scope

**`Mutex` (hardened):**
- Existing: `new()`, `lock()`, `unlock()`, `try_lock()` — preserved.
- **Add:** `try_lock_for(timeout Duration) -> bool` (returns true if
  acquired within timeout).
- **Add:** `is_locked() -> bool` (best-effort observability; NOT
  reliable для CAS-test).
- **Add:** `with_lock[R](body fn() -> R) -> R` helper — closure form,
  defer-based release. **Preferred pattern в V1** (M15).
- **Add:** unfair-mode opt-in constructor `Mutex.new_unfair() -> Self`
  (для perf-critical contention; documented trade-off).
- **Documented invariants:** still NOT reentrant; double-unlock = debug
  panic (NOVA_SYNC_ASSERT в sync_primitives.h:236).

**`RwLock`:**
- `new() -> Self`
- `read()` / `try_read()` / `try_read_for(timeout Duration) -> bool` /
  `read_unlock()`
- `write()` / `try_write()` / `try_write_for(timeout Duration) -> bool` /
  `write_unlock()`
- `with_read[R](body fn() -> R) -> R` helper
- `with_write[R](body fn() -> R) -> R` helper
- **Fairness:** writer-priority default (M7) — escape writer-starvation;
  `RwLock.new_reader_priority()` opt-in для read-heavy workloads.
- Internal state: read counter (AtomicI32) + writer flag (AtomicBool) +
  writer-waiting flag (AtomicBool); waiter list под guard mutex.

**`ReentrantMutex` (opt-in):**
- `new() -> Self`
- `lock()` / `unlock()` / `try_lock()` / `try_lock_for(...)` — same surface
  as Mutex
- Reentrant logic: owner-fiber-id tracking (AtomicU64) + lock-count
  (AtomicI32). lock() от того же fiber'а — incr count; unlock() — decr,
  release когда count = 0.
- `with_lock` helper.
- **WARN на mix:** `ReentrantMutex` + `Condvar` (Plan 103.4) =
  surprising semantics (Java has historical pitfalls). Documented:
  `Condvar.wait()` releases lock entirely (count → 0), wake re-acquires
  с count = 1. AI-friendly diagnostic `W_REENTRANT_CONDVAR_RECOMMEND`.

### Out of scope

- `MutexGuard consume` — V2 sub-plan 103.9 (gated Plan 100.7).
- Mutex `data-carrying` (Rust `Mutex<T>`) — M4 reject.
- Mutex poisoning (Rust `LockResult`) — M5 reject.
- `RwLock` upgradeable read → write — отдельный future plan (Rust
  parking_lot has it; Nova v1 без, complex semantics).
- `Mutex.lock_async` (cancellable lock) — handled через `cancel_scope`
  pattern уже (mutex park через nova_sched, cancellation propagates).

## API design

### `Mutex` (extended)

```nova
module runtime.sync

/// Fair FIFO fiber-aware mutex. NOT reentrant.
#stable(since = "0.1")
export external fn Mutex.new() -> Self

/// Unfair (LIFO-leaning, perf-friendly) opt-in variant.
/// Use case: high-contention short critical sections where FIFO ordering
/// not required. Trade-off: starvation possible — pick only после benchmark.
#stable(since = "0.1")
export external fn Mutex.new_unfair() -> Self

#stable(since = "0.1")
export external fn Mutex mut @lock()
#stable(since = "0.1")
export external fn Mutex mut @unlock()
#stable(since = "0.1")
export external fn Mutex mut @try_lock() -> bool

/// Try to acquire within timeout. Returns true if acquired, false if
/// timeout expired. Uses Plan 22 sleep/timer infrastructure.
#stable(since = "0.1")
export external fn Mutex mut @try_lock_for(timeout Duration) -> bool

/// Best-effort observability. NOT for CAS-style test-and-set —
/// state может measurement → action race.
#stable(since = "0.1")
export external fn Mutex @is_locked() -> bool

/// PREFERRED PATTERN (M15 V1). Closure-form lock/unlock с defer-based
/// release — никогда не забывается unlock даже при panic в body.
///
/// V2 (Plan 103.9): этот helper становится thin wrapper над
/// `MutexGuard consume` API; user code не меняется.
///
/// # Examples
///
/// ```nova
/// let m = Mutex.new()
/// let result = m.with_lock { ||
///     compute_under_lock()
/// }
/// ```
#stable(since = "0.1")
export fn Mutex mut @with_lock[R](body fn() -> R) -> R {
    self.lock()
    defer self.unlock()
    body()
}
```

### `RwLock`

```nova
/// Fiber-aware reader-writer lock. Writer-priority default (M7).
/// Use case: read-heavy shared data (config snapshot, computed cache).
#stable(since = "0.1")
export external fn RwLock.new() -> Self

/// Reader-priority opt-in (allows readers to starve writers if many
/// short reads keep coming). Use case: extreme read-heavy workloads.
#stable(since = "0.1")
export external fn RwLock.new_reader_priority() -> Self

#stable(since = "0.1")
export external fn RwLock mut @read()
#stable(since = "0.1")
export external fn RwLock mut @read_unlock()
#stable(since = "0.1")
export external fn RwLock mut @try_read() -> bool
#stable(since = "0.1")
export external fn RwLock mut @try_read_for(timeout Duration) -> bool

#stable(since = "0.1")
export external fn RwLock mut @write()
#stable(since = "0.1")
export external fn RwLock mut @write_unlock()
#stable(since = "0.1")
export external fn RwLock mut @try_write() -> bool
#stable(since = "0.1")
export external fn RwLock mut @try_write_for(timeout Duration) -> bool

#stable(since = "0.1")
export fn RwLock mut @with_read[R](body fn() -> R) -> R {
    self.read()
    defer self.read_unlock()
    body()
}
#stable(since = "0.1")
export fn RwLock mut @with_write[R](body fn() -> R) -> R {
    self.write()
    defer self.write_unlock()
    body()
}

#stable(since = "0.1")
export external fn RwLock @reader_count() -> int  // best-effort
#stable(since = "0.1")
export external fn RwLock @is_write_locked() -> bool  // best-effort
```

### `ReentrantMutex`

```nova
/// Reentrant mutex: same fiber can lock multiple times без deadlock.
/// Tracks owner-fiber-id + recursion count.
///
/// **Use case:** legacy migration, callbacks-into-locked-context patterns.
/// **Recommended default**: regular `Mutex` (deadlock-detection early).
///
/// **Interaction с Condvar:** `Condvar.wait()` releases ENTIRE lock
/// (count → 0), wake re-acquires с count = 1 — original recursion
/// depth NOT preserved. AI-warning `W_REENTRANT_CONDVAR_RECOMMEND`
/// если mix detected.
#stable(since = "0.1")
export external fn ReentrantMutex.new() -> Self

#stable(since = "0.1")
export external fn ReentrantMutex mut @lock()
#stable(since = "0.1")
export external fn ReentrantMutex mut @unlock()
#stable(since = "0.1")
export external fn ReentrantMutex mut @try_lock() -> bool
#stable(since = "0.1")
export external fn ReentrantMutex mut @try_lock_for(timeout Duration) -> bool

/// Returns recursion depth для current fiber, 0 если не locked этим fiber'ом.
#stable(since = "0.1")
export external fn ReentrantMutex @lock_count() -> int

#stable(since = "0.1")
export fn ReentrantMutex mut @with_lock[R](body fn() -> R) -> R {
    self.lock()
    defer self.unlock()
    body()
}
```

### Ф.2 — RwLock impl (~0.6d)

- `nova_rt/sync_primitives.h`: `Nova_RwLock` struct (mu, read_count,
  write_locked flag, write_waiting flag, waiter lists).
- Algorithm (writer-priority):
  - `read()`: if write_locked or write_waiting → park reader; else
    incr read_count.
  - `write()`: set write_waiting=true, wait for read_count=0 and
    write_locked=false → set write_locked=true.
  - `read_unlock()`: decr read_count; if 0 and write_waiting → wake
    one writer.
  - `write_unlock()`: write_locked=false; if write_waiters → wake one
    writer; else wake all readers.
- `reader_priority` flag: skip write_waiting check on read() path.
- `try_read`/`try_write`/`try_*_for(Duration)`: short-circuit без park.
- `with_read[R]` / `with_write[R]` Nova-body helpers.

Acceptance: 6 positive tests (multiple readers, single writer mutual
exclusion, writer-priority, reader-priority opt-in, try_read timeout).

### Ф.3 — ReentrantMutex impl (~0.4d)

- `nova_rt/sync_primitives.h`: `Nova_ReentrantMutex` struct (mu,
  owner_fiber_id, lock_count, waiter list).
- `lock()`: if owner == current fiber → incr count; else acquire (park
  если занято) → set owner, count=1.
- `unlock()`: assert owner == current fiber; decr count; if count = 0
  → release owner, wake next waiter.
- Owner-fiber-id: получить через `_nova_active_slot` + `_nova_active_scope`
  hash (или dedicated fiber UUID если есть).
- `lock_count()`: relaxed-load.
- `with_lock` Nova-body helper.

Acceptance: 4 positive tests (recursive lock from same fiber, other
fiber blocked, unlock symmetry).

## Risks + mitigations

| Риск | Mitigation |
|---|---|
| RwLock writer-priority может deadlock'ить read-heavy workloads | `new_reader_priority()` opt-in; lint guidance в D173 (103.7) |
| `try_lock_for(Duration)` использует libuv timer — может race condition с уже-acquired lock | Pattern proven (channel sleep, Plan 22); test 11+12 explicitly covers race |
| ReentrantMutex + Condvar interaction (release count entirely) — surprise | `W_REENTRANT_CONDVAR_RECOMMEND` warning (Plan 103.4 hook); D169 phsyical contract documented |
| `with_lock[R]` generic R — Plan 99 routing pattern может conflict | Тест integration в Ф.1; fallback на non-generic `with_lock(fn () -> ())` если generic breaks |
| Unfair mutex starvation — кто-то может never acquire | Acceptable; docs warn (opt-in only после benchmark); `new_unfair()` явная intent |
| Owner-fiber-id для ReentrantMutex может collision (если fiber recycled) | Use slot+scope combo + monotonic spawn-id (если existing); fallback на dedicated UUID — отдельный sub-decision Ф.3 |
| Backward compat ломается | Ф.0 + smoke test sync_mutex.nv PASS перед commit |

### Existing Mutex C-impl

**Расположение:** `compiler-codegen/nova_rt/sync_primitives.h:881-980` (НЕ
:185-252 как в исходном плане — line-numbers устарели после 103.x merges).

**Архитектура:**
- `typedef struct { nova_mutex_t mu; bool locked; NovaMutexWaiter* head, tail; } Nova_Mutex;`
  — fair FIFO via doubly-linked waiter list.
- `NovaMutexWaiter` хранит `scope + slot + next + prev`.
- `lock()`: fast-path try, иначе non-fiber spin (CPU yield) или fiber
  `nova_sched_park_with_unlock` (lost-wakeup-free).
- `unlock()`: direct ownership transfer следующему waiter'у (no
  thundering herd) — `locked` остаётся `true`, fiber wake'ится.

**⚠ Подтверждённый баг (apply pattern из 103.5):**
- Строка `964`: `NOVA_SYNC_ASSERT(m->locked, "Mutex.unlock() called on an unlocked mutex")`
  — no-op в Dev mode (NOVA_SYNC_ASSERT = #ifdef NOVA_DEBUG only).
- **Fix в Ф.1:** unconditional `if (!m->locked) { Nova_Fail_fail(...); nova_throw(...); }`
  как в `Nova_Once_method_done` (sync_primitives.h:446 после 103.5 fix).

### libuv `uv_timer_t` pattern (Plan 22)

**Reference impl:** `channels.h:1280-1334` (`Nova_Time_after`).

Pattern для `try_lock_for` (Ф.1):
```c
uv_timer_t timer;
uv_timer_init(nova_current_loop(), &timer);
timer.data = waiter_state;
uv_timer_start(&timer, _timeout_cb, delay_ms, 0);
// ... park fiber ...
// On wake (acquired): uv_timer_stop + uv_close.
// On timeout (_timeout_cb fires): mark waiter cancelled, wake, return false.
```

Cleanup via `uv_close((uv_handle_t*)&timer, close_cb)` — handle ownership
переходит к libuv; не GC-managed (raw malloc по аналогии с
`NovaAfterState`, Plan 44.1 R8).

### Owner fiber-id mechanism (для ReentrantMutex Ф.3)

**Существующие thread-locals (effects.c:167-168, 184-185):**
```c
__declspec(thread) NovaFiberQueue* _nova_active_scope = NULL;
__declspec(thread) int             _nova_active_slot  = -1;
```

**Uniqueness анализ:** `(scope, slot)` pair уникален для активного
fiber'а только пока fiber жив. После завершения slot переиспользуется.
Для `ReentrantMutex` это безопасно: если fiber A locked → completed
→ fiber B берёт тот же slot → B lock'ает same mutex → видит count > 0 от
A → ВЕРНО WRONG (B != A but same identity).

**Решение для Ф.3:** хранить полную (scope_ptr, slot_idx) пару как
owner-id (NULL/–1 = unlocked); проверять equality. Risk
переиспользования slot'а внутри Mutex'а — невозможен пока он locked
(fiber жив, slot reserved). Если нужна 100% protection — UUID/sequence
counter в fiber control block (defer to Plan 103.7 final).
