## ⚠️ ГЛАВНЫЙ РИСК (урок 172.1.2, simplifications.md) — decouple, НЕ extend
Расширение общего `infer_expr_type` (types/mod.rs:~9646) для richer-инференса **РЕГРЕССО-ОПАСНО**: прошлая
сессия (172.1.2 STEP 1) получила **4 регрессии** — (a) f3 chain-receiver → ложный `[E_EXTENSION_METHOD_NEEDS_IMPORT]`;
(b) inline-rippled binding-тип флипнул `[]T`/`Vec[T]` spelling → ранний Vec-mono → **conservative-GC segfault (plan154)**.
Решение тогда: отдельная `infer_method_call_channel_type` (канал-only), `infer_expr_type` НЕТРОНУТ. **ФАЗА 2 ОБЯЗАНА
следовать тому же паттерну: channel-аннотация в producer-блоке (f1_expr:6374-6388) через ОТДЕЛЬНУЮ
type-computation, НЕ через extend общего `infer_expr_type`** — иначе вскроется тот же perturbation-класс.

Producer-блок УЖЕ есть (f1_expr:6374-6388): `if matches!(e.kind, Block|If|IfLet|Match|Path) { if let Some(tr) =
infer_expr_type(e) { resolved_types_buf.insert(e.id, from_type_ref(tr)) } }` — **inert**, т.к. infer_expr_type=None
для этих kind. Активация = дать producer'у тип БЕЗ extend-perturbation (decoupled helper).

## Ранжированный план (synthesis)

### STEP 1 — Path (numeric-type-constant + module/enum-qualified)  ~121 сайтов
- **Legacy:** `numeric_type_constant_mapping(parts)` (emit_c.rs:36527) → c_ty (`int.MAX`→nova_int, `u8.MIN`→nova_byte,
  `f64.EPSILON`→nova_f64). Прямая таблица, НЕ infer.
- **DECOUPLED-подход (рекоменд.):** в producer для Path вычислять primitive ResolvedType ПРЯМО через
  checker-эквивалент numeric_type_constant_mapping (НЕ добавлять Path-арм в `infer_expr_type` — perturbation).
  Нужен checker-side numeric-constant-резолвер (проверить, есть ли; если нет — портировать таблицу).
  Byte-identical (та же таблица). module/enum-qualified Path — отдельный gated subset.
- Risk: low (если decoupled). Coverage: 121.

### STEP 2 — TurboFish (delegate-to-base)  ~448 (но residual мал — ctor-turbofish УЖЕ seeded number_exprs)
- **Legacy:** `infer_expr_c_type(base)` (emit_c.rs:36523) — делегация в **codegen**-инференс base.
- **⚠️ НАХОДКА:** наивный decouple `infer_expr_type(base)` (checker) **INERT** — для type-name base (`Foo[T]`,
  base=Foo) checker `infer_expr_type(Foo)`=None (scope не содержит type-name), тогда как codegen
  `infer_expr_c_type(Foo)` даёт C-тип. checker ≠ codegen для type-base. → TurboFish НЕ чистый quick-win;
  нужен checker-side type-name resolver ИЛИ оставить legacy для TurboFish. Coverage residual мал → низкий приоритет.

### STEP 3 — Member(field-read) + Index — RECEIVER-WIDENING (MOVE A)  ДОМИНИРУЮЩИЙ остаток Member
- Field-read УЖЕ в канале (RANK 2 Record:8438/NamedTuple:8581 gated primitive_gate/concrete_value_named).
- **Остаток = field-on-non-resolvable-receiver** (`a.b.c`, `xs[i].field`, method-chain): f3_check_member бейлит
  на `infer_expr_type(obj)=None` (:8311), т.к. infer_expr_type НЕТ Member/Index-receiver-арма.
- **MOVE A:** дать infer_expr_type Member/Index-receiver-резолв (для concrete Named record/named-tuple receiver)
  → unblock'ает существующий primitive_gate insert. **⚠️ ЭТО ТОЧНО RANK 4 / 172.1.2 perturbation-класс** —
  extend infer_expr_type Member/Index рипплит в extension-policy + GC-layout. HIGH-risk; decouple обязателен;
  ПОЛНЫЙ регресс. Gated subset: container-field (`[]T`→`[]int`) + residual type-param (`[M-172.1-typeparam-infer-erasure]`) — DEFER (mono-hazard §9).

### STEP 4 — Block (primitive-trailing copy)  ~63

  Ident, связанный ВНУТРИ блока, нет в outer scope → None (safe-skip). Concrete-trailing only.
