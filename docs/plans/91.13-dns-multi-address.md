# Plan 91.13 — DNS Multi-Address API (vtable erasure fix)

**Status:** ✅ CLOSED 2026-06-16  
**Branch:** plan-91-13 (worktree: D:\Sources\nv-lang\nova-p91-13)

## Goal

1. Fix vtable Ok-type erasure in `emit_c.rs` so that generic mono instances
   like `Nova_Vec____NovaValue_SocketAddr*` are not misclassified as stubs.
2. Upgrade DnsNet V1 API from single-address to multi-address:
   `lookup(host, port) -> Result[[]SocketAddr, NetError]`.
3. Add an opt-in real DNS integration test (_slow).

## Root Cause

In `compiler-codegen/src/codegen/emit_c.rs`, function `is_generic_stub_c`:

```rust
fn is_generic_stub_c(&self, s: &str) -> bool {
    if let Some(inner) = s.strip_prefix("Nova_").and_then(|x| x.strip_suffix('*')) {
        let name = inner.trim();
        return !name.is_empty()
            && !self.record_schemas.contains_key(name)
            && !self.sum_schemas.contains_key(name)
            && !self.generic_types.contains(name)
            && !self.opaque_ffi_types.contains(name);
    }
    false
}
```

`"Nova_Vec____NovaValue_SocketAddr*"` → inner = `"Vec____NovaValue_SocketAddr"`
→ not in any registry → `is_generic_stub_c` returns `true` → `Result` arm
falls to erased fallback `"NovaRes_nova_int_nova_str*"`.

Fix: add `&& !name.contains("____")` — monomorphized generic instances
always contain `____` in their mangled name and are ALWAYS concrete.
