## 1. Контекст

V6 / V6.1 emit cache count metrics. After V4.2 chain prefix sharing
landed, raw cache count может уменьшиться даже когда реальный CPU
saving stays the same или improves (prefix sharing emits ONE let
to cover several `@<root>` walks). Count-based gate falls behind.

V6.2 adds a static **cycles-saved estimate** computed from
ExplainReport, fed to Plan 57 `nova bench gate` as a robust
regression metric.

## 2. Изменения

### 2.1 field_cache.rs

New public API:
- `CpuSavingsReport` struct: `estimated_cycles_saved` (aggregate),
  per-layer breakdown, `methods_with_savings`.
- `cpu_savings_estimate(report: &ExplainReport) -> CpuSavingsReport`.

Cycle weights (env-tunable):
- `NOVA_FC_LOAD_CYCLES` (default 4)
- `NOVA_FC_CALL_CYCLES` (default 40)
- `NOVA_FC_LOOP_ITERS` (default 8 — multiplier for LICM hoists)

Per-layer formula:
- ro/mut: `count × LOAD_CYCLES`
- LICM: `count × (LOOP_ITERS × LOAD_CYCLES)`
- pure: `count × CALL_CYCLES`
- chain: `sum_over_chains(chain_depth × LOAD_CYCLES)`

### 2.2 nova-cli/src/main.rs

`cmd_check_telemetry_cache`:
- New aggregation counters for cycles_saved + per-layer cycles.
- Per-file: `cpu_savings_estimate(&report)` calls saturating sum.
- JSON emit: 7 new fields (`cycles_saved_estimate`,
  `cycles_methods_with_savings`, 5 per-layer).
- Human-readable summary: extra "Plan 57 CPU bench integration (V6.2)"
  section.
- Baseline gate: if baseline JSON has `cycles_saved_estimate`,
  compare relative drop > 10% → exit 1.

### 2.3 Tests

2 new unit tests в field_cache::tests:
- `v62_cpu_savings_estimate_aggregates_layers` — sanity на real module.
- `v62_cpu_savings_estimate_empty_report` — empty input → zero.
