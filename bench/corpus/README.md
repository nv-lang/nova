# Bench corpus — frozen reference programs

> Plan 57 §3 L8 — canonical corpus для compiler performance measurement.

Эти файлы — **frozen reference programs** для measuring compile-time
(parse + type-check + mono-pass + codegen + C-compile). Изменения требуют
explicit acknowledgement (commit msg `corpus: update X — reason`) +
update `CHECKSUMS` lockfile.

Не trivially-correlated с runtime benches в `../micro/` — те measure
behaviour generated code, а эти — performance компилятора самого.

## CHECKSUMS

`CHECKSUMS` — sha256 lockfile, generated by:

```sh
cd bench/corpus
for f in $(find . -name "*.nv" | sort); do sha256sum "$f"; done > CHECKSUMS
```

Verify before bench runs (CI hook):

```sh
sha256sum -c CHECKSUMS
```

Корпус **stable contract** — изменения требуют:
1. Update `CHECKSUMS` (run command above).
2. Commit `corpus: update <file> — <reason>` с explicit rationale.
3. (Optional) record old vs new в bench-history orphan branch для
   semver-style impact analysis.

## Usage

```sh
# Single file benchmark — measure compile time:
nova bench corpus 01_hello.nv  # TBD Plan 57.B.1 — wraps nova build с PerfTimer.

# Or manually via PerfTimer flags (NOVA_PERF_TIMER=1):
NOVA_PERF_TIMER=1 nova build bench/corpus/03_generic_heavy.nv -o /tmp/out
# → emits __PERF__ <pass> <ns> на stderr.
```

Per-pass breakdown (parse / type-check / mono-pass / codegen / c-compile /
link) — via PerfTimer hooks (Plan 57.B.1 follow-up).
